Isomorphic Strings

LeetCode 205 — Check if two strings have the same character mapping structure
Easy Hash Tables Bijection Mapping O(n) Time O(n) Space
1 Understanding the Problem ▼

Two strings s and t are isomorphic if we can replace each character in s with another character to get t, with consistent one-to-one mapping. This is a bijection — no two characters in s can map to the same character in t, and vice versa.

Walk through the examples:

s = "egg", t = "add"
e → a ✓, g → d ✓, g → d ✓ (consistent!) → True

s = "foo", t = "bar"
f → b ✓, o → a ✓, o → r ✗ (conflict: o already maps to a!) → False

s = "paper", t = "title"
p → t ✓, a → i ✓, p → t ✓, e → l ✓, r → e ✓ → True
💡 Key Insight

A single map (s→t) is not enough. Consider s="ab" and t="aa": forward map would set a→a, b→a — both "succeed" even though two different chars map to the same target. We need a reverse map (t→s) to catch this.

2 The Algorithm: Two-Map Bijection Check ▼

We maintain two hash maps simultaneously — one mapping s→t, the other mapping t→s. At each index we pair the characters and check consistency in both directions.

1If lengths differ, return False immediately.
2Create two empty maps: s_to_t and t_to_s.
3For each index i, take cs = s[i] and ct = t[i].
4If cs is in s_to_t and its mapped value ≠ ct, return False.
5If ct is in t_to_s and its mapped value ≠ cs, return False.
6Otherwise, record s_to_t[cs] = ct and t_to_s[ct] = cs.
7If we finish without conflict, return True.

⚠️ Why one map fails

s="ab", t="aa": a single forward map sets a→a, b→a. Both "succeed" because we never check if 'a' in t is already claimed by another character. The reverse map catches this: when 'b' tries to map to 'a', t_to_s['a'] already equals 'a' ≠ 'b' → False.

3 Why It Works: The Invariant ▼
🔐 The Invariant

After processing the first i pairs, s_to_t and t_to_s together form a bijection: every mapped char from s maps to exactly one char in t, and every mapped char from t maps back to exactly one char in s. No two keys share a value in either direction.

Why this guarantees correctness

Isomorphism is exactly a bijection between character sets. Our two maps enforce this directly:

  • Injectivity (one-to-one from s): The reverse map t_to_s rejects a char in t that is already mapped to a different char in s.
  • Injectivity (one-to-one from t): The forward map s_to_t rejects a char in s that already maps to a different char in t.
  • Consistency: Both maps must agree throughout — any conflict means not isomorphic.

Edge Cases

  • Different lengths: s="abc", t="ab" → instant False.
  • Empty strings: s="", t="" → trivially True.
  • Same string: s="abc", t="abc" → True (identity mapping).
  • All same char: s="aaa", t="bbb" → True (one mapping a→b).
4 The Code (Annotated) ▼

Solution — Two Hash Maps (Interview Pick)

# Two maps for bijection check
def isIsomorphic(s: str, t: str) -> bool:
if len(s) != len(t):
return False
s_to_t = {}
t_to_s = {}
for cs, ct in zip(s, t):
# Check forward: s char already mapped?
if cs in s_to_t:
if s_to_t[cs] != ct:
return False
# Check reverse: t char already claimed?
elif ct in t_to_s:
return False
else:
# Both new — establish mapping
s_to_t[cs] = ct
t_to_s[ct] = cs
return True

Alternative — Index-Based One-Liner

def isIsomorphic(s: str, t: str) -> bool:
# Map each char to its first occurrence index
return [s.index(c) for c in s] == [t.index(c) for c in t]
💡 Why the index trick works

Two strings are isomorphic if and only if they have the same "pattern" of first-occurrence indices. For "egg" → [0, 1, 1] and "add" → [0, 1, 1]. For "foo" → [0, 1, 1] vs "bar" → [0, 1, 2] — different patterns!

⚠️ Common Mistake

Using only s_to_t without t_to_s. This misses cases like s="ab", t="aa" where two different chars map to the same target.

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Recognize the Pattern

Keywords: "one-to-one mapping", "replace each character", "no two characters may map to same". This signals bijection check.

Step 2: Identify the Failure Modes

A bijection can fail in two ways: (1) a char in s maps to two different chars in t, (2) two chars in s map to the same char in t. Show both cases with examples.

Step 3: Two Maps Solution

Maintain both s→t and t→s maps. At each position, verify consistency in both directions before adding new mappings.

Step 4: Walk Through Example

Use "foo"/"bar" to show failure: f→b ✓, o→a ✓, then o→r conflicts with existing o→a.

🗣️ What to Say to the Interviewer

"This is a bijection problem — I need to verify that each character in s maps to exactly one character in t, and vice versa. A single map isn't enough because it doesn't catch when two different characters try to map to the same target. I'll use two hash maps: one for s→t and one for t→s. At each position, I check both maps for consistency before establishing new mappings. This runs in O(n) time and O(n) space."

7 Summary & Key Takeaways ▼
✓ A bijection requires checking both directions — s→t and t→s.
✓ Two hash maps ensure no character collision in either direction.
✓ Alternative: Compare first-occurrence index patterns.
✓ O(n) time, O(n) space where n = string length.

  • Two sequences must have matching structure
  • Each element maps to exactly one element, and vice versa
