LeetCode #703

Kth Largest Element in a Stream

Design a class that tracks the k-th largest element in a stream of integers. Each add(val) call inserts a value and returns the current k-th largest.

Easy Heaps Min Heap of Size K Time: O(n log k) Space: O(k)
1 Understanding the Problem ▾

You're building a live leaderboard that must always answer: "what is the k-th best score we've ever seen?" New scores keep arriving one by one in a stream. After every new score, you must report the k-th largest score across all scores received so far.

💡 Key Insight

The k-th largest element is the smallest number in the top-k set. If you maintain exactly the k largest values seen so far in a min-heap, the heap's root — the minimum of those k values — is always your answer. It's the element that "just barely" earned a spot in the top k.

Let's make this concrete. With k=3 and stream = [4, 5, 8, 2]:

3rd largest = 4 = min of top-3
heap = [4, 5, 8] heap[0] = 4
→ Answer is always heap[0]

✗ Naive: Sort entire stream

Each add() → sort O(n log n)
100 adds → O(100 × n log n)
Heap: each add → O(log k) only
100 adds → O(100 × log k) ✓

Edge cases to consider: The initial nums list may have fewer than k elements — that is valid, and the heap simply grows toward size k. The problem guarantees that by the time we query for the k-th largest (via add), at least k elements will have been seen in total. Duplicate values are handled naturally by the heap.

2 The Algorithm: Min-Heap of Size K ▾

We maintain a min-heap limited to exactly k elements — those k elements are always the k largest values seen so far. A min-heap surfaces its smallest element at index 0, so heap[0] is always the k-th largest of everything we've processed.

1 Store k, initialize empty heap. Then call self.add(num) for each element in the initial nums list — this reuses the same push+trim logic and avoids code duplication.
2 Push the new value onto the min-heap with heapq.heappush. The heap may now temporarily contain k+1 elements.
3 If heap size exceeds k, pop the minimum element with heapq.heappop. The popped element is smaller than all others in the heap — it cannot be in the top-k, so discarding it is safe.
4 Return heap[0] — the minimum of the top-k values, which is the k-th largest element across the entire stream seen so far.
🔑 Why Min-Heap for k-th LARGEST? (Counterintuitive!)

It feels backwards: we want the k-th largest but we use a min-heap. The trick: we're maintaining the top-k club. To find which candidate should be evicted from the club, we want to know the weakest member — the minimum. A min-heap gives us that in O(1). When a new challenger arrives and beats the weakest member, we evict the weakest and add the challenger.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After every add(val) call, the min-heap contains exactly the k largest values seen so far (or all values seen if fewer than k have arrived). Therefore heap[0] — the minimum of those k values — is the k-th largest across the entire stream.

Why the invariant holds after every push+pop:

  • If the new value is larger than the current k-th largest (heap[0]): it pushes the old k-th largest down to (k+1)th place. We pop the min — the old k-th largest — and the new value earns its top-k spot.
  • If the new value is smaller or equal to heap[0]: it gets pushed in and immediately popped as the new minimum. The top-k set is unchanged.

Either way, after the push+pop, the heap has exactly k elements which are the k largest seen so far.

⚠️ Common Mistake: Forgetting to Pop

If you push but never pop when len(heap) > k, the heap grows to O(n) elements. Then heap[0] becomes the overall minimum of the stream — not the k-th largest. Always pop when size exceeds k.

💡 Why we can pop before checking size

An alternative (equally correct) implementation pops only when size exceeds k, not before. Some solutions write: if len(heap) > k: heappop(heap). Both approaches preserve the invariant. The version in our code is idiomatic Python — push first, then trim.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

The phrase "k-th largest" + "stream" (or "dynamically added") is the clearest signal for a min-heap of size k. Whenever a problem asks for a rank-based order statistic in a dynamic dataset, heap = your first tool.

Step 2: State the Brute Force

"I could keep all values in a list, sort on every add, and return index k-1. That's O(n log n) per add — too slow if there are many adds. I need O(log k)."

Step 3: Arrive at the Heap Insight

"I don't need all n values — just the top k. If I maintain exactly the top-k values in a min-heap, the heap root is always the k-th largest. Each add is O(log k) and space is O(k), independent of total stream size."

Step 4: Handle the Min vs Max Confusion

"I use a min-heap, not a max-heap — this is counterintuitive. The reason: I need to quickly evict the weakest member of the top-k club when a stronger element arrives. The weakest member is the minimum, so a min-heap makes eviction O(log k)."

🗣️ What to Say to the Interviewer

"This is a streaming top-k problem. Key insight: k-th largest = minimum of top-k.
I'll use a min-heap of fixed size k. Each add() does heappush then, if size > k, heappop.
That's O(log k) per add, O(k) space — independent of how many elements we've seen total.
For __init__, I'll feed the initial nums through the same add() to avoid duplicate logic.
The heap root heap[0] is always the answer — O(1) to read."
🔮 Follow-Up Questions to Expect
  • k-th smallest? — Use a max-heap of size k (negate values for Python's min-heap).
  • Find median from data stream? — Two heaps: max-heap (lower half) + min-heap (upper half).
  • k-th largest in a static array? — Use Quickselect: O(n) average, O(n log n) worst.
  • What if k == 1? — Just track the maximum. Heap works but a simple variable suffices.
  • What if the stream is too large for memory? — Heap still works: O(k) memory only!

7 Summary & Key Takeaways ▾
A min-heap of size k always holds the k largest elements seen so far.
heap[0] = minimum of top-k = k-th largest overall. O(1) to read.
Each add() is O(log k) — independent of total stream length n.
Space is O(k) — we never store more than k elements at once.
For k-th smallest: use a max-heap of size k (negate values in Python).
Reusing add() in __init__ is the clean DRY pattern for this class design.
🎯 When to Use This Pattern
  • "k-th largest/smallest" + dynamic/streaming data → min/max-heap of size k
  • Need top-k elements with efficient updates → heap beats sorting
  • Memory constraint: you can't store all n elements → heap stores only k
  • Running order statistic: answer must update with each new element
