Best Time to Buy and Sell Stock II

Deep Dive: State Machine Dynamic Programming

Medium State Machine DP Two States O(n) Time · O(1) Space
1 Understanding the Problem

You're given daily stock prices. You can buy and sell as many times as you want, but you can only hold one share at a time. You must sell before buying again. Goal: maximize profit.

💡 Key Insight

At any moment in time, you're in one of exactly two states:

  • HOLDING — You own a stock (you've bought but not sold)
  • CASH — You don't own any stock (free to buy)

This is what makes it a "state machine" — you're always in one of these two states, and each day you make a decision that might transition you to the other state.

3 Why is This Dynamic Programming?

Dynamic Programming has two key ingredients. This problem has both:

✅ Optimal Substructure

The best solution at day i can be built from the best solution at day i-1. If I know the optimal profit for each state yesterday, I can compute today's optimal.

✅ Overlapping Subproblems

Without DP, we'd recalculate the same scenarios repeatedly. "What's the best profit if I'm holding on day 3?" gets asked many times in a naive recursive approach.

🎯 What we're computing

For each day, we compute:

  • hold = Maximum profit achievable if I END this day holding a stock
  • cash = Maximum profit achievable if I END this day not holding
📐 The Recurrence Relations
hold[i] = max(hold[i-1], cash[i-1] - price[i])
↳ Either keep holding from yesterday, OR buy today (spend price from cash)
cash[i] = max(cash[i-1], hold[i-1] + price[i])
↳ Either stay in cash from yesterday, OR sell today (add price to hold value)
4 The Code (Annotated)
# State Machine DP for unlimited transactions
def maxProfit(prices):
# Initialize states for day 0
hold = -prices[0] # If holding on day 0, we bought at prices[0]
cash = 0 # If not holding on day 0, profit is 0
# Process each remaining day
for price in prices[1:]:
# Update hold: keep holding OR buy today (from cash)
hold = max(hold, cash - price)
# Update cash: stay cash OR sell today (from hold)
cash = max(cash, hold + price)
# Answer is cash state (we want to end without holding)
return cash
⚠️ Why return cash, not hold?

At the end, we want maximum profit with no stock in hand. The cash state represents "best profit when not holding" — exactly what we need. Holding stock at the end means unrealized value.

6 Summary & Key Takeaways
🎓 Why State Machine DP Works

Instead of thinking about "which days to buy/sell", we think about "what's the best I can do in each state". The DP automatically figures out the optimal sequence of transactions by always picking the better option at each step.

Time: O(n)
One pass through prices
Space: O(1)
Only 2 variables needed
Result: $7
Buy@1→Sell@5, Buy@3→Sell@6
💡 When to Use State Machine DP

This pattern extends beautifully to harder stock problems:

  • With cooldown — Add a third state: "resting" (can't buy)
  • With fee — Subtract fee when selling
  • At most K transactions — Track state per transaction count
