LeetCode #2444

Count Subarrays With Fixed Bounds

Count contiguous subarrays where the minimum value equals minK and the maximum value equals maxK.

Hard Two Pointers Three Pointers Tracking Time: O(n) Space: O(1)
1 Understanding the Problem ▾

You have an integer array nums and two integers minK and maxK. A fixed-bound subarray is a contiguous subarray that satisfies both:

  • The minimum value in the subarray is exactly minK
  • The maximum value in the subarray is exactly maxK

Return the count of all such subarrays. Note that minK ≤ maxK is guaranteed.

💡 Key Insight

Any element outside the range [minK, maxK] acts as a hard wall — it violates the min or max constraint for every subarray containing it. This means we only need to search within segments of the array that are entirely in-range. Within each segment, a subarray is valid exactly when it contains at least one occurrence of minK and at least one of maxK.

✓ Valid: nums=[1,3,5,2,7,5], minK=1, maxK=7

[1,3,5,2,7] → min=1✓, max=7✓
[1,3,5,2,7,5] → min=1✓, max=7✓
All elements in [1,7], both bounds present.

✗ Invalid: element out of bounds

nums=[1,3,8,5,7], minK=1, maxK=7
8 > maxK=7 → wall at index 2.
No subarray crossing index 2 can be valid.

Edge Cases: minK == maxK (every in-range element must equal that value), arrays where minK or maxK never appears (answer = 0), and out-of-range elements at the very start or end of the array.

2 The Algorithm: Three-Pointer Single Pass ▾

We perform a single left-to-right sweep of the array, maintaining three integer "bookmarks". For each right endpoint r, we compute in O(1) how many valid left endpoints exist — and accumulate that count.

bad_pos Index of the most recent element outside [minK, maxK]. Any subarray that includes this position cannot be valid — its min or max would be out of bounds. Subarrays ending at r must start after bad_pos.
min_pos Index of the most recent element equal to minK. The subarray must include this position or one further left — but if the left boundary is greater than min_pos, no minK remains in the window.
max_pos Index of the most recent element equal to maxK. Same reasoning as min_pos — the subarray left boundary must be at most max_pos for the window to contain a maxK.
1 Initialize: Set bad_pos = min_pos = max_pos = -1. Initialize ans = 0. All three bookmarks start "before the array" — nothing has been seen yet.
2 Sweep right: For each index r, examine nums[r]. If it is outside [minK, maxK], update bad_pos = r. If it equals minK, update min_pos = r. If it equals maxK, update max_pos = r. These checks are independent (not else-if).
3 Count valid left endpoints: For the current right end r, a left endpoint l is valid if and only if: l > bad_pos (no wall in window) AND l ≤ min_pos AND l ≤ max_pos. Combining the upper bounds: l ≤ min(min_pos, max_pos). The count is max(0, min(min_pos, max_pos) − bad_pos).
4 Accumulate: Add that count to ans. After the loop, return ans. Total time: O(n), space: O(1).
added at step r = max(0, min(min_pos, max_pos) − bad_pos)
🔑 Why This Formula Counts Correctly

Valid left endpoints l satisfy: bad_pos < l ≤ min(min_pos, max_pos). The number of integers in this half-open interval is min(min_pos, max_pos) − bad_pos. We use max(0, …) because if min_pos or max_pos is still −1 (not seen yet), or is ≤ bad_pos (walled off), the result is negative — meaning 0 valid subarrays.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing index r, the variable ans contains the count of all valid fixed-bound subarrays whose right endpoint is ≤ r. At each step we add exactly the number of new valid subarrays ending at the current r, with no double-counting.

Why bad_pos creates a hard boundary: If element at index b is outside [minK, maxK], then any subarray containing index b has either min < minK or max > maxK (or both). No adjustment of the right endpoint can fix this — the element is permanently poisonous. So the earliest valid left start for any subarray ending at r ≥ b is b + 1 = bad_pos + 1.

Why both min_pos and max_pos constrain the left end: To guarantee min = minK, the window must contain at least one element equal to minK. The closest such element to the right is at min_pos. If we set l > min_pos, no minK remains in the range [l, r]. Similarly for maxK. So l must be at most the smaller of the two: l ≤ min(min_pos, max_pos).

No overcounting: Every valid subarray [l, r] is counted exactly once — at the step where r equals its right endpoint. For that specific r, the left endpoint l satisfies the valid range, so it contributes exactly 1 to the count.

⚠️ Off-by-One: bad_pos starts at -1

When no bad element has been seen, bad_pos = -1. The formula min(min_pos, max_pos) − (−1) = min(min_pos, max_pos) + 1 gives the count of valid left endpoints from index 0 to min(min_pos, max_pos) — which is correct (indices 0, 1, …, min(min_pos, max_pos), that's min(min_pos, max_pos) + 1 values). Do NOT subtract an extra 1.

🚨 What if minK == maxK?

When minK == maxK, an element equal to that value updates both min_pos and max_pos to the same index. min(min_pos, max_pos) equals that index, and the formula works correctly. The checks if x == minK and if x == maxK are NOT else-if — both fire on the same element.

4 The Code (Annotated) ▾

Approach 1 — Brute Force (O(n²), for intuition)

def countSubarrays_brute(nums, minK, maxK):
n, ans = len(nums), 0
for l in range(n): ← try every left endpoint
lo, hi = nums[l], nums[l] ← running min/max for this left start
for r in range(l, n): ← extend right endpoint
lo = min(lo, nums[r])
hi = max(hi, nums[r])
if hi > maxK: break ← max exceeded, extending further won't help
if lo == minK and hi == maxK: ans += 1
return ans ← O(n²) — TLE for n=10^5

Approach 2 — Three Pointers Optimal (O(n), O(1)) ⭐ Interview Pick

def countSubarrays(nums, minK, maxK):
ans = 0 ← total count of valid subarrays
bad_pos = min_pos = max_pos = -1 ← all bookmarks start "before array"
for r, x in enumerate(nums): ← single left-to-right sweep
if x < minK or x > maxK: ← out-of-range element: hard wall
bad_pos = r ← update most-recent wall position
if x == minK: ← track most-recent minK (NOT elif!)
min_pos = r
if x == maxK: ← track most-recent maxK
max_pos = r
ans += max(0, min(min_pos, max_pos) - bad_pos)← count valid left endpoints for this r
return ans
⚠️ Do NOT use elif between the three checks

Using elif x == minK or elif x == maxK would miss the case where x is both out-of-range and equal to minK (impossible, but structurally: the checks are conceptually independent). More critically, when minK == maxK, the same value must update both min_pos and max_pos. Only plain if handles this correctly.

💡 Complexity Summary

Brute Force: Time O(n²), Space O(1)
Three Pointers (optimal): Time O(n), Space O(1) — one loop, three integer bookmarks

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: State the Brute Force

Fix every left endpoint l, extend r right while tracking running min and max. When max exceeds maxK, break (extending further can't help). Count whenever min==minK and max==maxK. This is O(n²) — correct but too slow for n=10⁵.

Step 2: Spot the Wall Structure

Key observation: an element outside [minK, maxK] poisons every subarray crossing it. So valid subarrays live entirely within "clean segments" between walls. This suggests we can reset our search at each wall, which motivates a single-pass approach.

Step 3: Count From the Right End

Instead of enumerating subarrays, ask: "for this right end r, how many left ends l are valid?" A left end is valid if (a) it's after the last wall, and (b) both the most-recent minK and most-recent maxK are still inside the window. This gives the O(1) per-step formula.

Step 4: Derive the Formula

Valid l must satisfy bad_pos < l ≤ min(min_pos, max_pos). Count of integers in this range = min(min_pos, max_pos) − bad_pos (take max with 0 to avoid negatives when bounds not yet seen).

🗣️ What to Say to the Interviewer

"I'll observe that elements outside [minK, maxK] act as walls — no valid subarray crosses them.
For each right endpoint r, I need to count how many left endpoints l give a valid subarray.
l must be: (1) strictly after the most recent wall, and (2) at or before both the most recent
minK position and the most recent maxK position. That's max(0, min(min_pos, max_pos) - bad_pos).
I track these three positions in one pass — O(n) time, O(1) space."
🔮 Likely Follow-up Questions
  • "What if minK > maxK?" → Return 0 immediately; impossible to satisfy both constraints.
  • "What if all elements are equal to minK=maxK?" → Every subarray is valid — answer is n*(n+1)/2.
  • "Can you also output the subarrays?" → Yes, but output size is O(n²) worst case; time becomes O(n²).
  • "Variant: count subarrays with min in [a, b] only?" → Simpler: just need bad_pos and min_pos.
