LeetCode #1456

Maximum Number of Vowels in a Substring of Given Length

Given a string s and an integer k, return the maximum number of vowel letters in any substring of s with length k.

Medium Sliding Window Fixed Window Time: O(n) Space: O(1)
1 Understanding the Problem ▾

We are given a string s and an integer k. We need to find the maximum number of vowel characters (a, e, i, o, u) in any contiguous substring of length exactly k.

💡 Key Insight

We don't need to examine every possible substring independently. When we slide a window of size k one position to the right, only one character leaves (the leftmost) and one character enters (the new rightmost). We just adjust the count — no need to re-scan the entire window.

✗ Brute Force: Re-scan Every Window

s = "abciiidef", k = 3
"abc" → count vowels: 1
"bci" → count vowels: 1
"cii" → count vowels: 2
"iii" → count vowels: 3 ← max
...O(n·k) work total

✓ Sliding Window: O(1) per Step

"abc" → vowels = 1 (a)
slide: 'a' out(-1), 'i' in(+1) → 1
slide: 'b' out(0), 'i' in(+1) → 2
slide: 'c' out(0), 'i' in(+1) → 3 ← max
O(n) total

Vowels are: a, e, i, o, u — store them in a set for O(1) membership lookup. Edge cases: k equals len(s) means only one window; all-vowel input gives answer k; no vowels gives answer 0.

2 The Algorithm: Fixed-Size Sliding Window ▾

The fixed-size sliding window technique is ideal here because the window size never changes — it is always exactly k. We build the first window from scratch in O(k), then slide it across the string one step at a time, each step in O(1).

1 Initialize: Count vowels in the first window s[0..k-1]. Store this as count (current window vowels) and max_count (global best so far).
2 Slide the window: For each index i from k to len(s)-1: the new character entering is s[i], and the character leaving is s[i−k].
3 Update count: count += (s[i] in VOWELS) — adds 1 if entering char is a vowel. Then count -= (s[i-k] in VOWELS) — subtracts 1 if leaving char was a vowel. Booleans act as integers in Python.
4 Track maximum & early exit: max_count = max(max_count, count). If max_count == k, every character in this window is a vowel — the answer cannot be higher, so return immediately.
5 Return: After all windows are processed, return max_count.
🔑 Fixed vs Variable Sliding Window

A variable window expands and shrinks based on a condition (e.g., "at most k distinct characters"), requiring a nested while-loop to shrink. A fixed window like this problem — where size is always exactly k — never needs to shrink; we only slide. This is simpler: no inner loop, just constant-time add-one/remove-one per step.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the start of every loop iteration (when right index is i), count always equals the exact number of vowels in the current window s[i-k..i-1]. This is established before the loop and maintained by each add/remove update.

Why the invariant holds by induction:

  • Base case: After the sum over s[:k], count is exactly the vowels in s[0..k-1]. Trivially correct.
  • Inductive step: Sliding from window [i-k, i-1] to [i-k+1, i] changes the set by removing s[i-k] and adding s[i]. We subtract 1 if the removed character was a vowel, add 1 if the new one is. This is exactly the delta — count stays correct.

Completeness: We examine every possible starting position from 0 to n-k, covering all valid k-length substrings. Because we track the running maximum, max_count is guaranteed globally optimal at the end.

⚠️ Off-by-One: Which Character Leaves?

When the entering index is i, the character that just left is at s[i - k], not s[i - k + 1]. To verify: the new window is [i-k+1 .. i], so the old window's leftmost was i-k. One off-by-one here silently corrupts the count every single step — test with a known example first.

4 The Code (Annotated) ▾

Solution 1 — Sliding Window (Optimal, O(n) — Interview Pick)

def maxVowels(s: str, k: int) -> int:
VOWELS = set('aeiou') ← O(1) membership; avoids O(5) string scan on every check
count = sum(c in VOWELS ← Count vowels in the first window s[0..k-1]
for c in s[:k]) ← Generator expression — 1 per vowel, 0 per consonant
max_count = count ← Best so far = first window
for i in range(k, len(s)): ← i is the index of the character entering the window
count += (s[i] in VOWELS) ← New char enters: True=+1 if vowel, False=+0 if consonant
count -= (s[i-k] in VOWELS) ← Old char exits: True=-1 if was vowel, False=-0 if consonant
max_count = max(max_count, count)← Update global best after each slide
if max_count == k: return k ← Early exit: all-vowel window — can't do better
return max_count
⚠️ Python Bool-as-Int Trick

count += (s[i] in VOWELS) works because Python booleans are integers: True == 1, False == 0. This is idiomatic and concise. In a Java/C++ interview you would write explicit if-statements instead. In Python, this is perfectly acceptable and actually preferred.

Solution 2 — Brute Force (for contrast only, O(n·k))

def maxVowels_brute(s: str, k: int) -> int:
VOWELS = set('aeiou')
best = 0
for i in range(len(s) - k + 1): ← O(n) windows
cnt = sum(c in VOWELS for c in s[i:i+k]) ← O(k) per window — this is the costly re-scan
best = max(best, cnt)
return best ← Total: O(n·k) — too slow for large inputs
💡 Complexity Summary

Sliding Window: Time O(n) — one initialization pass + n-k slides each O(1). Space O(1) — only a few integers (the 5-char vowel set is a constant).
Brute Force: Time O(n·k) — up to O(n²) when k ≈ n. For n = 10⁵, k = 10⁴: brute = 10⁹ ops vs sliding = 10⁵ ops.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Pattern Signal

"The problem asks for a property of a substring of fixed length k." That phrase — fixed length window — is the textbook signal for fixed-size sliding window. Commit this pattern: fixed-k window + aggregate (sum/count/max) = sliding window.

Step 2: State Brute Force First

"Naively, I'd iterate every starting index and count vowels in each k-length window — O(n·k). With n = 10⁵, k = 10⁴ that's 10⁹ operations. Can we reduce the per-step work?"

Step 3: Identify the Reuse

"When I slide right by 1, the next window shares k-1 characters with the previous window. I already counted those. Only one char left and one entered — O(1) adjustment. No need to re-scan."

Step 4: Code and Verify Off-by-One

"The leaving character is at s[i - k], not s[i - k + 1]. I'll verify with example: i = 3, k = 3 → leaving = s[0]. Window is now s[1..3]. Correct." Always trace one step mentally before submitting.

🗣️ What to Say to the Interviewer

"Fixed-length substring aggregate — I'll use a fixed sliding window.
Initialize: count vowels in s[0..k-1]. That's O(k).
Then slide n-k times: each step, add 1 if the entering char is a vowel,
subtract 1 if the leaving char was a vowel. Track the running max.
Early exit if count hits k — can't do better. Time O(n), space O(1)."
🔮 Likely Follow-up Questions
  • "Maximum sum subarray of size k?" → Same pattern, use sum instead of vowel count (#643)
  • "What if the window size can vary?" → Variable sliding window with shrink condition (#3, #76)
  • "Count windows with at least k vowels?" → Count all windows where count >= k
  • "What if k > len(s)?" → Return 0 (or handle via guard at the start)

7 Summary & Key Takeaways ▾
Fixed-length window: initialize O(k), then slide O(1) per step — total O(n)
Add entering character's contribution, subtract leaving character's — both O(1)
Use a set for vowel lookup — O(1) per check vs O(5) for string search
Early exit when count == k — provably optimal, saves remaining iterations
Pattern signal: "contiguous subarray/substring of fixed length k" + aggregate
ApproachTimeSpaceNotes
Brute Force O(n·k) O(k) Re-scans; TLE for large n,k
Sliding Window ✓ O(n) O(1) Optimal; standard interview answer
🎯 When to Use Fixed-Size Sliding Window
  • Problem says "subarray/substring of length exactly k"
  • You need to maximize/minimize/count over all k-length windows
  • The aggregate can be updated incrementally (add one element, remove one)
  • Contrast: if window size varies based on a condition → variable sliding window
