LeetCode #144

Binary Tree Preorder Traversal

Given the root of a binary tree, return the preorder traversal of its nodes' values (Root → Left → Right).

Easy Trees DFS: Preorder Stack Time: O(n) Space: O(h)
1 Understanding the Problem ▾

A tree traversal visits every node exactly once and collects values in a specific order. Preorder follows the rule: visit the current node first, then the left subtree, then the right subtree. The mnemonic is Root → Left → Right.

Think of reading a book's table of contents: you note the chapter title first (root), then all left-side sub-chapters, then right-side sub-chapters — depth-first, always noting where you are before going deeper.

💡 Key Insight

In preorder traversal, a node is added to the output the moment you first encounter it — before descending into any children. The word "pre" means "before": process self, THEN process children.

Preorder (#144)
Root → L → R
Output NOW, recurse later
Inorder (#94)
L → Root → R
Recurse left, output, recurse right
Postorder (#145)
L → R → Root
Recurse both, then output

Concrete example: For the tree shown below, preorder gives [1, 2, 4, 5, 3].

1 ← 1st visited (root)
/ \
2 3 ← 2nd visited (left child), 5th visited (right child)
/ \
4 5 ← 3rd visited (left-left), 4th visited (left-right)

Edge cases to consider: empty tree (return []), single node (return [root.val]), right-skewed tree (deep stack), left-skewed tree (like a linked list — biggest stack-depth concern), and a complete balanced binary tree.

2 The Algorithm: Iterative Stack Simulation ▾

There are two natural approaches. The recursive solution maps directly to the definition — but it uses the hidden call stack, which can overflow on deeply skewed trees (10,000+ nodes). The iterative stack approach uses an explicit stack we control, making it safer and more production-ready.

1 Initialize: Create a stack containing just the root node. Create an empty result list. If root is None, return [] immediately.
2 Loop while stack is non-empty: Pop the top node. This is the current node to process.
3 Output immediately (PRE-order): Append the popped node's value to result. This happens BEFORE pushing children — that is what makes it pre-order.
4 Push children in reverse order: Push the RIGHT child first, then the LEFT child. Since the stack is LIFO, the left child (pushed last) comes off first — ensuring left-before-right processing.
5 Repeat until empty: Return the result list.
✗ Concern: Naive Recursive

def preorder(root):
  if not root: return []
  return ([root.val]
    + preorder(root.left)
    + preorder(root.right))

⚠ O(n) call-stack depth on skewed trees. Also builds O(n) intermediate lists on each + concatenation.

✓ Optimal: Iterative Stack

stack = [root]
result = []
while stack:
  node = stack.pop()
  result.append(node.val)
  if node.right: stack.append(right)
  if node.left: stack.append(left)
✓ Full control over stack depth; safe for deep trees

🔑 Why Push Right Before Left?

Stack is LIFO: the last thing pushed is the first thing popped. We need the LEFT subtree processed before the RIGHT subtree. So: push RIGHT first (it waits at bottom), then push LEFT (it sits on top and is popped next). This is the single most important implementation detail to memorize for iterative tree traversal.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point during execution, the stack contains exactly the nodes whose subtrees have not yet been fully processed, ordered so that the next node to process is on top. Every node is pushed onto the stack exactly once and popped exactly once — giving O(n) total operations.

Why right-before-left guarantees correct preorder? When we pop node N, we output it (pre-order step). We then push N.right, then N.left. The next pop retrieves N.left — which outputs N.left.val, then pushes N.left's children. This ensures N.left's entire subtree is drained before N.right is ever touched. This exactly mirrors how recursive preorder works: process self → drain left subtree → drain right subtree.

Proof sketch: For a single-node tree: push root, pop root, output root.val, no children to push, stack empty, done — correct. For a node with children: by induction, the left subtree produces preorder output for its nodes in order, and only after the left subtree is exhausted does the right subtree begin. Combined: root output + left-subtree preorder + right-subtree preorder = preorder of the whole tree. QED.

⚠️ Critical Bug: Pushing None

Always guard with if node.right and if node.left before pushing. If you push None onto the stack, the next iteration calls None.val and crashes with AttributeError. This is the #1 bug in iterative tree traversal.

Space complexity: The stack holds at most O(h) nodes simultaneously, where h is the tree height. For a balanced tree h = O(log n). For a right-skewed tree (every node has only a right child), h = n and space is O(n). In practice, LeetCode trees are limited to 100 or 10,000 nodes so this distinction matters.

node = stack.pop() ← Pop the top node (LIFO — last pushed = first processed)
result.append(node.val) ← PRE-order: append value BEFORE pushing children
if node.right: ← Push right FIRST so it waits below left

stack.append(node.left)
return result ← All n nodes visited exactly once → O(n) time
⚠️ Most Common Mistake: Pushing Left Before Right

Pushing left then right flips the order — you get Root → Right → Left instead of Root → Left → Right. Memory trick: Right goes to bed first (pushed first, sleeps at the bottom), Left wakes up first (pushed second, pops first). The "wrong" push order produces a valid traversal called "reverse preorder" — useful for computing postorder iteratively!

💡 Complexity Summary

Iterative Stack: Time O(n) — visit each node once. Space O(h) — stack holds at most h nodes. No hidden call-stack usage.
Recursive: Time O(n). Space O(h) — uses Python's call stack. Risk of RecursionError on deep trees.
Morris Traversal: Time O(n). Space O(1) — no stack at all; threads right-child pointers temporarily. Complex to implement; rarely required in interviews.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: State the Traversal Rule Out Loud

"Preorder is Root → Left → Right. At any node: record its value first, then visit the entire left subtree, then the entire right subtree." Saying this aloud shows pattern fluency and gives you a blueprint to code from.

Step 2: Offer the Recursive Solution First

"The recursive solution maps directly to the definition: append root.val, call preorder(left), call preorder(right). O(n) time, O(h) call-stack space. Clear and correct." Code this first if time is short.

Step 3: Offer the Iterative Version Proactively

"To avoid call-stack overflow on skewed trees, I'll simulate the call stack explicitly with a data structure. The key insight: stack is LIFO, so push right before left to ensure left is processed first." This shows depth beyond the trivial solution.

Step 4: Mention Morris Traversal as Bonus

"For O(1) space, Morris Traversal threads right-child pointers temporarily. I wouldn't implement it under time pressure, but it shows I know the space can be reduced further."

🗣️ What to Say to the Interviewer

"Preorder is Root → Left → Right — output a node before its children.
Recursive: base case null node, append val, recurse left, recurse right. O(n)/O(h).
Iterative (safer): use an explicit stack, start with root.
Pop a node, append its value, push right child then left child.
Stack is LIFO so left (pushed second) gets processed first — correct order.
Guard each push with a null check to avoid AttributeError.
Time O(n), Space O(h). Done when stack is empty."

7 Summary & Key Takeaways ▾
Preorder = Root → Left → Right: output a node the instant you first encounter it
Iterative stack: push RIGHT child first, then LEFT child (LIFO guarantees left-first processing)
Always null-check before pushing children — never push None onto the stack
Space is O(h): O(log n) for balanced trees, O(n) for skewed trees
Preorder serializes a tree structure — from preorder + inorder you can reconstruct the tree
