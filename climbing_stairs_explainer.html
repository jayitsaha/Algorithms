LeetCode #70

Climbing Stairs

You are climbing a staircase with n steps. Each time you can climb 1 or 2 steps. Return the number of distinct ways you can climb to the top.

Easy Dynamic Programming Same as Fibonacci Time: O(n) Space: O(1)
1 Understanding the Problem ▾

You stand at the ground (step 0) and want to reach the top (step n). At each point you may move up exactly 1 step or 2 steps. How many distinct sequences of moves lead you to the top?

💡 Key Insight

The only way to arrive at step n is from step n−1 (one step away) or step n−2 (two steps away). So the total number of ways to reach step n is exactly ways(n−1) + ways(n−2). This is the Fibonacci recurrence in disguise — dp[n] = dp[n-1] + dp[n-2].

Pattern: 5 = ways(4) = ways(3) + ways(2) = 3 + 2. The Fibonacci numbers are 1, 1, 2, 3, 5, 8, 13 … and climbStairs(n) = Fibonacci(n+1).

⚠️ Edge Cases

n = 1: exactly 1 way (one single step). n = 2: exactly 2 ways (take two singles, or one double). These form our base cases — we must hard-code them since the recurrence needs at least two prior values.

2 The Algorithm: Bottom-Up DP (Fibonacci Rolling Window) ▾

Why DP applies here — both pillars are present:

1 Optimal Substructure: The number of ways to reach step n depends entirely on the answers for step n−1 and step n−2. You don't need to re-enumerate every path — just their counts.
2 Overlapping Subproblems: A naive recursion computes ways(n−1) and ways(n−2), each of which re-computes ways(n−2) and ways(n−3), and so on. The call tree explodes exponentially (O(2ⁿ)). Memoization or tabulation computes each state once.
📐 The Recurrence Relation
dp[i] = dp[i-1] + dp[i-2]
State: dp[i] = number of distinct ways to reach step i
Base: dp[1] = 1, dp[2] = 2

Space optimization: Since dp[i] only depends on the two immediately preceding values, we can replace the O(n) array with just two variables — a rolling window.

1 Handle base cases: if n ≤ 2, return n immediately.
2 Initialize rolling window: prev = 1 (dp[1]), curr = 2 (dp[2]).
3 Roll forward: for i from 3 to n: prev, curr = curr, prev + curr. Each iteration computes dp[i] and shifts the window one step forward.
4 Return curr: after the loop, curr holds dp[n].

3 Why It Works: The Invariant & Proof of Correctness ▾
🔐 The Invariant

At the start of every loop iteration for step i, prev holds the exact count of ways to reach step i−2 and curr holds the exact count of ways to reach step i−1. After the update, curr holds the exact count for step i, and the invariant rolls forward.

Proof by induction:

B Base cases: dp[1] = 1 — only one way to climb 1 stair (a single step). dp[2] = 2 — two ways to climb 2 stairs (1+1, or 2). Both directly enumerable and trivially correct.
I Inductive step: Assume dp[k−1] and dp[k−2] are correct. Every path to step k ends with either a 1-step move from k−1, or a 2-step move from k−2. These two groups are mutually exclusive and exhaustive. By the sum rule of counting: dp[k] = dp[k−1] + dp[k−2]. ∎
⚠️ Why Not Pure Recursion?

A naive recursive solution recomputes the same sub-problems exponentially. f(n) calls f(n−1) and f(n−2); each calls f(n−2) and f(n−3) again. The call tree has O(2ⁿ) nodes — catastrophic for n=50 but fine for n=5. Memoization/tabulation makes every state O(1) work, bringing total to O(n).

💡 Analogy: Counting Stair Sequences

Imagine every route is a code made of 1s and 2s summing to n. The number of such codes is the Fibonacci sequence. Each new entry just asks: "how many codes ended one short, and how many ended two short?" Add them together. That's all DP is doing here.

4 The Code (Annotated) — Tabulation & Memoization ▾

Approach A — Bottom-Up Tabulation, O(1) Space (Interview Pick ✓)

def climbStairs(n: int) -> int:
if n <= 2: ← Handle base cases: dp[1]=1, dp[2]=2 → return n
return n
prev, curr = 1, 2 ← prev = dp[i-2], curr = dp[i-1]; start at step 2
for i in range(3, n + 1): ← Fill dp[3] through dp[n] one step at a time
prev, curr = curr, prev + curr ← Roll window: new curr = dp[i]; old curr → prev
return curr ← curr now holds dp[n], the answer
⚠️ The Simultaneous Assignment Trick

prev, curr = curr, prev + curr — Python evaluates the right-hand side before any assignment occurs. So prev + curr uses the old values of both. Without this, you'd need: tmp = prev + curr; prev = curr; curr = tmp. The one-liner is idiomatic and avoids the temporary variable.

Approach B — Top-Down Memoization, O(n) Space

def climbStairs(n: int) -> int:
memo = {} ← Cache: maps step i → number of ways to reach step i
def dp(i):
if i <= 2: return i ← Base cases: same as tabulation
if i in memo: return memo[i]← Cache hit: return stored result, avoid recomputing
memo[i] = dp(i-1) + dp(i-2) ← Recurrence: store result before returning
return memo[i]
return dp(n) ← Kick off recursion from target step

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the DP Signal

The problem asks "how many ways" (counting), involves a choice at each step (1 or 2), and the answer for n depends on smaller sub-problems. These three signals together — counting paths, local choices, sub-problem dependency — are classic DP cues.

Step 2: Define the State Precisely

Say: "Let dp[i] = the number of distinct ways to reach exactly step i." This state captures everything needed — no extra context required, since every path to step i is fully counted by the paths to i−1 and i−2.

Step 3: Write the Recurrence Out Loud

"To reach step i, I either came from step i−1 (one step) or step i−2 (two steps). These are mutually exclusive and exhaustive. Therefore dp[i] = dp[i−1] + dp[i−2]. Base cases: dp[1]=1, dp[2]=2."

Step 4: Propose the Space Optimization

"I only need the last two dp values at any step, so instead of an O(n) array, I'll use two variables — prev and curr — and roll them forward. That gives O(1) space with the same O(n) time."

🗣️ What to Say to the Interviewer

"This is a DP problem. At each step I have two choices: take 1 or take 2.
The number of ways to reach step n equals ways(n-1) + ways(n-2). That's Fibonacci.
Base cases: dp[1]=1, dp[2]=2.
I'll implement bottom-up with two rolling variables — O(n) time, O(1) space.
I could also do top-down with memoization if you want to see the recursive form."
🔮 Likely Follow-up Questions
  • "What if you could take 1, 2, or 3 steps?" → dp[i] = dp[i-1] + dp[i-2] + dp[i-3]; use 3-variable window
  • "What if some steps are broken?" → Skip those steps; set dp[broken] = 0 in the loop
  • "Can you minimize cost instead of count?" → Min Cost Climbing Stairs (#746) — same structure, min() instead of sum
  • "O(log n) solution?" → Matrix exponentiation: [[1,1],[1,0]]ⁿ gives Fibonacci in O(log n)

7 Summary & Key Takeaways ▾
climbStairs(n) equals Fibonacci(n+1) — the recurrence dp[i] = dp[i-1] + dp[i-2] is the Fibonacci relation
State definition: dp[i] = number of distinct ways to reach step i; only the step index matters
Base cases dp[1]=1, dp[2]=2 are directly enumerable without any recursion
Two-variable rolling window reduces space from O(n) to O(1) — always mention this optimization
Memoization (top-down) is equally correct but uses O(n) stack space; both forms are worth knowing

🎯 When to Recognize This Pattern (Fibonacci-style Linear DP)
  • "How many distinct ways to reach X?" with a fixed, small set of step sizes
  • Current state depends only on the last 1–2 (or k) states — rolling window compresses to O(k) space
  • No global weight or ordering constraint — just counting sequences
  • Keywords: "distinct ways", "reach top", "cover floor", "decode string"
