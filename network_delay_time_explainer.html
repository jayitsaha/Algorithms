LeetCode #743

Network Delay Time

Given a weighted directed graph, find the minimum time for a signal from node k to reach all nodes. Return -1 if unreachable.

Medium Graph Dijkstra's Algorithm Time: O((V+E) log V) Space: O(V+E)
1 Understanding the Problem ▾

Imagine a network of n servers numbered 1 to n. We send a signal from server k. Each directed link (u, v, w) takes w time units to travel from u to v. The signal propagates along all paths simultaneously. The question is: when does the last server receive the signal?

💡 Key Insight

This is Single-Source Shortest Path (SSSP). The signal arrives at each node at its shortest-path time from k. The network is fully synchronized when the last (farthest) node gets the signal. So the answer is max(shortest_path_distance for all nodes). If any node has no path from k, return -1.

Example: times=[[2,1,1],[2,3,1],[3,4,1]], n=4, k=2

dist[2]=0  (source)
dist[1]=1  (2→1, cost 1)
dist[3]=1  (2→3, cost 1)
dist[4]=2  (2→3→4, cost 2)
Answer = max(0,1,1,2) = 2

Edge Cases to Handle

n=1, k=1 → return 0
Disconnected node → -1
Parallel edges → keep min
Self-loops → safe to ignore

Why not BFS? BFS finds shortest paths in unweighted graphs (each edge costs 1). Here edges have different weights, so BFS gives wrong answers. We need Dijkstra's algorithm, which handles non-negative weights efficiently with a min-heap.

2 The Algorithm: Dijkstra's Shortest Path ▾

Dijkstra's algorithm is a greedy explorer that always commits to the cheapest unvisited node next. Because edge weights are non-negative, once we "settle" a node (commit to its distance), no future path can improve it.

1 Build adjacency list: Convert times into graph[u] = [(v, w), ...] for fast neighbor lookup. Directed edges only.
2 Initialize: dist[k] = 0, all others = ∞. Push (0, k) onto a min-heap. The heap stores (tentative_distance, node).
3 Extract minimum: Pop the node u with the smallest tentative distance from the heap. If it's stale (heap contains old entry for u), skip it.
4 Relax edges: For each neighbor (v, w) of u: if dist[u]+w < dist[v], update dist[v] and push (dist[v], v) to the heap.
5 Repeat until heap is empty. Every settled node has its optimal distance locked in.
6 Answer: ans = max(dist.values()). Return -1 if any node is still ∞ (unreachable).
🔬 Dijkstra Algorithm Deep-Dive

Origin: Edsger W. Dijkstra, 1956. Solves SSSP on graphs with non-negative edge weights in O((V+E) log V).

Core Invariant: When node u is extracted from the min-heap, dist[u] is already optimal. No future path through unprocessed nodes can improve it (all remaining edges add ≥ 0 weight).

When to recognize: Weighted directed/undirected graph + shortest path + non-negative weights. Keywords: "minimum time", "minimum cost", "minimum delay".

3 Why It Works: The Invariant ▾
🔐 The Greedy Invariant

The moment we pop (d, u) from the min-heap, d is the shortest possible distance to u. Proof: any alternative path to u must pass through some unvisited node x currently in the heap. But x's heap distance ≥ d (we always pop the minimum). Since all future edges add ≥ 0 weight, that alternative path costs at least d. Therefore d is optimal.

Why Dijkstra breaks with negative weights: If edges could be negative, a longer initial path might become shorter after traversing a negative edge — violating the greedy invariant. That's why Bellman-Ford is needed for negative weights (it relaxes all edges V-1 times, catching all improvements).

⚠️ The Stale Entry Problem (Lazy Deletion)

When we update dist[v] with a better distance, we push a new entry (new_dist, v) to the heap — but the OLD entry (old_dist, v) is still in there. When the old entry is popped, we check: d > dist[u]. If true, it's stale (we've already found and processed a better path). We continue to skip it. This "lazy deletion" avoids complex decrease-key operations and is O(E log V) overall.

Why max(dist)? The broadcast signal travels all paths simultaneously. Each node receives it at time equal to its shortest-path distance. The total time is determined by the slowest node — the one with the maximum shortest-path distance. This is why we return max(dist.values()) — not the sum, not the average.

4 The Code (Annotated) ▾

Solution 1 — Dijkstra with Min-Heap (Optimal, Interview Pick)

import heapq
def networkDelayTime(times, n, k):
graph = {} ← adjacency list: u → [(v, weight)]
for u, v, w in times: ← directed edges only
graph.setdefault(u, []).append((v, w))
dist = {i: float('inf') for i in range(1,n+1)} ← all start at infinity
dist[k] = 0 ← source is free to reach
heap = [(0, k)] ← (dist, node) min-heap
while heap:
d, u = heapq.heappop(heap) ← extract cheapest unprocessed node
if d > dist[u]: ← stale entry — skip it
continue
for v, w in graph.get(u, []): ← explore outgoing edges
nd = d + w ← candidate distance to v via u
if nd < dist[v]: ← found shorter path to v?
dist[v] = nd ← update dist[v]
heapq.heappush(heap, (nd, v)) ← push new candidate
ans = max(dist.values()) ← bottleneck = last node reached
return ans if ans < float('inf') else -1 ← -1 if unreachable node exists
⚠️ Why d > dist[u] and not d != dist[u]?

We only skip if d > dist[u] (stale — we found a better path already). We never want d < dist[u] because that would mean we somehow pushed something better without updating dist. In practice, only d > dist[u] is possible for stale entries.

Solution 2 — Bellman-Ford (O(V·E), handles negative weights)

def networkDelayTime_bf(times, n, k):
dist = [float('inf')] * (n + 1) ← 1-indexed
dist[k] = 0
for _ in range(n - 1): ← at most n-1 hops in shortest path
for u, v, w in times: ← relax every edge
if dist[u] + w < dist[v]: ← improvement found?
dist[v] = dist[u] + w
ans = max(dist[1:]) ← nodes 1..n
return -1 if ans == float('inf') else ans
💡 Complexity Summary

Dijkstra (min-heap): Time O((V+E) log V)  ·  Space O(V+E) — optimal for non-negative weights
Bellman-Ford: Time O(V·E)  ·  Space O(V) — handles negative weights
Dijkstra (array scan): Time O(V²) — better for dense graphs (E ≈ V²)

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

The signal delay to each node = shortest path from k. "When does the last node receive it?" = max of shortest paths. This is SSSP (Single-Source Shortest Path). Weighted + directed + minimize = Dijkstra.

Step 2: Confirm Non-Negative Weights

Check constraints: "1 ≤ w ≤ 100". All non-negative → Dijkstra is safe and optimal. If negative weights existed, say "I'd use Bellman-Ford instead."

Step 3: Name the Data Structures

Three things: adjacency list (graph lookup), distance dict (track best known distances), min-heap (always process cheapest next). State these before coding.

Step 4: Handle the Stale Entry Check

Proactively mention: "When we pop (d, u), we check if d > dist[u] — if so, it's a stale entry from an old push, and we skip it." This demonstrates deep understanding of the algorithm.

🗣️ What to Say to the Interviewer

"This is a shortest-path problem. Signal delay = shortest path from k.
The answer is max(all shortest path distances) — or -1 if any node unreachable.
Since weights are positive, I'll use Dijkstra with a min-heap.
I'll build an adjacency list, initialize dist[k]=0, all others infinity.
Main loop: pop cheapest, skip stale entries (d > dist[u]), relax outgoing edges.
After the heap is empty, return max(dist.values()) or -1 if any are still infinity.
Time complexity: O((V+E) log V) with a binary heap."
🔮 Likely Follow-up Questions
  • "What if weights could be negative?" → Bellman-Ford (O(V·E)) or SPFA
  • "What if you need the actual path?" → Add parent[v] = u dict on each relaxation
  • "What if the graph is undirected?" → Add both directions during graph construction
  • "Cheapest flights within K stops?" → Modified Bellman-Ford with stops constraint (#787)
  • "What if you want the path minimizing max edge?" → Dijkstra tracking max, not sum
