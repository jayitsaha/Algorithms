LeetCode #79

Word Search

Given an m×n grid of characters and a string word, return true if word exists in the grid. Adjacent cells connect horizontally or vertically, and each cell may only be used once per path.

Medium Graph DFS Backtracking Time: O(m·n·4^L) Space: O(L)
1 Understanding the Problem ▾

You have a rectangular grid of letters. Given a target word, you must determine whether that word can be spelled out by traveling through adjacent cells (up, down, left, right — not diagonal), where each cell may be visited at most once per path.

💡 Key Insight

Think of the grid as a graph where each cell is a node and each orthogonal adjacency is an edge. The question becomes: does a simple path exist that spells out the word character by character? Since we must try every possible starting cell and every possible route, this is an exhaustive search — but one we can prune aggressively using backtracking whenever a letter doesn't match.

Edge Cases: Single-cell grid and single-char word (trivially True if they match). Word longer than total cells (return False early). Grid full of same character as the word (worst-case exponential search). Word with repeated letters — the same letter can appear at multiple grid positions, just not on the same active path.

🔑 Why Backtracking is Essential

When we step into cell (r,c) and mark it as "on this path," we're making a commitment. If the path fails later, we need to un-commit — unmark the cell — so it's available for different paths starting from different roots. Without backtracking, a valid path could be blocked by a failed one. The mark is temporary: it exists only for the duration of one DFS branch.

2 The Algorithm: DFS with In-Place Backtracking ▾

Two-level structure: an outer loop that tries every cell as a starting point, and a recursive DFS helper that walks character by character, using temporary cell mutation to avoid revisiting cells on the current path.

1 Outer scan — try all starts: For every cell (r, c) in the grid, call dfs(r, c, 0). If any call returns True, immediately return True — word found!
2 Base case — success: Inside dfs(r, c, k), check if k == len(word) first. If so, all k characters have been matched — return True.
3 Bounds + character check: If (r, c) is out of bounds, or board[r][c] != word[k] (including when the cell is already marked '#'), return False immediately.
4 Mark visited: Save board[r][c] and set board[r][c] = '#'. Since '#' never appears in the word, future calls will detect this cell as "occupied" and return False automatically.
5 Recurse into 4 neighbors: Try down, up, right, left — dfs(r+1,c,k+1), etc. Use short-circuit OR: stop as soon as any direction succeeds.
6 Backtrack: After all recursive calls finish, restore board[r][c] to its saved character. This is the critical step — it un-commits this cell so other paths can use it.

3 Why It Works: The Invariant ▾
🔐 The DFS Backtracking Invariant

At any moment during dfs(r, c, k), every cell currently marked '#' belongs to the unique path from the DFS root to cell (r,c). This guarantees no cell on the current path is reused. When we backtrack, '#' is restored, and that cell becomes available to paths that don't include the current cell.

Why '#' is a safe sentinel: The word consists only of lowercase or uppercase English letters — none are '#'. So board[r][c] != word[k] will catch '#' correctly without any additional logic. The visited-cell check piggybacks on the character mismatch check for free.

Why check k == len(word) first: When k equals the word length, we've successfully matched every character and should return True. If we checked bounds first, we'd be fine — but checking k first is slightly cleaner and makes the logic read as "success before failure." More importantly, never access word[k] when k == len(word) — that's an index out of range.

⚠️ Why Forgetting to Backtrack Causes Wrong Answers

Suppose we search for "ABA" in a grid where A is at (0,0) and (0,2), and B is at (0,1). Without backtracking: start at A(0,0), use B(0,1), but now A(0,0) is permanently marked and we can't use A(0,2) either (wrong path). With backtracking: after the path A(0,0)→B(0,1) fails to find a second A adjacent to B (since (0,0) is on the same path), we backtrack and could try a completely different starting A.

Exhaustiveness: The DFS explores ALL possible paths from ALL possible starting cells. If no path spells the word, the algorithm correctly returns False — not because it got unlucky, but because it checked every possibility. The 4^L branching factor is the maximum work per start, and we have m·n starts.

4 The Code (Annotated) ▾

Solution 1 — DFS with In-Place Backtracking (Optimal — Interview Pick)

def exist(board, word): ← m×n grid, target word
rows, cols = len(board), len(board[0]) ← grid dimensions
def dfs(r, c, k): ← at cell (r,c), matching word[k]
if k == len(word): ← base case: all chars matched!
return True
if r < 0 or r >= rows: ← row out of bounds
return False
if c < 0 or c >= cols: ← column out of bounds
return False
if board[r][c] != word[k]: ← wrong char, or '#' (already visited)
return False
temp = board[r][c] ← save original letter
board[r][c] = '#' ← mark visited (sentinel)
found = (dfs(r+1, c, k+1) or ← down
dfs(r-1, c, k+1) or ← up
dfs(r, c+1, k+1) or ← right
dfs(r, c-1, k+1)) ← left (short-circuit OR)
board[r][c] = temp ← BACKTRACK: restore always
return found
for r in range(rows): ← try every cell as start
for c in range(cols):
if dfs(r, c, 0): ← start matching from word[0]
return True
return False ← exhausted all starts
⚠️ The backtrack runs even when found=True

board[r][c] = temp runs after the four recursive calls complete — regardless of whether found is True or False. When True, the restoration doesn't matter (we're about to return all the way up). When False, it's critical. You might be tempted to skip it on success, but don't: it's cleaner to always restore, and for problems returning multiple paths, not restoring would corrupt further searches.

Solution 2 — DFS with Explicit Visited Set (Same complexity, immutable board)

def exist_v2(board, word):
rows, cols = len(board), len(board[0])
visited = set() ← shared set (add/remove = backtrack)
def dfs(r, c, k):
if k == len(word): return True
if r < 0 or r >= rows or c < 0 or c >= cols: return False
if (r, c) in visited: return False ← explicit visited check
if board[r][c] != word[k]: return False
visited.add((r, c)) ← mark
found = any(dfs(r+dr, c+dc, k+1)
for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)])
visited.discard((r, c)) ← backtrack: unmark
return found
return any(dfs(r, c, 0)
for r in range(rows)
for c in range(cols))
💡 In-Place '#' vs Explicit Visited Set

In-place ('#'): O(1) extra memory per recursion level. Temporarily mutates input — must restore. Preferred when mutation is allowed and memory is at a premium.
Visited set: O(L) extra memory (one tuple per path cell). Board stays immutable throughout. Preferred when input must not be modified (e.g., concurrent readers).
Both have identical time complexity O(m·n·4^L) and recursion stack depth O(L).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize Grid + Path = Graph DFS

The moment you see a 2D grid with "find a path" or "adjacent cells" constraints, think graph. Cells are nodes, adjacencies are edges. Since we want path existence (not shortest path), DFS is natural — BFS finds shortest paths, DFS explores one full path before trying alternatives.

Step 2: Identify "each cell at most once" = Backtracking

The constraint that cells can't be reused means we need path-scoped visited tracking. A global visited array won't work — it would bleed across different starting paths. We need the visited set to be active only for the current DFS branch, then undone when we leave. That's the definition of backtracking.

Step 3: Define the Recursive State

State = position (r, c) + how far into the word we are (k). We need all three to make a decision: where are we in the grid, and which letter are we trying to match next? The base cases fall out naturally: k equals word length means done, out-of-bounds or mismatch means fail.

Step 4: Choose and Justify the Visited Strategy

Propose in-place mutation as the cleaner solution. Mention the visited-set alternative for immutable inputs. This shows you understand both options. An interviewer who asks "what if the board can't be modified?" wants to see you pivot to the set — be ready for that.

🗣️ What to Say to the Interviewer

"This is a graph path-search on a 2D grid — I'll use DFS with backtracking.
I'll try every cell as a potential starting point.
For each start, I recurse: if the current cell matches the current letter, mark it visited by replacing it with '#', then explore all 4 neighbors for the next letter. Always restore the cell after recursing — that's the backtrack.
Base cases: k equals word length means we matched everything; out-of-bounds or wrong character returns False.
Time complexity: O(m·n·4^L) where L is the word length. Space: O(L) for the recursion stack."
🔮 Likely Follow-up Questions
  • "Find all words from a list?" → Build a Trie from the word list, then DFS once with Trie traversal (#212 Word Search II)
  • "Board is read-only?" → Switch to explicit visited set — same algorithm, just add/discard instead of '#' swap
  • "Can we do better than 4^L?" → Prune: if remaining cells can't possibly contain needed chars, cut early. Reverse the word if word[-1] is rarer than word[0]
  • "What if diagonal moves allowed?" → Add 4 diagonal direction tuples: (1,1),(1,-1),(-1,1),(-1,-1)
  • "Worst-case input?" → Grid full of 'A's, word = "AAAA...AAB" — explores 4^(L-1) dead-end paths before finding no 'B'

7 Summary & Key Takeaways ▾
Grid path problems → DFS; path existence (not shortest) → DFS over BFS
Mark with '#' on entry, restore on exit — this is the backtracking skeleton
Check k == len(word) BEFORE bounds — avoids index error and is the success base case
Try every cell as a start — the word can begin anywhere in the grid
Short-circuit OR stops early once any branch succeeds — big practical speedup
Time O(m·n·4^L), Space O(L) for the call stack — L is the word length
ApproachTimeSpaceNotes
DFS + In-Place '#' ✓ O(m·n·4^L) O(L) Optimal; mutates board temporarily
DFS + Visited Set O(m·n·4^L) O(L) Same; cleaner for immutable boards
🎯 When to Recognize DFS Backtracking
  • Grid problem with "find a path" or "each cell at most once"
  • Explore all possibilities (not minimize distance)
  • "Does there exist a configuration / arrangement?" questions
  • Building a solution character by character, decision by decision
  • Whenever you need to commit a resource during recursion and uncommit after
