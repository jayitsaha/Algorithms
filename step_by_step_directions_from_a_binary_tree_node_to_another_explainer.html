LeetCode #2096

Step-By-Step Directions From a Binary Tree Node to Another

Find the shortest path between two nodes in a binary tree; return directions as a string of 'L', 'R', 'U' moves.

Medium Trees LCA + Path Building Time: O(n) Space: O(n)
1 Understanding the Problem ▾

You are given a binary tree and two distinct node values, startValue and destValue. Return a string of directions — using only 'L' (left), 'R' (right), and 'U' (up/parent) — that describes the shortest path from the start node to the destination node.

💡 Key Insight

Every path between two nodes in a tree passes through their Lowest Common Ancestor (LCA). The path shape is always: go up from start to LCA, then go down to dest. No detours, no cycles — trees guarantee a single unique path.

start → U U U … → LCA → L R L … → dest
✓ Correct Path: start=5, dest=1

Tree: 3→{5,1}, 5→{6,2}, 1→{0,8}
LCA(5,1) = node 3 (root)
LCA → start: go L (3→5) → len=1
LCA → dest: go R (3→1) → path="R"
Answer = "U" + "R" = "UR"

✗ Naive Mistake: Ignore LCA

Root → start = "L"
Root → dest = "R"
Strip common prefix "" → concat
"L" + "R" = "LR" ← WRONG!
Should be "UR" (go up, then right)

Edge cases to consider: Start equals destination (return empty string ""). One node is an ancestor of the other — path is only U's or only L/R's. Single-node tree. Very deep path — recursion stack depth is O(n) for a degenerate (linked-list-like) tree.

2 The Algorithm: LCA + Dual DFS Path Finding ▾

The solution has three clean stages. Each is an O(n) tree traversal, giving O(n) total:

1 Find the LCA — Recursively search for the lowest node that has both start and dest in its subtree (including possibly being start or dest itself). This is the "turning point" where the path changes from going up to going down.
2 DFS from LCA to start — Record each step as 'L' or 'R' in a list, backtracking when we go down a wrong branch. The result is the sequence of left/right moves from LCA to start. Its length tells us how many 'U's we need.
3 DFS from LCA to dest — Same DFS process targeting destValue. The result is the exact L/R sequence from LCA down to dest.
4 Combine — Answer = 'U' * len(src_path) + ''.join(dst_path). The src_path L/R directions are discarded — only the count matters. The dst_path gives us the exact downward route.
🔑 The Elegant Trick: U's Don't Need Direction

Going from start up to LCA: every single step is "go to parent" = 'U'. It doesn't matter whether each step was a left-child or right-child relationship — going up is always 'U'. So we only need how many steps (the length), not the actual L/R values. That's why 'U' * len(src_path) works perfectly without reversing or storing direction details.

3 Why It Works: The Invariant ▾
🔐 The Core Invariant

In any tree, there is exactly one simple path between any two nodes. That path always passes through the LCA. Therefore: path(A→B) = path(A→LCA) reversed + path(LCA→B). The path is automatically shortest because there are no alternatives.

Why LCA is the deepest turning point: The LCA is the deepest ancestor of both start and dest. Going any higher would visit a more distant common ancestor, adding unnecessary steps. The LCA is the exact node where the path "turns the corner" from ascending to descending.

Why DFS backtracking gives correct paths: The findPath function uses the pattern: append direction on entry, pop on failure. When a recursive call returns False (target not in that subtree), the direction we just appended is popped off — exactly restoring the path to its pre-call state. When True is returned all the way up, every direction in the list is a confirmed correct step.

⚠️ Common Mistake: Finding LCA from Root for Both Paths

Some solutions find paths from the original root to start and dest, then strip the common prefix. This works logically but has tricky off-by-one handling and requires careful string slicing. The cleaner approach: once you have the LCA, start both DFS path searches from the LCA — eliminating any shared prefix by design.

Special case — start is ancestor of dest (or vice versa): The LCA equals start (or dest). When we call findPath(lca_node=start, target=start), it returns immediately with an empty path (src_path = []). So we generate 'U' * 0 = "" and only the L/R path to dest. The algorithm handles this transparently.

4 The Code (Annotated) ▾

Solution 1 — LCA + DFS Path Building (Optimal · Interview Pick)

def getDirections(root, startValue, destValue):
def lca(node): ← Find Lowest Common Ancestor
if not node: return None ← Empty subtree: contains neither target
if node.val in (startValue, destValue): ← This node IS start or dest → LCA is at or above
return node ← Return early — no need to search deeper
L = lca(node.left) ← Recurse into left subtree
R = lca(node.right) ← Recurse into right subtree
if L and R: return node ← One target in each subtree → THIS is the LCA
return L or R ← Only one side found → bubble that result up
def findPath(node, target, path): ← DFS: record L/R path to target (with backtracking)
if not node: return False ← Dead end: target not in this subtree
if node.val == target: return True ← Found! path already has the correct route
path.append('L') ← Try left — tentatively record 'L'
if findPath(node.left, target, path): ← If found in left subtree…
return True ← …path is complete
path.pop() ← Left failed → undo the 'L' (backtrack!)
path.append('R') ← Try right — tentatively record 'R'
if findPath(node.right, target, path): ← If found in right subtree…
return True ← …path is complete
path.pop() ← Right also failed → undo the 'R'
return False ← Target not in either subtree of this node
lca_node = lca(root) ← Stage 1: find LCA
src_path, dst_path = [], [] ← Two lists for L/R sequences
findPath(lca_node, startValue, src_path) ← Stage 2: LCA → start
findPath(lca_node, destValue, dst_path) ← Stage 3: LCA → dest
return 'U'*len(src_path) + ''.join(dst_path) ← Stage 4: combine (up from start + down to dest)
⚠️ Why path.pop() is Non-Negotiable

The path list is mutated in-place (passed by reference). If we try going left and it fails, we must remove the tentative 'L' before trying right. Without path.pop(), failed-branch directions pile up in the list, giving a completely wrong path. This append/pop pattern is the canonical DFS backtracking idiom — memorize it.

Solution 2 — Parent Map + BFS (Conceptually simpler, more memory)

from collections import deque
def getDirections(root, startValue, destValue):
par, nmap = {}, {} ← par: child→(parentVal, dir); nmap: val→node
stk = [root] ← Iterative DFS to build parent map
while stk:
nd = stk.pop(); nmap[nd.val] = nd
if nd.left: par[nd.left.val]=(nd.val,'L'); stk.append(nd.left)
if nd.right: par[nd.right.val]=(nd.val,'R'); stk.append(nd.right)
vis, q = set(), deque([(startValue, "")]) ← BFS from start with running path string
while q:
v, path = q.popleft()
if v == destValue: return path ← BFS guarantees shortest path
vis.add(v); nd = nmap[v]
if nd.left and nd.left.val not in vis: q.append((nd.left.val, path+'L'))
if nd.right and nd.right.val not in vis: q.append((nd.right.val, path+'R'))
if v in par and par[v][0] not in vis: ← Go UP to parent
q.append((par[v][0], path+'U'))
💡 Complexity Comparison

Solution 1 (LCA + DFS): Time O(n) — 3 tree passes, each O(n). Space O(n) for recursion stack and paths.
Solution 2 (Parent Map + BFS): Time O(n), Space O(n) — but extra O(n) for par and nmap dicts, plus BFS queue grows O(n) in worst case. More intuitive to explain, more memory-hungry.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

Signals: "path between two nodes in a binary tree" + "directions L/R/U." In any tree, the shortest path between two nodes passes through their LCA. The problem reduces to: find LCA, find path from LCA to each node, combine. This is a standard LCA problem with a path-recording twist.

Step 2: Sketch the Path Shape

Draw start → ... → LCA → ... → dest. The left segment goes UP (every step is 'U', regardless of whether it was left or right). The right segment goes DOWN (every step is 'L' or 'R'). You only need: (a) the LENGTH of the first segment, and (b) the exact L/R sequence of the second.

Step 3: Propose Both Solutions

Mention the simpler BFS approach first: "Build a parent map in O(n), then BFS from start treating the graph as undirected, appending 'U' for parent moves." Then offer the cleaner LCA approach: "Alternatively, find LCA in O(n), then DFS twice for paths — more elegant, same complexity."

Step 4: Code the LCA Solution

The LCA function is standard #236 logic. The findPath function uses backtracking (append + pop). Call findPath twice from lca_node. Combine with 'U' * len(src) + ''.join(dst). Make sure to call lca() on root, not on lca_node itself.

🗣️ What to Say to the Interviewer

"The key observation: any path in a tree passes through the LCA of the two endpoints.
So the path is: go UP from start to LCA (all 'U's), then go DOWN to dest (exact L/R).
Step 1: find LCA with standard recursive logic — if both targets found in left and right subtrees, this node is LCA.
Step 2: DFS from LCA to start with backtracking — gives us path length → that many 'U's.
Step 3: DFS from LCA to dest — gives us the exact L/R string.
Result: 'U' * len(src_path) + ''.join(dst_path). Time O(n), Space O(n)."
🔮 Likely Follow-up Questions
  • "What if start == dest?" → LCA is the node itself; both paths are empty → return "".
  • "Can you do it iteratively?" → Yes: parent map + BFS (Solution 2 above).
  • "What if the tree is a general n-ary tree?" → Same LCA logic; DFS tries all children with direction index.
  • "What if values are not unique?" → Problem guarantees uniqueness, but if not: find nodes first, then do LCA on node references.
  • "Stack overflow on deep tree?" → Use iterative DFS with an explicit stack.

7 Summary & Key Takeaways ▾
Every path between two tree nodes passes through their LCA — use LCA as the decomposition point.
Going up from start to LCA: you only need the COUNT of steps (all become 'U'), not the L/R directions.
Going down from LCA to dest: DFS with backtracking (append/pop) gives the exact L/R sequence.
The standard recursive LCA algorithm runs in O(n) and handles edge cases (node is ancestor, or IS one of the targets) naturally.
Total time: O(n) — three tree traversals (LCA, findPath×2), each O(n). Total space: O(n) for recursion and paths.
🎯 When to Use This Pattern
  • Shortest path between two nodes in an unweighted tree.
  • Path involves going up ('U') and down ('L'/'R') — LCA is the turning point.
  • Any problem where you need the "meeting point" of two tree paths.
  • The words "binary tree" + "directions" + "path from A to B" should immediately suggest LCA.
