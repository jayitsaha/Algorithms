LeetCode #1482

Minimum Number of Days to Make m Bouquets

Find the minimum number of days to wait so that m bouquets of k consecutive adjacent bloomed flowers can be formed.

Medium Binary Search Binary Search on Days Time: O(n log(max)) Space: O(1)
1 Understanding the Problem ▾

You have a garden of n flowers in a row. Flower i blooms on day bloomDay[i]. You want to make m bouquets; each bouquet needs exactly k consecutive adjacent bloomed flowers. What is the minimum number of days you must wait?

💡 Key Insight

If you wait d days, a flower is "available" if its bloom day ≤ d. You can check in O(n) whether d days is enough. The key observation: if d days works, then d+1 days also works (more flowers bloom, not fewer). This monotone property means we can binary search on d rather than try every day from 1 to max(bloomDay).

Edge cases: m×k > n → return −1 immediately. m=1, k=1 → return min(bloomDay). Very large bloom day values (up to 10⁹) — binary search handles this; a linear scan over days would TLE.

2 The Algorithm: Binary Search on Days ▾

Instead of trying every day from 1 to max(bloomDay), we binary search over that range. For each candidate day mid, a greedy O(n) scan counts how many bouquets we can form. This reduces O(n × max) brute force to O(n × log(max)).

🔑 Why Binary Search Works Here

The function canMake(d) is monotonically non-decreasing: if day d works, day d+1 also works (same flowers still bloomed, possibly more). There is a threshold day T below which it's impossible and at or above T it's always possible. Binary search finds T exactly.

1 Early exit: If m × k > n, return −1. We can never have enough flowers even if all bloom.
2 Set search bounds: lo = 1 (earliest day), hi = max(bloomDay) (all flowers bloomed). The answer always lies in [lo, hi].
3 Binary search loop: While lo ≤ hi, compute mid = (lo + hi) // 2. Run canMake(mid). If True → record ans = mid, shrink hi = mid − 1 (try smaller). If False → grow lo = mid + 1 (need more days).
4 canMake(d): Scan left to right. If bloomDay[i] ≤ d, extend streak. Otherwise reset streak to 0. When streak reaches k, increment bouquets and reset streak. Return bouquets ≥ m.
5 Return ans: The minimum valid day found during binary search.
3 Why It Works: The Invariant ▾
🔐 The Binary Search Invariant

At every iteration of the while loop, the true answer (if it exists) is always within [lo, hi]. We only set ans when canMake = True, so ans is always valid. We narrow toward the minimum by shrinking hi whenever True. We never discard the real answer.

Why the greedy canMake scan is correct: On day d, a flower is available iff bloomDay[i] ≤ d. We scan left to right, tracking a consecutive streak of available flowers. When the streak hits k, we greedily "claim" those k flowers as a bouquet (start fresh). A gap (unavailable flower) breaks any bouquet, so we reset. This greedy is optimal: starting each bouquet as early as possible leaves the most flowers for subsequent bouquets.

⚠️ Critical Mistake: hi = n instead of hi = max(bloomDay)

The days in bloomDay can be up to 10⁹. If you set hi = n (number of flowers), you'll only search day 1 through n, completely missing the actual answer for inputs where bloomDay values exceed n. Always set hi = max(bloomDay).

Why we can stop early: When lo > hi, the entire feasibility space [1, max(bloomDay)] has been explored. The smallest day for which canMake returned True was recorded in ans. That's our answer.

💡 The "Leftmost True" Pattern

This is a canonical binary search template: find the leftmost position in a monotone boolean array where the condition becomes True. When canMake(mid) = True, we record ans and look left (hi = mid−1). When False, we look right (lo = mid+1). The final ans is the leftmost True.

4 The Code (Annotated) ▾

Solution 1 — Binary Search on Days (Optimal, Interview Pick)

def minDays(bloomDay, m, k):
n = len(bloomDay)
if m * k > n: ← Impossible: not enough flowers in the garden
return -1
lo, hi = 1, max(bloomDay) ← Day range: 1 (min) to latest bloom (max)
ans = hi ← Worst-case fallback (always valid if early-exit passed)
while lo <= hi:
mid = (lo + hi) // 2 ← Candidate day to evaluate
if canMake(bloomDay, m, k, mid):← O(n) feasibility check
ans = mid ← Valid day! Record it
hi = mid - 1 ← Search left for a smaller valid day
else:
lo = mid + 1 ← Not enough flowers; need more days → search right
return ans
def canMake(bloomDay, m, k, day): ← O(n): can we form m bouquets by this day?
bouquets = streak = 0
for d in bloomDay: ← Scan each flower left to right
if d <= day: ← This flower has bloomed by target day
streak += 1 ← Extend consecutive bloomed run
else:
streak = 0 ← Gap breaks the run; reset
if streak == k: ← Full bouquet of k consecutive flowers!
bouquets += 1 ← Claim it
streak = 0 ← Flowers used; start fresh
return bouquets >= m ← Did we get enough bouquets?
⚠️ Why reset streak when streak == k (not >)?

We use == because we reset streak to 0 immediately after claiming a bouquet. So streak can never exceed k. This keeps the logic tight: the moment we complete k consecutive flowers, we take the bouquet and start counting fresh. Flowers in one bouquet cannot be shared with the next.

Solution 2 — Brute Force (Try Every Day), O(n × max), TLE

def minDays_brute(bloomDay, m, k):
if m * k > len(bloomDay): return -1
for day in range(1, max(bloomDay) + 1): ← Try each day from 1 to max
if canMake(bloomDay, m, k, day): ← O(n) check
return day ← First valid day = answer
return -1
💡 Complexity Summary

Brute Force: Time O(n × max(bloomDay)), Space O(1). TLE for bloomDay values up to 10⁹.
Binary Search: Time O(n × log(max(bloomDay))), Space O(1). For n=10⁵ and max=10⁹, roughly 10⁵ × 30 ≈ 3×10⁶ operations — very fast.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the "minimum days / minimum X" Signal

The problem asks for the minimum number of days. Whenever you see "minimum X such that [condition holds]" and the condition is monotone in X, suspect binary search on the answer. The key test: if X works, does X+1 also work? Yes (more days = more flowers) — binary search applies.

Step 2: Separate Feasibility from Optimization

Design the helper first: "Given day d, can I make m bouquets?" This is a much simpler subproblem. Write canMake() and test it mentally on small examples. Only then wrap it in a binary search.

Step 3: Choose Bounds Carefully

lo = 1 (no flower blooms before day 1 is the first possible day). hi = max(bloomDay) (by this day, all flowers have bloomed — guaranteed feasible if m*k ≤ n). Never use hi = n.

Step 4: Leftmost-True Binary Search Template

When canMake(mid) = True: ans = mid, hi = mid−1. When False: lo = mid+1. At the end, ans holds the minimum True. This is the standard "find leftmost True in a boolean array" template — memorize it.

🗣️ What to Say to the Interviewer

"The problem asks for a minimum — I'll check if binary search on the answer applies.
Key observation: if we can make m bouquets on day d, we can definitely do it on day d+1.
Feasibility is monotone → binary search works.
Search range: [1, max(bloomDay)]. For each midpoint, run a greedy O(n) scan:
extend consecutive bloomed-flower streaks, claim a bouquet every k flowers.
If feasible, record the day and narrow left. Otherwise, go right.
Time: O(n log max), Space: O(1)."

7 Summary & Key Takeaways ▾
"Minimum X such that condition holds" + monotone condition = binary search on the answer
canMake(d) is a greedy O(n) scan: count consecutive streaks, claim bouquet when streak = k
Search range is [1, max(bloomDay)] — bloom days can be up to 10⁹, NOT bounded by n
Early exit: if m × k > n, return −1 — impossible no matter how long you wait
Leftmost-True pattern: when True set ans=mid and hi=mid−1; when False set lo=mid+1

🎯 When to Recognize Binary Search on Answer
  • "Find the minimum/maximum X such that [condition]" — classic signal
  • Direct computation is hard, but verification (canMake) is easy and O(n)
  • The condition is monotone: once True for X, always True for X+1 (or vice versa)
  • The search range is a continuous numeric interval (days, speed, capacity, distance)
