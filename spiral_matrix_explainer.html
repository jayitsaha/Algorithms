LeetCode #54

Spiral Matrix

Given an m × n matrix, return all elements in clockwise spiral order — outer ring first, peeling inward.

Medium Matrix Four Boundaries Time: O(m·n) Space: O(1)
1 Understanding the Problem ▾

Think of peeling an onion. You strip the outermost layer, revealing a smaller rectangle inside. Then you strip that layer, and so on until nothing is left. For a matrix, one "layer" is the perimeter of the active rectangle — walk it clockwise: top row left→right, right column top→bottom, bottom row right←left, left column bottom→top.

💡 Key Insight

Track four boundary pointers: top, bottom, left, right. After walking each edge, shrink that boundary inward by one. The loop terminates when top > bottom or left > right — the rectangle has collapsed.

✓ 3×3 Example

 [4, 5, 6],
 [7, 8, 9]]
→ [1,2,3,6,9,8,7,4,5]

Edge Cases: 1×1 matrix (return the single element), 1×n single row (walk left-to-right only), m×1 single column (walk top-to-bottom only), non-square matrices where m ≠ n — the boundary shrinks asymmetrically.

2 The Algorithm: Four Boundaries, Shrink Inward ▾

We maintain four variables — top, bottom, left, right — that define the active (unvisited) rectangle. Each outer loop iteration processes exactly one ring of that rectangle: four edge walks followed by four boundary shrinks.

1 Initialize: top=0, bottom=m-1, left=0, right=n-1. Result list is empty.
2 Walk top row →: for col in [left..right], append matrix[top][col]. Then top += 1 (seal this row).
3 Walk right column ↓: for row in [top..bottom], append matrix[row][right]. Then right -= 1.
4 Walk bottom row ← (guarded): only if top <= bottom: for col in [right..left], append matrix[bottom][col]. Then bottom -= 1.
5 Walk left column ↑ (guarded): only if left <= right: for row in [bottom..top], append matrix[row][left]. Then left += 1.
6 Repeat while top <= bottom and left <= right. One iteration = one full ring peeled.
🔑 Why the Boundary Guards?

After walking the top row and right column, the rectangle might have become a single remaining row (top == bottom) or single remaining column (left == right). The guard if top <= bottom before the bottom row walk prevents re-collecting cells already captured in the top-row pass. Similarly if left <= right guards the left column pass.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the start of every outer loop iteration, the rectangle [top..bottom] × [left..right] contains exactly the cells not yet added to result. After processing one full ring, that ring is sealed by shrinking all four boundaries — the invariant is restored for the next (smaller) rectangle.

Why no cell is visited twice: Each boundary advances immediately after its edge is walked. The top boundary increments right after the top row — future passes begin from the new top row and can never touch the old one. The same exclusion logic applies to each of the four edges independently.

Why no cell is missed: The while condition top <= bottom and left <= right guarantees we keep looping as long as any non-empty rectangle remains. When both pairs of boundaries cross, all cells have been collected — this follows directly from the shrink-by-exactly-one property: we never skip a row or column.

⚠️ The Single-Row Trap — Concrete Example

Matrix = [[1,2,3]]. After walking top row (1,2,3), top becomes 1. Now top=1 > bottom=0. The while loop exits — correct, we're done. Without the guard, if we tried to walk the bottom row backward (3,2,1), we'd produce [1,2,3,3,2,1] — six elements from a three-element row!

Space complexity: We only need four integer pointers. No visited[][] matrix, no direction arrays, no queue — the boundary pointers themselves encode all the state we need. Output list excluded from space analysis (it is the answer, not auxiliary storage).

4 The Code (Annotated) ▾

Solution 1 — Four Boundaries (Optimal: O(m·n) time, O(1) extra space) — Interview Pick

def spiralOrder(matrix):
result = [] ← output list (not counted in space)
top, bottom = 0, len(matrix) - 1 ← inclusive row boundaries
left, right = 0, len(matrix[0]) - 1 ← inclusive col boundaries
while top <= bottom and left <= right: ← any unvisited rectangle?
for col in range(left, right + 1): ← walk top row left→right
result.append(matrix[top][col])
top += 1 ← seal top row, shrink downward
for row in range(top, bottom + 1): ← walk right col top→bottom
result.append(matrix[row][right])
right -= 1 ← seal right col, shrink leftward
if top <= bottom: ← guard: rows still remain?
for col in range(right, left - 1, -1): ← walk bottom row right→left
result.append(matrix[bottom][col])
bottom -= 1 ← seal bottom row, shrink upward
if left <= right: ← guard: cols still remain?
for row in range(bottom, top - 1, -1): ← walk left col bottom→top
result.append(matrix[row][left])
left += 1 ← seal left col, shrink rightward
return result
⚠️ Most Common Bug: Missing the Guards

The two if guards before the bottom and left walks are the most commonly omitted lines. Many correct implementations on the web skip them — and pass LeetCode — because most test cases don't hit the single-row/single-column case directly. But on a 2×3 or 1×5 matrix, missing the guards produces duplicates. Always include them.

Solution 2 — Direction Vector Simulation (O(m·n) time, O(m·n) space)

def spiralOrder(matrix):
m, n = len(matrix), len(matrix[0])
result = []; seen = [[False]*n for _ in range(m)] ← visited grid: O(m·n) space
dr = [0, 1, 0, -1]; dc = [1, 0, -1, 0] ← right, down, left, up
r = c = di = 0
for _ in range(m * n):
result.append(matrix[r][c]); seen[r][c] = True
nr, nc = r + dr[di], c + dc[di]
if 0 <= nr < m and 0 <= nc < n and not seen[nr][nc]:
r, c = nr, nc ← keep going same direction
else:
di = (di + 1) % 4; r, c = r+dr[di], c+dc[di] ← turn clockwise
return result
💡 Complexity Summary

Four Boundaries: Time O(m·n), Extra Space O(1) — optimal, interview pick
Direction Simulation: Time O(m·n), Space O(m·n) — intuitive but wastes memory
Both visit every cell exactly once; the boundary approach achieves this without any bookkeeping matrix.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Draw It Out — Spiral = Rings

On a 3×3 example, trace the path by hand. You'll immediately see: outer ring (8 cells), inner point (1 cell). The pattern is "walk outer perimeter, peel it off, repeat." This is your mental model — translate it to code.

Step 2: Propose Direction Simulation as Baseline

"I can simulate the walk with four direction vectors [→,↓,←,↑]. When the next cell is out of bounds or already visited, turn clockwise. O(m·n) time and O(m·n) space for a visited matrix." — establishes you understand the problem.

Step 3: Optimize with Four Boundaries — O(1) Space

"The boundary pointers replace the visited matrix. I know exactly which cells are unvisited — they're inside the current rectangle. After each edge, shrink that boundary. Two guards handle the single-row/column degenerate cases. O(1) extra space."

Step 4: Test Edge Cases Verbally

Walk through 1×1 (one element, loop runs once, both guards fail, done), 1×3 (top row fires, top becomes 1 > bottom=0, loop exits — correct), 3×1 (top row has 1 elem, right col fires for remaining rows, guards stop bottom/left).

🗣️ What to Say to the Interviewer

"I'll use four boundary pointers — top, bottom, left, right — defining the unvisited rectangle.
Each loop iteration walks one full ring: right along top row, down right column,
left along bottom row (if rows remain), up left column (if cols remain).
After each edge I shrink that boundary. Two guards prevent re-collecting a single
remaining row or column. Time O(m·n), Space O(1) extra."

7 Summary & Key Takeaways ▾
Spiral traversal = peel rings. Four boundary pointers define the active rectangle at each iteration.
Shrink each boundary inward immediately after its edge is walked — this is what prevents revisiting.
The two guards (if top <= bottom, if left <= right) are critical for single-row and single-column cases.
Every cell is visited exactly once: O(m·n) time. No visited matrix needed: O(1) extra space.
The same boundary pattern powers Spiral Matrix II (fill), plus all "ring-processing" matrix problems.
