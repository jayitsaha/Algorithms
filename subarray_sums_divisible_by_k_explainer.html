Subarray Sums Divisible by K

Given an integer array nums and an integer k, return the number of non-empty subarrays that have a sum divisible by k.

Medium Prefix Sum Hash Map Time: O(n) Space: O(k)
1 Understanding the Problem ▾

We need to count all contiguous subarrays of nums whose elements add up to a multiple of k. A sum is "divisible by k" means sum % k == 0. The array can contain negative numbers, which is a crucial complication.

💡 Key Insight

Two prefix sums with the same remainder mod k mean that the subarray between those two positions has a sum divisible by k. Track how many times each remainder has appeared using a hash map — when you encounter a remainder you've seen before, each prior occurrence contributes one valid subarray.

✓ Example: nums=[4,5,0,−2,−3,1], k=5

Prefix sums: 0, 4, 9, 9, 7, 4, 5
Prefix mods: 0, 4, 4, 4, 2, 4, 0
Same remainders → valid subarrays
Answer: 7

Concrete walkthrough (mini-example): nums=[5,0], k=5. Prefix sums: [0, 5, 5]. Remainders: [0, 0, 0]. Two positions have rem=0 already (the seed and after index 0): that gives 1 match, subarray [5]. After index 1 we have rem=0 again — now 2 prior positions have rem=0, giving 2 more matches: [0] and [5,0]. Total = 3.

Edge cases: All-zero arrays (every subarray qualifies), negative numbers (Python's modulo always returns non-negative, but C++ requires manual adjustment: ((rem % k) + k) % k), a single-element array, and the "whole array" case where the seed {0:1} fires at the end.

2 The Algorithm: Prefix Sum + Modulo + Hash Map ▾

The trick relies on a beautiful modular arithmetic identity. If we define prefix[i] = sum of nums[0..i], then the sum of subarray nums[j+1..i] = prefix[i] − prefix[j]. For that to be divisible by k:

🔢 The Core Math
(prefix[i] − prefix[j]) % k == 0
⟺ prefix[i] % k == prefix[j] % k

Two prefix sums share the same remainder mod k if and only if the subarray between them sums to a multiple of k. So instead of checking each subarray sum, we only need to track how often each remainder has appeared.

This gives us the algorithm: maintain a running prefix sum remainder rem, and a hash map seen that counts how many times each remainder has appeared so far. Before starting, seed seen[0] = 1 to handle subarrays that start from index 0.

1 Initialize: seen = {0: 1}, rem = 0, count = 0. The seed seen[0]=1 represents the "virtual" prefix sum of 0 before the array begins.
2 Scan each element x: Update rem = (rem + x) % k. In Python, this is always non-negative. In C++/Java, add + k) % k to handle negatives.
3 Count matches: count += seen.get(rem, 0). Every prior position with the same remainder contributes one valid subarray ending at the current index.
4 Record current position: seen[rem] = seen.get(rem, 0) + 1. This position's remainder is now available for future elements to match against.
5 Return count after processing all elements. Total matches = total valid subarrays.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point in the scan, seen[r] equals the number of prefix positions (including the virtual position −1 via the seed) whose prefix sum has remainder r mod k. When the current position also has remainder r, each of those prior positions forms a valid subarray with the current position as the endpoint.

Why the seed {0:1} is necessary: Consider a subarray starting from index 0, say nums[0..i]. Its sum is exactly prefix[i] − prefix[−1], where prefix[−1] = 0 (the empty sum). For this to be divisible by k, we need rem(prefix[i]) == rem(0) == 0. The seed ensures that when rem hits 0, we count this as one valid subarray. Without the seed, we'd miss all subarrays starting at index 0.

Why counting matches before recording is correct: When we arrive at position i with remainder r, we add seen[r] (prior positions) to count. Then we record this position in seen. This ordering ensures we count prior positions that share the remainder, not the current position itself — avoiding off-by-one errors.

⚠️ Negative Numbers in C++/Java

In Python, -2 % 5 == 3 (always non-negative). In C++ and Java, -2 % 5 == -2 (sign follows dividend). For C++/Java you must write: rem = ((rem + x) % k + k) % k. In Python, the plain % k always works correctly.

Why this is O(n) time and O(k) space: We scan the array once (O(n)). The hash map can contain at most k distinct keys (remainders 0 through k−1), so space is O(k). For very large k and small n, the space is actually O(n), bounded by min(n, k).

4 The Code (Annotated) ▾

Solution 1 — Prefix Sum + Hash Map (Optimal, Interview Pick)

def subarraysDivByK(nums, k):
seen = {0: 1} ← Seed: virtual prefix sum=0 before start
rem, count = 0, 0 ← rem = running prefix remainder; count = answer
for x in nums: ← Single pass through array
rem = (rem + x) % k ← Extend prefix sum, keep only the remainder
count += seen.get(rem, 0) ← # prior positions with same rem = new valid subarrays
seen[rem] = seen.get(rem, 0) + 1 ← Record this position's remainder for future use
return count ← Total valid subarrays
⚠️ Why count before record?

If you add to seen before counting, you'd include the current position as a "prior" position, incorrectly counting a zero-length subarray. Always: count first (look up seen[rem]), then record (update seen[rem]).

Solution 2 — Brute Force (O(n²), for reference only)

def subarraysDivByK_brute(nums, k):
n, count = len(nums), 0
for i in range(n): ← Outer loop: start index
total = 0
for j in range(i, n): ← Inner loop: extend subarray
total += nums[j] ← Build subarray sum incrementally
if total % k == 0: ← Check divisibility
count += 1
return count
💡 Complexity Summary

Brute Force: Time O(n²), Space O(1) — TLE for n > 10,000
Prefix Hash Map (Optimal): Time O(n), Space O(min(n, k)) — passes all test cases
Note: You can also use an array of size k instead of a dict if k is small, for better cache performance.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern Signals

"Subarray sum" immediately suggests prefix sums. "Divisible by k" with negatives suggests modular arithmetic. "Count subarrays" suggests we need a counting data structure — a hash map of remainders. These three signals together point directly to the prefix-sum + modulo + hash-map sub-pattern.

Step 2: Start with Brute Force

"My brute force is to try all pairs (i, j) and accumulate the sum. That's O(n²). For n=50,000, this is too slow. I need to find subarrays without re-summing."

Step 3: Apply the Math Identity

"Instead of sum(nums[j..i]), I can use prefix sums: sum = prefix[i] − prefix[j]. For this to be divisible by k, I need prefix[i] % k == prefix[j] % k. So I only need to count how often each remainder has appeared before."

Step 4: Handle the Seed and Edge Cases

"I need to handle subarrays starting at index 0. I can do this by seeding seen = {0: 1}, treating the position before index 0 as having prefix sum 0. I also need to handle negative numbers — Python % always gives non-negative, but C++ would need (rem % k + k) % k."

🗣️ What to Say to the Interviewer

"Two subarrays [j+1..i] has sum divisible by k iff prefix[i] and prefix[j] have the same remainder mod k.
So I'll track seen = {0: 1} for the virtual prefix of 0, and a running remainder rem.
At each index: update rem = (rem + x) % k, add seen.get(rem, 0) to count, then increment seen[rem].
The seed handles subarrays starting at index 0. Time O(n), space O(k)."
🔮 Likely Follow-up Questions
  • "What if k could be 0?" → Division by zero — handle as a special case or assume k ≥ 1

Two prefix sums with the same remainder mod k mean the subarray between them is divisible by k
Seed seen={0:1} before scanning to correctly count subarrays starting at index 0

Space is O(k), not O(n) — only k distinct remainders exist (0 through k−1)
