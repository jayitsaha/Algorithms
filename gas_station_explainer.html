LeetCode #134

Gas Station

Find the starting gas station from which you can complete a circular route — or return -1 if no solution exists.

Medium Greedy Start Where Deficit Recovers Time: O(n) Space: O(1)
1 Understanding the Problem ▾

There are n gas stations arranged in a circle. At station i you collect gas[i] liters of fuel, then spend cost[i] liters traveling to station i+1 (wrapping around). Your car starts empty with an unlimited tank. Find the unique index from which you can complete the full circuit without your tank going negative — or return -1 if it is impossible.

💡 Key Insight

Define the net gain at each station as gas[i] - cost[i]. If the total of all net gains is negative the trip is globally impossible. If it is non-negative, exactly one valid start exists. We can find it greedily: whenever our running tank balance goes negative, every station from the current candidate start up to the current index is proven to be a bad start — so we jump to the next station and reset the tank.

Example: gas=[1,2,3,4,5], cost=[3,4,5,1,2]

2 The Algorithm: One-Pass Greedy with Two Accumulators ▾

Instead of naively trying every station as a starting point (O(n²) brute force), we observe that a failed run gives us information we can use. When our running tank balance goes negative at index i, we know that every station from the current candidate start through i is also a bad start — so we skip all of them at once and jump to i+1.

1 Initialize three variables: tank = 0 tracks running fuel from the current candidate start (resets when we jump). total = 0 tracks the global net gain across all stations (never resets). start = 0 is the current candidate starting index.
2 Scan every station i: Compute net = gas[i] - cost[i]. Add it to both tank and total. These are updated together but serve different purposes.
3 If tank < 0: The current candidate start is provably bad. Set start = i + 1 (next candidate) and reset tank = 0. Note: we do not reset total — it keeps counting toward the global feasibility check.
4 After the loop: If total < 0, the circuit is globally impossible — return -1. Otherwise return start. (When total ≥ 0, the final value of start is guaranteed to be the unique valid starting point.)
🔑 The Greedy Skip — Why It's Valid

When tank goes negative at station i, it means the sum of nets from start to i is negative. For any intermediate index j (where start < j ≤ i), the cumulative net from j to i would be even more negative (because we skip the early positive-or-neutral prefix from start to j-1 but still face the same negative tail). So every index between start and i inclusive is also a bad start — we can safely jump to i+1.

3 Why It Works: The Invariant ▾
🔐 Two Core Invariants

Invariant 1 — Global feasibility: total at the end equals the sum of all net gains. If total < 0, no starting point can work (you'd run out of gas no matter what). If total ≥ 0, exactly one valid start exists (provable by the problem constraints).

Invariant 2 — Candidate correctness: At the end of the loop, start is the index such that the cumulative sum from start to n-1 is always non-negative. This is the suffix of the array with no internal resets. The global non-negativity of total then guarantees that the "wrap-around" prefix (stations 0 to start-1) will also be covered by the surplus built up in the suffix.

Proof of wrap-around coverage: Let S = sum of nets from start to n-1 (we know S ≥ 0 because tank never went negative after the final reset). Let P = sum of nets from 0 to start-1 (the prefix). Then S + P = total ≥ 0, so P ≥ -S. This means the maximum deficit accumulated during the prefix is at most |P| ≤ S. The surplus built up in the suffix (at least S) is enough to cover whatever the prefix owes — so the tank stays non-negative throughout the full circuit.

⚠️ Common Confusion: Do I Need to Simulate the Wrap-Around After Finding start?

No — if you trust the invariant. The mathematical proof above shows the wrap-around is always covered when total ≥ 0. In an interview, you can mention: "I could verify by simulating from start, but the greedy proof makes it unnecessary." Simulation just confirms what the math guarantees.

Uniqueness of the answer: The problem statement guarantees at most one valid starting index when total ≥ 0. This can be proven: if two different starts both worked, the surplus at each would have to cover the deficit of the other, which (by pigeon-hole on the net sums) is impossible unless they are the same start.

4 The Code (Annotated) ▾

Solution 1 — Greedy One-Pass (Optimal, O(n), Interview Pick)

def canCompleteCircuit(gas, cost):
tank, total, start = 0, 0, 0 ← three accumulators: tank resets; total never resets; start = candidate index
for i in range(len(gas)): ← single pass through all n stations
net = gas[i] - cost[i] ← net fuel gain at station i (can be negative)
tank += net ← accumulate from current candidate start (local tank)
total += net ← accumulate global net (never reset — feasibility check)
if tank < 0: ← tank negative → candidate start + all intermediate indices are bad
start = i + 1 ← jump candidate start to i+1
tank = 0 ← reset local tank for the new candidate
return start if total >= 0 else -1 ← global check: total<0 → impossible; else start is the answer
⚠️ Why two separate variables (tank vs total)?

tank answers: "starting from my current candidate, can I keep driving?" It resets each time we move the start forward.
total answers: "does the whole circuit have enough fuel?" It accumulates all nets without ever resetting.
If you only track one variable, you conflate local reachability with global feasibility — you'd either wrongly reject a valid start or miss an impossible circuit.

Solution 2 — Brute Force O(n²) (for intuition building only)

def canCompleteCircuit_brute(gas, cost):
n = len(gas)
for start in range(n): ← try every station as starting point
tank, ok = 0, True
for step in range(n): ← simulate full n-station loop
i = (start + step) % n ← wrap around with modulo
tank += gas[i] - cost[i]
if tank < 0: ok = False; break
if ok: return start ← first valid start found
return -1
💡 Complexity Summary

Greedy One-Pass: Time O(n) — single iteration over n stations. Space O(1) — three scalar variables.
Brute Force: Time O(n²) — n candidate starts × n steps each. Space O(1).
The greedy approach is optimal since we must read each station at least once.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Brute Force and Observe

"The naive approach: try each of the n stations as a start, simulate the full loop — O(n²). But I notice we're redoing a lot of work. When starting at station 0 fails at station 2, we throw that away and restart from 1 — but we already know something about positions 0, 1, 2."

Step 2: Spot the Key Elimination

"If I start at position s and fail at position i, then any position j between s and i would face the same deficit at i but with a smaller head start. So I can skip the entire range [s, i] and jump directly to i+1."

Step 3: Design Two Accumulators

"I need two running sums: tank for the local reachability from my current candidate (resets on skip), and total for the global feasibility check (never resets). One pass. O(n)."

Step 4: Justify the Global Check

"At the end, total < 0 means the entire circuit has more cost than gas — impossible. If total ≥ 0, the suffix surplus from our chosen start covers the prefix deficit in the wrap-around, so the answer is start."

🗣️ What to Say to the Interviewer

"Brute force is O(n²): try all starts. We can do O(n) with a greedy insight:
when the running tank goes negative at i, no position from start to i can work,
so we jump start to i+1 and reset tank. We also track total (never reset)
to detect the globally impossible case. After one pass:
total < 0 → return -1; otherwise return start.
Time O(n), Space O(1)."
🔮 Likely Follow-up Questions
  • "Why is the answer guaranteed to be unique?" → If two valid starts existed, their surplus-deficit argument leads to contradiction.
  • "What if gas and cost could be negative?" → Algorithm still works — it operates on net sums, not absolute values.
  • "How is this related to Jump Game?" → Both use greedy "skip bad positions" — Jump Game tracks max reachable index; Gas Station tracks surplus.
  • "What if we need the minimum number of refueling stops?" → LC 871: extends to max-heap to retroactively pick the best past station to refuel at.

If sum(gas) < sum(cost), the circuit is impossible — return -1 immediately
When tank goes negative at i, skip the entire range [start, i] — all proven bad starts
Two accumulators: tank resets (local reachability); total never resets (global feasibility)

🎯 When to Recognize "Start Where Deficit Recovers"
  • Circular array with resource produced and consumed at each node
  • "Find a valid starting index" — not a fixed starting point
  • Global sum ≥ 0 guarantees a solution (hint the greedy argument applies)
  • Proof that failing at i eliminates all intermediate candidates
