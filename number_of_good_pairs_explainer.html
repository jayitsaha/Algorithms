Number of Good Pairs

LeetCode #1512 – Count matching value pairs in an array

Easy Hash Table Count Pairs O(n) Time O(n) Space
1 Understanding the Problem ▼

Given an array of integers nums, we need to count "good pairs" — pairs of indices (i, j) where:

  • nums[i] == nums[j] (same value)
  • i < j (i comes before j)

Example: For nums = [1, 2, 3, 1, 1, 3]

• Value 1 appears at indices 0, 3, 4 → pairs: (0,3), (0,4), (3,4) = 3 pairs
• Value 3 appears at indices 2, 5 → pairs: (2,5) = 1 pair
• Total: 4 good pairs

💡 Key Insight

When we encounter a value that's appeared k times before, we can form k new pairs with those previous occurrences. We don't need the formula — just count as we go!

If a value appears c times → c × (c-1) / 2 pairs
This is the combinations formula: choosing 2 items from c items (order doesn't matter)
2 The Algorithm: Running Count ▼

Instead of counting all occurrences first and using a formula, we use a clever running count approach:

1 Initialize a hash map to track how many times we've seen each value
2 For each number in the array: add its current count to result (pairs with previous occurrences)
3 Then increment that number's count in the hash map
4 Return the total result
🎯 Why "Add Count First, Then Increment"?

When we see a number, we want to pair it with all previous occurrences. So we add the count (how many times we've seen it before), then update the count for future numbers.

3 Why It Works: The Invariant ▼
🔐 The Invariant

At any point during iteration, count[num] represents exactly how many times we've seen num before the current position. This is the number of valid pairs we can form.

Why does adding counts give us the total pairs?

Consider a value that appears at positions [p₁, p₂, p₃, p₄]:

  • At p₁: count = 0 → add 0 pairs
  • At p₂: count = 1 → add 1 pair (with p₁)
  • At p₃: count = 2 → add 2 pairs (with p₁, p₂)
  • At p₄: count = 3 → add 3 pairs (with p₁, p₂, p₃)

Total: 0 + 1 + 2 + 3 = 6 pairs

This equals 4 × 3 / 2 = 6 — the formula for choosing 2 from 4!

💡 Connection to Math Formula

The sum 0 + 1 + 2 + ... + (n-1) equals n(n-1)/2. Our running sum naturally computes this without explicitly using the formula!

4 The Code (Annotated) ▼
# Count good pairs using running count approach
def numIdenticalPairs(nums):
count = {} # Value → how many times seen
result = 0 # Total good pairs
for num in nums:
# Add pairs with previous occurrences
result += count.get(num, 0)
# Increment count for this value
count[num] = count.get(num, 0) + 1
return result
⚠️ Common Mistake

Don't increment the count before adding to result! We want pairs with previous occurrences, not including the current element.

Complexity:

  • Time: O(n) — Single pass through the array
  • Space: O(n) — Hash map stores at most n distinct values
5 🎤 Interview Strategy ▼
Step 1: Recognize the Pattern

Keywords: "pairs", "count", "same value" → Think hash map to track occurrences. "i < j" tells us order matters but we're counting, not listing.

Step 2: Consider Brute Force First

O(n²) nested loops checking all pairs. Mention this to show you understand the problem, then optimize.

Step 3: The Optimization Insight

"Instead of checking all pairs, I can count as I go. Each new occurrence of a value can pair with all previous occurrences."

🗣️ What to Say to the Interviewer

"I'll use a hash map to count occurrences. When I see a value, I add its current count to my result — that's the number of pairs I can form with previous occurrences — then increment the count. This gives O(n) time and O(n) space."
🎯 Follow-up Questions to Expect
  • Alternative approach? — Formula-based: count all frequencies, then sum c*(c-1)/2
  • Space optimization? — If values are bounded (e.g., 1-100), use array instead of hash map
  • Extension: k-tuples? — For triplets, track pairs count, not just single count

7 Summary & Key Takeaways ▼
✓ "Count pairs" problems often use hash maps to track occurrences
✓ Running count: add count first, then increment
✓ Sum of 0+1+2+...+(n-1) = n(n-1)/2 naturally emerges
✓ One pass through array is enough — O(n) time
🎯 When to Use This Pattern
  • Counting pairs with some property (same value, sum to target)
  • Need O(n) time instead of O(n²) brute force
  • "How many pairs" rather than "list all pairs"
  • Each new element can pair with previous elements
