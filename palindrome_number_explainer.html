LeetCode #9

Palindrome Number

Given an integer x, return true if x is a palindrome, and false otherwise — without converting to a string.

Easy Mathematics & Geometry Reverse Half Time: O(log₁₀ n) Space: O(1)
1 Understanding the Problem ▾

A palindrome is a value that reads the same forward and backward. For numbers: 121 reads "1-2-1" left-to-right and "1-2-1" right-to-left — same! But 123 reads "1-2-3" vs "3-2-1" — different, so not a palindrome.

💡 Key Insight

A palindrome number's first half of digits mirrors its second half. We only need to reverse the second half and compare it to the first half — no string conversion, no overflow risk from reversing the full number.

✓ Palindrome: x = 1221

First half: 12
Reversed second half: 12
12 == 12 → True

✗ Not Palindrome: x = 123

First half: 1
Reversed second half: 32
1 ≠ 32 → False

Immediate edge cases:

  • Negative numbers — -121 reads "-121" forward but "121-" backward. The minus sign has no mirror. Always False.
  • Ends in zero (but isn't zero) — If a number ends in 0, its reverse would start with 0, which integers cannot have as a leading digit. Return False. Exception: x == 0 is a palindrome.
  • Single digit 0–9 — Trivially a palindrome (one digit is the same both ways).
  • Even vs Odd digit count — Handled naturally: even = exact match of halves; odd = match after dropping middle digit.
2 The Algorithm: Reverse Half the Number ▾

Instead of reversing all digits (which risks overflow for large numbers), we peel digits off the right of x and build a reversed second half, stopping as soon as the reversed portion is as large as the remaining x. Then compare the two halves.

1 Handle edge cases: If x < 0, return False. If x % 10 == 0 and x != 0, return False (trailing zero prevents valid palindrome).
2 Initialize reversed_half = 0. This will accumulate the digits peeled from the right of x, in reversed order.
3 Peel loop — while x > reversed_half: Extract the last digit of x with x % 10. Append it to reversed_half: reversed_half = reversed_half * 10 + x % 10. Shrink x: x //= 10.
4 Loop stops at midpoint: When reversed_half >= x, we've extracted exactly half the digits. reversed_half now holds those digits in reversed order.
5 Compare: Even-length → x == reversed_half. Odd-length → the middle digit sits in reversed_half's ones place; ignore it: x == reversed_half // 10. Return True if either matches.

For 1221: left half = 12, reversed right half = 12. Match → palindrome.

🔑 Why "Reverse Half" Beats "Reverse All"

Reversing the entire number can overflow for large values: e.g., reversing 2000000001 produces 1000000002 — that's fine here, but in general, the reversed form of a number close to INT_MAX would exceed 32-bit range. Reversing only half stays safely bounded — the reversed portion is at most half as many digits as the original.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every iteration: reversed_half contains exactly the digits peeled from the right of the original x, in reversed order. The loop condition x > reversed_half ensures we stop precisely at the midpoint — once the reversed portion "catches up" to the remaining portion in magnitude, we have processed exactly half the digits.

Proof for even-length (x = 1221, 4 digits):

  • Start: x=1221, reversed_half=0. (1221 > 0) → enter loop
  • Extract 1: reversed_half=1, x=122. (122 > 1) → continue
  • Extract 2: reversed_half=12, x=12. (12 == 12, not >) → stop
  • Compare: x (12) == reversed_half (12) → True

Proof for odd-length (x = 12321, 5 digits):

  • Start: x=12321, reversed_half=0. Enter loop
  • Extract 1: reversed_half=1, x=1232. Continue
  • Extract 2: reversed_half=12, x=123. Continue
  • Extract 3 (middle!): reversed_half=123, x=12. (12 < 123) → stop
  • Middle digit '3' is now in reversed_half's ones place. Drop it: 123 // 10 = 12
  • Compare: x (12) == reversed_half // 10 (12) → True
⚠️ The Trailing-Zero Edge Case

Consider x = 10. After one extraction: reversed_half = 0, x = 1. Loop stops (1 > 0). Compare: 1 == 0 (False) or 1 == 0 // 10 = 0 (False) → correctly False. The algorithm would actually handle trailing zeros correctly, but we add the early exit for clarity and to avoid the subtle case where 0 itself (a valid palindrome) would match the "trailing zero" check. The guard x != 0 preserves 0 as a valid palindrome while rejecting all other trailing-zero numbers immediately.

4 The Code (Annotated) ▾

Solution 1 — Reverse Half (Optimal, O(log₁₀ n) time, O(1) space) — Interview Pick

def isPalindrome(x: int) -> bool:
if x < 0 or (x % 10 == 0 and x != 0): ← Negative, or ends in 0 (but isn't 0): not a palindrome
return False
reversed_half = 0 ← Accumulates second half of digits in reversed order
while x > reversed_half: ← Loop until reversed portion catches up to remaining x
reversed_half = reversed_half * 10 + x % 10 ← Shift left + append last digit of x
x //= 10 ← Remove last digit from x (shrink from the right)
return x == reversed_half or x == reversed_half // 10
# x == reversed_half ← even-length: both halves exactly equal
# x == reversed_half // 10 ← odd-length: middle digit is in reversed_half's ones place; drop it
⚠️ Why reversed_half // 10 for Odd-Length Numbers?

For a 5-digit palindrome like 12321: when the loop exits, reversed_half = 123 and x = 12. The middle digit "3" is sitting in reversed_half's ones place because we extracted one too many digits to reach the stop condition. Integer-dividing by 10 removes it: 123 // 10 = 12. Then 12 == 12 → True. The middle digit doesn't need a mirror — a palindrome's center is always symmetric with itself.

Solution 2 — String Conversion (Simpler, but O(n) space)

def isPalindrome(x: int) -> bool:
if x < 0: return False ← Negative numbers can't be palindromes
s = str(x) ← Convert integer to string — O(n) space
return s == s[::-1] ← Python reverse slice and compare — very readable
💡 Complexity Comparison

Reverse Half (Solution 1): Time O(log₁₀ n) — processes only ⌊d/2⌋ digits where d = number of digits. Space O(1) — just one integer variable.
String Conversion (Solution 2): Time O(n), Space O(n) for the string and reversed copy. Simpler to read, acceptable when no constraint forbids string conversion.
