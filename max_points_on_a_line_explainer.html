LeetCode #149

Max Points on a Line

Given an array of points on a 2D plane, return the maximum number of points that lie on the same straight line.

Hard Math Count Slopes per Point Time: O(n²) Space: O(n)
1 Understanding the Problem ▾

You're given a list of 2D points like [[1,1],[2,2],[3,3],[0,2]]. Any set of points either lie on the same infinite straight line, or they don't. Your task: find the straight line that passes through the most given points, and return that count.

💡 Key Insight

Two points always define exactly one line. A third point lies on that same line if and only if the slope from point A to C equals the slope from point A to B. So: fix one "anchor" point, compute the slope to every other point, and count the most common slope — that count tells us how many points share a line through the anchor.

✓ Collinear: slopes match

A=(1,1), B=(2,2), C=(3,3)
slope(A→B) = (2-1)/(2-1) = 1
slope(A→C) = (3-1)/(3-1) = 1
All on y = x → 3 points

✗ Not collinear: slopes differ

A=(1,1), B=(2,2), D=(0,2)
slope(A→B) = 1
slope(A→D) = (2-1)/(0-1) = -1
D not on y=x line → skip

Edge cases to consider: Duplicate points (same coordinates) are collinear with every line — count them separately and add to each slope's total. If n ≤ 2, answer is n. If all points are identical, answer is n. Vertical lines (dx=0) are handled naturally by the reduced-fraction representation.

2 The Algorithm: Count Slopes per Anchor Point ▾

We try every point as an anchor. From each anchor we compute the slope to every later point (j > i), store slopes in a hash map, and find the maximum bucket size. The global answer is the max over all anchors.

1 Quick return: if n ≤ 2, return n — any 1 or 2 points trivially form a line.
2 Fix anchor at index i. Create a fresh slopes map {}, set dups = 0, local_max = 0.
3 For each j > i: compute dx = points[j][0] - points[i][0], dy = points[j][1] - points[i][1]. If both are 0 → duplicate, add to dups and continue.
4 Reduce fraction: divide dx, dy by gcd(|dx|, |dy|). Normalize sign so denominator is positive. Store (dy, dx) as the slope key. Increment its bucket.
5 Update answer: ans = max(ans, local_max + dups + 1). The +1 is the anchor itself; +dups adds points that lie on every line through the anchor.
🔐 Why Reduced Fractions, Not Floats?

Slopes like 2/4 and 1/2 represent the same direction, but as floats they might not compare equal due to IEEE-754 precision. Instead, reduce dy/dx by their GCD and store the integer tuple (dy//g, dx//g). This is exact and hashable. Always force the denominator positive so (-1, 2) and (1, -2) hash the same.

3 Why It Works: The Invariant ▾
🔐 The Invariant

For a fixed anchor A, points B and C lie on the same line through A if and only if slope(A→B) = slope(A→C). Grouping all points by their reduced-fraction slope from A, and taking the largest group + duplicates + 1 (anchor), gives the maximum collinear count through A.

Why we try every anchor: The maximum collinear set of k points must pass through each of those k points. When we process any one of them as anchor, we'll observe all k-1 others sharing the same slope, giving us a bucket of size k-1 → total k. So the global max is guaranteed to be found.

Why j > i suffices: The line through points i and j is the same regardless of direction. When anchor=0, we process pairs (0,1),(0,2),(0,3). When anchor=1, we need only (1,2),(1,3) — pair (0,1) was already covered. Using j > i halves the work with no loss of correctness.

⚠️ The Duplicate Point Trap

If points[i] == points[j] (same coordinates), there's no defined slope — they're the same location in space. These duplicates lie on every possible line through the anchor. Count them separately (dups) and add to every bucket's total at the end. Omitting this produces wrong answers on inputs with repeated points.

All-duplicates edge case: If every point is identical, the slope map stays empty, local_max=0, and we return 0 + (n-1) + 1 = n. Correct — all n identical points are trivially collinear.

4 The Code (Annotated) ▾
from math import gcd— need gcd to reduce fractions to lowest terms
 
def maxPoints(points):
n = len(points)— total number of points
if n <= 2: return n— 0,1,2 points: trivially collinear, return n
ans = 0— global best count across all anchors
 
for i in range(n): # try each point as anchor
slopes = {} # slope_key → count of points with that slope
dups = 0 # points identical to anchor[i]
local_max = 0 # best bucket size for this anchor
 
for j in range(i + 1, n): # only forward to avoid double-counting
dx = points[j][0] - points[i][0]— horizontal delta
dy = points[j][1] - points[i][1]— vertical delta
if dx == 0 and dy == 0: # duplicate point
dups += 1— lies on every line through anchor
continue
g = gcd(abs(dx), abs(dy))— reduce to lowest terms
dx //= g; dy //= g— e.g. (2,4) → (1,2)
if dx < 0: dx, dy = -dx, -dy— canonical sign: denominator always positive
key = (dy, dx) # hashable slope representation
slopes[key] = slopes.get(key, 0) + 1— increment this slope's bucket
local_max = max(local_max, slopes[key])— track largest bucket
 
# local_max other pts + dups + 1 (anchor) = total on best line
ans = max(ans, local_max + dups + 1)
 
return ans
⚠️ Sign Normalization — Critical Detail

The slope from A→B and from B→A is the same geometric line, but dx and dy flip signs. Without normalization, slope (1, -2) and slope (-1, 2) would land in different hash buckets even though they represent identical direction. Fix: if dx < 0, negate both — this forces the "denominator" (dx) to always be non-negative. Special case: if dx == 0 (vertical), we need dy to be positive — but gcd + sign of dy handles this naturally since we only flip when dx < 0.

📐 Complexity

Time: O(n²) — the outer loop runs n times; inner loop runs up to n-1 times. GCD is O(log(max_coord)), negligible. Total: O(n² log C).
Space: O(n) — the slopes dictionary holds at most n-1 entries per anchor (one entry per distinct direction).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Observe the Search Space

There are infinitely many lines in a plane, but every candidate line must pass through at least two of the n given points. So the real search space is O(n²) pairs — and O(n²) is tractable for n ≤ 300 (LeetCode constraint). Say this out loud to show you've bounded the problem.

Step 2: The Anchor Strategy

The hardest part is grouping points by line efficiently. Key insight: fix one point as anchor, compute slopes to all others, and group by slope. The best group size = collinear count through anchor. Since every optimal line passes through some point, trying all anchors guarantees we find it.

Step 3: Kill Floating-Point Errors Immediately

Interviewers expect you to address float precision proactively. "I'll store slopes as reduced integer fraction tuples (dy//g, dx//g) rather than floats, to avoid precision bugs." This is the single most important implementation detail — and the reason this is rated Hard.

Step 4: Handle Edge Cases Explicitly

Name them: (a) duplicate points — count separately, add to every slope; (b) vertical lines — dx=0, handled by the fraction naturally; (c) n ≤ 2 — trivial return; (d) all identical points — slope map empty, answer = n.

🗣️ What to Say to the Interviewer

"Any line in the optimal answer must pass through at least one of the n input points. So I'll try each point as an anchor — that's n iterations.

From each anchor, I compute the slope to every other point and group by slope using a hash map. The largest slope group gives the most collinear points through that anchor.

The tricky part: I can't use float slopes because of precision. Instead I'll store slopes as reduced fraction tuples using GCD, normalized so the denominator is always positive.

I also need to handle duplicate points — they have zero distance and lie on every line, so I count them separately and add at the end.

Final complexity: O(n²) time, O(n) space."
🎯 Follow-Up Questions to Expect
  • Can you do better than O(n²)? (No known general sub-quadratic algorithm.)
  • What if coordinates overflow int? (In Python, integers are arbitrary precision — no overflow.)
  • What about 3D points? (Track two slope components: dy/dx and dz/dx — same GCD approach.)
  • What if you must return the actual line, not just the count? (Store the anchor index and best slope key alongside the max count.)

7 Summary & Key Takeaways ▾
Fix each point as "anchor" in turn; group all other points by their slope from the anchor using a hash map.
Represent slopes as reduced integer fraction tuples (dy//gcd, dx//gcd) to avoid all floating-point precision bugs.
Normalize the sign of the fraction (force denominator positive) so directions like (1,-2) and (-1,2) hash identically.
Count duplicate points (same coordinates as anchor) separately and add to every slope bucket's count at the end.
Answer for each anchor = max slope bucket count + duplicates + 1 (anchor itself). Return the global max.
