Number of Matching Subsequences

LeetCode #792 – Count words that are subsequences of s

Medium Hash Map Bucket by First Char O(|s| + total_chars) Time
1Understanding the Problem▼

Given string s and array words, count how many words are subsequences of s.

Example: s="abcde", words=["a","bb","acd","ace"] → 3 (a, acd, ace match)

💡 Key Insight

Instead of checking each word independently (O(|s|) per word = slow), bucket words by their current needed character. When we see character c in s, only advance words waiting for c.

⚠️ Naive is too slow

Checking each word independently = O(|words| × |s|). Bucket approach = O(|s| + sum of word lengths) — single pass over s.

2The Algorithm: Bucket by First Char▼
1Create 26 buckets (one per letter). Put each word in the bucket matching its first character, storing an iterator/index pointer.
2Scan s character by character.
3For each char c: take all words from bucket[c], advance their pointer by 1.
4If word pointer reaches end → it's fully matched, increment count. Else move it to the bucket matching its new next character.
3The Code (Annotated)▼
from collections import defaultdict
def numMatchingSubseq(s, words):
buckets = defaultdict(list)
for w in words: # place each word in bucket of 1st char
buckets[w[0]].append((w, 1)) # (word, next_idx)
count = 0
for c in s:
waiting = buckets[c]
buckets[c] = []
for w, i in waiting:
if i == len(w): # fully matched
count += 1
else:
buckets[w[i]].append((w, i+1))
return count
4🎤 Interview Strategy▼
Key Observation

Instead of scanning s for each word, scan s once and push words forward lazily — only when their needed char appears.

🗣️ What to Say

"I'll bucket words by their current needed character. For each char in s, I take all words waiting for that char, advance them, and either count them as matched or re-bucket them. Single pass over s gives O(|s| + total word chars)."
