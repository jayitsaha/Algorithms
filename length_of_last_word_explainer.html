Length of Last Word

Reverse Iteration — Finding the Right Starting Point

#58 Easy Reverse Iteration String Processing O(n) Time · O(1) Space
1 Understanding the Problem

Given a string s consisting of words and spaces, return the length of the last word. A word is a maximal substring of non-space characters.

💡 Key Insight: Start from the End

Since we want the last word, starting from the end of the string is much more efficient than scanning from the beginning. We only need to:

  • Skip any trailing spaces
  • Count characters until we hit a space (or the beginning)
2 The Algorithm: Reverse Iteration

Instead of processing the entire string, we iterate from the end — finding exactly what we need with minimal work.

1 Start at the end: Set pointer i = len(s) - 1
2 Skip trailing spaces: Move i left while s[i] == ' '
3 Count the word: Move i left while s[i] != ' ', incrementing length
4 Return length: When we hit a space (or run out of string), return the count
🔑 Why Reverse Iteration?

By iterating backwards, we:

  • Avoid processing the entire string — we stop as soon as we've counted the last word
  • Handle trailing spaces elegantly — just skip them first
  • Need no extra space — unlike split() which creates a list
Approach Time Space Notes
Split + Last O(n) O(n) Simple but creates array of all words
Reverse Iteration O(n) O(1) Optimal — stops early, no extra space
3 Why It Works: The Two-Phase Approach
🔐 The Invariant

After Phase 1 (skip spaces): i points to the last character of the last word (or -1 if all spaces).

After Phase 2 (count word): length equals the number of characters from where we started counting back to (but not including) the preceding space.

PHASE 1: SKIP SPACES
Move left while s[i] is a space. This handles any trailing whitespace.
PHASE 2: COUNT WORD
Move left while s[i] is NOT a space. Each move = one more character in the word.
📝 Edge Cases Handled
  • Single word: "Hello" → Phase 1 does nothing (no trailing spaces), Phase 2 counts all 5
  • All spaces: " " → Phase 1 moves i to -1, length stays 0
  • Trailing spaces: "Hello " → Phase 1 skips 2 spaces, Phase 2 counts "Hello"
  • Leading spaces: " Hello" → Phase 1 does nothing, Phase 2 counts "Hello", stops at space
4 The Code (Annotated)
def lengthOfLastWord(s: str) -> int:
i = len(s) - 1 # Start from end
# Phase 1: Skip trailing spaces
while i >= 0 and s[i] == ' ':
i -= 1
# Phase 2: Count the last word
length = 0
while i >= 0 and s[i] != ' ':
length += 1
i -= 1
return length

✨ Python One-Liner (Alternative)

return len(s.rstrip().split()[-1]) if s.strip() else 0

This works but uses O(n) space due to split(). The reverse iteration is more efficient and shows deeper understanding.

⚠️ Why "i >= 0" checks come first?

Python evaluates conditions left-to-right with short-circuit logic. If i >= 0 is false, s[i] won't be evaluated — preventing an index-out-of-bounds error!

5 🎤 Interview Strategy: How to Arrive at This
Step 1: Clarify the Edge Cases

"Can there be trailing spaces? Leading spaces? Multiple spaces between words? Only spaces?" — This reveals the need to handle whitespace carefully.

Step 2: Consider Simple Approach First

"I could split the string and take the last element, but that uses O(n) extra space. Can I do better?"

Step 3: Optimize with Reverse Iteration

"Since I only need the LAST word, I can start from the end. First skip any trailing spaces, then count characters until I hit a space."

Step 4: Identify the Two Phases

"This naturally breaks into two while loops: (1) skip spaces, (2) count word."

🗣️ What to Say to the Interviewer
"To find the last word's length, I'll iterate from the end of the string. First, I skip any trailing spaces — this handles cases like 'Hello World '. Then I count characters until I hit another space or reach the start of the string. This gives O(n) time and O(1) space — better than split() which would use O(n) space."
🔄 Follow-up Questions
  • "What if the string has Unicode characters?" — Same approach works; Python handles Unicode natively.
  • "How would you find the length of the K-th word from the end?" — Count K word boundaries while iterating backwards.
  • "What about multiple consecutive spaces?" — Our approach handles this naturally.
