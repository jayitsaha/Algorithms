LeetCode #224

Basic Calculator

Implement a basic calculator to evaluate a string expression containing +, -, (, ), and spaces. No multiplication or division.

Medium Stacks Expression Evaluation Time: O(n) Space: O(n)
1 Understanding the Problem ▾

You are given a string like "1 + (2 - 3)" and must compute its integer value — without using Python's eval(). The expression contains non-negative integers, + and - operators, parentheses (), and spaces. No multiplication or division complicates precedence.

💡 Key Insight

Without parentheses, this is trivial — scan left to right, apply each operator. The challenge is that parentheses create nested sub-expressions that must be fully evaluated before combining with the outer expression. A stack is the natural tool: save your current state when you dive into a sub-expression, restore it when you surface.

✓ Easy: No Parentheses

"1 + 2 - 3"
result=0, sign=+1
+1 → result=1
+2 → result=3
−3 → result=0

✗ Hard: Nested Negation

"1 - (2 + 3)"
Inner = 2+3 = 5
The '-' NEGATES the inner!
1 − 5 = -4

The critical complication: When we see -(, the minus sign applies to the entire sub-expression, not just the first term inside. Deeply nested parentheses can flip signs multiple times. We need to propagate this sign context across nesting levels.

Edge cases to consider: multi-digit numbers (e.g. "123"), multiple consecutive spaces, unary minus at the very start (e.g. "- (3 + 4)"... actually LeetCode guarantees valid input so numbers are always non-negative in isolation), deeply nested parentheses like "((((1))))", and expressions with a trailing number like "1+2" (no operator after the last digit).

2 The Algorithm: Single-Pass Stack Evaluation ▾

We scan the input character by character, maintaining four variables. The stack saves and restores the outer context each time we cross a parenthesis boundary.

1 Digit: Build multi-digit numbers digit-by-digit. num = num × 10 + int(ch). Don't flush to result yet — wait until the number is complete (i.e., we see a non-digit next).
2 Operator + or -: The current number is now complete. Flush: result += sign × num; num = 0. Then record the new sign for the next term: sign = +1 for +, sign = -1 for -.
3 Open paren (: We're about to dive into a sub-expression. Push the outer context: stack.append(result), then stack.append(sign). Then reset result = 0; sign = 1 to start fresh inside.
4 Close paren ): Flush the last pending number: result += sign × num; num = 0. Then pop and apply: result *= stack.pop() (the outer sign), then result += stack.pop() (the outer running total).
5 Space: Simply skip. No else clause needed — do nothing.
6 After loop: Flush any trailing number: return result + sign × num. (The last number in the string has no operator after it, so we never flushed it during the loop.)

🔑 Why Push Both result AND sign at (?

When we push context at (, we need two pieces of information to correctly restore state at ): (1) the outer running total so far, and (2) the sign that should scale the inner sub-expression before adding it to the outer total. Pushing only one of them loses critical information.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any moment during parsing, result holds the fully evaluated sum of all completed terms at the current nesting level. The stack holds one (outer_result, outer_sign) pair per open parenthesis above the current level. When we close a parenthesis, the inner result is scaled by outer_sign and merged into outer_result.

Full trace of "1 - (2 + 3)":

Start: result=0, sign=+1, num=0, stack=[] '1': digit → num = 0*10+1 = 1 ' ': space → skip '-': operator → result = 0+(+1)*1 = 1; num=0; sign=-1 ' ': space → skip '(': push result=1, push sign=-1 → stack=[1,-1]; result=0, sign=+1 '2': digit → num = 0*10+2 = 2 ' ': space → skip '+': operator → result = 0+(+1)*2 = 2; num=0; sign=+1 ' ': space → skip '3': digit → num = 0*10+3 = 3 ')': close paren → flush: result = 2+(+1)*3 = 5; num=0 pop sign=-1: result = 5 * (-1) = -5 pop outer=1: result = -5 + 1 = -4 After loop: return -4 + (+1)*0 = -4 ✓

Why sign-flip propagates correctly: The sign recorded just before ( is stored on the stack. When the sub-expression finishes, we multiply the entire inner result by that sign. For -(-(1)): inner evaluates to 1, gets multiplied by inner's sign −1 → −1, then multiplied by outer sign −1 → +1. The double negation cancels, giving the correct answer 1.

⚠️ The Flush-Before-Action Rule

Any time you see a +, -, or ), you must flush num into result first. Multi-digit numbers accumulate across multiple character iterations, and you only know a number is complete when you hit a non-digit. Forgetting to flush is the single most common bug in student implementations. It silently drops the last digit group.

4 The Code (Annotated) ▾

Solution 1 — Stack Iterative (Interview Pick, O(n) time, O(n) space)

def calculate(s: str) -> int:
stack = [] ← stores (result, sign) pairs at each '('
result = 0 ← running sum at the current nesting level
sign = 1 ← next operator: +1=add, -1=subtract
num = 0 ← accumulates the current multi-digit integer
for ch in s: ← one character at a time
if ch.isdigit(): ← build multi-digit number
num = num * 10 + int(ch) ← '1','2','3' → 1,12,123
elif ch in '+-': ← operator: flush num, record new sign
result += sign * num ← FLUSH the completed number into result
num = 0 ← reset for next number
sign = 1 if ch == '+' else -1 ← record the operator as ±1
elif ch == '(': ← entering sub-expression
stack.append(result) ← save outer running total
stack.append(sign) ← save sign that applies to this sub-expression
result = 0 ← fresh start inside parens
sign = 1 ← first term inside is positive by default
elif ch == ')': ← closing sub-expression
result += sign * num ← FLUSH last number inside parens
num = 0
result *= stack.pop() ← pop outer sign, apply to inner result
result += stack.pop() ← pop outer total, add back
# ch == ' ': implicit skip (no else) ← spaces need no action
return result + sign * num ← flush any trailing number after the loop
⚠️ Pop Order: Sign First, Then Result

We push result first, then sign (so sign is on top). At ) we pop in reverse: stack.pop() gives sign (multiply), then stack.pop() gives result (add). If you push in the opposite order or pop in the wrong order, you silently compute the wrong answer — no error, just a wrong number.

Solution 2 — Recursive Descent (Clean mental model, same complexity)

def calculate(s: str) -> int:
i = 0
def parse() -> int:
nonlocal i
result, sign = 0, 1
while i < len(s):
ch = s[i]; i += 1
if ch.isdigit():
num = int(ch)
while i < len(s) and s[i].isdigit():
num = num * 10 + int(s[i]); i += 1
result += sign * num
elif ch == '+': sign = 1
elif ch == '-': sign = -1
elif ch == '(': result += sign * parse() ← recurse for sub-expression
elif ch == ')': break ← return sub-expression result to caller
return result
return parse()
💡 Complexity Summary

Stack iterative (Solution 1): Time O(n), Space O(n) — stack depth bounded by nesting depth
Recursive descent (Solution 2): Time O(n), Space O(n) — call stack depth = nesting depth. May hit Python's recursion limit (default 1000) for extremely deeply nested inputs. The iterative version is safer in production.
Both are optimal — we must read every character at least once.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

The moment you see parentheses in an expression, think stack. Parentheses create a "push/pop" structure — you save context on ( and restore it on ). Also note: only + and −, so there's no operator-precedence complexity (no × or ÷ to worry about).

Step 2: Define the State Variables

Ask: "What information do I need at each moment?" Four things: result (running total at current depth), sign (±1 for the next term), num (current integer being assembled), stack (saved context per open paren). Mention all four to the interviewer — it shows methodical thinking.

Step 3: Derive the Push/Pop Logic

Walk the interviewer through a small example: "1-(2+3)". Show that when we hit ( with sign=-1, we must save both result=1 and sign=-1 so we can later compute 1 + (-1)*(2+3) = -4. The push order and pop order must match.

Step 4: Call Out Common Bugs

Proactively mention: (1) Flush num before processing any operator or ). (2) Flush trailing num after the loop. (3) Multi-digit numbers need num = num*10 + digit. Interviewers love candidates who anticipate bugs rather than discover them during testing.

🗣️ What to Say to the Interviewer

"The key challenge is parentheses — they create nested sub-expressions that must evaluate completely
before combining with the outer expression. I'll use a stack to save/restore context at boundaries.
My state: result (running total), sign (+1/-1), num (current integer), stack.
On digit: build num with num*10+d. On operator: flush num to result, record sign.
On '(': push (result, sign), reset both. On ')': flush num, pop sign*inner + outer.
Trailing number: handled by return result + sign*num after the loop.
Time O(n), Space O(n). Let me trace '1-(2+3)' to verify the push/pop logic..."

7 Summary & Key Takeaways ▾
Parentheses = nesting = stack. Each ( pushes context; each ) pops and merges.
Push BOTH result AND sign at every (. Pop sign first, then result at ).
Always flush num before processing any operator or close-paren.
Multi-digit numbers: num = num * 10 + int(ch) accumulates across loop iterations.
Trailing number: return result + sign * num after the loop handles the last term.
Spaces: skip silently — no else clause needed.
