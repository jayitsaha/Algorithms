Split Array into Consecutive Subsequences

LeetCode #659 – Check if array can be split into consecutive subsequences of length ≥ 3

Medium Greedy Two Maps O(n) Time O(n) Space
1Understanding the Problem▼

Given a sorted integer array, determine if it can be split into one or more subsequences where each subsequence consists of consecutive integers and has length ≥ 3.

Example: [1,2,3,3,4,5] → [1,2,3] and [3,4,5] ✅

💡 Key Insight

Use two hash maps: freq (how many times each number is available) and tails (how many subsequences end at each number). For each number, greedily append to an existing sequence if possible, otherwise start a new one.

2The Algorithm: Greedy + Two Maps▼
1Build freq map: frequency of each number
2Build tails map: # of open subsequences ending at each value
3For each number: if tails[num-1] > 0, append to that subsequence (decrement tails[num-1], increment tails[num])
4Else if freq[num+1] > 0 and freq[num+2] > 0, start a new subsequence (num, num+1, num+2)
5Else return False
⚠️ Why append first?

Always prefer appending to existing sequences before starting new ones. Starting a new sequence uses up 3 elements; extending an existing one uses just 1. Greedy: preserve resources for harder-to-satisfy constraints.

3Why It Works▼
🔐 The Invariant

tails[v] = number of valid (length ≥ 1 so far) subsequences whose last element is v. Appending to them is always better than or equal to starting fresh — so greedy is optimal here.

If at any point we can neither append nor start a new 3-element chain, it's impossible — return False immediately.

4The Code (Annotated)▼
from collections import Counter
def isPossible(nums):
freq = Counter(nums)
tails = Counter() # seqs ending at key
for num in nums:
if freq[num] == 0: continue
if tails[num-1] > 0: # append
tails[num-1] -= 1
tails[num] += 1
elif freq[num+1] > 0 and freq[num+2] > 0: # new
freq[num+1] -= 1
freq[num+2] -= 1
tails[num+2] += 1
else: return False
freq[num] -= 1
return True
5🎤 Interview Strategy▼
Key Observation

Two maps: freq tracks availability, tails tracks open sequences. Greedy: extend before creating.

🗣️ What to Say

"I'll use two counters — freq for available numbers, tails for open subsequence endpoints. For each number, I greedily extend an existing sequence if possible, else I try to start a new 3-element chain. If neither works, return false."
🎯 Follow-ups
  • What if subsequence length must be exactly k? — Same pattern, track tails of length k-1
  • What if unsorted? — Sort first O(n log n), or use different approach
