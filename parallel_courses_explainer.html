LeetCode #1136

Parallel Courses

Given n courses and prerequisite pairs, return the minimum number of semesters to complete all courses (each semester you can take any number of courses whose prerequisites are met), or -1 if impossible due to a cycle.

Medium Graph Topological Sort BFS Level by Level Time: O(V+E) Space: O(V+E)
1 Understanding the Problem ▾

You have n university courses labeled 1 to n. Some courses have prerequisites: relations[i] = [prevCourse, nextCourse] means prevCourse must be completed before nextCourse. In each semester, you can take any number of courses simultaneously, as long as all their prerequisites are already done. What is the minimum number of semesters to finish everything?

💡 Key Insight

The courses form a directed acyclic graph (DAG). The minimum number of semesters equals the length of the longest chain of dependencies. You can parallelize independent courses, but sequential dependencies force you to wait. BFS level-by-level finds these layers naturally — each BFS level is exactly one semester.

✓ Valid: n=3, relations=[[1,3],[2,3]]

(3 needs both 1 and 2 first)
Semester 2: take 3
Answer: 2 semesters

✗ Cycle: n=2, relations=[[1,2],[2,1]]

Course 1 needs course 2
Course 2 needs course 1
Deadlock! Neither can start.
Answer: -1 (impossible)

Edge cases to consider: (1) No relations at all — answer is 1 (all courses in one semester). (2) A single long chain 1→2→3→...→n — answer is n (no parallelism possible). (3) A cycle — return -1. (4) Isolated courses (no edges) automatically count as semester 1.

🔑 Real-World Analogy

Think of a college course catalog. You want to graduate as fast as possible by front-loading every course you're eligible to take each semester. This greedy strategy of "take everything available now" is exactly what BFS level-by-level implements — and it's provably optimal.

2 The Algorithm: Kahn's BFS Level by Level ▾

Kahn's Algorithm is a BFS-based topological sort. We track each node's in-degree (number of unmet prerequisites). Nodes with in-degree 0 are ready. After processing them, we reduce in-degrees of their successors. This naturally groups courses into layers — each layer is a semester.

1 Build the graph: For each relation [prev, next], add next to graph[prev]'s list (prev unlocks next), and increment indegree[next].
2 Seed the queue: Find all courses with indegree 0 (no prerequisites). These are the courses available in semester 1 — add them all to the BFS queue.
3 BFS level by level: Each iteration of the outer while loop is one semester. Process all current queue entries (snapshot size first!), incrementing semesters each time.
4 Unlock next courses: For each course taken, decrement the in-degree of all courses that depend on it. Any that reach 0 join the queue for next semester.
5 Cycle check: After BFS ends, if taken < n, some courses were never reachable (trapped in a cycle). Return -1.

3 Why It Works: The Invariant & Algorithm Deep-Dive ▾
🔐 The Invariant

At the start of each BFS level (semester), the queue contains exactly all courses whose prerequisites have been completely satisfied. This invariant is established at initialization (all indegree-0 courses) and maintained by each step (we only add to queue when indegree hits 0).

Kahn's Algorithm — Deep Dive

Kahn's Algorithm (published 1962 by Arthur B. Kahn) computes a topological ordering of a directed graph using BFS. Its core invariant: a node is processed only after all its predecessors have been processed. For acyclic graphs, this always terminates with all nodes processed. For cyclic graphs, at least one node in each cycle can never reach in-degree 0.

Why BFS and not DFS for this problem? DFS topological sort gives you an ordering, but counting the "depth" (minimum levels) is awkward. BFS level-by-level makes levels explicit — each queue snapshot is exactly one level. This maps perfectly to "minimum semesters".

Greedy optimality: Taking every available course each semester is provably optimal. Delaying any available course can only push later courses back — never forward. So the greedy "take all available" strategy achieves the minimum.

⚠️ The Critical Detail: Snapshot the Queue Size

Inside the while queue loop, you must capture level_size = len(queue) before iterating. If you use a simple while queue: course = queue.popleft(), you'll mix courses from different semesters into the same iteration — breaking the level count.

Why does total_taken < n detect a cycle? In a DAG, every node eventually has all its predecessors processed, so every node reaches in-degree 0 and gets queued. In a graph with a cycle, every node in the cycle has at least one predecessor also in the cycle — no node in the cycle ever reaches in-degree 0. They stay permanently frozen.

4 The Code (Annotated) ▾

Solution 1 — BFS Kahn's Algorithm (Optimal — Interview Pick)

from collections import deque← deque for O(1) popleft in BFS
def minNumberOfSemesters(n, relations):
indegree = [0] * (n + 1)← indegree[c] = number of unsatisfied prerequisites for course c
graph = [[] for _ in range(n + 1)]← graph[c] = list of courses that require course c
for prev, nxt in relations:
graph[prev].append(nxt)← prev → nxt means "taking prev unlocks nxt"
indegree[nxt] += 1← nxt has one more dependency to satisfy
queue = deque(c for c in range(1, n+1) if indegree[c] == 0)← seed: all courses with no prerequisites
semesters, taken = 0, 0← semesters = level count; taken = cycle detection counter
while queue:← each outer iteration = one semester
semesters += 1← starting a new semester
for _ in range(len(queue)):← *** CRITICAL: snapshot size to process exactly this level ***
course = queue.popleft()← take this course this semester
taken += 1← count it toward cycle detection
for nxt in graph[course]:← notify all courses that depended on this one
indegree[nxt] -= 1← one fewer prerequisite remaining
if indegree[nxt] == 0:← all prerequisites met!
queue.append(nxt)← enqueue for next semester
return semesters if taken == n else -1← taken < n means cycle exists
⚠️ Common Mistake

Forgetting for _ in range(len(queue)) and using a single while loop instead. This mixes semester boundaries together — you'd process all courses before counting semesters, losing the level structure entirely.

Solution 2 — DFS Topological Sort (Alternative)

DFS can also compute a topological order, but counting the minimum "levels" requires computing the longest path to each node — this is a separate O(V+E) pass after DFS. BFS is cleaner for this problem because levels are explicit.

def minNumberOfSemesters_dfs(n, relations):
graph = [[] for _ in range(n + 1)]
for prev, nxt in relations: graph[prev].append(nxt)
depth = [0] * (n + 1)← depth[c] = min semester for course c
visited = [0] * (n + 1)← 0=unvisited, 1=in-stack (cycle!), 2=done
def dfs(u):
if visited[u] == 1: return float('inf')← cycle: return infinity
if visited[u] == 2: return depth[u]
visited[u] = 1
depth[u] = 1 + max((dfs(v) for v in graph[u]), default=0)
visited[u] = 2; return depth[u]
result = max(dfs(c) for c in range(1, n+1))
return -1 if result == float('inf') else result
💡 Complexity Analysis

BFS Kahn's — Time: O(V+E) — Every course is enqueued and dequeued once (V). Every edge is examined once when the source course is processed (E).
Space: O(V+E) — Graph adjacency list (E entries) + indegree array (V) + queue (at most V).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

The keywords "prerequisite", "before", "dependency", and "parallel" together signal a topological sort on a DAG. When the question asks for the minimum number of rounds/levels/semesters, it's specifically BFS-based topo sort — because BFS gives you levels naturally, whereas DFS does not.

Step 2: Verify it's a DAG Problem

Ask yourself: do we have directed relationships? Can there be cycles (which would make it impossible)? If yes and we need to detect them, Kahn's algorithm is perfect — cycle detection comes for free via the taken-vs-n check at the end.

Step 3: Build the Template

Kahn's has a fixed template: (1) build adjacency list + indegree array, (2) seed queue with indegree-0 nodes, (3) BFS with for _ in range(len(queue)) inner loop, (4) check taken == n. Memorize this template — it applies to 10+ LeetCode problems.

Step 4: State Complexity

Every node is processed once, every edge is examined once — clearly O(V+E) time. Space is O(V+E) for the graph plus O(V) for the queue and indegree array.

from collections import deque
def minNumberOfSemesters(n, relations):
indegree = [0] * (n + 1)
graph = [[] for _ in range(n + 1)]
for prev, nxt in relations:
graph[prev].append(nxt); indegree[nxt] += 1
queue = deque(c for c in range(1,n+1) if indegree[c]==0)
semesters, taken = 0, 0
while queue:
semesters += 1
for _ in range(len(queue)):
course = queue.popleft()
taken += 1
for nxt in graph[course]:
indegree[nxt] -= 1
if indegree[nxt] == 0: queue.append(nxt)
return semesters if taken == n else -1

7 Summary & Key Takeaways ▾
In-degree tracks unmet prerequisites. When it hits 0, all dependencies are satisfied — the course is ready.
BFS level-by-level makes semesters explicit: each queue snapshot processed together = one semester.
Greedy optimality: always taking every available course this semester minimizes total semesters. Waiting never helps.
Cycle detection is free: after BFS, if taken < n, some courses were permanently blocked in a cycle.
The critical implementation detail: use for _ in range(len(queue)) to process exactly one level at a time.
🎯 When to Use BFS Topological Sort (Kahn's)
  • "X must come before Y" → dependency graph → topological sort
  • Need minimum rounds/levels/semesters → BFS (not DFS) for natural level structure
  • Need to detect cycles in directed graph → Kahn's "taken vs n" check
  • Processing in "waves" where each wave unlocks the next
