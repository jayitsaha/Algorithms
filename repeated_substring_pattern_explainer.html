LeetCode #459

Repeated Substring Pattern

Given a string s, check if it can be constructed by taking a substring of it and appending multiple copies of that substring together.

Easy String Processing KMP Failure Function Time: O(n) Space: O(n)
1 Understanding the Problem ▾

We need to determine whether string s can be built by repeating some shorter substring at least twice. In other words: is s a "tile" made entirely of identical smaller tiles?

💡 Key Insight

If s is built from repeated copies of a unit of length p, then p must divide len(s) evenly, and s[:p] tiled n/p times must exactly reconstruct s. The KMP failure function reveals this period in O(n) time without trying every divisor explicitly.

✓ True: s = "abcabcabc"

Unit = "abc" (len=3)
"abc" × 3 = "abcabcabc" ✓
3 divides 9 → True

✗ False: s = "aba"

"a" → "aa" ≠ "aba"
"ab" → can't divide 3 evenly
No valid period → False

Key structural observations:

  • The repeating unit must divide len(s) — so only try divisors of n
  • The unit must repeat at least twice — so unit length < n
  • Single-character strings are always False (can't repeat twice without duplicating)
  • All-same strings like "aaaa" are True — the unit "a" repeats 4 times
2 The Algorithm: Two Elegant Approaches ▾

Approach 1 — The s+s Concatenation Trick (Interview Pick)

If s is periodic with period p, then in the doubled string s+s, there exists a rotation of s (shifted by p) that exactly recovers s. We strip off the first and last character of s+s to prevent trivially matching at position 0 and position n.

s = "abab", n = 4
s + s  =  aababb  ← s found at index 1 (not 0, not 4)
(s+s)[1:-1] = "ababab"  →  "abab" in "ababab" = True ✓
1 Form doubled = (s + s)[1:-1] — concatenate s with itself, then strip off the first and last character to prevent trivial self-matches.
2 Check s in doubled — if s appears anywhere in the stripped double, a valid period exists. Python's in uses an O(n) optimised search internally.
3 Return True/False — found means periodic, not found means no period. One line!

Approach 2 — KMP Failure Function (shows algorithmic depth)

The KMP failure function (lps array) computes the longest proper prefix of each prefix that is also a suffix. Its final value encodes the period of the string directly.

1 Build lps[0..n-1]: For each index i, store the length of the longest proper prefix of s[0..i] that is also a suffix. Use two pointers: i (current position) and length (current border length).
2 Read the period: L = lps[n-1]. The minimal period length is n - L. If L > 0 and n % (n - L) == 0, then s is perfectly tiled by that period.
3 Return the boolean — both conditions needed: a non-trivial border (L > 0) and even division (n % period == 0).
🔑 Why Both Approaches Are Equivalent

If s has period p, then in s+s, the string s appears at position p (not at 0 or n). Conversely, if s appears in (s+s)[1:-1], that position gives us a valid period. The KMP failure function computes this period directly from the prefix-suffix structure of the string. They are provably equivalent — two different lenses on the same mathematical fact.

3 Why It Works: The KMP Failure Function Invariant ▾
🔬 Algorithm Deep-Dive: KMP Failure Function

Origin: Knuth, Morris, and Pratt (1977). The failure function was invented to speed up pattern matching — after a mismatch, instead of backtracking all the way, we "fail" to the longest prefix we know still matches. But it also encodes a rich periodicity structure in the string.


Core invariant: lps[i] = length of the longest proper prefix of s[0..i] that is also a suffix of s[0..i]. "Proper" means strictly shorter than the string itself.


How it reveals repetition: If lps[n-1] = L, then the string has a "border" of length L — a prefix that equals a suffix. The "minimal period" is n - L. When n % (n-L) == 0, the entire string is tiled by that period.


When to recognize this algorithm: Any problem asking about periodicity, shortest repeating unit, pattern search, or prefix-suffix overlap. The key signal is "does some prefix also appear as a suffix?" — that's exactly what lps stores.

Worked example — s = "ababab" (n=6):

i 012345
s[i] ab ab ab
lps[i] 00 12 34

lps[-1] = 4 → period = 6 - 4 = 2 → unit = "ab". Since 6 % 2 == 0 → True ✓

🔐 The Invariant (Proof Sketch)

At every step of building lps, we maintain: lps[i] is the longest border of s[0..i]. The fallback step length = lps[length-1] works because if s[0..length-1] is a border of s[0..i-1] and s[i] ≠ s[length], we need the next-longest border — which is lps[length-1], already computed. This amortized argument gives O(n) total: the variable length can increase at most n times and the fallback can decrease it at most n times total.

⚠️ Common Mistake: lps[-1] > 0 Is Not Enough

You also need n % (n - lps[-1]) == 0. Example: "abaab" has lps[-1]=2, so period candidate = 3, but 5 % 3 ≠ 0 — so "abaab" is NOT a repeated pattern despite having a non-zero border. The divisibility check is mandatory.

The fallback step demystified: When s[i] ≠ s[length] and length > 0, we don't give up — we try the next-shorter border of the current candidate. This is what makes KMP O(n) instead of O(n²): we never re-compare characters we already know match. The key insight is that the next border of a border is lps[length-1].

4 The Code (Annotated) ▾

Solution 1 — s+s Trick (Interview Pick, elegant one-liner)

def repeatedSubstringPattern(s: str) -> bool:
return s in (s + s)[1:-1] ← Concat, strip endpoints, check if s is inside
🔑 Why [1:-1]?

In s+s, the string s trivially appears at index 0 (the first copy) and index n (the second copy). By removing the first and last character of s+s, we eliminate both trivial positions. Any remaining occurrence of s in the trimmed string must be a "rotated" copy — which means it corresponds to a valid repeating period.

Solution 2 — KMP Failure Function (O(n) with explicit proof)

def repeatedSubstringPattern(s: str) -> bool:
n = len(s) ← Store length for repeated use
lps = [0] * n ← Failure function; lps[i] = longest proper prefix-suffix of s[0..i]
length = 0 ← Length of the current prefix-suffix border being extended
i = 1 ← Start from 1: lps[0] = 0 always (no proper prefix exists)
while i < n:
if s[i] == s[length]: ← Characters match: we can extend the current border
length += 1 ← Grow the border by one
lps[i] = length ← Record the longest border at position i
i += 1 ← Move to next character
elif length > 0: ← Mismatch, but we had a partial border: fall back smartly
length = lps[length-1]← KEY: next-longest border uses already-computed info (O(n) total)
else: ← Mismatch with length=0: no border possible at i
lps[i] = 0 ← Explicitly set (already 0, but clear for readability)
i += 1 ← Advance to next character
L = lps[-1] ← Longest border of the entire string
period = n - L ← Candidate minimal repeating unit length
return L > 0 and n % period == 0 ← Border exists AND period divides n evenly

Solution 3 — Brute Force (mention as baseline, O(n²))

def repeatedSubstringPattern(s: str) -> bool:
n = len(s) ← Try every possible period length
for p in range(1, n // 2 + 1):← Period must be at most n//2 (repeats at least twice)
if n % p == 0: ← Period must divide n evenly
if s[:p] * (n // p) == s: ← Tile the period; compare to original — O(n) each
return True
return False ← No divisor worked
💡 Complexity Summary

Brute Force: Time O(n × d(n)) where d(n) = number of divisors. Worst case O(n√n), practically near O(n log n). Space O(n) for the tile string.
s+s Trick: Time O(n) (Python string search internally uses KMP or Boyer-Moore). Space O(n) for the doubled string.
KMP Explicit: Time O(n) (amortized — length increases at most n times and falls back at most n times total). Space O(n) for lps array.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Periodicity Problem

The keyword is "repeated substring." This signals periodicity — the string has a repeating tile structure. Your brain should fire: "divisors of n, KMP failure function, or string rotation trick."

Step 2: Propose the Brute Force Baseline

"I can try every divisor p of n from 1 to n//2. For each, check if s[:p] repeated n/p times equals s. This is O(n × number_of_divisors) — correct but not optimal."

Step 3: Offer the Elegant s+s Trick

"There's a beautiful one-liner: return s in (s+s)[1:-1]. If s is periodic, there exists a rotation of s (by the period) inside s+s. Stripping the endpoints removes trivial self-matches."

Step 4: Impress with KMP (if asked to go deeper)

"For an O(n) solution with a formal proof: build the KMP failure function. lps[-1] gives the longest border; the period is n - lps[-1]. If L > 0 and n divides evenly by the period, we have a repeated pattern. This directly follows from the prefix-suffix structure."

7 Summary & Key Takeaways ▾
A repeated-pattern string appears as a rotation in s+s (excluding endpoints) — this is the one-line interview answer
KMP lps[-1] = L encodes the period directly: period length = n - L

The KMP fallback step (length = lps[length-1]) is the non-obvious insight: it reuses already-known prefix info for O(n) amortized performance
