LeetCode #1186

Maximum Subarray Sum with One Deletion

Return the maximum sum of a non-empty subarray with at most one element deleted. An element can only be deleted once.

Medium Kadane's Algorithm Forward + Backward Kadane Time: O(n) Space: O(n)
1 Understanding the Problem ▾

Classic Maximum Subarray (LeetCode #53) asks for the highest-sum contiguous subarray. This variant adds one powerful twist: you may optionally remove one element from your chosen subarray to boost the sum. The subarray must remain non-empty after any deletion.

Think of it this way: you're looking at a streak of numbers and you can "white out" one value if it's dragging the total down. The question is — which one element (if any) should you erase?

💡 Key Insight

If we delete element at position i, the resulting subarray is split into a left piece ending at i-1 and a right piece starting at i+1. The maximum of each piece is exactly what Kadane's algorithm gives us when run forward (left piece) and backward (right piece). Precomputing both in O(n) lets us evaluate every possible deletion in one more O(n) sweep.

Example: arr = [1, -2, 3, 4, -1, 3]

No deletion: [3, 4, -1, 3] = 9
Delete -2 (idx 1): [1, 3, 4, -1, 3] = 10
Delete -1 (idx 4): [3, 4, 3] = 10
→ Answer: 10

Edge: arr = [-1] (single element)

Edge cases to watch: all-negative array (answer = max single element, deletion cannot help), single-element array (return that element, no deletion possible), deletion at endpoint (covered by standard Kadane's result), and very large negative element in the middle separating two large positive runs.

2 The Algorithm: Forward + Backward Kadane ▾

We run Kadane's algorithm twice — once left-to-right to build fwd[], once right-to-left to build bwd[]. Each array precomputes the best subarray ending (or starting) at every index. Then one more sweep considers every possible deletion by bridging fwd[i-1] and bwd[i+1].

1 Forward Kadane — build fwd[]: fwd[i] = max subarray sum ending exactly at index i. Base: fwd[0] = arr[0]. Transition: fwd[i] = max(arr[i], fwd[i-1] + arr[i]) — either restart at arr[i] alone, or extend the previous best.
2 Backward Kadane — build bwd[]: bwd[i] = max subarray sum starting exactly at index i. Base: bwd[n-1] = arr[n-1]. Transition: bwd[i] = max(arr[i], bwd[i+1] + arr[i]) — symmetric to forward, but scanning right-to-left.
3 Initialize answer — no-deletion case: ans = max(fwd). This is exactly standard Kadane's on the full array. We must include this because deletion is optional — sometimes no deletion maximizes the sum.
4 Deletion sweep — try every interior index: For i in range(1, n-1), compute fwd[i-1] + bwd[i+1]. This is the sum you'd get by deleting arr[i] and joining the best left and right segments. Update ans = max(ans, fwd[i-1] + bwd[i+1]).
5 Return ans. The answer is the maximum over the no-deletion case and all single-element deletion cases.

🔑 Why Two Separate Kadane Arrays?

A single left-to-right Kadane can track whether we've used our deletion (a 2-state DP). But the forward+backward approach is easier to reason about: "fwd[i-1] is the best left segment, bwd[i+1] is the best right segment, we delete whatever is in between." Both approaches are O(n); choose whichever you can explain most clearly in an interview.

3 Why It Works: The Invariant ▾
🔐 The Core Invariant (Kadane's)

After the forward pass: fwd[i] = maximum sum of any subarray whose last element is arr[i]. After the backward pass: bwd[i] = maximum sum of any subarray whose first element is arr[i]. These are canonical Kadane invariants applied in opposite directions.

Why the bridge formula is correct: Any subarray with element i deleted has the form arr[l..i-1] + arr[i+1..r] for some l ≤ i-1 and r ≥ i+1. The left piece's maximum over all l is exactly fwd[i-1] (by invariant). The right piece's maximum over all r is exactly bwd[i+1]. So fwd[i-1] + bwd[i+1] is the best possible subarray that bridges the gap at i. Sweeping all interior i covers every single deletion.

Why endpoints are excluded from the sweep: Deleting index 0 leaves subarray arr[1..r], whose max is bwd[1]. But bwd[1] ≤ max(bwd) which could be captured via fwd anyway. More precisely, bwd[1] as a standalone subarray is already covered by max(fwd) (Kadane's can choose any subarray, including one starting at index 1). Same argument for deleting the last element.

⚠️ Most Common Bug: Skipping the No-Deletion Case

The problem says "at most one deletion" — zero deletions is valid! Always initialize ans = max(fwd) before the sweep. If you only do the deletion sweep, you'll get wrong answers when the array has no negative interior elements and any deletion would reduce the sum (e.g., arr = [5, 3, 2] → answer 10, but every deletion reduces it to at most 8).

All-negatives correctness: Kadane's restarts (fwd[i] = arr[i]) whenever extending hurts. With all negatives, every fwd[i] equals arr[i]. max(fwd) picks the largest (least-negative) single element. Deletion sweep: fwd[i-1] + bwd[i+1] connects two negatives, producing something even more negative. So ans correctly remains the least-negative single element.

4 The Code (Annotated) ▾

Solution 1 — Forward + Backward Kadane (Optimal, interview-safe)

def maximumSum(arr): ← main function
n = len(arr) ← cache length
fwd = [0] * n ← fwd[i] = best subarray sum ending at i
fwd[0] = arr[0] ← base case: only one element
for i in range(1, n): ← forward Kadane's pass, left → right
fwd[i] = max(arr[i], fwd[i-1] + arr[i]) ← restart or extend
bwd = [0] * n ← bwd[i] = best subarray sum starting at i
bwd[-1] = arr[-1] ← base case: last element only
for i in range(n-2, -1, -1): ← backward Kadane's, right → left
bwd[i] = max(arr[i], bwd[i+1] + arr[i]) ← restart or extend rightward
ans = max(fwd) ← no-deletion case = standard Kadane result
for i in range(1, n-1): ← try deleting each interior element
ans = max(ans, fwd[i-1] + bwd[i+1]) ← bridge: best left + best right
return ans
⚠️ Why range(1, n-1) — not 0 to n-1?

If we delete index 0, the best we can do is bwd[1] — but that's a subarray starting at index 1, which is already considered in max(fwd). If we delete index n-1, the best is fwd[n-2] — also captured. Only interior deletions (1 to n-2) can genuinely bridge a left half and a right half that wouldn't naturally appear in standard Kadane's output.

Solution 2 — State Machine DP (O(1) space, interview pick for space)

def maximumSum(arr):
# dp0: best sum ending here, 0 deletions used
# dp1: best sum ending here, 1 deletion used
dp0 = dp1 = arr[0] ← both states init at first element
ans = arr[0]
for x in arr[1:]: ← single left-to-right pass
dp1 = max(dp0, dp1 + x) ← delete x (use dp0) OR extend after prev deletion
dp0 = max(x, dp0 + x) ← restart or extend, no deletion used yet
ans = max(ans, dp0, dp1) ← update global best from both states
return ans
💡 Why Update dp1 BEFORE dp0?

The dp1 transition uses the old dp0 (from the previous element): dp1 = max(dp0_old, dp1_old + x). If we updated dp0 first, dp1's "delete x" option would use the new dp0, which already incorporated x — effectively deleting the wrong element. Order matters: dp1 must be updated using the previous dp0.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Connect to the Known Base Problem

"This is Maximum Subarray (#53) with one optional deletion. I know Kadane's solves #53 in O(n). Let me think about what changes when we add a deletion."

Step 2: State the Brute Force Clearly

"O(n²) brute force: for each of the n elements, remove it and run Kadane's on the rest. That's n × O(n) = O(n²) total. We can do better."

Step 3: Arrive at the O(n) Insight

"If I delete element at index i, I need the best subarray ending at i-1 (forward Kadane precompute) and the best starting at i+1 (backward Kadane precompute). Both can be precomputed in O(n). Then one O(n) sweep evaluates all deletions. Total: O(n) time, O(n) space."

Step 4: Offer the O(1) Space Variant

"If O(1) space is needed: track two states — dp0 (best ending here with no deletion) and dp1 (best ending here with one deletion used). The key transition: dp1 = max(dp0, dp1 + x). Order matters — dp1 must be updated before dp0."

🗣️ What to Say to the Interviewer

"I recognize this as Maximum Subarray with one optional deletion.
Brute force is O(n²) — try all n deletions and run Kadane's each time.
To optimize: precompute fwd[i] = best subarray ending at i, bwd[i] = best starting at i.
Initialize ans = max(fwd) — this handles the no-deletion case.
Then sweep i from 1 to n-2: ans = max(ans, fwd[i-1] + bwd[i+1]).
Total: O(n) time, O(n) space. With the 2-state DP we can reduce space to O(1)."
🔮 Likely Follow-up Questions
  • "Can you do O(1) space?" → Yes — the dp0/dp1 state machine. Watch the update order!
  • "What if up to k deletions are allowed?" → DP with k+1 states; O(n·k) time
  • "What if the array is circular?" → Combine with circular Kadane (complement trick)
  • "Classic Maximum Subarray — no deletion?" → Standard Kadane's, O(n) O(1) (#53)

7 Summary & Key Takeaways ▾
Deletion is optional — always initialize ans with the no-deletion case (max(fwd))
fwd[i] and bwd[i] are standard Kadane's applied in opposite directions
Bridge formula: delete index i → combine fwd[i-1] + bwd[i+1]
Only interior indices (1 to n-2) need the deletion sweep; endpoints are already covered
O(1) space: use 2-state DP (dp0=no deletion, dp1=one deletion); update dp1 BEFORE dp0
ApproachTimeSpaceNotes
Brute Force (try all deletions) O(n²) O(1) Run Kadane's n times; too slow
Fwd + Bwd Kadane ✓ O(n) O(n) Easiest to explain & verify
State Machine DP (dp0/dp1) O(n) O(1) Best space; trickier update order
🎯 When to Recognize This Pattern
  • "Max/min subarray with at most one element removed/modified"
  • "Best contiguous segment bridging across a possible skip"
  • You need both 'best ending at i' AND 'best starting at i' for the same array
  • Whenever two Kadane passes (forward + backward) can answer queries for all positions in O(1)
