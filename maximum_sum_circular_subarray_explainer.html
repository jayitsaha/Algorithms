LeetCode #918

Maximum Sum Circular Subarray

Given a circular integer array nums, return the maximum possible sum of a non-empty subarray (elements may wrap around from end to start).

Medium Kadane's Algorithm Circular + Min Subarray Time: O(n) Space: O(1)
1 Understanding the Problem ▾

A circular array wraps around — the element after the last one is the first one. You can pick any contiguous subarray, including one that "wraps" from the end back to the beginning. Return the maximum possible sum among all such choices.

💡 Key Insight: Only Two Cases Exist

Every possible subarray in a circular array is one of exactly two types:

  • Case A — No wrap: The subarray sits entirely within the linear array (standard problem).
  • Case B — Wraps around: The subarray starts near the end, wraps past index n-1, and continues from index 0. Crucially, the elements we exclude form a contiguous middle block, so: wrap_sum = total_sum − min_middle_sum.
Case A: No Wrap (Linear)

Subarray is entirely within the array. This is classic Kadane's.

answer = max_kadane(nums)

Example: [5, -3, 5] → [5,-3,5] = 7

Case B: Wraps Around

The excluded middle block should be minimized.

answer = total − min_subarray

Example: [5, -3, 5] → 7 − (−3) = 10

Why these two cases cover everything: A subarray in a circular array of length n either (a) does not include the boundary between index n-1 and index 0, or (b) it does. There is no third case. The final answer is max(Case A, Case B).

⚠️ Critical Edge Case: All Negatives

If every element is negative, max_kadane < 0. In this situation the formula total − min_subarray gives 0 (because min_subarray = total — the entire array is the minimum block). But the subarray must be non-empty, so 0 is wrong. Guard: if max_g < 0, return max_g directly.

✓ Case A wins: nums = [1, 2, 3, -10, 2]

Kadane max = [1,2,3] = 6
total = -2; min = [-10] = -10
circular = -2 − (−10) = 8
Answer = max(6, 8) = 8 ← wrap wins!

✗ All negatives: nums = [-3, -2, -3]

max_g = -2 (least negative)
total = -8; min_g = -8 (all)
circular = -8 − (−8) = 0 ← WRONG!
Guard: return max_g = -2

2 The Algorithm: Dual Kadane in One Pass ▾

In a single traversal, run two simultaneous Kadane passes — one to find the maximum subarray sum (Case A), one to find the minimum subarray sum (for Case B). Accumulate the total sum. Then compare the two cases.

1 Initialize: max_cur = min_cur = max_g = min_g = nums[0], total = nums[0]. All tracking variables start at the first element.
2 For each num in nums[1:]: Update max_cur = max(num, max_cur + num). This is Kadane's choice: extend the running subarray or restart fresh at num. Update max_g = max(max_g, max_cur).
3 Same for minimum (mirror Kadane): min_cur = min(num, min_cur + num), then min_g = min(min_g, min_cur). Finds the minimum contiguous subarray sum in the same pass.
4 Accumulate total: total += num in every iteration.
5 All-negatives guard: If max_g < 0, all elements are negative. Return max_g — the least-negative single element.
6 Return: max(max_g, total - min_g). max_g = best Case A. total - min_g = best Case B (whole array minus worst middle block).
🔑 The Kadane's Algorithm Deep-Dive

Kadane's Algorithm (Joseph Born Kadane, 1984) solves "Maximum Subarray Sum" in O(n) time, O(1) space.

Core Invariant: At each step i, cur holds the maximum subarray sum ending exactly at index i. The global maximum is the best we ever saw across all positions.

Why the restart works: If the running sum drops below the current element (cur + num < num), the previous history is only a drag — better to start fresh. This is equivalent to saying: "any prefix with negative contribution should be discarded."

Recognition signal: "Find max (or min) sum of a contiguous subarray" with O(n) time constraint → Kadane's.

3 Why It Works: The Invariant ▾
🔐 Invariant: max_cur = best subarray sum ending at index i

After processing index i, max_cur always holds the maximum possible sum of any contiguous subarray that ends at nums[i]. Similarly, min_cur holds the minimum. These invariants are maintained by the restart-or-extend decision at each step.

Why total − min_g = best wrap subarray: A wrapping subarray = all elements except some contiguous middle block. To maximize the wrap sum, we minimize the excluded block. The excluded block is a contiguous subarray, so its minimum is exactly min_g. Therefore wrap_max = total − min_g.

Formal completeness argument: Every contiguous circular subarray of length 1 to n falls into exactly one of the two cases (linear or wrap). Case A is handled by max_g. Case B is handled by total − min_g, provided the excluded block is strictly shorter than the full array. The all-negatives guard prevents the degenerate wrap where the "excluded middle" is the entire array.

💡 Intuition: Duality of Max and Min

The brilliance of this approach is recognizing that maximizing the circular subarray is dual to minimizing its complement. Running Kadane's for minimum takes exactly the same code with min instead of max. We get both in a single O(n) pass — essentially solving two problems simultaneously for free.

⚠️ Why Double-Array is Inferior

You could concatenate nums + nums and find max subarray of length ≤ n using a deque-based sliding window. This is O(n) time and O(n) space — same time but worse space. The dual Kadane approach achieves O(1) space by exploiting the complementary structure. Always prefer it.

4 The Code (Annotated) ▾

Solution 1 — Brute Force O(n²) — intuition only

def maxSubarraySumCircular(nums):
n, best = len(nums), float('-inf') ← start with worst possible answer
for i in range(n): ← try every starting index
cur = 0
for k in range(1, n + 1): ← try every length 1..n
cur += nums[(i + k - 1) % n] ← % n wraps indices circularly
best = max(best, cur)
return best ← O(n²) time, O(1) space

Solution 2 — Optimal: Dual Kadane O(n), O(1) — Interview Pick ✓

def maxSubarraySumCircular(nums):
max_cur = min_cur = nums[0] ← Kadane running sums (both start same)
max_g = min_g = nums[0] ← best global max and min subarray sums
total = nums[0] ← accumulate entire array sum
for num in nums[1:]: ← single O(n) pass
max_cur = max(num, max_cur + num) ← restart vs extend for max
max_g = max(max_g, max_cur) ← update Case A answer
min_cur = min(num, min_cur + num) ← restart vs extend for min (mirror)
min_g = min(min_g, min_cur) ← update min subarray (needed for Case B)
total += num ← sum entire array
if max_g < 0: ← all-negatives guard
return max_g ← return least-negative element
return max(max_g, total - min_g) ← max(Case A, Case B)
⚠️ Don't Forget the All-Negatives Guard

Without if max_g < 0: return max_g, when all elements are negative the formula total − min_g computes total − total = 0. But a non-empty subarray of all-negative values must return a negative number (the maximum = least-negative element). The guard is essential.

💡 Complexity Summary

Brute Force: Time O(n²), Space O(1)
Dual Kadane (optimal): Time O(n), Space O(1) — single pass, five integer variables
Double Array + Deque: Time O(n), Space O(n) — valid but wastes space

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Base Problem

"This is Maximum Subarray (#53) with a circular twist. I know Kadane's handles the linear case in O(n)/O(1). Let me think about what 'circular' adds."

Step 2: Case Analysis (the aha moment)

"A subarray in a circular array either wraps around the boundary or it doesn't. Two cases — I can solve each. Case A (no wrap) = standard Kadane. Case B (wrap) = the elements I include are everything except some contiguous middle. To maximize the wrap sum, I minimize what I exclude: total − min_subarray."

Step 3: Combine into One Pass

"Kadane's for max and Kadane's for min run independently. I can run them simultaneously in one loop: track max_cur, max_g, min_cur, min_g, and total. Single O(n) pass, O(1) space."

Step 4: Handle the All-Negatives Edge Case

"When all elements are negative, min_g equals total, so total − min_g = 0 — wrong since we need a non-empty subarray. Guard: if max_g < 0, return max_g."

🗣️ What to Say to the Interviewer

"I see two cases: no-wrap (standard Kadane) and wrap-around.
For the wrap case, the elements I exclude form a contiguous middle block.
To maximize what I include, I minimize what I exclude: wrap_max = total − min_subarray.
I'll run two Kadane passes simultaneously — O(n) time, O(1) space — one for max, one for min.
Edge case: all negatives → guard against the formula returning 0."
