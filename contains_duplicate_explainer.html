Contains Duplicate

LeetCode 217 — Return true if any value appears at least twice

Easy Hash Set O(n) time O(n) space
1 Understanding the Problem ▾

Given an array of integers, determine if any value appears more than once. Return true if there's a duplicate, false if every element is unique.

Example: [1, 2, 3, 1] → The value 1 appears at index 0 and index 3. That's a duplicate → return true.

Counter-example: [1, 2, 3, 4] → All values are distinct → return false.

💡 Key Insight

We need a way to remember what we've already seen. A hash set provides O(1) lookup — for each element, we check "have I seen this before?" If yes, it's a duplicate. If no, add it to the set and move on.

2 The Algorithm: Hash Set Insert Check ▾

The optimal approach uses a hash set as a "memory" of elements seen so far. We scan left to right, and for each element we ask one question: "Is this already in the set?"

1 Create an empty hash set called seen.
2 For each number num in the array:
3 If num is already in seen → return true (duplicate found!).
4 Otherwise, add num to seen and continue.
5 If we finish the loop without returning → return false (all unique).

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing nums[0..i], the set seen contains exactly the distinct values from that subarray. If nums[i+1] is already in seen, it must be a duplicate.

Correctness: A hash set has O(1) membership test. Every element is checked against all previously seen elements. If any element was seen before, we immediately detect it. If we finish without detection, every element was encountered exactly once.

Early termination: We return as soon as the first duplicate is found — no need to scan the entire array. In the best case (duplicate at position 1), it's O(1).

⚠️ Why Not Sort?

Sorting works — duplicates become adjacent — but it's O(n log n) time and modifies the input (or needs a copy). The hash set approach is O(n) and doesn't mutate the array, making it strictly better when extra space is acceptable.

4 The Code (Annotated) ▾
# Hash Set Insert Check — Interview Pick
def containsDuplicate(nums):
seen = set() # empty hash set for O(1) lookup
for num in nums: # scan each element once
if num in seen: # O(1) membership check
return True # duplicate found!
seen.add(num) # remember this value
return False # all values were unique
📊 Complexity

Time: O(n) — one pass through the array, with O(1) set operations per element.
Space: O(n) — in the worst case (all unique), the set stores every element.

⚠️ One-Liner Shortcut

return len(nums) != len(set(nums)) — Works, but builds the entire set before comparing. The loop version above can short-circuit early on the first duplicate, which is faster in practice.

5 🎤 Interview Strategy ▾
Step 1: Recognize the Pattern

"Check if something exists" or "find duplicates" → hash set. Any time you need O(1) existence checks, a set is the tool.

Step 2: Mention Alternatives

Briefly note brute force O(n²) and sorting O(n log n) to show awareness, then propose the hash set O(n) approach as optimal.

Step 3: Handle Follow-ups

If asked "what if memory is limited?" → sorting is O(1) space. If asked "what about Contains Duplicate II?" → use a hash map tracking indices, or a sliding window set of size k.

🗣️ What to Say to the Interviewer

"I'll use a hash set to track seen values. For each element, I check if it's already in the set — if yes, return true. Otherwise add it. This gives O(n) time and O(n) space. I could also sort for O(1) space but that's O(n log n)."
