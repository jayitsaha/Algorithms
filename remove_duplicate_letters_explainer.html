LeetCode #316

Remove Duplicate Letters

Return the smallest in lexicographic order among all possible results of removing duplicate letters so that every letter appears once and only once.

Medium Stacks Monotonic Stack Time: O(n) Space: O(k)
1 Understanding the Problem ▾

You're given a string with duplicate characters. Your mission: remove enough characters so that every unique character appears exactly once. But you don't get to rearrange — you're selecting a subsequence that preserves the original order. Among all valid subsequences (one of each char), return the lexicographically smallest one.

💡 Key Insight

The "which occurrence to keep" decision is greedy: you want smaller characters as early as possible. But you can only skip an occurrence of a character if that character still appears later in the string — otherwise skipping it would lose the character entirely from your result.

✓ "bcabc" → "abc"

Smallest possible → "abc"

✓ "cbacdcbc" → "acdb"

"abcd" looks better but 'd' must follow
its only occurrence → "acdb"

Key constraints to keep in mind:

  • Every unique character must appear exactly once in the result — no character can be left out.
  • The relative order must be preserved from the original string — this is a subsequence problem.
  • Among all valid selections, return the lexicographically smallest result.

Edge cases: single character (trivially itself), all same characters like "aaaa" (result is "a"), a string already with all unique chars (just return it).

2 The Algorithm: Greedy Monotonic Stack ▾

We scan left-to-right and maintain a monotonic stack (kept in non-decreasing lexicographic order). Two helper structures: a remaining count map (how many more times a char still appears ahead) and a seen set (which chars are already committed to the result).

1 Pre-count: Walk the full string once and count how many times each character appears. This is our "remaining" map. Before visiting position i, we decrement — so remaining[c] always reflects how many future occurrences are still ahead.
2 Already committed? Before pushing any character, check if it's already in the seen set. If so, skip it — we already have a copy in the stack at an earlier (better) position. Always decrement remaining first, even when skipping.
3 Greedy pop loop: While the stack is not empty AND stack[-1] > c (top is lexicographically larger than current) AND remaining[stack[-1]] > 0 (top will appear again later, so safe to delay): pop the top, remove it from seen.
4 Push and commit: Push c onto the stack. Add it to seen. Continue to the next character.
5 Read the answer: The stack (from bottom to top) is the lexicographically smallest valid subsequence. Join and return.
🔑 The Two-Condition Pop Guard

Both conditions must hold simultaneously to pop:

  • stack[-1] > c — We'd get a lexicographically better result by placing c here instead of the top. Without this, we'd pop unnecessarily.
  • remaining[stack[-1]] > 0 — The top still appears later in the string. If this is its last occurrence, popping it would make the result invalid (missing a required character). This guard is mandatory.
3 Why It Works: The Invariant ▾
🔐 The Invariant

At every point in the algorithm, the stack contains a valid prefix of the optimal answer: (1) all characters in the stack are distinct, (2) the stack is in non-decreasing lexicographic order, (3) every character ever popped from the stack still appears at least once in the remaining unprocessed portion of the string.

Why the non-decreasing order guarantees lexicographic minimality: The lexicographically smallest string minimizes the character at the earliest position where a choice exists. By greedily popping any top that is larger than the current character (and can be replaced), we ensure our prefix is always optimal. Any future character we encounter will either extend the stack or trigger more pops — always keeping the prefix as small as possible.

Why we cannot always pop: If remaining[top] == 0, popping would permanently lose that character from the result. An invalid result (missing a required unique character) is worse than a non-optimal order. The guard prevents this.

Why the seen set is correct: When we encounter a character already in the stack, the copy already in the stack is at an earlier position. Since we want the smallest subsequence and relative order must be preserved, the earlier occurrence is always at least as good as a later one. Skipping duplicates is safe and necessary.

⚠️ Decrement Before Skip — Critical Order

You must do remaining[c] -= 1 BEFORE the if c in seen: continue check. Even if we skip a character (it's already committed), we still physically consumed it from the string. The remaining count must reflect this. Getting this order wrong produces incorrect pop decisions later.

4 The Code (Annotated) ▾

Solution 1 — Greedy Monotonic Stack (Optimal, Interview Pick)

def removeDuplicateLetters(s: str) -> str:
remaining = {} ← Will hold future occurrence counts
for c in s: ← Pre-count pass
remaining[c] = remaining.get(c, 0) + 1 ← Total occurrences of each char
seen = set() ← Chars committed to result
stack = [] ← Result built bottom-to-top
for c in s: ← Scan left to right
remaining[c] -= 1 ← Consume this occurrence (MUST be first)
if c in seen: ← Already in result — skip this duplicate
continue
while (stack ← Monotonic stack: try to improve prefix
and stack[-1] > c ← Top is lexicographically larger
and remaining[stack[-1]] > 0): ← Top still appears later (safe to pop)
seen.discard(stack.pop()) ← Undo commitment, allow it to be re-added
stack.append(c) ← Commit this character
seen.add(c) ← Mark as used
return ''.join(stack) ← Read bottom-to-top = lexicographically smallest
⚠️ Most Common Bug: Wrong Order of Operations

Many solutions fail by putting if c in seen: continue BEFORE remaining[c] -= 1. This means skipped characters don't get their count decremented. As a result, the algorithm thinks a character appears more times in the future than it actually does, leading to incorrect pops. Always decrement first.

Solution 2 — Brute Force via Recursion (Exponential time, for intuition)

def removeDuplicateLetters_brute(s: str) -> str:
if not s: return s ← Base case
cnt = Counter(s) ← Count all chars in current string
pos = 0 ← Find leftmost char we can safely remove
for i in range(len(s)):
if s[i] < s[pos]: ← Prefer the smallest char at earliest pos
pos = i
cnt[s[i]] -= 1
if cnt[s[i]] == 0: break ← Stop before losing any unique char
return s[pos] + removeDuplicateLetters_brute(s[pos+1:].replace(s[pos], '')) ← Recurse
💡 Complexity Summary

Greedy Stack (Optimal): Time O(n) — each char pushed/popped at most once. Space O(k) where k = unique chars (≤ 26 for a–z → effectively O(1))
Brute Force: Time O(n² × k) per level, O(n) recursion depth — impractical for large inputs

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern Signals

"I need to select one of each unique character. The order must be preserved (subsequence). I want the smallest result — that screams greedy. Whenever we want a minimal result with an 'undo last decision' ability, stacks are the right data structure."

Step 2: Identify What You Need to Track

"I need to know: (1) which characters I've already committed to — a seen set. (2) whether a character I might pop still appears later — a remaining count map. Without these two, I can't make correct greedy decisions."

Step 3: Define the Greedy Choice

"At each character c: if top > c and top appears again later, pop the top — I can put c earlier and still include top later. Both conditions must hold. If top is larger but this is its last copy, I'm stuck with it. If top is smaller, no benefit in popping."

Step 4: Remember the Order of Operations

"Always: (1) decrement remaining, (2) skip if seen, (3) pop while conditions hold, (4) push and mark seen. Get this order wrong and the solution silently fails on tricky inputs."

🗣️ What to Say to the Interviewer

"This is a greedy monotonic stack problem. I'll pre-count occurrences of each character. Then scan left-to-right. For each character c: 1. Decrement remaining[c] — we've consumed this copy. 2. If c is already in our result (seen set), skip it. 3. While the stack top is larger than c AND the top still appears later in the string, pop the top — we can include it later at a better (smaller-first) position. 4. Push c, mark it seen. The stack reads bottom-to-top as our answer. Time O(n), Space O(k)."

7 Summary & Key Takeaways ▾
The stack is monotonic (non-decreasing bottom→top). Each push and pop is amortized O(1), giving O(n) total.
Decrement remaining[c] before the seen check — even skipped chars must be counted as consumed.
Pop only when BOTH: top > current (lexicographic gain) AND remaining[top] > 0 (won't lose the character).
The seen set prevents duplicates. An earlier occurrence in the stack is always at least as good as a later one.
This exact algorithm also solves LC #1081 (Smallest Subsequence of Distinct Characters) — the problems are identical.
🎯 When to Recognize This Pattern
  • Select a subsequence with uniqueness constraints
  • Optimize for lexicographic order with a greedy choice at each step
  • "Can I undo a previous commitment?" — stack gives O(1) undo
  • "Does this decision depend on future data?" — pre-count + remaining map
