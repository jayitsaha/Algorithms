LeetCode #783

Minimum Distance Between BST Nodes

Given the root of a Binary Search Tree, return the minimum difference between the values of any two different nodes in the tree.

Easy Trees In-Order Traversal Time: O(n) Space: O(h)
1 Understanding the Problem ▾

You're given a Binary Search Tree with unique integer values. Find the minimum absolute difference between values of any two distinct nodes. The tree can have thousands of nodes — we want better than brute-force.

💡 Key Insight

In a BST, an in-order traversal (Left → Root → Right) visits every node in strictly ascending sorted order. The minimum gap between any two node values must be the gap between some pair of consecutive values in that sorted order — never between non-adjacent ones.

Consider the BST below with nodes [4, 2, 6, 1, 3]:

4 / \ 2 6 / \ 1 3 In-order (sorted): 1 → 2 → 3 → 4 → 6 Consecutive gaps: |2-1|=1 |3-2|=1 |4-3|=1 |6-4|=2 Answer: 1 (minimum of the consecutive gaps)

Edge cases to think about: A BST with only 2 nodes (return their difference). A skewed tree (all nodes in a line — space becomes O(n) for the recursion stack). Nodes with very large values — no overflow since we only subtract adjacent sorted values.

2 The Algorithm: In-Order Traversal with Prev Pointer ▾

We run a standard recursive in-order traversal (Left → Root → Right) across the BST. We carry one extra variable — prev — which always holds the value of the most recently visited node. Each time we process a node, we compute node.val − prev (always non-negative since sorted) and keep a running minimum.

1 Initialize: min_diff = float('inf') — no minimum found yet. prev = [None] — a mutable list so the inner recursive function can write to it (Python scoping requirement).
2 Recurse left first: inorder(node.left). In-order always visits the left subtree first, which contains all values smaller than the current node. This ensures we arrive at nodes in ascending order.
3 Process current node: If prev[0] is not None (i.e., we've visited at least one previous node), compute diff = node.val − prev[0] and update min_diff = min(min_diff, diff). Then set prev[0] = node.val.
4 Recurse right: inorder(node.right). After processing the current node, visit the right subtree — all values larger than the current node, processed next in sorted order.
5 Return result: After traversing all n nodes, min_diff holds the answer — the smallest gap between any two consecutive values in the BST's sorted sequence.
🔑 The BST In-Order Sorted-Order Guarantee

BST invariant: left subtree values < current node < right subtree values. An in-order traversal follows: left subtree (smallest values first), then root, then right subtree (largest values). This produces a strictly increasing sequence. The minimum absolute difference between any two nodes is therefore the minimum of the n−1 consecutive differences in this sorted sequence — because any non-adjacent pair's gap is a sum of adjacent gaps and thus ≥ the smallest adjacent gap.

3 Why It Works: The Invariant ▾
🔐 The Invariant

When we process any node X in the in-order traversal, prev[0] contains the value of the node immediately before X in sorted order — i.e., the largest BST node value that is still smaller than X.val. Therefore X.val − prev[0] is always positive and equals the gap between consecutive values.

Why the minimum must be adjacent in the sorted sequence: Let the sorted in-order values be a₁ < a₂ < ... < aₙ. For any non-adjacent pair (aᵢ, aⱼ) with j > i+1:

aⱼ − aᵢ = (aᵢ₊₁−aᵢ) + (aᵢ₊₂−aᵢ₊₁) + ... + (aⱼ−aⱼ₋₁) ≥ min adjacent gap

A non-adjacent gap is a sum of adjacent gaps. It can never be smaller than the smallest individual adjacent gap. So checking only adjacent pairs is sufficient — we find the global minimum.

⚠️ Common Mistake: Mutable State in Python Closures

If you write prev = None and try to do prev = node.val inside a nested function, Python will raise UnboundLocalError. The inner function sees prev as a new local variable. Fix: use nonlocal prev (Python 3), or wrap in a list: prev = [None] and write prev[0] = node.val. Alternatively, use self.prev on the Solution class.

First-node case: The leftmost (smallest) node is the very first node visited. At that moment prev[0] is None — there is no predecessor. The if prev[0] is not None guard skips the difference calculation and simply records this node as prev. Correct handling of this edge case is essential.

⚠️ Why no abs() around the difference?

Because in-order traversal guarantees we visit values in ascending order, every new node.val is strictly greater than prev[0]. So node.val − prev[0] is always positive — we never need abs(). This only holds for a BST with unique values — a generic binary tree would require abs().

Solution 2 — Iterative In-Order with Explicit Stack

def minDiffInBST(root):
stack, prev, min_diff = [], None, float('inf')
curr = root
while curr or stack: ← keep going while there's unprocessed work
while curr: ← go as far left as possible (find smallest unvisited)
stack.append(curr) ← save node to return to after visiting its left subtree
curr = curr.left
curr = stack.pop() ← process leftmost unvisited node
if prev is not None: ← skip diff on first node
min_diff = min(min_diff, curr.val - prev)
prev = curr.val
curr = curr.right ← now process the right subtree on next iteration
return min_diff
💡 Complexity Summary

Recursive In-Order: Time O(n), Space O(h) where h = height. Balanced BST → O(log n); skewed tree → O(n).
Iterative Stack: Time O(n), Space O(h) — same complexity, but avoids Python's recursion limit (~1000 by default).
Brute Force (all pairs): Time O(n²), Space O(n) — never use in an interview.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the BST Property

The moment you see "BST," think: "in-order traversal gives me sorted order for free." This is the most commonly tested BST property. Don't treat this like a generic tree — that BST structure is your key to an O(n) solution.

Step 2: Propose Brute Force, Then Optimize

"Naively, I could collect all values and check all pairs — O(n²). Or collect and sort, then check adjacent pairs — O(n log n). But since it's a BST, the in-order traversal is already sorted. I can check adjacent pairs in O(n) with just a single pass."

Step 3: Explain the Prev Pointer

"I only need the immediately preceding value in sorted order — not all past values. A single prev variable is enough. At each node, compute the difference with prev, update the minimum, then set prev to this node's value."

Step 4: Mention Python Scoping

"In Python, I'll wrap prev in a list or use self.prev on the class, since Python closures can't reassign outer-scope variables without nonlocal. For the iterative version, this isn't needed — prev is just a local variable."

🗣️ What to Say to the Interviewer

"Since this is a BST, in-order traversal visits nodes in sorted ascending order.
The minimum difference must be between consecutive values in that sorted order.
I'll do one in-order pass tracking 'prev' — the last visited node's value.
At each node: diff = node.val − prev (always positive in sorted order).
Update min_diff = min(min_diff, diff). Set prev = node.val. Recurse right.
Time: O(n) — each node visited once. Space: O(h) — recursion call stack."
🔮 Likely Follow-Up Questions
  • "What if the tree is very skewed (like a linked list)?" → Recursion stack becomes O(n). Switch to the iterative stack version to avoid stack overflow.
  • "What if it were a generic binary tree, not a BST?" → Can't rely on in-order = sorted. Collect all values in any order, sort them, then check adjacent pairs. O(n log n).
  • "Find the K-th smallest element in BST?" → Same in-order traversal, decrement counter K at each node, return when K = 0.
  • "This is identical to LeetCode #530 (Minimum Absolute Difference in BST)." → Yes! Same problem, same solution. Show you know both problem numbers.

7 Summary & Key Takeaways ▾
BST in-order traversal always produces values in strictly ascending sorted order.
The minimum difference between any two nodes is always between adjacent values in that sorted sequence — never non-adjacent.
Track only prev (previous node's value) — you never need to store all visited values.
In sorted order, node.val − prev is always positive — no need for abs().
In Python: use prev = [None] (mutable list), nonlocal prev, or self.prev to share state across recursive calls.
🎯 When to Use This Pattern: In-Order + Track Prev
  • Problem asks for a property of consecutive BST values (min difference, validation, k-th smallest)
  • Exploit the BST's sorted structure — don't treat it as a generic tree
  • Need O(n) time with only O(h) extra space (no auxiliary array)
  • Comparing adjacent sorted values (predecessor/successor relationships)
