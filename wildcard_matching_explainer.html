LeetCode #44

Wildcard Matching

Given a string s and a pattern p containing ? (matches any single char) and * (matches any sequence), determine if p matches s entirely.

Hard Dynamic Programming 2D DP · Star Handling Time: O(m·n) Space: O(m·n)
1 Understanding the Problem ▾

You have a string s and a pattern p. Two wildcard characters are allowed in the pattern: ? matches exactly one character (any character), and * matches zero or more characters (any sequence, including empty). The pattern must match the entire string — not just a substring.

💡 Key Insight

The * wildcard is what makes this a Hard problem — it can absorb 0, 1, 2, … characters from s. We cannot decide greedily. The trick is to define dp[i][j] = "do the first i chars of s match the first j chars of p?" Each cell depends on just 1–2 previously-computed cells, turning O(2n) backtracking into O(m·n) table filling.

✓ Matches: s="adceb", p="*a*b"

* matches "" (zero chars)
a matches 'a'
* matches "dce"
b matches 'b'
→ True

✗ No match: s="cb", p="?a"

? matches 'c' (ok)
a must match 'b' — FAILS
No wildcard to save this
→ False

Wildcard rules summary:

  • ? — matches exactly one character (any character)
  • * — matches zero or more characters (any sequence)
  • Regular character — must match the same character in s

Edge cases: p = "*" matches any string including empty; p = "" only matches s = ""; consecutive stars "**" are equivalent to one "*"; a pattern of all stars matches anything.

2 The Algorithm: 2D Bottom-Up DP ▾

We build a 2D boolean table where dp[i][j] = "do the first i characters of s match the first j characters of p?" Rows index into s (0 = empty prefix), columns index into p (0 = empty prefix). The answer is dp[m][n].

1 Base case dp[0][0] = True: Empty string matches empty pattern exactly.
2 Base row (i=0, empty string): dp[0][j] = dp[0][j-1] only if p[j-1] == '*'. A star can match the empty string, so a run of leading stars still matches empty s. Any non-star character immediately breaks this chain (dp[0][j] = False).
3 Letter or ? match: If p[j-1] == s[i-1] or p[j-1] == '?', then dp[i][j] = dp[i-1][j-1]. Both pointers advance — take the diagonal.
4 Star case: dp[i][j] = dp[i][j-1] (star matches empty — skip it, pattern advances) OR dp[i-1][j] (star matches one more char — string advances, star stays put). The star "grows" by keeping column j fixed and moving row i upward.
5 Otherwise False: Characters don't match and it's not a wildcard → dp[i][j] = False.
📐 Recurrence Relations
dp[0][0] = True
dp[0][j] = dp[0][j-1]  if p[j-1] == '*' else False

if p[j-1] == s[i-1] or p[j-1] == '?':
    dp[i][j] = dp[i-1][j-1]    ← diagonal

elif p[j-1] == '*':
    dp[i][j] = dp[i][j-1] or dp[i-1][j]
               ↑ star=ε    ↑ star eats s[i-1]

else:
    dp[i][j] = False
🔑 Why dp[i-1][j] Lets the Star Eat a Character

When p[j-1] == '*', the star occupies column j. dp[i-1][j] asks: "could this star (still at column j) have already matched s[0..i-2]?" If yes, then the same star can consume one more character s[i-1], making dp[i][j] True. The star does not advance its column — that's how it expands across multiple characters.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At all times, dp[i][j] is True if and only if s[0..i-1] (the first i characters of s) can be fully matched by p[0..j-1] (the first j characters of p), using the wildcard rules. Every cell is derived from smaller subproblems (top, left, or diagonal) that were already computed — this is the optimal substructure property.

Why not greedy? Greedy works for simple patterns, but a * creates ambiguity: how many characters should it consume? Consider s="aab", p="a*b". A greedy approach might let * match nothing (leaving "b" unmatched) or match "ab" (consuming too much). DP explores all possibilities implicitly: dp[i][j-1] (skip) OR dp[i-1][j] (eat), and takes True if either works.

Overlapping subproblems: Pure recursion on (i, j) from the top re-visits the same pair from many branches. Consider any * in the pattern: it fans out to call match(i-1, j), match(i-2, j), etc., each of which calls each other recursively. Memoisation (top-down) or tabulation (bottom-up) collapses this to one computation per cell.

⚠️ Common Mistake: Confusing Wildcard Matching (#44) with Regex Matching (#10)

In Regex Matching (LC #10), * is a quantifier for the preceding element: "a*" means "zero or more a's". Its recurrence is dp[i][j] = dp[i][j-2] OR (dp[i-1][j] if chars match). In Wildcard Matching, * stands alone and matches any sequence. Its recurrence is dp[i][j-1] OR dp[i-1][j] — no need to look two columns back. Mixing these up is the #1 interview bug.

4 The Code (Annotated) ▾

Solution 1 — 2D DP Bottom-Up (Interview Pick, O(mn) time & space)

def isMatch(s: str, p: str) -> bool:
m, n = len(s), len(p) ← lengths of s and p
dp = [[False]*(n+1) for _ in range(m+1)] ← (m+1)×(n+1) table; index 0 = empty prefix
dp[0][0] = True ← empty string vs empty pattern = match
for j in range(1, n+1): ← base row: empty s vs pattern prefix
if p[j-1] == '*': ← star can absorb empty string
dp[0][j] = dp[0][j-1] ← carry True forward only if chain of stars
for i in range(1, m+1): ← fill rows 1..m (each char of s)
for j in range(1, n+1): ← fill cols 1..n (each char of p)
if p[j-1] == '*': ← star case: two sub-cases
dp[i][j] = dp[i][j-1] or dp[i-1][j] ← star=ε OR star eats s[i-1]
elif p[j-1] == s[i-1] or p[j-1] == '?': ← char match or ? wildcard
dp[i][j] = dp[i-1][j-1] ← diagonal: both advance
# else: dp[i][j] remains False ← mismatch, no wildcard
return dp[m][n] ← full s vs full p
⚠️ Why dp[i][j-1] for "star skips"?

dp[i][j-1] asks: "does s[0..i-1] match p[0..j-2]?" If yes, adding a * that matches zero characters still gives a match — so dp[i][j] is also True. This is the "empty consumption" case: the star contributes nothing and we just skip past it.

Solution 2 — Space-Optimised Rolling Array (O(n) space)

def isMatch(s: str, p: str) -> bool:
m, n = len(s), len(p)
prev = [False] * (n + 1) ← represents dp[i-1][*], the "previous row"
prev[0] = True
for j in range(1, n + 1): ← base row initialization
prev[j] = prev[j-1] and p[j-1] == '*'
for i in range(1, m + 1):
curr = [False] * (n + 1) ← current row (i-th row)
for j in range(1, n + 1):
if p[j-1] == '*':
curr[j] = curr[j-1] or prev[j] ← skip* OR eat (prev[j] = dp[i-1][j])
elif p[j-1] == s[i-1] or p[j-1] == '?':
curr[j] = prev[j-1] ← diagonal = dp[i-1][j-1]
prev = curr
return prev[n]
💡 Complexity Summary

2D DP: Time O(m·n), Space O(m·n) — most readable for interviews
Space-Optimised DP: Time O(m·n), Space O(n) — rolling two rows
Brute Force Recursion: Time O(2m+n) — exponential, times out

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

"Two strings, one is a pattern — 'does p fully match s?' The * wildcard can consume any number of characters, so there's no greedy choice. That plus the phrase 'match the entire string' both signal 2D DP on prefixes."

Step 2: Define the State

"dp[i][j] = True if the first i chars of s can be matched by the first j chars of p. I need (m+1)×(n+1) cells. Answer: dp[m][n]."

Step 3: Write the Recurrence (3 cases)

"Case 1: p[j-1] is a letter or '?' that matches s[i-1] → take diagonal dp[i-1][j-1]. Case 2: p[j-1]=='*' → dp[i][j-1] (star=empty) OR dp[i-1][j] (star eats one char). Case 3: mismatch → False."

Step 4: Set Base Cases Carefully

"dp[0][0]=True. Row 0 (empty s): dp[0][j] = dp[0][j-1] only if p[j-1]=='*'. Column 0 (empty p): all False except dp[0][0] — a non-empty pattern cannot match empty s."

Step 5: Mention the Space Optimization

"Since each row only depends on the previous row, I can roll two arrays to reduce space from O(mn) to O(n). Worth noting, though the 2D version is clearer for an interview."

🗣️ What to Say to the Interviewer

"I'll use 2D DP. dp[i][j] = can s[0..i-1] be matched by p[0..j-1]?
Base: dp[0][0]=True. Empty s matches all-star prefixes, so dp[0][j]=dp[0][j-1] when p[j-1]=='*'.
Fill: For each (i,j) — if p[j-1]=='*', take dp[i][j-1] OR dp[i-1][j].
If char match or '?', take dp[i-1][j-1]. Otherwise False.
Answer: dp[m][n]. Time O(mn), Space O(mn) reducible to O(n)."
🔮 Likely Follow-up Questions
  • "Can you reduce space to O(n)?" → Yes — roll prev/curr arrays (Solution 2 above)
  • "How is this different from Regex Matching (#10)?" → In regex, * quantifies the preceding char; here * stands alone
  • "What if * can match at most k characters?" → Change dp[i-1][j] to only look back k rows
  • "Implement top-down with memoisation?" → Same recurrence, add @lru_cache to recursive function

7 Summary & Key Takeaways ▾
dp[i][j] = "first i chars of s match first j chars of p?" — the canonical 2D string DP state
Star case is OR of two cells: dp[i][j-1] (empty match) and dp[i-1][j] (eat one char)
Letter/? case takes the diagonal only: dp[i-1][j-1]
Base row: empty s matches a prefix of p only if that prefix is all stars
Space reduces to O(n) with rolling prev/curr arrays — same O(mn) time
Critical distinction from Regex (#10): here * is standalone, not a quantifier
ApproachTimeSpaceNotes
Brute Force (recursion) O(2m+n) O(m+n) TLE — revisits (i,j) pairs exponentially
2D DP ✓ (Interview Pick) O(m·n) O(m·n) Clear state, easy to explain
Space-Optimised DP O(m·n) O(n) Rolling prev/curr rows
🎯 When to Recognise the 2D String DP Pattern
  • Two strings, "does A match/transform to B?" — 2D DP on prefixes
  • Pattern matching where wildcards create ambiguous choices — DP over all splits
  • Must match the entire string (not substring) — dp[m][n] is the answer
  • Each character has OR of sub-results → True if any branch works
