LeetCode #32

Longest Valid Parentheses

Given a string containing only '(' and ')', return the length of the longest valid (well-formed) parentheses substring.

Hard Stacks Stack Index Trick Time: O(n) Space: O(n)
1 Understanding the Problem ▾

We have a string of only '(' and ')' characters, possibly mixed with mismatched brackets. We want the length of the longest contiguous substring that forms a valid, well-matched parentheses sequence.

💡 Key Insight

"Valid" means every '(' has a corresponding ')' after it, and vice versa — with no unmatched brackets breaking the run. Matched pairs can nest and chain, but they must be contiguous (no gaps).

More examples:

  • "(()())" → entire string is valid, answer = 6
  • "))))" → no valid pairs at all, answer = 0
  • "" → empty string, answer = 0
  • "()()" → chained pairs, answer = 4

Edge Cases: empty string (return 0), all '(', all ')', alternating mismatches like ")()()()", deeply nested "(((())))", and single characters.

2 The Algorithm: Stack Index Trick ▾

The key idea: instead of pushing characters onto a stack, push indices. The stack always contains the index of the most recent unmatched character — the "wall" that defines where any current valid run started from.

1 Initialize: stack = [-1], max_len = 0. The -1 is a sentinel wall before position 0. It ensures length calculations work even when a valid sequence starts from the very beginning of the string.
2 Scan left to right. For each character at index i:
3 If ch == '(': Push index i. This open bracket waits on the stack until its matching ')' arrives.
4 If ch == ')': Pop the stack. We've either consumed a matching '(' or the sentinel wall.
5 After pop — if stack is non-empty: A wall still exists. The current valid run extends from stack[-1] + 1 to i. Length = i - stack[-1]. Update max_len.
6 After pop — if stack is empty: This ')' is completely unmatched. Push i as the new wall. No future valid substring can start before this point.
7 Return max_len. After processing all characters, we have the answer.
🔑 Why -1 as the Sentinel?

If a valid substring starts at index 0, we need i - stack[-1] to give the full length. With sentinel -1: match at index 1 gives 1 - (-1) = 2 — exactly the length of "()". Without it, we'd need a special case for substrings starting at index 0.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every point during the scan, stack[-1] holds the index of the most recent unmatched character — either an unmatched '(' (waiting for a partner) or an unmatched ')' (that broke a potential sequence). Every valid substring we measure must fit entirely after this boundary.

Why length = i - stack[-1] captures the full run (not just one pair):

  • When we pop at ')' at index i, we matched one pair.
  • The new stack[-1] is the index just before the current valid run's start.
  • i - stack[-1] = end − (one before start) = the entire contiguous valid length, including any previously matched and chained pairs.
Chain: "()()" → length 4

i=1 '(': push → [-1,0,1]
i=2 ')': pop 1 → [-1,0]
  len=2-0=2, max=2
i=3 ')': pop 0 → [-1]
  len=3-(-1)=4, max=4 ✓

⚠️ The Critical Pop-then-Check Order

We always pop first, then check if the stack is empty. This is because when we pop: (a) if the stack had a '(' index on top, we consumed the match; (b) if the stack only had the sentinel/wall, we consumed that — making the stack empty, so we push the current ')' as the new wall. Pop-first handles both cases uniformly.

Correctness on edge cases:

  • "(((" → push 0, 1, 2. No pops. max stays 0. ✓
  • ")))" → i=0: pop -1, empty → push 0. i=1: pop 0, empty → push 1. i=2: pop 1, empty → push 2. max=0. ✓
  • "()" → i=0: push 0. i=1: pop 0, stack=[-1], len=1-(-1)=2. max=2. ✓
4 The Code (Annotated) ▾

Solution 1 — Stack Index Trick (Optimal, O(n) time, O(n) space) — Interview Pick

def longestValidParentheses(s: str) -> int:
stack = [-1] ← Sentinel wall at virtual position -1, before the string
max_len = 0 ← Track best answer found so far
for i, ch in enumerate(s): ← Scan every character with its index
if ch == '(': ← Open bracket: save position for future matching
stack.append(i) ← Push index i; we'll pop it when matching ')' found
else: ← Close bracket: try to complete a pair
stack.pop() ← Pop — either the matching '(' index, or the current wall
if stack: ← A wall still exists → we have a valid run
max_len = max(max_len, i - stack[-1]) ← Length = end - (one before the valid run start)
else: ← Stack empty → this ')' is completely unmatched
stack.append(i) ← Push as new wall; future substrings can't cross this position
return max_len ← Best valid length found across the entire string
⚠️ Common Confusion: What Gets Popped?

When we see ')' and pop, what comes off the stack? It could be: (1) a '(' index we pushed earlier — meaning we just matched a pair. Or (2) the sentinel/wall we previously pushed. After popping, if the stack is non-empty, we have a valid run. If empty, this ')' is unmatched and becomes the new wall.

Solution 2 — Dynamic Programming (O(n) time, O(n) space)

Define dp[i] = length of the longest valid substring ending exactly at index i. Only ')' can end a valid substring; all dp[i] where s[i]=='(' stay 0.

def longestValidParentheses(s: str) -> int:
if not s: return 0
dp = [0] * len(s) ← dp[i] = longest valid ending at i
for i in range(1, len(s)):
if s[i] == ')': ← Only ')' can end a valid sequence
if s[i-1] == '(': ← Case 1: "…()" — direct adjacent pair
dp[i] = (dp[i-2] if i >= 2 else 0) + 2 ← Extend valid run before the '('
elif dp[i-1] > 0: ← Case 2: s[i-1]=')' and it ended a valid block
j = i - dp[i-1] - 1 ← Jump past the inner valid block to find its opening '('
if j >= 0 and s[j] == '(': ← If there's a matching '(' for our current ')'
dp[i] = dp[i-1] + 2 + (dp[j-1] if j > 0 else 0) ← Inner block + new pair + prefix valid
return max(dp) ← Best value across all positions

Solution 3 — Two-Pass Counters (O(n) time, O(1) space)

Scan L→R counting left and right: when balanced, record length; when right > left, reset. Then scan R→L to catch the "excess open brackets" case.

def longestValidParentheses(s: str) -> int:
left = right = max_len = 0
for ch in s: ← Left-to-right pass
if ch == '(': left += 1
else: right += 1
if left == right: max_len = max(max_len, 2 * right) ← Balanced → valid!
elif right > left: left = right = 0 ← More ')' than '(' → reset
left = right = 0
for ch in reversed(s): ← Right-to-left pass (catches leading unmatched '(' case)
if ch == '(': left += 1
else: right += 1
if left == right: max_len = max(max_len, 2 * left)
elif left > right: left = right = 0 ← In reverse: excess '(' is the new blocker
return max_len
💡 Why Two Passes in the Counter Approach?

The L→R pass misses the case where there are more '(' than ')' — we'd never trigger a reset and miss some valid substrings inside. The R→L pass catches this by treating excess '(' (from the reverse direction) as the blocker instead.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Bracket Pattern → Stack

"I see only '(' and ')'. Bracket matching always calls for a stack. But I need to measure length, not just check validity — that's the extra challenge here."

Step 2: Characters vs. Indices on the Stack

"If I push characters, I can only tell whether something matched, not how long the valid run is. Pushing indices lets me compute length as i - stack[-1] after each pop."

Step 3: The Sentinel / Wall Insight

"I need a boundary before position 0. I seed the stack with -1 so that a match starting at index 0 correctly computes length = i - (-1) = i + 1. Unmatched ')' also push their index as new boundaries."

Step 4: Trace a Non-Trivial Example Live

"Let me trace ')()())': stack=[-1]. i=0 ')': pop→empty, push 0, stack=[0]. i=1 '(': push 1. i=2 ')': pop 1, stack=[0], len=2-0=2. i=3 '(': push 3. i=4 ')': pop 3, stack=[0], len=4-0=4. i=5 ')': pop 0, empty→push 5. Answer = 4. ✓"

🗣️ What to Say to the Interviewer

"This is a bracket matching problem — stack is the right tool.
The twist: I need to measure the LENGTH of valid runs, not just validate.
I'll push indices instead of characters. I initialize with [-1] as a sentinel wall.
For '(': push the index. For ')': pop. If stack is non-empty, length = i - stack[-1].

🔮 Likely Follow-up Questions
  • "Can you do O(1) space?" → Two-pass left-right counter approach

7 Summary & Key Takeaways ▾
Push indices, not characters — enables length measurement by subtraction
Seed the stack with -1 as a sentinel wall before the string begins
Always pop first on ')'. Then: if stack non-empty → measure length; if empty → push new wall
