LeetCode #2390

Removing Stars From a String

Process a string left-to-right: each * removes the nearest non-star character to its left. Return the resulting string after all stars are removed.

Medium Stacks Stack Pop on Star Time: O(n) Space: O(n)
1 Understanding the Problem ▾

You have a string containing lowercase letters and * characters. Each * acts like a backspace key — it deletes the nearest non-star character to its left. The problem guarantees that a valid deletion always exists (no star appears without a prior letter to erase).

💡 Key Insight

This is exactly the backspace operation in a text editor. Imagine typing the string one character at a time. Letters build up your text; each * presses Backspace. A stack models this perfectly: push letters onto it, pop the top when you see a star. The stack always holds the characters that have survived so far.

2 The Algorithm: Stack as a Backspace Buffer ▾

We simulate the typing process with a stack that holds only the characters that survive to the final result. Each character triggers exactly one O(1) operation. At the end, the stack from bottom to top is exactly the answer.

1 Initialize an empty list stack = []. Python lists give us O(1) amortized append and pop from the right end.
2 Scan left to right: For each character c in s, ask: is this a letter or a star?
3 Letter → push: stack.append(c). The letter joins the survivors. It is now the "most recently typed character" — i.e., the stack top.
4 Star → pop: stack.pop(). The star erases the most recent survivor (the top). The problem guarantees the stack is non-empty when we pop.
5 Join and return: "".join(stack) reads the stack left-to-right (bottom-to-top), producing the final string in correct document order.
🔑 Why a List (Stack) and Not a String?

Python strings are immutable. Building a result with res += c or res = res[:-1] creates a brand-new string object each time — O(n) per operation, O(n²) total. A list's append() and pop() are O(1) amortized, making the whole algorithm O(n). One final "".join(stack) at the end is O(n) and unavoidable — but it only happens once.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every point during the scan, the stack contains exactly the characters that would appear in the final result if we stopped processing right now. Equivalently: the stack holds all letters seen so far that have NOT yet been erased by a subsequent star.

Why popping is always correct: When we encounter a *, the problem says it erases "the closest non-star character to its left." The stack top is always the most recently pushed letter — which is exactly the closest surviving character to the left of our current position. Popping the top is precisely the correct operation.

Cascading stars handled automatically: If you see two stars in a row (**), the first pop removes the most recent letter, and then the second star sees a new top (the second-most-recent surviving letter) and pops that. No special logic is needed; the stack structure handles cascading deletions naturally.

⚠️ Common Mistake: String Slice in a Loop

Some code you might write first: res = res[:-1] if c == '*' else res + c. This is logically correct but O(n²). Each res[:-1] allocates a new string of length n−1, copying all characters. On a 100K-character input with 50K stars, that is 50K allocations of average 50K characters each — roughly 2.5 billion character copies. Switch to a list immediately when you need a mutable sequence.

Edge cases covered by the invariant:

  • No stars: Every character is pushed, nothing is popped → result = input string
  • All erased: Stack empties out → "".join([]) == ""
  • Stars at end: Letters pushed first, then all popped in reverse order
  • Single char + star: Push char, pop it → empty stack → empty result
4 The Code (Annotated) ▾

Solution 1 — Stack / List Buffer (Optimal) ← Interview Pick

def removeStars(s: str) -> str:
stack = [] ← Mutable buffer; tracks surviving characters in order
for c in s: ← Single left-to-right scan, O(n) total
if c == '*': ← Star: erase the closest surviving character to the left
stack.pop() ← O(1) amortized — removes the rightmost element in place
else: ← Regular letter: add to the surviving result
stack.append(c) ← O(1) amortized — list grows dynamically
return "".join(stack) ← One final O(n) join; reads bottom-to-top = left-to-right
💡 Why "".join(stack) gives the right order

We always append to the right end and pop from the right end. So the list grows from left (oldest) to right (newest). "".join(stack) reads left-to-right, which is exactly the left-to-right document order of surviving characters.

Solution 2 — String Slice Loop (O(n²) — do NOT use)

def removeStars(s: str) -> str:
res = "" ← Immutable string — every change allocates a new object
for c in s:
if c == '*':
res = res[:-1] ← Creates a new string copy of len(res)-1 — O(n) per call!
else:
res += c ← Also O(n) per call without CPython's intern optimization
return res ← Total: O(n²) — too slow for large inputs
💡 Complexity Summary

Stack (list): Time O(n), Space O(n) — each character is pushed at most once and popped at most once. Total operations ≤ 2n.
String slice: Time O(n²), Space O(n) — correct but unacceptable for large inputs.
In-place two-pointer: Time O(n), Space O(1) — advanced follow-up; works on a char array.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the "Closest to Left" Signal

"Closest non-star character to its left" — the word closest and left together scream stack. The stack top is always the closest character to the left because we process left-to-right and push in order.

Step 2: Reframe as Text Editor

Say aloud: "This is exactly how a text editor cursor works — letters type a character, stars press Backspace." Text editor cursor state = stack. This mental model instantly gives you the data structure.

Step 3: Explain Why List and Not String

Proactively mention: "I'll use a list as a mutable buffer, not a string, because string concatenation and slicing are O(n) each in Python — that gives O(n²) overall. List append/pop are O(1)."

Step 4: Confirm Edge Cases and Complexity

Each character triggers exactly one operation (push or pop). O(n) total. Space is O(n) for the stack in the worst case (no stars). Mention the problem guarantees valid deletions — no need to guard empty stack.

🗣️ What to Say to the Interviewer

"The phrase 'closest non-star character to its left' immediately signals a stack.
The stack top is always the most recent surviving character — exactly the closest one to the left.
So: iterate left to right. Letter → push. Star → pop. Return ''.join(stack) at the end.
I'm using a list as my stack so append and pop are O(1), giving O(n) overall.
String slice (res[:-1]) would be O(n²) — creating a new string each time. I'll avoid that.
No guard needed on pop because the problem guarantees a valid deletion always exists."

  • "Can you do it in O(1) space?" → Two-pointer in-place on a char array: write-pointer advances on letter, retreats on star. Same O(n) time, O(1) extra.
