Ugly Number

Determine if n is an ugly number — a positive integer whose only prime factors are 2, 3, and 5.
Easy Math · Divide by 2, 3, 5 Time O(log n) Space O(1)
1 Understanding the Problem ▾

An ugly number is a positive integer whose only prime factors are 2, 3, and 5. By convention, 1 is ugly (the empty product has no prime factors at all).

Examples: 6 = 2 × 3 (ugly), 12 = 2² × 3 (ugly), 30 = 2 × 3 × 5 (ugly), 1 (ugly by definition).
Not ugly: 14 = 2 × 7 — because 7 is a prime factor not in {2, 3, 5}.

Walk-through by hand:
n = 12: 12 ÷ 2 = 6, 6 ÷ 2 = 3, 3 ÷ 3 = 1. Reached 1 → ugly.
n = 14: 14 ÷ 2 = 7. 7 is not divisible by 2, 3, or 5 — stuck at 7 ≠ 1 → not ugly.
n = 1: no division needed, remainder = 1 → ugly (edge case).

💡 Key Insight

An ugly number, after you strip out every factor of 2, 3, and 5, must equal exactly 1. If anything > 1 remains, that leftover must contain at least one prime not in {2, 3, 5} — meaning n is not ugly.

⚠️ Edge Cases to Know

n ≤ 0: Ugly numbers are positive by definition. Return False for 0 and negatives.
n = 1: Special case — 1 has no prime factors at all, and the convention is "ugly". The algorithm naturally returns True (1 == 1 after the loop).
Large n (e.g., 2³⁰): Still works — each division cuts n in half at minimum; terminates in O(log n) steps.

2 The Algorithm: Repeated Division ▾

The approach is called Repeated Division by Allowed Primes. We methodically divide n by 2, 3, and 5 — each as many times as possible — and observe what remains. If n started with only factors of 2, 3, and 5, after removing them all we must be left with 1.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every iteration, n equals its original value divided by all the 2s, 3s, and 5s removed so far. We are systematically peeling away "allowed" prime factors. Once we cannot peel any more allowed factor, what remains is either 1 (all factors were allowed) or some value > 1 containing at least one disallowed prime.

Fundamental Theorem of Arithmetic: Every integer > 1 has a unique prime factorization. By dividing out all occurrences of 2, 3, and 5, we partition the prime factors into "allowed" and "other." If "other" is empty, the product is 1.

Order does not matter: We can remove 2s, 3s, 5s in any order. Divisibility by 3 is unaffected by dividing by 2. The operations commute.

Termination guarantee: Every division n = n ÷ p strictly reduces n (since p ≥ 2, n/p ≤ n/2 < n). The total number of divisions is at most log₂(n) + log₃(n) + log₅(n), which is O(log n).

💡 Analogy: Recipe Taste Test

Imagine a "clean" recipe that uses only salt (2), pepper (3), and olive oil (5). To verify a dish is "clean," you taste out all the salt, all the pepper, and all the olive oil. If nothing remains, the chef used only the allowed ingredients. If some mystery flavor lingers, something unauthorized was added.

4 The Code (Annotated) ▾
def isUgly(n: int) -> bool:
if n <= 0: return False ← Ugly numbers are positive; reject 0 and negatives immediately
for p in (2, 3, 5): ← Iterate over the three allowed prime factors in order
while n % p == 0: ← Keep dividing while p still divides n exactly (remainder is zero)
n //= p ← Strip one factor of p; integer division guaranteed exact here
return n == 1 ← If n reached 1, no foreign primes existed → ugly. Otherwise not.
⚠️ Common Mistake: Forgetting n ≤ 0

If you skip the guard, n = 0 enters the for-loop. Since 0 % 2 == 0 is always True, you'd compute 0 ÷ 2 = 0 → infinite loop! Always guard non-positive n first.

⚠️ Common Mistake: Returning n == 0

After the loop, you want to check n == 1, not n == 0. The loop ends when n can no longer be divided; the residue is 1 for ugly numbers and some integer > 1 for non-ugly ones. Zero is never the result of any valid run.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

The phrase "only prime factors are X, Y, Z" is the signal. This is a divisibility and factorization problem. Whenever you need to check whether a number belongs to a set defined by its prime factors, think: divide out allowed primes one by one and check the remainder.

Step 2: Handle Edge Cases Upfront — Say It Out Loud

Tell the interviewer: "Since the problem says n is a positive integer, I return False for n ≤ 0. And n = 1 is ugly by convention — my code returns 1 == 1 = True without any division, which is correct."

Step 3: Trace an Example Live

n = 12: 12 → 6 → 3 (two ÷2 steps), 3 → 1 (one ÷3), 1 % 5 ≠ 0 (skip), return 1==1=True.
n = 14: 14 → 7 (one ÷2), 7 % 3 ≠ 0, 7 % 5 ≠ 0, return 7==1=False.

Step 4: State Complexity Confidently

Total inner-loop iterations ≤ log₂(n) + log₃(n) + log₅(n) = O(log n). Space is O(1). For n up to 2³¹ ≈ 2 × 10⁹, this runs in at most ~60 divisions.

🗣️ What to Say to the Interviewer

"An ugly number has only 2, 3, and 5 as prime factors. My approach: strip all factors of 2, then 3, then 5 via repeated division. After stripping, if n == 1, every factor was in the allowed set, so n is ugly. If n > 1 remains, some prime outside {2,3,5} divides n, so it's not ugly. Edge cases: n <= 0 returns False (positive by definition), n = 1 returns True (empty product = ugly by convention). Time is O(log n), space O(1)."
🧩 Likely Follow-Up Questions

"What if the allowed primes were {2, 7, 11}?" — Same algorithm, just loop over (2, 7, 11) instead of (2, 3, 5).
"Find the nth ugly number." (LC #264) — Different problem requiring three-pointer min-heap or DP.
"What if n can be 10^18?" — Python handles big ints natively; algorithm is unchanged. In C++, use long long.
