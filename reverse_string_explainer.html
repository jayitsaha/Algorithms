Reverse String

Two Pointers Swap — The Classic In-Place Reversal

#344 Easy Two Pointers Swap String Processing O(n) Time · O(1) Space
1 Understanding the Problem

Given an array of characters s, reverse it in-place with O(1) extra memory. This means we can't create a new array — we must modify the existing one.

Example: ["h", "e", "l", "l", "o"] → ["o", "l", "l", "e", "h"]

💡 Key Insight: Meet in the Middle

To reverse, we need to swap pairs of elements: first with last, second with second-to-last, and so on. We stop when the two pointers meet in the middle!

  • Swap s[0] ↔ s[n-1]
  • Swap s[1] ↔ s[n-2]
  • ...continue until left ≥ right
🔑 Why Two Pointers?

With one pointer from each end moving toward the center, we naturally visit all swap pairs. This is the most intuitive in-place reversal pattern and requires no extra space beyond the two index variables.

2 The Algorithm: Two Pointers Swap

The algorithm is beautifully simple: use two pointers converging from opposite ends.

1 Initialize pointers: Set left = 0 (start) and right = n - 1 (end).
2 While left < right: Swap s[left] and s[right].
3 Move pointers: Increment left, decrement right.
4 Stop when they meet: When left >= right, the entire array is reversed.

⚠️ Why left < right, not left <= right?

If we used <=, when there's an odd number of elements, we'd try to swap the middle element with itself — harmless but wasteful. < correctly stops when pointers meet or cross.

3 Why It Works: The Invariant
🔐 The Invariant

After each iteration:

  • Elements at indices [0, left) and (right, n-1] are in their final reversed positions.
  • Elements at indices [left, right] are yet to be processed.

When left >= right, all elements have been swapped — reversal complete!

📝 Proof Sketch
  • Base case: Initially, no elements processed. Invariant holds trivially.
  • Induction: Each swap places two more elements in final position. Pointers move inward.
  • Termination: Each iteration decreases (right - left) by 2. We stop when difference ≤ 0.
  • Correctness: Every index pair (i, n-1-i) gets swapped exactly once.
🧮 Number of Swaps

For array of length n, we perform ⌊n/2⌋ swaps:

  • Even n: exactly n/2 swaps
  • Odd n: (n-1)/2 swaps (middle element stays in place)
4 The Code (Annotated)
def reverseString(s):
left, right = 0, len(s) - 1 # Two pointers at ends
while left < right: # Until they meet
s[left], s[right] = s[right], s[left] # Swap
left += 1 # Move left inward
right -= 1 # Move right inward
Time
O(n)
n/2 swaps, each O(1)
Space
O(1)
Only two index variables
✨ Python One-Liner (Alternative)

s[:] = s[::-1] — This also works in-place because of slice assignment, but the two-pointer approach is more explicit and educational.

5 🎤 Interview Strategy: How to Arrive at This
Step 1: Understand the Constraint

"In-place with O(1) space means I can't create a new array. I need to modify the existing array by swapping elements."

Step 2: Identify Swap Pairs

"To reverse, s[0] becomes s[n-1], s[1] becomes s[n-2], etc. So I need to swap pairs (0, n-1), (1, n-2), ..."

Step 3: Recognize Two Pointers

"Two pointers from opposite ends naturally generate these pairs! They converge toward the center."

Step 4: Handle Termination

"Stop when left >= right — at that point all pairs have been swapped."

🗣️ What to Say to the Interviewer
"To reverse in-place with O(1) space, I'll use two pointers — one starting at the beginning and one at the end. At each step, I swap the characters at these positions, then move both pointers toward the center. I continue until the pointers meet or cross. This gives O(n) time since I visit each element once, and O(1) space since I only use two index variables."
🔄 Follow-up Questions
  • "What if it's a string (immutable in Python)?" — Convert to list, reverse, convert back. Or use slicing s[::-1].
  • "Reverse words in a sentence?" — Reverse entire string, then reverse each word individually.
  • "Reverse only vowels?" — Two pointers, but skip non-vowels.

7 Summary & Key Takeaways
✓ Two pointers from opposite ends naturally generate swap pairs
✓ Stop when left >= right (pointers meet or cross)
✓ Each swap places TWO elements in final position
✓ ⌊n/2⌋ swaps total — O(n) time, O(1) space
✓ The invariant: outside pointers = done, inside = pending
🎯 When to Use Two Pointers (Opposite Direction)

  • Palindrome checking
  • Two Sum in sorted array
  • Container with most water
  • Any problem requiring pairs from opposite ends
