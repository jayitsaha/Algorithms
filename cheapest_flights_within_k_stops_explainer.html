LeetCode #787

Cheapest Flights Within K Stops

Find the cheapest price from source to destination using at most K intermediate stops. Return -1 if no such route exists.

Medium Graph Bellman-Ford Time: O(K·E) Space: O(n)
1 Understanding the Problem ▾

You are given n cities numbered 0 to n-1, and a list of flights where each entry is [from, to, price]. Starting from city src, you want to reach city dst using at most k stops (intermediate cities). Find the cheapest total cost, or -1 if it's impossible.

💡 Key Insight

This is a constrained shortest path problem. The constraint is not just on cost but on number of edges used. "K stops" means K intermediate cities, so the path uses at most K+1 flights. Bellman-Ford's round structure maps perfectly to this: after round i, we have the cheapest path using at most i flights.

✓ Example: k=1 allowed

0→1→3: 100+300 = 400 ← 1 stop
0→2→3: 500+100 = 600 ← 1 stop
Best: 400

✗ k=0: direct flights only

0→3 doesn't exist directly.
No path with 0 stops → -1

Key terminology: A "stop" is an intermediate city you land at between source and destination. A path with 1 stop uses 2 flights. A path with k stops uses k+1 flights. When k=0, you can only take a single direct flight.

Edge cases: src == dst (price 0), no path within k stops (return -1), multiple paths with same cost (any is valid), k large enough to allow all paths (standard shortest path).

2 The Algorithm: Bellman-Ford with K+1 Rounds ▾

Bellman-Ford normally relaxes all edges n-1 times to find shortest paths. Here, we run it exactly k+1 times — one round per additional allowed flight. After round i, dist[v] holds the cheapest cost using at most i flights.

1 Initialize: Create dist[0..n-1] = ∞ and set dist[src] = 0. City src costs zero to reach — it's where we start.
2 Outer loop — k+1 rounds: Run the relaxation loop exactly k+1 times. Each iteration represents "taking one more flight."
3 Snapshot before each round: Copy dist into a temporary array temp = dist[:]. All reads in this round use dist (frozen), all writes go to temp. This prevents cascading updates within one round.
4 Relax every edge: For each flight [u, v, w], if city u is reachable (dist[u] != ∞), try temp[v] = min(temp[v], dist[u] + w).
5 Commit the round: Set dist = temp. After all k+1 rounds, return dist[dst] if not ∞, else -1.
🔑 The Bellman-Ford Algorithm — Origin and Invariant

Origin: Richard Bellman (1958) and Lester Ford (1956) independently discovered this algorithm. It finds shortest paths in weighted directed graphs, including those with negative edge weights.

Standard invariant: After pass i of relaxing all edges, dist[v] holds the shortest path to v using at most i edges. This round-by-round structure is exactly what makes it perfect for "at most K edges" problems.

Why K stops = K+1 flights: Stops are intermediate cities. Source and destination don't count as stops. So k=1 means 1 intermediate city = 2 flights = 2 rounds needed.

3 Why It Works: The Invariant ▾
🔐 The Round-Based Invariant

After completing round i: dist[v] = minimum cost to reach city v from src using at most i flights. The copy-before-relax enforces this.

Why the snapshot (copy) is essential: Suppose we update dist[1] early in round 0, then later in the same round try to extend from node 1 to node 3. Without the copy, we'd read the newly updated dist[1] and effectively allow 2 flights in 1 round — that would be wrong. The frozen snapshot dist[u] guarantees we only extend paths that existed at the start of this round.

Tracing the invariant on our example (src=0, k=1):

  • Start: dist = [0, ∞, ∞, ∞]. City 0 reachable via 0 flights.
  • After Round 0 (1 flight): dist = [0, 100, 500, ∞]. Cities directly reachable from 0.
  • After Round 1 (2 flights): dist = [0, 100, 200, 400]. City 3 reached via 0→1→3 for 400.
⚠️ Off-by-One: k Stops vs k+1 Flights

The problem says "at most k stops." Stops ≠ flights. If k=1, you can make 1 stop → 2 flights → run 2 rounds (range(k+1)). Using range(k) would give you only k flights = k-1 stops. This is the most common bug — always loop range(k+1).

🔑 Why Not Dijkstra?

Dijkstra expands the globally cheapest node next. It finds the cheapest cost, but doesn't naturally count how many edges were used. You'd need to embed stop-count into the state: (cost, city, stops_remaining). This modified Dijkstra works but is more complex to code correctly. Bellman-Ford's round structure gives us the hop limit for free.

4 The Code (Annotated) ▾

Solution 1 — Bellman-Ford K Rounds (Interview Pick)

def findCheapestPrice(n, flights, src, dst, k):
INF = float('inf') ← Sentinel: "city not yet reachable"
dist = [INF] * n ← Cheapest cost to each city using ≤ i flights so far
dist[src] = 0 ← Source costs zero — we start here
for i in range(k + 1): ← k stops = k+1 flights = k+1 rounds
temp = dist[:] ← Snapshot! Read from dist, write to temp (no in-round chaining)
for u, v, w in flights: ← Try every available flight
if dist[u] < INF: ← Only extend from reachable cities
temp[v] = min(temp[v], dist[u] + w) ← Can we reach v cheaper via u?
dist = temp ← Commit round: this round's snapshot becomes next round's base
return dist[dst] if dist[dst] < INF else -1 ← Unreachable = -1
🚨 Critical Bug: Reading temp[u] Instead of dist[u]

Writing temp[v] = min(temp[v], temp[u] + w) instead of dist[u] + w is wrong. If you updated temp[u] earlier in this same loop iteration, you'd chain two flights in one round — allowing k+2 flights instead of k+1. Always read from the frozen dist and write to temp.

Solution 2 — Modified Dijkstra with Stop State

import heapq
from collections import defaultdict
def findCheapestPrice(n, flights, src, dst, k):
graph = defaultdict(list) ← Adjacency list: city → [(neighbor, price)]
for u, v, w in flights:
graph[u].append((v, w))
heap = [(0, src, 0)] ← (total_cost, current_city, stops_used)
best = {} ← (city, stops) → min cost to avoid revisiting
while heap:
cost, u, stops = heapq.heappop(heap)
if u == dst: return cost ← Heap is min-ordered: first arrival is cheapest
if stops > k: continue ← Exceeded stop budget — prune
if best.get((u, stops), float('inf')) <= cost: continue
best[(u, stops)] = cost
for v, w in graph[u]:
heapq.heappush(heap, (cost + w, v, stops + 1))
return -1
💡 Complexity Comparison

Bellman-Ford K Rounds: Time O((k+1) × E), Space O(n). Simple, correct, interview-friendly.
Modified Dijkstra: Time O(E × k × log(n×k)), Space O(n×k). Faster in practice on sparse graphs but more complex to implement correctly.
For most interviews, Bellman-Ford K Rounds is the expected answer.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Pattern Signals

Keywords: "cheapest", "flights", "at most k stops" = weighted directed graph + shortest path + edge-count constraint. The edge-count constraint is the twist that rules out standard Dijkstra.

Step 2: Propose and Dismiss Brute Force

"I could explore all paths using DFS, tracking cost and stops remaining. But that's exponential — O(n^k) in the worst case. I need something smarter."

Step 3: Recognize Bellman-Ford Round Structure

"Bellman-Ford naturally expresses the invariant I need: after round i, I have shortest paths using at most i edges. I just run it k+1 times and use a copy-before-relax to prevent same-round chaining."

Step 4: Be Explicit About the Off-By-One

"k stops = k intermediate cities = k+1 total flights = k+1 rounds. I'll use range(k+1)." Say this out loud — interviewers love seeing you reason carefully about boundary conditions.

🗣️ What to Say to the Interviewer

Standard Dijkstra doesn't track how many edges a path uses, so I'll use Bellman-Ford.

🔮 Likely Follow-up Questions
  • "What if some prices are negative?" → Bellman-Ford handles negative weights; Dijkstra does not. (However, LeetCode guarantees non-negative prices here.)
  • "Can you use Dijkstra?" → Yes, with state (cost, city, stops_used) in the heap. More complex but faster in practice.
  • "What's the space complexity if k is large?" → Still O(n) — we only keep the current dist array, not all previous rounds.
  • "What if we also want the actual path?" → Track predecessors: prev[v] = u when temp[v] is updated. Reconstruct by following prev from dst back to src.

7 Summary & Key Takeaways ▾
k stops = k+1 flights = run Bellman-Ford for k+1 rounds (loop range(k+1))
Always copy dist into temp before each round — prevents same-round edge chaining
Read from dist (frozen snapshot), write to temp — this is the round-isolation invariant
After round i, dist[v] = cheapest path to v using at most i flights
If dist[dst] == ∞ after all rounds, no valid path exists — return -1

🎯 When to Recognize the Bellman-Ford K Iterations Pattern
  • "Cheapest / shortest path" + "at most K edges / stops / hops" → Bellman-Ford K rounds
  • Negative weights allowed → Bellman-Ford is the only safe choice
  • No stop constraint + non-negative weights → standard Dijkstra
  • DAG with hop limit → DP on layers (equivalent approach)
