LeetCode #152

Maximum Product Subarray

Given an integer array nums, find a contiguous subarray with the largest product, and return that product.

Medium Kadane's Algorithm Track Max & Min Time: O(n) Space: O(1)
1 Understanding the Problem ▾

You are given an array of integers — positive, negative, and possibly zero. Find the contiguous subarray (at least one element) whose elements multiply together to give the largest possible product. Return that product value (not the subarray itself).

💡 Key Insight

This looks like "Maximum Subarray" (Kadane's) but with products. The critical difference: two negatives multiply to a positive. A deeply-negative running product can instantly become the new maximum when multiplied by another negative. We can never safely discard a negative running product — it's a future opportunity. This forces us to track both the running maximum AND the running minimum at every step.

Key edge cases:

  • Zero: Any subarray containing a zero has product 0. Zero resets both running trackers.
  • Even count of negatives: They all cancel — result is likely the full array product.
  • Odd count of negatives: We exclude one negative (the leftmost or rightmost) to maximize.
  • Single element: Answer is that element, even if negative.
  • All same sign: If all positive — take the whole array. If all negative — take the pair with greatest product.
2 The Algorithm: Dual Kadane's (Track Max & Min Product) ▾

Standard Kadane's tracks the best subarray ending at each position. For products, we track two values at every position: the maximum product subarray ending here, and the minimum product subarray ending here. The minimum matters because a big negative flipped by another negative becomes a big positive.

1 Initialize: Set max_prod = min_prod = result = nums[0]. Seed with the first element. This handles the length-1 case and gives both trackers a starting point.
2 Scan from index 1: For each element num, compute all three candidates for the new max/min ending here: (a) num alone — start a fresh subarray, (b) max_prod × num — extend the max subarray, (c) min_prod × num — extend the min subarray (this becomes the new max if num is negative).
3 Compute all candidates at once: candidates = (num, max_prod * num, min_prod * num). Grouping them in a tuple ensures we use the old values of both max_prod and min_prod when computing the new ones.
4 Update running trackers: max_prod = max(candidates), min_prod = min(candidates). The new max is the best of all three candidates; the new min is the worst (most negative).
5 Update global result: result = max(result, max_prod). The answer is the best max_prod ever seen across all positions — not just the final one.
Why Track Minimum?

A large negative running product, when multiplied by another negative element, becomes a large positive — potentially the new global maximum. Discarding the minimum means missing subarrays like [-2, 3, -4] → 24.

The Three Candidates

1. num — start fresh here
2. max_prod × num — extend max
3. min_prod × num — flip the min

🔑 Algorithm Deep-Dive: Kadane's Algorithm (Adapted)

Origin: Joseph Kadane, 1984. Originally designed to find the maximum sum subarray in O(n) time and O(1) space. The core idea: at each position, the best subarray ending here is either the current element alone (restart) or the previous best extended by the current element (continue).

Core invariant: After processing index i, max_prod holds the maximum product of any non-empty contiguous subarray ending at index i.

Adaptation for products: Since multiplication is not monotone (negatives can flip the order), we maintain both the maximum and minimum simultaneously. The minimum today is the maximum's best friend tomorrow if the next element is negative.

Recognize when: "Maximum/minimum value of some function over all contiguous subarrays" — especially when the function can have sign-flipping behavior (products, XOR).

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every index i: max_prod holds the maximum product of any contiguous subarray ending exactly at index i, and min_prod holds the minimum product (most negative). The global result tracks the overall best max_prod seen across all positions 0..i.

Why are exactly three candidates sufficient? The best subarray ending at index i either:

  • Starts at i (just nums[i] alone) — when the previous product worsens things, e.g., after a zero
  • Extends the max-product subarray ending at i−1: max_prod_prev × nums[i]
  • Extends the min-product subarray ending at i−1: min_prod_prev × nums[i] — only wins when nums[i] is negative

No other subarray ending at i can beat these three — because any subarray ending at i is one of these by definition of how products chain.

Proof by induction sketch: Base case i=0: all three variables equal nums[0], trivially correct. Inductive step: assume invariant holds at i−1. Then max(num, max_prev×num, min_prev×num) covers all possible subarrays ending at i, so the invariant holds at i. QED.

⚠️ The Most Common Bug: Updating in Place

If you compute max_prod = max(num, max_prod*num, min_prod*num) and then compute min_prod = min(num, max_prod*num, min_prod*num), the second line uses the already-updated max_prod. This corrupts the result. Always save both old values first, or compute all candidates into a tuple simultaneously before assigning either.

💡 Zero Handling: Automatic Reset

When nums[i] = 0, all three candidates become 0. So max_prod = 0, min_prod = 0. The next iteration naturally starts fresh with nums[i+1] as the sole candidate for either tracker. No special zero-handling code is needed — the algorithm handles it automatically!

4 The Code (Annotated) ▾

Solution 1 — Dual Kadane's, Track Max & Min (Optimal · Interview Pick · O(n) / O(1))

def maxProduct(nums: list[int]) -> int:
max_prod = min_prod = result = nums[0] ← seed all three with nums[0]; handles len=1 automatically
for num in nums[1:]: ← scan left to right, starting from index 1
candidates = (num, ← candidate 1: start a fresh subarray here
max_prod * num, ← candidate 2: extend the current max-product subarray
min_prod * num) ← candidate 3: flip the min (big negative × negative = big positive)
max_prod = max(candidates) ← best product of any subarray ending at current index
min_prod = min(candidates) ← worst product ending here — saved for future sign-flip
result = max(result, max_prod) ← update global best seen across all positions so far
return result ← best product across every possible contiguous subarray
💡 Why a tuple for candidates?

Wrapping all three products in a tuple before assigning guarantees we read the old max_prod and min_prod when computing all three. Python evaluates the right-hand side of an assignment completely before storing it — the tuple captures both old values simultaneously. No temp variables needed.

Solution 2 — Brute Force O(n²) / O(1) — for understanding only

def maxProduct(nums: list[int]) -> int:
result = nums[0] ← at minimum, a single element is a valid subarray
for i in range(len(nums)): ← try every start position
prod = 1
for j in range(i, len(nums)): ← try every end position
prod *= nums[j] ← extend subarray by one element
result = max(result, prod)
return result
⚠️ Why Brute Force Times Out

O(n²) — for n = 20,000 elements that's 400 million multiplications. Correct on small inputs; use it to verify your optimal solution. Never submit it for the real problem.

🔑 Complexity Summary

Dual Kadane's (optimal): Time O(n) · Space O(1) — one pass, three variables
Brute Force: Time O(n²) · Space O(1) — correct but too slow
Prefix Products (alternative O(n)/O(1)): Scan left-to-right and right-to-left simultaneously, tracking running products. Zeros auto-reset. Max of both scans gives the answer.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Kadane's Pattern

"This is Maximum Subarray but with products instead of sums. Kadane's should apply — but I need to think about what's different when we multiply."

Step 2: Spot the Negative-Flip Difference

"With sums, a negative number always hurts — I can restart when sum goes negative. With products, two negatives give a positive. A large negative running product is a sleeping giant — it can flip to the global maximum when hit by another negative."

Step 3: Derive the Dual-Tracker Solution

"Since I can't ignore the minimum, I'll track both. At each element, the new max comes from max(element alone, old_max×element, old_min×element). The old_min×element candidate handles the sign flip. I also need the new min for the next iteration."

Step 4: Mention the Tuple Trick for Correctness

"I'll compute all three candidates into a tuple before updating either tracker. This avoids the common bug of using a newly-updated value when computing the other."

🗣️ What to Say to the Interviewer

"This is similar to Maximum Subarray, but products have a key quirk: negatives can flip.
My Kadane's approach needs to track two running values: max_prod and min_prod at each position.
At each element, the new max is the best of: the element alone, old_max × element, old_min × element.
The third candidate handles the sign-flip case — a big negative times a negative becomes a big positive.
Global result tracks the best max_prod ever seen. Time O(n), Space O(1). No extra array needed."

7 Summary & Key Takeaways ▾
Track both max_prod and min_prod — negatives can flip the minimum into the new maximum
Always compute all candidates from old values before updating either tracker (use a tuple)
Zero automatically resets both trackers to 0; no special handling needed
The global result is the best max_prod ever seen — update it after every position
Time O(n), Space O(1) — one forward pass, three variables
🎯 When to Use This Pattern
  • Problem asks for maximum or minimum product over a subarray
  • Input contains both positives and negatives (sign-flipping is possible)
  • O(n) time and O(1) space are required
  • Any problem where a running value's extreme (minimum) could become the optimal if the operator changes sign
