LeetCode #214

Shortest Palindrome

Given a string s, find the shortest palindrome you can find by adding characters in front of it.

Hard String Processing KMP Time: O(n) Space: O(n)
1 Understanding the Problem ▾

You are given a string s and allowed to add any characters in front of it (prepend). Your goal is to make the result a palindrome while adding as few characters as possible.

💡 Key Insight

The shortest palindrome is formed by finding the longest palindromic prefix of s. The characters you must prepend are the reverse of the suffix that isn't part of that palindromic prefix.

✓ Example: s = "aacecaaa"

Longest palindromic prefix: "aacecaa"
Remaining suffix: "a"
Prepend reverse("a") = "a"
Result: "aaacecaaa"

Another: s = "abcd"

Longest palindromic prefix: "a"
Remaining suffix: "bcd"
Prepend reverse("bcd") = "dcb"
Result: "dcbabcd"

Why prepend and not append? We are forced to add only in front. Adding to the back would be a different (easier) problem. The challenge is that the front of the string must "extend" an already-palindromic prefix.

Edge Cases: Empty string → return "". String already a palindrome → return as-is. Single character → already palindrome. All-same characters like "aaaa" → already palindrome.

2 The Algorithm: KMP Failure Function on s + "#" + reverse(s) ▾

The naive O(n²) approach: try every prefix of s, check if it is a palindrome, keep the longest. We can do better with a clever KMP trick.

We construct a special combined string: t = s + "#" + reverse(s). Then we compute KMP's failure function (prefix-suffix table) over t. The last value of the failure function tells us the length of the longest palindromic prefix of s.

1 Reverse s to get rev. Build combined string t = s + "#" + rev. The "#" separator is a character not in s — it prevents false-positive matches that would cross the boundary.
2 Compute the KMP failure function (LPS array — Longest Proper Prefix that is also a Suffix) for the combined string t. This runs in O(n) time.
3 Read off the answer: lps[-1] gives the length of the longest palindromic prefix of s. The portion of s beyond that length is s[lps[-1]:]. Prepend its reverse to s.
4 Return s[lps[-1]:][::-1] + s. We reverse the non-palindromic suffix and prepend it.

🔑 Why the "#" Separator Matters

Without "#", the prefix of s could falsely match characters from the reversed half that shouldn't cross the boundary. The "#" is a sentinel — it forces any match to be entirely within the original s portion or the reversed portion, never spanning both.

3 Why It Works: KMP Failure Function Deep-Dive ▾
🔐 The Invariant

The KMP failure function lps[i] stores the length of the longest proper prefix of t[0..i] that is also a suffix. A "proper" prefix is shorter than the full string itself.

KMP Algorithm — Origin and Purpose: Invented by Donald Knuth, Vaughan Pratt, and James Morris in 1977. Originally designed for pattern matching in O(n + m) time without backtracking. Its key structure — the failure function — captures the self-similarity of the pattern string: "if the current comparison fails at position j, how far back can we restart without re-examining already-compared characters?"

💡 The Core Invariant of KMP Failure Function

For each position i in the string, lps[i] = length of the longest proper prefix of t[0..i] that is also a suffix of t[0..i]. This is computed in O(n) time using a two-pointer approach: pointer i marches forward, pointer j tracks the length of the current matching prefix.

Why does lps[-1] give the palindromic prefix length?

Consider t = s + "#" + rev(s). A palindromic prefix of s of length k reads the same forward and backward. So s[0..k-1] == reverse(s[0..k-1]) == rev[-k:]. This means s[0..k-1] appears both as a prefix of t (from the s part) and as a suffix of t (from the rev(s) part). That is exactly what the LPS array measures — the longest prefix that is also a suffix! The "#" prevents any match longer than len(s).

Building the failure function:

def kmp_failure(t): ← build LPS array for combined string t
n = len(t)
lps = [0] * n ← lps[0] always 0 (no proper prefix)
j = 0 ← j: length of previous longest prefix-suffix
for i in range(1, n): ← i scans forward through t
while j > 0 and t[i] != t[j]: ← mismatch: fall back using lps itself!
j = lps[j - 1] ← brilliant: skip to next candidate
if t[i] == t[j]: ← match: extend the prefix-suffix
j += 1
lps[i] = j ← record length at this position
return lps
⚠️ The "Fall Back" Step is the Heart of KMP

When a mismatch happens at position j, we don't reset to 0. Instead we set j = lps[j-1]. Why? Because we know the last lps[j-1] characters already matched. We reuse that knowledge. This is what gives KMP its O(n) amortized cost — j can only increase at most n total times, so the total fallback steps is also bounded by n.

When to recognize KMP applies: (1) You need to search for a pattern in a text, (2) You need to find the longest prefix of A that matches a suffix of B, (3) You need to find repeated structure in a string (like the minimum period), (4) Any "prefix meets suffix" question where naive O(n²) is too slow.

4 The Code (Annotated) ▾

Solution 1 — KMP on Combined String (Optimal, O(n))

def shortestPalindrome(s: str) -> str:
if not s: return s ← edge case: empty string
rev = s[::-1] ← reverse of s
t = s + "#" + rev ← combined string with sentinel separator
n = len(t)
lps = [0] * n ← failure function array
j = 0
for i in range(1, n): ← compute failure function
while j > 0 and t[i] != t[j]: ← mismatch: fall back
j = lps[j - 1]
if t[i] == t[j]:
j += 1
lps[i] = j
longest_pal_prefix = lps[-1] ← last LPS value = palindromic prefix length
suffix = s[longest_pal_prefix:] ← the non-palindromic tail of s
return suffix[::-1] + s ← prepend reverse of suffix
⚠️ Why Reverse the Suffix (Not s)?

We prepend suffix[::-1], not rev. We only need to prepend the characters that make the result a palindrome — those are exactly the characters in s[longest_pal_prefix:] reversed. Prepending the full rev would over-add characters.

Solution 2 — Brute Force O(n²)

def shortestPalindrome_brute(s: str) -> str:
def is_palindrome(t): ← helper: O(n) palindrome check
return t == t[::-1]
for i in range(len(s), -1, -1): ← try longest prefix first
if is_palindrome(s[:i]): ← O(n) check at each of O(n) positions → O(n²)
return s[i:][::-1] + s ← prepend reverse of remaining suffix

Solution 3 — Rabin-Karp Rolling Hash O(n)

def shortestPalindrome_hash(s: str) -> str:
base, mod = 26, 10**9 + 7
fwd = rev_h = 0; power = 1; best = 0 ← track forward and backward hashes
for i, c in enumerate(s):
fwd = (fwd * base + ord(c)) % mod ← forward hash of s[0..i]
rev_h = (rev_h + ord(c) * power) % mod ← reverse hash of s[0..i]
power = power * base % mod
if fwd == rev_h: ← hashes match → s[0..i] is a palindrome
best = i + 1 ← track longest palindromic prefix seen
return s[best:][::-1] + s ← same answer construction
💡 Complexity Summary

Brute Force: Time O(n²), Space O(n) — too slow for large inputs
KMP: Time O(n), Space O(n) — optimal and guaranteed correct
Rabin-Karp Hash: Time O(n) expected, Space O(1) extra — hash collisions possible (rare with large prime mod)

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Reframe the Problem

Say: "We can only add characters to the front. Adding the minimum means we need to touch as few characters as possible. Ideally, a long prefix of s is already a palindrome — then we only reverse the short remaining suffix and prepend it."

Step 2: Propose the O(n²) Solution First

Start with brute force: try every prefix from longest to shortest, check if it's a palindrome with s[:i] == s[:i][::-1], return once found. This is O(n²) and correct. The interviewer will ask you to optimize.

Step 3: Identify the Bottleneck

The bottleneck is checking "is this prefix a palindrome?" for each of n prefixes. Each check is O(n). We need an O(n) way to find the longest palindromic prefix.

Step 4: Connect to KMP / String Matching

Hint yourself: "A prefix of s is a palindrome if and only if it equals its own reverse. That means s's prefix matches a suffix of reverse(s). Finding the longest prefix of X that equals a suffix of Y — that's a string matching problem, solvable with KMP's failure function!"

Step 5: Construct t = s + "#" + rev(s)

By concatenating with a separator, the KMP failure function on t tells you the length of the longest proper prefix of t (which is a prefix of s) that is also a suffix of t (which is a substring of rev(s)). The last LPS value is exactly the length of the longest palindromic prefix.

🗣️ What to Say to the Interviewer

"We can only prepend characters. Minimum additions means finding the longest palindromic prefix of s.
Brute force: try every prefix, check palindrome — O(n²).
Key observation: a prefix of s of length k is a palindrome iff s[0..k-1] == rev(s)[-k:].
So the longest palindromic prefix of s = longest prefix of s that matches a suffix of rev(s).
That's exactly what KMP's failure function computes on t = s + '#' + rev(s).
lps[-1] gives the length. Prepend reverse of s[lps[-1]:] to s.
Time O(n), Space O(n)."
🔮 Likely Follow-up Questions
  • "What if you could also append?" → Then it's the longest palindromic substring starting at index 0 or ending at n-1.
  • "What's the longest palindromic substring in general?" → Manacher's Algorithm O(n) (#5)

7 Summary & Key Takeaways ▾
Prepend minimum = find longest palindromic prefix, then prepend the reverse of the remaining suffix
Build t = s + "#" + reverse(s); KMP's lps[-1] gives the palindromic prefix length in O(n)
The "#" separator prevents cross-boundary spurious matches in the failure function
KMP failure function falls back using itself — j = lps[j-1] — giving amortized O(n) computation
Same pattern works for: shortest rotation to make a string equal to another, minimum period of a string
