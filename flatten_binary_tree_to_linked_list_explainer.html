LeetCode #114

Flatten Binary Tree to Linked List

Flatten a binary tree in-place into a linked list following preorder traversal order — every left pointer null, right pointer threads the next node.

Medium Trees Post-order Pointer Rewiring Time: O(n) Space: O(h)
1 Understanding the Problem ▾

Given the root of a binary tree, flatten it in-place into a linked list. The resulting list must follow preorder traversal order: root → left subtree → right subtree. After flattening, every node's left child is null, and right points to the next node in preorder.

💡 Key Insight

The flattened order is exactly what you get from a preorder DFS walk: visit the root, then the entire left subtree recursively, then the entire right subtree. The last node of the left subtree's preorder sequence must be stitched to the first node of the right subtree. If we process nodes bottom-up (after their children), each node's subtrees are already flattened when we arrive — we just connect them.

Input: Binary Tree

      1
     / \
    2   5
   / \  \
  3   4   6

Output: Flattened List (right pointers only)

1 → 2 → 3 → 4 → 5 → 6

Each .left = None
Each .right = next node

What traversal is this? Preorder visits: root(1), left subtree of 1 → [2, left subtree of 2 → [3], right subtree of 2 → [4]], right subtree of 1 → [5, right subtree of 5 → [6]]. That gives [1, 2, 3, 4, 5, 6] — exactly the linked list order.

Edge cases: empty tree (root = None) — return immediately; single node — already flat; a tree that is already a right-skewed list — no-op; tree with only left children — must shift everything to right pointers.

2 The Algorithm: Reverse Preorder with a Trailing Pointer ▾

The elegant solution visits nodes in reverse preorder (right → left → root) and maintains a single prev pointer. Because reverse preorder is the mirror of preorder, when we arrive at any node, the node that should come right after it in the final list is already pointed to by prev. We simply wire node.right = prev, clear node.left, and advance prev = node.

1 Keep a prev pointer (starts as None). After processing any node, prev always holds the "head" of the already-flattened suffix — everything that comes after the current node in preorder is already a correct linked list starting at prev.
2 Recurse right subtree first, left subtree second. This visits nodes in reverse preorder: [6, 5, 4, 3, 2, 1] for our example. When node N is processed, nodes that come after N in preorder have all been visited and wired.
3 At each node: node.right = prev (attach the already-wired suffix), node.left = None (remove left child — it has already been folded into the right chain via prev), prev = node (this node is now the new head of the processed list).
4 Base case: if not node: return. Empty subtrees are silently skipped, leaving prev unchanged.
🔑 The Three Approaches at a Glance

1. Collect + Rewire: Do a preorder traversal, collect nodes in a list, then re-link right pointers. O(n) space — simple but wasteful.
2. Reverse Preorder (interview pick): Visit in reverse preorder, trailing prev pointer. O(h) space.
3. Morris-style Iterative: For each node with a left subtree, find the rightmost node of that left subtree and stitch it to the current node's right subtree. O(1) space, fully iterative.

3 Why It Works: The Invariant ▾
🔐 The Invariant

When we arrive at node N during the reverse-preorder traversal, prev is the head of a fully-correct linked list containing all nodes that follow N in the preorder sequence. Therefore setting N.right = prev is always correct and never needs adjustment.

Informal proof: Reverse preorder visits nodes in the exact reverse of [1, 2, 3, 4, 5, 6], giving [6, 5, 4, 3, 2, 1]. The last visited is the first in the final list. At step 1 we process 6: prev=None, so node6.right=None — correct, 6 is the tail. At step 2 we process 5: prev=node6, so node5.right=node6 — correct, 5 comes before 6. And so on. Each step extends the front of the already-correct list by one node.

Why recurse right before left matters: In preorder, after visiting a root, the entire left subtree comes before the entire right subtree. In reversed preorder, the right subtree must be visited before the left subtree. If you accidentally recurse left before right, prev will point into the left subtree when you process the root, producing an incorrect link.

⚠️ The Classic Mistake: Recurse Left First

If you write dfs(node.left) before dfs(node.right), you'll recurse in standard postorder (left, right, root). When you process a node, prev holds the last node of the right subtree — not the first node of the right subtree. This wires every root directly to the end of its right subtree, skipping the left subtree entirely. The order of the two recursive calls is the entire trick.

Why is node.left = None safe to do after recursing? We recurse on node.left before we clear it, so the recursive call sees the real left subtree. Only after both recursive calls return do we set node.left = None. The left subtree's nodes are already wired into the right chain via prev at that point.

4 The Code (Annotated) ▾

Solution 1 — Reverse Preorder DFS (Interview Pick, O(h) space)

def flatten(root: TreeNode) -> None: ← Modifies tree in-place; no return value needed
prev = [None] ← List trick: lets inner func mutate this reference
def dfs(node): ← Reverse preorder: right, left, then process node
if not node: return ← Base case: null node → nothing to do, prev unchanged
dfs(node.right) ← CRITICAL: right subtree first (reverse of preorder)
dfs(node.left) ← Left subtree second (still before processing root)
node.right = prev[0] ← Attach: this node's right → already-wired suffix
node.left = None ← Clear: left subtree was already folded into right chain
prev[0] = node ← Advance: this node is now the new head of the list
dfs(root) ← Kick off; root is processed last, becoming list head
⚠️ Why prev = [None] instead of prev = None?

Python nested functions can read enclosing scope variables freely, but cannot rebind them without nonlocal. Wrapping in a single-element list (prev[0]) lets us mutate the content. The alternative is nonlocal prev (Python 3 only) or making prev an instance attribute (self.prev = None). All three approaches work identically.

Solution 2 — Collect Preorder + Re-wire (O(n) space, simplest)

def flatten(root: TreeNode) -> None:
nodes = [] ← Will hold all nodes in preorder
def preorder(n): ← Standard preorder: root → left → right
if not n: return
nodes.append(n) ← Collect node before recursing into children
preorder(n.left)
preorder(n.right)
preorder(root) ← nodes = [1,2,3,4,5,6] for our example
for i in range(len(nodes) - 1): ← Re-wire: each node points to the next
nodes[i].right = nodes[i + 1] ← Thread right pointer to successor
nodes[i].left = None ← Clear left pointer

Solution 3 — Morris-style Iterative (O(1) space)

def flatten(root: TreeNode) -> None:
curr = root
while curr: ← Walk through every node iteratively
if curr.left: ← Only act when there's a left subtree to fold in
prev = curr.left ← Start at root of left subtree
while prev.right: ← Walk to rightmost node of left subtree
prev = prev.right ← (This is where left subtree's preorder ends)
prev.right = curr.right ← Stitch: rightmost of left → curr's right subtree
curr.right = curr.left ← curr's right becomes the left subtree (now prepended)
curr.left = None ← Clear left pointer
curr = curr.right ← Advance (always safe — left was just moved to right)
💡 Complexity Summary

Collect + Rewire: Time O(n), Space O(n) — explicit node list
Reverse Preorder DFS: Time O(n), Space O(h) — recursion stack (h = tree height)
Morris-style Iterative: Time O(n) amortized, Space O(1) — no extra memory (inner while loop costs O(h) total across all iterations)

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Required Order

Trace the example by hand: [1, 2, 3, 4, 5, 6]. Ask yourself: what traversal produces root first, then all left nodes, then all right nodes? That's preorder. Say this explicitly to the interviewer before writing any code.

Step 2: Propose the O(n) Space Solution First

"My first approach: do a preorder DFS to collect all nodes in a list, then re-wire each node's right pointer to the next, setting left to null. O(n) time, O(n) space." Write this first — it's always correct and demonstrates understanding.

Step 3: Discover the Space Optimization

"Can I avoid the list? I need each node to know its successor in preorder. If I process nodes in reverse preorder — right subtree first, then left — the successor is always the last node I processed (i.e., prev). So I can wire backward without storing anything."

Step 4: For O(1) Space, Describe Morris Technique

"For true O(1) space, I can avoid recursion entirely. For each node with a left child, find the rightmost node of that left subtree and stitch it to the current node's right child. Then move the entire left subtree to the right. Repeat until no left children remain."

🗣️ What to Say to the Interviewer

"The output is preorder order — root first, then left subtree, then right subtree.
Simple solution: collect preorder in a list, re-wire right pointers — O(n) space.
Better: process in reverse preorder (recurse right, then left, then current node).
By the time I reach any node, its suffix list is already wired starting at prev.
Just set node.right = prev, node.left = None, prev = node.
This is O(h) space. For O(1): Morris technique — stitch rightmost of left subtree to right child before moving left to right."
🔮 Likely Follow-up Questions
  • "Can you do it iteratively with O(1) space?" → Morris-style: stitch rightmost of left to right, move left to right
  • "What if the tree is very deep (100k nodes)?" → DFS recursion risks stack overflow; use Morris or explicit stack
  • "What if we need inorder instead of preorder?" → Same idea but recurse left first, then process, then right
  • "Why is the inner while loop in Morris O(n) total?" → Each node's right pointer is followed at most twice (once by inner while, once by curr = curr.right)

7 Summary & Key Takeaways ▾
The flattened order is preorder: root → entire left subtree → entire right subtree
Reverse preorder (right → left → root) visits nodes in exactly the reverse of the desired output order
A single prev pointer is sufficient: when we arrive at any node, all its successors are already wired into a list at prev
Recurse RIGHT before LEFT — this single rule is the entire cleverness of the algorithm
Morris-style achieves O(1) space: stitch the rightmost of the left subtree to the current right subtree, then pivot
