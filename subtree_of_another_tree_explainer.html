LeetCode #572

Subtree of Another Tree

Given the roots of two binary trees root and subRoot, return true if there is a subtree of root with the same structure and node values of subRoot and false otherwise.

Easy Trees Pre-Order DFS Time: O(m·n) Space: O(m)
1 Understanding the Problem ▾

A subtree of a binary tree root is a node inside root along with all of its descendants. We need to check whether subRoot appears as a subtree anywhere inside root — meaning the structure AND node values must match exactly, including which children are null.

💡 Key Insight

This is a two-function problem. The outer function isSubtree walks every node of root looking for a starting candidate. The inner helper isSameTree checks if two trees are completely identical from those starting nodes. Together: try each node as a candidate, validate completely, or keep searching.

✓ True: subRoot [4,1,2] found at node 4

root: [3, 4, 5, 1, 2]
subRoot: [4, 1, 2]
Node 4 → left=1, right=2
subRoot → left=1, right=2 ✓

✗ False: subRoot [4,1,2] doesn't fit anywhere

root: [3, 4, 5, 1, 2, null, null, 0]
subRoot: [4, 1, 2]
Node 4 has extra child 0
Not identical structure ✗

Edge cases to consider: subRoot equal to the whole root (valid — a tree is a subtree of itself). subRoot is a single node. Trees with all identical values (must match structure too). A completely skewed tree (worst case for time complexity).

⚠️ Subtree vs Sub-Structure

A subtree must match every descendant — including null children. If node 4 in root has children 1, 2, and 3, but subRoot's node 4 only has children 1 and 2, that is not a valid subtree match. All null positions must also agree.

2 The Algorithm: Check at Each Node (Pre-Order DFS) ▾

We use two cooperating recursive functions. The outer one is a navigator — it pre-order DFS walks every node of root. The inner one is a validator — it checks whether two trees are structurally identical.

1 Base case for isSubtree: If root is None, we've exhausted the entire tree with no match. Return False.
2 Check here — isSameTree: Test whether the subtree rooted at this exact node is identical to subRoot. If yes — return True immediately.
3 Recurse — left then right: If the current node is not the match, try root.left and root.right. Use short-circuit OR — return True as soon as either side finds a match.
4 isSameTree logic: Both null → True (perfect empty match). One null → False (structural mismatch). Values differ → False. Otherwise recursively check left AND right children.

🔑 Why Pre-Order?

We check the current node first, then recurse into children. This is pre-order (Root → Left → Right). It's natural here because we want to test "does subRoot begin exactly at this node?" — which is a root-first question. Post-order would be used if we needed child results before deciding about the current node.

3 Why It Works: The Invariant ▾
🔐 The Invariant

isSubtree(root, subRoot) returns True if and only if there exists at least one node n reachable from root such that isSameTree(n, subRoot) is True. The DFS guarantees we test every reachable node before giving up.

Why isSameTree is correct: Two trees are identical iff: both are null (empty match), OR both are non-null with equal root values AND identical left subtrees AND identical right subtrees. This recursive definition covers every possible structural difference. It cannot produce false positives.

Why short-circuit OR is safe and efficient: We only need one valid location. As soon as isSubtree(root.left, subRoot) returns True, Python's or skips evaluating isSubtree(root.right, subRoot). This prunes work in cases where the match appears early in the tree.

⚠️ Common Mistake: Missing the "one null" case in isSameTree

If you forget if not p or not q: return False and jump straight to comparing p.val != q.val, you'll get an AttributeError when one of them is None. Always check: are both null (True), then is one null (False), then compare values. The order matters!

Worst-case analysis: Imagine root = [1, 1, 1, 1, 1, ...] (all ones, m nodes) and subRoot = [1, 1, 1, ...] (all ones, n nodes). At every node in root, isSameTree runs for O(n) before finding a final mismatch or match. Total: O(m·n). This is unavoidable without hashing/serialization.

4 The Code (Annotated) ▾

Solution 1 — DFS + isSameTree (Interview Pick)

class Solution:
def isSubtree(self, root, subRoot) -> bool:
if not root: ← Searched entire tree, no match found
return False
if self.isSameTree(root, subRoot): ← Does subRoot begin exactly here?
return True
return (self.isSubtree(root.left, subRoot) or ← Try left branch …
self.isSubtree(root.right, subRoot)) ← … then right (short-circuit)
def isSameTree(self, p, q) -> bool:
if not p and not q: return True ← Both null → perfectly equal
if not p or not q: return False ← One null, one not → mismatch
if p.val != q.val: return False ← Different values → mismatch
return (self.isSameTree(p.left, q.left) and ← Left children must match …
self.isSameTree(p.right, q.right)) ← … and right children too
💡 Complexity

Time: O(m · n) where m = nodes in root, n = nodes in subRoot. In the worst case (all same values) we run isSameTree at every node.
Space: O(m) recursion stack for isSubtree (balanced: O(log m), skewed: O(m)).

Solution 2 — Serialization + Substring Match (O(m+n) time)

def isSubtree(self, root, subRoot) -> bool:
def serialize(node): ← Pre-order serialization with delimiters
if not node: return "#" ← # marks null children (structural info)
return (f"^{node.val}" ← ^ prefix prevents "1"+"2" matching "12"
+ serialize(node.left)
+ serialize(node.right))
return serialize(subRoot) in serialize(root) ← O(m+n) substring check
⚠️ The Delimiter Trick is Non-Negotiable

Without the ^ prefix on values, a root with node value 1 followed by node value 2 would serialize as "12", falsely matching a subRoot with node value 12 serialized as "12". The ^ delimiter makes node 1 serialize to "^1" and node 12 to "^12" — no collision possible. Similarly, the # for null is essential for structural matching.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Two-Function Pattern

When a problem says "check if X exists anywhere in a tree," think: "I need to walk the tree (outer DFS) and validate at each node (inner check)." These two responsibilities should be separate functions — don't try to combine them into one messy recursion.

Step 2: Write isSameTree First

Start with the simpler sub-problem: "How would I check if two trees are identical?" Say: "both null → True; one null → False; values differ → False; else recurse on children." This is LeetCode #100 and demonstrates strong tree fundamentals. Interviewers appreciate the decomposition.

Step 3: Write isSubtree Using the Helper

The outer function writes itself naturally: "If root is null, return False. If isSameTree(root, subRoot) is True, return True. Otherwise, OR together the recursive results on root.left and root.right." Clean, readable, and provably correct.

Step 4: Offer the O(m+n) Serialization Upgrade

If the interviewer asks about time complexity or large inputs, mention: "We can serialize both trees into strings with delimiter-separated node values and use substring matching. This gives O(m+n) time at the cost of O(m+n) space. The key is adding a delimiter before each value to prevent false matches."

🗣️ What to Say to the Interviewer

"I'll break this into two functions. First, a helper isSameTree: both null → True; one null → False; different values → False; else recursively check left and right children.
Then isSubtree: walk root in pre-order DFS. At each node, call isSameTree. If it returns True, we found our match — return True immediately.
If root becomes null, we've searched everything with no match — return False.
Otherwise recurse into root.left and root.right using OR for short-circuiting.
Time: O(m·n) worst case. Space: O(m) for the call stack.
For large inputs, we could serialize both trees with delimiters and do a substring check for O(m+n) time."
🔮 Likely Follow-up Questions
  • "Can you do O(m+n)?" → Serialize + substring (or KMP for true O(m+n))
  • "What if trees are huge — say 100K nodes?" → Rabin-Karp rolling hash for O(m+n) time and O(1) extra space
  • "Find ALL positions where subRoot appears?" → Collect nodes where isSameTree returns True into a list
  • "Same problem but nodes hold strings?" → No change — isSameTree compares p.val != q.val regardless of type

7 Summary & Key Takeaways ▾
A subtree match requires complete structural and value equality — every null child position must also agree
Decompose into two functions: outer DFS navigator (isSubtree) + inner validator (isSameTree)
Pre-order traversal: test the current node first, then recurse left and right
Short-circuit OR in the recursion prunes work as soon as any match is found
Serialization with delimiters achieves O(m+n) time but requires O(m+n) space
