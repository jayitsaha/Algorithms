First Unique Character in a String

LeetCode #387 · Find the first non-repeating character
Easy Hash Table Frequency Count O(n) Time O(1) Space
1 Understanding the Problem ▼

Given a string s, find the first character that appears exactly once and return its index. If no such character exists, return -1.

Example: For s = "leetcode", the answer is 0 because 'l' is the first character that appears only once.

Example: For s = "loveleetcode", the answer is 2 because 'v' is the first unique character (at index 2).

💡 Key Insight

We need to know two things for each character: (1) how many times it appears, and (2) where it first appears. A hash map gives us both — count as the value, and we can find position by a second scan.

Why can't we solve it in one pass? When we see a character for the first time, we don't yet know if it will repeat later. We must see the entire string before we can confirm any character is unique.

2 The Algorithm: Two-Pass Frequency Count ▼

This is a classic two-pass algorithm using a hash map to count character frequencies.

1 Pass 1 — Count: Scan the string and build a frequency map. For each character, increment its count in the map.
2 Pass 2 — Find: Scan the string again (in order). For each character, check if its count in the map is 1. Return the index of the first such character.
3 No unique found: If we complete Pass 2 without finding count=1, return -1.

🔑 Why Two Passes?

We can't determine uniqueness until we've seen the entire string. The first pass collects all the information, the second pass uses it to find the answer while preserving the original order.

3 Why It Works: The Invariant ▼
🔐 The Invariant

After Pass 1, count[c] holds the exact number of times character c appears in the string. In Pass 2, we iterate in original order, so the first character with count[c] == 1 is guaranteed to be the answer.

Correctness Proof:

  • Completeness: Every character is counted exactly once per occurrence in Pass 1.
  • Order preservation: Pass 2 iterates left-to-right, so we find the first unique character by index.
  • No false positives: If count[c] == 1, the character appeared exactly once.
⚠️ Edge Cases
  • Empty string: No characters → return -1
  • All duplicates: Every char appears ≥2 times → return -1
  • Single character: Always unique → return 0
4 The Code (Annotated) ▼
# Two-Pass Frequency Count — O(n) time, O(1) space (26 letters max)
def firstUniqChar(s: str) -> int:
# Pass 1: Build frequency map
count = {}
for c in s:
count[c] = count.get(c, 0) + 1 # Increment count
# Pass 2: Find first character with count == 1
for i, c in enumerate(s):
if count[c] == 1:
return i # Found first unique
return -1 # No unique character found

Using Counter (Pythonic):

from collections import Counter
def firstUniqChar(s: str) -> int:
count = Counter(s) # Built-in frequency counter
for i, c in enumerate(s):
if count[c] == 1:
return i
return -1
💡 Space Complexity Note

The hash map stores at most 26 entries (lowercase English letters), so space is O(1) — constant regardless of input size. This is often described as O(k) where k=26.

5 🎤 Interview Strategy ▼
Step 1: Recognize the Pattern

Keywords: "first", "unique", "non-repeating", "count" → This signals frequency counting with a hash map.

Step 2: Ask Clarifying Questions

"Are all characters lowercase letters?" (affects space complexity) "Can the string be empty?" (edge case)

Step 3: Explain the Two-Pass Approach

Explain why one pass isn't enough: "We can't confirm uniqueness until we've processed the entire string."

Step 4: State Complexity Upfront

"This approach is O(n) time with two passes, and O(1) space since we have at most 26 characters."

🗣️ What to Say

"I'll use a frequency counting approach. First, I'll count how many times each character appears. Then I'll scan the string again and return the index of the first character with count 1. This gives O(n) time since we make two passes, and O(1) space since we store at most 26 letters."

Follow-up questions they might ask:

  • "Can you do it in one pass?" — Yes, with an ordered structure (LinkedHashMap in Java, or track indices)
  • "What if the string contains unicode?" — Space becomes O(k) where k = unique characters
