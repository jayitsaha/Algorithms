Text Justification

LeetCode #68 — Greedy Line Building + Space Distribution
Hard String Transformation Greedy Line Building O(n·maxWidth) O(n·maxWidth)
1 Understanding the Problem ▾

Given words and a maximum width, produce lines exactly maxWidth characters wide. Words are packed greedily (as many as fit), then spaces are distributed evenly between them. The last line is left-justified with trailing spaces.

Example: words = ["This","is","an","example","of","text","justification."], maxWidth = 16

Line 1: "This····is····an" — 3 words, 10 chars, 6 extra spaces split into 3+3.
Line 2: "example··of·text" — 3 words, 13 chars, 3 spaces split into 2+1.
Line 3: "justification.··" — last line, left-justified, pad right.

💡 Key Insight

The hard part isn't finding which words go on each line — it's distributing spaces. Use divmod: total_spaces ÷ gaps = base_spaces with remainder extra spaces sprinkled left-to-right.

⚠️ Three Special Cases
  • Single word on a line: Left-justify, pad right with spaces
  • Last line: Left-justify (single space between words), pad right
  • Middle lines: Fully justify — distribute extra spaces left-to-right
2 The Algorithm: Greedy Pack + Justify ▾

Two distinct phases per line:

1 Pack words greedily: Starting from word i, keep adding words while total_chars + spaces_needed ≤ maxWidth. The line gets words [i..j-1].
2 Calculate spaces: total_spaces = maxWidth - chars_in_words. Number of gaps = word_count - 1.
3 Distribute evenly: base = total_spaces // gaps, extra = total_spaces % gaps. First extra gaps get base+1 spaces, rest get base.
4 Handle edges: Single word → pad right. Last line → one space between, pad right.
📐 Space Distribution Formula
gaps = word_count - 1
total_spaces = maxWidth - sum(len(word) for word in line_words)
base, extra = divmod(total_spaces, gaps)
gap[k] = base + (1 if k < extra else 0)
3 Why It Works: The Invariant ▾
🔐 The Greedy Invariant

Pack as many words as possible per line. This is optimal because (1) words can't be split, (2) fewer words per line means more wasted padding later, and (3) the problem requires maximum packing.

Why divmod gives even distribution: If we have total_spaces = 7 and gaps = 3, then 7 ÷ 3 = 2 remainder 1. That means 2 base spaces per gap, plus 1 extra space for the first gap: gaps get [3, 2, 2] = 7 total. The left-bias rule is exactly what k < extra achieves.

Edge cases:

  • Single word: gaps = 0, so divmod would divide by zero. Handle separately — all spaces go right.
  • Last line: Override the justify logic — just use 1 space between, pad right.
  • Word exactly fills line: total_spaces = gaps (one space each), extra = 0. Works perfectly.
4 The Code (Annotated) ▾
# Greedy Pack + Justify (Interview Pick)
def fullJustify(words, maxWidth):
res, line, line_len = [], [], 0
for word in words:
# Can we fit this word? (chars + 1 space per word already in line)
if line_len + len(word) + len(line) > maxWidth:
# Line is full — justify and emit
for i in range(maxWidth - line_len):
line[i % (len(line)-1 or 1)] += ' '
res.append(''.join(line))
line, line_len = [], 0
line.append(word)
line_len += len(word)
# Last line — left justified, pad right
res.append(' '.join(line).ljust(maxWidth))
return res
⚠️ The Clever i % (len(line)-1 or 1) Trick

When distributing extra spaces, we cycle through gaps using modulo. If only 1 word on the line, len(line)-1 = 0 would cause ZeroDivisionError. The or 1 handles single-word lines — all spaces go after the word.

📊 Complexity

Time: O(n · maxWidth) — each word is processed, space distribution is O(maxWidth) per line.
Space: O(n · maxWidth) — output lines.

Step 2: Identify the Edge Cases Early

Three cases: normal middle lines (full justify), single word on a line (pad right), and last line (left justify). Clarify these with the interviewer before coding.

Step 3: The Space Formula

"For a middle line with W words and C total characters: total_spaces = maxWidth - C, gaps = W - 1. Use divmod for even distribution with left bias."

🗣️ What to Say to the Interviewer

"I'll use a greedy approach: scan words, greedily pack as many as fit per line. When a line is full, justify it by distributing spaces evenly using divmod, with extra spaces biased left. For the last line, I'll left-justify instead. The key edge case is a single word on a line — all padding goes right. This is O(n · maxWidth) time, O(n · maxWidth) space for the output."
