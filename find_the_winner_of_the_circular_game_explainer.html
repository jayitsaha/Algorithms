LeetCode #1823

Find the Winner of the Circular Game

n friends sit in a circle, counting k steps clockwise and eliminating — find the last one standing.

Medium Queues Josephus Problem Time: O(nk) or O(n) Space: O(n) or O(1)
1 Understanding the Problem ▾

n friends numbered 1 to n are seated in a circle. Starting from friend 1, you count clockwise. When you reach the k-th person in the count, that person is eliminated from the circle. You then immediately start counting again from the next surviving person. This repeats until exactly one person remains — that person is the winner.

💡 Key Insight

This is the famous Josephus Problem — one of the oldest puzzles in CS. A deque (queue) naturally models the circle: move surviving players to the back (rotate), then eliminate the front (the k-th person). There is also a brilliant O(n) mathematical recurrence that computes the winner's position without any simulation.

Walk-through example — n=5, k=2:

Round-by-round elimination

Circle: [1, 2, 3, 4, 5]
Count 2 from 1 → eliminate 2
Count 2 from 3 → eliminate 4
Count 2 from 5 → eliminate 1
Count 2 from 3 → eliminate 5
Winner: 3 🏆

Why a queue is perfect

The deque front is the "current" player. Rotate k−1 survivors to the back (they live), then popleft() the k-th player (they're out). After rotation, the player at front is exactly who the count lands on.

Edge cases to consider: k=1 means the first player each round is always eliminated — winner is player n. n=1 means player 1 wins immediately. k larger than the current circle size wraps around using the queue's natural rotation.

2 The Algorithm: Queue Simulation + Josephus Recurrence ▾

Approach 1 — Queue Simulation: A deque models the circle. Each round, we rotate k−1 players (surviving the count) to the back, then popleft the k-th player (eliminated). Continue until size 1.

1 Initialize: q = deque(range(1, n + 1)). Players 1..n in order, with player 1 at the front (start of count).
2 Rotate k−1 times: q.append(q.popleft()) repeated k−1 times. Each rotation moves one player from front (counted but surviving) to the back of the queue.
3 Eliminate: q.popleft() — the player now at the front is the k-th in the count. They are removed from the circle.
4 Repeat: Go to step 2 while len(q) > 1. The surviving front player after k−1 rotations of the previous step automatically becomes the next "counter start."

Approach 2 — Josephus Recurrence (O(n), O(1)): Instead of simulating, use the mathematical insight that winning positions follow a recurrence across subproblem sizes:

Josephus Recurrence (0-indexed positions)
J(1) = 0
J(i) = (J(i − 1) + k) % i    for i = 2, 3, ..., n
Final Answer = J(n) + 1   (convert 0-indexed → 1-indexed)
🔑 When to use which approach

In an interview, code the queue simulation first — it's intuitive and easy to explain. Then mention: "There's actually an O(n) mathematical solution using the Josephus recurrence — want me to implement that too?" This demonstrates both practical and theoretical depth.

3 Why It Works: Queue Invariant & Josephus Recurrence ▾
🔐 The Queue Invariant

At the start of every elimination round, the deque contains exactly the surviving players in their correct circular order, with the "next count-start" at the front. After k−1 rotations, the player at the front is precisely the k-th person in the current count — they are correctly eliminated. This invariant holds at every round, so the simulation is always correct.

Why the Josephus recurrence works: After one elimination, n−1 players remain in a new circle. The game of n−1 players has the same structure. The key observation: if the winner is at 0-indexed position p in the (n−1)-player game, then in the n-player game, the same physical person was at position (p + k) % n. This mapping applies because eliminating the k-th player shifts all subsequent positions by k. Build from base case J(1)=0 up to J(n).

Verify for n=5, k=2:
J(1)=0, J(2)=(0+2)%2=0, J(3)=(0+2)%3=2, J(4)=(2+2)%4=0, J(5)=(0+2)%5=2 → 2+1=3 ✓ (matches simulation!)

⚠️ Off-by-one: k−1 rotations, not k

We count k people total. The first k−1 "survive" this round's count (rotate to back). Only the k-th person is eliminated (popleft). If you rotate k times and then popleft, you eliminate the (k+1)-th — a silent, hard-to-debug off-by-one error. Think: "skip k−1 survivors, then eliminate the front."

⚠️ Josephus result is 0-indexed — add 1!

The recurrence works in 0-indexed space (positions 0 to n−1). Players are numbered 1 to n. The final answer is josephus(n, k) + 1. Returning the raw recurrence result gives the wrong player number.

4 The Code (Annotated) ▾

Solution 1 — Queue Simulation (Interview Pick)

from collections import deque
def findTheWinner(n: int, k: int) -> int:
q = deque(range(1, n + 1)) ← Build circle: players 1..n, player 1 at front
while len(q) > 1: ← Keep going until one survivor remains
for _ in range(k - 1): ← Skip k−1 players (they survive this count)
q.append(q.popleft()) ← Move front to back: "counted past, still in"
q.popleft() ← k-th player is at front → eliminate (discard)
return q[0] ← Last surviving player is the winner
⚠️ q.rotate(-(k-1)) vs the explicit loop

Python's deque has q.rotate(-(k-1)) which does the same rotation in one call. Both are correct. The explicit loop is more readable during an interview; rotate() is the one-liner. In terms of complexity, both are O(k) per round (rotation is not O(1) — it moves k−1 elements).

Solution 2 — Josephus Recurrence (O(n), O(1))

def findTheWinner(n: int, k: int) -> int:
pos = 0 ← Base case: 1 player, winner is at 0-indexed position 0
for i in range(2, n + 1): ← Extend: solve for 2 players, then 3, ..., up to n
pos = (pos + k) % i ← Shift winner's position by k (mod i = circle size this round)
return pos + 1 ← 0-indexed → 1-indexed player number
💡 Complexity Summary

Queue Simulation: Time O(nk), Space O(n) — n rounds, k rotations each
Josephus Recurrence: Time O(n), Space O(1) — single loop, two variables
For n=5, k=2: queue runs 4 rounds × 1 rotation = 4 ops. Recurrence runs 4 iterations. Fast input is better; large k makes queue much slower.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize "Circle + Count + Eliminate" = Josephus

The problem says "circular" arrangement, counting k steps, and eliminating. These three signals together almost always mean Josephus Problem or queue rotation. A deque simulates the circle directly — the front is the current player, and rotating moves the window.

Step 2: Code the Queue Simulation First

"I'll model the circle with a deque initialized to [1, 2, ..., n]. Each round I rotate k−1 times (survivors move to back) and then popleft the k-th player. This is O(nk) time and O(n) space." Start here — it's easy to code and easy to verify.

Step 3: Propose the Mathematical Optimization

"If the interviewer asks for O(n), I can use the Josephus recurrence. J(1)=0; for i from 2 to n: J(i)=(J(i−1)+k)%i. The answer is J(n)+1. This runs in O(n) time with O(1) space — just two variables." Even mentioning this gets you points.

Step 4: Cross-Verify with the Example

Always trace through n=5, k=2 using your code. Queue gives 3; recurrence gives (((0+2)%2+2)%3+2)%4+2)%5 + 1 = 2+1 = 3. Both agree — builds confidence in both solutions.

🗣️ What to Say to the Interviewer

"This is the Josephus counting-out game. I'll simulate with a deque.
Players 1..n go in with 1 at front. Each round: rotate k−1 players to the back
(they're counted past but survive), then popleft to eliminate the k-th player.

7 Summary & Key Takeaways ▾
A deque models a circle: rotate-to-back simulates "counting past" a player, popleft eliminates them
Rotate exactly k−1 times per round (not k) — the k-th player then sits at the front to be eliminated
Josephus recurrence J(i) = (J(i−1) + k) % i gives the 0-indexed winner in O(n) time, O(1) space
Always add 1 to the Josephus result to get the 1-indexed player number
The historical Josephus problem (37 CE) is the same puzzle — one of the oldest known CS problems

Brute Force (List pop) O(n²) O(n) List.pop(i) is O(n) — avoid
