LeetCode #215

Kth Largest Element in an Array

Given an integer array nums and an integer k, return the kth largest element in the array. Note that it is the kth largest element in sorted order, not the kth distinct element.

Medium Sorting QuickSelect Time: O(n) avg Space: O(1)
1 Understanding the Problem ▾

We are given an unsorted array nums and a positive integer k. We want the kth largest element when the array is sorted in descending order. Think of it as: if you sorted everyone by their salary from highest to lowest, what is the salary of the person in position k?

Crucially, "kth largest" counts with repetition. If the array is [5, 5, 4] and k=2, the answer is 5, not 4, because 5 appears twice and occupies positions 1 and 2.

💡 Key Insight

The kth largest element is the same as the (n − k)th smallest element in the array (0-indexed). If we could magically place one element at its correct sorted index without sorting the whole array, we'd have our answer. QuickSelect does exactly this: it places a pivot at its final sorted position, then only recurses into the half that contains the target index.

✓ Example: nums=[3,2,1,5,6,4], k=2

Sorted ascending: [1, 2, 3, 4, 5, 6]
n = 6, target index = n−k = 4
nums[4] = 5 ← 2nd largest

⚠ Duplicates: nums=[3,2,3,1,2,4,5,5,6], k=4

Sorted: [1,2,2,3,3,4,5,5,6]
target = 9−4 = 5 → value 4
Answer = 4 (not 3 or 5)

Edge cases to consider: single-element array (return it), k=1 (find the maximum), k=n (find the minimum), all-same values (any element is the answer), and negative numbers mixed with positives.

2 The Algorithm: QuickSelect ▾

QuickSelect is a partial sorting algorithm derived from QuickSort. QuickSort fully sorts by recursing into both halves after partitioning. QuickSelect only recurses into one half — the one containing the target index. On average, this cuts the work in half each time, giving O(n) total work versus O(n log n) for full sorting.

1 Translate kth-largest to a target index: target = len(nums) − k. The kth largest element occupies this zero-based index in sorted-ascending order. For k=2 in a 6-element array: target = 4.
2 Partition around a random pivot: Pick any element as the pivot (swap it to the end). Scan from left to right; move every element ≤ pivot to the left side by swapping with a "store" pointer. Finally, place the pivot at the store position. The pivot is now at its permanently correct sorted index.
3 Compare pivot index to target: If p == target, return nums[p] — done! If p < target, the answer is in the right half: set lo = p + 1. If p > target, the answer is in the left half: set hi = p − 1.
4 Repeat on the smaller subarray only: We never revisit elements we've already partitioned past. Average case: n + n/2 + n/4 + ... = 2n operations total → O(n) overall.

🔑 Alternative: Min-Heap of Size k

Maintain a min-heap of the k largest elements seen so far. Scan the array; push each element; if the heap grows beyond k, pop the smallest. After the scan, the heap's minimum is the kth largest. Time: O(n log k), Space: O(k). This is the go-to for streaming data where the array is too large to hold in memory, or when k is small relative to n.

3 Why It Works: The Invariant ▾
🔐 The Partition Invariant

After partition(lo, hi) returns index p, the following holds for the entire array:
— Every element at indices [lo .. p−1] is ≤ nums[p]
— Every element at indices [p+1 .. hi] is ≥ nums[p]
— nums[p] is at its permanently correct sorted position and will never move again.

Why this guarantees correctness: If p == target, there are exactly p elements smaller or equal to the left, and n − p − 1 elements larger or equal to the right. That makes nums[p] the (p+1)-th smallest = the (n−p)-th largest. When n−p = k, it is the kth largest.

Why the average case is O(n): A random pivot splits the remaining work roughly in half each time. The recurrence is T(n) = T(n/2) + O(n), which solves to T(n) = O(n) by the geometric series (n + n/2 + n/4 + ... converges to 2n). We only recurse into one half — unlike QuickSort which recurses into both.

⚠️ Worst Case O(n²) — Why and How to Avoid It

If we always pick the smallest or largest remaining element as our pivot, each partition reduces the search space by only 1. This gives T(n) = T(n−1) + O(n) = O(n²). This happens on already-sorted arrays when always picking the first or last element as pivot. Solution: pick a random pivot index (random.randint(lo, hi)) before partitioning. In practice this makes the worst case astronomically unlikely. For a provably O(n) worst-case, use the Median-of-Medians algorithm — but it has higher constants and is rarely needed in interviews.

Correctness with duplicates: The Lomuto partition scheme (used here) handles duplicates correctly. Elements equal to the pivot satisfy the ≤ condition, so they land in the left zone. The final pivot position p is still valid — it represents one element at its correct sorted rank, even if that value appears multiple times.

4 The Code (Annotated) ▾

Solution 1 — QuickSelect with random pivot (Optimal, O(n) average, O(1) space) — Interview Pick

import random
def findKthLargest(nums: list[int], k: int) -> int:
target = len(nums) - k ← kth largest = element at index (n−k) in sorted-asc
def partition(lo: int, hi: int) -> int: ← Returns pivot's final sorted index
pi = random.randint(lo, hi) ← Random pivot: avoids O(n²) worst case
nums[pi], nums[hi] = nums[hi], nums[pi] ← Move pivot to end so scan loop can ignore it
pivot = nums[hi] ← The value we partition around
store = lo ← "write pointer": next slot for a ≤pivot element
for i in range(lo, hi): ← Scan every element except the pivot at hi
if nums[i] <= pivot: ← Element belongs to the left partition
nums[store], nums[i] = nums[i], nums[store] ← Swap it into left zone
store += 1 ← Advance write pointer
nums[store], nums[hi] = nums[hi], nums[store] ← Place pivot at its correct sorted position
return store ← Pivot's final index: now permanently placed
lo, hi = 0, len(nums) - 1
while lo <= hi: ← Iterative version (avoids recursion stack depth)
p = partition(lo, hi) ← Partition current subarray
if p == target: ← Pivot landed exactly at target index!
return nums[p]
elif p < target: ← Target is in right half
lo = p + 1 ← Discard left half forever
else: ← Target is in left half
hi = p - 1 ← Discard right half forever
⚠️ Why swap the pivot to index hi before scanning?

We want the scan loop for i in range(lo, hi) to process all non-pivot elements. By placing the pivot at nums[hi], the loop naturally stops before it and we avoid accidentally comparing the pivot against itself. The final swap(store, hi) inserts the pivot into position store — the exact spot where all elements to its left are ≤ pivot and all to its right are ≥ pivot.

Solution 2 — Min-Heap of size k (O(n log k), preferred for streaming)

import heapq
def findKthLargest(nums: list[int], k: int) -> int:
heap = [] ← Python's heapq is always a min-heap
for num in nums:
heapq.heappush(heap, num) ← O(log k) per push
if len(heap) > k: ← Heap grew too large
heapq.heappop(heap) ← Evict the SMALLEST — we keep the k largest
return heap[0] ← Heap min = smallest of the k largest = kth largest

Solution 3 — Sort (O(n log n), simplest for interviews to propose first)

def findKthLargest(nums: list[int], k: int) -> int:
nums.sort() ← Timsort, O(n log n). Modifies nums in-place.
return nums[-k] ← -k indexes from the end; -1 = largest, -2 = 2nd largest
💡 Which to Use in an Interview?

Always propose all three: Sort first (trivially correct, establishes a baseline), then Min-Heap (good for streams — O(n log k) with O(k) space), then QuickSelect (theoretically best — O(n) average, O(1) space). Mention random pivot to show awareness of worst-case pitfalls. Implement whichever the interviewer asks for.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the "Order Statistic" Pattern

Any problem asking for the kth largest, kth smallest, or the median of an array is an order statistic problem. The key question is: do we need all elements sorted, or just one element's position? Since we only need one position, we don't need full sorting.

Step 2: Propose Sort as Baseline

"The simplest solution: sort the array and return nums[-k]. That's O(n log n) time, O(1) space with in-place sort. Correct and easy to implement in 1 line." This gives you a working solution immediately.

Step 3: Introduce Heap for the Streaming Case

"If the array is too large to fit in memory — say it's a stream — I'd use a min-heap of size k. Each element costs O(log k) to process, total O(n log k). This is optimal when k << n." This shows practical thinking.

Step 4: QuickSelect for O(n) Average

"For O(n) average with O(1) space, I can use QuickSelect. I translate kth-largest to the (n−k)th element in sorted-ascending order, pick a random pivot, partition, and only recurse into the subarray containing the target index."

Step 5: Proactively Address Worst Case

"QuickSelect degrades to O(n²) if we always pick a bad pivot — like always picking the minimum. I always randomize the pivot choice to avoid this. The expected runtime is then O(n). For a guaranteed O(n) worst case, Median-of-Medians exists, but it has high constants and isn't practical."

🗣️ What to Say to the Interviewer

"The brute force is just sort: O(n log n), one line. I'll use that as the baseline.
To do better: we observe we only need one element's sorted rank, not the full sort.
QuickSelect partitions around a pivot in O(n), placing the pivot at its sorted index.
We then recurse into only the half containing index n-k.
On average, we halve the remaining work each time: n + n/2 + n/4 + ... = O(n).
I'll randomize the pivot to prevent the O(n²) worst case on sorted inputs.
The implementation is iterative to avoid stack overflow on pathological inputs."
🔮 Likely Follow-up Questions
  • "What if the array is read-only (can't modify it)?" → Use the heap approach. QuickSelect modifies the array in-place.
  • "What if it's a stream of numbers?" → Min-heap of size k; O(log k) per new element.
  • "Find the k closest points to the origin?" → Same pattern (#973): QuickSelect by distance.
  • "Top K Frequent Elements?" → Bucket sort or heap by frequency (#347).
  • "Guaranteed O(n) worst case?" → Median of Medians — discuss tradeoffs (higher constant, complex to implement).

7 Summary & Key Takeaways ▾
kth largest = element at index n−k in sorted-ascending order
QuickSelect partitions once per level; only recurses into one half — O(n) on average
After partition, pivot's index is permanent — it's in its correct sorted position forever
