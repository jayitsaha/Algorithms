LeetCode #741

🔴 Cherry Pickup

Traverse an N×N grid twice (there and back) to collect the maximum cherries, avoiding thorns.

Hard Dynamic Programming 3D DP Two Paths Time: O(N³) Space: O(N³)
1 Understanding the Problem ▾

You have an N×N grid where each cell is one of three values:

  • 0 — empty cell you can walk through
  • 1 — cell with a cherry 🍒 (collected once walked on, then becomes 0)
  • -1 — thorn 🌵 (impassable)

You start at (0,0) and want to reach (N-1, N-1), then return to (0,0). Only moves allowed: right or down on the way there; left or up on the return. You want to maximize total cherries collected. Return 0 if no path exists.

💡 Key Insight — The Two-Path Equivalence

A round trip (go + return) is mathematically equivalent to two people walking simultaneously from (0,0) to (N-1,N-1). Both take steps at the same time — always moving right or down. If they land on the same cell, only one cherry is picked. This transforms a sequential problem into a simultaneous two-path problem.

✗ Naïve: Two greedy passes

Greedily maximize first pass,
then try to maximize second.
FAILS — first greedy pick can block the second path from collecting more overall.

✓ Correct: Simultaneous two paths

Consider both paths at once via DP.
State = (step t, col of P1, col of P2).
Row of each person is derived: r = t - c.

Edge Cases: If (0,0) or (N-1,N-1) is a thorn → return 0. If any required path is fully blocked → return 0. A cell shared by both paths yields only 1 cherry.

2 The Algorithm: 3D DP — Simultaneous Two-Path ▾

Instead of thinking about one path going forward and one returning, we model two people (P1 and P2) both starting at (0,0) and moving to (N-1,N-1) in synchrony. At each time step t, both move one step right or down.

📐 State Definition & Recurrence
State: dp[t][c1][c2] = max cherries collectible when P1 is at column c1 and P2 is at column c2 after t total steps

Row of P1: r1 = t - c1    Row of P2: r2 = t - c2

dp[t][c1][c2] = cherries_at(r1,c1) + cherries_at(r2,c2) [if c1≠c2 else once]
     + max over 4 previous (dc1, dc2) pairs of dp[t-1][c1-dc1][c2-dc2]
1 Symmetry trick: Enforce c1 ≤ c2 always — by symmetry, if P2 is to the left of P1 just swap them. This halves the state space.
2 Validate cells: For a given (t, c1, c2), compute r1 = t - c1 and r2 = t - c2. Skip if any row/col is out of bounds or the cell is a thorn (-1).
3 Count cherries: Add grid[r1][c1]. If c1 ≠ c2, also add grid[r2][c2] (distinct cells). If c1 == c2, both persons occupy the same cell — count only once.
4 Transition: Each person came from either the cell above (row-1, same col) or the cell to the left (same row, col-1). There are 4 combinations: (up,up), (up,left), (left,up), (left,left).
5 Answer: max(0, dp[2*(N-1)][N-1][N-1]) — when both reach the bottom-right corner after 2*(N-1) steps. Return 0 if this is negative infinity (no path exists).
🔐 Why t Steps Implies r = t - c

Each step is either right (c += 1) or down (r += 1). So after t total steps, r + c = t, giving r = t - c. Both persons always have the same total step count — this is the synchronization invariant that reduces 4D state (t, r1, c1, r2, c2) to 3D (t, c1, c2).

3 Why It Works: The Invariant & Proof ▾
🔐 The Invariant

At every time step t, both persons have made exactly t total moves (right or down). This means row + col = t for each person at all times. The DP explores all possible column assignments for both persons at each step — this covers all valid pairs of paths simultaneously.

Optimal Substructure: The maximum cherries collectible from (0,0) to (N-1,N-1) by two paths is fully determined by the maximum cherries at the previous step plus whatever each person picks today. No future step affects past decisions — once a cherry is picked, it's gone.

Overlapping Subproblems: Without memoization, the same (t, c1, c2) state is visited exponentially many times through different sequences of moves. The DP table stores each state once — O(N³) entries total.

⚠️ Why Sequential Greedy Fails

Consider this 3×3 grid: path 1 greedily collects the center cherry, then path 2 is forced around it — possibly missing cherries that a joint optimization would have found. The first path's choice permanently alters the landscape for the second. Only considering both paths simultaneously captures the correct global optimum.

Edge case handling: If a cell is a thorn, dp[t][c1][c2] = -∞ (propagated as a sentinel). The final answer is max(0, dp) — returning 0 if the corner is unreachable.

4 The Code (Annotated) ▾
def cherryPickup(grid):// entry point
N = len(grid)// grid is N×N
NEG_INF = float('-inf')// sentinel for "no valid path reaches here"
# dp[c1][c2] for current step t (rolling 2D array)
dp = [[NEG_INF]*N for _ in range(N)]// init all states as unreachable
dp[0][0] = grid[0][0]// both start at (0,0); collect its cherry if any
for t in range(1, 2*N - 1):// t goes from 1 to 2*(N-1) total steps
ndp = [[NEG_INF]*N for _ in range(N)]// new layer for step t
for c1 in range(max(0,t-N+1), min(N,t+1)):// valid col range for P1
for c2 in range(c1, min(N,t+1)):// c2 >= c1 (symmetry; skip c2 < c1)
r1, r2 = t-c1, t-c2// derive rows from t = r + c
if r1 >= N or r2 >= N: continue// row out of bounds
if grid[r1][c1]==-1 or grid[r2][c2]==-1: continue// thorn — skip
cherries = grid[r1][c1]// P1's cherry at this step
if c1 != c2: cherries += grid[r2][c2]// P2's cherry, only if different cell
best = NEG_INF
for dc1 in (0, 1): # P1 came from up(0) or left(1)
for dc2 in (0, 1): # P2 came from up(0) or left(1)
pc1, pc2 = c1-dc1, c2-dc2// previous columns
if 0<=pc1<=pc2// must be in bounds and pc1<=pc2
best = max(best, dp[pc1][pc2])// pick best previous state
if best != NEG_INF:// at least one valid predecessor exists
ndp[c1][c2] = best + cherries// update with cherries collected this step
dp = ndp// advance to next time step
return max(0, dp[N-1][N-1])// 0 if unreachable; both must end at (N-1,N-1)
⚠️ Common Mistake — Forgetting the Symmetry Constraint

If you allow c2 < c1, you duplicate every state (swapping P1 and P2 gives the same scenario). Always enforce c1 ≤ c2. When looking up predecessors, also enforce pc1 ≤ pc2.

📊 Complexity Analysis

Time: O(N³) — outer loop is O(N) steps; inner two loops are O(N²) column pairs; inner 4 transitions are O(1). Total: O(N · N² · 4) = O(N³).
Space: O(N²) with rolling array (shown above), or O(N³) if storing all t layers.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize Sequential → Simultaneous

The problem says "go then return" but the return path (bottom-right to top-left, moving left or up) is the same as a second forward path (top-left to bottom-right, moving right or down) traversed in reverse. Flip the return into a second forward path.

Step 2: Identify Why Greedy Fails

Greedy pass 1 can "use up" cherries that, had they been saved, could have been combined with pass 2 for a better total. This interdependence means we need DP — we must consider all joint decisions.

Step 3: Define the State

Both persons always move right or down, so after t steps each has r + c = t. Track (t, c1, c2) — 3D state. Row is implicit. This is the key dimensionality reduction from 4D to 3D.

Step 4: Handle Same-Cell Collision

When c1 == c2 (and r1 == r2), both persons are on the same cell — count cherry only once. This is a subtle but critical detail that distinguishes correct from incorrect solutions.

Step 5: Exploit Symmetry

By always keeping c1 ≤ c2, you avoid computing mirror states. Halves work. This optimization also simplifies the predecessor lookup constraint.

🗣️ What to Say to the Interviewer

"The return path moving left/up is equivalent to a second person walking right/down from the start simultaneously. So I'll model this as two synchronized paths on the grid."

"My state is (t, c1, c2) where t is the step count and c1, c2 are the columns of person 1 and 2. Since both make exactly t moves total, row of person i is t minus their column."

"At each state I pick up cherries from both cells — counting only once if they're on the same cell. I transition from the 4 possible previous (dc1, dc2) combinations."

"Time complexity is O(N³), space O(N²) with a rolling array. The answer is dp[N-1][N-1] at step 2*(N-1)."
🎯 Pattern Recognition Signals
  • "Round trip" or "go and return" on a grid
  • Must maximize combined collection over two traversals
  • Items disappear once collected (shared resource between paths)
  • Only right/down moves (or their reversal equivalents)

7 Summary & Key Takeaways ▾
A round trip = two simultaneous forward paths. The return path reversed is identical to a second forward path.
State (t, c1, c2) tracks step count and both persons' columns. Row is derived: r = t - c.
When both persons share a cell (c1 == c2 implies r1 == r2), count the cherry only once.
Enforce c1 ≤ c2 by symmetry — halves the state space and simplifies transitions.
A cell becoming a thorn (-1) propagates -∞ through the DP — any state depending on it remains unreachable.
Final answer: max(0, dp[N-1][N-1]) — 0 if bottom-right is unreachable from any valid two-path combo.

  • When one path's choice affects the other's available resources
