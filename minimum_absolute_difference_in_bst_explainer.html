Minimum Absolute Difference in BST

Given the root of a Binary Search Tree (BST), return the minimum absolute difference between the values of any two different nodes.

Easy Trees BST · In-Order Traversal Time: O(n) Space: O(h)
1 Understanding the Problem ▾

We have a Binary Search Tree (BST). We need to find the smallest possible difference between the values of any two distinct nodes in the tree — not just parent and child, but any pair anywhere in the tree.

💡 Key Insight

In a BST, an in-order traversal (Left → Root → Right) visits nodes in strictly ascending sorted order. Once the values are sorted, the minimum difference can only come from two adjacent values in that sorted sequence — values far apart can never produce the minimum.

Consider this example BST where root=4, left subtree has [2,1,3], and right=6:

In-order traversal gives: 1, 2, 3, 4, 6. The differences between adjacent pairs: |2-1|=1, |3-2|=1, |4-3|=1, |6-4|=2. Minimum = 1.

✓ Optimal: use BST in-order

In-order → [1, 2, 3, 4, 6]

One pass, O(n) time, O(h) space

✗ Brute force: check all pairs

Edge Cases: Minimum 2 nodes (guaranteed by constraints). Very large values up to 10⁵. Heavily skewed tree (O(n) call stack depth). All values identical is not possible in a valid BST with distinct values.

2 The Algorithm: In-Order Traversal with a Prev Pointer ▾

We perform a standard BST in-order DFS (Left → Root → Right) while tracking the value of the previously visited node in a variable called prev. Each time we land on a node, we compute its gap with prev and update our running minimum.

1 Initialize: Set min_diff = infinity and prev = None. prev tracks the value of the last-visited node. It starts as None because no node has been visited yet.
2 Recurse left first: Call inorder(node.left). This reaches the smallest values first, ensuring our traversal is ascending. The current node waits while the left subtree is fully processed.
3 Process current node: If prev is not None, compute diff = node.val - prev (always non-negative since we visit in ascending order). Update min_diff = min(min_diff, diff). Then set prev = node.val.
4 Recurse right: Call inorder(node.right) to process larger values. They will compare against the current prev which now holds this node's value.
5 Return min_diff: After visiting all n nodes, min_diff holds the minimum gap between any two adjacent values in sorted order — which is provably equal to the minimum gap across all possible pairs.
🔑 Why Adjacent Pairs Suffice (Proof Sketch)

Suppose sorted values are a < b < c. Then |a−c| = (c−a) = (c−b) + (b−a) = |c−b| + |b−a|. So |a−c| ≥ max(|b−a|, |c−b|) ≥ min(|b−a|, |c−b|). The difference between any non-adjacent pair is at least as large as some adjacent pair's difference. Therefore the global minimum must come from an adjacent pair.

3 Why It Works: The Invariant ▾
🔐 The Two-Part Invariant

Invariant 1: At any point during traversal, prev holds the value of the most recently visited node in sorted order — i.e., the largest BST value seen so far.

Invariant 2: min_diff holds the minimum difference seen so far among all adjacent pairs processed. Since we visit ascending, node.val − prev is always ≥ 0 — no need for abs().

Why in-order produces sorted output: The BST property states: all values in the left subtree < root < all values in the right subtree. In-order traversal visits left → root → right. By structural induction, this produces values in strictly ascending order for any valid BST.

Why we process root between left and right: Left subtree values are all smaller than root, and right subtree values are all larger. Processing left-then-root-then-right respects this ordering, making prev always hold the largest value seen so far when we arrive at the next node.

⚠️ Common Mistake: Storing the TreeNode instead of its value

A frequent bug is storing the entire TreeNode object in prev, then computing node.val - prev.val. This works but is error-prone if prev is None (you'd get None.val). Store prev = node.val (just the integer) and guard with if self.prev is not None.

⚠️ Python Closure Pitfall with nonlocal

If you define prev = None; min_diff = float('inf') in the outer function and try to rebind them inside a nested inorder() function, Python raises UnboundLocalError. Fix with either: (a) nonlocal prev, min_diff at the top of inorder(), or (b) use self.prev / self.min_diff as instance variables, or (c) wrap in a mutable container like state = [None, float('inf')].

Solution 2 — Collect In-Order Array, then Scan (Simpler, O(n) space)

def getMinimumDifference(root):
vals = [] ← collect all node values
def inorder(node):
if not node: return
inorder(node.left)
vals.append(node.val) ← append in sorted order
inorder(node.right)
inorder(root)
return min(vals[i]-vals[i-1] for i in range(1,len(vals))) ← scan adjacent pairs

Solution 3 — Brute Force: collect all values, sort explicitly, then scan (O(n log n))

def getMinimumDifference(root):
vals = []
def dfs(node): ← any order works; we sort later
if not node: return
vals.append(node.val)
dfs(node.left); dfs(node.right)
dfs(root)
vals.sort() ← sort separately (in-order gives this free!)
return min(vals[i]-vals[i-1] for i in range(1,len(vals)))
💡 Complexity Summary

In-Order with Prev (Solution 1): Time O(n) — one visit per node. Space O(h) — call stack. h = O(log n) balanced, O(n) skewed.
Collect Array (Solution 2): Time O(n), Space O(n) — extra array holds all values.
Brute Force Sort (Solution 3): Time O(n log n) for sort, Space O(n). Ignores BST structure entirely.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Notice It's a BST, Not a Generic Binary Tree

The moment you see "BST" and "minimum difference" together, the signal is clear: in-order traversal = sorted sequence. BST's defining property is that in-order output is sorted. This is the key insight every interviewer wants to hear.

Step 2: State the Brute Force (and its flaw)

"The naive approach: collect all values, sort them, scan adjacent pairs — O(n log n) time, O(n) space. But this ignores the BST structure, which already gives us sorted order for free."

Step 3: Propose the Optimization

"In-order traversal visits values in sorted order in O(n). I don't need to collect the whole array — I only need to compare each node to the previous one. So I track a single 'prev' variable, compute the gap, and update the minimum. This cuts space from O(n) to O(h)."

Step 4: Clarify the Space Complexity

"Space is O(h) where h is tree height — O(log n) for a balanced BST, O(n) for a degenerate skewed tree (essentially a linked list). The interviewer may ask about the worst case: always O(n)."

🗣️ What to Say to the Interviewer

"Key observation: it's a BST, so in-order traversal gives values in ascending sorted order.
Minimum difference between any two nodes = minimum gap between adjacent values in sorted order.
Plan: DFS in-order. Track prev = last visited value. At each node: diff = curr.val - prev.
Update min_diff = min(min_diff, diff). Then set prev = curr.val.
Time O(n) — one pass. Space O(h) — recursion stack. O(log n) for balanced BST."
🔮 Likely Follow-up Questions
  • "What if it were a general binary tree?" → Collect all values, sort them, then scan adjacent pairs. O(n log n) time. Can't exploit BST property.
  • "Implement this iteratively?" → Use an explicit stack to simulate in-order DFS. Same O(n)/O(h) complexity, no recursion.
  • "Kth smallest element in BST?" → Same in-order traversal, decrement a counter at each node, stop when counter hits 0 (#230).
  • "How to validate a BST?" → In-order must produce strictly ascending values. Track prev, check curr.val > prev at each step (#98).

7 Summary & Key Takeaways ▾
In-order traversal of a BST always yields values in ascending sorted order — no explicit sort needed
Minimum difference between any two nodes equals the minimum gap between adjacent values in sorted order
Track only prev (a single integer) — no need to store the full array, saving O(n) space
The difference node.val − prev is always non-negative during in-order traversal; no need for abs()
Space is O(h): O(log n) for balanced BST, O(n) for a degenerate chain (the call stack depth)
