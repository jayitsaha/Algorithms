Merge Sorted Array

LeetCode #88 – Merge two sorted arrays in-place, filling from the end

Easy Two Pointers Merge from End O(m+n) Time O(1) Space
1Understanding the Problem▼

Merge nums2 into nums1 in-place. nums1 has m real elements followed by n zeros (space reserved). nums2 has n elements.

💡 Key Insight

Merge from the end instead of the front. No overwriting risk — we write to positions that are already "done" or empty (zeros). Three pointers: p1 at end of nums1's real elements, p2 at end of nums2, k at the last slot.

⚠️ Why not front-to-back?

Merging from the front would overwrite unprocessed elements of nums1. Merging from the end writes to the safe empty zone first.

2The Algorithm: Merge from End▼
1p1 = m-1 (last real element of nums1), p2 = n-1 (last of nums2), k = m+n-1 (last slot)
2While p1 ≥ 0 and p2 ≥ 0: place max(nums1[p1], nums2[p2]) at nums1[k], advance that pointer and k--
3If p2 ≥ 0 remaining: copy rest of nums2 into nums1 (p1 exhausted, nums2 elements are smaller)
3The Code (Annotated)▼
def merge(nums1, m, nums2, n):
p1, p2, k = m-1, n-1, m+n-1
while p1 >= 0 and p2 >= 0:
if nums1[p1] > nums2[p2]:
nums1[k] = nums1[p1]; p1 -= 1
else:
nums1[k] = nums2[p2]; p2 -= 1
k -= 1
# copy remaining nums2 if any
nums1[:p2+1] = nums2[:p2+1]
