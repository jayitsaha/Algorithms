LeetCode #904

Fruit Into Baskets

Find the maximum number of fruits you can collect — you carry at most 2 basket types (contiguous subarray with at most 2 distinct values).

Medium Sliding Window At Most 2 Types Time: O(n) Space: O(1)
1 Understanding the Problem ▾

You walk along a row of trees, each holding one type of fruit. You have exactly 2 baskets, and each basket holds only one fruit type — but unlimited quantity of that type. You must pick from a contiguous stretch of trees and want to maximize the total fruit count.

💡 Key Insight

Strip away the story. The problem is asking: "Find the longest contiguous subarray containing at most 2 distinct values." This is the canonical "at most K distinct" sliding window problem with K = 2. Once you see this, the algorithm follows directly.

✓ Example: fruits = [1, 2, 1]

Basket A = type 1 (count: 2)
Basket B = type 2 (count: 1)
Pick all 3 trees → Answer: 3

✗ Example: fruits = [0, 1, 2, 2]

Can't pick all 4 (types 0, 1, 2 = 3 baskets needed)
Best subarray: [1, 2, 2] → length 3
Answer: 3

What makes a window invalid? Having 3 or more distinct fruit types simultaneously inside the window. When that happens, we must shrink from the left until exactly one type disappears entirely.

Edge cases: Single tree (answer = 1). All same type (answer = length). All alternating two types (answer = length). All distinct values (answer = 2). Empty array (answer = 0).

2 The Algorithm: Variable Sliding Window ▾

We use a variable-size sliding window. A hash map basket counts how many fruits of each type are currently inside the window. The window expands right on every iteration; it shrinks left reactively whenever a 3rd distinct type enters.

1 Initialize: left = 0, res = 0, basket = {}. The window [left, right] starts empty.
2 Expand right: For each index right, add fruits[right] to the basket map: basket[fruit] = basket.get(fruit, 0) + 1. The window now includes this fruit.
3 Shrink if invalid (while loop): If len(basket) > 2, we have a 3rd type. Remove fruits[left] from basket (decrement count; delete if count reaches 0). Advance left += 1. Keep shrinking until len(basket) ≤ 2.
4 Update result: After shrinking, window [left, right] is valid. res = max(res, right - left + 1). This records the largest valid window seen ending at right.
5 Return res: After right finishes the full scan, res holds the answer.
🔑 Why not just track two variables instead of a map?

You could store the two active types as typeA and typeB and count occurrences directly. It works, but becomes tricky when a type exits and a new one enters — you need to know how many of the exiting type remain. The map handles this naturally, and generalizes to "at most K distinct" with zero code change beyond replacing 2 with K.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After every shrink loop, the window [left, right] contains at most 2 distinct fruit types, and it is the largest valid window ending at that exact right position. Every candidate answer is captured by res at the moment right reaches the end of that answer's subarray.

Why shrinking is correct: When a 3rd type enters at position right, any valid window ending at right must exclude at least one of the three types entirely. We shrink from the left — removing occurrences one at a time — until one type's count drops to zero and gets deleted from the map. The left pointer can only move forward, never backward, which is safe because shorter windows ending at an earlier right have already been recorded.

Why we never miss the optimal answer: Suppose the optimal answer is subarray [i, j]. When right reaches j, the left pointer can only be ≤ i (because the shrink only moves left forward when forced). Therefore right - left + 1 ≥ j - i + 1, so res is at least the optimal value.

⚠️ Common Mistake: Updating res BEFORE shrinking

If you write res = max(...) before the while loop, you record invalid windows (those with 3 types). Always ensure the window is valid first, then record the size. The shrink must be a while loop — not if — because a single left step may only reduce one fruit's count without eliminating the type.

4 The Code (Annotated) ▾

Solution 1 — Variable Sliding Window with Hash Map (Optimal, Interview Pick)

def totalFruit(fruits): ← input: list of ints (fruit type per tree)
basket = {} ← type → count of that type in current window
left = 0 ← left boundary of the sliding window
res = 0 ← maximum valid window length found so far
for right, fruit in enumerate(fruits): ← right boundary moves forward every step
basket[fruit] = basket.get(fruit, 0) + 1 ← add new fruit: create entry or increment
while len(basket) > 2: ← 3 distinct types → window invalid, must shrink
left_fruit = fruits[left] ← what type is at the left edge?
basket[left_fruit] -= 1 ← remove one of that type from window
if basket[left_fruit] == 0: ← if count hits zero, type fully leaves window
del basket[left_fruit] ← clean up map → now ≤ 2 keys
left += 1 ← shrink left boundary by one position
res = max(res, right - left + 1) ← window valid: record its size
return res ← longest subarray with ≤ 2 distinct types
⚠️ Why check basket[left_fruit] == 0 before deleting?

If there are multiple trees of type left_fruit in the window, removing one instance does not evict the type entirely — it's still in the map with count > 0. The delete only happens when we have exhausted every occurrence of that type in the current window. Not deleting means len(basket) stays at 3 and the while loop continues correctly.

Solution 2 — Brute Force O(n²) (Good starting point)

def totalFruit(fruits): ← try every possible starting index
res = 0
for i in range(len(fruits)): ← O(n) start positions
basket = set() ← track distinct types in this window
for j in range(i, len(fruits)): ← O(n) end positions
basket.add(fruits[j])
if len(basket) > 2: break ← 3rd type → this start can't extend further
res = max(res, j - i + 1) ← valid window → record length
return res ← O(n²) time — TLE on n=10^5
💡 Complexity Summary

Variable Sliding Window: Time O(n) — each element enters the window once (right pass) and exits at most once (left shrink). The total work is O(2n) = O(n). Space O(1) — the basket map holds at most 3 keys momentarily, then ≤ 2.
Brute Force: Time O(n²) — all pairs. Space O(1) for the set.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Abstract Away the Story

The first thing to say: "This is equivalent to finding the longest subarray with at most 2 distinct integers." Fruit types → integer values. Baskets → distinct value limit. Contiguous picking → subarray. This reframing immediately connects to a known pattern.

Step 2: Propose Brute Force

"The naive approach tries every starting index and extends right until a 3rd type appears. O(n²). With n up to 10⁵, this is too slow. We need a linear approach."

Step 3: Identify the Sliding Window Signals

Two signals: (1) "Longest contiguous subarray" and (2) "at most K distinct values." Together, these almost always mean variable sliding window. The critical question: what makes the window invalid? Ans: more than 2 distinct types. What shrinking strategy restores validity? Remove from the left until one type is evicted.

Step 4: Code it up precisely

Key implementation points: use a hash map (not a set) because you need counts to know when a type is fully evicted. Use a while loop for the shrink, not an if statement. Record res after the while loop, not before.

Step 5: Generalize unprompted

Say "this directly generalizes to K baskets — just change the constant 2 to K. Same O(n) algorithm." This is #340 (Longest Substring with At Most K Distinct Characters) and shows depth beyond the immediate problem.

🗣️ What to Say to the Interviewer

"This is really asking for the longest subarray with at most 2 distinct values.
I'll use a variable sliding window: expand right each step, shrink left when we have 3 distinct types.
A hash map tracks the count of each type in the window. When a type's count hits zero, I remove it.
Each element enters once and exits at most once — total O(n). Space is O(1) since the map stays ≤ 3 keys.
Generalizes to K baskets with a one-character change."
🔮 Likely Follow-up Questions
  • "What if you had K baskets?" → Change > 2 to > K. LeetCode #340.
  • "What if you want exactly 2 distinct types?" → atMost(2) - atMost(1). LeetCode #992 technique.
  • "What if the trees are arranged in a circle?" → Run on array doubled, cap window at n.
  • "What's the minimum window containing at least one of each type?" → Different constraint; track satisfaction count.

7 Summary & Key Takeaways ▾
Abstract the story: "fruit/baskets" → "values/distinct limit" → "longest subarray with ≤ 2 distinct"
Variable sliding window: expand right greedily, shrink left reactively
Use a hash map of counts — not a set — so you know when a type is fully evicted
The shrink is a while loop, not if: one left step may not eliminate the extra type
Record res AFTER the while loop, ensuring the window is always valid when measured
Each element enters and exits the window at most once: total work O(2n) = O(n)
Generalize: replace 2 with K for "at most K baskets" — same algorithm, LeetCode #340
ApproachTimeSpaceNotes
Brute Force (nested loops) O(n²) O(1) TLE on large inputs; fine starting point
Variable Sliding Window ✓ O(n) O(1) Optimal; interview pick; generalizes to K
🎯 When to Recognize the Variable Sliding Window Pattern
  • "Longest/shortest contiguous subarray/substring satisfying a constraint" → variable window
  • "At most K distinct elements/characters" → variable window + hash map counting
  • The constraint is violated by adding one element → shrink left until valid
  • You need to know when an element fully leaves the window → map counts, not just set membership
