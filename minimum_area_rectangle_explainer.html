LeetCode #939

Minimum Area Rectangle

Given a set of points in the XY plane, find the minimum area of an axis-aligned rectangle formed by any four of those points. Return 0 if no such rectangle can be formed.

Medium Math Hash Set Diagonal Check Time: O(n^2) Space: O(n)
1 Understanding the Problem ▼

We are given a list of integer coordinate points. We want to find four points that form a rectangle with sides parallel to the X and Y axes (axis-aligned), and we want the smallest possible area among all such rectangles. If no rectangle exists, return 0.

Key Insight

An axis-aligned rectangle is completely defined by two opposite diagonal corners. If you pick any two points that differ in both x and y, they could be diagonal corners. The other two corners are then forced to be (x1, y2) and (x2, y1). If both exist in our point set, we have a rectangle. This collapses a 4-point search into a 2-point search.

Valid Rectangle

Diagonal: (1,1) and (3,3)
Forced: (1,3) and (3,1)
Both exist in set
Area = |3-1| x |3-1| = 4

No Rectangle

Diagonal: (0,0) and (2,2)
Forced: (0,2) and (2,0)
(2,0) missing from set
Skip this pair

Key constraints: Points have integer coordinates, no duplicate points. Only axis-aligned rectangles count (sides horizontal and vertical). With up to 500 points, brute-force over all 4-point combinations would be O(n^4) = 500^4 / 24 which is too slow.

Edge cases: Fewer than 4 points; all points on one horizontal or vertical line; all points collinear in any axis direction. All these return 0.

2 The Algorithm: Hash Set Diagonal Enumeration ▼

We enumerate every pair of points as candidate diagonal corners of a rectangle. For valid candidates (x coords differ AND y coords differ), we check if the two forced corners exist via O(1) hash set lookup.

1 Build a point set: Convert all points to a set of tuples. This is the O(n) preprocessing step that enables O(1) membership checks throughout the algorithm.
2 Enumerate all unique pairs: Use nested loops for i in range(n): for j in range(i+1, n). Each unique pair (p1, p2) is examined exactly once as a candidate diagonal.
3 Guard the collinearity condition: Skip any pair where x1 == x2 or y1 == y2. Such pairs are on the same vertical or horizontal line and cannot form a proper diagonal (rectangle would have zero area).
4 Check the two forced corners: If the pair passes the guard, look up (x1, y2) and (x2, y1) in the set. If both exist, compute area = |x2-x1| * |y2-y1| and update the running minimum.
5 Return result: After all pairs, if min_area was updated from infinity, return it. Otherwise return 0.
Why This Works Geometrically

For an axis-aligned rectangle, knowing one diagonal means knowing all four vertices. If the diagonal runs from (x1, y1) to (x2, y2), then geometry forces the other two corners to be (x1, y2) (same column as the first, same row as the second) and (x2, y1) (vice versa). There is zero freedom. This is unique to axis-aligned shapes.

Complexity: O(n^2) pairs, each with O(1) lookup = O(n^2) total time, vs O(n^4) brute-force. Space O(n) for the set.

3 Why It Works: The Invariant ▼
The Invariant

At every point during the nested loop, min_area holds the minimum valid axis-aligned rectangle area from all pairs examined so far. When we finish all n*(n-1)/2 pairs, it holds the global answer.

Correctness: Every axis-aligned rectangle has exactly one pair of diagonals. Our loop covers every unordered pair of points exactly once (i < j ensures no repeat). If any valid rectangle exists, its diagonal pair will be encountered, the forced corners will be found in the set, and the area will be computed and tracked.

Why require x1 != x2 AND y1 != y2? If x1 == x2, the two points share a column. The "rectangle" they would form has zero width (degenerate). Similarly, equal y gives zero height. Both conditions together ensure we only consider proper non-degenerate diagonals. Without this guard, degenerate corners like (x1, y1) itself might be found in the set, producing false zero-area hits.

Completeness Argument

We cannot miss any rectangle. Every valid axis-aligned rectangle has exactly 2 diagonal pairs. Our O(n^2) pair enumeration visits every pair. When we reach the diagonal pair of the minimum rectangle, both forced corners will be in the set and the area will be recorded. QED.

Edge case handling: If no pair ever passes both the collinearity guard and the forced-corner check, min_area stays at float('inf'), and we return 0 as required.

4 The Code (Annotated) ▼
def minAreaRect(points):
point_set = set(map(tuple, points)) # Build O(n) lookup structure: {(1,1),(1,3),(3,1),(3,3),(2,2)}
min_area = float('inf') # Sentinel: any real area will beat this
n = len(points)
for i in range(n): # First corner of diagonal candidate
x1, y1 = points[i]
for j in range(i + 1, n): # j > i avoids duplicate pairs
x2, y2 = points[j]
if x1 != x2 and y1 != y2: # Both coords must differ = valid diagonal
if (x1, y2) in point_set and (x2, y1) in point_set: # O(1) lookups
area = abs(x2 - x1) * abs(y2 - y1) # Width x Height
min_area = min(min_area, area) # Track minimum
return min_area if min_area != float('inf') else 0 # 0 if no rectangle found
Common Mistake: Missing the Collinearity Guard

If you remove if x1 != x2 and y1 != y2, you will try to look up forced corners for degenerate pairs. For two points sharing x=1 as (1,1) and (1,3), the forced corners become (1,3) and (1,1) — which are the points themselves! Both exist in the set, and area = 0. You will incorrectly return 0 even when valid rectangles exist. This guard is essential.

Complexity Summary

Time: O(n^2) — two nested loops each O(n), with O(1) hash set lookups inside. For n=500 that is ~125,000 operations, not 500^4 = 62.5 billion. Space: O(n) — the point set. No extra DP table or matrix needed.

5 Interview Strategy: How to Arrive at This ▼
Step 1: Exploit Axis-Alignment

The phrase "sides parallel to the X and Y axes" is the critical constraint. Axis-aligned rectangles have a special property: one diagonal uniquely determines all four corners. Immediately think: can I reduce from 4-point enumeration to 2-point enumeration?

Step 2: Identify the Lookup Bottleneck

After picking a diagonal pair O(n^2), you need to verify two forced corners. Naive verification is O(n) scan per pair = O(n^3) total. Build a hash set upfront for O(1) verification = O(n^2) total. Classic preprocess-then-query pattern.

Step 3: Guard Degenerate Cases

Before checking forced corners, verify x1 != x2 AND y1 != y2. Mentioning this during an interview signals careful thinking. It prevents false positives and shows you understand what "diagonal" means geometrically.

Step 4: State the Approach Cleanly

Initialize min_area = inf. Try all pairs as diagonals. On each valid rectangle, update minimum. Return 0 if no rectangle. This is the clean "minimum over valid configurations" pattern common in geometry problems.

What to Say to the Interviewer

"For axis-aligned rectangles, any two points with different x AND y
coordinates define a potential diagonal. The other two corners are
forced by geometry. So I will preprocess all points into a hash set
for O(1) lookup, then try all O(n^2) pairs as diagonal candidates.
For each valid candidate I check if both forced corners are in the set.
If yes, compute area and track the minimum. Total: O(n^2) time, O(n) space."

Follow-up: What if rectangles need not be axis-aligned? This is LC #963 (Minimum Area Rectangle II). You enumerate pairs as potential sides, check perpendicularity via dot product (two vectors must be orthogonal), and use the vector to find the fourth point. Much more complex and O(n^2) with heavier constant.

Follow-up: Can we do better than O(n^2)? For axis-aligned rectangles, O(n^2) is essentially optimal since we must examine each pair of points at least once to check if they form a valid diagonal. The hash set already makes each check O(1), so this is the best achievable complexity with this approach.

7 Summary & Key Takeaways ▼
An axis-aligned rectangle is fully determined by its two diagonal corners
Build a hash set first: O(n) preprocessing enables O(1) per-pair lookup
Guard both x1 != x2 AND y1 != y2 before looking up forced corners
Initialize min_area = infinity; return 0 if never updated (no rectangle found)
Pattern: reduce search by exploiting geometric constraints, verify via hash set
When to Use This Pattern
  • Geometric problems where axis-alignment constrains the structure
  • Any "find minimum/maximum configuration from points" where fixing 2 determines 4
  • When you need existence checks over a large set — hash set is the tool
  • Problems of the form: "enumerate k-1 elements, look up the k-th in O(1)"
