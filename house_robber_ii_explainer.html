LeetCode #213

House Robber II

Houses arranged in a circle — you cannot rob adjacent houses. Return the maximum loot without triggering the alarm.

Medium Dynamic Programming Two Passes: Exclude First / Last Time: O(n) Space: O(1)
1 Understanding the Problem ▾

You are a robber on a circular street. There are n houses, each holding nums[i] dollars. Adjacent houses have a shared security alarm — robbing two neighbors in a row triggers it. Because the street is circular, house 0 and house n-1 are also neighbors. Return the maximum money you can rob without activating any alarm.

The Circular Twist

House 0 and house n-1 are neighbors. You cannot rob both. This breaks the simple House Robber I approach.

The Reduction

Either you rob house 0 or you don't. Run linear DP twice — once excluding the last, once excluding the first. Take the max.

💡 Key Insight

In any valid robbery plan, house 0 and house n-1 are mutually exclusive — you'll never rob both. So the optimal answer lives entirely within one of two sub-arrays: nums[0..n-2] (exclude last) or nums[1..n-1] (exclude first). Run House Robber I on each; the answer is the maximum of both results.

Concrete example: nums = [2,7,9,3,1]. House 0 (=2) and house 4 (=1) are circular neighbors. Pass A on [2,7,9,3]: best = rob houses 0+2 = 2+9 = 11. Pass B on [7,9,3,1]: best = rob houses 1+3 = 7+3 = 10. Answer = max(11, 10) = 11.

2 The Algorithm: Two-Pass Linear DP ▾

House Robber I (the linear version) uses this recurrence at every house:

📐 Recurrence — House Robber I
dp[i] = max( dp[i-1],  dp[i-2] + nums[i] )
               ↑ skip house i      ↑ rob house i (can't have robbed i-1)

We use pprev and prev rolling variables (no array) — O(1) space. House Robber II wraps this helper in two calls:

1 Edge case: If only one house, return nums[0] — nothing to worry about.
2 Pass A — exclude last house: run rob_linear(nums[0..n-2]). This allows house 0 to be robbed but removes its circular neighbor (house n-1) from consideration.
3 Pass B — exclude first house: run rob_linear(nums[1..n-1]). This allows house n-1 to be robbed but removes house 0 (its circular neighbor).
4 Return max(pass_a, pass_b). Every valid circular robbery plan is covered by one of the two passes.
🔐 Why Two Passes Are Exhaustive

The optimal solution either does include house 0 or it doesn't. If it includes house 0, house n-1 is off-limits — Pass A already handles this by ending at n-2. If it doesn't include house 0, house n-1 is free — Pass B handles this by starting at 1. No valid plan is missed.

3 Why It Works: Optimal Substructure & the Invariant ▾
🔐 The DP Invariant

After processing each house i, prev always holds the maximum loot obtainable from houses 0 through i, inclusive, without ever robbing two adjacent houses. This invariant is maintained by the rolling update: pprev, prev = prev, max(prev, pprev + num).

Optimal Substructure: The best answer for houses 0..i depends only on the best answers for 0..i-1 and 0..i-2. This allows us to build the solution bottom-up without reconsidering earlier houses.

Overlapping Subproblems: A naive recursion would recompute f(i) from scratch on every branch of the decision tree, exponentially duplicating work. The rolling DP computes each state exactly once.

⚠️ Common Wrong Approach: Skipping Both Endpoints

A tempting shortcut is to run DP only on nums[1..n-2], skipping both the first and last house. This is wrong. It misses the optimal solution whenever house 0 or house n-1 is the single best house to rob. For example, [100, 1, 1, 1]: skipping both endpoints yields 1; but the correct answer is 100 (rob only house 0).

Edge Cases: n=1 — return nums[0] directly. n=2 — return max(nums[0], nums[1]) (both passes handle this but the single-element edge case is caught early). n≥3 — two-pass DP works correctly for all arrangements.

4 The Code (Annotated) ▾

Solution 1 — Two-Pass Bottom-Up DP, O(n) time O(1) space Interview Pick

def rob(nums: list[int]) -> int:
if len(nums) == 1:  → single house: no circular neighbor, just take it
return nums[0]
 
def rob_linear(arr):  → classic House Robber I on any sub-array
pprev, prev = 0, 0  → best loot through i-2 and i-1; start at 0 (nothing robbed)
for num in arr:  → scan each house left-to-right
curr = max(prev, pprev + num)  → skip house (keep prev) OR rob it (pprev + num)
pprev, prev = prev, curr  → slide window: old prev → pprev, curr → prev
return prev  → prev holds maximum loot for the full arr
 
pass_a = rob_linear(nums[:-1])  → exclude house n-1 (nums[0..n-2])
pass_b = rob_linear(nums[1:])   → exclude house 0 (nums[1..n-1])
return max(pass_a, pass_b)  → best of both; covers all valid circular plans
⚠️ Off-by-One Slice Trap

nums[:-1] = indices 0 to n-2 (drops the last). nums[1:] = indices 1 to n-1 (drops the first). A frequent bug is using nums[:-1] for both passes — this runs Pass B on the wrong sub-array and gives a wrong answer.

Solution 2 — Top-Down Memoization (same complexity, useful for derivation)

from functools import lru_cache
 
def rob(nums: list[int]) -> int:
if len(nums) == 1: return nums[0]
 
def rob_range(lo, hi):   → max loot on nums[lo..hi] inclusive
@lru_cache(maxsize=None)
def dp(i):   → max loot from house i to hi
if i > hi: return 0   → base: no houses left
skip = dp(i + 1)   → skip house i; move to i+1
rob = nums[i] + dp(i + 2)   → rob i; skip i+1 (adjacent); move to i+2
return max(skip, rob)
result = dp(lo)
dp.cache_clear()   → clear cache so pass B starts fresh
return result
 
return max(rob_range(0, len(nums)-2),   → Pass A
rob_range(1, len(nums)-1))   → Pass B

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1 — Recognize the DP Pattern

Signals: "maximize sum of non-adjacent elements," "array of values with exclusion rule." Immediately say: "This is the House Robber family." Confirm you know the O(1)-space linear DP for it.

Step 2 — Identify the Circular Twist

"Houses are in a circle" means house 0 and house n-1 are neighbors. Ask yourself: in any valid plan, can I rob both? No. So one of them is always excluded from the optimal answer.

Step 3 — Reduce to the Known Problem

Since exactly one endpoint is excluded, try both exclusions and take the max. This turns a circular problem into two independent linear ones — both solvable with your existing Robber I helper.

Step 4 — Implement and Justify

Write rob_linear first (unit-testable). Then call it twice with the right slices. State complexity: two O(n) passes = O(n) total, O(1) space. Mention the edge case for n=1.

🗣️ What to Say to the Interviewer

"This is essentially House Robber I but with houses arranged in a ring, meaning house 0 and the last house are neighbors — so we can never rob both. The key observation: the optimal solution always excludes at least one endpoint. So I run the linear DP helper twice: once on nums[0..n-2] (excluding the last), and once on nums[1..n-1] (excluding the first). The max of both covers every valid circular configuration. Each pass is O(n) time, O(1) space with rolling variables pprev and prev."
💡 Follow-Up Questions to Expect
  • "Avoid slicing?" — Pass start/end indices to rob_linear instead of creating new arrays.
  • "What about n=2?" — max(nums[0], nums[1]); each pass handles this naturally.
  • "What if it's a tree?" — House Robber III (LC 337): post-order tree DP.
  • "What if at most k adjacent can be robbed?" — Generalize to k-state DP.

7 Summary & Key Takeaways ▾
Circular constraint = house 0 and house n-1 are mutually exclusive. Exploit this!
Run House Robber I twice: on nums[0..n-2] and nums[1..n-1]. Take the max.
Rolling variables pprev / prev give O(1) space — no DP array needed.
Each house: curr = max(prev, pprev + num) — skip or rob, take the better option.
Two O(n) passes → O(n) total time, O(1) space. Optimal for this problem.
🎯 When to Use This Pattern
  • Circular array + adjacency exclusion constraint
  • "Maximize/minimize" over non-adjacent picks
  • You already know the linear DP — circular is just two passes
  • Signal phrases: "arranged in a circle," "first and last are neighbors"
