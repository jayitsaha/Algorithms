LeetCode #992

Subarrays with K Different Integers

Given an integer array nums and an integer k, return the number of good subarrays — subarrays with exactly k different integers.

Hard Sliding Window AtMost(K) Trick Time: O(n) Space: O(k)
1 Understanding the Problem ▾

A good subarray is a contiguous slice of nums that contains exactly k distinct integers. We need to count every such slice efficiently.

Example: nums = [1,2,1,2,3], k = 2. There are exactly 7 valid subarrays (each containing exactly 2 distinct values):

[1,2]
[2,1]
[1,2]
[2,3]
[1,2,1]
[2,1,2]
[1,2,1,2]
Key Insight: Exactly K = AtMost(K) − AtMost(K−1)

We cannot directly use a single sliding window for "exactly K" because when a new element pushes us to K+1 distinct values and we shrink left, the shrunken window has an ambiguous distinct count — it could be anywhere from 1 to K. The standard fix: exactly(K) = atMost(K) − atMost(K−1). We write one clean helper and call it twice.

Why direct sliding window fails for "exactly K": A sliding window naturally handles "at most K" — when distinct count exceeds K, shrink left until the window is valid again. "Valid" for at-most means distinct ≤ K, which has a single well-defined left boundary. "Exactly K" has no such clean boundary after shrinking — we cannot distinguish it from "somewhere between 1 and K distinct". The subtraction trick sidesteps this entirely.

2 The Algorithm: AtMost(K) − AtMost(K−1) ▾

We define a helper atMost(k) that counts subarrays with at most k distinct integers. The sliding window for "at most" is well-defined: expand right, shrink left whenever distinct count exceeds k. At each valid right, every subarray ending there with left boundary in [left..right] is valid, so we add right − left + 1.

1 Define helper atMost(k): count subarrays with at most k distinct integers, using a frequency map and a sliding window with two pointers.
2 Expand right: for each right in the array, add nums[right] to the frequency map (increment its count by 1).
3 Shrink left while invalid: while len(count) > k, decrement count[nums[left]]; if it hits 0, delete the key (distinct count drops by 1); advance left.
4 Count subarrays ending at right: add right − left + 1 to result. This counts all valid subarrays [left..right], [left+1..right], ..., [right..right].
5 Combine: return atMost(k) − atMost(k−1). The subtraction isolates exactly those subarrays with exactly k distinct values.
3 Why It Works: The Invariant ▾
The Invariant

At every right, after the shrink loop, left is the smallest index such that nums[left..right] has at most k distinct integers. All subarrays [left..right], [left+1..right], ..., [right..right] are valid — exactly right − left + 1 of them.

Why right − left + 1 is correct: Since the invariant holds, every sub-window within [left..right] that ends at right has at most k distinct values — a larger window only adds elements, so smaller sub-windows have fewer or equal distinct values. There are exactly right − left + 1 choices of left boundary. Summing over all right gives the total with no double-counting, because each subarray is counted once when processing its rightmost index.

Why subtraction works mathematically: Let S(x) = set of subarrays with at most x distinct integers. Then S(k) \ S(k−1) = set of subarrays with exactly k distinct integers. Since S(k−1) ⊆ S(k), we have |exactly k| = |S(k)| − |S(k−1)|. The two helper calls measure these set sizes directly.

Why delete the key when frequency reaches 0?

The map's key count — len(count) — is what we use as the distinct-element counter. When we decrement a frequency to 0, that value is no longer present in the window. Keeping the zero-frequency key would falsely inflate the distinct count. Deleting it ensures len(count) always equals the number of distinct integers currently in [left..right].

Trace summary for nums=[1,2,1,2,3], k=2: atMost(2) = 12 (verified step-by-step in Section 6). atMost(1) = 5 (only single-distinct windows: [1],[2],[1],[2],[3]). Answer = 12 − 5 = 7.

4 The Code (Annotated) ▾

Solution: AtMost(K) − AtMost(K−1) — O(n) time, O(k) space — Interview Pick

def subarraysWithKDistinct(nums, k):
def atMost(k): ← helper: count subarrays with ≤ k distinct
count = {} ← frequency map of elements in current window
left = 0 ← left boundary of sliding window
result = 0 ← accumulator for valid subarray count
for right in range(len(nums)): ← expand window rightward one step at a time
count[nums[right]] = count.get(nums[right], 0) + 1 ← add right element to window
while len(count) > k: ← too many distinct → must shrink
count[nums[left]] -= 1 ← remove left element from frequency map
if count[nums[left]] == 0: ← if frequency hits zero
del count[nums[left]] ← remove key so len(count) drops by 1
left += 1 ← advance left pointer past removed element
result += right - left + 1 ← count all valid subarrays ending at right
return result
return atMost(k) - atMost(k - 1) ← exactly K = at most K − at most (K−1)
Why not directly track exactly k distinct?

When we add an element that pushes us to k+1 distinct, we advance left until we drop back to k distinct — but we might drop to fewer than k. The resulting window doesn't give a clean subarray count for "exactly k". We'd need to track two left boundaries (one for >k and one for >k−1) simultaneously — which is exactly what the subtraction trick does, but more elegantly via two separate helper calls.

Brute Force O(n²) — for comparison only:

def subarraysWithKDistinct_brute(nums, k):
count, n = 0, len(nums)
for i in range(n): ← every possible starting index
seen = set()
for j in range(i, n): ← extend subarray to j
seen.add(nums[j])
if len(seen) == k: count += 1 ← exactly k distinct: count it
if len(seen) > k: break ← can only grow; no point continuing
return count
5 Interview Strategy ▾
Step 1 — Recognize the Pattern

"Exactly K distinct" in a sliding window context almost always maps to atMost(K) − atMost(K−1). When you see "count subarrays with exactly K [property]", jump to this decomposition immediately.

Step 2 — Prove the Helper

The atMost(k) helper is a textbook sliding window: expand right, shrink left when too many distinct. The left boundary after shrinking is always uniquely the smallest valid index — clean invariant, easy to prove correct.

Step 3 — Count Subarrays Per Right

right − left + 1 is a key interview technique: at each valid right, this counts all subarrays ending at that position whose left boundary is anywhere in [left..right]. Summing over all right gives the total — no overcounting.

Step 4 — Combine and State Complexity

Two calls to atMost, each O(n) time and O(k) space. Total: O(n) time, O(k) space. Mention that each element enters and leaves the window at most once per call, giving the O(n) amortized guarantee.

What to Say to the Interviewer

"I notice we need exactly K distinct integers. Directly tracking K with
a sliding window is tricky — the window's left boundary isn't uniquely
determined when we have exactly K distinct after shrinking.

The standard trick: exactly(K) = atMost(K) - atMost(K-1).
I'll write an atMost helper: expand right, add to freq map,
shrink left while distinct count exceeds k, then add right-left+1.
That's O(n) per call, O(k) space for the map.

Call it twice with k and k-1, subtract. Total O(n) time, O(k) space."
Likely Follow-up Questions
  • "Can you do it in one pass?" — Yes: maintain two left pointers simultaneously (lo1 for atMost(k), lo2 for atMost(k−1)), one right pointer. Add (right − lo1 + 1) − (right − lo2 + 1) per step. Same O(n), one loop.
  • "What if you need the actual subarrays, not just the count?" — Store (left, right) pairs when distinct == k. Output can be O(n²) in the worst case.
  • "What about exactly K occurrences rather than K distinct?" — Different technique; prefix sum counting or two-pointer on a transformed array.
