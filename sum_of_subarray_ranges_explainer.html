Sum of Subarray Ranges

For every contiguous subarray, compute max − min. Return the sum of all those ranges.

Medium Monotonic Stack Sum of Max − Sum of Min Time: O(n) Space: O(n)
1 Understanding the Problem ▾

You're given an integer array nums. For every contiguous subarray, its range is defined as its maximum element minus its minimum element. Return the sum of all these ranges.

With nums = [1, 2, 3], the 6 subarrays and their ranges are:

SubarrayMinMaxRange (max − min)
[1]110
[2]220
[3]330
[1, 2]121
[2, 3]231
[1, 2, 3]132

Total: 0 + 0 + 0 + 1 + 1 + 2 = 4.

💡 The Critical Decomposition

∑(max − min) across all subarrays = (∑ of all subarray maxima) − (∑ of all subarray minima). This linearity means we can compute both sums independently in O(n) and subtract — instead of computing n(n+1)/2 ranges individually.

🔑 Contribution Thinking

For each element nums[i], ask: "In how many subarrays is this element the minimum?" Multiply that count by nums[i] to get its total contribution to ∑mins. Repeat for maxima. A monotonic stack answers the boundary question in O(1) amortized per element.

Edge cases: Single-element array (all ranges = 0, answer = 0). All equal values (all ranges = 0). Negative numbers: the math works unchanged since we compare values.

2 The Algorithm: Monotonic Stack — Contribution Counting ▾

We use the identity answer = sumMax − sumMin. For each sum, we count every element's contribution using a monotonic stack that finds "previous/next smaller" or "previous/next larger" boundaries.

📐 Contribution Formula for Element i
left_span[i] = i − (index of previous element strictly < nums[i], or −1 if none)
right_span[i] = (index of next element ≤ nums[i], or n if none) − i

nums[i] is the minimum of exactly: left_span[i] × right_span[i] subarrays
→ Contribution to sumMin = nums[i] × left_span[i] × right_span[i]

Flip the inequalities to compute sumMax contributions.
1 Compute sumMin — monotonic increasing stack: Maintain a stack of indices in increasing value order. When element i is smaller than the stack top, pop: the popped element m now has its right boundary = i, left boundary = new top. Add nums[m] × left × right to sumMin.
2 Flush remaining elements with a sentinel (i = n): After the loop, any index still in the stack has its right boundary = n (past the end). A virtual sentinel iteration processes them all.
3 Compute sumMax — monotonic decreasing stack: Same idea, but pop when the current element is greater than the stack top. Contribution formula is the same; just different pop conditions.
4 Return sumMax − sumMin. Mathematically identical to summing (max − min) for every subarray, achieved in linear time.
⚠️ The Duplicate Tie-Breaking Rule (Critical)

When two elements are equal, we must assign ownership consistently. For sumMin: pop when nums[top] >= nums[i]. For sumMax: pop when nums[top] <= nums[i]. This asymmetry ensures each subarray has exactly one "minimum owner" (the rightmost equal element) and one "maximum owner" (the leftmost equal element) — no double-counting.

3 Why It Works: The Invariant ▾
🔐 The Stack Invariant

A monotonic increasing stack stores indices such that their corresponding values are strictly increasing from bottom to top. When we process element i and pop index m, it guarantees: nums[i] is the first element to the right of m that is strictly less than nums[m]. The new stack top (after popping m) is the first element to the left of m that is less than or equal to nums[m]. These are exactly the right and left boundaries for the subarrays where nums[m] is the minimum.

Why contribution counting is correct: For nums[m] to be the minimum of a subarray [l, r], the subarray cannot contain any element smaller than nums[m]. So l must lie in the range (prevSmaller, m] and r must lie in [m, nextSmaller). The number of such (l, r) pairs is exactly left_span × right_span, and the contribution to sumMin is nums[m] times that count.

Why linearity holds: ∑(maxᵢ − minᵢ) = ∑maxᵢ − ∑minᵢ by the distributive property of addition. This is not an approximation — it's exact arithmetic. Each of the n(n+1)/2 subarrays contributes its max to sumMax and its min to sumMin exactly once. Their difference is the same as computing each range individually.

⚠️ Why O(n) Not O(n²)?

Each element is pushed onto the stack exactly once and popped at most once. So across all n iterations, the total number of push+pop operations is at most 2n — O(n) total. This amortized analysis is why contribution counting via monotonic stack achieves linear time despite processing n(n+1)/2 subarrays.

4 The Code (Annotated) ▾

Solution 1 — Brute Force O(n²)

def subArrayRanges(nums): ← n² approach, expand each subarray
n, ans = len(nums), 0
for i in range(n): ← i = left boundary of subarray
lo = hi = nums[i] ← single-element: min = max = nums[i]
for j in range(i, n): ← j = right boundary, expand rightward
lo = min(lo, nums[j]) ← rolling min as window grows
hi = max(hi, nums[j]) ← rolling max as window grows
ans += hi - lo ← add this subarray's range to total
return ans ← O(n²) time, O(1) space

Solution 2 — Monotonic Stack O(n) — Interview Pick ✓

def subArrayRanges(nums): ← Decompose: answer = sumMax − sumMin
n = len(nums)
def sum_of_mins(): ← Monotonic increasing stack
total, stk = 0, []
for i in range(n + 1): ← Extra iteration: i=n is sentinel to flush stack
while stk and (i == n or nums[stk[-1]] >= nums[i]):← Pop when top ≥ current (or flush)
m = stk.pop() ← m: the element being finalized
left = m - (stk[-1] if stk else -1) ← subarrays m can extend to the left
right = i - m ← subarrays m can extend to the right
total += nums[m] * left * right ← m is min in left×right subarrays
stk.append(i) ← push current index
return total
def sum_of_maxs(): ← Monotonic decreasing stack (mirror of above)
total, stk = 0, []
for i in range(n + 1):
while stk and (i == n or nums[stk[-1]] <= nums[i]):← Pop when top ≤ current
m = stk.pop()
left = m - (stk[-1] if stk else -1)
right = i - m
total += nums[m] * left * right ← m is max in left×right subarrays
stk.append(i)
return total
return sum_of_maxs() - sum_of_mins() ← linearity: ∑(max−min) = ∑max − ∑min
⚠️ Why the Pop Conditions Differ Between Min and Max?

For sumMin: pop when nums[top] >= nums[i]. This means equal elements are also popped, so the rightmost among equals is the one that survives longest. For sumMax: pop when nums[top] <= nums[i]. Equals are popped, so the leftmost among equals persists. This deliberate asymmetry ensures each subarray's min and max are each counted exactly once.

💡 Complexity Summary

Brute Force: Time O(n²), Space O(1) — correct but fails at n = 10⁵
Monotonic Stack: Time O(n), Space O(n) — each element pushed and popped at most once
Parent problem: LeetCode #907 (Sum of Subarray Minimums) is just the sum_of_mins() half of this solution.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: State the Brute Force

"Brute force: iterate every pair (i, j), track running min and max as j expands. Add range each step. O(n²) time, O(1) space. Correct, but too slow for n ≤ 10⁵."

Step 2: Apply the Linearity Trick

"Key observation: ∑(max − min) = ∑max − ∑min. I can compute these two sums independently. This separates the problem into two symmetric subproblems."

Step 3: Recognize the Contribution Pattern

"For ∑mins, instead of computing min(subarray) for every subarray, I ask: for each element nums[i], in how many subarrays is it the minimum? That's left_span × right_span. A monotonic increasing stack gives these boundaries in O(n) total."

Step 4: Address Duplicates Proactively

"If nums has duplicate values, I need tie-breaking to avoid double-counting. I use '≥' for the min stack (pop equals left), '≤' for the max stack (pop equals right). This assigns each subarray's min to its rightmost equal element, and max to its leftmost."

🗣️ What to Say to the Interviewer

"I'll observe that sum of ranges = sum of maxima − sum of minima across all subarrays.
For sum of minima: each element i contributes nums[i] × (# subarrays it's the minimum of).
A monotonic increasing stack finds: previous index with value < nums[i], next index with value <= nums[i].
Those give left and right spans → product = subarray count where i is the min.
Same logic (flipped comparisons) for sum of maxima.
Total: O(n) time, O(n) space. Handles negatives and duplicates correctly."

7 Summary & Key Takeaways ▾
∑(max − min) = ∑max − ∑min — linearity lets you split the problem into two independent subproblems
Each element contributes: value × left_span × right_span to the sum of mins (or maxes)
A monotonic stack finds left/right boundary spans in O(1) amortized — each element pushed and popped at most once
Use >= for min stack pops, <= for max stack pops — asymmetric tie-breaking prevents double-counting
A sentinel iteration (i = n) cleanly flushes all remaining stack elements without special-case code
