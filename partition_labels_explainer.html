LeetCode #763

Partition Labels

Partition a string into as many parts as possible so each letter appears in at most one part. Return a list of the sizes of these parts.

Medium Intervals Greedy Time: O(n) Space: O(1)
1 Understanding the Problem ▾

You are given a string like "ababcbacadefegde". You need to split it into the maximum number of parts such that no letter appears in more than one part. Return the size of each part.

💡 Key Insight

A character "stakes out territory." If the letter 'a' appears at indices 0, 2, 6, and 8, then any partition containing 'a' must reach at least index 8 — otherwise some occurrence of 'a' would escape into a different partition, violating the rule. So the last occurrence of a character is its minimum right boundary.

✗ Invalid: naive split

s = "ab | abcbacadefegde"
Split at index 2? NO!
'a' appears in both parts.
Rule violated.

✓ Valid: extend to cover all

last['a']=8, last['b']=5, last['c']=7
Partition must reach index 8.
Part 1 = "ababcbaca" (size 9).
Then start fresh at index 9.

Key variables: last[c] = rightmost index where character c appears. start = left edge of current partition. end = the furthest right we must reach (max of all last[c] for chars seen so far).

Edge cases: All unique chars like "abcde" → 5 parts of size 1 each (no char repeats). All same chars like "aaaa" → 1 part of size n. Single char → size 1.

2 The Algorithm: Last Occurrence + Greedy Extend ▾

This is a two-pass greedy. Pass 1 precomputes where each character last appears. Pass 2 walks left-to-right, greedily extending the partition window to cover every character seen, and cuts the earliest moment it is safe to do so.

1 Precompute last occurrences: One pass through s. For each (i, c), set last[c] = i. Overwriting means the final value is the rightmost index. Only 26 possible keys for lowercase English.
2 Initialize window: start = 0 (partition left edge), end = 0 (must-reach boundary). result = [] to collect sizes.
3 Walk and extend: At each index i with character c, do end = max(end, last[c]). This "rubber-bands" the partition right to include every future occurrence of every character seen.
4 Cut at boundary: When i == end, the partition is closed — no character in [start..end] appears past end. Record end - start + 1, then start = i + 1.
5 Return result: The list of sizes. At most 26 partitions (one per distinct character in the worst case where all chars are unique).
🔑 The "Rubber Band" Mental Model

Imagine a rubber band around the current partition. As you scan each character, you snap it rightward to that character's last occurrence if it's further than the current right edge. When you physically arrive at the right edge of the rubber band, it's perfectly sealed — nothing inside can escape, nothing outside has crept in. Cut it off and start a new band.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any index i during the scan, end equals the maximum last-occurrence index of any character seen so far in the current partition. When i == end, every character from start to end has last[c] ≤ end — meaning none of them appear to the right of end. The partition is truly closed.

Why this maximizes the number of parts: We cut at the earliest possible valid position. Any later cut would only merge currently-separable partitions, which strictly reduces the count. This greedy earliest-cut is optimal: it never misses a valid split and never makes an invalid one.

Why no character leaks: When we close partition at index end, by construction every c in [start..end] satisfies last[c] ≤ end. So the character sets of consecutive partitions are disjoint — the problem's requirement is satisfied exactly.

⚠️ Common Mistake: Size Formula

Partition size is end - start + 1 (both endpoints inclusive), not end - start. And after recording, advance start = i + 1, not start = end + 1 (they happen to be equal, but i == end at cut time, so both work — but i + 1 is the clearer expression).

All-unique chars edge case: For "abcde", last['a']=0, last['b']=1, .... At each index i, end = max(end, i) = i. So i == end at every step → 5 partitions of size 1. Correct!

4 The Code (Annotated) ▾

Solution 1 — Last Occurrence + Greedy Extend (Optimal, Interview Pick)

def partitionLabels(s: str) -> list[int]:
last = {c: i for i, c in enumerate(s)} ← Overwriting gives rightmost index naturally
result, start, end = [], 0, 0 ← sizes list; start = partition left; end = must-reach boundary
for i, c in enumerate(s): ← Walk left to right, one character at a time
end = max(end, last[c]) ← Extend right boundary if c's last occurrence is further
if i == end: ← We've reached the must-reach point → safe to cut
result.append(end - start + 1) ← Inclusive size of this partition
start = i + 1 ← Next partition begins immediately after the cut
return result ← List of partition sizes
⚠️ Why dict comprehension overwrites correctly

{c: i for i, c in enumerate(s)} iterates left-to-right, and for repeated characters each new assignment overwrites the previous one. Since later indices are larger, the final value stored is always the last (rightmost) occurrence. Clean and idiomatic Python.

Solution 2 — Brute Force (O(n²), for reference only)

def partitionLabels_brute(s: str) -> list[int]:
result, start = [], 0
while start < len(s):
end = start
while end < len(s): ← Expand until stable
new_end = max(s.rfind(c) for c in s[start:end+1]) ← O(n) rfind per char → O(n²)
if new_end == end: break
end = new_end
result.append(end - start + 1)
start = end + 1
return result
💡 Complexity Summary

Optimal (last map + greedy): Time O(n) — two single-pass scans. Space O(1) — at most 26 keys for lowercase English.
Brute Force: Time O(n²) — rfind called inside nested expansion. Space O(1).
Interval Merge view: Same O(n) — build [first, last] per char, merge overlapping, return lengths. Conceptually equivalent.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Core Constraint

"Each letter must appear in exactly one part. That means for any character c, all of its occurrences must be in the same partition. The latest occurrence of c determines the minimum right edge of that partition." This is the key observation — say it out loud.

Step 2: Design the Two-Pass Solution

"I'll precompute last[c] for each character in one O(n) pass. Then I'll scan left-to-right, maintaining a window [start, end]. At each character c, I extend end = max(end, last[c]). When i reaches end, I cut and record the size."

Step 3: Justify the Greedy Choice

"Cutting at the earliest valid position is optimal because it maximizes future opportunities. The window [start, end] is the minimum possible partition — shrinking it would violate the constraint, growing it would only merge separable parts."

Step 4: Code and Trace

Write the 8-line solution, then trace "ababcbacadefegde": Part 1 ends at index 8 (last 'a'=8, last 'b'=5, last 'c'=7 → max=8). Size=9. Part 2 ends at 15 (last 'e'=15 controls). Size=7. Output: [9, 7].

🗣️ What to Say to the Interviewer

"Key observation: every character must live entirely within one partition,
so the partition must extend to the last occurrence of every character it contains.

Pass 1: build last[c] = rightmost index of c. O(n), O(1) space.
Pass 2: scan left-to-right. Track start (partition left) and end (must-reach boundary).
At each char c: end = max(end, last[c]).
When i == end: we're at the boundary. Cut! Record end - start + 1. Set start = i + 1.
Total: O(n) time, O(1) space."
🔮 Likely Follow-up Questions
  • "Return the actual substrings?" → Return s[start:end+1] instead of end-start+1.
  • "Works with uppercase / arbitrary chars?" → Yes, dict handles all; space is O(k) for k distinct chars.
  • "How does this relate to Merge Intervals?" → Each char defines interval [first_occ, last_occ]. Merging overlapping intervals and returning lengths gives the same answer.
  • "What's the maximum number of partitions?" → 26 for lowercase English (all unique chars). n in general.

7 Summary & Key Takeaways ▾
Precompute last[c] in one pass — iterating left-to-right and overwriting naturally yields the rightmost index.
Walk the string maintaining [start, end]. Extend end = max(end, last[c]) to "rubber-band" the window.
Cut when i == end — earliest valid split, guaranteeing maximum number of partitions.
Partition size = end - start + 1 (inclusive both ends). Advance start = i + 1.
O(1) space: at most 26 entries in last[] for lowercase English — effectively constant.
