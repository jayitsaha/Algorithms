LeetCode #172

Factorial Trailing Zeroes

Given an integer n, return the number of trailing zeroes in n! (n factorial).

Medium Math Count Factors of 5 Time: O(log n) Space: O(1)
1 Understanding the Problem ▾

A trailing zero in a number is any zero that appears at the right end — after all non-zero digits. For example, 100 has 2 trailing zeroes, and 10200 has 2 trailing zeroes (the 00 at the end, not the 0 between 2 and the other digits).

The question asks: if I compute n! = 1 × 2 × 3 × … × n, how many trailing zeroes does the result have? We can't just compute n! because it grows astronomically fast — 100! has 158 digits. We need a smarter approach.

💡 Key Insight

A trailing zero is produced by a factor of 10 = 2 × 5 in the product. Since 2s are far more plentiful than 5s in any factorial, the number of trailing zeroes equals the number of times 5 divides n! — i.e., how many factors of 5 exist across all numbers from 1 to n.

✓ Concrete: n = 10

10! = 3,628,800
Trailing zeroes: 2
Factors of 5: 5 (×1), 10 (×1) = 2
Factors of 2: 2,4,6,8,10 → 8 total
min(8, 2) = 2 ✓

✓ Tricky: n = 25

Multiples of 5: 5,10,15,20,25 = 5 fives
But 25 = 5² gives an extra factor!
Total 5s: 5 + 1 = 6
25! ends in 6 trailing zeroes

trailing_zeroes(n) = ⌊n/5⌋ + ⌊n/25⌋ + ⌊n/125⌋ + ⌊n/625⌋ + …
Sum continues until 5^k > n — typically O(log₅ n) terms

Edge Cases: n = 0 → 0! = 1, no trailing zeroes. n = 4 → 24, no trailing zero. Any n < 5 → 0 trailing zeroes. Large n (e.g., 10^9) still runs in O(log n) ≈ 13 iterations.

2 The Algorithm: Iterative Floor Division ▾

We need to count total factors of 5 in all numbers 1 through n. Numbers that are multiples of 5 contribute one factor. Numbers that are multiples of 25 contribute an additional factor. Multiples of 125 contribute yet another. And so on.

1 Start with count = 0 and the original value of n.
2 Floor-divide n by 5: n //= 5. This counts how many numbers in [1..n] are multiples of 5 (each contributing at least one factor of 5).
3 Add n to count. After the division, n now represents multiples-of-5 count. Accumulate it.
4 Repeat: divide by 5 again. Now n represents multiples of 25 (which have a second factor of 5). Add to count. Continue until n < 5 (no more factors).
5 Return count. The accumulated sum is the total number of factors of 5 — equal to the trailing zeroes.
🔑 Why This Trick Works

⌊n/5⌋ counts numbers with ≥1 factor of 5. ⌊n/25⌋ counts numbers with ≥2 factors (i.e., multiples of 5²). By computing n //= 5 iteratively and summing, we count each "layer" of factors exactly once. After dividing by 5 once, the new n is exactly ⌊original_n / 5⌋, so adding it gives multiples-of-5 count. After dividing again, it's ⌊n/25⌋, etc.

⚠️ Why Don't We Count Factors of 2?

Every even number contributes a factor of 2. Since half of all integers are even, and only 1 in 5 contribute a factor of 5, there are always at least 2× more factors of 2 than 5 in any factorial. The limiting reagent is always the factor of 5 — so we only need to count 5s.

3 Why It Works: The Invariant ▾
🔐 The Invariant (Legendre's Formula)

The total exponent of prime p in n! is given by Legendre's Formula: the sum of ⌊n/p^k⌋ for k = 1, 2, 3, … until p^k > n. For trailing zeroes, p = 5 (since p = 2 is never the bottleneck).

Why the iterative approach is equivalent: Instead of computing ⌊n/5⌋ + ⌊n/25⌋ + ⌊n/125⌋ + … directly, we do:

n = 100÷ 5 →20← counts multiples of 5
n = 20÷ 5 →4← counts multiples of 25
n = 4< 5 → stop
Total = 20 + 4 = 24← 100! has 24 trailing zeroes

This works because: when we do n //= 5 once and get 20, we're counting all numbers in [1..100] divisible by 5¹ = 40 such numbers, but we only add 20 because we've already "used up" one factor from each of the 20 multiples-of-25 in the next iteration. Actually simpler: sum = ⌊100/5⌋ + ⌊100/25⌋ = 20 + 4 = 24. The iterative trick computes exactly this sum without needing to compute 5^k for each step.

💡 Worked Proof: n = 25

Numbers 1..25 with factor of 5: 5, 10, 15, 20, 25 → that's 5 numbers.
But 25 = 5 × 5, so it contributes TWO factors of 5, not one.
Total 5-factors = 5 (one each from 5,10,15,20,25) + 1 (extra from 25) = 6.
Algorithm: n=25 → 25÷5=5, count=5 → 5÷5=1, count=6 → 1<5 stop. Returns 6 ✓

4 The Code (Annotated) ▾

Solution 1 — Iterative Floor Division (Optimal, O(log n)) — Interview Pick

def trailingZeroes(n: int) -> int:
count = 0 ← accumulate total factors of 5
while n >= 5: ← stop when no more multiples of 5 exist
n //= 5 ← collapse: ⌊n/5⌋ counts multiples of this power
count += n ← add count of numbers with ≥(iteration) factors of 5
return count ← total 5-factors = total trailing zeroes
⚠️ Common Mistake: Using n = original value each iteration

Beginners sometimes write count += n // 5; count += n // 25; count += n // 125 etc. That's equivalent but requires generating the powers 5, 25, 125... The iterative n //= 5; count += n pattern is cleaner: after dividing once, n already is the floor of original/5, so adding it is correct for the second-layer count.

Solution 2 — Recursive (same complexity, elegant)

def trailingZeroes(n: int) -> int:
if n < 5: ← base case: no multiples of 5
return 0
return n // 5 + trailingZeroes(n // 5) ← count this layer + recurse for higher powers

Solution 3 — Naive Brute Force (TLE for large n)

def trailingZeroes_naive(n: int) -> int: ← O(n) or worse; NEVER do this
import math
factorial = math.factorial(n) ← computing n! is already O(n log n) with big integers
count = 0
while factorial % 10 == 0: ← count trailing zeroes manually
count += 1
factorial //= 10 ← FAILS: 1000! has 249 zeros, int ops are huge
return count
💡 Complexity Summary

Optimal (iterative/recursive): Time O(log₅ n), Space O(1) (or O(log n) stack for recursive)
Naive factorial: Time O(n log n) or worse (big integer arithmetic), Space O(n) — impractical for n > 10,000
Key: We never actually need to compute n! — just count its prime factor 5.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Reframe — Where Do Trailing Zeroes Come From?

"A trailing zero comes from 10 = 2 × 5 in the prime factorization of n!. I need to count how many (2, 5) pairs exist in the product. Since 2s appear much more frequently than 5s, the bottleneck is the number of 5s."

Step 2: Recognize the Counting Problem

"How many times does 5 divide into n!? Every multiple of 5 contributes one factor. But 25 = 5² contributes two factors, 125 = 5³ contributes three, etc. This is exactly Legendre's Formula."

Step 3: Derive the Formula Naturally

"Among 1..n, there are ⌊n/5⌋ multiples of 5, ⌊n/25⌋ multiples of 25, ⌊n/125⌋ multiples of 125... So the answer is the sum of these floor divisions. I can compute this iteratively: divide n by 5 repeatedly, adding each quotient."

Step 4: State Complexity

"The loop runs O(log₅ n) times — about 13 iterations for n = 10⁹. Time is O(log n), Space is O(1). Very efficient."

🗣️ What to Say to the Interviewer

"A trailing zero requires a factor of 10 = 2 × 5. In n!, 2s are plentiful, so trailing zeros = number of 5-factors in n!. Among 1 to n: ⌊n/5⌋ multiples of 5 (each giving one factor), plus ⌊n/25⌋ extra for multiples of 25, plus ⌊n/125⌋ for multiples of 125, and so on. So I'll loop: while n >= 5, divide n by 5 and add to count. O(log n), O(1) space."
🔮 Likely Follow-up Questions
  • "What if we want leading zeroes?" → Leading zeroes don't make sense (leading zeros aren't stored in integers)
  • "Can you generalize to base b instead of base 10?" → Factorize b, find its prime factors, sum Legendre's formula for each, take the minimum
  • "Preimage Size of Factorial Zeroes Function (LC 793)?" → Binary search on the answer using trailingZeroes as a monotonic function
  • "Count trailing zeroes in n! in base 12?" → 12 = 2² × 3, count min(⌊factors_of_2/2⌋, ⌊factors_of_3/1⌋)

7 Summary & Key Takeaways ▾
Trailing zeroes come from factors of 10 = 2 × 5. Count factors of 5 (never 2, always the bottleneck).
Use Legendre's Formula: sum ⌊n/5^k⌋ for k=1,2,3,… until 5^k > n
Iterative trick: repeatedly do n //= 5 and add n to count — equivalent, clean, O(1) space
Never compute n! itself — it's astronomically large and unnecessary
Loop runs O(log₅ n) ≈ 13 times for n = 10⁹. Always O(log n) time, O(1) space
This is Legendre's Formula for the prime factor 5 — generalizable to any prime p

🎯 When to Recognize the "Count Factors" Pattern

  • "How many times does prime p divide n!?" → Legendre's Formula
  • Any problem involving factorial and a divisibility property → avoid computing the factorial
  • Binary search over trailingZeroes when the answer is a count rather than a specific n
