LeetCode #516

Longest Palindromic Subsequence

Given a string s, find the length of the longest subsequence of s that is a palindrome.

Medium Dynamic Programming Interval DP Time: O(n²) Space: O(n²)
1 Understanding the Problem ▾

A subsequence is formed by deleting some characters (or none) without changing the order of the remaining ones. A palindrome reads the same forwards and backwards. We want the longest subsequence of s that is also a palindrome.

Crucially, this is NOT the same as finding the longest palindromic substring. A substring must be contiguous. A subsequence can skip characters.

✓ Example: s = "bbbab"

Subsequences that are palindromes:
"b" (len 1), "bb" (len 2), "bbb" (len 3)
Best: "bbbb" (idx 0,1,2,4) → len 4
"bbbb" ← skip the 'a'!

✗ Common mistake: Contiguous only

"bbb" (idx 0,1,2) is a contiguous palindrome
len 3, but we can do better by skipping 'a'
Subsequence allows gaps → "bbbb"
Answer = 4, not 3

💡 Key Insight: Two Powerful Reductions

Reduction 1 — LCS on reverse: LPS(s) = LCS(s, reverse(s)). Any palindromic subsequence of s must be a common subsequence of s and its reverse. This reduces to a well-known problem.

Reduction 2 — Interval DP: Define dp[i][j] = LPS of s[i..j]. Shrink the problem inward from both endpoints — check if they match, then recurse into the inner substring.

Edge Cases: Single character → 1 (always a palindrome). All same characters (e.g., "aaaa") → n (the whole string). Two distinct characters → 1. Empty string → 0.

2 The Algorithm: Interval DP (Bottom-Up Tabulation) ▾

We define the state dp[i][j] = length of the longest palindromic subsequence within the substring s[i..j]. We fill this table for substrings of increasing length.

📐 The Recurrence Relations
Base case (length = 1): dp[i][i] = 1 ← single char is always a palindrome

If s[i] == s[j]: dp[i][j] = dp[i+1][j-1] + 2
If s[i] != s[j]: dp[i][j] = max(dp[i+1][j], dp[i][j-1])

Answer: dp[0][n-1]

1 Initialize diagonal: dp[i][i] = 1 for all i. Single-character palindrome.
2 Outer loop over length: for length in range(2, n+1). We compute subproblems from smallest to largest, ensuring all dependencies are ready.
3 Inner loop over start index: for i in range(n - length + 1), with j = i + length - 1 as the right endpoint.
4 Match case: If s[i] == s[j], the two endpoints form the outermost layer of a palindrome — add 2 to the inner result: dp[i][j] = dp[i+1][j-1] + 2.
5 No match case: If s[i] != s[j], at least one endpoint cannot be in the final palindrome. Try excluding each endpoint: dp[i][j] = max(dp[i+1][j], dp[i][j-1]).

3 Why It Works: The Invariant & Why This is DP ▾
🔐 The Invariant

At all times, dp[i][j] holds the exact length of the longest palindromic subsequence within s[i..j]. When we compute dp[i][j] for length L, every smaller subproblem (all lengths < L) is already correctly computed — guaranteed by the outer loop order.

Why Optimal Substructure holds: Any palindromic subsequence of s[i..j] either includes both s[i] and s[j], or excludes at least one of them.

  • If s[i] == s[j]: we can always extend the LPS of the inner string s[i+1..j-1] by wrapping both endpoints. The gain is exactly +2.
  • If s[i] != s[j]: the two endpoints cannot both appear in any palindrome (a palindrome's first and last characters must match). We must exclude at least one. The optimal answer is the better of the two sub-problems.

Why Overlapping Subproblems arise: A naive recursion for lps(i, j) spawns calls to lps(i+1, j), lps(i, j-1), and lps(i+1, j-1). For a string of length n, the same sub-range (say, i=2, j=5) gets recomputed from many different parent calls — the recursion tree has exponential branches but only O(n²) distinct states. Memoization or tabulation collapses this to O(n²) work.

⚠️ Why Fill By Length, Not By Row?

When computing dp[i][j], we need dp[i+1][j-1] (one row down, one column left — the inner diagonal), dp[i+1][j] (one row down), and dp[i][j-1] (one column left). If we filled row-by-row, dp[i+1][j] for the same length would not yet be computed. Filling by length guarantees all substrings shorter than the current one are complete.

LCS Proof Sketch: Let P be a palindromic subsequence of s. Since P is a palindrome, P = reverse(P). The characters of P appear in s in order (by subsequence definition) AND in reverse(s) in order (since P = reverse(P)). Therefore P is a common subsequence of s and reverse(s). The longest such P is therefore the LCS(s, reverse(s)).

4 The Code (Annotated) ▾

Solution 1 — Interval DP Tabulation (Interview Pick) · O(n²) time · O(n²) space

def longestPalindromeSubseq(s: str) -> int:
n = len(s) ← Length of the input string
dp = [[0]*n for _ in range(n)] ← n×n table; dp[i][j] = LPS of s[i..j]
for i in range(n): ← Base case: all single-char substrings
dp[i][i] = 1 ← s[i..i] has LPS = 1 (itself)
for length in range(2, n + 1): ← Outer: grow substring length 2 → n
for i in range(n - length + 1): ← Inner: slide the left endpoint i
j = i + length - 1 ← Right endpoint from i and length
if s[i] == s[j]: ← Endpoints match → can extend!
dp[i][j] = dp[i+1][j-1] + 2 ← Inner LPS + 2 (both endpoints added)
else: ← Endpoints differ → skip one
dp[i][j] = max(dp[i+1][j], dp[i][j-1]) ← Best of: skip left OR skip right
return dp[0][n-1] ← LPS of the entire string
⚠️ Edge: dp[i+1][j-1] when length == 2

When the substring has length 2 and s[i] == s[j], we compute dp[i+1][j-1] = dp[i+1][i]. Since i+1 > i, this is an empty (invalid) range. It evaluates to 0 by initialization. So dp[i][j] = 0 + 2 = 2. Correct! Two matching characters form a length-2 palindrome.

Solution 2 — Top-Down Memoization (Recursive) · O(n²) time · O(n²) space

def longestPalindromeSubseq(s: str) -> int:
from functools import lru_cache ← Python's built-in memoization
n = len(s)
@lru_cache(maxsize=None) ← Cache calls by (i, j) — up to n² states
def dp(i, j): ← Returns LPS length for s[i..j]
if i > j: return 0 ← Empty range: 0 (handles length-2 base case)
if i == j: return 1 ← Single character: 1
if s[i] == s[j]: ← Match: extend inward
return dp(i+1, j-1) + 2 ← Recurse on inner + 2
return max(dp(i+1, j), dp(i, j-1)) ← No match: try both skips
return dp(0, n-1) ← Start with full string

Solution 3 — LCS Reduction · O(n²) time · O(n²) space

def longestPalindromeSubseq(s: str) -> int:
t = s[::-1] ← Reverse of s
n = len(s)
dp = [[0]*(n+1) for _ in range(n+1)] ← LCS table; extra row/col for empty string
for i in range(1, n+1): ← For each character of s
for j in range(1, n+1): ← For each character of t (= reverse s)
if s[i-1] == t[j-1]: ← Characters match
dp[i][j] = dp[i-1][j-1] + 1 ← Extend LCS by 1
else: ← No match
dp[i][j] = max(dp[i-1][j], dp[i][j-1]) ← Take best from either direction
return dp[n][n] ← LCS(s, reverse(s)) = LPS(s)
💡 Space Optimization: O(n²) → O(n)

In the tabulation approach, dp[i][j] depends only on dp[i+1][j-1] (diagonal), dp[i+1][j] (next row), and dp[i][j-1] (current row, previous column). With careful iteration, you only need to maintain the current and previous rows — O(n) space. Worth mentioning in interviews after the O(n²) solution is accepted.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the DP Signal

The problem asks for the longest subsequence satisfying a property. "Longest something" over a sequence → classic DP. The palindrome property adds a symmetry constraint that suggests comparing from both ends.

Step 2: Define the State

"If I shrink the problem, what are the natural boundaries?" For substring problems, the boundaries are (i, j). State: dp[i][j] = LPS in s[i..j]. The answer will be dp[0][n-1].

Step 3: Write the Recurrence

"What are my cases?" Any palindrome either uses both endpoints (they match) or skips at least one (they differ). This exhausts all possibilities — every palindromic subsequence obeys one of these two rules.

Step 4: Choose Implementation Style

Top-down (add @lru_cache to the recursive function) is easiest to write and explain. Bottom-up (fill by length) shows deeper understanding of dependencies. Both are O(n²).

Step 5: Drop the LCS Reduction Bonus

"I also know LPS(s) = LCS(s, reverse(s)) — a palindromic subsequence is exactly a string that appears in order in both s and its reverse. Same complexity, different table structure."

🗣️ What to Say to the Interviewer

"This is a classic interval DP problem. I'll define dp[i][j] as the length of the longest palindromic subsequence within s[i..j]. Base case: dp[i][i] = 1 (single chars are palindromes). Recurrence: If s[i] == s[j]: dp[i][j] = dp[i+1][j-1] + 2 (both endpoints extend the inner LPS) If s[i] != s[j]: dp[i][j] = max(dp[i+1][j], dp[i][j-1]) (skip one endpoint) I fill the table by increasing substring length to ensure all dependencies are ready. Time O(n²), space O(n²), reducible to O(n) with rolling arrays."
🔮 Likely Follow-up Questions
  • "Can you reduce space?" → Yes, O(n) with two rolling arrays. The key is that dp[i][j] only needs row i+1 and the previous column in the current row.
  • "Print the actual subsequence?" → Backtrack the table: at each (i,j) if match go diag with character, else follow the max direction.
  • "Minimum insertions to make palindrome?" → n − LPS(s). Remove everything not in LPS, then insert to mirror.
  • "Longest palindromic SUBSTRING (contiguous)?" → Different! Expand around center or Manacher's algorithm — O(n).

7 Summary & Key Takeaways ▾
LPS uses Interval DP: dp[i][j] = LPS of s[i..j], filled by increasing substring length
If s[i] == s[j]: both endpoints wrap the inner palindrome → dp[i][j] = dp[i+1][j-1] + 2
If s[i] != s[j]: cannot use both endpoints → dp[i][j] = max(dp[i+1][j], dp[i][j-1])
Fill order is by length (not row), so all dependencies are computed first
Alternative: LPS(s) = LCS(s, reverse(s)) — elegant reduction to a classic problem
Minimum insertions to make palindrome = n − LPS(s)
ApproachTimeSpaceNotes
Brute Force (all subsequences) O(2ⁿ) O(n) Try all 2ⁿ subsequences — infeasible for n>20
Memoization (top-down) O(n²) O(n²) Easy to write, recursive overhead, same result
Interval DP Tabulation ✓ O(n²) O(n²) → O(n) Optimal; space reducible with rolling arrays

🎯 When to Recognize Interval DP / Palindrome DP
  • "Longest/optimal something in a substring" + no contiguous constraint → Interval DP with dp[i][j]
  • Problem involves checking/building palindromes from characters at two ends → expand inward
  • "Minimum operations to make a string palindrome" → n − LPS
  • LCS on s and reverse(s) is a reliable equivalent formulation
