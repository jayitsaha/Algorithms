LeetCode #1047

Remove All Adjacent Duplicates In String

Given a string s, repeatedly remove adjacent duplicate characters until no two adjacent characters are equal. Return the final string.

Easy Stacks Stack Character Match Time: O(n) Space: O(n)
1 Understanding the Problem ▾

Imagine a row of colored tiles. Whenever two adjacent tiles match, they explode and disappear — and after the explosion, the tiles on either side collapse together, potentially creating a new matching pair. You keep exploding pairs until there are no adjacent matches left. What tiles remain?

💡 Key Insight

This is a cascade problem — removing one pair can expose a new pair that also needs removing. A naive repeated-scan approach is O(n²). The key insight: use a stack to track the "surviving" characters. When a new character matches the stack's top, they annihilate each other (pop). Otherwise, the character survives (push). The stack's final contents are the answer.

✗ Naive: Repeated Scan (O(n²))

"abbaca"
→ find "bb" → remove → "aaca"
→ find "aa" → remove → "ca"
Needs multiple passes!

✓ Stack: Single Pass O(n)

'a' → push → [a]
'b' → push → [a,b]
'b' → match! POP → [a]
'a' → match! POP → []
'c' → push → [c]
'a' → push → [c,a] → "ca"

Edge Cases: Empty string → return "". Single character → return it unchanged. All identical characters ("aaaa") → even-length returns "", odd-length returns one character. String with no adjacent duplicates → return unchanged. The result is always uniquely determined regardless of removal order.

2 The Algorithm: Stack Character Match ▾

We scan the input string exactly once, left to right. A stack acts as a running result buffer. At every character, we face a binary decision: does this character survive, or does it cancel its neighbor?

1 Initialize an empty stack — it will hold characters that have "survived" so far (no adjacent duplicate is waiting to cancel them).
2 Scan each character c left to right:
• If stack is not empty AND stack[-1] == c: adjacent duplicate found! Pop the top — both the top and c are removed.
• Otherwise: Push c — it survives for now (no match with its left neighbor).
3 Return "".join(stack) — the remaining stack characters have no adjacent duplicates anywhere.

Why does a single pass work? When we push a character onto the stack, its only potential duplicate neighbor is the top of the stack. That top already survived the same filtering. If the new character matches the top, they cancel immediately — we never need to re-scan because the stack invariant guarantees the prefix is already fully reduced.

🔑 Stack as a "Deduplication Buffer"

Think of the stack as the current result string being built character by character. At each step, the stack holds the maximally-reduced prefix with no adjacent duplicates. Adding a new character either extends this prefix safely (push) or cancels the last character of the prefix (pop). The final stack is the answer.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point during the scan, the stack contains the maximally-deduplicated version of all characters processed so far. Specifically: no two adjacent characters in the stack are equal. This property holds after every single character is processed.

Proof by induction:

  • Base case: Empty stack — no adjacent duplicates. Invariant holds trivially.
  • Inductive step: Assume stack has no adjacent duplicates. We process character c:
    • If stack[-1] == c: pop the top. The newly-exposed top (if any) differs from the character below it (invariant held before). No new adjacency created. ✓
    • If stack[-1] != c (or stack empty): push c. New top c differs from the previous top. No adjacent duplicate added. ✓

Why is this the correct final answer? The problem's final result is uniquely determined regardless of which pairs you remove first — it can be proven by showing the result is order-independent. So our greedy left-to-right approach always reaches the same unique answer as any other valid removal order.

⚠️ Common Mistake: Forgetting Empty Stack Check

Before checking stack[-1] == c, you must verify the stack is not empty. An empty stack access causes an IndexError in Python. Always guard: if stack and stack[-1] == c:. The and short-circuits — if stack is empty (falsy), the second condition is never evaluated.

Edge case walkthrough — "aaaa": Push 'a' → [a]. 'a' matches top → pop → []. Push 'a' → [a]. 'a' matches top → pop → []. Stack is empty → result is "". ✓

Edge case walkthrough — "ab": Push 'a' → [a]. 'b' ≠ 'a' → push → [a, b]. No more chars. Result = "ab". ✓ (no adjacent duplicates to remove)

4 The Code (Annotated) ▾

Solution 1 — Stack (Optimal, O(n) — Interview Pick)

def removeDuplicates(s: str) -> str:
stack = [] ← Python list as stack: append=push, pop()=pop, [-1]=top; O(1) each
for c in s: ← Single pass O(n): each char processed exactly once
if stack and stack[-1] == c: ← Guard empty first (short-circuit), then check top == current char
stack.pop() ← Match! Pop top. Current c is discarded (both chars annihilated)
else:
stack.append(c) ← No match: c survives. Push onto stack as new top.
return "".join(stack) ← Stack = deduplicated result. join() is O(n), not O(n²) like string +=
⚠️ Why "".join(stack) and not result += c?

Python strings are immutable. Using result += c inside a loop creates a brand-new string each iteration: O(1) + O(2) + ... + O(n) = O(n²) total. "".join(list) is a single O(n) operation. Always use join when building strings from a list.

Solution 2 — Brute Force: Repeated Replace (O(n²))

def removeDuplicates_brute(s: str) -> str:
changed = True
while changed: ← Repeat until no pair removed in a full pass
changed = False
i = 0
while i < len(s) - 1:
if s[i] == s[i+1]: ← Found adjacent duplicate
s = s[:i] + s[i+2:] ← O(n) string creation each removal
changed = True
else:
i += 1
return s ← Correct but O(n²) — up to n/2 passes of O(n) each
💡 Complexity Summary

Stack (Optimal): Time O(n) — one pass, each char pushed/popped at most once. Space O(n) — stack holds at most n chars.
Brute Force: Time O(n²) — up to n/2 passes of O(n). Space O(n).
In-place (write pointer): Time O(n), Space O(1) extra — treat the input array itself as the stack using a write index. Only feasible with mutable arrays (C++/Java); Python strings are immutable.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Cascade Signal

"Repeatedly remove adjacent duplicates" — the word repeatedly is the key. It means removing one pair can create another pair to remove. This cascading behavior is the hallmark of a stack problem. Any time you see "the result of one step affects what you check next from a previously-processed character," think stack.

Step 2: Propose Brute Force First

"Naive: scan repeatedly, each time removing any pair found. O(n²) worst case — imagine 'aabb...': each pass removes exactly one pair from the middle, requiring O(n) passes of O(n) each."

Step 3: Derive the Stack Optimization

"Better: for each character, the only neighbor I need to check is the immediately previous surviving character. A stack gives me exactly that — O(1) access to the last survivor. This turns O(n²) into O(n) with a single left-to-right pass."

Step 4: State the Invariant

"The stack always contains the maximally-deduplicated prefix of all characters processed so far — no two adjacent chars in the stack are equal. This invariant proves the approach is correct and that a single pass is sufficient."

🗣️ What to Say to the Interviewer

"The cascade tells me to use a stack.
I scan left to right. At each character c, I check the stack's top.
If the top equals c — adjacent duplicate — I pop (both chars removed).
Otherwise, I push c. The stack always holds the deduplicated prefix.
At the end, join the stack into a string. Time O(n), Space O(n).
If asked for O(1) extra space: use the array itself as the stack with a write index."
🔮 Likely Follow-up Questions
  • "Remove k adjacent duplicates?" → Stack of (char, count) pairs; pop when count reaches k. (#1209, Medium)
  • "O(1) extra space?" → Treat input char array as the stack with a write pointer (C++/Java).
  • "What if you can only remove one pair total?" → Simpler: find first adjacent pair and remove it.
  • "Verify the result has no adjacent duplicates?" → Same stack approach on the result, or just check linearly — our invariant already guarantees it.
