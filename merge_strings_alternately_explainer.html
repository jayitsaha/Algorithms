Merge Strings Alternately

LeetCode #1768 — Two Pointers: Same Direction Interleave
Easy Two Pointers Same Direction Interleave O(m+n) Time · O(m+n) Space
1
Understanding the Problem
▼

Given two strings word1 and word2, create a new string by taking one character from each in turn, starting with word1. When one string runs out, append all remaining characters of the other.

💡 Key Insight

Think of a zipper: teeth from the left side and right side interlock one at a time. When one side runs out of teeth, the remaining ones from the other side just extend past the end.

Example: word1 = "abc", word2 = "pqrs"

Step 1: Take 'a' (word1) → "a"
Step 2: Take 'p' (word2) → "ap"
Step 3: Take 'b' (word1) → "apb"
Step 4: Take 'q' (word2) → "apbq"
Step 5: Take 'c' (word1) → "apbqc"
Step 6: word1 done, take 'r' (word2) → "apbqcr"
Step 7: Take 's' (word2) → "apbqcrs"

Edge cases: Empty word1 → return word2. Equal length → no tail. word1 longer → word1's tail appended.

2
The Algorithm: Same-Direction Two Pointers
▼

Use two indices i (word1) and j (word2), both starting at 0, advancing simultaneously.

1Initialise i = 0, j = 0, result = []
2While i < len(word1) OR j < len(word2):
3   If word1 not exhausted: append word1[i], i += 1
4   If word2 not exhausted: append word2[j], j += 1
5Return ''.join(result)
🔑 Why One Loop Handles Everything

The while ... OR ... condition keeps the loop alive until both strings are exhausted. Inside, the two if guards prevent index-out-of-bounds. When word1 is done, only the word2 guard fires — naturally appending the tail without any special-case code.

3
Why It Works: The Invariant
▼
🔐 The Invariant

After each loop iteration, result contains exactly i + j characters, and no character from either string has been skipped or repeated. The relative order within each source string is preserved.

When word1 is exhausted (i ≥ m), the if i < m guard silently skips — word2 continues appending at its natural pace. Neither pointer ever moves "backwards" or "past the end". Each pointer visits its string once: O(m + n) total operations.

✅ Correctness

Every character of both strings appears in the output exactly once, in the order specified by the problem. The loop terminates because both i and j are strictly increasing (at least one advances per iteration).

4
The Code (Annotated)
▼
def mergeAlternately(word1: str, word2: str) -> str: i, j = 0, 0 # one pointer per string result = [] # list is O(1) append; join at end avoids O(n²) concat while i < len(word1) or j < len(word2): # keep going until BOTH exhausted if i < len(word1): # guard: word1 has chars left result.append(word1[i]) # take one char from word1 i += 1 # advance word1 pointer if j < len(word2): # guard: word2 has chars left result.append(word2[j]) # take one char from word2 j += 1 # advance word2 pointer return ''.join(result) # O(m+n) final join
⚠️ Common Mistake: AND vs OR

Using while i < m and j < n exits as soon as the shorter string ends. You then need separate result += word1[i:] and result += word2[j:] lines to append the tail. The OR version handles the tail automatically — simpler and harder to get wrong.

🎤 Interview Strategy
▼
1Reframe aloud: "Zip them together; leftovers from the longer string are appended."
2Brute force mention: Python's zip + chain works but is less explicit about the pointer logic.
3Edge cases to state: one empty string, equal lengths, length-1 strings.
4Complexity: O(m+n) time, O(m+n) space (output). Cannot do better — must read every character.
5Mention string building: list + join is preferred over repeated string concat in Python (avoids O(n²) copies).
