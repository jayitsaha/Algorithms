LeetCode #233

Number of Digit One

Given an integer n, count the total number of digit 1 appearing in all non-negative integers less than or equal to n.

Hard Dynamic Programming Digit DP Time: O(log n) Space: O(1)
1 Understanding the Problem ▾

We need to count how many times the digit 1 appears across all integers from 0 to n. This counts every individual 1 digit in every number — not just how many numbers contain a 1.

✓ Example: n = 13 → Answer = 6

1  → one 1
10 → one 1
11 → two 1s ← counted twice!
12 → one 1
13 → one 1
Total = 1+1+2+1+1 = 6

✗ Common Mistake: Counting numbers with 1

Numbers containing 1: {1,10,11,12,13}
That is 5 numbers...
But 11 has TWO 1-digits!
Wrong answer: 5
Correct answer: 6

💡 Key Insight

n can be up to 2×10⁹ — iterating all numbers is impossible. The breakthrough: count 1s position by position. For each digit position (ones, tens, hundreds, …), independently count how many times 1 appears at that exact column across all numbers from 0 to n. Sum over all positions. This runs in O(log n) time — just one pass per digit position.

Edge Cases: n = 0 → 0. n = 1 → 1. n = 10 → 2 (one from "1", one from "10"). n = 11 → 4 (1, 10, 11, 11). For large n ≈ 10⁹, output can be ~8.6×10⁸ — use int64 in C++/Java.

2 The Algorithm: Count 1s at Each Position ▾

For each digit position, we split n into three parts: the higher digits (digits left of current position), the current digit, and the lower digits (digits right of current position).

📐 The Core Formula (for each position pos = 1, 10, 100, …)
higher = n // (pos * 10) # digits to the LEFT of pos current = (n // pos) % 10 # the digit AT pos lower = n % pos # digits to the RIGHT of pos if current == 0: contribution = higher * pos if current == 1: contribution = higher * pos + lower + 1 if current >= 2: contribution = (higher + 1) * pos
1 Initialize: count = 0, pos = 1 (start at ones place).
2 Loop while pos ≤ n: Extract higher, current, lower from n at position pos.
3 Apply three-case formula: digit = 0 (full cycles only), digit = 1 (full cycles + partial), digit ≥ 2 (one extra full cycle).
4 Advance: pos *= 10. Repeat. The loop runs exactly ⌊log₁₀(n)⌋ + 1 times.
🔑 Why Three Cases?

Think of numbers grouped into blocks of pos × 10. In each complete block, the digit at position pos cycles through 0–9, so the digit 1 appears exactly pos times. The partial block at the end depends on what current is: if it's 0, we haven't reached the 1-cycle yet; if it's 1, we're in the middle of it; if it's ≥ 2, we've completely passed through the 1-cycle.

3 Why It Works: The Invariant ▾
🔐 The Invariant

For position pos, the 1s contributed at that column by numbers 0 through n equals the number of times the digit 1 appears in the pos-column when you write out 0, 1, 2, …, n vertically. The three-case formula computes this count exactly by reasoning about complete and partial cycles of size pos × 10.

Worked example: n = 3141, pos = 100 (hundreds column)

  • higher = 3, current = 1, lower = 41
  • Complete cycles (higher = 3): ranges 0–999, 1000–1999, 2000–2999. In each block of 1000, the hundreds digit = 1 for exactly 100 numbers (x100–x199). So 3 × 100 = 300 ones from complete cycles.
  • Partial cycle (current = 1): We're in 3000–3141. The hundreds digit is 1 for 3100, 3101, …, 3141 — that's 41 + 1 = 42 numbers. Formula: lower + 1 = 41 + 1 = 42.
  • Total at hundreds place: 300 + 42 = 342 ones.
⚠️ Walking Through All Three Cases

current = 0 (e.g., n = 3041, hundreds = 0): The partial cycle hasn't reached the 1-century (3100+). Only 3 complete cycles contribute: 3 × 100 = 300.

current = 1 (e.g., n = 3141, hundreds = 1): The partial cycle started (3100) but hasn't ended (3199). We've seen lower + 1 = 42 instances of hundreds = 1.

current ≥ 2 (e.g., n = 3241, hundreds = 2): The partial cycle (3100–3199) is fully done. 4 complete cycles: (3 + 1) × 100 = 400.

Independence of positions: A digit 1 at the tens place of 314 is a different occurrence from the 1 at the hundreds place of 3141. Counting independently per position and summing covers every 1-digit exactly once without double-counting.

4 The Code (Annotated) ▾

Solution 1 — Position-by-Position Count (Optimal — Interview Pick)

def countDigitOne(n: int) -> int: ← count all 1-digits from 0 to n
count = 0 ← accumulator: total 1s found
pos = 1 ← current digit position: 1, 10, 100, ...
while pos <= n: ← process each existing digit position
higher = n // (pos * 10) ← digits strictly LEFT of this position
current = (n // pos) % 10 ← the digit AT this position
lower = n % pos ← digits strictly RIGHT of this position
if current == 0: ← digit is 0: only full cycles contribute
count += higher * pos
elif current == 1: ← digit is 1: partial cycle in progress
count += higher * pos + lower + 1
else: ← digit ≥ 2: partial cycle fully passed 1
count += (higher + 1) * pos
pos *= 10 ← advance to next digit position
return count ← sum of all 1s across all positions
⚠️ Why does the loop condition use pos <= n?

When pos exceeds n, the digit at that position in n is 0 and higher = 0, so the contribution is 0. We stop early because: (n // pos) % 10 = 0 when pos > n, meaning no digits exist at that position. The condition is tight — for n = 13, we check pos = 1 (ones) and pos = 10 (tens), but not pos = 100 since 100 > 13.

Solution 2 — Brute Force (O(n log n) — Educational Only, TLE on large n)

def countDigitOne(n: int) -> int:
return sum(str(i).count('1') for i in range(n + 1)) ← O(n log n), TLE for n ≥ 10⁶
💡 Unified One-Liner (Advanced)

The three cases collapse into one expression, useful once you internalize the logic:

count += (higher + (1 if current > 1 else 0)) * pos + (lower + 1 if current == 1 else 0)

Use the three-branch version in interviews — it's far more readable and debuggable.

💡 Complexity Summary

Position Count (Optimal): Time O(log₁₀ n) ≈ O(10) for n ≤ 10⁹, Space O(1)
Brute Force: Time O(n log n), Space O(log n) per string. TLE for n > 10⁵.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize That O(n) is Impossible

n ≤ 2×10⁹. Any per-number loop is dead on arrival. Immediately say: "With n up to 2 billion, I need a mathematical or digit-based approach, definitely O(log n)." This signals you understand constraints before diving into code.

Step 2: Propose Brute Force, Then Discard It

"Naively: iterate 0 to n, convert each to string, count '1's. O(n log n). Way too slow. Let me think structurally about where 1s appear."

Step 3: The Position Decomposition Insight

"Instead of iterating numbers, fix a digit position and count how many numbers from 0 to n have a 1 in that column. The columns are independent, so I can sum them up."

Step 4: Derive the Three Cases at the Board

Work through n = 13, ones place: higher=1, current=3 ≥ 2 → (1+1)×1 = 2. Then tens place: higher=0, current=1 → 0+3+1 = 4. Total = 6. Verify with brute force mentally. Then generalize the three cases.

🗣️ What to Say to the Interviewer

"I'll iterate over each digit position — ones, tens, hundreds, and so on.
For each position pos, I split n into: higher (left digits), current (the digit at pos), lower (right digits).
The contribution of 1s at this position follows three cases:
current = 0: only full cycles — higher × pos ones.
current = 1: full cycles plus partial — higher × pos + lower + 1.
current ≥ 2: full cycles plus one complete extra cycle — (higher+1) × pos.
Sum over all positions. Time O(log n), space O(1)."

  • "Count 1s from a to b?" → f(b) − f(a−1) where f(n) is this function.

7 Summary & Key Takeaways ▾
Think position-by-position, not number-by-number — O(log n) not O(n)
Split n into higher / current / lower at each digit position
Three cases: current=0 (higher×pos), current=1 (higher×pos+lower+1), current≥2 ((higher+1)×pos)
Positions are independent — contributions don't overlap, sum them directly
Digit DP signal: "count integers in [0,n] satisfying a per-digit property" with large n
