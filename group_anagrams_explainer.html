Group Anagrams

LeetCode 49 — Group strings that are anagrams of each other
Medium Hash Tables Sorted String as Key O(n·k log k) O(n·k) Space
1 Understanding the Problem ▼

Given an array of strings, group all anagrams together. An anagram is a word formed by rearranging the letters of another word — same characters, potentially different order.

Walk through the example:

Input: ["eat", "tea", "tan", "ate", "nat", "bat"]

"eat", "tea", "ate" → all have letters {a, e, t} → Group 1
"tan", "nat" → all have letters {a, n, t} → Group 2
"bat" → letters {a, b, t} → Group 3 (alone)

Output: [["eat","tea","ate"], ["tan","nat"], ["bat"]]
💡 Key Insight

Two strings are anagrams if and only if they have the same characters with the same frequencies. We need a way to "fingerprint" each string so that anagrams produce the same fingerprint.

The core question: What "fingerprint" (key) should we use to identify anagrams?

  • Option 1: Sort the characters → "eat" → "aet", "tea" → "aet" ✓
  • Option 2: Character count tuple → "eat" → (1,0,0,0,1,...,1,...) for a,e,t ✓
2 The Algorithm: Sort as Key ▼

The simplest approach: sort each string and use the sorted string as a hash key. All anagrams will produce the same sorted string.

1Create an empty hash map: groups = {}
2For each word in the input array:
3 Compute the key: key = sorted(word) → "tea" becomes "aet"
4 Append word to groups[key]
5Return all values from the hash map

⚠️ Why We Need a Hash Map

Without a hash map, we'd have to compare every pair of strings — O(n²) comparisons. The hash map lets us group in O(n) passes.

3 Why It Works: The Invariant ▼
🔐 The Invariant

Two strings are anagrams ⟺ They have identical sorted forms.

Sorting is a canonical representation — it maps all anagrams to the same unique string, making it a perfect hash key.

Proof Sketch

If two strings are anagrams:

  • They contain exactly the same characters
  • Sorting arranges characters by ASCII value
  • Same characters → same sorted result

Conversely, if two strings have the same sorted form, they must be anagrams (same character multiset).

Edge Cases

  • Empty string: sorted("") = "" — groups with other empty strings
  • Single character: sorted("a") = "a" — correct grouping
  • All same word: All map to same key → one big group
  • No anagrams: Each word gets its own group
4 The Code (Annotated) ▼

Solution 1 — Sorted String as Key

from collections import defaultdict
def groupAnagrams(strs: list[str]) -> list[list[str]]:
groups = defaultdict(list) # key → list of anagrams
for word in strs:
key = "".join(sorted(word)) # "tea" → "aet"
groups[key].append(word) # Add to group
return list(groups.values()) # Return all groups

Solution 2 — Character Count as Key (Optimal)

from collections import defaultdict
def groupAnagrams(strs: list[str]) -> list[list[str]]:
groups = defaultdict(list)
for word in strs:
count = [0] * 26 # Fixed-size array for a-z
for c in word:
count[ord(c) - ord('a')] += 1
key = tuple(count) # Tuple is hashable
groups[key].append(word)
return list(groups.values())
💡 Why Character Count is Faster

Sorting takes O(k log k) per string. Counting characters takes O(k). For long strings, the count approach is faster. Both use O(n·k) space overall.

⚠️ Common Mistake

Using a list as a dictionary key — lists are unhashable! Convert to tuple: tuple(count) or string: "".join(sorted(word)).

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Recognize the Grouping Pattern

"Group by X" problems almost always use a hash map where X becomes the key. Here X = "anagram equivalence".

Step 2: Define the Key

What makes two strings anagrams? Same characters. How to represent "same characters"? Sort them or count them.

Step 3: Choose Data Structure

defaultdict(list) is perfect — auto-creates empty list for new keys, then append.

Step 4: Discuss Trade-offs

Mention both approaches. Sort is simpler to code; count is asymptotically faster. Ask interviewer which they prefer.

🗣️ What to Say to the Interviewer

"This is a grouping problem — I need to put anagrams in the same bucket. Anagrams have identical character frequencies, so I'll use a canonical representation as the hash key. The simplest is sorting: 'eat' and 'tea' both become 'aet'. I'll iterate once, compute each key, and append to a dictionary. Time is O(n·k log k), space O(n·k). If strings are very long, I can optimize to O(n·k) time using character count tuples as keys."
