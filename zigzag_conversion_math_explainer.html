LeetCode #6 — Solution 2

Zigzag Conversion — Mathematical Index Jump

Medium String Processing Mathematical O(n) Time · O(1) Extra Space
1 Understanding the Problem ▼

Write s = "PAYPALISHIRING" in a zigzag with numRows = 3, then read row by row:

P··· A··· H··· N·
·A·P ·L·S ·I·I ·G
··Y· ··I· ··R· ··

Solution 1 simulates this with row buffers. But this Solution 2 asks: can we skip the simulation and directly calculate which source indices belong to each row?

💡 Key Insight: The Zigzag is Periodic

The zigzag pattern repeats every period = 2 × (numRows − 1) = 4 characters. Within each period, every row gets characters at predictable offsets. We can iterate row-by-row, jumping directly to each row's characters — no simulation needed.

2 The Algorithm: Period-Based Index Calculation ▼
📐 The Period Formulas
period = 2 × (numRows − 1) = 4
Downstroke index = j    (j = row, row + period, row + 2·period, ...)
Upstroke index = j + period − 2 × row
Upstroke only exists for middle rows (not row 0 or last row)

Each row has two types of characters per period cycle:

  • Downstroke — the character placed while the zigzag goes DOWN. Found at index j.
  • Upstroke — the character placed on the way back UP. Found at index j + period - 2*row. Only middle rows have these — top and bottom rows have one char per cycle.
RowDownstroke indices (j)Upstroke indices (j + period − 2×row)Characters
00, 4, 8, 12— (top row, no upstroke)P A H N
11, 5, 9, 133, 7, 11  (j+4−2=j+2)A P L S I I G
22, 6, 10— (bottom row, no upstroke)Y I R
1Compute period = 2 × (numRows − 1)
2For each row (0 to numRows−1):
3 Jump through source by period: j = row, row+period, row+2·period, ...
4 At each j, pick the downstroke char: s[j]
5 If middle row, compute mid = j + period − 2×row. If mid < len(s), pick the upstroke char: s[mid]
6Join all picked chars → result
🔑 Why O(1) Extra Space?

Unlike Solution 1 which stores characters in numRows separate buffers, this approach reads characters in the exact output order. We only need a single result list (which is the output itself). No intermediate row buffers needed.

3 Why It Works: Deriving the Formulas ▼
🔐 The Invariant

After processing row r, the result contains exactly the characters that appear in rows 0..r of the zigzag grid, in left-to-right reading order.

Why period = 2 × (numRows − 1)?

One full zigzag cycle goes: down numRows steps, then back up numRows − 2 steps (excluding the top and bottom endpoints which are shared). Total = numRows + (numRows − 2) = 2 × (numRows − 1).

Why the downstroke is at index j?

Row r's first downstroke char is at position r in the source. Each period later, the next downstroke is at r + period, then r + 2·period, etc.

Why the upstroke is at j + period − 2×row?

The upstroke char is the "mirror" within the period. From position j (downstroke at row r), the next occurrence of row r on the upstroke is period − 2r positions later. For row 0 this equals period (the next downstroke — no separate upstroke). For the last row, period − 2(numRows−1) = 0 — again no separate upstroke.

⚠️ The mid < len(s) Check

The upstroke index might exceed the string length if the string doesn't complete a full period. Always bounds-check mid < len(s) before reading. Also, only emit upstroke for middle rows — top and bottom rows have mid == j + period or mid == j, which are already covered.

Solution 1: Row Buffers

Scan string once, bounce row index, append to row buffers. O(n) time, O(n) space. Simpler code, easier to explain.

Solution 2: Math Index Jump

Iterate row by row, jump to each index mathematically. O(n) time, O(1) extra space. More complex code, shows deeper understanding.

4 The Code (Annotated) ▼
def convert(s: str, numRows: int) -> str: # Edge cases: no zigzag possible if numRows == 1 or numRows >= len(s): return s period = 2 * (numRows - 1) # Full zigzag cycle length (e.g., 4 for 3 rows) result = [] for row in range(numRows): # Process each row top to bottom for j in range(row, len(s), period): # Jump by period within this row result.append(s[j]) # Downstroke char — always exists mid = j + period - 2 * row # Upstroke char index # Only middle rows have upstroke; bounds-check mid if row != 0 and row != numRows - 1 and mid < len(s): result.append(s[mid]) # Upstroke char return ''.join(result)
⚠️ Common Mistake: Forgetting the Row Guard

Without row != 0 and row != numRows - 1, the top/bottom rows would emit duplicates. For row 0: mid = j + period, which is the next downstroke — it would be added twice. For the last row: mid = j, the same index — also a duplicate.

⏱ Complexity

Time: O(n) — each character is visited exactly once across all row iterations.
Space: O(1) extra — beyond the output list (which is mandatory), no row buffers needed.

5 🎤 Interview Strategy ▼
When to Offer This Solution

Start with Solution 1 (row buffers) — it's simpler and faster to code. Then say: "I can also do this with O(1) extra space by computing indices mathematically." This shows depth.

Derive the Period on the Whiteboard

Draw the zigzag for a small example. Count chars in one full down+up cycle. Show: period = 2(numRows-1). Then identify the two formulas: downstroke at j, upstroke at j + period - 2*row.

Emphasize the Guard Condition

Explicitly state: "Top and bottom rows have no upstroke — only middle rows. And mid might exceed string length for partial periods." This prevents bugs.

🗣️ What to Say to the Interviewer

"The zigzag repeats every period = 2*(numRows-1) characters. For each row, I can jump through the source by this period to find the downstroke chars. Middle rows also have an upstroke char at offset period - 2*row within each cycle. I iterate row by row, collect chars in output order, and join. This avoids the row buffers entirely — O(1) extra space."

7 Summary & Key Takeaways ▼
The zigzag is periodic: period = 2 × (numRows − 1)
Each row has downstroke chars at row, row+period, row+2·period, ...
Middle rows also have upstroke chars at j + period − 2×row
Top/bottom rows: one char per cycle. Middle rows: two chars per cycle
O(n) time, O(1) extra space — no intermediate row buffers
🎯 When to Use This Pattern
  • When the interviewer asks for O(1) space after you present the simulation
  • Any periodic/cyclic pattern where you can pre-compute index offsets
  • When the output order differs from input order but follows a mathematical rule
