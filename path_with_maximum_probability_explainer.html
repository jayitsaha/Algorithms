LeetCode #1514

Path with Maximum Probability

Find the path from start to end that maximizes the probability of success in an undirected weighted graph.

Medium Graph Dijkstra's (Max Product) Time: O((E+V) log V) Space: O(V+E)
1 Understanding the Problem ▾

You're given an undirected graph with n nodes. Each edge has a success probability between 0 and 1. When you traverse an edge with probability p, there's a p chance of successfully crossing it.

Along any path, the total probability of successfully reaching the destination is the product of all edge probabilities along that path — because each hop is an independent event. A 0.5 edge then a 0.8 edge gives 0.5 × 0.8 = 0.40 total success probability.

💡 Key Insight

This is Dijkstra's algorithm in disguise. Instead of minimizing the sum of edge weights, we maximize the product of edge probabilities. The greedy property still holds: since all weights are ≤ 1.0, multiplying can only decrease (or maintain) a value. So the first time we reach a node via the max-heap, we have found its optimal probability.

✓ Best Path: 0 → 2 → 3

0→2: p = 0.6
2→3: p = 0.8
Total: 0.6 × 0.8 = 0.480

✗ Worse Path: 0 → 1 → 3

0→1: p = 0.5
1→3: p = 0.7
Total: 0.5 × 0.7 = 0.350

Edge Cases to consider: no path exists between start and end (return 0.0), start equals end (return 1.0), single direct edge connecting start and end, edges with probability 0.0 (effectively no connection), disconnected graph components.

2 The Algorithm: Modified Dijkstra (Max-Product) ▾

Classic Dijkstra greedily expands the node with the minimum known distance. Here we flip it: greedily expand the node with the maximum known probability. Everything else is structurally identical — just swap addition for multiplication and min-heap for max-heap.

1 Build adjacency list: For each edge [u, v] with probability p, add (v, p) to graph[u] and (u, p) to graph[v] since the graph is undirected.
2 Initialize probability array: prob[i] = 0.0 for all nodes. Set prob[start] = 1.0 — we start here with 100% certainty.
3 Push start onto max-heap: Push (-1.0, start). Python's heapq is a min-heap; negating probabilities turns it into a max-heap.
4 Greedy expansion: Pop the node with highest current probability. If it is end, return immediately — this is optimal by Dijkstra's invariant. For each neighbor, if prob[node] × edge_p > prob[neighbor], update and push.
5 Stale entry pruning: When popping (-p, node), if p < prob[node], this is an outdated heap entry — skip it. This "lazy deletion" avoids the need to decrease-key in the heap.
🔬 Dijkstra's Algorithm Deep-Dive

Origin: Edsger W. Dijkstra, 1956. Solves single-source shortest path on graphs with non-negative edge weights in O((V+E) log V) with a binary heap.

Core invariant: When a node is extracted from the priority queue, its stored value is finalized — no future path can be better. This holds because edge weights are non-negative (can't "undo" a bad path by adding more edges).

In this problem: "Non-negative" maps to "all probabilities ≤ 1.0." Extending any path by one more edge multiplies by something ≤ 1, so probabilities can only decrease or stay flat. The max-heap invariant holds perfectly.

Recognize when: "Weighted graph, find optimal path, all weights non-negative" → reach for Dijkstra. The operation (sum, product, max, min over edges) adapts, but the heap-based greedy structure stays the same.

3 Why It Works: The Invariant ▾
🔐 The Invariant

When a node is popped from the max-heap, prob[node] is finalized as the globally maximum probability to reach that node. No later path can exceed it.

Proof sketch: Suppose node v is popped with probability p. For any alternative path to v that passes through some unvisited node u: we must have prob[u] ≤ p (since v was popped first, meaning p ≥ prob[u] at that moment). The edge weight w(u, v) ≤ 1.0. So that alternative achieves at most prob[u] × 1.0 ≤ p. The current value is already optimal.

Why not just BFS? BFS finds the shortest path in terms of hop count on unweighted graphs. Here, edges have different probabilities, so the path with fewest hops is not necessarily the highest-probability path. We need to account for the weights.

Why not Bellman-Ford? Bellman-Ford is needed when edges can have negative weights (which can undo progress, requiring multiple passes). Here all probabilities are ≥ 0, so Dijkstra's one-pass greedy approach is correct and faster.

⚠️ The Heap Negation Trick

Python's heapq only supports min-heaps. To get max-heap behavior: store -probability. The entry with the smallest negative value corresponds to the largest probability. Example: (-0.9, node2) is popped before (-0.5, node1) — giving us node2 first, which has the higher probability 0.9. Un-negate on pop with p = -neg_p.

4 The Code (Annotated) ▾

Solution 1 — Modified Dijkstra with Max-Heap (Optimal, Interview Pick)

import heapq ← Python's heap is min-heap by default
from collections import defaultdict
def maxProbability(n, edges, succProb, start, end):
graph = defaultdict(list) ← Adjacency list: node → [(neighbor, prob), ...]
for (u, v), p in zip(edges, succProb): ← Pair each edge with its probability
graph[u].append((v, p)) ← Undirected: add both directions
graph[v].append((u, p))
prob = [0.0] * n ← Best probability found so far for each node
prob[start] = 1.0 ← Starting node: certain (probability = 1)
heap = [(-1.0, start)] ← Negate to use min-heap as max-heap
while heap:
neg_p, node = heapq.heappop(heap) ← Pop node with highest probability
p = -neg_p ← Un-negate to get actual probability
if node == end: ← First time we pop end = optimal (Dijkstra invariant)
return p
if p < prob[node]: ← Stale heap entry — better path was found later
continue ← Skip: lazy deletion instead of decrease-key
for neighbor, edge_p in graph[node]: ← Explore all neighbors
new_p = p * edge_p ← Multiply: independent probabilities combine by product
if new_p > prob[neighbor]: ← Found a better path to this neighbor
prob[neighbor] = new_p ← Update best probability
heapq.heappush(heap, (-new_p, neighbor)) ← Push negated for max-heap behavior
return 0.0 ← Heap exhausted — no path to end exists
⚠️ Common Mistake: Not Handling Stale Entries

A node can be pushed multiple times as we find progressively better paths. Without the if p < prob[node]: continue check, we'd process outdated entries and potentially push even more stale entries, causing wasted work and wrong updates. Always prune stale entries.

Solution 2 — Bellman-Ford Relaxation (Simpler to Remember)

def maxProbability_bellman(n, edges, succProb, start, end):
prob = [0.0] * n; prob[start] = 1.0 ← Initialize
for _ in range(n - 1): ← Relax all edges n-1 times
updated = False
for (u, v), p in zip(edges, succProb): ← Try both directions (undirected)
if prob[u] * p > prob[v]: ← Can we improve path to v via u?
prob[v] = prob[u] * p; updated = True
if prob[v] * p > prob[u]: ← Can we improve path to u via v?
prob[u] = prob[v] * p; updated = True
if not updated: break ← Early exit if no progress
return prob[end] ← 0.0 if unreachable
💡 Complexity Summary

Dijkstra (max-heap): Time O((E+V) log V), Space O(V+E) — optimal, interview pick
Bellman-Ford: Time O(V×E), Space O(V) — simpler to code from memory
Both return 0.0 if no path exists — Dijkstra because heap exhausts; Bellman-Ford because prob[end] stays 0.0

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize Weighted Graph + Path Optimization

The problem says: weighted graph, find path maximizing some value between two nodes. This is the textbook setup for Dijkstra or Bellman-Ford. Ask yourself: "Are all weights non-negative?" Probabilities are in [0, 1] — yes, non-negative. Dijkstra applies.

Step 2: Identify the Twist — Max Product, Not Min Sum

Standard Dijkstra computes dist[v] = min(dist[v], dist[u] + w). Here we compute prob[v] = max(prob[v], prob[u] × w). The algorithm skeleton is identical; only the update formula and heap direction change. Mention Bellman-Ford O(VE) as a simpler fallback, then offer Dijkstra O((V+E) log V) as the optimal.

Step 3: Implement the Max-Heap Trick

Python's heapq is a min-heap. The standard trick: negate values on push, un-negate on pop. Initialize with (-1.0, start) for probability 1.0 at start. Prune stale entries with the p < prob[node]: continue guard.

Step 4: Handle Edge Cases Explicitly

If start == end, return 1.0 immediately (already at destination). If the heap exhausts, return 0.0 (no path). Mention these proactively — shows completeness.

🗣️ What to Say to the Interviewer

"This is a weighted graph path problem — I'll use modified Dijkstra.
The twist: instead of minimizing sum, I maximize product of probabilities.
Since all probabilities are in [0,1], the greedy invariant holds — the first
time we pop a node from the max-heap, its probability is finalized.
I'll negate probabilities to use Python's min-heap as a max-heap.

🔮 Likely Follow-up Questions
  • "How would you reconstruct the actual path?" → Track parent[v] = u on each update; backtrack from end to start.
  • "What if the graph were directed?" → Remove one of the two adjacency insertions; algorithm unchanged.
  • "Could you use log probabilities?" → Yes! log(p1 × p2) = log(p1) + log(p2). Maximize sum of log probs — now standard Dijkstra with negated log weights.
  • "What if edges had probabilities > 1?" → Dijkstra breaks (could loop). Use Bellman-Ford with iteration limit.

7 Summary & Key Takeaways ▾
Dijkstra adapts to max-product: swap min-heap for max-heap, swap addition for multiplication
Negate probabilities to use Python's min-heap as a max-heap (standard trick)
Greedy invariant holds: all probabilities ≤ 1 means extending a path can only decrease probability
Prune stale heap entries with if p < prob[node]: continue (lazy deletion)
First time end is popped = maximum probability (Dijkstra's invariant)
Return 0.0 when heap exhausts — no path from start to end exists
ApproachTimeSpaceNotes
Bellman-Ford O(V × E) O(V) Simple; slower; works for all edge weights
BFS / SPFA O(V × E) worst O(V+E) Fast in practice; not worst-case optimal
Dijkstra (max-heap) ✓ O((E+V) log V) O(V+E) Optimal; interview pick; early exit
🎯 When to Recognize Dijkstra Variants
  • Weighted graph + optimize a path metric → Dijkstra (if weights ≥ 0)
  • Maximize product along path → Max-heap Dijkstra (this problem)
  • Minimize max bottleneck edge → Modified Dijkstra with max() instead of sum()
  • Shortest path with extra constraints (k stops, fuel) → DP + BFS or modified Dijkstra
  • Negative weights possible → Bellman-Ford instead
