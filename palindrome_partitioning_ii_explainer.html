LeetCode #132

Palindrome Partitioning II

Given a string s, partition s such that every substring of the partition is a palindrome. Return the minimum number of cuts needed for a palindrome partitioning of s.

Hard Dynamic Programming Min Cuts + isPalindrome DP Time: O(n²) Space: O(n²)
1 Understanding the Problem ▾

A palindrome partition splits a string into substrings where every piece reads the same forwards and backwards. For "aab", one valid partition is ["a","a","b"] — that uses 2 cuts. But the minimum is ["aa","b"] — just 1 cut. We want that minimum.

💡 Key Insight

Think about it as: "where is the last cut?" If we cut right before index j, the right piece s[j..i] must be a palindrome, and the left piece s[0..j-1] needs to be optimally partitioned too. This gives us a recurrence: dp[i] = min(dp[j-1] + 1) over all j where s[j..i] is a palindrome. We need two DP tables — one for palindrome precomputation, one for minimum cuts.

Suboptimal: s = "aab"

"a" | "a" | "b"
→ 2 cuts
Every piece is a palindrome ✓
But NOT the minimum.

Optimal: s = "aab"

"aa" | "b"
→ 1 cut ← minimum
"aa" is a palindrome ✓
"b" is a palindrome ✓

Edge cases to consider: A string already a palindrome like "aba" (0 cuts), all unique chars like "abc" (n−1 cuts), single character (0 cuts), and repeated chars like "aaaa" (0 cuts). An empty string trivially returns 0.

🔑 Why is This DP?

Optimal substructure: The minimum cuts for s[0..i] depends on the minimum cuts for shorter prefixes — we can reuse already-computed smaller answers.
Overlapping subproblems: A naive recursive approach checks the same (start, end) palindromes from many parent calls, causing exponential re-computation. DP stores each result exactly once.

2 The Algorithm: Two-Pass DP ▾

We split the problem cleanly into two independent DP passes. Pass 1 answers "which substrings are palindromes?" in O(n²). Pass 2 uses those answers to find "what's the minimum cuts for each prefix?" also in O(n²).

1 Build isPal[i][j] — the palindrome precomputation: A 2D boolean table. isPal[i][j] = True iff s[i..j] is a palindrome. Fill by increasing substring length: length 1 (all True), length 2 (equal chars), length 3+ (outer chars equal AND inner is palindrome). This O(n²) pass makes every future palindrome check O(1).
2 Define dp[i] — the minCuts DP: dp[i] = minimum cuts needed to partition s[0..i] into palindromes. dp[0] = 0 (single char, always a palindrome). Worst case initialization: dp[i] = i (cut every character into its own piece).
3 Early exit — entire prefix is a palindrome: Before trying all cuts, check isPal[0][i]. If True, the whole prefix is already a palindrome, so dp[i] = 0. No inner loop needed.
4 Try every possible last palindrome chunk: For each j from 1 to i, if isPal[j][i] is True, then dp[i] = min(dp[i], dp[j-1] + 1). This tries every possible "last palindromic piece" and picks the best split.
5 Return dp[n-1] — the minimum cuts for the entire string.
📐 The Recurrence Relations
isPal[i][i] = True ← single char always palindrome isPal[i][i+1] = (s[i] == s[i+1]) ← pair: palindrome iff equal isPal[i][j] = (s[i] == s[j]) AND isPal[i+1][j-1] ← expand outer chars dp[i] = 0 if isPal[0][i] (early exit) dp[i] = min(dp[j-1] + 1) for all j in [1..i] where isPal[j][i]
3 Why It Works: The Invariant ▾
🔐 The Invariant

When computing dp[i], all values dp[0], dp[1], ..., dp[i-1] are already correct. Therefore, when we try dp[j-1] + 1 for some valid j, we know dp[j-1] is the true minimum for the left prefix. We are guaranteed to find the global minimum for dp[i] by taking the best over all valid last palindromic chunks.

Why fill isPal by length (not row-by-row): When checking isPal[i][j] for a 3+ character substring, we need isPal[i+1][j-1] — a shorter substring. If we fill by increasing length, shorter substrings are always computed before longer ones that depend on them. Filling row-by-row would access uninitialized entries.

Why worst-case dp[i] = i (not infinity): For a prefix of length i+1, cutting every character requires exactly i cuts — this is the provable worst case, not just a safe upper bound. Every single character is a palindrome, so this is always achievable.

⚠️ Common Mistake: Checking Palindrome Inside the dp Loop

If you compute isPal(j, i) on-the-fly inside the dp loop (e.g., by scanning inward), each check is O(n), making the whole algorithm O(n³). The two-pass approach decouples these: precompute all O(n²) palindrome answers first, then each lookup is O(1) during the cuts DP. Always separate the precomputation!

Why the early exit for isPal[0][i] matters: If s[0..i] is a palindrome, dp[i] = 0 — zero cuts. If we skipped this and went to the inner loop, we'd still find dp[i] = 0 eventually (j=0 gives dp[-1]+1 which we'd define as 0+1=1, which is wrong). The early exit is logically essential, not just an optimization.

4 The Code (Annotated) ▾

Solution 1 — Two-Pass DP: isPalindrome table + minCuts (Optimal, Interview Pick)

def minCut(s: str) -> int:
n = len(s)← total length of s
# ── Pass 1: Build isPal[i][j] ──
isPal = [[False] * n for _ in range(n)]← n×n table, default False
for length in range(1, n + 1): ← fill by increasing substring length
for i in range(n - length + 1): ← i is start index
j = i + length - 1 ← j is end index
if length == 1:
isPal[i][j] = True ← single char: always a palindrome
elif length == 2:
isPal[i][j] = (s[i] == s[j]) ← pair: palindrome iff both chars equal
else:
isPal[i][j] = (s[i] == s[j] and isPal[i+1][j-1])← outer chars match AND inner already computed
# ── Pass 2: dp[i] = min cuts for s[0..i] ──
dp = [0] * n← initialize all to 0
for i in range(n):
if isPal[0][i]: ← entire prefix is a palindrome → 0 cuts needed
dp[i] = 0
else:
dp[i] = i ← worst case: i cuts (cut every character)
for j in range(1, i + 1): ← try each position for the "last" cut
if isPal[j][i]: ← s[j..i] is a palindrome?
dp[i] = min(dp[i], dp[j-1] + 1)← optimal left + 1 cut before j
return dp[n - 1] ← min cuts for the full string
⚠️ Why dp[i] = i and not dp[i] = float('inf')?

Using i as the initial value for dp[i] is exact — for a prefix of length i+1, cutting every character into its own palindrome requires exactly i cuts. Starting with float('inf') would also work correctly, but i is tighter and more meaningful. Also, this guarantees the condition dp[i] = i skips needing a special case for "no valid j found" (which can't happen since every single character is a palindrome).

💡 Complexity Summary

Two-Pass DP (bottom-up): Time O(n²) for both passes. Space O(n²) for isPal table + O(n) for dp array.
Memoized Recursion: Same time O(n²) with memoization, O(n²) space for cache + call stack.
Manacher's + 1D DP: O(n²) time, O(n) space — Manacher's builds palindrome info in O(n), avoiding the full table. Advanced interview bonus.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the DP Signal

"I need to minimize the number of cuts. Every valid configuration is a palindrome partition. The answer for the full string depends on answers for shorter substrings — classic optimal substructure. This is a 1D DP on prefix length."

Step 2: Define State and Recurrence

"Let dp[i] = min cuts for s[0..i]. Transition: try each possible last palindromic piece s[j..i]. If s[j..i] is a palindrome, dp[i] = min(dp[i], dp[j-1] + 1). Base: if s[0..i] is itself a palindrome, dp[i] = 0."

Step 3: Spot the Bottleneck and Separate

"Naively checking isPalindrome(j, i) inside the dp loop is O(n) per check → O(n³) total. I'll precompute all O(n²) palindrome results in a separate table first, filled by increasing length. Then each lookup inside the dp loop is O(1) → O(n²) total."

Step 4: Mind the Fill Order for isPal

"When checking isPal[i][j] for length ≥ 3, I need isPal[i+1][j-1]. Filling by increasing substring length ensures every sub-result is ready before it's needed."

🗣️ What to Say to the Interviewer

"This is a minimization problem with optimal substructure — DP.
I define dp[i] = min cuts for s[0..i]. The recurrence is:
dp[i] = 0 if s[0..i] is a palindrome, else
dp[i] = min(dp[j-1] + 1) for all j where s[j..i] is a palindrome.
Naively that's O(n³) if I check palindromes inline.
I'll precompute isPal[i][j] in a separate O(n²) DP, filling by length.
Then the dp loop is O(n²) with O(1) palindrome lookups.
Overall: O(n²) time, O(n²) space."

7 Summary & Key Takeaways ▾
Two separate DP tables: isPal[i][j] precomputes all palindrome checks in O(n²)
dp[i] = min cuts for prefix s[0..i]; transition: try every valid last palindromic chunk s[j..i]
Fill isPal by increasing length so sub-sub-answers are always ready
Early exit: if s[0..i] is a palindrome, dp[i] = 0 immediately — skip inner loop
Worst-case initialization dp[i] = i is exact — every character cut individually
Without the isPal table, checking palindromes inline causes O(n³) — always precompute

🎯 When to Recognize the Min Cuts + isPalindrome DP Pattern
  • "Minimum cuts / partitions of a string where every piece satisfies property X" → DP on prefixes + precompute X
  • Palindrome checks inside a DP loop → always precompute in a separate O(n²) pass
  • 1D dp[i] depends on dp[j-1] plus checking s[j..i] → interval DP precomputation pattern
