LeetCode #1975

Maximum Matrix Sum

Given an n×n integer matrix, you can multiply any two adjacent elements by −1 any number of times. Return the maximum possible sum.

Medium Greedy Count Negatives Flip Pairs Time: O(n²) Space: O(1)
1 Understanding the Problem ▾

You have an n × n matrix of integers. One allowed operation: pick any two adjacent cells (sharing a horizontal or vertical edge) and flip both their signs — multiply both by −1 simultaneously. You may perform this operation any number of times. Goal: maximize the total sum of all elements.

💡 The Core Insight: Negatives Are Mobile

Flipping two adjacent cells can move a negative sign from one cell to its neighbor. By chaining adjacent flips, you can slide any negative sign to any cell in the matrix. This means: you have complete freedom over which cells are negative — but not over how many are negative.

Why can you move a negative? Suppose cell A = −5 and its neighbor B = +3. Flip both: A becomes +5, B becomes −3. The "−" moved from A to B. Chain this rightward, downward, diagonally (via two steps), anywhere. You can transport any negative to any position.

Why can't you change parity? Each flip changes the sign of exactly two cells. The count of negative cells changes by −2 (two negatives become positive), 0 (one of each, signs swap), or +2 (two positives become negative). In every case, the change is even — so the parity of the negative count never changes.

Edge cases: If a zero exists in the matrix, it's the ideal "negative bearer" for the odd case — placing the remaining − on a zero incurs zero penalty. Single-cell matrix: 1×1, no adjacency, no flips possible.

2 The Algorithm: One-Pass Greedy Scan ▾

We never need to actually simulate any flips. Instead, we collect three statistics in one pass, then apply the decision formula:

1 Scan all cells once: accumulate total_abs (sum of |each cell|), neg_count (how many cells are negative), and min_abs (smallest |value| across all cells).
2 Check parity: Is neg_count even or odd?
3 Even: All negatives can be eliminated. Return total_abs — the ideal case where every cell contributes its full magnitude positively.
4 Odd: One negative is unavoidable. Move it to the cell with the smallest absolute value to minimize the penalty. Return total_abs − 2 × min_abs.
neg_count % 2 == 0?

🔐 Why "subtract twice"?

total_abs was computed as if every cell were positive. When we must make one cell negative — say the cell with value v — its actual contribution is −v instead of +v. The difference is −v − (+v) = −2v. So: answer = total_abs − 2 × min_abs.

3 Why It Works: The Parity Invariant ▾
🔐 The Invariant

After any sequence of operations, the parity (odd/even) of the count of negative numbers in the matrix is unchanged from the initial state. This is preserved by every single flip.

Proof (case analysis on one flip):

  • Both cells positive (+, +) → (−, −): neg_count increases by 2 → parity unchanged.
  • Both cells negative (−, −) → (+, +): neg_count decreases by 2 → parity unchanged.
  • Mixed (+, −) → (−, +) or (−, +) → (+, −): neg_count unchanged → parity unchanged.

Every possible flip falls into one of these three cases. In no case does the parity of neg_count change. Therefore: starting parity = final parity, regardless of how many flips you do.

Reachability: Given that we can move any negative anywhere, and parity is the only constraint, we can reach any configuration with the same parity. This means: even start → can make all positive; odd start → can make exactly one negative, placed anywhere we choose.

⚠️ Common Misconception

Students sometimes think: "Can't I flip a negative cell with an adjacent positive, making both positive?" No — flipping multiplies both by −1. A negative becomes positive, but the positive becomes negative. The negative sign moves, not disappears. You need to catch two negatives together to eliminate both.

💡 Greedy Optimality

Given that exactly one cell must be negative (odd case), and we want to maximize the sum, we want to minimize the magnitude of that negative cell. The optimal placement is the cell with the smallest absolute value. This is why we track min_abs.

4 The Code (Annotated) ▾

Optimal Solution — O(n²) time, O(1) space (Interview Pick)

def maxMatrixSum(matrix: List[List[int]]) -> int:
total_abs = 0 ← Accumulates sum of |val| for every cell
neg_count = 0 ← Counts how many cells have val < 0
min_abs = float('inf') ← Tracks the smallest |val| across all cells
for row in matrix: ← Outer loop: iterate over each row
for val in row: ← Inner loop: each individual cell value
total_abs += abs(val) ← Always add the absolute value (as if all were positive)
if val < 0: ← Is this cell negative?
neg_count += 1 ← Yes: increment the negative counter
min_abs = min(min_abs, abs(val)) ← Keep track of the globally smallest |val|
if neg_count % 2 == 0: ← Even negatives: we can eliminate them all
return total_abs ← Perfect case: every cell contributes positively
else: ← Odd negatives: exactly one unavoidable negative
return total_abs - 2 * min_abs ← Place the negative on the smallest cell; penalty = 2×min_abs
⚠️ Why not just return total_abs − min_abs in the odd case?

total_abs included +min_abs for that cell. But its actual value is −min_abs. To fix: subtract min_abs (cancel the mistaken +) and subtract another min_abs (add the correct −). Total correction = −2 × min_abs.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Notice "unlimited operations" — find the invariant

When problems allow unlimited operations, the interesting question is: what is preserved? Here, each flip negates exactly two cells. Ask: how does this affect the count of negatives? Answer: it changes by 0 or ±2, never ±1 → parity is invariant.

Step 2: Verify mobility — can you move negatives freely?

Confirm with a small example: can you move a negative from (0,0) to (2,2)? Yes — shift it right by flipping (0,0)+(0,1), then flip (0,1)+(0,2), then flip (0,2)+(1,2), etc. The matrix is connected via adjacency, so any cell is reachable.

Step 3: Solve the two-case decision

Even count of negatives: move pairs together and flip → all become positive → answer = sum of |values|. Odd count: one negative survives → greedily minimize its magnitude → place it on the min-absolute-value cell → penalty = 2 × min_abs.

Step 4: Verify edge cases

Zero in matrix with odd neg_count: min_abs = 0, penalty = 0 → same result as even case. All same values: if even count → total = n²×|v|. Single element: no operations possible, return matrix[0][0].

🗣️ What to Say to the Interviewer

"The operation flips the sign of two adjacent cells simultaneously. With unlimited operations,
I want to find what's invariant. Each flip changes neg_count by 0 or ±2 — so the parity
(even/odd) of neg_count never changes. Also, since the matrix is connected via adjacency,
I can move any negative to any cell by chaining flips.

So: if neg_count is even, I pair up all negatives and flip them positive — answer = sum of
absolute values. If neg_count is odd, one negative is inescapable — move it to the cell with
the smallest absolute value to minimize the penalty.

Formula: scan once to get total_abs, neg_count, min_abs. Return total_abs if even,
total_abs − 2×min_abs if odd. Time O(n²), space O(1)."
💡 Follow-up Questions to Expect
  • "What if operations were limited to k flips?" — Greedy breaks. Would need BFS/DP to determine reachable configurations.
  • "What if you could only flip diagonally adjacent pairs?" — Need to re-examine connectivity and parity on the diagonal subgraph.
  • "Is there a zero in the matrix?" — Zero is the perfect "negative bearer": makes the odd-case penalty vanish entirely.
  • "What about a 1×1 matrix?" — No operations possible (no adjacent pair). Return matrix[0][0] as-is.

7 Summary & Key Takeaways ▾
Each flip changes neg_count by 0 or ±2 — the parity of negative count is an invariant you cannot escape.
Negatives are "mobile" across the matrix via chaining adjacent flips — connectivity of the adjacency graph ensures you can place them anywhere.
Even negatives → eliminate all → answer = sum of absolute values. Odd → one survives → place it on the min-abs cell.
Formula: total_abs − 2 × min_abs in the odd case. The "−2×" corrects for the cell that must flip from +v to −v.
A zero in the matrix is the perfect "negative bearer" for the odd case — penalty is 2×0 = 0.
One O(n²) scan, three variables, O(1) extra space. Clean, fast, interview-ready.
🎯 When to Use This Pattern (Count Negatives / Flip Pairs)
  • Problem says "multiply two elements by −1, any number of times"
  • You need to maximize a sum involving signs
  • Operations come in pairs → look for parity invariants
  • The operation is symmetric (flips two, not one) → conservation law present
