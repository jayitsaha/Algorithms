LeetCode #78

Subsets

Given an integer array nums of unique elements, return all possible subsets (the power set). The solution set must not contain duplicate subsets.

Medium Backtracking Include / Exclude Each Element Time: O(n · 2ⁿ) Space: O(n)
1 Understanding the Problem ▾

The power set of a set S is the collection of every possible subset of S — including the empty set [] and S itself. For input [1, 2, 3], there are exactly 8 subsets (2³ = 8). The result can be in any order; there are no duplicates because all elements of nums are unique.

💡 Key Insight

For each element in the array, you face exactly one binary decision: include it in the current subset, or exclude it. Making this decision independently for each of n elements generates all 2ⁿ subsets — no duplicates, no omissions. This binary choice structure is the heart of the include/exclude backtracking pattern.

Constraints to note: Elements are guaranteed unique, so no deduplication logic is needed (contrast with Subsets II, #90, which has duplicates and requires sorting + skipping). The order of subsets in the output does not matter, and elements within a subset can appear in any order.

⚠️ Not Combinations or Permutations

Subsets differ from combinations (subsets of a fixed size k) and permutations (all orderings of elements). Here we want all sizes from 0 to n, and order within each subset is irrelevant. This distinction matters for choosing the right template in interviews.

2 The Algorithm: Include / Exclude Backtracking ▾

We use a depth-first search (DFS) with a running path list. At each index we make two recursive calls — one including the current element, one excluding it. When we've decided all n elements (reached the end of the array), the path is a complete subset and we record a copy of it.

1 Initialise: Start with an empty path [] and index 0. The path holds the partial subset being built; the index says which element we're currently deciding about.
2 Base case — record the subset: When index == len(nums), all n decisions have been made. Append path[:] (a copy!) to the result list and return.
3 Include branch: path.append(nums[index]), then call backtrack(index + 1). This explores every subset that contains the current element.
4 Exclude branch (backtrack): After the include branch returns, path.pop() to undo the addition, then call backtrack(index + 1) again. This explores every subset that does NOT contain the current element.
5 Return results: The recursion tree has exactly 2ⁿ leaves, one per unique subset. Collect them all and return.

🔑 Iterative / BFS-style Alternative

Start with result = [[]]. For each number num, take every existing subset and create a new one with num appended. After processing all n numbers, result contains all 2ⁿ subsets. This is O(n·2ⁿ) time and O(n·2ⁿ) space (all subsets in memory). Very clean code — one loop, one list comprehension — but gives less insight into the backtracking pattern that generalizes to harder problems.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the moment we call backtrack(index, path), the variable path contains a consistent partial decision: for every element at position < index, we have already decided to include or exclude it, and path reflects those choices. Elements from index onward have not been decided. Completing all remaining decisions in all possible ways generates exactly all subsets that extend path.

Why no duplicates? Each element occupies a unique index and is processed exactly once in fixed left-to-right order. At each index we branch into exactly two states: include or exclude. No element is ever revisited. Every distinct include/exclude sequence of length n maps to a distinct subset, and every subset corresponds to exactly one such sequence.

Why exactly 2ⁿ subsets? The recursion tree is a complete binary tree of depth n. Every internal node has exactly 2 children. The number of leaves in a complete binary tree of depth n is 2ⁿ. Each leaf corresponds to a unique sequence of n binary decisions, and thus to a unique subset.

⚠️ The Copy Trap: path vs path[:]

When recording at the base case, you must store a copy of path — not a reference to path itself. Since path is a shared mutable list, storing a reference means every recorded subset will reflect path's final state (typically empty after all backtracking). Always use result.append(path[:]) or result.append(list(path)).

Edge cases handled naturally:

  • nums = [] → index 0 equals len(nums) immediately, records []. Result: [[]]. Correct.
  • nums = [5] → two leaves: [5] and []. Result: [[5], []]. Correct.
  • Large n → O(n·2ⁿ) is unavoidable — the output itself has that many characters.
4 The Code (Annotated) ▾

Solution 1 — Include/Exclude Backtracking (Interview Pick)

def subsets(nums): ← Input: list of unique integers
result = [] ← Accumulate completed subsets here
path = [] ← Shared mutable path (partial subset)
def backtrack(index): ← DFS: decide nums[index] next
if index == len(nums): ← Base case: all decisions made
result.append(path[:]) ← path[:] copies the list — CRITICAL
return
# --- Include branch ---
path.append(nums[index]) ← Add current element to subset
backtrack(index + 1) ← Recurse: decide remaining elements
# --- Exclude branch (backtrack) ---
path.pop() ← Undo: restore path to pre-include state
backtrack(index + 1) ← Recurse without the element
backtrack(0) ← Start from index 0, empty path
return result ← All 2ⁿ subsets collected
💡 Complexity

Time O(n·2ⁿ): The recursion makes 2ⁿ calls that reach the base case (one per subset). Each records a copy of path costing O(n). Total = O(n·2ⁿ).
Auxiliary Space O(n): The call stack is at most n levels deep. The path list holds at most n elements. Output space (O(n·2ⁿ)) is not counted in auxiliary space complexity.

Solution 2 — Iterative (very clean, O(n·2ⁿ))

def subsets(nums):
result = [[]] ← Start: only the empty subset
for num in nums: ← Process one element at a time
result += [s + [num] ← For each existing subset s, create
for s in result] ← a new subset s ∪ {num}. Doubles count!
return result

Solution 3 — Bitmask (O(n·2ⁿ), clever but less readable)

def subsets(nums):
n = len(nums)
result = []
for mask in range(1 << n): ← 0..2ⁿ-1; each integer IS a subset
subset = [nums[i] for i in range(n) ← Bit i of mask = include nums[i]?
if mask & (1 << i)] ← 1 << i is 2^i; & tests if bit is set
result.append(subset)
return result
⚠️ Bitmask: Know When to Use It

The bitmask solution is elegant and correct, but in an interview it can be a black box if you can't explain it fluently. Only present it if you're comfortable explaining why iterating mask from 0 to 2ⁿ - 1 covers all subsets, and why mask & (1 << i) checks whether bit i is set. Otherwise, the backtracking or iterative approach is safer.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Establish the Output Size

"For n unique elements, the answer must contain exactly 2ⁿ subsets — I can verify: n=3 → 8. This tells me any algorithm is at least Ω(2ⁿ), so I'm targeting O(n·2ⁿ) where the extra n factor accounts for copying each subset."

Step 2: Name the Pattern

"Each element has exactly two options: included or excluded. This binary decision per element screams include/exclude backtracking — the same template used for combinations, subsets with sum, etc."

Step 3: Translate to DFS Code

"State = (current index, current path). Two recursive calls at each node: append + recurse, then pop + recurse. Base case = index hits the end: record a copy of path."

Step 4: Call Out the Copy Gotcha

"One subtle point: I must append path[:] not path. Since path is shared across the recursion, storing a reference would mean all recorded subsets see the same (eventually empty) list."

🗣️ What to Say to the Interviewer

"The output has 2ⁿ subsets, so O(n·2ⁿ) is our target — this is tight to the output size.
I'll use include/exclude backtracking: for each element, branch into 'include it' and 'exclude it'.
Base case: when index reaches the end, record a COPY of the current path.
Backtrack: pop the element after the include branch before the exclude branch.
Auxiliary space is O(n) for the call stack. The output itself is O(n·2ⁿ)."
🔮 Common Follow-up Questions
  • "What if nums has duplicates?" → Sort + skip same element at the same recursion depth (Subsets II, #90)
  • "Only subsets of size exactly k?" → Add size guard to base case (Combinations, #77)
  • "Subsets summing to target?" → Add sum guard; this becomes Subset Sum / Combination Sum (#39, #40)
  • "Count subsets, don't list them?" → Same recursion but count leaves; optimize with DP for large inputs
  • "Can you do it iteratively?" → Yes: start with [[]], double the result for each element

7 Summary & Key Takeaways ▾
Power set size is exactly 2ⁿ — any solution must be at least O(2ⁿ), and copying each subset costs O(n), giving O(n·2ⁿ) total
Each element has one binary decision: include or exclude. n decisions = 2ⁿ unique outcomes
The append-recurse-pop pattern (backtracking) maintains path invariant: add, recurse deeply, undo, recurse differently
ALWAYS copy path at the base case: path[:] not path (shared reference = silent bug)
Auxiliary space is O(n) for the call stack — independent of the exponential output
This same template generalises to Combinations, Subsets II, Combination Sum, Letter Combinations, and many more

🎯 When to Recognise the Include/Exclude Pattern
  • "Generate all subsets / power set" → Include/Exclude backtracking
  • "Count subsets with property X" → Same recursion, count at leaves
  • "Does a subset summing to target exist?" → Same recursion, early-exit True at hit
  • "Subsets with duplicates" → Sort first + skip same element at the same recursion depth
  • "Select k elements from n" → Combinations — add size guard at base case
