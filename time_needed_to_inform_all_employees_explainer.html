LeetCode #1376

Time Needed to Inform All Employees

Find the total minutes for the company head to inform all employees through a weighted manager-tree, where each relay takes a different amount of time.

Medium Graph DFS from Root Time: O(n) Space: O(n)
1 Understanding the Problem ▾

A company has n employees numbered 0 to n-1. You are given:

  • headID — the index of the company head (no manager)
  • manager[i] — who manages employee i (head has manager[headID] = -1)
  • informTime[i] — how many minutes employee i takes to tell ALL their direct reports simultaneously

Information flows from the head downward. Each manager tells all their direct reports at once (in parallel), but the relay takes informTime[i] minutes. Then each of those employees relays to their own reports, and so on until every employee is informed.

💡 Key Insight

The time for any employee to be informed equals the sum of informTime values along the path from the head to that employee. All branches inform in parallel, so the total time equals the maximum such path sum — the slowest branch determines when everyone is done.

✓ Concrete Example

Tree: 0 is head
0 → {1, 2}
1 → {3, 4}
2 → {5}

informTime = [6, 4, 2, 0, 0, 0]
Path 0→1→3: 6+4 = 10
Path 0→1→4: 6+4 = 10

2 The Algorithm: DFS with Accumulated Time ▾

The manager[] array defines a rooted tree with headID as root. We perform Depth-First Search (DFS) from the root, tracking the total time accumulated from the head to each node. The answer is the maximum accumulated time seen at any node.

1 Invert manager[] to children[]: The manager array is a parent-pointer representation. Reverse it: for each employee i, add i to children[manager[i]]. This gives us direct reports for each manager — the adjacency list we need for DFS.
2 DFS from headID with time=0: The head receives the message at minute 0. Call dfs(headID, 0). The second argument is the accumulated time — the minute at which this node has been informed.
3 Update global maximum at every node: At each call dfs(node, time), run ans = max(ans, time). This works because every employee — leaf or not — is someone being informed, and the maximum time across all employees is the answer.
4 Recurse into children: For each direct report c of node, call dfs(c, time + informTime[node]). The child receives the message only after node has had informTime[node] minutes to relay it.
5 Return ans: After DFS completes, ans holds the maximum accumulated time at any node — the time when the last employee gets informed.

🔑 Alternative: Bottom-Up DFS

Instead of passing accumulated time down, each call returns the max time within its subtree: return informTime[node] + max(dfs(c) for c in children[node], default=0). Both are O(n). The top-down version (tracking ans globally) is easier to explain in interviews; the bottom-up version is more compositional and handles the n=1 edge case cleanly via default=0.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every call dfs(node, t), the value t equals exactly the minute at which node receives the message. This is maintained because: the head hears at t=0, and every edge from parent to child adds the parent's informTime, which is the delay before the child hears. The sum along any root-to-node path equals when that node is informed.

Why the maximum gives the answer: Since all branches relay information in parallel (a manager tells all direct reports simultaneously), the last employee to be informed is the one on the path with the highest accumulated sum. Taking the maximum over all nodes correctly identifies this critical path.

Why DFS (not BFS) for weighted trees: BFS processes nodes level by level, which works for unweighted trees where each level = 1 unit of time. Here, each edge has a different weight (informTime). Level ≠ time. DFS with the accumulated-time parameter handles this naturally — the parameter carries the exact inform-time at each node, regardless of depth.

⚠️ Common Trap: Looking for the Deepest Node

The deepest node (maximum depth) is not necessarily the last to be informed. Consider a chain of depth 10 where every informTime=0 vs. a depth-2 tree where both edges have informTime=100. Maximum depth = 10, but maximum time = 200 (from the shallow tree). Always track accumulated time, not depth.

Edge case: n=1: children[headID] is empty. DFS sets ans = max(0, 0) = 0 and terminates. Correct — the head is already informed, nobody else needs to be told.

4 The Code (Annotated) ▾

Solution 1 — DFS with Accumulated Time (Interview Pick)

def numOfMinutes(n, headID, manager, informTime):
from collections import defaultdict
children = defaultdict(list) ← manager → [direct reports]
for emp, mgr in enumerate(manager):
if mgr != -1: ← skip the head (manager[head] = -1)
children[mgr].append(emp) ← invert parent[] to children[]
ans = 0 ← track maximum inform time seen
def dfs(node, time): ← time = minute node received the message
nonlocal ans
ans = max(ans, time) ← every node is a candidate for the answer
for child in children[node]: ← relay to all direct reports
dfs(child, time + informTime[node]) ← child hears after node's delay
dfs(headID, 0) ← head hears at time 0
return ans
⚠️ Why update ans at EVERY node, not just leaves?

Leaf employees (those with no direct reports) still receive the message and count as "informed." By updating ans at every node visited, we capture the time for both intermediate managers and leaf employees. Since accumulated time only increases as we go deeper, leaves always have the largest time in their branch — but updating everywhere is safe and correct.

Solution 2 — Bottom-Up DFS (return max from subtree)

def numOfMinutes(n, headID, manager, informTime):
from collections import defaultdict
children = defaultdict(list) ← same tree construction
for emp, mgr in enumerate(manager):
if mgr != -1: children[mgr].append(emp)
def dfs(node): ← returns max time in node's subtree
if not children[node]: ← leaf: contributes no additional relay time
return 0
return informTime[node] + max(dfs(c) for c in children[node])
return dfs(headID) ← root call gives the total answer
💡 Complexity Summary

Both DFS solutions: Time O(n) — every employee visited exactly once.
Space O(n) — adjacency list O(n) + recursion stack O(n) worst case (linear chain of managers).
Iterative (BFS/DFS with stack): Same O(n)/O(n) but avoids Python's recursion limit for very deep trees.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Data Structure

The manager[] array is a parent-pointer array — the hallmark of a rooted tree. "Inform all employees through managers" = propagate from root to leaves. Signal words: "manager relationship", "head of company", "inform all" → tree traversal from root.

Step 2: Recognize Maximum Path Sum

When is the last employee informed? On the slowest branch. The question becomes: find the root-to-leaf path with maximum weighted sum. This is a classic DFS pattern — explore all paths, track the maximum.

Step 3: Build the Tree (Invert manager[])

We need to traverse children, not parents. Invert: children[manager[i]].append(i) for all i where manager[i] != -1. This one-liner converts the parent-pointer representation to an edge-list tree.

Step 4: DFS with Accumulated Time

Pass time as a parameter representing "when this node was informed." At each child call, add informTime[parent]. Track the global maximum. Alternatively, return max from subtree bottom-up — both are correct, choose whichever is cleaner under interview pressure.

🗣️ What to Say to the Interviewer

"The manager[] array defines a rooted tree — this is a tree path sum problem.
I need the maximum weighted path from root to any node, where edge weights are informTime values.
Step 1: Invert manager[] to get a children adjacency list — O(n).
Step 2: DFS from headID, passing accumulated time as a parameter.
At each node: update global max. For each child: recurse with time + informTime[node].
Time O(n), Space O(n). Edge case n=1: returns 0."
🔮 Likely Follow-up Questions
  • "Can you do it iteratively?" → Use an explicit stack with (node, accumulated_time) tuples
  • "What if there are multiple heads?" → Create a virtual root node pointing to all heads with informTime=0
  • "What if informTime values can be negative?" → The problem guarantees non-negative, but if negative: DFS still works, max logic handles it
  • "How to find the slowest branch?" → Track path alongside time during DFS

7 Summary & Key Takeaways ▾
The problem is: find the maximum weighted root-to-any-node path sum in a tree
First invert manager[] (parent-pointer) to children[] (adjacency list) — this enables forward DFS
DFS passes accumulated time as a parameter — each child gets parent's time + parent's informTime
Update the global maximum at every node visited, not just leaves
Branches inform in parallel — the answer is always the maximum path sum, never the sum of all paths
ApproachTimeSpaceNotes
Brute Force: trace each employee's path to root O(n²) O(n) Recomputes shared path prefixes repeatedly
Top-Down DFS — Accumulated Time ✓ O(n) O(n) Interview pick; intuitive; single global max
Bottom-Up DFS — Return subtree max O(n) O(n) Same complexity; cleaner for recursion purists
Iterative BFS/DFS with (node, time) pairs O(n) O(n) No recursion limit risk for deep trees
🎯 When to Recognize the DFS from Root Pattern
  • "Propagate information/infection/value from a root node through a tree"
  • "Find maximum/minimum time for all nodes to receive a message"
  • Manager-employee hierarchy → always a rooted tree; invert to get children
  • Edge weights differ → cannot use BFS level = cost; use DFS with accumulated cost
  • Contrast with unweighted: if all informTime=1, BFS levels would work
