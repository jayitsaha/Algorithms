Find All Numbers Disappeared in an Array

LeetCode 448 — Use the array itself as a marker board
Easy Hash Tables Index Marking O(n) Time O(1) Space*
1 Understanding the Problem ▼

Given an array nums of n integers where each nums[i] is in the range [1, n], return all integers in [1, n] that do not appear in the array.

Let's walk through an example:

nums = [4, 3, 2, 7, 8, 2, 3, 1]
n = 8, so we expect numbers 1 through 8
Present: 1, 2, 2, 3, 3, 4, 7, 8
Missing: 5, 6
💡 Key Insight

Since all values are in range [1, n] and the array has exactly n slots, we can use the array indices as a "presence board". Each value v can mark index v-1 to indicate "I exist!"

🔐 The Constraint That Enables the Trick

Values in [1, n] map perfectly to indices [0, n-1]. This 1:1 correspondence lets us use the array itself as a hash set with O(1) space.

2 The Algorithm: Index Marking ▼

Instead of using extra space for a hash set, we mark visited indices by negating the value at that position. After marking, any index with a positive value was never visited — its corresponding number is missing.

1Marking Phase: For each value v, compute target index idx = |v| - 1. If nums[idx] is positive, negate it to mark "number |v| exists".
2Collection Phase: Scan the array. Any index i where nums[i] > 0 means number i + 1 never appeared. Add it to the result.

Value-to-Index Mapping:

1→index 0(1 - 1 = 0)
2→index 1(2 - 1 = 1)
...→...
n→index n-1(n - 1 = n-1)
⚠️ Why Use Absolute Value?

When we encounter a value, the cell might already be negative (marked by a previous iteration). We use |v| to get the original value regardless of sign.

3 Why It Works: The Invariant ▼
🔐 The Invariant

After the marking phase: nums[i] < 0 if and only if the number i + 1 exists somewhere in the original array.

Why This Guarantees Correctness

  • Existence ⟹ Negative: If number v exists, we find it during iteration and mark index v-1 negative.
  • Negative ⟹ Existence: If index i is negative, some iteration must have found value i+1 and marked it.
  • Positive ⟹ Missing: If nums[i] > 0 after marking, no value i+1 was ever encountered — it's missing!

Edge Cases

  • Duplicates: Marking the same index twice is harmless — already negative stays negative.
  • All same values: [1,1,1] → marks index 0 three times, indices 1,2 stay positive → missing: 2,3.
  • Complete array: [1,2,3] → all indices marked → result is empty.
4 The Code (Annotated) ▼

Solution — Index Marking (Interview Pick)

# Two-pass index marking approach
def findDisappearedNumbers(nums):
# Phase 1: Mark indices corresponding to present numbers
for num in nums:
idx = abs(num) - 1 # Map value to index
if nums[idx] > 0: # Only negate if positive
nums[idx] = -nums[idx] # Mark as "seen"
# Phase 2: Collect indices that were never marked
result = []
for i in range(len(nums)):
if nums[i] > 0: # Still positive = never marked
result.append(i + 1) # Convert index back to number
return result

Alternative — Hash Set (Simpler but O(n) Space)

def findDisappearedNumbers(nums):
present = set(nums)
return [i for i in range(1, len(nums) + 1) if i not in present]
⚠️ Common Mistake

Forgetting to use abs(num). On second encounter of a value, the cell might already be negative, so num - 1 would give a wrong (negative) index!

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Recognize the Constraint

Values in [1, n] with array size n is a huge hint. This bijection between values and indices enables in-place tricks.

Step 2: Start with O(n) Space Solution

Mention the hash set approach first: "I could use a set to track present numbers, then find missing ones in O(n) time and space."

Step 3: Optimize to O(1) Space

"Since values map to indices, I can use the array itself as a marker. Negating values marks presence without losing information."

Step 4: Handle the Gotcha

"I need absolute value because a cell might already be marked negative when I read it."

🗣️ What to Say to the Interviewer

"The key observation is that values are bounded by [1, n], which maps perfectly to indices [0, n-1]. Instead of using extra space for a hash set, I can mark presence by negating values at corresponding indices. After one pass of marking, any index with a positive value indicates its number is missing. This gives O(n) time with O(1) auxiliary space."

7 Summary & Key Takeaways ▼
✓ Values in [1, n] can be used as indices into an array of size n.
✓ Negating marks presence without losing the original value.
✓ Always use abs() when reading a potentially-marked value.
✓ Positive values after marking reveal which numbers are missing.
🎯 When to Use This Pattern
  • Values bounded by [1, n] or [0, n-1] with array size n
  • Need O(1) auxiliary space
  • Finding missing/duplicate numbers
  • Can modify the input array (or restore afterward)
