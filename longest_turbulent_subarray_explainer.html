LeetCode #978

Longest Turbulent Subarray

Find the length of the maximum-size turbulent subarray — one whose consecutive differences strictly alternate between positive and negative.

Medium Kadane's Algorithm Two-State Kadane Time: O(n) Space: O(1)
1 Understanding the Problem ▾

A subarray is turbulent if the sign of each consecutive pair's comparison strictly alternates. In other words, consecutive differences must go: positive, negative, positive, negative… (or start the other way). Equal adjacent elements are forbidden — they produce a zero difference and break the alternation.

💡 Key Insight

Only three things can happen between arr[i-1] and arr[i]: the step goes up (+), down (−), or stays equal (0). Turbulence requires + and − to alternate with no zeros. So we can reduce the problem to tracking the sign of each step and asking: "how long can this alternating sign pattern continue?"

✓ Turbulent: [9, 4, 7, 2, 10]

9 → 4: ↓ (sign −)
4 → 7: ↑ (sign +)
7 → 2: ↓ (sign −)
2 → 10: ↑ (sign +)
Pattern: − + − + → alternating!

✗ Not Turbulent: [4, 8, 8, 16]

4 → 8: ↑ (sign +)
8 → 8: = (sign 0)
8 → 16: ↑ (sign +)
Pattern: + 0 + → zero breaks it!

What we return: the length (number of elements, not differences) of the longest turbulent subarray. A single element always qualifies — it vacuously satisfies the condition. The answer is always at least 1.

Edge cases to watch: all equal elements (answer = 1), monotonically increasing/decreasing (answer = 2 since any pair is turbulent), length 1 array (answer = 1).

2 The Algorithm: Two-State Kadane ▾

Classic Kadane's algorithm tracks a single value: the length of the best valid subarray ending at the current position. For turbulent subarrays, we need two such values because the extension rule depends on what direction the current position ends in.

Two-State Kadane — State Machine
↑
inc
Longest turbulent window ending with an upward step
arr[i] > arr[i-1]:
→
inc = dec+1, dec=1
⇅
arr[i] < arr[i-1]:
→
dec = inc+1, inc=1
↓
dec
Longest turbulent window ending with a downward step

Equal elements reset both states to 1. After each element: ans = max(ans, inc, dec)

1 Initialize: inc = dec = 1. Every single element is a turbulent subarray of length 1, so both states start at 1. Set ans = 1.
2 Loop i from 1 to n-1: Compare arr[i] with arr[i-1]. The old values of inc and dec are read simultaneously before any update.
3 Step UP (arr[i] > arr[i-1]): An upward step can continue any subarray that was last heading downward. So new_inc = old_dec + 1. The dec-state resets to 1 (can't extend a downward run when just going up).
4 Step DOWN (arr[i] < arr[i-1]): Symmetric. new_dec = old_inc + 1; inc resets to 1.
5 Equal (arr[i] == arr[i-1]): A zero-sign step breaks any turbulent run. Both inc and dec reset to 1.
6 Update answer: ans = max(ans, inc, dec). The best turbulent subarray ending here could end in either state, so we check both.
3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing index i:
  — inc = length of the longest turbulent subarray that ends at index i with the last step going up (i.e., arr[i] > arr[i-1]).
  — dec = length of the longest turbulent subarray that ends at index i with the last step going down (i.e., arr[i] < arr[i-1]).
  — If neither applies (i=0 or equal), both are 1.

Why the transitions preserve the invariant: Suppose at index i-1, dec = k — meaning there's a valid turbulent subarray of length k ending at i-1 with its last step going down. If arr[i] > arr[i-1] (step up), we can append this new up-step. The resulting subarray of length k+1 ends at index i with an up-step, and it's still turbulent because an up-step after a down-step satisfies the alternating condition. So inc = dec + 1 is correct.

Why we can't extend a same-direction run: If inc = k (last step up) and we encounter another up-step, the last two differences are both positive — violating turbulence. We cannot extend inc with another up-step. So we must reset inc to 1 (just the new element alone).

⚠️ The Critical "Old Values" Bug

When an up-step occurs, we need inc = old_dec + 1 and dec = 1. If you write this as two separate assignments, you must save old_dec first — otherwise updating dec first corrupts the value you need for inc. Python's tuple unpacking solves this elegantly: inc, dec = dec + 1, 1 evaluates both right-hand sides using the current (old) values before assigning either.

Why the answer is correct: The globally longest turbulent subarray must end at some index i, in either an up-step or a down-step state. Since we take max(ans, inc, dec) after every element, we never miss it. The answer could be 1 (single element) if the entire array is monotonically equal.

4 The Code (Annotated) ▾
def maxTurbulenceSize(arr): ← accepts a list of integers
n = len(arr) ← cache length to avoid repeated calls
if n < 2: return n ← 0 or 1 elements: answer is n itself
inc = dec = 1 ← both states start at 1 (single element)
ans = 1 ← global best; always at least 1
for i in range(1, n): ← scan from index 1 (need i and i-1)
if arr[i] > arr[i-1]: ← step goes UP
inc, dec = dec + 1, 1 ← inc extends old dec; dec resets (tuple trick!)
elif arr[i] < arr[i-1]: ← step goes DOWN
inc, dec = 1, inc + 1 ← dec extends old inc; inc resets
else: ← equal: zero sign, turbulence broken
inc = dec = 1 ← full reset; new window starts fresh
ans = max(ans, inc, dec) ← best ending here in either state
return ans ← global best turbulent subarray length
💡 The Python Tuple Trick — Why It Matters

Python evaluates the entire right-hand side of a, b = expr1, expr2 using the current values of all variables before assigning either. So inc, dec = dec + 1, 1 uses the old dec for the new inc, even though dec is being set to 1 simultaneously. This is the idiomatic Python way to do a simultaneous swap/update — no temp variable needed.

⚠️ Why Brute Force is O(n²) and Fails

A naive approach tries every subarray (i, j) and checks if it's turbulent: two nested loops plus a scan — that's O(n³) or O(n²) with optimization. With n up to 40,000, even O(n²) = 1.6 × 10⁹ operations. Two-State Kadane does a single pass: O(n) time, O(1) space — it's the only realistic approach at interview scale.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize "Longest Subarray" = Kadane's

Whenever you see "longest subarray satisfying some local condition," think Kadane's algorithm. The condition here — consecutive differences alternate sign — is purely local (depends only on adjacent pairs), which is exactly the kind of condition Kadane's handles.

Step 2: Notice the Condition is Direction-Dependent

Standard Kadane's tracks one quantity (max subarray sum ending here). Here, the extension rule depends on the direction of the current step. An up-step can only extend a down-ending subarray, and vice versa. This tells you to split into two states: inc and dec.

Step 3: Derive the Transitions by Logic

For each step direction (up / down / equal), think: "Which state can be extended?" If up: only dec-ending subarrays can be extended (because up after down = alternating). If equal: neither — turbulence breaks. Write out the three cases, mention the tuple trick to avoid the old-value bug.

Step 4: Verify on a Small Example

Walk through [9, 4, 7, 2] on the whiteboard. Steps: ↓ ↑ ↓. After each: (inc=1,dec=2), (inc=3,dec=1), (inc=1,dec=4). ans=4. Confirm by inspection that [9,4,7,2] is indeed turbulent.

🗣️ What to Say to the Interviewer

"The key observation: turbulence only depends on the sign of consecutive differences. A turbulent window ending at position i must end in either an up-step or a down-step. An up-step can extend any down-ending window, and vice versa. So I'll use a Kadane's variant with two states: inc = best turbulent length ending at i with an up-step dec = best turbulent length ending at i with a down-step Transitions: up: inc, dec = dec+1, 1 down: inc, dec = 1, inc+1 equal: inc = dec = 1 I use tuple unpacking to avoid the old-value bug. Update ans = max(ans, inc, dec) at each step. Time O(n), Space O(1)."
🎯 Follow-Up Questions to Expect
  • "Return the actual subarray, not its length?" — track a start pointer; update it when both states reset.
  • "Sliding window approach?" — maintain explicit left pointer; advance on turbulence break.
  • "Can you do it with one variable instead of two?" — track prev_sign and a single length; reset on sign repeat or zero.
  • "What about floating point input?" — same algorithm; comparisons still yield +/−/0 signs.

7 Summary & Key Takeaways ▾
Two-State Kadane: maintain inc (best turbulent length ending in an up-step) and dec (ending in a down-step).
An up-step extends a dec-ending window; a down-step extends an inc-ending window; equal resets both to 1.
Python tuple unpacking (inc, dec = dec+1, 1) evaluates both right-hand sides before assigning — elegant solution to the old-value bug.
Update ans = max(ans, inc, dec) after every element — the global best might end in either state.
O(n) time, O(1) space. A single scan with just four scalar variables.

Brute Force (check all subarrays) O(n²) O(1) Two loops + scan; TLE on large inputs

🎯 When to Use Two-State (or Multi-State) Kadane
  • Longest subarray with an alternating-sign local condition
  • Extension rule depends on the type of the last step, not just the length
  • Any "state machine Kadane" where there are 2–4 named states and steps transition between them
  • Classic examples: Maximum Product Subarray (max/min states), Stock with Transaction Fee (hold/cash states)
