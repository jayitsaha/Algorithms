LeetCode #2699

Minimum Cost Path with Edge Reversals

Find the minimum number of edge reversals needed to reach every node from node 0 in a directed graph.

Medium Graph 0-1 BFS Time: O(V + E) Space: O(V + E)
1 Understanding the Problem ▾

You are given n nodes (labeled 0 to n-1) and a list of directed edges. You can traverse any edge in its original direction for free. But if you need to go against the direction of an edge (i.e., reverse it), you pay a cost of 1.

The goal: for every node v, find the minimum number of edge reversals needed to reach v from node 0.

💡 Key Insight

This is a shortest-path problem on a graph with only two edge weights: 0 and 1. Going along an original edge costs 0; reversing an edge costs 1. When edge weights are binary (0 or 1), we can skip the min-heap entirely and use a deque-based BFS that runs in linear time O(V + E).

Example Graph (n=4, directed edges)

0 → 1 (original, cost 0)
1 → 2 (original, cost 0)
0 → 3 (original, cost 0)

dist[0] = 0, dist[1] = 0
dist[2] = 0, dist[3] = 0

Tricky case: edges pointing wrong way

2 → 1 (must reverse to get 1→2)
3 → 0 (must reverse to get 0→3)

dist[0] = 0, dist[1] = 1
dist[2] = 2, dist[3] = 1

Key edge cases: Node 0 always has cost 0. A node reachable via all forward edges has cost 0. A node behind k reversal-only edges has cost k. Disconnected nodes get cost ∞ (not reachable).

🔑 The Modeling Trick

For each undirected edge view (u, v), add two directed edges in the adjacency list: u → v with cost 0 (original direction) and v → u with cost 1 (reversal direction). Now any path in the resulting graph automatically encodes the number of reversals in its total weight. Shortest path on this graph = minimum reversals.

2 The Algorithm: 0-1 BFS with a Deque ▾

Standard BFS works for unweighted graphs. Dijkstra works for non-negative weighted graphs. When all weights are either 0 or 1, there is a much faster approach: 0-1 BFS. It uses a deque (double-ended queue) and achieves O(V + E) — the speed of BFS, with the correctness of Dijkstra.

Dijkstra (over-powered here)

Uses a min-heap (priority queue). Every push/pop is O(log V). Total: O((V + E) log V). Correct but wasteful when weights are only 0 or 1.

0-1 BFS (just right)

Uses a deque. Cost-0 neighbors go to the front, cost-1 neighbors go to the back. Processes nodes in non-decreasing distance order. Total: O(V + E).

1 Build the graph: For each directed edge (u, v), add two entries to the adjacency list: adj[u].append((v, 0)) for free traversal and adj[v].append((u, 1)) for paid reversal. This doubles the edge count but stays O(E).
2 Initialize distances: Set dist[0] = 0 and dist[i] = ∞ for all other nodes. Push node 0 into the deque.
3 0-1 BFS loop: While the deque is not empty, pop from the front. For each neighbor (v, w) of current node u: if dist[u] + w < dist[v], update dist[v]. Then push v to the front if w == 0, or to the back if w == 1.
4 Return distances: After the deque is empty, dist contains the minimum number of reversals to reach every node from node 0.
🔑 Why Front vs Back?

The deque stays sorted by distance at all times. Cost-0 neighbors are as close as the current node — they belong at the front (same distance level). Cost-1 neighbors are one step further — they belong at the back. This invariant mirrors Dijkstra's priority queue, but in O(1) per operation instead of O(log n).

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any moment during 0-1 BFS, the deque contains nodes at at most two distinct distances: the current minimum distance d at the front, and d + 1 at the back. When we pop from the front, we always process the globally nearest unvisited node first — exactly like Dijkstra, but without a heap.

Why this suffices: Since the only edge weights are 0 and 1, after processing all nodes at distance d (front of deque), the front of the deque will contain distance d+1 nodes. Cost-0 edges can never push a node further than the current front. Cost-1 edges push to the back of what are currently distance-d nodes, landing at distance d+1. The deque is self-sorting for this binary case.

Why the modeling trick is correct: By representing each original directed edge as two adjacency entries (one free, one paid-reverse), every path from source to any node in the modified graph corresponds 1-to-1 with a valid traversal in the original graph. The path weight equals exactly the number of reversals used. The shortest such path is therefore the minimum-reversal answer.

⚠️ Common Trap: Revisiting Settled Nodes

Unlike BFS (where each node is visited at most once), 0-1 BFS may push the same node into the deque multiple times — once when first discovered and again if a cheaper path is found later. Guard with the standard Dijkstra-style check: skip popped nodes if dist[u] is already optimal (i.e., the stored distance is less than what we'd compute). The if dist[u] + w < dist[v] condition does exactly this.

Edge case — disconnected nodes: If a node is not reachable from node 0 (even after adding reversal edges), its distance remains ∞. The problem guarantees a valid answer exists for reachable nodes.

4 The Code (Annotated) ▾

Solution 1 — 0-1 BFS (Optimal, O(V + E))

from collections import deque ← deque supports O(1) appendleft and append
def minEdgeReversals(n, edges):
adj = [[] for _ in range(n)] ← adjacency list: adj[u] = [(v, cost), ...]
for u, v in edges: ← process each directed edge (u→v)
adj[u].append((v, 0)) ← original direction: free (cost 0)
adj[v].append((u, 1)) ← reverse direction: pay 1 reversal
dist = [float('inf')] * n ← all nodes start at infinite cost
dist[0] = 0 ← source node: 0 reversals needed
dq = deque([0]) ← start BFS from node 0
while dq: ← process until deque is empty
u = dq.popleft() ← always pop from front (smallest distance)
for v, w in adj[u]: ← explore all neighbors of u
if dist[u] + w < dist[v]: ← found a cheaper path to v?
dist[v] = dist[u] + w ← update to the cheaper cost
if w == 0: ← free edge: v is at same distance level
dq.appendleft(v) ← push to front (process next!)
else: ← cost-1 edge: v is one level further
dq.append(v) ← push to back (process later)
return dist ← dist[i] = min reversals to reach node i
⚠️ Why Two Entries per Edge (not just one)?

We add adj[v].append((u, 1)) in addition to adj[u].append((v, 0)). Without the reverse entry, we could never "go backwards" against an edge, even if reversing it yields a shorter path. The directed graph only stores one direction, but traversal physically requires both. The reversal entry models the cost of "flipping" that edge during traversal.

Solution 2 — Dijkstra (correct but suboptimal, O((V+E) log V))

import heapq
def minEdgeReversalsDijkstra(n, edges):
adj = [[] for _ in range(n)]
for u, v in edges:
adj[u].append((v, 0))
adj[v].append((u, 1))
dist = [float('inf')] * n
dist[0] = 0
heap = [(0, 0)] ← (cost, node)
while heap:
d, u = heapq.heappop(heap) ← always pops the globally cheapest node
if d > dist[u]: continue ← stale entry: cheaper path already found
for v, w in adj[u]:
if d + w < dist[v]:
dist[v] = d + w
heapq.heappush(heap, (dist[v], v))
return dist
💡 Complexity Summary

0-1 BFS (Optimal): Time O(V + E), Space O(V + E). Each node and edge processed at most twice (original + reverse entry).
Dijkstra: Time O((V + E) log V), Space O(V + E). Correct, but uses heap where deque suffices.
Brute-force DFS/Bellman-Ford: Time O(V × E) — technically correct but far too slow.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern (Graph + Shortest Path)

"We need minimum cost to reach every node — that signals shortest-path. The 'cost' of each move is either 0 (forward) or 1 (backward). Binary edge weights."

Step 2: Model the Problem

"I'll model this by adding two entries per original edge: the original direction at cost 0 and the reverse at cost 1. Now I have a standard weighted graph and need single-source shortest paths from node 0."

Step 3: Propose Dijkstra, then Optimize

"Dijkstra would work — O((V+E) log V). But since weights are only 0 or 1, I can do better. 0-1 BFS uses a deque: cost-0 neighbors go to the front, cost-1 to the back. This maintains the same correctness as Dijkstra in O(V+E) — linear time."

Step 4: Code it up cleanly

Walk through: build adj list with both edge directions → init distances to ∞ except dist[0]=0 → deque BFS with front/back rule → return dist array.

🗣️ What to Say to the Interviewer

"This is a shortest-path problem. The cost to reach each node is the minimum number of edge reversals needed.
I'll model it: for each directed edge u→v, add (v, cost=0) to adj[u] — free traversal in original direction —
and (u, cost=1) to adj[v] — pay 1 to traverse it backwards.
Now I have a graph with only 0/1 edge weights. That's 0-1 BFS territory: use a deque,

🔮 Likely Follow-up Questions
  • "What if edge costs aren't 0 and 1, but arbitrary?" → Then use standard Dijkstra with min-heap, O((V+E) log V).
  • "What if costs can be negative?" → Bellman-Ford, O(V×E).
  • "Can you find the minimum-reversal path itself, not just the cost?" → Track parent pointers in the dist update step.
  • "What if we want to minimize reversals only for one target node?" → Same algorithm, stop early when target is settled.

def minEdgeReversals(n, edges):
adj = [[] for _ in range(n)]
for u, v in edges:
adj[u].append((v, 0)); adj[v].append((u, 1))
dist = [float('inf')] * n
dist[0] = 0
dq = deque([0])
while dq:
u = dq.popleft()
for v, w in adj[u]:
if dist[u] + w < dist[v]:
dist[v] = dist[u] + w
if w == 0: dq.appendleft(v)
else: dq.append(v)
return dist

7 Summary & Key Takeaways ▾
Model directed edges as two adjacency entries: (v, 0) for original direction and (u, 1) for reversal — this is the core trick
0-1 BFS = Dijkstra's correctness at BFS speed — use a deque, push cost-0 to front, cost-1 to back
The deque invariant: at any moment, it holds nodes at distances d and d+1 only (no heap needed)
O(V + E) time and space — linear, same as unweighted BFS
Recognize 0-1 BFS whenever edge weights are restricted to exactly {0, 1}

🎯 When to Recognize the 0-1 BFS Pattern
  • "Minimum number of [changes/flips/reversals] to reach target" → Binary cost → 0-1 BFS
  • Grid where each cell has an arrow/direction and you can change it → Cost 0 to follow, cost 1 to change → 0-1 BFS
  • Graph shortest path but all weights are 0 or 1 → 0-1 BFS beats Dijkstra
  • Any "walk for free, but pay 1 to override" problem
