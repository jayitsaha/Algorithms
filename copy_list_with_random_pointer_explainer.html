LeetCode #138

Copy List with Random Pointer

A linked list of nodes each with a next pointer and a random pointer that can point to any node or null. Return a deep copy of the entire list.

Medium Linked List Hash Map Time: O(n) Space: O(n)
1 Understanding the Problem ▾

You have a special linked list where every node carries two pointers instead of one. The next pointer is the usual forward link to the next node. The random pointer can point to any node in the list, or to null — it is completely arbitrary.

Your goal: produce a deep copy — an entirely new linked list made of new node objects, with the same values and the same structure of next and random pointers, but referencing only new nodes (not originals).

💡 Key Insight

Copying next is straightforward — it's sequential. The hard part is random: when you create the copy of node A and wire A'.random, its target might be a node you haven't created yet. You need O(1) lookup from "original node → its copy." That's exactly what a hash map gives you.

✗ Wrong: Shallow Copy

Copy nodes reference original nodes.
Modifying copy leaks into original.
Not an independent deep copy.

✓ Correct: Deep Copy

Brand-new node objects only.
copy[i].next = new copy of orig[i].next
copy[i].random = new copy of orig[i].random

Edge cases: empty list (head is null → return null), single node with random pointing to itself, a node whose random equals its next, multiple nodes with random pointing to the same target.

2 The Algorithm: Two-Pass Hash Map ▾

We use a hash map as a "node registry" — it maps every original node to its newly created copy. We then wire up pointers in a second pass when all copies are guaranteed to exist.

1 Edge case: If head is null, return null immediately.
2 Pass 1 — Create all copies: Walk the original list. For each node, create a brand new node with the same value. Register it: old_to_new[orig] = copy_node. At this stage, copy nodes have no next or random — just their values.
3 Pass 2 — Wire all pointers: Walk the original list again. For each original node n, look up its copy. Set copy.next = old_to_new[n.next] (if n.next is not null). Set copy.random = old_to_new[n.random] (if n.random is not null).
4 Return: old_to_new[head] — the copy of the original head is the head of the deep-copied list.
🔑 Why Two Passes?

In a single pass, when you process node A and its random points to node Z that hasn't been processed yet, node Z's copy doesn't exist — you'd have nothing to wire. By separating "create all copies" from "wire all pointers," Pass 2 is guaranteed that every lookup succeeds because Pass 1 already created every copy.

There is also an O(1) space interleaving approach: insert each copy node immediately after its original in the list (A → A' → B → B' → ...), set randoms using the invariant that orig.random.next is the copy of the random target, then split the two lists apart. It avoids the hash map but requires careful pointer surgery in three passes.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After Pass 1: old_to_new[x] exists for every node x reachable from head. Each value is a freshly created node with the correct value and null pointers. This means in Pass 2 every lookup is guaranteed to find a valid copy node.

Why null is handled safely: When curr.next or curr.random is null, we simply skip the lookup — no old_to_new[None] KeyError, and the copy's pointer defaults to null, which is correct.

Why this is truly a deep copy: The original nodes appear only as keys in the map — they are lookup handles, never included in the result. Every node in the returned list is a new object. Copy nodes only reference other copy nodes via wired next/random pointers.

⚠️ Classic Bug: Mixing Original and Copy Nodes

A subtle error: in Pass 2, accidentally writing copy.next = orig.next instead of copy.next = old_to_new[orig.next]. The copy's next now points to an original node — breaking the deep copy. Always look up through the map.

Why the interleaving trick works: After interleaving, every original node's copy is one step ahead: orig.next is orig_copy. So orig.random.next is the copy of orig.random — O(1) lookup without a hash map, using the list structure itself as the registry.

4 The Code (Annotated) ▾

Solution 1 — Two-Pass Hash Map (Interview Pick, O(n) time / O(n) space)

def copyRandomList(head): ← Entry point
if not head: ← Guard: empty list → return null
return None
old_to_new = {} ← Registry: original → copy node
curr = head ← Pass 1 pointer
while curr: ← Walk entire original list
old_to_new[curr] = Node(curr.val) ← Create copy with same value only
curr = curr.next
curr = head ← Reset for Pass 2
while curr: ← Walk again to wire pointers
if curr.next: ← Avoid KeyError: don't look up None
old_to_new[curr].next = old_to_new[curr.next] ← Map orig.next → copy.next
if curr.random: ← random can also be null
old_to_new[curr].random = old_to_new[curr.random] ← Map orig.random → copy.random
curr = curr.next
return old_to_new[head] ← Head of the deep-copied list
⚠️ Why guard with "if curr.next" before looking up?

If curr.next is null, old_to_new[None] raises a KeyError. The guard skips the assignment — the copy node's next stays null by default, which is exactly correct. Same logic for curr.random.

Solution 2 — O(1) Space: Interleaving (3 passes)

def copyRandomList(head):
if not head: return None
# Pass 1: interleave A → A' → B → B' → ...
curr = head
while curr:
copy = Node(curr.val) ← Create copy
copy.next = curr.next ← Copy's next = orig's old next
curr.next = copy ← Insert copy after orig
curr = copy.next ← Advance to next original
# Pass 2: set randoms — copy.random = orig.random.next
curr = head
while curr:
if curr.random:
curr.next.random = curr.random.next ← Trick: random's copy = random.next
curr = curr.next.next ← Skip over copy to next original
# Pass 3: separate the two interleaved lists
new_head = head.next
curr = head
while curr:
orig_next = curr.next.next ← Next original (skipping copy)
curr.next.next = orig_next.next if orig_next else None← Wire copy.next
curr.next = orig_next ← Restore original.next
curr = orig_next
return new_head
💡 Complexity Summary

Two-Pass Hash Map: Time O(n), Space O(n) — 2 clean passes, easiest to reason about
Interleaving: Time O(n), Space O(1) — optimal space, 3 passes, trickier to implement correctly

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Name the Core Difficulty

Say: "Copying next is trivial — sequential. The problem is random: when creating a copy of node A, A's random might point to a node I haven't created yet. I need to be able to look up any original → its copy in O(1)."

Step 2: Propose the Hash Map Approach

"I'll use a hash map as a registry. Pass 1: create all copy nodes (values only), store old→new. Pass 2: wire next and random by looking up originals in the map, guarding null pointers. O(n) time and space."

Step 3: Offer the O(1) Space Follow-up

"If we need O(1) space, I can use the interleaving trick: insert each copy right after its original, set randoms via orig.random.next, then split the two lists. Same O(n) time, no extra space — but requires three careful passes."

Step 4: Proactively Handle Edge Cases

Mention: "I'll guard null pointers — empty head returns null, and I check if curr.next and if curr.random before each map lookup to avoid KeyError on null."

🗣️ What to Say to the Interviewer

"The challenge is the random pointer — it can skip ahead to an uncopied node.
I'll use a two-pass hash map: old_to_new = {}.
Pass 1: walk list, create Node(curr.val) for each, store old_to_new[curr] = copy.
Pass 2: walk again, if curr.next set copy.next = old_to_new[curr.next], same for random.
Return old_to_new[head]. Time O(n), space O(n).
For O(1) space, interleaving is possible — want me to code that too?"
🔮 Likely Follow-up Questions
  • "Can you do O(1) space?" → Interleaving (3-pass technique above)
  • "What if random points to null?" → Handled — guard with if curr.random
  • "Clone a graph instead of a list?" → Same pattern, BFS/DFS + hash map (#133)
  • "Can you do it recursively?" → Yes, recursive DFS + memoization map

7 Summary & Key Takeaways ▾
The random pointer makes naive copying impossible — it can forward-reference nodes you haven't created yet
Hash map (old→new) is the registry that guarantees O(1) lookup for any pointer target
Two passes are required: Pass 1 creates all nodes, Pass 2 wires all pointers
Guard null pointers before map lookup to avoid KeyError on null next or random
O(1) space is achievable via interleaving (inserting copies between originals), same O(n) time
ApproachTimeSpaceNotes
Two-Pass Hash Map ✓ O(n) O(n) Clearest logic, interview pick

🎯 When to Recognize This Pattern
  • "Deep copy" of a structure with non-sequential references → hash map originals to copies
  • "Clone a graph" → exact same technique (#133)
  • Any pointer that can skip forward to an unprocessed node → two-pass approach
  • When one pass creates nodes another pass needs → separate creation from wiring
