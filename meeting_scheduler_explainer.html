LeetCode #1229

Meeting Scheduler

Given availability slots for two people and a meeting duration, find the earliest common free time window long enough for the meeting.

Medium Intervals Two Pointers on Sorted Time: O(n log n) Space: O(1)
1 Understanding the Problem ▾

You have two people, each with a list of availability slots — time windows when they are free. Each slot is a [start, end] pair representing a half-open interval. You also have a required duration. Find the earliest time window that fits inside both people's availability and is at least duration long. Return [start, start + duration], or [] if no such slot exists.

💡 Key Insight

Two intervals overlap when the later start comes before the earlier end: max(s1, s2) < min(e1, e2). The overlapping region is exactly [max(s1,s2), min(e1,e2)]. You don't need the full overlap — just the first duration-minute window inside it.

✓ Found: A=[10,50], B=[60,70], dur=8

Overlap: max(60,60)=60, min(120,70)=70
Length: 70−60=10 ≥ 8
Answer: [60, 68]

✗ Miss: A=[10,50], B=[60,70]

Overlap: max(10,60)=60, min(50,70)=50
60 > 50 → no overlap!
A ends before B starts → advance A

Edge cases: slots may not be sorted in the input; overlap might exist but be shorter than duration; no overlap at all between the two people; one person has zero slots.

2 The Algorithm: Sort + Two Pointers ▾

After sorting both lists by start time, use two pointers — one into each list — scanning forward. At each step, compute the overlap between the current pair of slots. If it is long enough, return the answer. Otherwise, advance whichever pointer's slot ends sooner — that slot cannot contribute to future overlaps.

1 Sort both lists by start time. This lets us process intervals in chronological order and make a single forward pass through each list.
2 Initialize two pointers: i = 0 into person A's sorted slots, j = 0 into person B's sorted slots.
3 Compute overlap: lo = max(A[i][0], B[j][0]) (later start) and hi = min(A[i][1], B[j][1]) (earlier end). If hi − lo ≥ duration, return [lo, lo + duration].
4 Advance the earlier-ending pointer: whichever slot has the smaller end time is "used up" — it ends before the other slot, so it cannot overlap with any future slot in the other list either.
5 Repeat until one pointer goes out of bounds. If no valid window was found, return [].

At each step — exactly one decision:

Advance i (A ends first)
A[i][1] < B[j][1]
i += 1
or
Advance j (B ends first or tie)
A[i][1] ≥ B[j][1]
j += 1
🔑 Why "advance the one that ends first"?

If A[i] ends at time T, then A[i] cannot overlap with B[j+1], B[j+2], ... if they all start after T. But A[i+1] might — so we keep B[j] and advance A. This is the classic merge-two-sorted-lists pivot: the shorter-lived slot is discarded, never the longer one.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every iteration, all slots before index i in A and before index j in B have been fully considered. No discarded slot can participate in a valid meeting window with any remaining slot from the other list.

Why discarding the earlier-ending slot is safe: Suppose A[i] ends at time T and we advance i. Any future slot B[k] (k > j) starts at some time S ≥ B[j][0]. If B[j][0] > T, then certainly all later B slots also start after T, so A[i] overlaps with none of them. If B[j][0] ≤ T, we already checked the pair (A[i], B[j]). Either way, A[i] is done.

Completeness: Every pair (A[i], B[j]) that could possibly form a valid overlap is visited exactly once — because both pointers only move forward and each pair is checked before either pointer advances past it.

⚠️ Must Sort First

The problem does not guarantee sorted input. If you skip sorting, you might advance past a valid slot because your current comparison looked at a later slot first. Always sort both lists as the first step.

Tie-breaking on end time: If A[i][1] == B[j][1], it doesn't matter which you advance — the overlap (if any) was already checked at this step. The code uses else to advance B on ties, which is equally correct.

4 The Code (Annotated) ▾

Solution 1 — Sort + Two Pointers (Optimal · Interview Pick)

def minAvailableDuration(slots1, slots2, duration):
slots1.sort() ← Sort A by start time (in-place, O(n log n))
slots2.sort() ← Sort B by start time (in-place, O(m log m))
i, j = 0, 0 ← Two independent pointers, start at front
while i < len(slots1) and j < len(slots2):← Both lists must have remaining slots
lo = max(slots1[i][0], slots2[j][0])← Overlap start: the later of the two starts
hi = min(slots1[i][1], slots2[j][1])← Overlap end: the earlier of the two ends
if hi - lo >= duration: ← Overlap fits the meeting → found it!
return [lo, lo + duration] ← Return earliest window, not full overlap
if slots1[i][1] < slots2[j][1]: ← A ends sooner → A[i] is exhausted
i += 1 ← Advance A's pointer
else: ← B ends sooner or tie → B[j] is exhausted
j += 1 ← Advance B's pointer
return [] ← One list exhausted, no valid slot found
⚠️ Return [lo, lo+duration], not [lo, hi]

If the overlap is 30 minutes but the meeting only needs 15, returning [lo, hi] gives them an incorrectly long meeting block. You only schedule the required duration starting at the earliest possible moment lo.

Solution 2 — Brute Force O(n × m) (mention first, then optimize)

def minAvailableDuration_brute(slots1, slots2, duration):
result = None
for a in slots1: ← Try every pair of slots
for b in slots2:
lo = max(a[0], b[0]) ← Overlap start
hi = min(a[1], b[1]) ← Overlap end
if hi - lo >= duration:
if not result or lo < result[0]:← Keep the earliest one
result = [lo, lo + duration]
return result or []
💡 Complexity Summary

Sort + Two Pointers: Time O(n log n + m log m), Space O(1) additional
Brute Force: Time O(n × m), Space O(1)
With 1000 slots each: brute force = 1,000,000 checks vs. two pointers ≈ 2000 checks.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify the Problem

Ask: "Are the slots already sorted?" (Not guaranteed.) "Can slots from the same person overlap?" (Usually no.) "Return the earliest, or any?" (Earliest.) This tells you sorting is necessary and you want the chronologically first answer.

Step 2: State the Brute Force

"The naive approach tries every pair of slots — O(n × m). For each pair, compute the overlap and check its length. Correct, but too slow for large inputs. Can we do better?"

Step 3: Notice the Sorted-List Opportunity

"If I sort both lists, I can scan them with two pointers simultaneously, similar to merging two sorted arrays. The key observation: a slot that ends before the other's current slot even starts can never contribute to any future overlap — I can safely discard it."

Step 4: Code and Verify

Sort → i=j=0 → loop: compute lo/hi → check → advance smaller end → return []. Walk through the example. Verify edge cases: no overlap, all overlaps too short, one list empty.

🗣️ What to Say to the Interviewer

"First I sort both lists by start time — O(n log n + m log m).
Then two pointers i and j. At each step:
lo = max of two starts, hi = min of two ends.
If hi - lo >= duration, return [lo, lo+duration].
Otherwise advance the pointer whose slot ends first —
that slot is expired and can't overlap with anything later.
Total: O(n log n) time, O(1) extra space."
🔮 Likely Follow-up Questions
  • "k people, not just 2?" → Min-heap keyed by (end_time, person_index, slot_index) — pop the earliest-ending slot, check overlap with all current pointers
  • "Return all valid slots?" → Collect every (lo, lo+duration) where overlap ≥ duration; don't return early
  • "What if slots can overlap within one person's list?" → Must merge that person's intervals first (#56 Merge Intervals)
  • "Meeting Rooms I and II?" → Related: #252 (can attend all?) and #253 (min rooms needed)

7 Summary & Key Takeaways ▾
Sort both lists first — two-pointer logic requires sorted order
Overlap of two intervals = [max(starts), min(ends)] — valid only when max(starts) < min(ends)
Always advance the pointer whose slot ends sooner — that slot is exhausted
Return [lo, lo+duration], not [lo, hi] — schedule the meeting, not the full overlap
Each slot is processed at most once → O(n + m) scan after sorting
ApproachTimeSpaceNotes
Brute Force (all pairs) O(n × m) O(1) Simple; mention first in interview

🎯 When to Recognize This Pattern
  • "Find overlap between two sorted interval lists" → Two pointers, advance smaller end

  • k participants instead of 2 → Upgrade to heap-based merge
  • Sorted input (or sortable) + linear-time scan → Two pointers
