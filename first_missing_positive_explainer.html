First Missing Positive

Cyclic Sort / Index Mapping — Use Array as Hash Table

#41 Hard Cyclic Sort / Index Mapping Array Manipulation O(n) Time · O(1) Space
1 Understanding the Problem

Given an unsorted integer array nums, find the smallest positive integer that is NOT in the array. Must run in O(n) time with O(1) auxiliary space.

Example: [3, 4, -1, 1] → Answer: 2

💡 Key Insight: The Answer is Bounded!

For an array of length n, the answer must be in the range [1, n+1].

  • Best case: Array contains all of 1, 2, ..., n → answer is n+1
  • Worst case: Array has no positive integers → answer is 1
  • Why? If any number in [1, n] is missing, that's our answer. Otherwise, it's n+1.

This means we only care about values in [1, n]. Everything else (negatives, zeros, values > n) is irrelevant!

🔑 The Brilliant Trick: Use Array as Hash Table

We can't use extra space for a hash set. But wait — the array itself has n slots!

  • Place value x at index x-1 (if 1 ≤ x ≤ n)
  • After placing, nums[i] should equal i+1 if that number exists
  • Scan for first index where nums[i] ≠ i+1 → that's i+1, the missing number!
2 The Algorithm: Cyclic Sort / Index Mapping

The algorithm has two phases: Placement Phase (put each number in its "home" index) and Scan Phase (find the first violation).

1 Placement Phase: For each position i, while nums[i] is in range [1, n] AND not in its correct spot AND won't create infinite loop — swap nums[i] to its correct index.
2 The "Correct Spot" Rule: Value x should be at index x-1. So nums[i] should go to index nums[i]-1.
3 Why "while" not "if"? After swapping, a new value arrives at position i. We must keep swapping until position i has nothing useful to place.
4 Scan Phase: After all placements, find first i where nums[i] ≠ i+1. Return i+1. If all match, return n+1.

⚠️ Critical: Avoiding Infinite Loops

The swap condition must check nums[i] != nums[target] (not just positions). Why?

If the array has duplicates like [1, 1], swapping positions 0 and 0 forever would loop infinitely. Checking values ensures we stop when the destination already has the correct value.

3 Why It Works: The Invariants
🔐 Invariant 1: Each Swap Fixes One Position

Every swap places at least one element in its correct "home" position. An element is never moved away from its correct position once placed there.

Why O(n) swaps total? Each of the n positions can be "fixed" at most once. So we do at most n swaps across the entire algorithm.

🔐 Invariant 2: After Placement, What's Missing is Revealed

After the placement phase:

  • If value x ∈ [1, n] exists in the array, it's now at index x-1
  • If nums[i] ≠ i+1, then value i+1 was NEVER in the array (or was a duplicate)
  • The first such violation gives us the smallest missing positive
📝 Edge Cases Handled
  • Negatives / Zeros: Ignored (not in [1, n])
  • Values > n: Ignored (can't be placed, answer must be ≤ n+1)
  • Duplicates: One goes to correct spot, others stay (checked via value comparison)
  • All elements present: Every nums[i] = i+1, return n+1
4 The Code (Annotated)
def firstMissingPositive(nums):
n = len(nums)
# Phase 1: Place each value x at index x-1 (if valid)
for i in range(n):
# Keep swapping while nums[i] can be placed correctly
while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
# Swap nums[i] to its "home" index
target = nums[i] - 1
nums[i], nums[target] = nums[target], nums[i]
# Phase 2: Find first index where nums[i] != i+1
for i in range(n):
if nums[i] != i + 1:
return i + 1 # This positive integer is missing
# All 1..n present, missing is n+1
return n + 1

⚠️ Why compare VALUES not indices?

The condition nums[nums[i] - 1] != nums[i] checks if the target position already has the correct value. This handles:

  • Duplicates: Don't swap if destination already correct
  • Self-referential: If nums[i] = i+1 already, no swap needed
5 🎤 Interview Strategy: How to Arrive at This
Step 1: Recognize the Constraints

"O(n) time and O(1) space with an unsorted array... I can't sort (O(n log n)) or use a hash set (O(n) space). What can I use? The array itself as storage!"

Step 2: Bound the Answer

"What's the range of possible answers? For n elements, answer is in [1, n+1]. So I only care about values in [1, n]."

Step 3: Index as Hash

"If I put value x at index x-1, after placement I can scan for the first mismatch. That's Cyclic Sort!"

Step 4: Handle Edge Cases

"Negatives, zeros, values > n → ignore. Duplicates → check value at destination before swapping."

🗣️ What to Say to the Interviewer
"The key insight is that for an array of size n, the answer must be between 1 and n+1. This means I can use the array itself as a hash table — place each value x at index x-1. After placing all valid values, I scan for the first index i where nums[i] ≠ i+1. That tells me i+1 is missing. This gives O(n) time because each element is swapped at most once, and O(1) auxiliary space since I modify in-place."
🔄 Follow-up Questions
  • "What if we can't modify the array?" — Need O(n) space (hash set or boolean array)
  • "What about negative marking instead?" — Alternative: mark presence by negating nums[x-1]
  • "Find ALL missing positives?" — Same approach, collect all violations

7 Summary & Key Takeaways
✓ Answer is always in [1, n+1] — so only care about values in that range
✓ Use array as hash table: place value x at index x-1
✓ While loop for swapping handles chain reactions
✓ Compare VALUES not indices to avoid infinite loops with duplicates
✓ Scan phase finds first violation — that index+1 is the answer
🎯 When to Use Cyclic Sort / Index Mapping
  • Find missing/duplicate in range [1, n] or [0, n-1]
  • O(1) space constraint
  • Values naturally map to indices (pigeonhole principle)
  • "First missing positive" type problems
