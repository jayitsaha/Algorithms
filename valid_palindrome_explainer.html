Valid Palindrome

Skip non-alphanumeric characters, then check if cleaned string reads the same forwards and backwards

Easy String Processing Two Pointers + Skip Non-Alnum Time: O(n) Space: O(1)
1 Understanding the Problem ▲

Given a string s, determine if it is a palindrome after:

  1. Converting all uppercase letters to lowercase
  2. Removing all non-alphanumeric characters (spaces, punctuation, etc.)

A palindrome reads the same forwards and backwards. Examples of valid palindromes: "racecar", "abcba", "a", "" (empty string is trivially a palindrome).

✗ Raw Input
"A man, a plan, a canal: Panama"

Has spaces, commas, colons, uppercase

✓ After Cleaning
"amanaplanacanalpanama"

Only lowercase alphanumeric → palindrome ✓

💡 Key Insight

We don't need to actually build the cleaned string. We can use two pointers starting from each end, skipping non-alphanumeric characters on the fly, and comparing the letters/digits they land on. If all pairs match, it's a palindrome.

⚠️ Edge Cases to Remember
  • Empty string or all non-alphanumeric characters → cleaned string is "" → True (empty is a palindrome)
  • Single alphanumeric character → always True
  • Case insensitive: 'A' == 'a'
  • Digits count: "0P" → cleaned: "0p" → not a palindrome
2 The Algorithm: Two Pointers + Skip Non-Alnum ▲

Two pointers left and right start at opposite ends of the string. They converge toward the centre, skipping non-alphanumeric characters and comparing what they land on.

1 Initialize left = 0, right = len(s) - 1.
2 While left < right: advance left right past any non-alphanumeric chars.
3 While left < right: advance right left past any non-alphanumeric chars.
4 Compare s[left].lower() with s[right].lower(). If they differ → return False.
5 Move pointers inward: left += 1, right -= 1. Repeat from step 2.
6 If pointers cross without mismatch → return True.
ApproachIdeaTimeSpace
Clean then Reverse Filter → new string → compare to reverse O(n) O(n)
Two Pointer In-Place ✓ Skip non-alnum on the fly, compare at pointers O(n) O(1)
💡 Why Opposite-Direction Two Pointers?

A palindrome by definition mirrors around its centre: s[0] == s[n-1], s[1] == s[n-2], etc. Checking these pairs from the outside in naturally matches the problem structure. Each comparison either confirms a mirror pair or immediately reveals a mismatch — no need to look at the whole string if we find a mismatch early.

3 Why It Works: The Invariant ▲
🔐 The Loop Invariant

At the start of each iteration (when left < right and both point at alphanumeric chars), all mirror pairs outside the current window [left, right] have already been verified as equal. If the current pair matches, we shrink the window inward; if not, the string is not a palindrome.

Why Skipping Non-Alphanumeric is Correct

The problem explicitly says to remove non-alphanumeric characters before checking. Our pointer skipping is logically equivalent: we're looking at only the alphanumeric characters in their original relative order, checking them as mirror pairs. Non-alphanumeric characters simply have no role in the palindrome test.

•Empty after cleaning (s = " "): Pointers never satisfy left < right with both on valid chars → return True.
•Single alnum char (s = "a!"): left lands on 'a', right skips '!' back to 'a', then left ≥ right → loop ends → return True.
•Early exit: First mismatch returns False immediately — no need to scan the rest of the string.
4 The Code (Annotated) ▲

Optimal: Two Pointers + Skip Non-Alnum  ← Interview Pick

def isPalindrome(s: str) -> bool:
left, right = 0, len(s) - 1 # start from both ends
while left < right:
# skip non-alphanumeric from the left
while left < right and not s[left].isalnum():
left += 1
# skip non-alphanumeric from the right
while left < right and not s[right].isalnum():
right -= 1
# compare (case-insensitive)
if s[left].lower() != s[right].lower():
return False
left += 1 # matched — move both inward
right -= 1
return True # all mirror pairs matched
⚠️ Why left < right guard inside inner while loops?

If all remaining characters to the right (or left) are non-alphanumeric, the inner loop would run past the opposite pointer without the guard. The guard ensures we don't advance left past right while skipping, which would make the outer comparison meaningless.

Alternative: Clean-then-Compare (Simpler to Read, O(n) Space)

def isPalindrome_clean(s: str) -> bool:
cleaned = [c.lower() for c in s if c.isalnum()]
return cleaned == cleaned[::-1]

Clean and readable — but allocates O(n) extra space. Use the two-pointer approach in interviews when O(1) space is expected.

5 🎤 Interview Strategy: How to Arrive at This ▲
Step 1 — Clarify the Cleaning Rules

State explicitly: "After converting to lowercase and removing non-alphanumeric characters, does it read the same both ways?" Confirm digits count as valid characters. Often interviewers add this wrinkle specifically to test whether you read the problem carefully.

Step 2 — Lead with the Clean Approach, Then Optimise

It's perfectly valid to say "My first thought is to build a cleaned string and compare to its reverse — O(n) time, O(n) space." Then immediately say "I can do O(1) space with two pointers, skipping non-alnum characters in-place." This shows progression of thought.

Step 3 — Explain the Skip Guard

Mention why both inner while loops also check left < right: "If all remaining characters are punctuation, a pointer could overshoot, so I guard both inner loops." This shows attention to detail.

Step 4 — Walk Through Edge Cases

Mention: empty string (True), all non-alnum (True), single character (True), digits matter ("race a car" → False), case insensitive ("A" == "a").

7 Summary & Key Takeaways ▲
Two pointers converging from both ends — the natural structure for palindrome checking
Skip non-alphanumeric on the fly with inner while loops (no extra space needed)
Always guard inner skip loops with left < right to prevent overshoot
Compare with .lower() for case-insensitive matching
Empty string (after cleaning) → True — loop never runs
