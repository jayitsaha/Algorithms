LeetCode #648

Replace Words

Given a dictionary of roots and a sentence, replace each word with its shortest matching root. If no root matches, keep the word unchanged.

Medium Tries Trie for Root Lookup Time: O(N·L) Space: O(D·L)
1 Understanding the Problem ▾

In linguistics, a root is the base morpheme of a word. "cat" is the root of "cattle" and "catfish". This problem asks: for each word in a sentence, find its shortest matching root from a dictionary and replace the word with that root. If no root matches, keep the word.

💡 Key Insight

The core operation is: "Is any dictionary root a prefix of this word?" The Trie data structure is purpose-built for this — we insert all roots into a Trie, then for each sentence word we descend the Trie character by character. The first end-of-word marker we hit is the shortest matching root.

✓ Root Found: Replace

word: "cattle"
root: "cat" is a prefix ✓
result: "cat"

✗ No Root: Keep Word

word: "was"
roots: ["cat","bat","rat"]
result: "was" (unchanged)

Full example: dictionary = ["cat", "bat", "rat"], sentence = "the cattle was rattled by the battery"

Result: "the cat was rat by the bat" — three words got replaced, the rest stayed.

Why "shortest" root matters: If dictionary = ["cat", "ca"] and word = "cattle", both "ca" and "cat" match. We must return "ca" — the shortest root — so the problem requires us to stop at the shallowest match in the Trie.

Edge cases: Empty dictionary (keep all words). Word shorter than all roots (no match). Root exactly equals the word (replace with itself — correct). Multiple roots are prefixes of the same word (take shortest).

2 The Algorithm: Trie for Root Lookup ▾

A Trie (pronounced "try", from reTRIEval) is a tree where each node represents a character. Every path from the root to a marked node spells a complete word. Three roots "cat", "bat", "rat" share the same Trie depth-structure but branch from different starting nodes.

1 Build the Trie: Insert each root from the dictionary character by character. Mark the final character of each root with an end-of-word sentinel ('#'). Cost: O(D·L) where D = dictionary size, L = average root length.
2 Tokenize the sentence: Split on spaces with sentence.split() to get a list of words.
3 Lookup each word in the Trie: Walk character by character from the Trie root. At each node, check for '#' — if found, return word[:i+1] (the prefix so far). If the next character has no child node, fall off the Trie and return the full word unchanged.
4 Rejoin: Collect all results (replaced or unchanged) and join with " ".join(...).

Why the Trie finds the shortest root automatically: We descend from the Trie root toward the leaves. Shorter roots live at shallower depths. The first '#' we encounter during descent is always the shallowest (shortest) matching root. No explicit length comparison needed.

🔑 Hash Set Alternative (Simpler, Slower)

Store all roots in a set. For each word, try prefixes of length 1, 2, 3, … in order, checking if each prefix is in the set. The first hit is the shortest root. This is O(L) set lookups of O(L) slices each → O(L²) per word vs Trie's O(L).

3 Why It Works: The Invariant ▾
🔐 The Trie Descent Invariant

At each step during a Trie lookup for word w, after visiting character w[i], the current Trie node exactly represents the prefix w[0..i]. If '#' is in that node's children, then w[0..i] is a dictionary root — and since we stop immediately, it is provably the shortest such root (we stopped as early as possible).

Why O(L) per lookup: We visit one Trie node per character of the query word, performing one dict lookup per step (O(1) amortized with Python dicts). Total: O(L) regardless of how many roots exist in the dictionary.

Why building the Trie is worthwhile: The O(D·L) build cost is paid once. Every subsequent lookup is O(L). For a sentence with N words, total query cost is O(N·L). The Hash Set approach pays O(N·L²) — notice the extra L factor from repeated slicing. For long words or large sentences, the Trie wins decisively.

⚠️ Classic Mistake: Checking '#' After the Full Word

Some implementations traverse all of word in the Trie and only check '#' at the end. This fails for two reasons: it gives the longest match, not the shortest; and it misses any root that is shorter than the full word. The check must happen at every node during descent.

Edge: root exactly equals word: Lookup "cat" with root "cat": descend c → a → t, find '#' at depth 3, return word[:3] = "cat". Correct — word is returned as-is.

4 The Code (Annotated) ▾

Solution 1 — Trie (Optimal, Interview Pick)

def replaceWords(dictionary, sentence):
# Phase 1: Build Trie from all roots
trie = {} ← Trie as nested dicts: {'c':{'a':{'t':{'#':True}}}}
for root in dictionary: ← Insert every root
node = trie ← Reset to Trie root each time
for ch in root: ← Walk character by character
if ch not in node: ← Create child if absent
node[ch] = {}
node = node[ch] ← Descend into child
node['#'] = True ← Mark end of this root
# Phase 2: Lookup each word
def lookup(word):
node = trie ← Start at Trie root each call
for i, ch in enumerate(word): ← i = depth in Trie / prefix length
if ch not in node: ← No root with this prefix → stop
break
node = node[ch] ← Descend one level
if '#' in node: ← Hit an end-of-root marker!
return word[:i + 1] ← Shortest root = word prefix of length i+1
return word ← No root matched → return original
return ' '.join(lookup(w) for w in sentence.split()) ← Apply to every word, rejoin
⚠️ Why '#' as the end marker instead of a boolean field?

Using '#' (or '$') as a key inside the same dictionary avoids needing a class or separate field. Since '#' is not a lowercase letter, it cannot collide with any real character key. This is idiomatic Python Trie — compact, readable, no boilerplate class needed.

Solution 2 — Hash Set + Prefix Scan (Simpler, O(L²) per word)

def replaceWords(dictionary, sentence):
root_set = set(dictionary) ← O(1) membership test
result = []
for word in sentence.split():
replacement = word ← Default: keep word
for length in range(1, len(word) + 1): ← Try prefix lengths 1..len(word)
if word[:length] in root_set: ← O(L) slice + O(1) hash lookup
replacement = word[:length] ← Shortest match (first hit)
break
result.append(replacement)
return ' '.join(result)
💡 Complexity Summary

Trie: Build O(D·L), Query O(N·L). Total O((D+N)·L). Space O(D·L).
Hash Set: Build O(D·L), Query O(N·L²) — repeated slice O(L) × up to L iterations.
D = dict size, N = words in sentence, L = avg word/root length.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Prefix Lookup Signal

Any problem that says "replace with shortest matching prefix", "startsWith", or "find all words beginning with X" is a Trie signal. Tries are the canonical O(L) prefix data structure. This problem literally says "replace with shortest root" — it's screaming Trie.

Step 2: Propose Hash Set First, Offer Optimization

"The brute-force: store roots in a set, check all prefixes of each word shortest-first. O(N·L²). Simple to code. But we can reduce query time to O(N·L) with a Trie."

Step 3: Justify the Trie Build Cost

"Build cost is O(D·L) — paid once upfront. After that, every word lookup is O(L) — we walk at most L nodes, one per character, and stop early at the first root match. Total: O(D·L + N·L)."

Step 4: Implement With Nested Dicts

Python nested dicts make a clean Trie without boilerplate classes. Use '#' as the end-of-word sentinel — it cannot collide with letter keys. The lookup function returns early on the first '#' found, guaranteeing the shortest root.

🗣️ What to Say to the Interviewer

"This is a prefix lookup problem — Tries give us O(L) per query.
Phase 1: Insert all dictionary roots into a Trie. Mark root ends with '#'. O(D·L).
Phase 2: For each sentence word, walk the Trie character by character.
If we hit a '#' end marker, return the current prefix — it's the shortest root.
If the next char has no child node, fall off the Trie and return the full word.
Join results with spaces. Total time O((D+N)·L), space O(D·L)."
🔮 Likely Follow-up Questions
  • "Can you handle wildcards like '.'?" → Add DFS at wildcard nodes (Design Add and Search Words, #211)
  • "Implement Trie as a class?" → TrieNode with children: dict and is_end: bool
  • "What if the same root appears multiple times?" → No problem — inserting twice is idempotent
  • "Find longest prefix instead of shortest?" → Don't stop at first '#'; walk all the way, track last '#' seen
