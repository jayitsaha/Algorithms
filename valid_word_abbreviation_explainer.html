Valid Word Abbreviation

LeetCode #408 — Two Pointers: Two Pointers with Skip
Easy Two Pointers Two Pointers with Skip O(n) Time · O(1) Space
1
Understanding the Problem
▼

A word abbreviation replaces a substring of consecutive characters with a count. Given a word and an abbreviation, check if the abbreviation is valid for the word.

💡 Key Insight

Use two pointers: i on word, j on abbr. When abbr[j] is a digit, parse the full number (may be multi-digit) and jump i forward by that amount. When it's a letter, compare directly. Key constraint: no leading zeros allowed in numbers.

Examples: word="internationalization", abbr="i12iz4n" → true

word="apple", abbr="a2e" → true ("ppl" = 3? No, "pp" = 2) → i=0(a→a), j skip 2 → i=3(l), abbr[j]='e', word[3]='l' → false

Invalid abbr: "01" (leading zero), "0" (zero skip = skip nothing but occupied a digit → invalid)

2
The Algorithm
▼
1i=0 (word pointer), j=0 (abbr pointer)
2While i < len(word) and j < len(abbr):
3  If abbr[j].isdigit(): parse full number n (reject if leading zero). Advance i by n. Advance j past digits.
4  Else if abbr[j] == word[i]: advance both.
5  Else: mismatch → return False.
6Return True only if both i and j reached their ends.
🔑 Leading Zero Rule

If abbr[j]=='0' and it's the start of a number, it's invalid. "01" doesn't mean "skip 1 character" — leading zeros in abbreviations are not allowed. Check: if the first digit of a number is '0', return False.

3
The Code (Annotated)
▼
def validWordAbbreviation(word: str, abbr: str) -> bool: i = j = 0 # i: word pointer, j: abbr pointer while i < len(word) and j < len(abbr): if abbr[j].isdigit(): # number: skip characters in word if abbr[j] == '0': return False # leading zero: invalid num = 0 while j < len(abbr) and abbr[j].isdigit(): num = num * 10 + int(abbr[j]) # parse multi-digit number j += 1 i += num # advance word pointer by the number elif abbr[j] == word[i]: # matching letters i += 1; j += 1 else: # letter mismatch return False return i == len(word) and j == len(abbr) # both must be exhausted
⚠️ i Could Overshoot

If the number in abbr is too large (e.g., "a99" for "ab"), i may exceed len(word). The final check i == len(word) catches this correctly — if i overshot, it's > len, so the return is False.

🎤 Interview Strategy
▼
1Clarify: Can the abbreviation contain 0? (Yes, invalid if leading zero). Can numbers be multi-digit? (Yes, "i12n" = skip 12 chars).
2Core logic: Two pointers with conditional advancement — letter = compare & advance both, digit = parse number & advance word pointer by that amount.
3Edge cases: Leading zeros, number > remaining word length, abbr exhausted before word, empty abbr.
