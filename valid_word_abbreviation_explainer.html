LeetCode #408

Valid Word Abbreviation

Given a string word and an abbreviation abbr, return true if abbr is a valid abbreviation of word.

Easy Two Pointers Two Pointers with Skip Time: O(n) Space: O(1)
1 Understanding the Problem ▾

An abbreviation replaces one or more consecutive letters with a count of how many letters were skipped. For example, "internationalization" can be abbreviated as "i18n" (i + 18 skipped letters + n), or "internati6n", or the full word itself. Your job: given a candidate abbreviation, verify it actually matches the word.

💡 Key Insight

A valid abbreviation is exactly a two-pointer skip game: pointer i walks word one character at a time; pointer j walks abbr, but whenever it lands on a digit, it reads the full number and jumps i forward by that many positions — skipping those word characters in bulk.

✓ Valid: word="substitution", abbr="s10n"

s → matches word[0]='s'
10 → skip 10 chars (indices 1-10)
n → matches word[11]='n'
Both exhausted → True

✗ Invalid: word="apple", abbr="a2e"

a → matches word[0]='a'
2 → skip 2 chars → i=3
e → word[3]='l' ≠ 'e'
Mismatch → False

Critical rules to internalize:

  • No leading zeros: "02" is not a valid skip number. The moment you see abbr[j]=='0' at the start of a number, return False immediately.
  • Number cannot overshoot: If a skip number would move i past len(word), the final check catches it — the skip itself does not need an explicit bounds guard.
  • Both pointers must be exhausted simultaneously: After the loop, both i==len(word) and j==len(abbr) must hold — no trailing characters allowed in either string.
  • The word itself is always a valid abbreviation — all letters, zero numbers.
2 The Algorithm: Two Pointers with Skip ▾

We maintain two pointers — i for word and j for abbr — and advance them in lockstep. The trick is that when abbr[j] is a digit, we decode the full number (which may be multiple digits, e.g. "10", "127") and use it to leap i forward by that many characters, consuming multiple word letters at once.

1 Initialize: Set i = 0, j = 0. These are the current read positions in word and abbr respectively. Loop while both have characters remaining.
2 Digit case — decode the skip number: If abbr[j] is a digit, immediately reject if it is '0' (leading zero). Otherwise, initialize n = 0 and accumulate: while abbr[j] is a digit, do n = n * 10 + int(abbr[j]); j += 1. Then advance i += n, skipping n characters in word.
3 Letter case — exact match required: If abbr[j] is a letter, it must equal word[i]. If they don't match, return False. If they match, advance both: i += 1; j += 1.
4 Validate final state: After the loop ends, check i == len(word) and j == len(abbr). This catches trailing characters in either string — e.g., word had more letters that the abbreviation never consumed.

Decision tree at each abbr[j]:

🔢 abbr[j] is a digit

Reject if '0' (leading zero). Read full multi-digit number into n. Jump i forward by n, consuming n word letters.

🔤 abbr[j] is a letter

Must exactly match word[i]. Match: advance both i and j. No match: return False immediately.

✓ Loop exits

Return True only if i==len(word) AND j==len(abbr). Both sequences fully consumed.

🔑 Multi-Digit Number Parsing

The skip count in an abbreviation can be more than one digit: "a10b" skips 10 characters. We handle this with a standard integer-parsing inner loop: n = 0; while abbr[j].isdigit(): n = n * 10 + int(abbr[j]); j += 1. This correctly converts "10" → 10, "127" → 127, and so on.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every iteration of the outer while loop, the prefix word[0..i-1] has been fully and correctly validated against abbr[0..j-1]. Every letter in that prefix either matched its counterpart directly, or was accounted for by a skip number. The remaining suffixes word[i..] and abbr[j..] still need to be matched.

Why the leading-zero rule: A skip number of zero means "skip zero characters" — which is meaningless and redundant. More importantly, allowing leading zeros would create ambiguity: does "01" mean skip 0 then literal '1', or skip 1? The spec forbids leading zeros so every abbreviation has a unique canonical representation.

Why we need both pointers exhausted: Consider word="apple", abbr="a3". After matching 'a' and skipping 3, we get i=4, j=2. The loop exits because j==len(abbr), but i=4 is not len(word)=5 — the 'e' at index 4 was never accounted for. Without the final check, we'd wrongly return True.

Why a skip can overshoot safely: If skip n is too large, i becomes > len(word). The while loop condition i < len(word) prevents reading word[i] out of bounds. The final check i == len(word) catches the overshoot. No explicit bounds checking needed inside the skip branch.

⚠️ Subtle Out-of-Bounds Risk

In the letter branch, the code accesses word[i]. But could i be out of bounds at that point? No: the outer while condition guarantees i < len(word) before we enter the loop body. After a skip, we re-enter the loop top, where that check applies again. Safe by construction.

Edge cases handled:

  • abbr = word (no numbers) — every letter matches directly → True
  • abbr = "0" — leading zero check fires immediately → False
  • abbr = str(len(word)) — skip all n letters, j finishes, i==n==len(word) → True
  • abbr skips too many — i overshoots, final i!=len(word) → False
4 The Code (Annotated) ▾

Solution — Two Pointers with Skip (Optimal, O(n+m) time, O(1) space) — Interview Pick

def validWordAbbreviation(word: str, abbr: str) -> bool:
i, j = 0, 0 ← i: position in word; j: position in abbr
while i < len(word) and j < len(abbr): ← continue while both have remaining chars
if abbr[j].isdigit(): ← digit at abbr[j] → decode a skip number
if abbr[j] == '0': ← leading zero is forbidden by the spec
return False
n = 0 ← accumulator for the full integer value
while j < len(abbr) and abbr[j].isdigit(): ← consume all consecutive digits
n = n * 10 + int(abbr[j]) ← shift left and add: '1','0' → 1*10+0 = 10
j += 1 ← advance j past this digit character
i += n ← jump i forward by n: skip n word letters
else: ← abbr[j] is a letter → must match word[i]
if word[i] != abbr[j]: ← character mismatch → abbreviation is invalid
return False
i += 1 ← advance both past the matched letter
j += 1
return i == len(word) and j == len(abbr) ← both must be exactly exhausted
⚠️ Most Common Bug: Missing the Final Check

Beginners often return True as soon as the while loop exits. But consider word="apple", abbr="a3": after matching 'a' and skipping 3, i=4, j=2. The loop exits (j exhausted), but i=4 is not 5. Without return i == len(word) and j == len(abbr) you'd incorrectly return True. Always validate both ends.

💡 Complexity Summary

Time: O(n + m) where n = len(word), m = len(abbr). Each character is visited at most once — j advances through abbr, and each advance of j corresponds to at most one advance of i (letter) or a bulk advance (skip).
Space: O(1) — only two integer pointers and one temporary accumulator n.

Alternative 1 — Brute Force (enumerate all abbreviations): Generate all 2^n possible abbreviations of word and check if abbr appears. O(2^n) time and space. Impractical for anything beyond toy examples.

Alternative 2 — Regex: Build a regex from abbr (replace each digit sequence with .{n}) and match against word. Concise, but O(n) space for the regex pattern, harder to control edge cases, and depends on re library internals. The manual two-pointer solution is clearer and more robust in interviews.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Two-Pointer Pattern

Two strings must be matched in sync, but one string has "skip" tokens. When you see "match two sequences with possible jumps," think two pointers — one per string. The skip token governs how far the first pointer jumps. This is a classic "Two Pointers with Skip" sub-pattern.

Step 2: Identify Exactly Two Cases

At each step in abbr, there are exactly two cases: digit (decode the skip count, advance i) or letter (match directly with word[i], advance both). Handling these two cases correctly, plus the leading-zero and final-check edge cases, is the entire algorithm.

Step 3: Handle Multi-Digit Numbers Explicitly

Don't assume skip numbers are single digits. Use an inner while loop: n = n * 10 + int(abbr[j]). This is a standard pattern for parsing integers character by character that appears in many other problems (e.g., atoi, Roman numerals).

Step 4: Always Mention the Final Check

Proactively tell the interviewer: "After the loop, I verify both pointers are at their respective ends. This is important — the loop can exit early if one side is exhausted first, but both must be exhausted for a valid match." This shows you think about subtle post-conditions.

🗣️ What to Say to the Interviewer

"I'll use two pointers: i into word and j into abbr.
At each step: if abbr[j] is a digit, I check for a leading zero first — invalid if so.
Then I read all consecutive digits to build the full number n, and jump i forward by n.
If abbr[j] is a letter, it must exactly match word[i]; otherwise return False.
After the loop, both i and j must equal their respective lengths — this prevents
partial matches where one string still has remaining characters.
Time is O(n+m), space is O(1)."
