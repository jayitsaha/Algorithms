LeetCode #130

Surrounded Regions

Given an m×n board of 'X' and 'O', capture all surrounded regions by flipping enclosed 'O's to 'X'. 'O's connected to a border are never captured.

Medium Graph DFS from Borders Time: O(m·n) Space: O(m·n)
1 Understanding the Problem ▾

You have a 2D grid of 'X' and 'O' characters. A region of 'O' cells is surrounded if every path from it to any edge of the board passes through at least one 'X'. Surrounded 'O' regions must be flipped to 'X'. An 'O' that touches the border (or is connected to one that does) is safe and stays as 'O'.

💡 Key Insight: Invert the Problem

Instead of finding surrounded regions (which requires verifying ALL paths are blocked — hard), find the safe ones first. An 'O' is safe if and only if it is reachable from any border cell by a path through only 'O's. Everything else is surrounded and gets captured.

Input Board

X X X X
X O O X
X X O X
X O X X

Output Board

X X X X
X X X X
X X X X
X O X X

The 'O' at row 3, col 1 sits on the bottom border row — it is safe and stays. The three 'O's at (1,1), (1,2), (2,2) form an interior cluster with no path to any border — they are captured and become 'X'.

Edge Cases: All-X board (nothing to do). All-O board (border O's safe, rest captured). Single row or column (all cells are border cells — nothing captured). 1×1 board (always safe). Disconnected interior O-islands all get captured independently.

2 The Algorithm: Three-Phase DFS from Borders ▾

The algorithm uses a classic mark-then-flip pattern. We repurpose a temporary sentinel character 'S' to tag safe cells in-place, avoiding any extra visited array.

①
Mark Safe
DFS/BFS from every border 'O', marking reachable O's as 'S'
②
Capture
Scan board: any remaining 'O' is surrounded → flip to 'X'
③
Restore
Any 'S' sentinel → restore to 'O'. Board is final.
1 Border DFS — mark safe regions: Scan all four border edges. For each 'O' on the border, launch DFS inward. The DFS visits only 'O' cells, marks them 'S', and recurses to all 4-directional neighbors. This floods every connected component that has border access.
2 Capture surrounded O's: Scan the entire board. Any 'O' not marked 'S' was unreachable from the border — it is fully enclosed by 'X's. Flip it to 'X'.
3 Restore safe O's: Any cell with value 'S' was a border-connected O that we temporarily protected. Flip it back to 'O'. Board is now in the correct final state.
🔑 Why Start DFS from Borders?

Border cells are the only exit points from the grid. Any O connected to a border has an escape route and must survive. By flooding outward from these known-safe sources, we transitively discover all safe O's in one traversal — regardless of how far from the border they are, as long as they are connected.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After Phase 1, every cell marked 'S' is reachable from at least one border cell by a path through only 'O's. Every 'O' that remains unmarked has NO such path — it is genuinely enclosed by 'X's with no escape route.

Proof of correctness (by contradiction): Suppose an interior 'O' is connected to a border O but was NOT marked 'S'. Then there exists a path from that border O to our interior O through only O's. The DFS starting from that border O would follow this path and mark every O along it — including our interior O. Contradiction. Therefore, if it's unmarked, it truly has no border connection.

Why the sentinel works: Since the DFS only expands into cells that are still 'O' (skipping 'X' and 'S'), marking as 'S' also prevents re-visiting in the same traversal. No explicit visited set is needed. The three distinct values ('X', 'O', 'S') encode all necessary state in the board itself.

Single row/column edge case: Every cell in a 1×m or n×1 board is a border cell. The border scan marks all O's as S. Phase 2 finds no unmarked O's to capture. Phase 3 restores all S's. Result: board unchanged. Correct — nothing to capture when everything is on the border.

⚠️ The Naive Approach: Why It Fails

A naive approach DFS from each interior O and checks if it can reach a border. This is O((m·n)²) in the worst case — for each of m·n O-cells, you run an O(m·n) DFS. Moreover, connected components must be handled atomically (flip ALL or NONE), which requires bookkeeping across DFS calls. The border-DFS approach solves all of this elegantly in a single traversal.

# Phase 2 + 3: Flip + Restore in one pass← Single combined scan

for c in range(cols):
if board[r][c] == 'O': ← Unmarked O = surrounded → capture
board[r][c] = 'X'
elif board[r][c] == 'S': ← Safe sentinel → restore to original
board[r][c] = 'O'
⚠️ Python Recursion Depth on Large Boards

Python's default recursion limit is ~1000 frames. A connected O-region spanning a 200×200 board could have 40,000 cells — easily exceeding this. In interviews, mention this risk and offer the iterative BFS alternative. You can also call sys.setrecursionlimit(200*200) as a quick fix, but iterative BFS is the production-safe approach.

Solution 2 — Iterative BFS from Borders (Production-Safe)

from collections import deque ← BFS uses a queue
def solve(board):
if not board or not board[0]: return
rows, cols = len(board), len(board[0])
queue = deque() ← BFS frontier
for r in range(rows): ← Seed with all border O's
for c in [0, cols - 1]:
if board[r][c] == 'O':
queue.append((r, c)); board[r][c] = 'S' ← Mark when enqueued
for c in range(cols):
for r in [0, rows - 1]:
if board[r][c] == 'O':
queue.append((r, c)); board[r][c] = 'S'
while queue: ← BFS expansion of all safe regions
r, c = queue.popleft()
for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)]:← 4-directional neighbors
nr, nc = r + dr, c + dc
if 0<=nr<rows and 0<=nc<cols and board[nr][nc]=='O':
board[nr][nc] = 'S'; queue.append((nr, nc))
for r in range(rows): ← Flip + restore pass
for c in range(cols):
if board[r][c] == 'O': board[r][c] = 'X' ← Captured
elif board[r][c] == 'S': board[r][c] = 'O' ← Restored
💡 Complexity

Both approaches: Time O(m·n) — each cell visited at most once by the DFS/BFS, then once in the flip pass.
Space O(m·n) — recursion call stack (DFS) or BFS queue in worst case (all cells are safe O's).
No additional visited array needed — the in-place sentinel 'S' handles state tracking.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Problem Inversion

Finding surrounded regions requires verifying that ALL neighbors are blocked — exponential in the worst case. Flip the question: find the regions that are NOT surrounded. These are exactly the O's reachable from any border cell. This makes it a simple reachability/flood-fill problem.

Step 2: Identify the "Multi-Source" DFS Pattern

Border cells are all the sources of safety. Rather than running one DFS per border cell and merging results, we can treat all border O's as simultaneous sources. In DFS, this is natural: just call dfs(r,0) and dfs(r,cols-1) for each row, etc. In BFS, seed the queue with all border O's at once — classic multi-source BFS.

Step 3: Use Sentinel to Save Space

A visited set would work but costs O(m·n) extra space. By repurposing a third character value ('S') within the board itself, we get visited-tracking for free. The board has three possible values — three possible states. This is a general trick worth remembering for in-place grid problems.

Step 4: State Engineering Judgment

Proactively mention that recursive DFS can stack-overflow on large boards in Python. Offer BFS as a production-safe alternative. This separates "just correct" from "actually deployable" — exactly what interviewers at strong companies look for.

🗣️ What to Say to the Interviewer

"Finding surrounded regions is complex because I'd need to check all paths from each O.
Instead, I'll identify the SAFE regions: O's connected to the border — easier to compute.
Phase 1: DFS from every border O, marking all reachable O's with sentinel 'S'.
Phase 2: Scan the board — any remaining 'O' was surrounded → flip to 'X'.
Phase 3: Restore 'S' back to 'O'. Board is in final state.
Time O(m·n), Space O(m·n). In Python I'd use iterative BFS to avoid recursion depth issues."
🔮 Likely Follow-up Questions
  • "Could you use Union-Find?" → Yes: add a virtual "border" node, union all border O's to it, then union O-O adjacencies. O's in the same component as the virtual node are safe.
  • "What if diagonal connections count?" → Expand directions array from 4 to 8 entries: add (1,1), (1,-1), (-1,1), (-1,-1).
  • "Large board — stack overflow risk?" → Use BFS with deque, or sys.setrecursionlimit(m*n+100) as a shortcut.
  • "Can we do this in O(1) space?" → No — we need at least O(border length) = O(m+n) for the BFS queue. True O(1) space is impossible here.

7 Summary & Key Takeaways ▾
Invert the problem: find SAFE O's (border-connected) instead of finding surrounded O's directly
Multi-source DFS/BFS from all border cells simultaneously discovers all safe regions in one pass
Sentinel character ('S') enables in-place state tracking — no extra visited array needed
Three-phase pattern (Mark → Capture → Restore) is reusable for many flood-fill grid problems
Use iterative BFS in Python production code to avoid recursion depth issues
Time O(m·n), Space O(m·n) — each cell visited at most once
