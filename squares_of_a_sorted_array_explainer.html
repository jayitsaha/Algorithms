Squares of a Sorted Array

LeetCode #977 — Two Pointers: Opposite Direction (Abs Compare)
Easy Two Pointers Opposite Direction (Abs Compare) O(n) Time · O(n) Space
1
Understanding the Problem
▼

Given a sorted array of integers (possibly with negatives), return an array of the squares of each number, also in non-decreasing order.

💡 Key Insight

The array is sorted, so the largest absolute values (and therefore largest squares) are at the two ends. The smallest squared value is somewhere in the middle near zero. By comparing the two ends and filling the result from the back, we can avoid sorting.

Example: nums = [-4, -1, 0, 3, 10]

Naively: square all → [16, 1, 0, 9, 100] → sort → [0, 1, 9, 16, 100]
Optimal: abs(-4)=4 vs abs(10)=10 → 10 wins → place 100 at end. Continue inward.

Edge cases: All negative, all positive, single element, all zeros, mixed with no negatives.

2
Two Solutions: Naive Sort vs Optimal Two Pointers
▼

Solution 1 — Naive (O(n log n)): Square everything, then sort. Simple but doesn't exploit the sorted structure.

# Naive — O(n log n) def sortedSquares_naive(nums): return sorted(x * x for x in nums)

Solution 2 — Optimal Two Pointers (O(n)): Exploit sorted property. Use left and right pointers converging inward. Largest square is always at one of the ends; fill result from back.

1Initialise left=0, right=n-1, k=n-1 (fill result from back)
2While left ≤ right: compare |nums[left]| vs |nums[right]|
3Larger absolute value goes into result[k], move that pointer inward, k--
4Repeat until pointers cross
🔑 Why Fill from the Back?

We know the current maximum square but not the minimum. Filling from position k=n-1 downward means we always place values in the correct sorted order — large to small — using one pass.

3
Why It Works: The Invariant
▼
🔐 The Invariant

At any point, result[k+1 .. n-1] contains the k largest squares in sorted order (placed correctly). The next largest square is at nums[left]² or nums[right]² — whichever has the larger absolute value.

This holds because nums is sorted: the two endpoints always hold the current maximum absolute values of the unprocessed portion. After squaring the larger endpoint and placing it at result[k], the subproblem reduces to nums[left+1..right] or nums[left..right-1].

✅ O(n) vs O(n log n)

The two-pointer approach makes one pass, achieving O(n) time — a factor of log n faster than sort-based approaches on large arrays.

4
The Code (Annotated)
▼
def sortedSquares(nums): n = len(nums) result = [0] * n # pre-allocate output array left, right = 0, n - 1 # two pointers at both ends k = n - 1 # fill result from the back (largest first) while left <= right: l_sq = nums[left] ** 2 # square of left element r_sq = nums[right] ** 2 # square of right element if l_sq > r_sq: # left has larger absolute value result[k] = l_sq # place it at current back position left += 1 # move left pointer inward else: # right has larger or equal absolute value result[k] = r_sq # place it at current back position right -= 1 # move right pointer inward k -= 1 # move fill position one step left return result
⚠️ Common Mistake: Condition Should Be ≤ not <

Use while left <= right not left < right. When left==right, there's exactly one element left that must be squared and placed. Using strict less-than skips the last element.

🎤 Interview Strategy
▼
1Start with naive: "We can square each element and sort — O(n log n), O(n)."
2Observe: "The array is sorted, so largest absolute values are at the ends. Can we use that?"
3Insight: "Yes — converge two pointers from both ends, fill result from back. O(n)."
4Why fill from back? "We know the maximum, not the minimum. Filling largest-to-smallest ensures correct order in one pass."
5State edge cases: All negatives (right end wins first), all positives (left end wins first), symmetric array.
