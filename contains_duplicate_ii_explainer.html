Contains Duplicate II

LeetCode #219 – Find duplicates within distance k

Easy Hash Table Index Tracking O(n) Time O(min(n,k)) Space
1 Understanding the Problem ▼

We need to find if there are two equal elements whose indices are at most k apart.

Two conditions must both be true:

  • nums[i] == nums[j] — Same value
  • |i - j| ≤ k — Within distance k
✅ [1,2,3,1], k=3
indices 0 and 3
distance = 3 ≤ k
✓ TRUE
❌ [1,2,3,1,2,3], k=2
1 at 0,3 → dist=3 > k
2 at 1,4 → dist=3 > k
✗ FALSE
💡 Key Insight

We only need to track the most recent index of each value. If we see the same value again and the distance is ≤ k, we're done. Otherwise, update to the current index (it's now the most recent).

2 The Algorithm: Hash Map with Index Check ▼

Use a hash map to store value → last seen index. For each element:

1 Check if value exists in map (seen before)
2 If yes, check distance: current_index - last_index ≤ k
3 If distance ≤ k → return true (found!)
4 Update map with current index (always keep most recent)
🎯 Why Track Only the Most Recent Index?

If an earlier occurrence was too far away, it will always be too far for future indices. The most recent occurrence gives us the best chance of being within k.

3 Why It Works: The Invariant ▼
🔐 The Invariant

At any point, map[v] contains the most recent index where value v was seen. This is the closest possible match for any future occurrence of v.

Why doesn't keeping old indices help?

Suppose value 5 appears at indices [2, 7, 15] and k=5:

  • At index 7: distance from 2 is 5 ≤ k → but we check this
  • At index 15: distance from 2 is 13 > k, from 7 is 8 > k

If neither works, keeping index 2 is useless — index 7 is strictly better for future checks.

⚠️ Alternative: Sliding Window Set

You could also maintain a set of the last k elements. If an element is already in the set when you try to add it → duplicate found! This uses O(k) space guaranteed.

4 The Code (Annotated) ▼
# Hash Map with Index Check
def containsNearbyDuplicate(nums, k):
last_seen = {} # value → most recent index
for i, num in enumerate(nums):
if num in last_seen:
if i - last_seen[num] <= k:
return True # Found within distance!
last_seen[num] = i # Update to current index
return False # No valid pair found

Alternative: Sliding Window Set

def containsNearbyDuplicate(nums, k):
window = set()
for i, num in enumerate(nums):
if num in window:
return True
window.add(num)
if len(window) > k: # Window size exceeds k
window.remove(nums[i - k]) # Remove oldest
return False

Complexity:

  • Time: O(n) — Single pass, O(1) hash operations
  • Space: O(min(n, k)) — At most n unique values, or k+1 in window approach
5 🎤 Interview Strategy ▼
Step 1: Recognize the Pattern

Keywords: "duplicate", "distance k", "indices" → Need to track where each value was last seen. Hash map for O(1) lookup.

Step 2: Clarify the Constraint

"|i - j| ≤ k" means within k positions. If k=3 and indices are 2 and 5, distance is 3 which is valid.

Step 3: Choose the Approach

Two options: (1) Hash map storing last index, or (2) Sliding window set of size k. Hash map is simpler to implement correctly.

🗣️ What to Say to the Interviewer

"I'll use a hash map to track the most recent index of each value. As I scan, if I see a value that's in the map and the distance from its last index to current is ≤ k, I return true. Otherwise, I update the map with the current index. This is O(n) time and O(min(n,k)) space."
🎯 Follow-up Questions
  • What if k is very large? — Hash map approach uses O(n) space worst case
  • What if we need to find the pair? — Return (last_seen[num], i) instead of true
  • Contains Duplicate III? — Need buckets or sorted set for value range check
