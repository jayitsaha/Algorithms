LeetCode #378

Kth Smallest Element in a Sorted Matrix

Given an n×n matrix where each row and column is sorted ascending, find the kth smallest element in the matrix.

Medium Heaps Merge K Sorted Time: O(k log n) Space: O(n)
1 Understanding the Problem ▾

You are given an n×n matrix with a special structure: every row is sorted left-to-right and every column is sorted top-to-bottom. Your task is to return the kth smallest value across all n² elements — not the kth element in any particular row or column, but globally when all values are imagined in sorted order.

💡 Key Insight

The matrix is not fully sorted — matrix[0][2] may be larger than matrix[1][0]. However, the sorted-row, sorted-column structure guarantees that the globally smallest element is always at [0][0]. After extracting any element, its right neighbor (in the same row) and bottom neighbor (in the same column) are the next candidates. A min-heap lets us walk through values in globally sorted order without materializing all n² elements.

Critical observation about the structure: Row 0 is sorted (1 < 5 < 9), Row 1 is sorted (10 < 11 < 13), Column 0 is sorted (1 < 10 < 12). But 9 (row 0, col 2) > 10 (row 1, col 0), so the matrix is NOT in row-major sorted order.

✗ Naive: Flatten and Sort

Copy all n² elements, sort, return index k−1.
Time: O(n² log n²) = O(n² log n)
Space: O(n²)
Ignores the sorted structure entirely.

✓ Optimal: Min-Heap Row Traversal

Treat each row as a sorted stream.
Time: O(k log n) — only visit what we need
Space: O(n) — heap holds one entry per row
Exploits the sorted-row structure perfectly.

Edge cases: k=1 → always matrix[0][0]. k=n² → always matrix[n-1][n-1]. Duplicate values are handled naturally by the heap. 1×1 matrix → return matrix[0][0].

2 The Algorithm: Min-Heap Row Streams ▾

Think of the matrix as n independently sorted streams running in parallel — one per row. We need the globally kth element across all streams merged. This is precisely the Merge K Sorted Lists pattern, applied to a 2D matrix.

1 Seed the heap: Push (matrix[i][0], i, 0) for each row i. This loads the first (smallest) element of each row's stream. Heap size = n.
2 Pop the minimum: heappop() yields (val, row, col) — the globally smallest element not yet returned. The heap invariant guarantees this.
3 Advance that stream: If col + 1 < n, push (matrix[row][col+1], row, col+1). This moves row's pointer one step right. If col+1 == n, that row's stream is exhausted — don't push.
4 Repeat k times: The value from the kth pop is the kth smallest. Run the pop-and-push loop k−1 times, then do a final pop outside the loop.
🔐 The Invariant

At every moment, the heap contains exactly the "current front" of each active row stream. The heap minimum is always the globally smallest element not yet extracted. This invariant is established by seeding with column 0 and maintained by always pushing the next element from the popped row.

Why Only Push Right Neighbors?

Because each row is already seeded from column 0. Pushing "below" (row+1, same col) would double-count elements already covered by row r+1's own stream entry. Pushing only right keeps each cell pushed exactly once, achieving O(n) space instead of O(n²).

3 Why It Works: Correctness and Invariant ▾
🔐 Proof of Correctness

Claim: The value returned by the kth heappop() is the kth smallest element in the matrix.

Proof by induction:

  • Base case (k=1): The heap is seeded with matrix[i][0] for all i. The minimum of these is matrix[0][0] (since columns are sorted, matrix[0][0] ≤ matrix[i][0] for all i). The first pop correctly returns the global minimum.
  • Inductive step: Assume pops 1 through k−1 returned the k−1 smallest elements. Before pop k, the heap contains the "next candidate" from each stream — all elements that could possibly be the kth smallest. The heap minimum among these is exactly the kth smallest overall.

Why does the heap always contain the right candidates? After popping element at position (r, c), we push (r, c+1). Every element in the matrix either: (a) has already been popped, (b) is currently in the heap, or (c) will only be reachable after some heap-element is popped. Nothing is skipped.

⚠️ Common Mistake: Pushing Both Right and Down

Some implementations push both the right neighbor (r, c+1) and the bottom neighbor (r+1, c) after each pop, using a visited set to avoid duplicates. This works correctly but uses O(min(k, n²)) space for the visited set. The row-seeding approach is cleaner: each cell is pushed at most once, and no visited set is needed.

Binary Search alternative: Binary search on the value range [matrix[0][0], matrix[n-1][n-1]]. For each candidate mid, count how many matrix values are ≤ mid using an O(n) staircase walk from the bottom-left corner. This achieves O(n log(V)) time (V = value range) and O(1) space — better space, different time trade-off. The heap approach is preferred in interviews because it's more intuitive.

4 The Code (Annotated) ▾

Solution 1 — Min-Heap (Interview Pick)

import heapq← Python's min-heap module; heapq operates on plain lists in-place
def kthSmallest(matrix, k):
n = len(matrix)← matrix is n×n; len gives the row count = n
heap = [(matrix[i][0], i, 0) for i in range(n)]← seed with (value, row, col=0) for every row — the front of each stream
heapq.heapify(heap)← build valid min-heap from unsorted list in O(n); heap[0] is now the global min
for _ in range(k - 1):← do k−1 pops to discard elements 1 through k−1
val, r, c = heapq.heappop(heap)← extract globally smallest remaining element; O(log n) heap-down
if c + 1 < n:← if row r has more elements to the right…
heapq.heappush(heap, (matrix[r][c+1], r, c+1))← …push next element from row r's stream; O(log n) heap-up
return heapq.heappop(heap)[0]← kth pop: extract the kth smallest; [0] grabs the value (not row/col)
⚠️ Why range(k-1), not range(k)?

We want k total pops: k−1 inside the loop (discard), then 1 final pop that returns the answer. This avoids an if-condition inside the loop to detect the kth iteration. Alternatively you can do range(k) and capture the last val: for _ in range(k): val, r, c = heapq.heappop(...); ...

Solution 2 — Binary Search on Answer (O(1) Space)

def kthSmallest(matrix, k):
n = len(matrix)
lo, hi = matrix[0][0], matrix[n-1][n-1]← search range: smallest to largest value in matrix
while lo < hi:
mid = (lo + hi) // 2
count = _count_le(matrix, mid, n)← how many values ≤ mid? use O(n) staircase walk
if count < k: lo = mid + 1← fewer than k values ≤ mid, answer must be larger
else: hi = mid← at least k values ≤ mid; mid could be the answer, shrink right
return lo← lo converges to the kth smallest value (which exists in the matrix)
def _count_le(matrix, target, n):← staircase: O(n) count of elements ≤ target
count, r, c = 0, n - 1, 0← start at bottom-left corner
while r >= 0 and c < n:
if matrix[r][c] <= target:← current cell ≤ target → all cells above it in this column too
count += r + 1; c += 1← count r+1 cells (col c, rows 0..r), move right
else: r -= 1← cell too big, go up one row
return count

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Pattern Recognition

"Sorted rows, sorted columns, find kth smallest" is the Merge K Sorted Lists signal in disguise. Each row is one sorted list. Signal words: "sorted matrix", "kth", "multiple sorted sources". Jump straight to min-heap.

Step 2: Define the Heap Entry

Each entry needs three things: (value, row, col). The value drives heap ordering. Row + col let you locate the next element in that row's stream. Python tuples compare lexicographically — value first, perfect.

Step 3: Code the Pattern

Seed → loop k−1 (pop + conditional push right) → final pop. Confirm the boundary: if col+1 == n, that row is done — skip the push. This is the exact same loop as Merge K Sorted Lists.

Step 4: Proactively Mention Space

After presenting the heap solution say: "For O(1) space, I can binary search on the answer — binary search on values from [min, max], counting how many cells are ≤ mid using a staircase walk. Want me to show that?"

🗣️ What to Say to the Interviewer

"I recognize this as Merge K Sorted Lists applied to a matrix.
Each row is a sorted stream — I'll track the front of each with a min-heap.
Heap entry: (value, row, col). Seed with column 0 of every row.
Pop k times total: each pop gives the next global minimum.
After each pop, push that row's next element (col+1) if it exists.
Time: O(k log n) — k pops × O(log n) each, heap size ≤ n.
Space: O(n) for the heap.
There's also a binary search approach for O(1) space if needed."
🎯 Follow-Up Questions You Might Get
  • What if k ≈ n²? → Heap is O(k log n) ≈ O(n² log n); binary search is better: O(n log V)
  • What if the matrix is not square (m×n)? → Heap still works: seed with min(m,n) rows, each with n columns
  • What if only rows are sorted (not columns)? → Heap approach still valid; binary search's staircase count breaks
  • Find kth largest? → Same idea reversed: seed from last column, push left neighbor instead
  • Find kth smallest sum from two sorted arrays? → Same heap pattern (#373 Find K Pairs with Smallest Sums)

7 Summary & Key Takeaways ▾
This is Merge K Sorted Lists on a matrix — each row is one sorted stream
Seed the heap with the entire first column: one entry per row (the stream front)
Each pop gives the globally next-smallest; push its right neighbor to advance that stream
Heap never exceeds n entries — O(n) space, no visited set needed
The binary search approach achieves O(1) space but requires the staircase counting trick
Duplicates are handled naturally — heap tuple comparison includes (val, row, col) which is always unique
🎯 When to Recognize This Pattern
  • "Find kth smallest/largest from multiple sorted sources" → Min/max heap
  • "Sorted matrix, find kth element" → Row-seeded min-heap traversal
  • "Merge k sorted lists/arrays" → Min-heap with (value, list_id, index)
  • "Smallest sum from two sorted arrays, kth pair" → Same heap pattern
