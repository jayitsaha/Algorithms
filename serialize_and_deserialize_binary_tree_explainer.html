LeetCode #297

Serialize and Deserialize Binary Tree

Design an algorithm to serialize a binary tree to a string and deserialize that string back to the original tree structure, preserving both values and shape.

Hard Trees Preorder with Nulls Time: O(n) Space: O(n)
1 Understanding the Problem ▾

You must implement two functions: serialize (binary tree → string) and deserialize (string → binary tree), such that deserialize(serialize(root)) == root — the reconstructed tree has the exact same structure and node values as the original.

💡 Key Insight

A standard preorder traversal like [1, 2, 3] is ambiguous — it could represent many different trees. But if we also record null children explicitly with a sentinel like #, the sequence becomes unique. Every valid preorder-with-nulls sequence maps to exactly one tree.

✓ Preorder + Null Markers

Tree [1, 2, 3]:
serialize → "1,2,#,#,3,#,#"
→ Unique, unambiguous ✓

✗ Preorder Only (No Nulls)

"1,2,3" could mean:
root=1, left=2, right=3 OR
root=1, left=2, left.left=3 ✗

Edge cases: null root (empty tree) → serialize to "#"; single node → "1,#,#"; right-only skewed tree; trees with negative values or duplicate values.

2 The Algorithm: Preorder DFS with Null Markers ▾

We use preorder DFS (root → left → right) for serialization. The key rule: whenever we reach a None node, we write the sentinel # instead of nothing. This encodes the tree's structure in the token sequence itself.

Serialize — preorder DFS, null → #:
1 Base case: If node is None, append "#" to the output list and return.
2 Record root: Append str(node.val) to the output list.
3 Recurse left: Call dfs(node.left) — serializes the entire left subtree.
4 Recurse right: Call dfs(node.right) — serializes the entire right subtree.
5 Join: Return ",".join(tokens) as the final string.
Deserialize — mirror of serialize, consuming the deque top-down:
1 Split & wrap: deque(data.split(",")) — O(1) left-pops.
2 Pop next token: Each recursive call consumes exactly one token via popleft().
3 If "#": return None — this position in the tree is empty.
4 Otherwise: Create TreeNode(int(val)), then recurse for left and right children.
Example: Tree [1, 2, 3, null, null, 4, 5]

        1
      /   \
    2       3
   / \     /  \
  #   #   4    5
          /\   /\
          # #  # #

Serialized: "1,2,#,#,3,4,#,#,5,#,#"
11 tokens = 2×5 nodes + 1 = 2n+1 ✓
🔑 Why Preorder Works Best

Preorder processes the root before its children. During deserialization, this means we always know the current subtree's root before we need to build its children. This top-down construction is the most natural — no lookahead needed. Inorder or postorder would require more bookkeeping.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At each recursive call during deserialization, the next token in the deque is exactly the value (or null marker) for the root of the current subtree being built. The preorder structure guarantees that every call pops exactly the right token without any lookahead or backtracking.

Why does recording nulls make reconstruction unique?

In any binary tree with n nodes, there are exactly n+1 null pointers (leaf nodes each contribute 2 null children, but internal nodes "consume" one null pointer slot each). So the serialized string always contains exactly 2n + 1 tokens — n values plus n+1 null markers. When deserializing, each recursive call consumes exactly one token: the consumption is perfectly balanced and deterministic.

💡 The Bijection Argument

The mapping "preorder sequence with nulls" ↔ "binary tree" is a bijection. Every binary tree produces a unique token sequence, and every sequence where the parenthetical "brackets" balance (conceptually: each # closes a leaf, each value opens two sub-calls) reconstructs exactly one tree. This is why the deque-popleft approach always terminates correctly.

⚠️ Critical: Use deque for O(1) popleft

If you call list.pop(0), Python shifts all remaining elements left — that's O(n) per pop, giving O(n²) deserialization. Always use collections.deque and deque.popleft() for O(1) front removal. Interviewers will notice.

Empty tree edge case: A null root serializes to "#". Deserializing pops "#", returns None immediately. Clean, correct, no special-casing needed beyond the standard base case.

4 The Code (Annotated) ▾

Solution 1 — Preorder DFS + Null Markers (Optimal, Interview Pick)

from collections import deque
class Codec:
def serialize(self, root) -> str: ← Tree → string
tokens = [] ← Build list, join at end (faster than concat)
def dfs(node): ← Closure: captures tokens
if node is None: ← Base case: null child
tokens.append('#') ← '#' encodes "no node here"
return
tokens.append(str(node.val)) ← Preorder: root value first
dfs(node.left) ← Serialize entire left subtree
dfs(node.right) ← Then entire right subtree
dfs(root)
return ','.join(tokens) ← Comma-delimited output
def deserialize(self, data: str): ← String → tree
q = deque(data.split(',')) ← deque for O(1) popleft
def build(): ← Each call consumes 1 token
val = q.popleft() ← Next token in preorder
if val == '#': return None ← Null marker → empty position
node = TreeNode(int(val)) ← Create the node
node.left = build() ← Left subtree (mirrors serialize)
node.right = build() ← Right subtree
return node
return build()
⚠️ The Symmetry Principle

Notice that build() in deserialize has the exact same recursive structure as dfs() in serialize: base case first, then create/append, then recurse left, then recurse right. This symmetry is not a coincidence — if your serialize and deserialize share the same traversal order, reconstruction is guaranteed to be correct.

Solution 2 — BFS Level-Order (Alternative)

def serialize(self, root) -> str: ← BFS approach
if not root: return '#'
q, tokens = deque([root]), []
while q:
node = q.popleft()
if node is None: tokens.append('#')
else: ← Enqueue children including None
tokens.append(str(node.val))
q.append(node.left); q.append(node.right)
return ','.join(tokens)
💡 DFS vs BFS Trade-offs

DFS: Cleaner code, natural recursion. Stack depth = O(h) where h = tree height. Worst case O(n) for skewed trees (but Python's recursion limit is ~1000).
BFS: Iterative, no stack overflow risk for deep trees. Matches LeetCode's own serialization format. Slightly more verbose.
In interviews: DFS is preferred for elegance. Mention BFS as an alternative if stack overflow is a concern.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Core Challenge

"The tricky part isn't the traversal — it's making the encoded string unambiguous. I need a format where the tree structure is recoverable without any additional information."

Step 2: Recognize Why Standard Traversals Fail

"Preorder [1,2,3] alone is ambiguous — node 2 could be a left child or a right child. Inorder + preorder together work, but require O(n) lookup. The simplest fix: record null children explicitly."

Step 3: Commit to Preorder + Null Markers

"With '#' for null, I can do a preorder DFS. The order root→left→right means when deserializing, the next token is always the current root. I consume it, build the node, then recurse left and right — perfect symmetry."

Step 4: Highlight the deque Optimization

"I'll use a deque for O(1) popleft. Using list.pop(0) would be O(n) per call, making deserialization O(n²). deque keeps it O(n) overall."

🗣️ What to Say to the Interviewer

"I'll use preorder DFS with null markers.
Serialize: If node is None, append '#'. Otherwise, append the value, then recurse left then right. Join with commas.
Deserialize: Split on comma, wrap in deque. The recursive helper pops the next token — if '#', return None. Otherwise create a node and recurse for left and right.
The symmetry guarantees correctness: same traversal order for both functions.
Time: O(n) for both — visit each node once. Space: O(n) for the token list and recursion stack."

7 Summary & Key Takeaways ▾
Null markers ('#') are the key — without them, any tree traversal is ambiguous
An n-node tree always has exactly n+1 null pointers → serialized string has 2n+1 tokens
Serialize and deserialize must use the same traversal order (both preorder here)
Use collections.deque + popleft() — O(1) per pop vs O(n) for list.pop(0)
The recursive structure of build() mirrors dfs() exactly — symmetry guarantees correctness
