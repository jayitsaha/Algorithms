LeetCode #14

Longest Common Prefix

Find the longest common prefix string amongst an array of strings.

Easy String Processing Horizontal Scan Time: O(S) Space: O(1)
1 Understanding the Problem ▾

A prefix is a string that forms the beginning of another string. The longest common prefix is the longest string that appears at the beginning of every string in the array.

💡 Key Insight

We don't need to compare all pairs. The LCP of a set of strings is the LCP of any two that bound the rest — or we can progressively narrow a candidate prefix by comparing it against each string one at a time. If the prefix ever becomes empty, return "" immediately.

2 The Algorithm: Horizontal Scan ▾

The Horizontal Scan treats strs[0] as the starting candidate prefix, then trims it against each subsequent string until every string starts with what remains.

1 Seed from strs[0]: prefix = strs[0]. The LCP of all strings cannot be longer than the first string, so starting here is safe. No character comparisons yet.
2 Compare with each remaining string: For each s in strs[1:], check s.startswith(prefix). If True, move on. If False, shrink the prefix.
3 Shrink by one char at a time: prefix = prefix[:-1] removes the last character. Repeat until s.startswith(prefix) is True. If prefix empties, return "" immediately.
4 Return the survivor: After all strings are processed, whatever remains in prefix is the LCP. It can never grow — it can only shrink or stay the same.
✗ Brute Force — O(n·m²)

for i in range(min_len):
  c = strs[0][i]
  for s in strs[1:]:
    if s[i] != c:
      return strs[0][:i]
return strs[0][:min_len]

✓ Horizontal Scan — O(S)

prefix = strs[0]
for s in strs[1:]:
  while not s.startswith(prefix):
    prefix = prefix[:-1]
    if not prefix: return ""
return prefix

🔐 The Monotone Shrinking Property

The prefix only ever shrinks — never grows. Once a character is removed from the right, no future string can ever add it back. This is the key invariant: after comparing against string i, the prefix is the LCP of strs[0..i].

3 Why It Works: The Invariant ▾
🔐 Loop Invariant

After processing strs[0..i], prefix is exactly the longest common prefix of those i+1 strings. Starting with i=0, prefix = strs[0] is trivially the LCP of one string. Each iteration extends the invariant by one more string.

Why shrinking one character at a time is correct: s.startswith(prefix) fails exactly when prefix has a character at some position that s doesn't match (either a different character or s is shorter). Removing the last character of prefix shortens the candidate — we keep shrinking until the remaining prefix is entirely a prefix of s. Since we shrink to the exact point of agreement, we lose no more characters than necessary.

Why we can safely stop early: If prefix becomes "", no string can make it longer again (the prefix only monotonically shrinks). Return immediately — the answer is the empty string.

⚠️ Why Not Always Start from the Shortest String?

You could sort and compare only the first and last strings (they have the most different characters), which gives O(S log n). But horizontal scan doesn't need sorting: O(S) total character comparisons is already optimal — you can't do better without reading every character at least once.

4 The Code (Annotated) ▾

Solution 1 — Horizontal Scan (Interview Pick)

def longestCommonPrefix(strs: list[str]) -> str:
prefix = strs[0] ← Seed with first string; LCP ≤ len(strs[0])
for s in strs[1:]: ← Compare prefix against each remaining string
while not s.startswith(prefix): ← If s doesn't start with prefix, shrink it
prefix = prefix[:-1] ← Drop last character (prefix can only shrink)
if not prefix: ← Empty prefix = no common prefix at all
return ""
return prefix ← Survived all strings — this is the LCP
⚠️ Common Mistake: Forgetting the Empty Check

Without if not prefix: return "", prefix[:-1] on an empty string returns "" again — the while loop runs forever. Always guard against the empty-prefix case.

Solution 2 — Vertical Scan

def longestCommonPrefix(strs: list[str]) -> str:
for i in range(len(strs[0])): ← Scan column-by-column (position i in all strings)
c = strs[0][i] ← Reference character from strs[0]
for s in strs[1:]: ← Check same position in every other string
if i >= len(s) or s[i] != c: ← s ran out of chars OR mismatch at position i
return strs[0][:i] ← Return everything before the mismatch
return strs[0] ← strs[0] is a prefix of all others

Solution 3 — Sort + Compare Endpoints

def longestCommonPrefix(strs: list[str]) -> str:
strs.sort() ← Lexicographic sort: most-different are first/last
first, last = strs[0], strs[-1] ← After sort, LCP(all) = LCP(first, last)
i = 0 ← Shared prefix length
while i < len(first) and first[i] == last[i]:
i += 1 ← Extend shared prefix one char at a time
return first[:i] ← Prefix of length i
💡 Why Sort + Compare Works

After lexicographic sort, strs[0] is the lexicographically smallest and strs[-1] is largest. Any character that differs between them will also differ somewhere in the full array. So LCP(all strings) = LCP(first, last). Two-string LCP is then a simple while-loop scan.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify the Problem

Ask: can the array be empty? Can strings be empty? The answer is yes to both — handle them. The LCP is always a prefix of every string, so it can't be longer than the shortest one.

Step 2: Start with a Concrete Example

"With [flower, flow, flight]: 'fl' is common. 'flo' fails for 'flight'. Let me think about how to systematically find it." This leads naturally to scanning.

Step 3: Propose Brute Force + Optimize

"I could compare character by column (vertical scan) — O(S). Or I can reduce a prefix candidate against each string (horizontal scan) — also O(S). Both are optimal since we must read every character. I'll go with horizontal scan as it's clean."

Step 4: State the Invariant

"At any point, prefix is the LCP of all strings seen so far. After the loop, it's the LCP of all strings. The only operation is shrinking — never growing — which makes it correct."

🗣️ What to Say to the Interviewer

"I'll use horizontal scan: seed the prefix from strs[0], then for each subsequent string, shrink the prefix until that string starts with it. The prefix monotonically shrinks — it's the LCP of all strings seen so far. After the loop we have the global LCP. Time is O(S) — total characters — since each character is removed at most once. Space is O(1) excluding the output string."
🔮 Likely Follow-Ups
  • "Can you do better than O(S)?" → No — you must read every character at least once to confirm it's shared.
  • "What if the array is huge and you query multiple times?" → Build a Trie; then each query is O(m) where m = answer length.
  • "Longest common suffix?" → Reverse all strings, run LCP, reverse the result.
  • "Handle Unicode?" → Same algorithm; Python strings are Unicode by default.

7 Summary & Key Takeaways ▾
The prefix is a monotonically shrinking candidate — each comparison can only make it shorter
Horizontal scan is O(S) time, O(1) space — optimal for this problem
Early exit when prefix becomes empty avoids unnecessary comparisons
Vertical scan is equivalent but stops at the first mismatch column instead
Sort + compare endpoints is elegant but O(S log n) due to sorting overhead
🎯 When to Use This Pattern
  • Finding shared prefix/suffix among a set of strings
  • Progressive reduction: start with a candidate and trim until all elements agree
  • Any problem where you can initialize a "max possible" answer then monotonically reduce it
