LeetCode #37

Sudoku Solver

Fill all empty cells of a 9x9 board so that every row, column, and 3x3 box contains the digits 1-9 exactly once.

Hard Backtracking Constraint Propagation Time: O(9^m) Space: O(m)
1 Understanding the Problem ▾

You are given a 9x9 grid partially filled with digits 1-9 and dots '.' for empty cells. The goal is to fill every empty cell with exactly one digit such that:

  • Every row contains the digits 1-9 with no repetition
  • Every column contains the digits 1-9 with no repetition
  • Each of the nine 3x3 sub-boxes contains the digits 1-9 with no repetition

The problem guarantees exactly one valid solution. You must modify the board in-place — not return a new board.

Key Insight

This is a constraint satisfaction problem (CSP). Each empty cell has a domain {1-9}, but constraints from its row, column, and 3x3 box eliminate most candidates. We systematically try each valid digit, recurse to the next empty cell, and undo if we reach a dead end — this "try, recurse, undo" is the essence of backtracking.

Unsolved board (partial)

5 3 . | . 7 . | . . .
6 . . | 1 9 5 | . . .
. 9 8 | . . . | . 6 .
...

Solved board (result)

5 3 4 | 6 7 8 | 9 1 2
6 7 2 | 1 9 5 | 3 4 8
1 9 8 | 3 4 2 | 5 6 7
...

Edge cases: Given cells are never modified. A cell with no valid candidates (domain = empty) means we must backtrack. The "box" containing cell (r,c) is identified by (r//3, c//3), giving box indices 0-8.

2 The Algorithm: Backtracking with Constraint Check ▾

The algorithm is a depth-first search with pruning. We process one empty cell at a time: try each digit that satisfies all three constraint groups, recurse for the rest of the board, and undo (backtrack) if recursion fails.

1 Find the next empty cell: Scan left-to-right, top-to-bottom for the first '.'. If no empty cell is found, the board is complete — return True.
2 Collect valid candidates: For the current cell (row, col), a digit d is valid if d does not appear in row row, column col, or the 3x3 box containing (row, col).
3 Try each candidate: Place d, then call backtrack() recursively. If it returns True, propagate success upward. The board is already complete.
4 Backtrack on failure: If the recursive call returns False, undo the placement (set cell back to '.') and try the next candidate digit.
5 Signal failure: If all candidates are exhausted for this cell, return False to the parent call, which will undo its own placement and try a different digit.

Decision tree for one empty cell — how backtracking looks:

Try '1' → is_valid? No (conflicts in row)
Try '2' → is_valid? No (conflicts in col)
Try '4' → is_valid? Yes → place '4'
    Recurse(next cell) → returns False (dead end deeper)
    Undo '4' → cell = '.'
Try '6' → is_valid? Yes → place '6'
    Recurse(next cell) → returns True!
Constraint Propagation Optimization

Instead of calling is_valid() which scans 9 cells in each constraint group (O(27) per check), maintain three sets: rows[r], cols[c], boxes[b] that track used digits. Validity is then an O(1) set membership test. This is 3-5x faster in practice.

3 Why It Works: The Invariant ▾
The Invariant

At every recursive call, all digits already placed on the board form a valid partial solution — no row, column, or box constraint is violated among the placed digits. We only ever place a digit after verifying it satisfies all three constraint groups.

Why this guarantees correctness: Since we only place valid digits, and the problem guarantees a unique solution exists, the DFS will eventually find a complete board where all constraints are simultaneously satisfied. The constraint check prunes dead-end branches early rather than waiting for a full board to detect violations.

Completeness: Backtracking is a complete search — it explores every valid branch. If a branch leads to a dead end (some cell has no valid candidates), it backtracks and explores other branches. Because the invariant is maintained, we never accept a false solution.

Worst-Case Complexity Note

In the absolute worst case (empty board), there are theoretically 9^81 combinations. But constraints reduce candidates dramatically. Typical Sudoku puzzles have 25-30 given digits, leaving 51-56 empty cells. With constraints, most cells have just 1-3 valid candidates. Real boards solve in well under a millisecond.

The box index formula explained: Given cell (row, col), the box is box_id = (row//3) * 3 + (col//3). This maps 9 boxes to indices 0-8: top-left box = 0, top-right box = 2, bottom-right box = 8. Integer division groups rows 0-2 together, rows 3-5 together, and rows 6-8 together.

4 The Code (Annotated) ▾

Solution 1 — Clean Backtracking (Interview Pick)

def solveSudoku(board): Modifies board in-place; returns nothing
def is_valid(row, col, num): Check if num can be placed at (row, col)
box_r, box_c = 3*(row//3), 3*(col//3) Top-left corner of the 3x3 box
for i in range(9): Check all 9 slots in row, col, and box simultaneously
if board[row][i] == num: return False Duplicate in same row?
if board[i][col] == num: return False Duplicate in same column?
if board[box_r+i//3][box_c+i%3] == num: box row cycles 0,0,0,1,1,1,2,2,2; col cycles 0,1,2,0,1,2...
return False
return True Passed all three constraint checks
def backtrack(): Returns True if board is fully solved, False otherwise
for r in range(9): Scan for next empty cell
for c in range(9):
if board[r][c] != '.': continue Skip filled cells
for num in '123456789': Try each candidate digit
if is_valid(r, c, num): Only place if valid
board[r][c] = num PLACE: tentative assignment
if backtrack(): return True RECURSE: if solved, propagate True
board[r][c] = '.' UNDO: this digit led to failure
return False No digit worked for this cell; signal failure
return True No empty cells remain; puzzle solved!
backtrack() Kick off the search (board is modified in-place)
Why is 'return False' inside the inner loop, not after?

The return False is at the depth of the first empty cell found. Once we find an empty cell and exhaust all 9 digits without success, we signal failure immediately to our parent. We handle exactly one empty cell per call, then delegate the rest to the recursive call. Continuing to scan for more empty cells in the same call would be incorrect — we'd skip the backtracking step for the current cell.

Solution 2 — Optimized with Pre-computed Constraint Sets

def solveSudoku(board):
rows = [set() for _ in range(9)] Digits used in each row
cols = [set() for _ in range(9)] Digits used in each column
boxes = [set() for _ in range(9)] Digits used in each 3x3 box (index 0-8)
empty = [] List of (r, c) for empty cells
for r in range(9): Pre-fill sets from given digits
for c in range(9):
if board[r][c] != '.':
d = board[r][c]
bi = (r//3)*3 + c//3 Box index 0-8
rows[r].add(d); cols[c].add(d); boxes[bi].add(d)
else: empty.append((r, c))
def backtrack(idx): idx = index into 'empty' list
if idx == len(empty): return True All empty cells filled!
r, c = empty[idx]
bi = (r//3)*3 + c//3
for d in '123456789':
if d not in rows[r] and d not in cols[c] and d not in boxes[bi]:O(1) check
board[r][c] = d
rows[r].add(d); cols[c].add(d); boxes[bi].add(d)
if backtrack(idx+1): return True
board[r][c] = '.'
rows[r].discard(d); cols[c].discard(d); boxes[bi].discard(d)Undo sets too
return False
backtrack(0)
Complexity Summary

Solution 1 — Simple Backtrack: Time O(9^m), Space O(m) stack depth. m = number of empty cells.
Solution 2 — Set-Optimized: Same asymptotic bounds, but O(1) validity check instead of O(27). In practice 3-5x faster. Space O(m + 9*9) for sets + stack.

5 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Backtracking Signal

Sudoku has three hallmarks of backtracking: (1) you must fill a structure cell-by-cell, (2) each placement has hard constraints, (3) you may need to undo a choice if it leads to a dead end. Anytime you see "fill a grid with constraints" or "place items without conflicts", think backtracking.

Step 2: Define the Recursive Subproblem

"Solve the board from the current state" — each recursive call handles exactly one empty cell. This decomposition is natural: solve cell k given cells 0..k-1 are filled. The base case is when there are no empty cells left.

Step 3: Write is_valid() Separately First

Separate concerns. Write and mentally test the constraint checker before the recursive solver. This also shows the interviewer you understand the constraints. Three groups to check: row, column, 3x3 box. The clever single-loop box traversal using i//3 and i%3 is worth knowing.

Step 4: State the Template Explicitly

Every backtracking problem follows: choose, explore, unchoose. For Sudoku: choose = place digit, explore = recurse, unchoose = reset to '.'. Naming this template shows pattern recognition. Mention constraint propagation optimization as a follow-up.

What to Say to the Interviewer

"This is a constraint satisfaction problem — I'll use backtracking.
I'll scan for the next empty cell, try digits 1-9, and for each check if it's valid in that row, column, and 3x3 box.
If valid, place it and recurse. If recursion succeeds, return True up the stack. If it fails, undo and try the next digit.
If all digits fail for a cell, return False — triggering backtrack in the parent.
Time: O(9^m) where m is empty cells. Space: O(m) for recursion stack.
For optimization, I'd maintain row/col/box constraint sets for O(1) validity checks, instead of O(9) per check."
Likely Follow-up Questions
  • "How do you detect no solution?" — backtrack() returns False from the very top level call
  • "Can you optimize further?" — MRV heuristic: always pick the cell with the fewest valid candidates
  • "How would you validate a board (not solve it)?" — LeetCode #36, uses the same constraint groups
  • "What if there are multiple solutions?" — collect all solutions; return True after first for this problem

7 Summary & Key Takeaways ▾
Backtracking = DFS with undo: place, recurse, undo on failure — the "choose, explore, unchoose" template
The constraint validator (is_valid) is the pruning engine — it eliminates dead branches before they expand
Pre-computed row/col/box sets reduce validity check from O(27) scanning to O(1) set membership
One empty cell per recursive call — return False immediately when a cell has no valid candidates
MRV (Minimum Remaining Values) is the most impactful further optimization: pick the most-constrained cell first
ApproachTimeSpaceNotes
Brute Force (no pruning) O(9^81) O(81) Infeasible — no constraint checking
Backtracking (Solution 1) ✓ O(9^m) O(m) m = empty cells; prunes on violation
Backtracking + Constraint Sets O(9^m) O(m + 81) O(1) validity; 3-5x faster in practice

  • "Does a valid solution exist?" or "find one valid configuration"
