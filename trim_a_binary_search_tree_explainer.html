LeetCode #669

Trim a Binary Search Tree

Given the root of a BST and a range [low, high], remove all nodes with values outside the range and return the root of the trimmed, still-valid BST.

Medium Trees Recursive Trim Time: O(n) Space: O(h)
1 Understanding the Problem ▾

You are given a Binary Search Tree (BST) and two integers low and high. Your task: remove every node whose value is outside the range [low, high], and return the root of the resulting tree. The output must still be a valid BST — the relative ordering of all surviving nodes is preserved.

💡 Key Insight

Because it's a BST, when a node is out of range, you know which subtree to salvage. If node.val < low, the node and its entire left subtree are too small — but its right subtree might still contain valid values. If node.val > high, node and right subtree are too large — the left subtree might still be valid. This 3-way decision is the entire algorithm.

Important edge cases: The root itself might be out of range — then we must return a subtree as the new root, not null. The entire tree might fall outside the range. All nodes might already be valid. Single-node trees.

2 The Algorithm: Recursive Trim (3-Way Decision) ▾

At every node, we make exactly one of three decisions. The BST ordering property is what makes each decision safe — we can guarantee which subtrees are entirely invalid.

1 Base case: If node is None, return None. Nothing exists to trim.
2 Too small: If node.val < low, discard this node and its left subtree (all are < low). Recurse into the right subtree only — it may contain values ≥ low.
3 Too large: If node.val > high, discard this node and its right subtree. Recurse into the left subtree — it may contain values ≤ high.
4 In range: This node stays. Set node.left = trim(node.left) and node.right = trim(node.right) to clean up both subtrees, then return this node.
🔑 The Recursive Return Contract

The function trim(node) always returns the root of a valid trimmed BST for that subtree. When we write node.left = trim(node.left), we're saying: "replace the left pointer with whatever the trimmed version of that subtree looks like." This pointer-rewiring pattern is idiomatic for in-place tree modification.

3 Why It Works: The Invariant ▾
🔐 The Invariant

trim(node) always returns the root of a valid trimmed BST containing exactly the nodes from the original subtree at node whose values are in [low, high]. This guarantee holds at every level of recursion — so when a parent attaches the returned value as its child pointer, the overall tree stays valid.

Why "too small" is safe to skip entirely: In a BST, all values in any node's left subtree are strictly less than that node's value. If node.val < low, then every value in its left subtree is also < low. We can discard all of them with certainty — no valid values are hidden there.

Why "too large" is safe to skip entirely: Symmetric argument — if node.val > high, every value in the right subtree is also > high. Discard them all.

Why BST ordering survives reconnection: When we set node.left = trim(node.left), the trimmed left subtree still only contains nodes whose original values were less than node.val (that was the BST invariant). We just removed some of them — but the remaining ones are still all smaller. Same logic for right. No reordering happens.

⚠️ Common Mistake: Just Returning None When Out of Range

A naive approach might do: "if node is out of range, return None." This is wrong — it would discard valid nodes in the "inward" subtree. If a node is too small, its right subtree could still contain perfectly valid values. Always recurse into the salvageable direction before returning.

4 The Code (Annotated) ▾

Solution 1 — Recursive Trim (Optimal, Interview Pick)

def trimBST(root, low, high):
if not root: ← Base case: null → return null (nothing to trim here)
return None
if root.val < low: ← Value too small: node + all of left subtree are invalid
return trimBST(root.right, low, high) ← Salvage: right subtree may have valid values ≥ low
if root.val > high: ← Value too large: node + all of right subtree are invalid
return trimBST(root.left, low, high) ← Salvage: left subtree may have valid values ≤ high
root.left = trimBST(root.left, low, high) ← Node is valid; rewire left to trimmed left subtree
root.right = trimBST(root.right, low, high) ← Rewire right to trimmed right subtree
return root ← Return this node as root of the trimmed subtree
⚠️ Why We Don't Check the Other Subtree When Out of Range

When root.val < low, we only recurse on root.right — not on root.left. In a BST, root.left.val ≤ root.val < low is always true. The entire left is guaranteed invalid — no need to inspect it. Recursing into it would waste time and risk touching nodes we're discarding.

Solution 2 — Iterative (O(1) Space)

def trimBST(root, low, high):
# Step 1: find the new valid root
while root and (root.val < low or root.val > high):
root = root.right if root.val < low else root.left
if not root: return None
# Step 2: trim the left subtree (remove nodes < low)
node = root
while node:
while node.left and node.left.val < low:
node.left = node.left.right # skip invalid, promote right
node = node.left
# Step 3: trim the right subtree (remove nodes > high)
node = root
while node:
while node.right and node.right.val > high:
node.right = node.right.left # skip invalid, promote left
node = node.right
return root
💡 Complexity Summary

Recursive: Time O(n) — visits each node once. Space O(h) call stack; O(log n) balanced, O(n) skewed.
Iterative: Time O(n), Space O(1) — avoids call stack. Preferred when tree might be deeply skewed.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the BST Property

Whenever a BST problem involves deciding what to keep or skip, ask: "Can I use the ordering property to skip entire subtrees?" Here the answer is yes — if a node is too small, its whole left is too small. This is the unlock.

Step 2: Enumerate All Cases on a Small Example

Draw a small BST, pick a range, and trace what should happen at each node: too small → go right; too large → go left; in range → keep and recurse both. This reveals the 3-way decision naturally.

Step 3: Define the Return Value Contract

The key to writing clean recursive tree code: define precisely what the function returns. Here: "trimBST(node) returns the root of the valid trimmed subtree rooted at node." With that contract, node.left = trimBST(node.left) reads naturally.

Step 4: Mention the Iterative Alternative If Asked

Offer the O(1) space iterative solution: first walk down to find the valid new root, then do two passes (one for left chain, one for right chain) to clean up invalid links iteratively.

🗣️ What to Say to the Interviewer

"Since it's a BST, I can make a 3-way decision at each node:
If node.val is below low, the node and its entire left subtree are too small,
but the right subtree might have valid values — so I return trim(node.right).
If node.val is above high, the node and its right are too large,
so I return trim(node.left).
Otherwise the node is valid — I trim both children and return the node itself.
This visits every node once: O(n) time, O(h) space for the call stack."
🔮 Likely Follow-up Questions
  • "Can you do it O(1) space?" → Iterative approach: find new root, then two while-loop passes.
  • "What if it's a regular binary tree, not a BST?" → Must visit every node; can't skip entire subtrees. Still O(n) but no shortcut.
  • "Does the result always remain a BST?" → Yes — we only remove nodes and rewire existing pointers. Relative ordering of survivors is unchanged.
  • "What if low > high?" → The entire tree gets trimmed; naturally returns None.
