LeetCode #744

Find Smallest Letter Greater Than Target

Given a sorted character array letters and a character target, return the smallest letter in letters strictly greater than target. The letters wrap around.

Easy Binary Search Upper Bound Time: O(log n) Space: O(1)
1 Understanding the Problem ▾

You have a sorted list of letters and a target letter. Your job is to find the smallest letter in the list that comes strictly after the target in the alphabet. There is a twist: if no letter comes after the target (because target is as large as or larger than all letters), the list wraps around — returning the first letter.

💡 Key Insight

We want the first position where letters[i] > target. This is the classic upper bound binary search pattern — finding the leftmost index where a condition becomes true on sorted data. If no such index exists within the array, wrap to index 0.

✓ Normal: answer within array

letters = ['c','f','j','p','v']
target = 'h'
'j' is first letter > 'h'
Answer: 'j'

↺ Wrap: target ≥ largest letter

letters = ['c','f','j']
target = 'j' (or 'z')
No letter > target
Answer: 'c' (wrap to start)

Edge Cases to consider:

  • Target equals the largest letter → wrap to letters[0]
  • Target is smaller than all letters → answer is letters[0] (the upper-bound pointer naturally lands at 0)
  • Duplicate letters in the array → handled naturally; <= skips all equal values
  • Target equals some letters in the array → those equal entries are skipped (strictly greater only)
2 The Algorithm: Upper-Bound Binary Search ▾

The array is sorted — that is the key signal for binary search. Instead of scanning every element, we use the sorted property to halve the search space at each step, zeroing in on the leftmost position where the letter beats our target.

1 Set boundaries: lo = 0, hi = len(letters). Notice that hi starts at n (the length), not n-1. This allows the search space to represent "past the end" for the wrap case.
2 Binary search loop: While lo < hi, compute mid = (lo + hi) // 2 and inspect letters[mid].
3 Compare and narrow: If letters[mid] <= target, mid is NOT a valid answer — move lo = mid + 1 (search right half). If letters[mid] > target, mid IS a valid candidate but something smaller might also qualify — move hi = mid (search left half, keeping mid in range).
4 Return with modulo: When lo == hi, the loop ends. If lo == n, every letter was ≤ target — wrap using lo % n = 0. Otherwise lo is the answer index. Return letters[lo % n].

🔑 Linear Scan Alternative (Brute Force, O(n))

Scan left to right and return the first ch where ch > target. If the loop finishes without finding one, return letters[0]. Simple and correct, but doesn't exploit the sorted property. Always mention this brute force first, then propose the O(log n) binary search improvement.

3 Why It Works: The Invariant ▾
🔐 The Loop Invariant

Throughout the binary search, the answer (the first index i where letters[i] > target) always lies in the half-open interval [lo, hi). When lo == hi, the interval collapses to a single point — and that point is definitively the answer.

Why initialize hi = n instead of n-1? If every letter is ≤ target, the conceptual answer is index n (past the end). Initializing hi = n allows the invariant to stay valid even for this edge case. The final modulo converts n to 0 for the wrap.

Why does letters[mid] <= target move lo = mid + 1? If mid equals target, it's not strictly greater — not a valid answer. If mid is less than target, it's obviously not valid. Both cases mean the answer must be strictly to the right of mid, so we advance lo past mid.

Why does letters[mid] > target set hi = mid (not mid - 1)? When mid beats target, mid is a valid answer, but we haven't yet checked whether something to its left is also valid and smaller. We set hi = mid to keep mid as the right boundary (it remains in the interval) while we search left. This is the defining move of upper-bound / leftmost-valid binary search.

⚠️ The Most Common Mistake: Using hi = n-1

If you use hi = n - 1, you can't represent "past the end" without extra if-statements. The standard upper-bound template uses hi = n precisely so the loop converges to the correct index including index n (which modulo turns into the wrap). Never change this initialization without rethinking the whole invariant.

Trace for letters=['c','f','j','p','v'], target='h':

  • lo=0, hi=5 → mid=2, letters[2]='j' > 'h' → hi=2
  • lo=0, hi=2 → mid=1, letters[1]='f' <= 'h' → lo=2
  • lo==hi==2 → letters[2%5]='j' ✓
4 The Code (Annotated) ▾

Solution 1 — Upper-Bound Binary Search (Optimal, Interview Pick)

def nextGreatestLetter(letters: list, target: str) -> str:
lo, hi = 0, len(letters) ← hi = n (past-end sentinel enables wrap case)
while lo < hi: ← loop until search space collapses to one point
mid = (lo + hi) // 2 ← midpoint, integer division (no overflow in Python)
if letters[mid] <= target: ← mid is <= target: not a valid answer, must go right
lo = mid + 1 ← discard mid and everything left of it
else: ← letters[mid] > target: valid candidate, but search left for smaller
hi = mid ← keep mid in range [lo, hi); search left half
return letters[lo % len(letters)] ← lo==n → wrap (% gives 0); else lo is the answer
⚠️ Why lo % len(letters)?

If target is ≥ every letter, the loop ends with lo = len(letters) — an out-of-bounds index. The modulo wraps it: n % n = 0, giving letters[0]. When lo is 0 to n-1, lo % n = lo (unchanged). This single operation handles both the normal and wrap cases without an if-statement.

Solution 2 — Linear Scan (Brute Force, O(n))

def nextGreatestLetter(letters: list, target: str) -> str:
for ch in letters: ← iterate left to right (array is sorted, so first match is smallest)
if ch > target: ← strictly greater than target
return ch ← first valid letter found — return immediately
return letters[0] ← no letter beat target → wrap to the beginning

Python one-liner using bisect

from bisect import bisect_right
def nextGreatestLetter(letters, target):
return letters[bisect_right(letters, target) % len(letters)] ← same O(log n) algorithm
💡 Complexity Summary

Binary Search (Optimal): Time O(log n), Space O(1)
Linear Scan: Time O(n), Space O(1)
Both are O(1) space — no auxiliary data structures required.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Sorted Array Signal

"The array is sorted. Any time I see sorted data and a search task, binary search should be my first thought. Scanning linearly wastes the sorted property."

Step 2: Classify the Search Type

"I'm not looking for an exact match — I need the first index where the element is strictly greater than target. That's the upper-bound pattern: find the leftmost position where a predicate becomes true."

Step 3: Set hi = n, Not n-1

"I initialize hi to the length of the array, not length-1. This lets the pointer reach index n if all elements are ≤ target, which naturally encodes the wrap-around case without extra branching."

Step 4: Use Modulo for Wrap

"After the loop, lo % n handles the wrap elegantly: if lo is within bounds, modulo is a no-op; if lo equals n, modulo gives 0. Clean, branch-free."

🗣️ What to Say to the Interviewer

"The array is sorted, so I'll use binary search to avoid O(n) scanning.
I want the first index where letters[i] is strictly greater than target — that's an upper-bound search.
I'll set lo=0 and hi=n (length, not length-1, to represent the wrap case as a valid endpoint).
Loop: if letters[mid] <= target, discard mid and go right: lo = mid+1.
If letters[mid] > target, mid is a valid candidate but I search left for smaller: hi = mid.
When lo==hi, return letters[lo % n]. The modulo wraps index n to 0 for free.
Time: O(log n). Space: O(1)."
🔮 Likely Follow-up Questions
  • "What if the array has duplicates?" → Handled naturally — <= skips all duplicates equal to target.
  • "Find the largest letter smaller than target?" → Lower-bound variant: flip condition to <, adjust the pointer moves accordingly.
  • "What if wrapping is not required?" → Check if lo == n and return -1 or None instead of wrapping.
  • "Can you use Python's standard library?" → Yes: bisect_right(letters, target) % n is equivalent and one line.
