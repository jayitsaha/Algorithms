Word Pattern

LeetCode 290 — Determine if a string follows a pattern via bijection
Easy Hash Tables Bijection Check O(n) Time O(n) Space
1 Understanding the Problem ▼

Given a pattern string (e.g., "abba") and a sentence s (e.g., "dog cat cat dog"), we need to check if each letter in the pattern maps to exactly one word in the sentence, and vice versa. This is a bijection — a one-to-one, two-way mapping.

Walk through the example by hand:

pattern: a b b a
words: dog cat cat dog

a → dog ✓   b → cat ✓   b → cat ✓ (matches!)   a → dog ✓ (matches!)
💡 Key Insight

A single map (pattern→word) is not enough. Consider pattern "abba" and string "dog dog dog dog": a forward map would set a→dog and b→dog — both map to "dog" — so the check wrongly passes. We need a reverse map (word→pattern) too, to catch that "dog" is already claimed by 'a' when 'b' tries to use it.

2 The Algorithm: Two-Map Bijection Check ▼

We maintain two hash maps simultaneously — one mapping letters→words, the other mapping words→letters. At each index we pair the letter and word, and check consistency in both directions.

1Split the sentence into words. If len(pattern) ≠ len(words), return False immediately.
2Create two empty maps: char_to_word and word_to_char.
3For each index i, take c = pattern[i] and w = words[i].
4If c is already in char_to_word and its mapped word ≠ w, return False.
5If w is already in word_to_char and its mapped char ≠ c, return False.
6Otherwise, record char_to_word[c] = w and word_to_char[w] = c.
7If we finish the loop without conflict, return True.

⚠️ Why one map fails

pattern="abba", s="dog dog dog dog": a single forward map sets a→dog, b→dog. Both "succeed" because we never check if "dog" is already claimed by another letter. The reverse map catches this: when 'b' tries to map to "dog", word_to_char["dog"] already equals 'a' ≠ 'b' → False.

3 Why It Works: The Invariant ▼
🔐 The Invariant

After processing the first i pairs, char_to_word and word_to_char together form a bijection: every mapped char maps to exactly one word, and every mapped word maps back to exactly one char. No two keys share a value in either direction.

Why this guarantees correctness

A valid pattern match is exactly a bijection between the set of distinct pattern letters and the set of distinct words. Our two maps enforce this directly:

  • Injectivity (no two letters → same word): The reverse map word_to_char rejects a word that is already mapped to a different letter.
  • Surjectivity-on-the-fly (no two words → same letter): The forward map char_to_word rejects a letter that already maps to a different word.
  • Length check: Equal lengths ensure no unmapped letters or extra words remain.

Edge Cases

  • Length mismatch — pattern="abc", words=["dog","cat"] → instant False.
  • Single pair — pattern="a", words=["dog"] → trivially True.
  • All same letter — pattern="aaa", words=["dog","dog","dog"] → True (one mapping a→dog).
  • All same letter, different words — pattern="aaa", words=["dog","cat","dog"] → False at index 1.
4 The Code (Annotated) ▼

Solution — Two Hash Maps (Interview Pick)

# Split sentence into words
def wordPattern(pattern: str, s: str) -> bool:
words = s.split()
# Quick length check
if len(pattern) != len(words):
return False
# Two maps for bijection
char_to_word = {}
word_to_char = {}
for c, w in zip(pattern, words):
# Forward check: char already mapped?
if c in char_to_word:
if char_to_word[c] != w:
return False
# Reverse check: word already claimed?
elif w in word_to_char:
return False
else:
# Both new — establish mapping
char_to_word[c] = w
word_to_char[w] = c
return True

Alternative — Pythonic Zip One-Liner

def wordPattern(pattern: str, s: str) -> bool:
words = s.split()
# Bijection ⟺ # unique pairs == # unique chars == # unique words
return (len(pattern) == len(words) and
len(set(zip(pattern, words))) ==
len(set(pattern)) ==
len(set(words)))
💡 Why the zip trick works

If the mapping is a bijection, then the number of unique (char, word) pairs must equal the number of unique chars AND the number of unique words. If any char maps to two words (or vice versa), the pair count would exceed one of the other counts.

⚠️ Common Mistake

Forgetting the reverse check. Using only char_to_word lets two different chars map to the same word, which is not a valid bijection.

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Recognize the Pattern

Key words: "full match", "bijection", "maps to exactly one". Anytime the problem asks for one-to-one correspondence between two sets of elements, think two hash maps.

Step 2: Identify the Failure Modes

A bijection can fail in two ways: (1) a char maps to two different words, (2) a word is claimed by two different chars. Show both cases with examples — this impresses interviewers.

Step 3: Handle Edge Cases First

Start by splitting the string and checking lengths. This eliminates a class of inputs and simplifies the main loop.

Step 4: Code the Main Loop

Walk through zip(pattern, words). At each pair, check both maps. If both are new, add to both. If either conflicts, return False.

🗣️ What to Say to the Interviewer

"This is a bijection problem — I need to verify that the mapping between letters and words is one-to-one in both directions. I'll use two hash maps: one for letter→word and one for word→letter. First I'll split the string and check lengths match. Then I'll iterate through paired elements, checking consistency in both maps. This runs in O(n) time and O(n) space where n is the number of pairs."

7 Summary & Key Takeaways ▼
✓ A bijection requires checking both directions — forward (char→word) and reverse (word→char).
✓ Always start with the length check — it's the cheapest guard.
✓ The Pythonic zip + set trick is elegant but harder to extend; prefer two maps in interviews for clarity.
✓ O(n) time, O(n) space where n = number of chars/words.
🎯 When to Use This Pattern
  • Problem says "one-to-one", "bijection", "isomorphic", or "follows the same pattern"
  • Two sequences must have matching structure
  • Each element in one set maps to exactly one element in the other, and vice versa
