LeetCode #2

Add Two Numbers

Two non-empty linked lists represent non-negative integers stored in reverse order. Add the two numbers and return the sum as a linked list.

Medium Linked List Digit by Digit with Carry Time: O(max(m,n)) Space: O(max(m,n))
1 Understanding the Problem ▾

You are given two numbers encoded as linked lists. The twist: digits are stored in reverse order — the head of each list is the least significant digit (the ones place). Your task is to add them exactly like elementary school addition — right to left, one column at a time — and return the sum as a new reversed linked list.

💡 Key Insight

Because the digits are stored least-significant-first, the list order exactly matches the order you add columns in elementary school — left-to-right through the list is ones → tens → hundreds. You never need to reverse anything. The "reversed" representation is actually the natural representation for addition.

✓ Example: 342 + 465 = 807

l1: 2→4→3 (represents 342)
l2: 5→6→4 (represents 465)
col 0: 2+5=7, carry=0
col 1: 4+6=10 → digit=0, carry=1
col 2: 3+4+1=8, carry=0
Result: 7→0→8 (= 807) ✓

⚠️ Tricky: Final carry past all digits

l1: 9→9→9 (= 999)
l2: 1 (= 1)
col 0: 9+1=10 → digit=0, carry=1
col 1: 9+0+1=10 → digit=0, carry=1
col 2: 9+0+1=10 → digit=0, carry=1
col 3: 0+0+1=1 → 0→0→0→1 (=1000)

Edge cases to handle:

  • Lists of different lengths — treat missing digits as 0
  • Final carry — if sum produces a carry past the last digit, append one more node (999+1 → needs 4 nodes)
  • Single-node lists: 5+5=10 requires two result nodes
2 The Algorithm: Digit-by-Digit Simulation with Dummy Head ▾

Simulate elementary school addition. Walk both lists simultaneously, one node at a time. At each position, add the two digits and the carry-in, write the digit (sum mod 10), compute carry-out (sum div 10), and advance. Use a dummy head node to simplify result-list construction.

1 Dummy head + cur pointer: Create a sentinel dummy = ListNode(0). Maintain cur pointing to the last result node. The final answer is dummy.next — no special case for the first node ever.
2 Loop condition covers all cases: while l1 or l2 or carry. This continues as long as any input has digits remaining OR a carry still needs to propagate. All three edge cases in one condition.
3 Safe digit extraction: v1 = l1.val if l1 else 0. If one list runs out before the other, substitute 0. No need to branch on "which list is longer."
4 Column arithmetic: total = v1 + v2 + carry. The digit to write is total % 10. The carry-out is total // 10 (always 0 or 1, since max is 9+9+1=19).
5 Append and advance: cur.next = ListNode(total % 10), then advance cur and both input pointers (if not null).

🔑 Why a Dummy Head?

Without it, you'd need a special case for the first result node: if result is None: result = ListNode(digit) else: cur.next = ListNode(digit). With a dummy, cur.next = ListNode(digit) works for every node including the first. Return dummy.next to skip the sentinel.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the start of each iteration, carry holds exactly the value carried over from all previously processed digit columns — either 0 or 1. The result list built so far correctly encodes all lower-order digits of the true sum. This invariant is maintained because: (1) we always write total % 10 to the result, and (2) we always update carry = total // 10 before the next iteration.

Why the loop condition is correct: The loop exits when all three of these are simultaneously false: l1 is null, l2 is null, carry is 0. Any remaining carry means the sum has more digits that haven't been written yet. Checking only while l1 or l2 would leave the final carry unemitted — the classic bug on this problem.

Why carry is always 0 or 1: Maximum input per column is 9 (l1 digit) + 9 (l2 digit) + 1 (carry in) = 19. So carry out = 19 // 10 = 1 at most. The carry never "compounds." This means a single integer is sufficient to track it — no carry chain is needed.

⚠️ The Most Common Bug: Forgetting the Final Carry

Writing while l1 or l2: (without or carry) breaks on inputs like 999 + 1. After three iterations, both lists are null — the loop exits. But carry is still 1. The thousands-place '1' never gets appended. Always write while l1 or l2 or carry:.

4 The Code (Annotated) ▾

Optimal Solution — Digit-by-Digit with Dummy Head (Interview Pick)

class ListNode:
def __init__(self, val=0, next=None):
self.val = val; self.next = next
def addTwoNumbers(l1, l2):
dummy = ListNode(0) ← Sentinel node; answer = dummy.next at the end
cur = dummy ← Build pointer: always the last appended result node
carry = 0 ← Carry from previous column (starts at 0)
while l1 or l2 or carry: ← Continue while any work remains — all three cases!
v1 = l1.val if l1 else 0 ← Safe extract: 0 if l1 exhausted
v2 = l2.val if l2 else 0 ← Same for l2
total = v1 + v2 + carry ← Column sum: digit + digit + carry-in
carry = total // 10 ← Carry out: 0 if total < 10, else 1
cur.next = ListNode(total % 10) ← New result node = units digit
cur = cur.next ← Advance build pointer
if l1: l1 = l1.next ← Advance l1 if not exhausted
if l2: l2 = l2.next ← Advance l2 if not exhausted
return dummy.next ← Skip sentinel; return real head of result list
⚠️ Why total % 10 and total // 10?

Max total per iteration is 9+9+1=19. 19 % 10 = 9 (digit to store in node). 19 // 10 = 1 (carry to next column). For total ≤ 9, carry is always 0. This clean integer arithmetic replaces any conditional on overflow — no if-else needed.

💡 Complexity

Time: O(max(m, n)) — single pass through both lists; max(m,n)+1 iterations at most (extra +1 for trailing carry).
Space: O(max(m, n)) — result list has at most max(m,n)+1 nodes. No other significant space used.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Reversed Representation as a Gift

The problem says digits are in reverse order. Immediately recognize: "This means the head is the ones place — exactly the right starting point for addition. I can traverse both lists left-to-right and add column by column without any reversal." Say this aloud — it shows you understand the representation.

Step 2: Name the Three Things You Track

At each position you need: (1) current l1 digit (or 0 if exhausted), (2) current l2 digit (or 0 if exhausted), (3) carry from the previous column. These three fully determine each output digit. No other state is needed.

Step 3: Mention the Dummy Head Pattern by Name

"I'll use a dummy head to simplify building the result list — it eliminates the special case for the first node." Mentioning this pattern by name is a good signal. Return dummy.next at the end.

Step 4: Emphasize the Loop Condition — This Is Where People Fail

"My loop condition is while l1 or l2 or carry — I specifically include carry to handle the case where a final carry exists after both lists are exhausted. For example, 999 + 1 should produce 1000." Trace this example to prove correctness.

🗣️ What to Say to the Interviewer

"The reversed order is actually helpful — I can add column by column, left to right through the lists.
I'll use a dummy head so I don't special-case the first result node.
Loop: while l1, l2, or carry — the 'or carry' handles the 999+1 edge case.
Each iteration: extract digits (0 if list exhausted), total = d1+d2+carry, write total%10, carry = total//10.
Time O(max(m,n)), Space O(max(m,n)) for the output list."
🔮 Likely Follow-up Questions
  • "What if stored most-significant-first?" → Reverse both lists first, apply same algorithm (or use stacks)
  • "Add two numbers without creating a new list?" → Modify the longer list's nodes in-place with same carry logic
  • "Add two binary numbers?" → Same algorithm, base 2: digit = total%2, carry = total//2
  • "Can carry be more than 1?" → Never for two single digits (max 9+9+1=19), but for k numbers summed, yes

7 Summary & Key Takeaways ▾
Reversed-order storage is not a handicap — it matches the natural column-addition order perfectly
Dummy head removes the first-node special case; always return dummy.next
Loop condition: while l1 or l2 or carry — the carry clause handles 999+1 and all trailing-carry cases
Safe extraction l1.val if l1 else 0 handles lists of different lengths with zero branches
Carry is always 0 or 1 because max column sum is 9+9+1=19, so total//10 ≤ 1

🎯 When to Recognize the Digit-by-Digit Pattern
  • Numbers stored as arrays or lists (big integer arithmetic)
  • Any operation that processes corresponding positions of two sequences simultaneously
  • A carry or borrow propagating between positions (addition, subtraction, multiplication)
  • Problem says "stored in reverse order" or "most significant digit first" — carry logic is the core
