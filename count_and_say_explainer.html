Count and Say

LeetCode #38 — Run-Length Encoding Sequence Generation
Medium String Run-Length Encoding O(n · L)
1 Understanding the Problem ▼

The count-and-say sequence starts with "1". Each subsequent term is generated by describing the previous term out loud — literally "say what you see".

For each group of consecutive identical digits, record the count followed by the digit. This is called Run-Length Encoding (RLE).

TermStringHow to read it (→ next term)
1"1"one 1 → "11"
2"11"two 1s → "21"
3"21"one 2, one 1 → "1211"
4"1211"one 1, one 2, two 1s → "111221"
5"111221"three 1s, two 2s, one 1 → "312211"
💡 Key Insight

Each term is just the verbal description of the previous one. Scan left to right, group consecutive identical digits, and for each group write count + digit. That's it — pure simulation.

2 The Algorithm: Iterative RLE Scan ▼

Starting from "1", repeat the "describe" process n − 1 times. Each description pass works like this:

1Initialize — set result = "1" (term 1)
2For each iteration (n − 1 times), scan result from left to right
3At position i, read char = result[i] and count how many consecutive copies follow
4Append str(count) + char to the next term's string
5Jump past the group (i += count) and repeat until end of string
6Replace result with the newly built string; repeat for next iteration
🔐 Core Mechanism

The inner scan uses a technique similar to Two Pointer: Same Direction — a read pointer i jumps through the string in variable-size groups. Each group produces exactly 2 output characters (count + digit).

3 Why It Works: The Invariant ▼
🔐 The Invariant

After k iterations of the outer loop, result holds the correct (k+1)th term of the count-and-say sequence.

Base case: Before any iteration, result = "1" — correct for term 1.

Inductive step: If result is the correct term k, then our RLE scan produces the correct term k+1 because:

  • We start at position 0 and count consecutive identical characters
  • After writing a group, we jump i += count, which lands exactly at the start of the next group
  • This partitions the entire string into non-overlapping, exhaustive groups — no character is skipped or double-counted
  • Each group produces the correct count + digit encoding

Edge cases:

  • n = 1 → Returns "1" immediately (0 iterations)
  • Single-character string → One group of size 1, produces "1X" correctly
  • All same digits → One large group, e.g., "111" → "31"
4 The Code (Annotated) ▼
# Start with term 1
def countAndSay(n: int) -> str:
result = "1"
# Build each subsequent term by describing the previous
for _ in range(n - 1):
next_result = []
i = 0
# Scan the current term left to right
while i < len(result):
char = result[i] # Current digit
count = 1 # At least one occurrence
# Count consecutive identical chars
while i + count < len(result) and result[i + count] == char:
count += 1
# Emit: count followed by the character
next_result.append(str(count) + char)
i += count # Jump past this group
result = "".join(next_result)
return result
⚠️ Common Mistake: Count vs Digit Order

The output is count THEN digit — "11" means "one 1", NOT "digit 1, once". Getting the order wrong produces a completely different (and wrong) sequence. Think: "How many? Of what?"

⚠️ Bounds Check in Inner Loop

The inner while must check i + count < len(result) before accessing result[i + count]. Python's and short-circuits, so this is safe — but forgetting the bounds check causes an index error at the end of the string.

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Clarify the Sequence

Build terms 1–4 by hand on the whiteboard. This shows the interviewer you understand the problem and reveals the RLE pattern.

Step 2: Identify the Core Operation

Each term is computed from the previous by one pass of run-length encoding. Isolate this as a helper function or inner loop.

Step 3: Design the RLE Scan

Use a pointer i that advances through the string. At each position, count consecutive identical characters using a nested while loop, then emit count + char.

Step 4: Wrap in Iteration

Apply the RLE scan n − 1 times starting from "1". Each iteration transforms the current term into the next.

🗣️ What to Say to the Interviewer

"This is essentially iterative run-length encoding. I'll start with '1' and apply an RLE pass n−1 times. Each pass scans left to right, groups consecutive identical digits, and builds the next string by appending count + digit for each group. Time is O(n · L) where L is the max term length, and space is O(L) for the intermediate strings."
💡 Follow-up Questions

Growth rate? The string length grows by roughly a factor of ~1.304 per term (Conway's constant). Regex? Python's re.findall(r'(.)\1*', s) can replace the inner loop. Recursive? Yes, but iterative avoids stack depth issues for large n.
