LeetCode #1986

Minimum Number of Work Sessions to Finish the Tasks

Find the minimum number of work sessions needed to complete all tasks, given each session has a fixed time limit.

Medium Dynamic Programming Bitmask DP Time: O(2^n · n) Space: O(2^n)
1 Understanding the Problem ▾

You have n tasks (n ≤ 14), each with a fixed duration. You have a session time limit — the maximum total work allowed per session. You can schedule tasks in any order, but each task must be completed entirely within a single session (no splitting). A session can end at any time; you start a new one for tasks that don't fit.

Goal: Assign all tasks to sessions to minimize the number of sessions.

💡 Key Insight: Why Bitmask DP?

With n ≤ 14 tasks, there are 214 = 16,384 possible subsets. Represent "which tasks are done" as a bitmask integer — bit i = 1 means task i is complete. DP over all 2n subsets gives us the minimum sessions to complete each subset, building from smaller subsets to larger ones.

✓ Example: tasks=[2,3,2], sessionTime=4

Session 1: task0(2) + task2(2) = 4 ✓
Session 2: task1(3) = 3 ✓
Answer: 2 sessions

✗ Why brute force fails

n=14 tasks → 14! orderings = 87 billion
Bitmask DP: 2^14 × 14 = 229K ops only
Exponential vs tractable!

Edge Cases: All tasks fit in one session (answer=1). Every task exactly fills a session (answer=n). Two tasks together exceed the limit but each fits alone. n=1 always needs exactly 1 session.

2 The Algorithm: Bitmask DP with Packed State ▾

Define dp[mask] as the best (sessions_closed, time_in_open_session) pair to complete exactly the tasks indicated by bits in mask. We want to minimize sessions first, then minimize leftover time (to pack future tasks efficiently).

🔑 The Packing Trick

Store both values in one integer: dp[mask] = sessions * (sessionTime + 1) + time_used. Taking min() of this packed integer automatically minimizes sessions first, then time_used for ties. Unpack with divmod(dp[mask], sessionTime+1).

1 Base case: dp[0] = 0 — no tasks done, 0 sessions closed, 0 time used. All other dp[mask] = INF.
2 Iterate all masks 1 to 2n−1. For each mask, try every task i that IS in the mask — imagine task i was the LAST task added to reach this mask. Its predecessor state is dp[mask ^ (1<<i)].
3 Decision: Unpack prev = dp[mask^(1<<i)] → (s, t). If t + tasks[i] ≤ sessionTime, task i fits — keep same session: candidate = s*(T+1) + (t+tasks[i]). Otherwise open a new session: candidate = (s+1)*(T+1) + tasks[i].
4 Take minimum over all tasks i in mask. Final answer: unpack dp[(1<<n)−1], get s, return s + 1 (the +1 is for the last open session).
📐 The Recurrence (T = sessionTime+1)
For each mask with bit i set:
  prev = dp[mask ^ (1 << i)]
  s, t = divmod(prev, T)
  if t + tasks[i] <= sessionTime: candidate = s*T + (t + tasks[i])
  else: candidate = (s+1)*T + tasks[i]
  dp[mask] = min(dp[mask], candidate)

Answer: divmod(dp[(1<<n)-1], T)[0] + 1
⚠️ Why +1 at the End?

The DP tracks completed (closed) sessions — a session closes only when a new one must be opened. The final session containing the last task(s) is never "closed" in this sense. So the total is dp_sessions + 1 for that still-open last session.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After computing dp[mask]: it holds the Pareto-optimal (sessions_closed, time_in_open_session) pair for completing exactly the tasks in mask, where "optimal" means: fewest sessions first, least time-used-in-current-session as tiebreaker.

Why subset DP is correct: Any assignment of tasks to sessions can be described by which tasks go in which session, but we only care about the COUNT of sessions, not their identities. By encoding the entire "done" set as a bitmask, we collapse all orderings of the same subset into one state — avoiding redundant recomputation.

Why greedy packing is locally optimal: Given a fixed set of tasks in the current session, fitting more tasks in is always at least as good as starting a new session early. A task that fits in the current time budget never benefits from being moved to a new session.

Why the tiebreaker matters: Among all ways to complete a subset with the same number of sessions, the one leaving the LEAST time in the open session gives future tasks the most room — maximizing the chance they fit without needing yet another session. This greedy second-level optimization is correct because future tasks can only benefit from more remaining room.

💡 Optimal Substructure Proof Sketch

If dp[mask] is optimal, then for any task i not in mask: dp[mask|(1<<i)] can be computed from dp[mask] by considering whether to fit task i in the current session or open a new one. No better solution exists because that would imply a better solution for mask, contradicting our assumption.

4 The Code (Annotated) ▾

Solution 1 — Bitmask DP (Optimal · Interview Pick)

def minSessions(tasks, sessionTime):
n = len(tasks) ← n ≤ 14 is the key constraint
T = sessionTime + 1 ← multiplier: packs (sessions, time) into one int
INF = float('inf')
dp = [INF] * (1 << n) ← 2^n states, one per subset
dp[0] = 0 ← base: 0 tasks done, 0 sessions, 0 time used
for mask in range(1, 1 << n): ← iterate all non-empty subsets
for i in range(n): ← try each task as the "last added"
if not (mask >> i & 1): ← task i NOT in mask? skip it
continue
prev = dp[mask ^ (1 << i)]← predecessor: mask without task i
if prev == INF: continue ← predecessor unreachable, skip
s, t = divmod(prev, T) ← unpack sessions and time_used
if t + tasks[i] <= sessionTime: ← task fits in current session
cand = s * T + (t + tasks[i]) ← same sessions, add time
else: ← must open a new session
cand = (s + 1) * T + tasks[i] ← +1 session, reset time
dp[mask] = min(dp[mask], cand) ← best option for this mask
s, _ = divmod(dp[(1 << n) - 1], T)← unpack final state
return s + 1 ← +1 for last open session
⚠️ Three Common Mistakes

1. Wrong bit check direction: We check if task i IS in mask (to find the predecessor by removing it). Don't check for tasks NOT in mask — that's the forward direction and builds duplicates.

2. Forgetting the +1: The DP counts closed sessions. The final session is never closed — it just ends. So always return s + 1, not s.

3. Tuple comparison vs packed int: Comparing (s, t) tuples works in Python but is slower than packing. The packed int trick is both correct and faster.

Solution 2 — DFS Backtracking with Pruning (Alternative)

def minSessions_dfs(tasks, sessionTime):
tasks.sort(reverse=True) ← large tasks first → better pruning
n = len(tasks)
best = [n] ← worst case: one session per task
def dfs(i, sessions):
if len(sessions) >= best[0]: return ← prune: already at best or worse
if i == n: ← all tasks placed
best[0] = len(sessions); return
seen = set() ← deduplicate sessions with same capacity
for j in range(len(sessions)):
if sessions[j] >= tasks[i] and sessions[j] not in seen:
seen.add(sessions[j])
sessions[j] -= tasks[i] ← add task to this session
dfs(i+1, sessions)
sessions[j] += tasks[i] ← backtrack
sessions.append(sessionTime - tasks[i]) ← open new session
dfs(i+1, sessions); sessions.pop()
dfs(0, [])
return best[0]
💡 Complexity Summary

Bitmask DP: Time O(2n · n), Space O(2n). For n=14: ~229K operations — instant.
DFS Backtracking: Worst-case exponential, but in practice fast with pruning. Unpredictable for adversarial inputs.
When n ≤ 14 appears in a partition/assignment problem, Bitmask DP is the canonical optimal solution.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Bitmask Signal — n ≤ 14

The constraint "n ≤ 14" (or ≤ 20) in a problem involving subsets, assignment, or partitioning is the clearest possible hint for Bitmask DP. 214 = 16,384 — always fits in memory. Say this explicitly to the interviewer.

Step 2: Define the State

"What info do I need to optimally handle the next task?" You need: which tasks are done (bitmask), and the time remaining in the current session. Encode as dp[mask] = (sessions_closed, time_used_in_open_session).

Step 3: Build the Recurrence

"For each subset (mask), I consider each task i in it as the most recently added task. The predecessor state is mask without task i. From that state, I either fit task i in the current session or open a new one." This backward construction ensures no double-counting.

Step 4: The Packed Integer Trick

"To avoid storing tuples and enable clean min() comparison, I'll pack both values: s * (T+1) + t. Python's integer comparison then automatically minimizes sessions first, time second — exactly what I want."

🗣️ What to Say to the Interviewer

"n ≤ 14 means Bitmask DP — 2^14 = 16K states, trivially feasible.
State: dp[mask] = optimal (sessions_closed, time_in_open_session) for subset mask.
Recurrence: for each bit i set in mask, the predecessor is mask without task i.
Decision: if task i fits in the current session → extend it; else open a new session.
Pack both values into one int for clean min() comparison.
Base case: dp[0] = 0. Answer: dp[all_done].sessions + 1.
Time O(2^n * n), Space O(2^n). For n=14 that's about 229K operations."
🔮 Likely Follow-up Questions
  • "What if n = 30?" → Bitmask DP infeasible (230 = 1B). Need greedy/approximation or different formulation.
  • "Tasks can be split across sessions?" → Then answer is simply ceil(sum(tasks) / sessionTime) — much easier!
  • "Maximize utilization instead of minimizing sessions?" → Same DP, different objective function.
  • "What if each session has a different time limit?" → State needs to track which sessions are available, makes it much harder.

7 Summary & Key Takeaways ▾
n ≤ 14 is the bitmask DP signal — 214=16K states is always tractable
Encode "tasks done" as a bitmask integer: bit i = 1 means task i is complete
Pack (sessions, time_used) into one integer for clean min() comparison
Iterate from smaller to larger masks; each mask pulls from its sub-masks (predecessors)
Greedy sub-decision: fit task in current session if possible; open new session otherwise
Final answer is dp_sessions + 1 because the last session is never explicitly "closed"
