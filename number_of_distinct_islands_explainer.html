LeetCode #694

Number of Distinct Islands

Given a binary grid, return the number of distinct island shapes — two islands are the same shape if one can be translated (but not rotated or reflected) to overlap the other.

Medium Graph DFS + Shape Hash Time: O(m·n) Space: O(m·n)
1 Understanding the Problem ▾

You have a 2D binary grid where 1 = land and 0 = water. A group of 4-directionally connected land cells forms an island. The question asks: how many geometrically distinct shapes appear among all islands?

Two islands are considered the same if you can slide (translate) one to sit exactly on top of the other. Rotation and reflection do not make them equal — only translation.

💡 Key Insight

Record each island's cell positions relative to its top-left corner (the DFS starting cell). Two islands with identical relative-offset sequences are the same shape — position on the grid is irrelevant. Store these signatures in a set; the set's size is the answer.

Edge cases: all-zero grid (answer 0), all-land grid (one island, answer 1), islands of size 1 (all single-cell islands are the same shape), and islands that are reflections/rotations of each other (those are still counted as distinct by this problem's rules).

2 The Algorithm: DFS + Canonical Shape Hash ▾

This problem is a variation of the classic "Number of Islands" DFS pattern, with one twist: we need to detect when two islands have the same shape rather than just counting them. The insight is to compute a canonical fingerprint (hash key) for each island's shape.

1 Scan every cell: Iterate over all (r, c). When you find grid[r][c] == 1 (unvisited land), you've found the start of a new island. This cell becomes the anchor (r0, c0).
2 DFS to map the island: From (r0, c0), explore all 4-directional land neighbors recursively. For each cell (r, c) visited, record the relative offset (r − r0, c − c0). Flood-fill each visited cell to 0 so it won't be counted again.
3 Build the shape signature: The list of offsets, in DFS visit order, uniquely encodes this island's shape. Convert it to a tuple (hashable) and add it to a Python set. The set automatically deduplicates identical shapes.
4 Return the count: After scanning the entire grid, len(seen_shapes) is the number of distinct island shapes.
🔑 Why DFS Order Is Canonical

If we always explore directions in the same fixed order (e.g., down, up, right, left), then for any given shape, DFS will always visit cells in the same relative sequence. The resulting offset list is therefore a deterministic, position-independent fingerprint of the shape. Two islands with the same geometry will produce identical offset tuples.

⚠️ Pitfall: frozenset vs. tuple

Using frozenset(shape) loses traversal order and can conflate two geometrically distinct shapes that happen to share the same coordinate set visited in different orders. Always use an ordered tuple of the DFS-ordered offsets. Alternatively, use direction-path encoding (see Solution 2) for even more robustness.

3 Why It Works: The Invariant ▾
🔐 The Core Invariant

For any island starting at anchor (r0, c0), DFS with a fixed direction order visits every cell in a deterministic sequence. The sequence of offsets (r − r0, c − c0) is therefore a canonical fingerprint. Two islands have identical fingerprints if and only if they are translates of each other.

Why flood-fill (marking cells 0) is safe and efficient: Once a cell is incorporated into an island, it must never be the start of another island. Overwriting it with 0 achieves this while also serving as our visited marker — no extra data structure needed.

Why the set correctly counts distinct shapes: Python's set uses hashing and equality. Two tuples are equal iff they have the same length and identical elements at every position. Identical offset sequences ↔ identical shapes ↔ same set slot. Different sequences → different slots → counted separately.

⚠️ The Subtle Edge Case: Path Ambiguity

Consider an S-shape and a Z-shape. If you only store the set of offsets (unordered), they can appear identical. The DFS-ordered tuple avoids this because DFS visits cells in a deterministic order dictated by the shape's topology — the S and Z produce different sequences. The even safer fix is direction-path encoding with backtrack markers (Solution 2).

4 The Code (Annotated) ▾

Solution 1 — DFS + Offset Tuple (Interview Pick)

def numDistinctIslands(grid):
rows, cols = len(grid), len(grid[0]) ← grid dimensions
seen_shapes = set() ← stores canonical shape tuples
def dfs(r, c, r0, c0, shape): ← r0,c0 = anchor (island start)
if r < 0 or r >= rows or c < 0 or c >= cols:← bounds check
return
if grid[r][c] != 1: ← water or already visited
return
grid[r][c] = 0 ← flood-fill: mark visited
shape.append((r - r0, c - c0)) ← record relative offset from anchor
dfs(r+1, c, r0, c0, shape) ← explore down
dfs(r-1, c, r0, c0, shape) ← explore up
dfs(r, c+1, r0, c0, shape) ← explore right
dfs(r, c-1, r0, c0, shape) ← explore left
for r in range(rows): ← scan the full grid
for c in range(cols):
if grid[r][c] == 1: ← unvisited land found
shape = [] ← fresh list for this island
dfs(r, c, r, c, shape) ← anchor = this starting cell
seen_shapes.add(tuple(shape)) ← list→tuple (hashable) → set
return len(seen_shapes) ← distinct shapes = set size
⚠️ Why Not frozenset(shape)?

frozenset discards order. Two differently-shaped islands can share the same cells-at-same-offsets in different connectivity arrangements — their offset sets are equal but their shapes are not. The ordered tuple preserves DFS traversal order, keeping shapes distinct when they differ.

Solution 2 — Direction-Path Encoding (More Robust)

def numDistinctIslands(grid):
rows, cols = len(grid), len(grid[0])
seen = set()
DIRS = [(1,0,'D'),(-1,0,'U'),(0,1,'R'),(0,-1,'L')] ← direction labels
def dfs(r, c, path):
grid[r][c] = 0 ← mark visited
for dr, dc, label in DIRS:
nr, nc = r + dr, c + dc
if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
path.append(label) ← record direction taken
dfs(nr, nc, path)
path.append('B') ← 'B' = backtrack marker
for r in range(rows):
for c in range(cols):
if grid[r][c] == 1:
path = []
dfs(r, c, path)
seen.add(tuple(path)) ← direction sequence is the key
return len(seen)
💡 Why Backtrack Markers Are Critical

Without 'B' (backtrack) markers, a T-shaped branch and an L-shaped branch could produce the same direction sequence (e.g., 'D','R') if the DFS explores them the same way at the branch point. The 'B' records "I returned to the parent before going in a new direction," which uniquely encodes the tree structure of the DFS traversal — and thus the exact shape.

💡 Complexity

Both solutions — Time O(m·n): Each cell is visited at most twice (once for the outer scan, once for DFS).
Space O(m·n): DFS call stack depth ≤ m·n (worst case: all land, one snake-shaped island); shape set ≤ m·n entries; no separate visited array needed thanks to flood-fill.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the base pattern

Grid + 4-directional connectivity + counting groups = classic DFS island pattern. Start with: "I'll DFS to enumerate every island." This is the same as #200 Number of Islands, just with an extra requirement.

Step 2: Identify the "distinct" requirement

"Distinct shapes" means I can't just count — I need to tell apart islands of different geometries while merging islands of the same geometry. Signal: I need a canonical key per island. Ask yourself: what makes two shapes the same? Answer: translation only.

Step 3: Derive the relative-offset encoding

"If I subtract the starting cell's (r, c) from every cell I visit, positions become shape-relative. Two islands at different grid positions but same shape will produce identical offset sequences." This is the key observation — propose it clearly.

Step 4: Choose the right data structure

Store offset-tuples in a Python set(). The set deduplicates identical shapes automatically, so len(seen_shapes) is directly the answer. No need to compare all pairs.

7 Summary & Key Takeaways ▾
Two islands are the same shape iff their DFS-ordered relative-offset tuples are equal
The anchor (first DFS cell) is subtracted from all coordinates to make signatures position-independent
A Python set of tuples deduplicates shapes in O(1) per lookup — the set's size is the answer
Flood-fill (grid[r][c] = 0) avoids a separate visited set and prevents double-counting
Direction-path encoding with backtrack markers ('B') handles all edge cases robustly

🎯 When to Use DFS + Shape Hash
  • "Count distinct/unique shapes in a grid" → DFS + canonical signature + set
  • "Two connected components are equal up to translation" → relative-offset encoding
  • "Connected land groups in a binary grid" → DFS or BFS with flood-fill
  • Anytime you need to deduplicate structural/geometric patterns → encode as hashable key
