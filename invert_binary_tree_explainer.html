LeetCode #226

Invert Binary Tree

Given the root of a binary tree, invert the tree, and return its root.

Easy Trees DFS Postorder Time: O(n) Space: O(h)
1 Understanding the Problem ▾

Inverting a binary tree means producing its mirror image. Every node keeps its own value, but its left and right children are swapped — and this must happen at every node in the tree, all the way down to the leaves.

💡 Key Insight

Inverting a tree is just swapping children at every node. For any node: swap its left and right subtrees, then recursively invert each. The recursive structure of the tree tells you how to walk it — you never need to know the tree's shape up front.

Original Tree

    4
   / \
  2   7
 / \ / \
 1 3 6 9

Inverted Tree (mirror)

    4
   / \
  7   2
 / \ / \
 9 6 3 1

Edge cases to consider: empty tree (None — return None), a single-node tree (no children to swap — return the node), a tree that's already symmetric, and trees that are just a left or right chain (skewed trees — where recursion depth = n).

This problem became famous when Max Howell, creator of Homebrew, tweeted: "Google: 90% of our engineers use the software you wrote (Homebrew), but you can't invert a binary tree on a whiteboard so f*** off." It's now a beloved joke in the industry — a reminder that interview puzzles don't always reflect engineering skill.

2 The Algorithm: Recursive DFS — Swap Children (Postorder) ▾

The recursive solution reads almost like the problem description: "If there's a node, invert its left subtree, invert its right subtree, then swap them." This is a DFS postorder traversal — children are fully processed before their parent is modified.

1 Base case: If root is None, return None immediately. A null node is its own mirror — nothing to do. This also handles all leaf nodes' null children automatically.
2 Recurse left: Call invertTree(root.left). By the time this returns, the entire left subtree is a fully mirrored version of what it was.
3 Recurse right: Call invertTree(root.right). The entire right subtree is now fully mirrored.
4 Swap children: root.left, root.right = root.right, root.left. Python's tuple assignment makes this a single atomic operation — no temp variable needed.
5 Return root: Return the current node. Its children are now the two mirrored subtrees in their swapped positions.
⚠️ Pre-order vs Post-order — Does Swap Timing Matter?

You can swap children before recursing (pre-order) or after (post-order) — both produce the same correct final result, because each swap is local to one node. The canonical version below uses post-order because it most cleanly matches the natural "build-from-leaves-up" mental model.

Iterative BFS alternative: Use a deque as a queue. Enqueue the root. While the queue isn't empty: dequeue a node, swap its children, enqueue any non-null children. Same O(n) time, O(w) space where w is the max tree width.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After invertTree(node) returns, the subtree rooted at node is the exact mirror image of its original form. Every descendant has had its left and right children swapped, and node's own pointers now point to the two correctly mirrored sub-trees in their swapped positions.

Proof by structural induction:

  • Base case (null): None is trivially its own mirror — we return it as-is.
  • Base case (leaf): A leaf has two None children. Swapping None, None = None, None is a no-op. The leaf is its own mirror.
  • Inductive step: Assume invertTree correctly mirrors any subtree of height < h. For height h: the left subtree has height < h, so invertTree(root.left) returns its correct mirror L'. The right subtree similarly gives R'. Swapping them places R' on the left and L' on the right — exactly the mirror of the original node. ✓
💡 Why the Call Stack Handles Everything

We never need to "remember" the original tree structure because each recursive call owns its own scope. When invertTree(node) is executing, it doesn't need to know its parent — it just handles its own subtree and returns. The call stack is the data structure doing the bookkeeping.

Single-node edge case: root.left and root.right are both None. Both recursive calls immediately return None. The swap is None, None = None, None — a harmless no-op. The function returns the single node unchanged. Correct.

4 The Code (Annotated) ▾

Solution 1 — Recursive DFS Postorder (Optimal & Interview Pick)

def invertTree(root): ← Takes TreeNode or None; returns TreeNode or None
if root is None: ← Base case: null node has nothing to invert
return None ← Propagates None upward so parent's child stays None
invertTree(root.left) ← Mirror the entire left subtree (recursive leap of faith)
invertTree(root.right) ← Mirror the entire right subtree
root.left, root.right = root.right, root.left ← Atomic Python swap — no temp variable needed
return root ← Return the (now-mirrored) node to the caller
⚠️ Common Mistake: Forgetting return root

If you omit return root, the function returns None implicitly. LeetCode checks the return value of the top-level call — you'd pass None back instead of the modified tree. Always return the node you processed.

Solution 2 — Iterative BFS (Queue, no recursion)

from collections import deque ← O(1) popleft — never use list.pop(0) which is O(n)
def invertTree(root):
if not root: return None ← Guard: empty tree
q = deque([root]) ← Seed queue with root; process level by level
while q: ← Until every node has been swapped
node = q.popleft() ← Dequeue front node
node.left, node.right = node.right, node.left ← Swap children at this node
if node.left: q.append(node.left) ← Only enqueue non-null children
if node.right: q.append(node.right) ← Right child (after swap = original left)
return root ← Root value unchanged; children are now mirrored
💡 Complexity Summary

Recursive: Time O(n) — visit every node once. Space O(h) call stack; h = O(log n) balanced, O(n) skewed.
Iterative BFS: Time O(n). Space O(w) queue; w = max width = O(n) for a perfect binary tree (last level has n/2 nodes).
Verdict: Both are O(n) time. Recursive is cleaner; BFS avoids recursion stack overflow for extremely deep skewed trees.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Tree DFS Pattern

Tree problems almost always decompose as: "do something at this node, then recurse on children." The problem says "invert every node" — this screams DFS. The key operation at each node is just swapping left and right children.

Step 2: Nail the Base Case First

Always start with: "What does invertTree(None) mean?" — Nothing to do; return None. Then: "What does invertTree(leaf) mean?" — Swap two None children (no-op). If your base case is solid, the recursive step writes itself.

Step 3: Apply the Recursive Leap of Faith

Don't try to mentally trace the recursion. Instead, assume invertTree magically works on any smaller subtree. Then: call it on left, call it on right, swap the results at the current node. Trust the induction.

Step 4: Proactively Offer the Iterative Solution

Say: "I can also do this iteratively with a BFS queue — swap children when dequeuing each node, enqueue non-null children. This avoids stack overflow for very deep trees." This demonstrates breadth of knowledge.

🗣️ What to Say to the Interviewer

"At each node, all I need to do is swap its left and right children — then do the same for every descendant.
This is naturally recursive: base case is null (return null); recursive case is:
invert the left subtree, invert the right subtree, then swap them at the current node.
Time O(n) — we visit every node exactly once. Space O(h) for the call stack.
For very deep skewed trees I'd use iterative BFS to avoid stack overflow."
🔮 Likely Follow-up Questions
  • "Do it without recursion?" → BFS queue (shown above) or explicit DFS stack.
  • "Symmetric Tree (#101)?" → Check if the tree equals its own inversion. Simultaneous two-pointer DFS comparing mirrored positions.
  • "What if the tree has 10⁶ nodes in a skewed chain?" → Recursion hits Python's 1000-frame default stack limit — must use iterative BFS.
  • "Invert only nodes at even depth?" → Pass depth parameter; only swap when depth % 2 == 0.

  • "Mirror", "invert", or "flip" a tree → swap children at each node via DFS
  • "Apply operation to every node" → DFS or BFS over the full tree
  • Problem result depends on a subtree being fully processed before the parent → postorder DFS
  • "Is the tree symmetric?" → same pattern, but compare rather than modify (#101)
