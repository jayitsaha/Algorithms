LeetCode #73

Set Matrix Zeroes

Given an m × n integer matrix, if an element is 0, set its entire row and column to 0. Do it in-place.

Medium Matrix Use First Row/Col as Markers Time: O(m·n) Space: O(1)
1 Understanding the Problem ▾

You have a 2D grid of numbers. Whenever you see a 0 anywhere in the grid, you must wipe out its entire row and its entire column — setting every cell in that row and every cell in that column to 0. The modification must happen in-place: you cannot allocate a copy of the matrix.

💡 Key Insight

The dangerous pitfall is cascading zeroes: if you zero a row or column the moment you find a zero, the newly-created zeroes look like "original" zeroes to later iterations — triggering extra rows and columns to be zeroed that should stay untouched. You must record which rows and columns need zeroing first, then apply the changes in a second pass.

Input: zero at (1,1)

1  1  1
1  0  1
1  1  1

Zero found at row 1, col 1

Output: row 1 & col 1 zeroed

1  0  1
0  0  0
1  0  1

Row 1 and col 1 completely zeroed

Edge cases to consider: A matrix where every cell is 0 (entire output is 0). A matrix with no zeros (nothing changes). A zero sitting in the first row or first column — requires careful handling because we reuse those cells as marker storage. A 1×1 matrix containing [0] (the single cell stays 0).

⚠️ The Naive Trap

Scanning and zeroing simultaneously is incorrect. Say you find a 0 at (1,1) and immediately zero row 1. When you later scan (1,2) — which was originally 1 — it is now 0. Your algorithm thinks "there's a zero at (1,2), zero column 2!" — but column 2 never had an original zero. The cascading corruption snowballs.

2 The Algorithm: Use First Row/Col as Markers ▾

The O(1)-space trick exploits unused capacity already in the matrix. The first row has one cell per column — it can flag "this column needs zeroing." The first column has one cell per row — it can flag "this row needs zeroing." We repurpose these cells as boolean markers. The only extra state we need: two boolean variables to remember whether the first row and first column themselves originally had zeros (because we'll overwrite them during marking).

Phase 1: Record border zeros → Phase 2: Scan interior, write markers → Phase 3: Zero interior via markers → Phase 4: Zero border if flagged
1 Check the border first. Scan row 0 to see if it contains any original zero → store in first_row_zero. Scan col 0 to see if it contains any original zero → store in first_col_zero. These booleans "back up" the information we're about to overwrite.
2 Scan the interior (rows 1+, cols 1+). For each interior cell matrix[i][j] that equals 0: write matrix[i][0] = 0 (marking row i in column 0) and matrix[0][j] = 0 (marking col j in row 0). We do NOT touch the actual cell yet.
3 Apply zeroes to the interior. Re-scan interior cells. For each (i,j) where i≥1 and j≥1: if matrix[i][0] == 0 (row i is marked) OR matrix[0][j] == 0 (col j is marked), set matrix[i][j] = 0. The markers now drive the modifications.
4 Handle the border. If first_col_zero is True, zero all of column 0. If first_row_zero is True, zero all of row 0. Do column 0 before row 0 — this is conventional; both flags are independent booleans, so order doesn't create bugs here, but col-first is the standard safe practice.
🔑 Why This Achieves O(1) Space

No auxiliary arrays are created. The marker slots (matrix[i][0] for rows, matrix[0][j] for columns) already exist in the matrix. The two boolean flags first_row_zero and first_col_zero occupy O(1) space. Total extra space = 2 booleans = O(1), regardless of matrix size.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After Phase 2 (marking pass), for every interior cell position (i≥1, j≥1):
matrix[i][0] == 0 ↔ row i contains at least one original zero
matrix[0][j] == 0 ↔ col j contains at least one original zero

Why the marking doesn't corrupt itself: During Phase 2, we only scan interior cells (i≥1, j≥1) and write to border cells (first row and first column). The border cells are never scanned in Phase 2 — so writing markers there never falsely triggers additional markings.

The matrix[0][0] corner: This cell lies at the intersection of row 0 and col 0. We use it as the row-0 marker (its value being 0 means "row 0 originally had a zero"). Column 0's "needs zeroing" state is tracked separately by first_col_zero. If we tried to use matrix[0][0] for both, we'd have a logical conflict — a cell can't simultaneously signal two independent facts.

Why Phase 3 only needs to check two conditions: Because the marking pass guarantees completeness — every row with an original interior zero has its row-marker set, and every column with an original interior zero has its col-marker set. So checking the markers in Phase 3 is equivalent to checking "was there an original zero in this row or column?"

⚠️ Common Implementation Bug

Forgetting to capture first_row_zero and first_col_zero before the marking pass. If you scan row 0 for zeros after Phase 2, the markers you wrote in Phase 2 are now zeros in row 0 — you'll think "row 0 had zeros" when those zeros were just markers you placed for column zeroing. Always capture the border flags first.

4 The Code (Annotated) ▾

Solution 1 — O(1) Space: First Row/Col Markers (Interview Pick)

def setZeroes(matrix): ← in-place, no return value needed
m, n = len(matrix), len(matrix[0]) ← row and column dimensions
first_row_zero = any(matrix[0][j]==0 for j in range(n))← Phase 1a: does row 0 have a zero?
first_col_zero = any(matrix[i][0]==0 for i in range(m))← Phase 1b: does col 0 have a zero?
for i in range(1, m): ← Phase 2: scan interior rows (skip row 0)
for j in range(1, n): ← Phase 2: scan interior cols (skip col 0)
if matrix[i][j] == 0: ← found an original zero in the interior
matrix[i][0] = 0 ← mark row i: col-0 cell = 0
matrix[0][j] = 0 ← mark col j: row-0 cell = 0
for i in range(1, m): ← Phase 3: apply zeroes to interior
for j in range(1, n):
if matrix[i][0]==0 or matrix[0][j]==0: ← row or col is marked?
matrix[i][j] = 0 ← yes → zero this interior cell
if first_col_zero: ← Phase 4a: zero col 0 if it originally had a zero
for i in range(m): matrix[i][0] = 0
if first_row_zero: ← Phase 4b: zero row 0 if it originally had a zero
for j in range(n): matrix[0][j] = 0
⚠️ Why col before row in Phase 4?

If both first_row_zero and first_col_zero are True, the order of zeroing row 0 and col 0 doesn't matter for correctness — because both operations zero all cells in their respective row/column. matrix[0][0] ends up as 0 either way. The "col first" convention exists to match the mental model: first column markers are applied, then row markers.

Solution 2 — O(m+n) Space: Two Sets (Simpler, Present This First)

def setZeroes(matrix):
m, n = len(matrix), len(matrix[0])
zero_rows, zero_cols = set(), set() ← track which rows/cols to zero
for i in range(m): ← Pass 1: collect all zero positions
for j in range(n):
if matrix[i][j] == 0:
zero_rows.add(i); zero_cols.add(j) ← record but do NOT modify yet
for i in range(m): ← Pass 2: apply zeroing
for j in range(n):
if i in zero_rows or j in zero_cols:
matrix[i][j] = 0
💡 Complexity Summary

Two Sets approach: Time O(m·n), Space O(m+n) — clean, easy to code and explain
First Row/Col Markers: Time O(m·n), Space O(1) — optimal, requires careful border handling

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Cascading Danger

"If I zero rows and columns as I scan, newly-created zeros will look like originals on later iterations, causing incorrect extra rows/columns to be zeroed. I need to separate discovery from modification."

Step 2: Propose the Easy O(m+n) Solution First

"Two passes: first collect all (row, col) positions of zeros into two sets, then use those sets to do the actual zeroing. O(m·n) time, O(m+n) space. Always start here — it's clean and correct."

Step 3: Recognize the O(1) Space Opportunity

"To get O(1) space, I need to store m row-booleans and n col-booleans somewhere that already exists. The first row has n cells (one per column). The first column has m cells (one per row). I can repurpose them as my flag arrays — but first, I save two booleans to remember if the first row and column themselves had zeros."

Step 4: Explicitly Describe the 4 Phases

Name the four phases: record border zeros → mark interior (write to border cells) → zero interior (read border cells) → zero border if flagged. This shows structured thinking and makes the approach verifiable step-by-step.

🗣️ What to Say to the Interviewer

"The naive in-place approach breaks because newly-created zeros contaminate the scan.
Simple fix: two passes — collect zero positions in two sets, then zero. That's O(m+n) space.

To get O(1) space: repurpose the first row as column-flags and first column as row-flags.
But first, I capture whether row 0 and col 0 themselves have zeros — in two booleans.
Then scan interior, write markers to border, re-scan interior to apply zeroes,
then zero the border rows/columns based on the two captured booleans. Four phases, O(1) extra."
🔮 Likely Follow-up Questions
  • "Why two booleans and not three?" → matrix[0][0] covers row-0 marker; only col-0 needs its own boolean
  • "What if the matrix has no zeros?" → Both sets empty / no markers written — Phase 3 changes nothing
  • "What if every cell is zero?" → Every marker gets set, every interior cell is zeroed, border is zeroed — correct!
  • "Can we do it in a single pass?" → No, because you need to know ALL zeros before modifying anything

7 Summary & Key Takeaways ▾
Never zero cells while scanning — cascading corrupts the scan; always separate discovery from modification
Start with two-set solution (O(m+n) space) — it's clean, easy to explain, and always correct
O(1) upgrade: repurpose the first row (column flags) and first column (row flags) as marker storage
Before using border cells as markers, capture two booleans for first-row and first-column zero status
Four clear phases: check border → mark interior → zero interior → zero border

🎯 When to Recognize the "First Row/Col as Markers" Pattern
  • You need to zero / flag entire rows and columns based on interior cell values
  • Space constraint pushes toward O(1) — look for existing wasted space in the input
  • The matrix's border can naturally store one boolean per row (col 0) and per column (row 0)
  • Classic signal combination: "m×n matrix · in-place · O(1) space"
