LeetCode #1647

Minimum Deletions to Make Character Frequencies Unique

A string s is "good" if no two different characters share the same frequency. Return the minimum number of character deletions needed to make s good.

Medium Hash Tables Greedy Time: O(n) Space: O(k)
1 Understanding the Problem ▾

We are given a string of lowercase letters. A string is "good" if no two different characters share the same frequency (count of occurrences). We want the minimum number of character deletions to achieve this property.

💡 Key Insight

We can only delete characters — never add or rearrange. Each deletion reduces exactly one character's frequency by 1. The key realization: we only care about the frequency values, not which character has which count. The problem reduces to: given a multiset of positive integers (the frequencies), make them all distinct with minimum total decrements.

✗ Not Good: "aaabbbcc"

freq: a→3, b→3, c→2
a and b both have count 3
→ CONFLICT at frequency 3
→ 2 deletions needed

✓ Good: After 2 deletions

Delete 1 'b': b→2 (conflict with c→2)
Delete 1 'c': c→1
Now: a→3, b→2, c→1
All unique → Good!

What counts as "good"? Every non-zero character frequency must be unique. If a character is completely deleted (frequency becomes 0), that's fine — multiple characters can all have 0 occurrences. Only positive frequencies must be distinct.

Edge cases: Single character (trivially good). All characters have unique frequencies already (answer: 0). All same characters — one char with count = n (trivially good). All distinct characters — n chars each with count 1 — they all share frequency 1! We'd need to delete n−1 of them to leave only one character at count 1, or use the greedy approach to assign them unique slots like 1, 2, 3, ... but we can only decrease, so n−1 deletions minimum in this worst case.

2 The Algorithm: Greedy Decrement with Used-Set ▾

The optimal strategy: for each character's frequency, if that frequency slot is already taken, keep decrementing by 1 (each decrement = one deletion) until we find a free slot — or reach 0. This greedy approach always uses the minimum deletions because we stop the instant we find an available slot.

1 Count frequencies: Use Counter(s) to map each character to its count. For "aaabbbcc" → {a:3, b:3, c:2}. Extract just the values — we no longer care which char has which count.
2 Initialize used-set: A set tracking which frequency values are already "claimed" by characters processed so far. Empty at start.
3 For each frequency f: While f is in the used-set (conflict!) AND f > 0: decrement f by 1 and increment the deletions counter. Each iteration represents deleting one occurrence of that character.
4 Claim the slot: After the while loop, if f > 0 (we didn't delete all occurrences), add f to used-set. If f = 0, we don't add it — frequency 0 is "all deleted" and multiple characters can share this state.
5 Return total deletions.
🔑 Why This Greedy Choice is Optimal

For a frequency f conflicting with an already-claimed slot, the closest available slot below it is some f−k. We need exactly k deletions to reach it — we can't skip there in fewer steps, since each deletion reduces the count by exactly 1. Decrement-until-free is therefore the minimum-cost resolution for every conflict. Since we greedily minimize each conflict independently and conflicts don't interfere with each other (a claimed slot stays claimed), the total is globally optimal.

Example trace for "aaabbbcc" — frequencies [3, 3, 2]:

a: f=3→3 not in {} → claim itused={3}, del=0
b: f=3→3 in {3} → conflict! decrement: f=2, del=1del=1
b: f=2→2 not in {3} → claim itused={3,2}, del=1
c: f=2→2 in {3,2} → conflict! decrement: f=1, del=2del=2
c: f=1→1 not in {3,2} → claim itused={3,2,1}, del=2

Final answer: 2 deletions → resulting string: a×3, b×2, c×1

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing each character's frequency, the used-set contains exactly the set of distinct positive frequency values currently claimed by all already-processed characters. Every value in used-set corresponds to exactly one character's final (after-deletion) frequency. The structure is always a valid partial assignment of unique frequencies.

Why decrement-by-1 is always optimal (not by more): Suppose frequency f is taken and the next free slot below is at f−k. We must perform exactly k decrements (k deletions) to land there. There is no way to land at f−k with fewer than k deletions because each deletion reduces by exactly 1. Therefore, for every conflict, decrement-until-free is both necessary and sufficient for minimum cost.

Why order of processing doesn't affect correctness (but may affect intuition): No matter which character we process first, each one greedily claims the largest available slot ≤ its original frequency. The total "wasted" slots (deletions) is always the same. Sorting descending is optional but helps intuition — larger frequencies naturally take higher slots, leaving small slots for low-frequency characters.

⚠️ Critical: Don't add 0 to the used-set!

When a frequency decrements all the way to 0, we have effectively deleted all occurrences of that character. We do not add 0 to used-set. Why? Because multiple characters can all be fully deleted — they all "share" frequency 0 — and that's allowed. If we added 0, the next character reaching 0 would try to decrement below 0, which is logically nonsensical (you can't delete more characters than exist).

Proof of minimum: Consider any valid deletion plan. For each character, it ends at some non-negative frequency. All positive end-frequencies must be distinct. The number of deletions for character c is original_freq(c) − final_freq(c). To minimize total deletions, we want each final_freq as large as possible. The greedy approach maximizes each character's final_freq by stopping at the first available slot — you can't do better without violating uniqueness.

4 The Code (Annotated) ▾

Solution 1 — Greedy Decrement with Used-Set (Interview Pick)

from collections import Counter
def minDeletions(s: str) -> int:
freq = list(Counter(s).values()) ← Get frequency of each character; discard which char has which count
used = set() ← Set of frequency values already "claimed" by processed characters
deletions = 0 ← Accumulate total characters we must delete
for f in freq: ← Process each character's frequency independently
while f > 0 and f in used: ← While slot is taken and we haven't deleted all occurrences...
f -= 1 ← Delete one occurrence: try the next lower frequency slot
deletions += 1 ← Count this deletion
if f > 0: ← Only claim a slot if this char still has occurrences remaining
used.add(f) ← Mark this frequency as taken; no future char can use this value
return deletions ← Minimum deletions to make all frequencies unique
⚠️ Why is the guard "while f > 0 and f in used" — not just "while f in used"?

Without f > 0, if a character's frequency decrements to 0 and 0 happens to be in used (which we avoid adding, but hypothetically), the loop would go to −1, −2, etc. The guard ensures we stop at 0 — "we've deleted all occurrences, there's nothing left to keep." The if f > 0 before used.add(f) then skips adding 0, since frequency 0 is "all deleted" and is not a unique slot we need to protect.

Solution 2 — Sort + Sweep (slightly simpler logic, no set needed)

def minDeletions(s: str) -> int:
freq = sorted(Counter(s).values(), reverse=True) ← Sort descending: process highest freqs first
deletions = 0
for i in range(1, len(freq)): ← Compare each frequency to the one before it
if freq[i] >= freq[i-1]: ← Must be strictly less than previous (sorted descending = strictly decreasing)
new_f = max(0, freq[i-1] - 1) ← Reduce to just one below previous; clamp to 0
deletions += freq[i] - new_f ← Deletions = how much we reduced this frequency
freq[i] = new_f ← Update for next comparison (this becomes the "previous")
return deletions

Solution 3 — Brute Force: Sorting + Bucket Check

def minDeletions(s: str) -> int: ← Conceptually simplest but same complexity
freq = Counter(s) ← Count frequencies
deletions = 0
seen = set()
for count in freq.values(): ← For each character's frequency
while count > 0 and count in seen: ← Same greedy decrement — identical to Solution 1
count -= 1
deletions += 1
if count > 0: seen.add(count)
return deletions ← Solution 1 and 3 are equivalent; Solution 1 is cleaner
💡 Complexity Summary

Solution 1 (Greedy + Set): O(n) to build Counter; O(k · max_freq) for greedy loop, but since k ≤ 26 and total while-loop iterations are bounded by Σfreq = n, the greedy part is O(n). Total: O(n), Space: O(k) where k = distinct chars ≤ 26 = O(1).
Solution 2 (Sort + Sweep): O(n + k log k). Since k ≤ 26, this is also O(n) in practice. No set needed — uses the sorted array directly.
Both optimal. Solution 1 is preferred in interviews for its clean, self-documenting while-loop structure.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify the constraint

Ask: "Can we only delete characters, or also insert/replace?" (Only delete.) "Are empty characters allowed?" (Implicitly yes — fully deleting a character is fine.) This confirms we can only reduce frequencies, never increase them.

Step 2: Reframe the problem

Recognize that the identities of characters don't matter — only their frequencies. Restate: "Given a multiset of positive integers, reduce some values (min total reduction) so all are distinct." This reframing makes the greedy approach obvious.

Step 3: Identify the greedy insight

For each frequency that conflicts, the cheapest fix is to decrement by 1 and check again. We stop as soon as we find a free slot. This is O(1) amortized per character since total decrements ≤ n (total characters in the string).

Step 4: Handle the edge case of frequency 0

Frequency 0 = "this character was completely deleted." Multiple characters can all be at 0 — that's legal. Do NOT add 0 to the used-set. Do stop decrementing at 0 (can't delete negative characters).

🗣️ What to Say to the Interviewer

"The problem reduces to: given a list of integers (character frequencies), make them all distinct with minimum total decrements.

I'll use a greedy approach with a set to track used frequency slots.
For each character's frequency, if that slot is taken, I keep decrementing by 1 — each decrement is one deletion — until I find a free slot or reach 0.
I don't add 0 to the used-set since multiple characters can all be completely deleted.

Time: O(n) — counting is O(n), and total decrements across all characters is also bounded by n.
Space: O(k) where k is the number of distinct characters, at most 26 for lowercase — effectively O(1)."
🔮 Likely Follow-up Questions
  • "Can you solve it without a set?" → Yes — sort frequencies descending, sweep, ensure each is strictly < previous. O(k log k). Solution 2 above.
  • "What if we could also insert characters?" → Much harder: we can now increase frequencies, which opens higher slots. Requires a different approach.
  • "What's the maximum possible answer?" → At most n−1 deletions (e.g., n distinct characters all at frequency 1 — keep one, delete n−1). Bounded by n.
  • "Related problem: Find if a string can be rearranged with all unique frequencies naturally?" → Check: after Counter, are all values distinct? O(k) check.

Key reframe: discard character identities — only the multiset of frequencies matters
Greedy: for each conflicting frequency, decrement by 1 (one deletion) until a free slot is found
Never add 0 to the used-set — "fully deleted" characters don't occupy a unique slot
