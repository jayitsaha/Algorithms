LeetCode #2246

Longest Path With Different Adjacent Characters

Find the longest path in a tree where no two adjacent nodes share the same character label.

Hard Graph DFS + Two Max Children Time: O(n) Space: O(n)
1 Understanding the Problem ▾

You are given a tree of n nodes (numbered 0 to n-1), each with a single lowercase character label stored in string s. The tree is rooted at node 0, and each node's parent is given by array parent (with parent[0] = -1). A path is any sequence of adjacent nodes with no repeated nodes. You need to find the longest such path where every pair of consecutive nodes has different character labels.

💡 Key Insight

Every path in a tree has a unique apex node — the highest node (closest to root) on that path. When a path passes through an apex, it consists of at most two downward arms. To find the longest valid path overall, compute the longest path through each node as the apex, and take the maximum. At each apex, we only need the two longest valid arms from its children.

✓ Valid Path (all adjacent chars differ)

Nodes: 3(b) → 1(a) → 4(c)
b≠a ✓, a≠c ✓
All adjacent different → length 3

✗ Blocked Path (adjacent chars match)

Nodes: 1(a) → 0(a) → ...
a = a at edge 1→0 → BLOCKED!
Cannot extend path through this edge

Edge Cases: Single node (answer = 1). All nodes same character (answer = 1 — no two adjacent can be connected). A linear path with alternating characters (answer = n). A star with all children the same char as root (answer = 1).

2 The Algorithm: DFS with Two Max Children ▾

We perform a single post-order DFS from the root. At each node, we ask: "What is the longest valid path that bends here?" We collect the lengths of all valid downward arms (arms from children whose character differs from ours), track the top two, and update the global answer. The DFS function returns the best single arm — what the parent can use if it extends through this node.

Example: parent=[-1,0,0,1,1,2], s="aabccc"
node→char: 0→a, 1→a, 2→b, 3→b, 4→c, 5→c
0(a)
/ \
1(a) 2(b)
/ \ \
3(b) 4(c) 5(c)
Best path: 3(b)→1(a)→4(c) — length 3
1 Build children list: For each node i from 1 to n-1, append i to children[parent[i]]. Root (node 0) has no parent.
2 DFS returns arm length: dfs(node) returns the length of the longest valid path starting at node and going downward. A leaf returns 1.
3 Filter child arms: Recurse into every child c. Only count dfs(c) toward our arm selection if s[c] != s[node] — the edge must be valid.
4 Track top two: Among valid child arms, maintain top1 (longest) and top2 (second longest). Update in a single scan — no sorting needed.
5 Update global answer: ans = max(ans, top1 + top2 + 1). The +1 is for the current node itself (the apex of this path). Return top1 + 1 to the parent.
🔑 Why Only Two Arms?

A simple path in a tree cannot branch — it's a straight line through nodes. When it passes through apex node u, it enters from exactly one child subtree and exits into at most one other. The two longest arms give the optimal non-branching path through u. Adding a third arm would create a branch, which is not a valid simple path.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After dfs(node) completes, its return value is exactly the length of the longest valid path that starts at node and extends downward through its subtree, respecting the different-adjacent-characters rule at every step.

Why correctness is guaranteed: Every path in the tree has exactly one apex node. By running DFS on every node (via the recursive descent), we consider all possible apex nodes exactly once. At each apex, the formula top1 + top2 + 1 correctly captures the longest bend — it's the best left arm + best right arm + the apex node itself. Since DFS returns the pre-validated arm length (all character constraints within the subtree are already satisfied), we only need to check the one immediate edge from child to parent.

Why blocked edges don't corrupt results: We still call dfs(child) for a blocked child (same character as parent). This correctly computes paths that bend within that child's subtree — those paths just don't extend through the edge to the parent. The child's subtree is still explored; it just contributes 0 to the parent's arm selection.

⚠️ Common Mistake: Forgetting +1 for the Apex Node

A single node by itself is a valid path of length 1. When computing top1 + top2 + 1, the +1 counts the apex node itself. Without it, a node with two arms of length 2 would report a path of length 4 instead of the correct 5. Similarly, the returned arm includes the current node: top1 + 1, not just top1.

Leaf case: A leaf has no children. top1 = top2 = 0. Path through it = 0+0+1 = 1. Returned arm = 0+1 = 1. Correct — a single node is a valid path of length 1.

4 The Code (Annotated) ▾

Solution 1 — DFS with Two Max Children (Optimal, Interview Pick)

def longestPath(parent: List[int], s: str) -> int:
n = len(parent)← number of nodes 0..n-1
children = [[] for _ in range(n)]← adjacency list: node → list of children
for i in range(1, n): ← skip root (parent[0]=-1)
children[parent[i]].append(i) ← add i as a child of parent[i]
ans = 1 ← minimum path = 1 node (single node always valid)
def dfs(node): ← returns best arm length going downward from node
nonlocal ans
top1 = top2 = 0 ← best and second-best valid child arm lengths
for child in children[node]: ← process each child
child_len = dfs(child) ← recurse first (post-order) — always explore
if s[child] == s[node]: ← same char? edge is blocked
continue ← skip this arm — cannot extend through this edge
if child_len > top1: ← new best arm?
top2 = top1 ← old best becomes second-best
top1 = child_len ← update best
elif child_len > top2: ← beats second-best but not best?
top2 = child_len ← update second-best only
ans = max(ans, top1 + top2 + 1) ← longest path through node as apex
return top1 + 1 ← best arm (includes this node) for the parent
dfs(0) ← start DFS from root
return ans ← global best across all possible apex nodes
⚠️ Python Recursion Limit

Python's default recursion limit is ~1000. For a linear chain of n=10⁵ nodes, recursive DFS will crash with RecursionError. In an interview, mention: "I'd add sys.setrecursionlimit(200000) or convert to iterative DFS using an explicit stack with a post-order traversal." The iterative version processes nodes after all their children (topological order from leaves to root).

Solution 2 — Iterative (Stack-Safe, same O(n) complexity)

def longestPath(parent, s): ← iterative DFS; safe for large inputs
n = len(parent)
children = [[] for _ in range(n)]
for i in range(1, n): children[parent[i]].append(i)
arm = [1] * n ← arm[i] = best arm downward from node i
ans = 1
# Process in reverse BFS order (leaves first)
order = []
stack = [0]
while stack:
node = stack.pop()
order.append(node)
for c in children[node]: stack.append(c)
for node in reversed(order): ← process children before parents
top1 = top2 = 0
for child in children[node]:
if s[child] == s[node]: continue
cl = arm[child]
if cl > top1: top2=top1; top1=cl
elif cl > top2: top2=cl
ans = max(ans, top1+top2+1)
arm[node] = top1 + 1
return ans
💡 Complexity Summary

DFS (recursive): Time O(n) — each node visited once; Space O(n) — recursion depth + children list
Iterative: Time O(n); Space O(n) — explicit stack replaces call stack; no recursion limit issue
Brute Force (DFS from every node): Time O(n²) — TLE for n=10⁵

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Tree Path Pattern

Key signal: "longest path in a tree." The apex-node enumeration idea is the fundamental trick for all tree-path problems (Diameter of Binary Tree, Binary Tree Maximum Path Sum). When you see a path-in-tree problem, immediately think: "Every path has a unique highest node. I'll enumerate all possible apex nodes via DFS."

Step 2: Separate Two Concerns

The DFS function does two things: (a) It returns a value — the best arm downward — used by its parent. (b) It updates a global answer using the local path-through-apex formula. Don't confuse these two: you're returning an arm, not the full path.

Step 3: Handle the Constraint as a Filter

Always recurse into every child (to explore their subtrees). Only filter the arm after: if s[child] == s[node], that arm can't extend through this edge. Still call dfs(child) — paths within the child's subtree are still valid.

Step 4: Mention the Recursion Limit Proactively

For Python, mention the recursion limit concern without being asked. Propose the iterative solution (BFS order traversal, then process in reverse). This demonstrates production-grade thinking and often impresses interviewers.

🗣️ What to Say to the Interviewer

"This is a tree-path problem. Every path has a unique apex node — the highest node on the path.
I'll run DFS from the root. At each node, I collect the longest valid arms from children
that have a different character — valid means the adjacent-char constraint holds.
I track the top two arms, compute path-through-apex = top1 + top2 + 1, update global max.
DFS returns top1 + 1 — the best single arm extending downward, used by the parent.
Time O(n) — each node visited once. Space O(n).
Note: Python's recursion limit could be an issue for n=10^5. I'd use iterative DFS with
topological ordering (BFS then process in reverse) to avoid RecursionError in production."
🔮 Likely Follow-up Questions
  • "What if it's a general graph (not a tree)?" → Track visited nodes to avoid cycles; DFS still works
  • "What if you need the actual path nodes?" → Track which child gave top1 and reconstruct upward via parent links
  • "How would you handle weighted edges?" → Same structure; arm lengths become weighted sums
  • "What if characters must alternate (A-B-A-B)?" → Add state to DFS: what was the last character?

7 Summary & Key Takeaways ▾
Every tree path has a unique apex — enumerate all apex candidates via post-order DFS
At each apex, only the top 2 valid child arms matter (a path cannot branch)
Always recurse into blocked children — their subtrees contain valid internal paths
DFS returns arm length (top1+1), not the full path — the parent uses this for its own calculation
Time O(n): each of the n nodes is visited exactly once; Space O(n): stack + children list
For Python + large n: use iterative DFS (BFS traversal order reversed) to avoid RecursionError
