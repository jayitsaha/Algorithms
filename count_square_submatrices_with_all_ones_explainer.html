LeetCode #1277

Count Square Submatrices with All Ones

Given an m×n binary matrix, count the total number of square submatrices (of any size) that contain all ones.

Medium Dynamic Programming 2D Grid DP · Maximal Square Family Time: O(m·n) Space: O(1) in-place
1 Understanding the Problem ▾

You're given a binary matrix — only 0s and 1s. A square submatrix of size k×k is a contiguous region of k rows and k columns where every single cell is 1. You must count ALL such squares across ALL sizes — 1×1, 2×2, 3×3, and up.

The key constraint is that the square must be entirely filled with ones. A single 0 inside the k×k region disqualifies it. Squares can overlap — for example, two different 2×2 regions that share a column both count.

💡 The Core Insight

Every valid square submatrix has a unique bottom-right corner. At a cell (i,j), the question becomes: what is the largest all-ones square whose bottom-right corner is exactly here? If the answer is k, then this corner accounts for exactly k squares — a 1×1, a 2×2, …, a k×k. Sum all these values across every cell and you have your total count. This is the same DP used in Maximal Square (#221).

Example Input (3×4)

0  1  1  1
1  1  1  1
0  1  1  1

Answer: 15 total squares

Ten 1×1 squares (10 ones in matrix)
Four 2×2 squares
One 3×3 square
10 + 4 + 1 = 15

Edge cases to consider: All-zeros matrix → answer 0. Single cell → answer = that cell's value. Single row/column → only 1×1 squares possible. All-ones matrix → answer = sum of 1+2+...+min(m,n) triangular terms per diagonal.

2 The Algorithm: Min-of-Three-Neighbors DP ▾

We define dp[i][j] = the side-length of the largest all-ones square whose bottom-right corner is at (i,j). We compute this using the elegant recurrence below, processing cells left-to-right, top-to-bottom.

1 Base cases (first row and first column). A cell on the first row or first column can never be the bottom-right corner of anything larger than a 1×1 square — there's nothing above or to the left to extend from. So dp[i][j] = matrix[i][j] for edge cells (0 stays 0, 1 stays 1).
2 Interior 0-cells. If matrix[i][j] == 0, no square of any size can have its bottom-right corner here. dp[i][j] = 0.
3 Interior 1-cells: apply the recurrence. For a square of side k to end at (i,j), all three neighbors must support a square of side k−1. The minimum of the three neighbors is the bottleneck: dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1.
4 Accumulate the count. After computing dp[i][j], add it to total. Since dp[i][j] = k means this cell is the bottom-right corner of squares of sizes 1, 2, …, k, each dp value contributes exactly that many squares.
5 Space optimization. We can modify the matrix in-place, overwriting each 1 with its dp value. No extra array needed. Previous dp values in the same row/column are already computed before we need them.
📐 The Recurrence Relation
if matrix[i][j] == 0:   dp[i][j] = 0
if i == 0 or j == 0:   dp[i][j] = matrix[i][j]
otherwise:             dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1

answer = Σ dp[i][j] for all (i, j)
🔑 Why Exactly Three Neighbors?

For a k×k all-ones square to end at (i,j), three conditions must all hold:

  • Top (i-1, j): a (k-1)×(k-1) square ends here — gives us the top-right sub-block
  • Left (i, j-1): a (k-1)×(k-1) square ends here — gives us the bottom-left sub-block
  • Diagonal (i-1, j-1): a (k-1)×(k-1) square ends here — gives us the top-left sub-block

These three (k-1)×(k-1) sub-squares together tile the k×k region perfectly. The min is the weakest link — if any neighbor is less than k-1, we cannot form a k×k square regardless of the others.

3 Why It Works: DP Invariant and Proof Sketch ▾

Why is this Dynamic Programming? Two pillars make DP applicable:

Optimal Substructure

The largest all-ones square ending at (i,j) depends only on the same values at the three immediate neighbors. No cell further away needs to be consulted — it's fully captured by the three nearest ones.

Overlapping Subproblems

A brute-force approach checking every (i,j,k) triple re-scans the same regions repeatedly. Each cell (i,j) appears as a "sub-region" in many different queries. DP computes each cell's answer exactly once in O(1) and caches it in-place.

🔐 The Invariant

After processing (i,j), the value stored there equals the side-length of the largest all-ones square whose bottom-right corner is exactly at (i,j). This value simultaneously counts how many distinct all-ones squares have their bottom-right corner here: a value of k means squares of sizes 1×1, 2×2, …, k×k all end here.

Proof by induction that the recurrence is correct:

  • Base case: Edge cells are trivially correct — a 1×1 can form only if the cell is 1.
  • Inductive step: Assume dp is correct for all previously-processed cells. For (i,j)=1 with all three neighbors computed: let m = min of three neighbors. The three (m×m) sub-squares at (i-1,j), (i,j-1), (i-1,j-1) share overlapping regions that together tile the entire (m+1)×(m+1) area ending at (i,j). Since matrix[i][j]=1, this entire area is all-ones. We cannot do better than m+1 because whichever neighbor achieved the minimum is the bottleneck. QED.
⚠️ Why Summing dp Works — No Double Counting

Every all-ones square has a unique bottom-right corner. Two different squares cannot share the same bottom-right corner unless they are the same square (same top-left corner implied by same bottom-right and same size). So partitioning squares by their bottom-right corner is a perfect partition — exactly one way to assign each square. Summing over all cells sums over all partitions → counts each square exactly once.

total = 0 ← will hold sum of all dp[i][j]
for i in range(m): ← scan top-to-bottom
for j in range(n): ← scan left-to-right
if matrix[i][j] and i > 0 and j > 0: ← interior 1-cell: apply recurrence
matrix[i][j] = 1 + min( ← overwrite with dp value (min neighbor + 1)
matrix[i-1][j], ← top neighbor (already computed this pass)
matrix[i][j-1], ← left neighbor (just computed this iteration)
matrix[i-1][j-1]) ← diagonal neighbor

⚠️ Two Subtle Points

1. In-place is safe: When we process (i,j), we need matrix[i-1][j] (already overwritten in the previous row), matrix[i][j-1] (just overwritten this row), and matrix[i-1][j-1] (overwritten previous row, earlier column). All three are already finalized — no dependency issue.

2. The condition skips 0-cells AND edge cells: If matrix[i][j] == 0 the condition is false — we skip the recurrence and just add 0. If i == 0 or j == 0 the condition is also false — we leave the edge value unchanged (0 or 1) and add it. Both are correct because edge cells can only form 1×1 squares and 0-cells form none.

def countSquares_brute(matrix):
m, n = len(matrix), len(matrix[0])
total = 0
for i in range(m):
for j in range(n):

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

"Counting/optimizing regions in a binary 2D grid" → 2D DP. The fact that you need ALL sizes, not just the max, narrows it to: define a dp value per cell, then aggregate. The bottom-right corner framing is the standard move for any "square ending at cell" DP.

Step 2: Define the State

Ask: "What single number at each cell would let me avoid recomputation?" → dp[i][j] = side-length of largest all-ones square with bottom-right corner at (i,j). This is exactly the state in Maximal Square (#221). If you've seen that problem, you already have this.

Step 3: Derive the Recurrence

Think geometrically: if I want a k×k square ending at (i,j), I need (k-1)×(k-1) squares ending at (i-1,j), (i,j-1), and (i-1,j-1) — covering the top strip, left strip, and interior respectively. Binding constraint = min of three. So dp[i][j] = min(top, left, diagonal) + 1.

Step 4: Connect to Counting

The non-obvious leap: dp[i][j] = k means this cell is the corner of squares of sizes 1 through k — that's k distinct squares. Summing dp[i][j] over all cells counts every square exactly once, since each square has a unique bottom-right corner. Say this explicitly to the interviewer.

🗣️ What to Say to the Interviewer

"I notice this is a 2D grid problem where I need to count all-ones squares of any size.
I'll define dp[i][j] as the side-length of the largest all-ones square ending at (i,j).

For edge cells: dp = matrix value (0 or 1) — base case.
For interior 1-cells: dp[i][j] = min(top, left, diagonal) + 1.
The min captures that all three sub-regions must agree for a larger square to fit.

Key insight for counting: dp[i][j] = k means exactly k squares end at (i,j).
So answer = sum of all dp[i][j]. I can do this in-place for O(1) space.
This is the Maximal Square recurrence — just sum instead of max. Time O(m·n)."
🔮 Likely Follow-up Questions
  • "Can you do it without modifying the input?" → Use a separate dp array O(mn), or a rolling row (O(n) space with careful prev-diag tracking).
  • "What about Maximal Square (#221)?" → Same recurrence, but track max(dp[i][j]) and return max².
  • "What about all-ones rectangles?" → Much harder: histogram approach per row + largest rectangle in histogram (#85).
  • "Count squares with exactly one zero?" → Different approach; might use 2D prefix sums.

7 Summary & Key Takeaways ▾
dp[i][j] = side-length of largest all-ones square with bottom-right at (i,j)
A dp-value of k contributes exactly k squares to the answer (sizes 1×1 through k×k)
Recurrence: min(top, left, diagonal) + 1 — bottleneck principle: all three sub-regions must support the larger square
In-place modification of the matrix gives O(1) extra space — no separate dp array needed
Identical recurrence to Maximal Square (#221) — just sum all dp values instead of tracking the max
Every square has a unique bottom-right corner → summing dp[i][j] counts each square exactly once, no duplicates

🎯 When to Recognize this DP Pattern
  • "Count/find largest square region in binary grid" → min-of-three-neighbors DP
  • Any 2D grid where the answer at (i,j) depends only on immediate top, left, and diagonal → O(mn) tabulation
  • "Count something of ALL sizes" where each cell contributes its dp value as a count → sum all dp values
  • Connection to Maximal Square: if problem says "largest" instead of "count all" → max instead of sum
