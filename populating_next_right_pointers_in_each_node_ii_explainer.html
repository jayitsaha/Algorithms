LeetCode #117

Populating Next Right Pointers in Each Node II

Fill the next pointer of every node to point to its next right neighbour on the same level. Works on any binary tree — not just perfect. Rightmost nodes get NULL.

Medium Trees BFS Level Order Time: O(n) Space: O(1) optimal
1 Understanding the Problem ▾

Every node in this binary tree has an extra field: next. By default it's NULL. Your job is to set next on each node to point to the node immediately to its right on the same tree level. The rightmost node on each level gets next = NULL.

The tricky part: this is an arbitrary binary tree, not a perfect one. Some nodes may have one child, or none at all. You cannot assume that a gap on one side means there's a gap on the other side too. You must handle every combination.

💡 Key Insight

Think of each level as a linked list waiting to be wired up. If you could visit all nodes on level k in strict left-to-right order, you'd connect consecutive ones via next. BFS gives you exactly that order. The cleverness is in doing it without a queue by reusing the next pointers you've already set as free traversal infrastructure.

2 The Algorithm: Dummy-Head Level Linking ▾

The optimal solution uses a classic linked-list construction trick — a dummy sentinel node — to build each level's chain without knowing which child comes first. Here is the full algorithm:

1 curr = root. curr is the leftmost node we start from on each level. For level 0 that's the root. We traverse the current level via curr = curr.next.
2 Create dummy + tail for next level. dummy is a throwaway sentinel Node(0). tail starts at dummy and marches forward as we thread children into the chain. dummy.next will end up pointing at the first real child.
3 Visit every node on current level (inner loop). For each curr: if curr.left exists, do tail.next = curr.left; tail = tail.next. Then if curr.right exists, do the same. Advance: curr = curr.next.
4 Drop to next level. After the inner loop, dummy.next is the first node of the next level (or null if no children). Set curr = dummy.next. Repeat outer loop.
✗ BFS Queue (O(n) space)

Push nodes into a deque level by level. Track level size. For each level, node[i].next = node[i+1]. Easy but the queue can hold up to n/2 nodes (widest level).

✓ Dummy Head (O(1) space)

Use already-set next pointers as a free linked list for traversal. Build the next level's chain using dummy + tail. Only 3 pointers total.

🔑 Why Dummy Node Instead of a Null-Check?

Without dummy, you'd need: "if this is the first child I've found this level, it's the head; else append." That's a messy if/else. With dummy, you always do tail.next = child — the first child goes at dummy.next, later children extend the chain. Uniform code, zero special cases.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the start of each outer-loop iteration, all nodes on the current level form a complete left-to-right linked list via next pointers. Walking curr → curr.next → … visits every node on that level in left-to-right order. The inner loop leverages this free traversal to build the next level's linked list.

Inductive proof sketch:

  • Base (Level 0): Only the root. Its next is null. Traversal = just root. Invariant trivially holds.
  • Inductive step: Assume level k is a proper left-to-right linked list. We traverse it left-to-right, appending each child (left before right) to the dummy chain. Children of earlier nodes appear before children of later nodes — left-to-right ordering is preserved. After the traversal, dummy.next → … → tail is the complete chain for level k+1. We set curr = dummy.next. Invariant holds for k+1. ✓
⚠️ Left Before Right is Mandatory

In the inner loop we always check curr.left before curr.right. This ensures children are appended in the correct order. If you swapped it — right before left — the next-level chain would be built in a mixed order, giving incorrect next connections.

Handling gaps (the "General Case"): Suppose node 3 has no left child but does have right child 7. When curr = 3, the left-check skips. The right-check appends 7 directly after whatever was last in the chain (node 5 from node 2). So node 5's next will point to node 7 — skipping the missing position entirely. This is exactly correct: 5 and 7 are adjacent on level 2.

4 The Code (Annotated) ▾

Solution 1 — Dummy Head, O(1) Space (Interview Pick)

def connect(root): ← accepts root of any binary tree
curr = root ← start at leftmost node of level 0
while curr: ← outer: one pass per level
dummy = Node(0) ← sentinel anchor for next-level chain
tail = dummy ← tail extends the chain as children are found
while curr: ← inner: traverse current level via next-ptrs
if curr.left: ← left child first (left-to-right order)
tail.next = curr.left ← thread left child into chain
tail = tail.next ← advance tail
if curr.right: ← then right child
tail.next = curr.right ← thread right child into chain
tail = tail.next ← advance tail
curr = curr.next ← move to next node on current level
curr = dummy.next ← dive to next level (null if no children)
return root ← return modified tree
⚠️ Common Mistake: Forgetting to Reset dummy per Level

Notice that dummy = Node(0) is inside the outer while loop, so a fresh dummy is created for every level. If you moved it outside, the dummy's previous connections would bleed into the next level's chain, producing wrong results.

Solution 2 — BFS Queue (O(n) space, easier to explain first)

from collections import deque
def connect(root):
if not root: return root ← guard: empty tree
q = deque([root]) ← BFS queue, seeded with root
while q:
size = len(q) ← snapshot: exactly one level
for i in range(size):
node = q.popleft()
if i < size - 1: ← not the rightmost this level
node.next = q[0] ← next in queue = right neighbour
if node.left: q.append(node.left) ← enqueue children
if node.right: q.append(node.right)
return root
💡 Complexity Summary

Dummy Head: Time O(n) — each node visited once. Space O(1) — only curr, dummy, tail pointers.
BFS Queue: Time O(n). Space O(n) — queue holds at most n/2 nodes (the widest level).
Both visit each node exactly once; the dummy approach avoids the queue allocation entirely.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Level-Order Pattern

"The next pointer connects nodes on the same level — that's BFS level-order. Any level-order problem uses a queue or an equivalent traversal structure."

Step 2: Propose BFS with Queue First (O(n) space)

"Simple approach: use a deque. Track the size of each level. For each node in the level except the last, set node.next = q[0]. Clean, correct, O(n) space. Let me see if I can do O(1)."

Step 3: Recognise the Free Queue (O(1) space)

"After linking level k, I already have a linked list there via next. I can traverse level k for free using curr = curr.next. So I don't need a queue at all — the already-linked level acts as my queue for free."

Step 4: Introduce Dummy to Avoid First-Child Edge Case

"To build the next level's chain uniformly, I use a dummy sentinel. tail.next = child every time — no special case for the first child. dummy.next gives me the head after the loop."

🗣️ What to Say to the Interviewer

"This is level-order traversal — each level's nodes must be linked left to right.
Start simple: BFS queue, track level size, connect node[i].next = node[i+1]. O(n) space.
Can we do O(1)? Yes — once level k is linked, it's already a linked list we can walk.
Use curr.next to traverse level k, and a dummy sentinel to build level k+1's chain.
Always check left child before right to preserve order. Dummy.next gives chain start.
Time O(n) — one visit per node. Space O(1) — only 3 pointers: curr, dummy, tail."

7 Summary & Key Takeaways ▾
Level-order BFS naturally processes nodes left-to-right per level — the ideal fit for next-pointer problems
The dummy sentinel eliminates the "first child" special case — always append via tail, read the head from dummy.next
Once level k is linked, traverse it for free using next-pointers — no queue needed for O(1) space
Always process left child before right to preserve left-to-right ordering across the next level
Gaps (null children) are handled automatically — the chain simply skips over missing positions
