LeetCode #796

Rotate String

Return true if s can become goal after some number of left-shift rotations.

Easy String Processing Concatenate + Substring Check Time: O(n) Space: O(n)
1 Understanding the Problem ▾

A left shift moves the leftmost character to the rightmost position. Applying shifts repeatedly cycles through all rotations of s and then returns to s. We need to check if, after any number of shifts (including 0), s equals goal.

💡 Key Insight

There are exactly n distinct rotations of a length-n string (one per starting position). All of them appear as contiguous windows of width n inside s + s. So check: does goal appear anywhere in s + s?

Edge cases: different lengths → always False; s == goal → True (0 shifts); single character → True if same char.

2 The Algorithm: Concatenation Trick ▾

Instead of generating all n rotations and comparing each — O(n²) — we use one elegant O(n) trick:

1 Length guard: If len(s) != len(goal), return False immediately. A rotation preserves every character, so lengths must be equal.
2 Concatenate: Form s + s. The string "abcde" + "abcde" = "abcdeabcde" is a 10-character string that contains all 5 rotations as overlapping windows.
3 Substring check: Return goal in (s + s). If goal is found anywhere in s+s as a length-n substring, it's a valid rotation. Python's in uses KMP, so this is O(n).
✗ Brute Force — O(n²)

for i in range(n):
  rotation = s[i:] + s[:i]
  if rotation == goal:
    return True
return False
n iterations × O(n) = O(n²)

✓ Concat Trick — O(n)

if len(s) != len(goal):
  return False
return goal in s + s

O(n) concat + O(n) search

3 Why It Works: The Invariant ▾
🔐 The Core Property

The i-th rotation of s (i left shifts) is s[i:] + s[:i].
In the string s+s, this rotation appears starting at index i:
(s+s)[i : i+n] = s[i:n] + s[0:i] = s[i:] + s[:i]
For i = 0, 1, …, n−1, we cover all n possible rotations exactly once.

Proof for our example: s = "abcde" (n=5)
s+s = "abcdeabcde"
(s+s)[2:7] = "cdeab" = s[2:] + s[:2] = "cde" + "ab" ✓ — this is exactly 2 left shifts.

Why the length check matters: Without it, short strings might appear in s+s coincidentally. For example, if s = "ab" and goal = "a", goal appears in s+s = "abab" but is not a rotation (wrong length). The length guard ensures we only accept length-n matches.

⚠️ What About i = n (n shifts)?

After n left shifts, s returns to itself (full cycle). We only need to check windows at positions 0 through n−1. The window at position n would be (s+s)[n:2n] = s again (i=0 rotation). No need to go beyond — n positions cover all distinct rotations.

4 The Code (Annotated) ▾

Solution 1 — Concatenation Trick (Optimal, Interview Pick)

def rotateString(s: str, goal: str) -> bool:
if len(s) != len(goal): ← Different lengths → can't be a rotation
return False
return goal in s + s ← All rotations live in s+s; CPython uses KMP internally → O(n)
⚠️ Is goal in s + s actually O(n)?

Yes — CPython implements the in operator for strings using a combination of Boyer-Moore-Horspool and Sunday's algorithm, achieving O(n) average case. For interview purposes, calling it O(n) is accepted. If asked about worst case, mention that naive substring search is O(n²) but proper KMP guarantees O(n) always.

Solution 2 — Brute Force (O(n²))

def rotateString_brute(s: str, goal: str) -> bool:
if len(s) != len(goal): return False ← Length guard
n = len(s)
for i in range(n): ← Try each of the n starting positions
if s[i:] + s[:i] == goal: ← Build rotation i and compare — O(n) each
return True
return False ← No rotation matched
💡 Complexity Summary

Concat Trick: Time O(n + n) = O(n), Space O(n) for s+s string
Brute Force: Time O(n²) (n rotations × O(n) comparison), Space O(n) per rotation
Special case: s == goal → True (0 shifts). len(s) == 0 and goal == "" → True.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Structure

"Rotation/shift/cycle" problems involve moving characters from one end to the other. There are exactly n distinct rotations (then it cycles back). Ask: can I enumerate all n without O(n²) work?

Step 2: Start with Brute Force

"I'll try all n rotations: for each i, check if s[i:]+s[:i] == goal. This is O(n²). The interviewer probably wants better."

Step 3: Look for a Structure that Encodes All Rotations

"What if I had a single string that contained all n rotations? Then I just need one substring search. Concatenating s with itself does exactly that: s+s contains every rotation as a window."

Step 4: Justify and Check the Boundaries

"Length guard is needed. Then: goal in s+s is O(n) via KMP. Total: O(n) time, O(n) space. Done."

🗣️ What to Say to the Interviewer

"A rotation of s is s[i:]+s[:i] for some i. There are n such rotations.
Brute force checks each one individually — O(n²).
Key insight: all n rotations appear as substrings of length n in s+s.
So I just need to check goal in s+s, with a length guard first.
This is O(n) time, O(n) space for the concatenated string."
🔮 Likely Follow-up Questions
  • "What if shifts are right shifts instead?" → Same trick: right shift = left shift with goal reversed order. Or just test goal in s+s — right shifts of s also appear in s+s.
  • "Find all rotation positions?" → Find all starting indices of goal in s+s using KMP (returns list of positions).
  • "Minimum shifts to reach goal?" → Find the index where goal appears in s+s — that's the number of left shifts needed.
  • "Are two strings rotations using O(1) space?" → Need KMP or rolling hash; pure O(1) space is tricky.

7 Summary & Key Takeaways ▾
A length-n string has exactly n distinct rotations (indices 0 through n−1)
All n rotations appear contiguously in s+s — one check replaces n comparisons
Length equality guard is mandatory — prevents false positives from shorter strings
Python's in uses KMP internally → O(n) — this is the full optimal solution
The minimum shift count = index where goal first appears in s+s

🎯 Generalization: When to Use Concatenation
  • "Is A a rotation of B?" → B in A+A
  • "Find all rotation positions of B in A" → KMP on A+A, find all indices
  • "Minimum shifts to make A equal B" → First index of B in A+A
  • Any "circular arrangement" membership problem → concatenate and search
