LeetCode #426

Convert Binary Search Tree to Sorted Doubly Linked List

Convert a BST in-place to a sorted circular doubly linked list by reusing the left/right node pointers.

Medium Trees In-order Traversal Time: O(n) Space: O(h)
1 Understanding the Problem ▾

Given the root of a Binary Search Tree (BST), convert it to a circular doubly linked list (DLL) in-place. You must reuse the existing tree nodes — just rewire their left and right pointers. The result must be sorted in ascending order, and the list must be circular (tail points back to head, head's prev points to tail).

💡 Key Insight

BST in-order traversal (Left → Root → Right) visits nodes in ascending sorted order by the BST property. If we connect each visited node to its predecessor during traversal, we build the doubly linked list in the exact order we need — no sorting, no extra space for values.

Input BST

    4
  ↙ ↘
 2   5
↙ ↘
1  3

Output: Circular DLL

↔ 1 ↔ 2 ↔ 3 ↔ 4 ↔ 5 ↔
↑____________________↓
head=1 (circular, sorted)

Pointer reuse: Each BST node has left (its left child) and right (its right child). After conversion, left becomes the prev pointer and right becomes the next pointer in the DLL. No new nodes are allocated.

Edge cases: Empty tree (return None), single node (points to itself — self-circular).

2 The Algorithm: In-Order DFS with prev Tracking ▾

We run an in-order DFS with two persistent pointers: head (the first node ever visited — the minimum, fixed once) and prev (the most recently visited node, updated at each step). At each visit we wire the bidirectional link.

1 Recurse left: Go to node.left first. This ensures we visit the smallest element before the current node — the fundamental rule of in-order traversal.
2 Visit current node: If prev is None, this is the global minimum — set head = node (done only once). Otherwise: prev.right = node (forward link) and node.left = prev (backward link). Then set prev = node.
3 Recurse right: Go to node.right. This visits all larger nodes after the current node, maintaining ascending order.
4 Close the circle: After full traversal, prev = maximum (tail), head = minimum. Wire: prev.right = head and head.left = prev. The list is now circular.

In-order visit sequence for BST [4,2,5,1,3]:

1→ 2→ 3→ 4→ 5 ascending!
🔑 Why In-Order = Sorted Order in a BST?

BST invariant: every node in the left subtree is strictly smaller than the root, every node in the right subtree is strictly larger. In-order traversal visits the left subtree first (all smaller), then the root, then the right subtree (all larger). Applied recursively, this guarantees ascending order across the entire tree.

3 Why It Works: The Invariant ▾
🔐 The Invariant

When we are about to process node n during in-order traversal: prev always holds the immediately preceding node in sorted order. The segment from head to prev is already a correctly-wired doubly linked list containing all elements smaller than n.

Why linking is always correct: In-order visits nodes in non-decreasing order. When we process node n, prev is the last-visited node, which is exactly the predecessor of n in sorted order. Wiring prev.right = n (prev's next = n) and n.left = prev (n's prev = prev) correctly extends the DLL by one node. Setting prev = n restores the invariant for the next iteration.

Why the circle-close step is safe: After all n nodes are processed, head = minimum (set exactly once, when prev was None) and prev = maximum (last node visited). The two closing assignments don't disturb any of the n-1 internal links already established — they only fill in the two "boundary" pointers of the otherwise-open list.

⚠️ Common Trap: Forgetting the Circle-Close Step

The most frequent bug: returning head immediately after the recursion without closing the circle. The problem explicitly asks for a circular DLL. Without head.left = prev; prev.right = head, the head has no prev and the tail has no next — it's an open list, not circular.

Single-node edge case: After traversal, head = prev = node. The closing step: node.right = node and node.left = node — the node points to itself in both directions, which is the correct circular single-element DLL.

4 The Code (Annotated) ▾

Solution 1 — Recursive In-Order DFS (Interview Pick)

def treeToDoublyList(root): ← Entry point
if not root: ← Empty tree → return None
return None
head, prev = None, None ← head = first node (min); prev = last linked node
def inorder(node): ← Nested function captures head/prev via nonlocal
nonlocal head, prev
if not node: return ← Base case: null child → nothing to do
inorder(node.left) ← ① RECURSE LEFT: visit all smaller nodes first
if prev is None: ← ② VISIT: first node ever = minimum = head
head = node ← Set head exactly once (the minimum)
else: ← Otherwise, wire bidirectional link
prev.right = node ← prev → node (forward/next link)
node.left = prev ← node ← prev (backward/prev link)
prev = node ← Advance prev to current node
inorder(node.right) ← ③ RECURSE RIGHT: visit all larger nodes
inorder(root) ← Kick off traversal
head.left = prev ← Close circle: head's prev → tail
prev.right = head ← Close circle: tail's next → head
return head ← Return the minimum node
⚠️ Why nonlocal is Required

Inside the nested inorder(), assigning to head or prev (e.g., head = node) without nonlocal would create a new local variable inside inorder, leaving the outer head unchanged. The nonlocal declaration tells Python to look up and modify the variable in the enclosing function's scope.

Solution 2 — Iterative In-Order with Explicit Stack

def treeToDoublyList(root): ← Avoids recursion depth limits for huge trees
if not root: return None
head, prev, stack, node = None, None, [], root
while node or stack: ← Continue while nodes remain to process
while node: ← Push entire left spine onto stack
stack.append(node)
node = node.left
node = stack.pop() ← Pop = next in-order node
if prev is None: head = node ← First pop = minimum = head
else: prev.right = node; node.left = prev ← Wire bidirectional link
prev = node ← Advance prev
node = node.right ← Move to right subtree (may be None)
head.left = prev; prev.right = head ← Close circle
return head
💡 Complexity Summary

Both solutions — Time O(n): every node is visited exactly once.
Both solutions — Space O(h): where h = tree height. Balanced BST: O(log n). Degenerate (sorted-array-shaped) BST: O(n).
The iterative version eliminates Python's recursion stack but uses an explicit stack of the same depth.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Notice the Output Must Be Sorted

The DLL must be in ascending sorted order. BST in-order traversal produces exactly this. Say: "Since we have a BST and need sorted output, in-order traversal is the natural fit — it visits nodes from smallest to largest by the BST property."

Step 2: Observe That Pointers Can Be Reused In-Place

BST nodes already have two pointers (left, right). DLL nodes need two pointers (prev, next). After conversion: left = prev, right = next. No new nodes allocated — true in-place transformation.

Step 3: Track head and prev During Traversal

At each in-order visit: if prev is None, this is the head (minimum). Otherwise wire prev ↔ current bidirectionally, then advance prev. After traversal, close the circle with two pointer assignments.

Step 4: Mention Edge Cases and Complexity

Empty tree (return None), single node (self-circular). Time O(n) since every node visited once. Space O(h) for the call stack — mention the iterative version if stack depth is a concern.

🗣️ What to Say to the Interviewer

"BST in-order traversal gives ascending order — that's the key insight.
I'll reuse left as prev and right as next since we need an in-place conversion.
I'll track two outer pointers: head (set once for the minimum) and prev (last linked node).
At each in-order visit: if prev is None, set head. Else: prev.right = node, node.left = prev. Advance prev.
After traversal: head.left = prev and prev.right = head to close the circle.
Time O(n), Space O(h) for the call stack."
🔮 Likely Follow-up Questions
  • "Can you do it iteratively?" → Yes — explicit stack simulates the call stack (Solution 2)
  • "Can you do it in O(1) space?" → Morris in-order traversal (threads tree temporarily)
  • "What if it weren't circular?" → Skip the two closing lines; head.left = None, tail.right = None
  • "What if it's not a BST?" → In-order still works but won't be sorted; need collect + sort + relink

7 Summary & Key Takeaways ▾
BST in-order traversal visits nodes in ascending sorted order — the core property this problem exploits
Reuse the existing left/right pointers as prev/next — no new nodes allocated (true in-place)
Two outer pointers: head (set once for the minimum) and prev (last linked node, advanced each step)
At each visit: if prev is None → set head. Else: wire prev ↔ node bidirectionally. Always advance prev.
After traversal: close the circle with head.left = prev and prev.right = head (two lines, easy to forget!)
Time O(n): every node visited once. Space O(h): call stack depth equals tree height.
