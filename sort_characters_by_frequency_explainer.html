LeetCode #451

Sort Characters By Frequency

Given a string s, sort it in decreasing order based on the frequency of the characters and return the sorted string.

Medium Sorting Frequency Buckets Time: O(n) Space: O(n)
1 Understanding the Problem ▾

We're given a string and need to return a rearrangement of its characters where the most frequent characters come first. All copies of the same character must be grouped together, and characters with the same frequency can appear in any relative order.

💡 Key Insight

This is fundamentally a count → rank → reconstruct problem. Count how many times each character appears, rank characters by that count (highest first), then write each character exactly its count times. The clever twist: since frequencies are bounded by n (string length), we can use bucket sort to achieve O(n) instead of O(n log n).

Example: s = "tree"

freq: t→1, r→1, e→2
Rank: e(2) > t(1) = r(1)
Output: "eert" or "eetr"
(both valid — ties are arbitrary)

Example: s = "Aabb"

freq: A→1, a→1, b→2
Rank: b(2) > A(1) = a(1)
Output: "bbAa" or "bbaa"
('A' ≠ 'a' — case-sensitive!)

Edge Cases: single character string ("z" → "z"), all identical characters ("aaaa" → "aaaa"), all unique characters (any order valid), empty string ("" → ""). The problem guarantees s is non-empty, but handle gracefully.

⚠️ Case Sensitivity Trap

'A' and 'a' are distinct characters. Never lowercase the string before processing — it changes the result. "Aab" has A→1, a→1, b→1, not a→2, b→1.

2 The Algorithm: Frequency Bucket Sort ▾

We use a classic bucket sort strategy: since character frequencies range from 1 to n (length of string), we can create an array of n+1 "buckets" where bucket i holds all characters that appear exactly i times. Then reading the buckets from highest to lowest gives us our sorted output in O(n).

1 Count frequencies: Scan the string once. Build a hash map freq[c] = count. O(n) time.
2 Fill buckets: Create array buckets of size n+1. For each character c with count k, do buckets[k].append(c). O(n) time, O(n) space.
3 Build output (reverse scan): Loop i from n down to 1. For each character c in buckets[i], append c * i (the character repeated i times). O(n) total output characters.
4 Join and return: ''.join(result) produces the final string.

🔑 Why Bucket Sort Beats Comparison Sort Here

Standard sort (e.g., Python's sorted()) is O(n log n) — it must compare elements. But we're not sorting arbitrary values: frequencies are bounded in [0, n]. Bucket sort exploits this bounded range to skip comparisons entirely, achieving O(n). The trade-off is O(n) extra space for the buckets array — a fair deal.

3 Why It Works: The Invariant ▾
🔐 The Invariant

During the reverse bucket scan, every character written to the output so far has a frequency ≥ the frequency of every character not yet written. This directly satisfies the "decreasing order by frequency" requirement.

Why correctness is guaranteed: Each character c is placed in exactly one bucket — buckets[freq[c]]. When we scan from n down to 1, we always process higher-frequency buckets before lower-frequency ones. So every character written at step i came from a bucket with index ≥ every future bucket. The ordering invariant is maintained at every step.

Why the output length always equals n: Each character c contributes exactly freq[c] copies to the output (we write c * freq[c]). The sum of all frequencies equals n (every input character counted once). So total output characters = n. No characters are added or dropped.

Why ties are always valid: The problem accepts any valid answer. Two characters with the same frequency can appear in either order — both satisfy "decreasing frequency." Our bucket at index i stores all same-frequency characters, and we output them in whatever iteration order they appear, which is fine.

⚠️ Classic Bug: Writing Only Once Per Character

A frequent mistake: writing each character once instead of freq[c] times. If 'e' appears 3 times in the input, the output must contain "eee" (three e's), not just "e". Always use char * count when building the output — the multiplication is not optional.

4 The Code (Annotated) ▾

Solution 1 — Bucket Sort (Optimal, O(n) time)

def frequencySort(s: str) -> str:
freq = {} ← Will map each char to its occurrence count
for c in s: ← Single pass to count all characters
freq[c] = freq.get(c, 0) + 1 ← get(c,0) handles first-time chars safely
buckets = [[] for _ in range(len(s) + 1)] ← Index i = "chars that appear exactly i times"
for char, count in freq.items(): ← Place each char into its frequency bucket
buckets[count].append(char) ← e.g., freq('e')=2 → buckets[2].append('e')
result = []
for i in range(len(s), 0, -1): ← Scan from n down to 1 (skip 0: no char has freq 0)
for char in buckets[i]: ← For each char with this frequency
result.append(char * i) ← Write it exactly i times (the key line!)
return ''.join(result) ← Concatenate all repeated-char strings
⚠️ Why range(len(s), 0, -1) not range(len(s)-1, -1, -1)?

We start at len(s) because a single character could appear that many times (e.g., "aaaa" → freq=4=len). We stop before 0 because no character has frequency 0 — stopping at 1 (exclusive lower bound) is correct. The step -1 makes it descending.

Solution 2 — Counter + most_common() (Simpler, O(n log n))

from collections import Counter
def frequencySort(s: str) -> str:
count = Counter(s) ← Build frequency map in one call
return ''.join( ← Concatenate all the repeated-char strings
char * freq ← 'e' * 2 = "ee" — repeat char freq times
for char, freq in count.most_common() ← most_common() returns pairs sorted by freq desc
)
💡 Complexity Comparison

Counter + most_common(): Time O(n log n), Space O(n) — relies on Python's internal sort
Bucket Sort: Time O(n), Space O(n) — theoretically optimal
In interviews: Show both. Start with Solution 2 (simpler, fewer lines), then offer Solution 1 as an O(n) optimization when asked to go faster.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

Key signals: "sort/order by frequency," "characters grouped together," output ordering based on count. These point to Count → Sort → Reconstruct. The problem gives freedom in tie-breaking, which is a hint that the ordering mechanism doesn't need to be perfectly stable.

Step 2: Propose the Simple O(n log n) First

"I'll start with Counter(s).most_common() — count all characters, get them sorted by frequency descending, then join each char * freq. This is clean, correct, and O(n log n). Can I optimize further?"

Step 3: Identify the Optimization — Bounded Range

"The bottleneck is sorting. Notice: character frequencies are bounded by n (can't exceed string length). That's a bounded integer range — perfect for bucket sort! I can skip the O(n log n) sort entirely by placing each character directly into bucket[freq], then reading buckets n → 1."

Step 4: Discuss Trade-offs

"Both approaches use O(n) space. The bucket sort is O(n) time vs O(n log n) — the improvement is most dramatic when n is large and the character set is small (like only lowercase letters, where we have at most 26 distinct chars spread across n positions)."

🗣️ What to Say to the Interviewer

"The core task: count frequencies, then sort by those counts.
First approach: Counter(s) + most_common() + join. Clean, O(n log n).
Can we beat O(n log n)? Yes — frequencies are bounded by n, so use bucket sort.
Create buckets[0..n], put char in bucket[freq[char]], scan n→1, append char*i.
O(n) time and space. The key insight: bounded range → no comparison sort needed."
