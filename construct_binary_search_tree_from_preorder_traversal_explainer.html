LeetCode #1008

Construct Binary Search Tree from Preorder Traversal

Given an array of integers preorder, which represents the preorder traversal of a BST, construct the BST and return its root node.

Medium Trees / BST Upper Bound Tracking Time: O(n) Space: O(n)
1 Understanding the Problem ▾

A preorder traversal visits nodes in Root → Left → Right order. Given preorder = [8, 5, 1, 7, 10, 12], we must reconstruct the unique BST from which this traversal came.

💡 Key Insight

The first element is always the root. After the root, every element smaller than it belongs to the left subtree, and every element larger belongs to the right subtree. This BST property — combined with the preorder ordering — uniquely determines the tree without needing the inorder traversal.

✓ BST Property

Root = 8
Left subtree: < 8 → [5, 1, 7]
Right subtree: > 8 → [10, 12]
Recurse on each subarray

✗ Without BST Property

General tree preorder alone
is NOT uniquely decodable
— you'd need inorder too.
BST property IS the inorder!

Edge Cases: empty array (return null), single element (a leaf node), all values ascending (right-only chain like 1→2→3→4), all values descending (left-only chain).

🔑 Why Preorder Alone Is Sufficient for a BST

For a general binary tree, you need both preorder and inorder to reconstruct uniquely. But a BST's inorder is always sorted ascending — it's implicit. So preorder alone encodes the full BST structure. This is the crucial observation that unlocks the solution.

2 The Algorithm: Upper Bound Tracking (O(n) Optimal) ▾

We process preorder elements left to right using a shared index. At each recursive call, we pass an upper bound: any element in this subtree must be strictly less than this bound. When the next element exceeds the bound, it belongs to an ancestor — return None and let the index stay where it is for the ancestor to pick up.

1 Base case: If index is exhausted, OR preorder[i] >= upper, return None. The element doesn't belong in this subtree; the index stays put for the caller.
2 Create node: Take preorder[i] as the root of this subtree. Create a TreeNode and advance the shared index by 1.
3 Recurse left: Call build(node.val). The left subtree's upper bound tightens to the current node's value — all left descendants must be smaller.
4 Recurse right: Call build(upper). The right subtree inherits the same upper bound from the parent — the right child is larger than us, but still bounded by whatever constraint our ancestor imposed.
5 Return node: The shared index has advanced exactly once per element placed. Total work: O(n).
🔑 Why Upper Bound Beats Naive Split Search

The naive approach scans for the first element > root to find where the left subtree ends — O(n) per level, O(n²) worst case for a skewed tree. Upper bound tracking avoids this: the bound itself tells us when we've crossed into an ancestor's territory. The index advances naturally, consuming exactly the right elements at each level.

    build(5): val=1 < 5 → node=1, i=3

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any recursive call, all values that belong to this subtree appear consecutively in the remaining preorder array, and they are all strictly less than upper. The shared index points exactly to the next unprocessed element — no element is ever skipped or revisited.

Why left child gets upper = node.val: In BST, every node in the left subtree is smaller than its parent. By passing the current node's value as the upper bound, any left-subtree call stops the moment it sees a value that's too large — which means that value belongs in a right subtree, not here.

Why right child inherits the same upper: The right child is larger than the current node, but it must still respect the constraint of any ancestor. If the current node had an upper bound of 8, then its right subtree cannot contain 9 either (because 9 would violate the ancestor's BST constraint). The same bound propagates downward through all right children.

⚠️ The Mutable Index Trap

The index must be shared across all recursive calls. In Python, integers are immutable — you cannot update a local copy and expect the caller to see it. Use a list: self.i = [0] and write self.i[0] += 1. Alternatively, use Python's nonlocal i in a nested function. Both work; the list wrapper is the most portable pattern.

O(n) proof: The shared index starts at 0, only ever increments, and each increment corresponds to exactly one TreeNode created. With n nodes in the tree, there are exactly n increments. All calls that hit a base case (return None) do O(1) work with no increment. Total: O(n) time, O(h) space for recursion stack where h ≤ n.

4 The Code (Annotated) ▾

Solution 1 — Upper Bound Tracking, O(n) [Interview Pick]

class Solution:
def bstFromPreorder(self, preorder):
self.i = [0] ← mutable list wraps index so all calls share it
def build(upper): ← upper = max allowed value in this subtree
if self.i[0] == len(preorder): ← exhausted all elements — no node to create
return None
val = preorder[self.i[0]] ← peek: does next element belong in THIS subtree?
if val >= upper: ← NO — exceeds bound; belongs to an ancestor's subtree
return None ← do NOT advance index; caller will handle val
self.i[0] += 1 ← commit: this element is ours; advance shared index
node = TreeNode(val) ← create the BST node
node.left = build(val) ← left child: tighter upper bound = current node value
node.right = build(upper) ← right child: inherits same upper from parent
return node
return build(float('inf')) ← root has no constraint — any value allowed
⚠️ Why float('inf') for the root call?

The root can be any value — no ancestor constrains it. Passing float('inf') ensures the check val >= upper is always False for the first element, so it's placed unconditionally. This avoids a special case and keeps the code uniform across all recursion levels.

Solution 2 — Naive Split Search O(n²)

def bstFromPreorder(self, preorder): ← simpler to understand, but O(n²) worst case
if not preorder: return None
root = TreeNode(preorder[0]) ← first element is always the root
i = 1 ← scan to find split: where right subtree begins
while i < len(preorder) and preorder[i] < root.val:
i += 1 ← O(n) per level in worst case (skewed tree)
root.left = self.bstFromPreorder(preorder[1:i]) ← left: elements smaller than root
root.right = self.bstFromPreorder(preorder[i:]) ← right: elements larger than root
return root
💡 Complexity Summary

Upper Bound Tracking: Time O(n) — index advances once per node. Space O(h) recursion stack + O(n) output tree.
Naive Split Search: Time O(n²) worst (skewed tree), O(n log n) average (balanced). Space O(n²) due to array slicing creating subarrays.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Signal

"The problem says preorder traversal of a BST. Two key facts: (1) preorder → root always first, (2) BST → left/right split by value. Together these uniquely determine the tree without needing inorder."

Step 2: Propose the Naive Approach First

"Simple idea: root = preorder[0]. Scan to find where elements become > root — that's the left/right split. Recurse on both halves. This works but it's O(n²) for a skewed tree because each level scans O(n) elements."

Step 3: Optimize with the Upper Bound Insight

"Instead of scanning for the split, I can pass an upper bound into each recursive call. A value belongs to this subtree only if it's less than the bound. The shared index advances naturally — O(n) total, no scanning."

Step 4: Explain the Shared Index Trick

"The index must be mutable across calls. I'll use self.i = [0] as a list so all recursive frames see the same index. Left subtree gets bound = node.val; right subtree inherits the same bound from its parent."

🗣️ What to Say to the Interviewer

"First observation: preorder[0] is always the root of any subtree.
Second: BST property means left < root < right — this partitions the array.
Naive approach: find the split each time. O(n²). Can we do O(n)?
Yes: pass an upper bound. A node belongs here iff val < upper.
Left child: upper tightens to node.val. Right child: inherits parent's upper.
Shared index advances once per node created — exactly n times total. O(n)."
