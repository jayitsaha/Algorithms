LeetCode #101

Symmetric Tree

Given the root of a binary tree, check whether it is a mirror of itself (symmetric around its center).

Easy Trees Compare Left/Right Mirrors Time: O(n) Space: O(h)
1 Understanding the Problem ▾

A symmetric tree looks the same on both sides when folded along its vertical center axis — its left subtree is a mirror image of its right subtree. "Mirror" is not the same as "identical": a tree that is mirror-symmetric has its structure and values reflected, not copied identically.

💡 Key Insight

For a tree to be symmetric, two nodes are "mirrors" if their values are equal, AND the left child of one is a mirror of the right child of the other. This crossing — left-left vs right-right AND left-right vs right-left — is what makes this a mirror, not an identical copy.

✓ Symmetric: mirrors left ↔ right

       / \
      2   2
     / \ / \
    3  4 4  3

Outer pair (3,3) match. Inner pair (4,4) match. → True

✗ Not Symmetric: structure differs
        1
       / \
      2   2
       \   \
        3   3

Left has right child only, right has right child only — not mirrored. → False

Mirror rule visualized: When comparing nodes L (left subtree) and R (right subtree) as potential mirrors, check: (1) L.val == R.val, (2) L.left mirrors R.right (outer pair), (3) L.right mirrors R.left (inner pair). This crossing pattern is essential.

Edge cases: An empty tree (None) is symmetric. A single-node tree is trivially symmetric. When one side has a child and the mirror position is null, it's asymmetric.

2 The Algorithm: Recursive Mirror Comparison ▾

We define a helper isMirror(left, right) that checks if two subtrees are mirror images. The key is that we always compare across the axis: left's outer child against right's outer child, and left's inner child against right's inner child.

1 Base case — both null: isMirror(None, None) → True. Two empty positions mirror each other perfectly.
2 Base case — one null: isMirror(None, node) or isMirror(node, None) → False. Asymmetric structure detected; one side has a child, the other doesn't.
3 Value mismatch: If left.val != right.val → False. Values at mirror positions must be equal.
4 Recurse — outer pair: Check isMirror(left.left, right.right). These are the outermost children, farthest from the axis.
5 Recurse — inner pair: Check isMirror(left.right, right.left). These are the innermost children, adjacent to the axis of symmetry.
6 Entry: Call isMirror(root.left, root.right). The root is the axis — only its two children need to mirror each other.
🔑 The Crossed Comparison — This Is the Core

When node L and node R are supposed to be mirrors of each other, their children must be compared across the axis:
L.left ↔ R.right (outer edges — both point away from center)
L.right ↔ R.left (inner edges — both point toward center)
Writing L.left ↔ R.left checks identity, not symmetry. Always cross!

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every call, isMirror(L, R) returns True if and only if the subtree rooted at L is a structural and value mirror of the subtree rooted at R.

This invariant holds by structural induction:

  • Base cases are trivially correct: (None, None) → True; (None, node) or (node, None) → False. These match the definition of an empty vs non-empty mirror position.
  • Inductive step: Assume isMirror is correct for all smaller subtrees. If L.val == R.val, then the full subtrees rooted at L and R are mirrors iff their children are mirrors in the crossed arrangement. Our recursion checks exactly this.
  • Termination: Each recursive call operates on strictly smaller subtrees (children of current nodes), so we always reach base cases.
⚠️ The #1 Mistake: Same-Direction Comparison

Writing isMirror(L.left, R.left) checks whether the two subtrees are identical, not mirrored. Only a perfectly balanced tree like [1,1,1] would pass this incorrectly. Always compare across the axis: outer-with-outer, inner-with-inner.

Iterative equivalence: The BFS queue approach maintains the same invariant: every pair in the queue is a (should-be-mirror) pair. We pop pairs, verify values match, and enqueue their children in the crossed order. Any violation immediately returns False.

4 The Code (Annotated) ▾

Solution 1 — Recursive (Interview Pick)

def isSymmetric(root): ← entry point; root is the axis of symmetry
def isMirror(left, right): ← helper: are these two subtrees mirrors?
if not left and not right: ← both None → mirrors (matching empty positions)
return True
if not left or not right: ← one None → asymmetric structure, impossible mirror
return False
if left.val != right.val: ← both exist but different values → not mirrors
return False
outer = isMirror(left.left, right.right)← outer pair: L's left vs R's right (both outer)
inner = isMirror(left.right, right.left)← inner pair: L's right vs R's left (both inner)
return outer and inner ← both pairs must mirror for this level to mirror
return isMirror(root.left, root.right) ← start: compare the two halves
⚠️ Why not just check root.val?

The root is the axis of symmetry, not a mirror pair. It has no counterpart. We compare its two children (left vs right) as mirror candidates. If root is None, Python's root.left would crash — always handle the null root edge case if needed (LeetCode guarantees a non-null root here).

Solution 2 — Iterative BFS with Queue

from collections import deque ← use a queue for iterative BFS
def isSymmetric(root):
q = deque([(root.left, root.right)]) ← seed with the initial mirror pair
while q:
L, R = q.popleft() ← dequeue one should-be-mirror pair
if not L and not R: continue ← both None → OK, skip
if not L or not R: return False ← asymmetric structure
if L.val != R.val: return False ← values differ
q.append((L.left, R.right)) ← enqueue outer pair
q.append((L.right, R.left)) ← enqueue inner pair
return True ← all pairs passed

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Tree Recursion Pattern

Any tree problem that asks about structural properties across the whole tree — "is it symmetric?", "is it balanced?", "are these two trees the same?" — is almost always recursive. Trees are self-similar structures; recursion is natural.

Step 2: Reframe as a Two-Pointer Tree Problem

Instead of thinking about one tree, think of two pointers — L and R — that start at the two halves and walk inward together. The question becomes: "are these two nodes mirrors?" This is a simpler sub-problem.

Step 3: Define the Base Cases First

Write the stopping conditions before any recursion: (both null → True), (one null → False), (values differ → False). Only after handling these write the recursive case. This prevents you from forgetting edge cases under pressure.

Step 4: Derive the Crossed Comparison

Ask: if L and R are mirrors, what's true of their children? Draw it: L's left child must face outward, and R's right child also faces outward — they're the outer pair. L's right and R's left face inward — they're the inner pair. This gives you the crossing naturally.

🗣️ What to Say to the Interviewer

"I'll solve this with a helper function isMirror(L, R) that checks if two subtrees are mirrors. Base cases: both null → True (empty positions match). One null → False (structural mismatch). Values differ → False. Recursive case: mirror requires outer children to mirror (L.left, R.right) and inner children to mirror (L.right, R.left). The crossing is key — this is what makes it a mirror, not a copy. Entry: call isMirror(root.left, root.right). Root is the axis, not a mirror pair. Complexity: O(n) time — visit each node once. O(h) space — call stack depth equals tree height."
💡 Common Follow-up Questions

"Can you do it iteratively?" Yes — use a queue. Push the initial pair (root.left, root.right). Each iteration, dequeue a pair, check values, enqueue (L.left, R.right) and (L.right, R.left). Identical logic, no call stack.

"What's the space complexity?" Recursive: O(h) stack depth. For balanced trees h = O(log n); for a completely skewed tree h = O(n). The BFS uses O(w) queue space where w is the max width (O(n/2) at the widest level).

7 Summary & Key Takeaways ▾
Symmetric = left subtree is a mirror image of right subtree (not identical).
Mirror comparison always crosses: (L.left, R.right) for outer pair, (L.right, R.left) for inner pair.
Base cases first: both null → True; one null → False; values differ → False.
Time: O(n) — every node is visited once. Space: O(h) for call stack depth.
Iterative BFS with a queue of mirror-pairs is equivalent and avoids deep recursion stacks.
🎯 When to Use This Pattern
  • Any tree problem asking about left-right structural symmetry or equality.
  • Problems comparing two trees simultaneously (same tree, invert, subtree check).
  • Recursive tree problems where the structure is inherently self-similar.
