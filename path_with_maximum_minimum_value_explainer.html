LeetCode #1102

Path With Maximum Minimum Value

Given an m × n integer grid, find the path from (0,0) to (m-1,n-1) that maximizes the minimum cell value along the path.

Medium Graph Dijkstra (Max-Min) Binary Search + BFS Time: O(mn log mn) Space: O(mn)
1 Understanding the Problem ▾

You are given an m × n integer grid. You must travel from the top-left corner (0,0) to the bottom-right corner (m-1, n-1), moving only to adjacent cells (up, down, left, right). The score of a path is the minimum cell value encountered along the entire path.

Your goal: among all possible paths, find the one whose score (minimum cell value) is as large as possible. Return that maximum score.

💡 Key Insight

This is a bottleneck path problem. A path is only as good as its weakest link. Think of it like water flowing through pipes — the flow rate is limited by the narrowest pipe. You want to find a route where even the narrowest pipe is as wide as possible. The strategy: always prefer paths that avoid small-valued cells.

Edge Cases: 1×1 grid (answer is that single cell's value), grid where all values are equal (answer is that value), grid where every path must pass through a single low-value bottleneck cell.

2 The Algorithm: Modified Dijkstra with Max-Heap ▾

Dijkstra's algorithm normally finds shortest paths by always expanding the cheapest known path. We adapt it: instead of cumulative cost, we track the minimum cell value seen so far on the best path to each cell. We use a max-heap to always expand the most promising cell next — the one we can reach with the highest bottleneck score.

1 Initialize: Push (-grid[0][0], 0, 0) onto a max-heap (Python's heapq is a min-heap, so we negate values). The initial score at (0,0) is simply its own value. Mark (0,0) as visited.
2 Pop best cell: Pop the cell with the highest bottleneck score (most negative in heap). This is the "most promising" cell — the one reachable with the largest minimum on the path so far.
3 Check destination: If the popped cell is (m-1, n-1), return its score immediately. By Dijkstra's greedy correctness, this first pop of the destination is guaranteed optimal.
4 Expand neighbors: For each unvisited neighbor (nr, nc), compute: new_score = min(current_score, grid[nr][nc]). Mark it visited and push (-new_score, nr, nc) onto the heap.
5 Repeat: Continue until the destination is popped. Because we always pop the maximum-score cell first, the first time we pop (m-1, n-1), that score is the answer.
🔑 Standard Dijkstra vs. Max-Min Dijkstra
Standard Dijkstra (shortest path)    Max-Min Dijkstra (bottleneck)
────────────────────────────         ────────────────────────────
Score = cumulative edge sum          Score = minimum cell value on path
Goal  = MINIMIZE score               Goal  = MAXIMIZE score
Heap  = min-heap on distance         Heap  = max-heap on min-value (negate)
Update= score + edge_weight          Update= min(score, neighbor_value)

Alternative — Binary Search + BFS: Binary search on the answer threshold T. For each T, ask: "Can we reach (m-1,n-1) using only cells with value ≥ T?" The highest T where this is true is the answer. O(mn · log(maxVal)) — works but requires two nested loops.

3 Why It Works: The Invariant ▾
🔐 The Invariant

When a cell (r, c) is popped from the max-heap, the score associated with it equals the maximum possible minimum value for any path from (0,0) to (r, c). This score cannot be improved by any future path.

Why the invariant holds: Suppose cell X is popped with score S. Any other (unpopped) path to X must pass through cells still waiting in the heap. By the max-heap property, those cells all have scores ≤ S. Extending those paths to reach X can only apply more min() operations, which can only decrease the score further — never increase it. So S is the true optimal score for X.

The min() update rule explained: When extending from cell A (score S_A) to neighbor B (value V_B), the new score becomes min(S_A, V_B). If V_B ≥ S_A: the previous minimum still dominates, score stays S_A. If V_B < S_A: the new minimum becomes V_B. Either way, the score can only stay equal or decrease — a path score is monotonically non-increasing as you travel further.

⚠️ Why Not Just DFS or BFS?

BFS explores by distance (number of steps), not by bottleneck value. DFS explores depth-first with no ordering guarantee. Neither guarantees that the first time you reach the destination gives the optimal score. Dijkstra's greedy selection by bottleneck score is what gives the optimality guarantee.

Why mark visited on PUSH (not on POP): If we mark visited on pop, the same cell can be pushed multiple times by different neighbors, wasting memory and time. Since we always push a cell with the best possible score it can have (the first time any neighbor discovers it with max-heap ordering), marking on push is both correct and efficient.

4 The Code (Annotated) ▾

Solution 1 — Modified Dijkstra / Max-Min Heap (Interview Pick)

import heapq
def maximumMinimumPath(grid):
m, n = len(grid), len(grid[0]) ← grid dimensions
heap = [(-grid[0][0], 0, 0)] ← max-heap: negate value; score at start = its own value
visited = [[False]*n for _ in range(m)] ← O(mn) grid to track finalized cells
visited[0][0] = True ← mark start visited immediately on push
dirs = [(0,1),(0,-1),(1,0),(-1,0)] ← 4-directional movement
while heap:
score, r, c = heapq.heappop(heap) ← pop highest bottleneck cell
score = -score ← un-negate to get real score
if r == m-1 and c == n-1: ← reached destination — guaranteed optimal!
return score
for dr, dc in dirs:
nr, nc = r+dr, c+dc
if 0<=nr<m and 0<=nc<n and not visited[nr][nc]:
visited[nr][nc] = True ← mark visited on PUSH, not pop
new_score = min(score, grid[nr][nc]) ← bottleneck update: path limited by its weakest cell
heapq.heappush(heap, (-new_score, nr, nc))
⚠️ Common Mistake: Forgetting to Negate

Python's heapq is a min-heap. To simulate a max-heap, we push negative values. When we pop, we get the most negative value — which corresponds to the largest original score. Don't forget to un-negate with score = -score after popping, or all your comparisons will be wrong.

Solution 2 — Binary Search + BFS (Alternative)

from collections import deque
def maximumMinimumPath(grid):
m, n = len(grid), len(grid[0])
def can_reach(threshold): ← BFS: can we get to (m-1,n-1) using only cells ≥ threshold?
if grid[0][0] < threshold or grid[m-1][n-1] < threshold:
return False ← start/end blocked — impossible
seen = {(0,0)}; q = deque([(0,0)])
while q:
r, c = q.popleft()
if r == m-1 and c == n-1: return True
for dr,dc in [(0,1),(0,-1),(1,0),(-1,0)]:
nr,nc = r+dr,c+dc
if 0<=nr<m and 0<=nc<n and (nr,nc) not in seen and grid[nr][nc]>=threshold:
seen.add((nr,nc)); q.append((nr,nc))
return False
lo, hi = 0, max(max(r) for r in grid) ← binary search over all possible answer values
while lo < hi:
mid = (lo + hi + 1) // 2 ← upper mid avoids infinite loop when lo+1==hi
if can_reach(mid): lo = mid ← threshold T is achievable, try higher
else: hi = mid - 1 ← too restrictive, try lower
return lo
💡 Complexity Summary

Dijkstra (Max-Min): Time O(mn·log(mn)), Space O(mn) — heap holds at most mn entries
Binary Search + BFS: Time O(mn·log(V)) where V = max cell value, Space O(mn)
For sparse grids or small V, Binary Search + BFS can win. Dijkstra is preferred in interviews for clarity.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Bottleneck Pattern

Key signal: "maximize the minimum" on a path. This is a bottleneck problem. The two standard tools are: (1) Modified Dijkstra with max-heap, or (2) Binary search on the answer + BFS feasibility check. Mention both upfront.

Step 2: Verify Dijkstra Applies

Dijkstra works when: the "score" of a path is determined by local decisions (here: min of cells visited), and the greedy property holds (popping the best-score cell gives us its optimal score permanently). Both hold here — confirm this to the interviewer.

Step 3: Code the Key Adaptation

The only changes from standard Dijkstra: (1) use max-heap (negate values in Python), (2) update score with min() instead of addition, (3) mark visited on push (not pop). Three lines of change from the classic template.

Step 4: State the Termination Condition

Explicitly say: "The moment I pop the destination cell from the heap, I return its score. I don't need to process all cells. Dijkstra guarantees this is optimal because any other path to the destination would have a lower or equal bottleneck score."

🗣️ What to Say to the Interviewer

"This is a maximize-the-minimum bottleneck path problem. I'll use a modified Dijkstra.
Instead of tracking cumulative distance, I track the minimum cell value on the best path to each cell.
I use a max-heap so I always expand the cell with the highest bottleneck score first.
Score update: new_score = min(current_score, neighbor_value).
When I pop the destination, that score is guaranteed optimal — return it immediately.
Time: O(mn log mn) for heap operations. Space: O(mn) for the visited grid and heap."
🔮 Likely Follow-up Questions
  • "Solve it with binary search instead?" → Binary search on threshold T, BFS checks if we can reach destination using cells ≥ T
  • "What if you could move diagonally?" → Add 4 diagonal directions to dirs; same algorithm, same complexity
  • "Path With Minimum Effort?" → Same Dijkstra, but score = max(absolute difference between adjacent cells), use min-heap (#1631)
  • "Swim in Rising Water?" → Minimize the max cell on any path — same pattern, different objective (#778)
