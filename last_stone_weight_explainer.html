LeetCode #1046

Last Stone Weight

Repeatedly smash the two heaviest stones together; return the weight of the last remaining stone, or 0 if none remain.

Easy Heaps Max Heap Simulation Time: O(n log n) Space: O(n)
1 Understanding the Problem ▾

You have a collection of stones, each with a positive integer weight. Each round, pick the two heaviest stones and smash them together. Two outcomes:

  • If both weigh the same (x == y): both are destroyed — nothing remains from this pair.
  • If they differ (x < y): the lighter stone is destroyed, the heavier survives with weight y − x.

Repeat until at most one stone remains. Return that stone's weight, or 0 if no stones are left.

💡 Key Insight

Each round you need the two largest elements — fast extraction from the top and re-insertion of a remainder. This is exactly what a max-heap (priority queue) is built for. Each smash is O(log n), and we do at most n smashes → O(n log n) total.

✓ Example: stones = [2, 7, 4, 1, 8, 1]

Round 1: smash 8 & 7 → remainder 1
Heap: [4, 2, 1, 1, 1]
Round 2: smash 4 & 2 → remainder 2
Heap: [2, 1, 1, 1]
Round 3: smash 2 & 2 → both gone
Heap: [1, 1]
Round 4: smash 1 & 1 → both gone
Heap: [] → return 0

Edge Cases

[1] → return 1 (no smash possible)
[3, 3] → return 0 (equal, both gone)
[5, 3] → return 2 (5−3=2 survives)
All equal → all cancel → return 0

The core challenge: we need the maximum element repeatedly, and after each round we may insert a new value. A sorted array requires O(n) re-insertion. A heap gives O(log n) — that is the right tool here.

2 The Algorithm: Max Heap Simulation ▾

Python's heapq module provides only a min-heap. The standard trick to simulate a max-heap: negate all values before insertion, negate again when reading. The smallest negative maps to the largest original value.

1 Negate & heapify: Convert every weight to its negative: heap = [-s for s in stones]. Then call heapq.heapify(heap) in O(n) to build the min-heap in one pass. Popping gives the most negative value = the largest original weight.
2 Smash loop: While len(heap) > 1, pop twice. The first pop yields y (heaviest), the second yields x (second heaviest). Both are un-negated: y = -heapq.heappop(heap), x = -heapq.heappop(heap). Guaranteed: y ≥ x.
3 Apply smash rule: If y > x, push -(y − x) back (remainder survives). If y == x, push nothing — both destroyed. The heap re-balances after each push via heapq.heappush.
4 Return answer: Loop exits when 0 or 1 stones remain. Return -heap[0] if the heap has one element, or 0 if it is empty.
🔑 Max-Heap via Negation — The Standard Python Trick

Python's heapq is always a min-heap. Negating input makes the min-heap behave as a max-heap. −8 is "smaller" than −7, so popping gives −8, which un-negated is 8 — the correct maximum. Every heap problem in Python uses this pattern.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After every round, the heap contains exactly the weights of all surviving stones — no more, no fewer. The max-heap property guarantees that each pair of pops yields the two globally largest stones in the current collection, which is exactly what the problem requires.

Why two consecutive pops give the two largest: A max-heap's root is always the global maximum. After popping the root, the next root becomes the new maximum. So two consecutive pops yield elements #1 and #2 in sorted-descending order — that is y ≥ x by construction.

Why the smash rule is correctly simulated:

  • If y == x: both gone. Two pops, zero pushes. Heap shrinks by 2. Correct.
  • If y > x: remainder y−x pushed back. Heap shrinks by 1 (two pops, one push). Heap invariant restored by heappush. Correct.

Termination guarantee: Each round reduces the stone count by at least 1. With n initial stones, the loop runs at most n−1 rounds. Total: O(n) rounds × O(log n) heap operations = O(n log n).

⚠️ Common Confusion: Order of Pops

The first heappop from the negated min-heap returns the smallest negative = largest positive. That is y. The second pop returns the next largest. Always assign: y = -heapq.heappop(heap) first, then x = -heapq.heappop(heap). Then y ≥ x is guaranteed — never swap them.

4 The Code (Annotated) ▾

Solution 1 — Max Heap (Optimal, Interview Pick)

import heapq← Python's standard heap module (min-heap)
def lastStoneWeight(stones: list) -> int:
heap = [-s for s in stones] ← negate all weights → min-heap acts as max-heap
heapq.heapify(heap) ← O(n) heapify — faster than n individual pushes
while len(heap) > 1: ← keep going while 2+ stones remain
y = -heapq.heappop(heap) ← pop most-negative → largest stone (y)
x = -heapq.heappop(heap) ← pop next → second largest (x); y ≥ x guaranteed
if y > x: ← stones differ → remainder y-x survives
heapq.heappush(heap, -(y-x)) ← push negated remainder back into heap
# if y == x: both destroyed, push nothing
return -heap[0] if heap else 0 ← un-negate last survivor; 0 if heap is empty
⚠️ Why heapify over a loop of heappush?

heapq.heapify uses Floyd's algorithm — it sifts down from the middle outward — running in O(n). Building a heap by n individual heappush calls costs O(n log n). For this problem the sizes are small, but always prefer heapify: it is the correct and faster idiom.

Solution 2 — Brute Force Sorting (simpler, slower)

def lastStoneWeight_brute(stones):
stones = list(stones) ← copy input — don't mutate original
while len(stones) > 1:
stones.sort() ← O(n log n) per round — huge waste
y = stones.pop() ← largest (after sorting, last element)
x = stones.pop() ← second largest
if y > x:
stones.append(y - x) ← insert remainder (unsorted position OK, will re-sort)
return stones[0] if stones else 0
💡 Complexity Summary

Max Heap: Time O(n log n) — heapify O(n) + at most n rounds × O(log n) per pop/push. Space O(n) for heap.
Brute Force Sort: Time O(n² log n) — O(n log n) sort × O(n) rounds. Space O(1) extra (sort is in-place).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Repeated Operation

"What operation repeats?" — extracting the two maximum elements. "What changes?" — stones are removed and sometimes a new one is added. Repeated max-extraction with dynamic insertion = priority queue signal.

Step 2: Propose Brute Force First

"Naively, I can sort the array each round and pop the last two elements. O(n² log n). Correct but wasteful — I'm re-sorting the entire array just to find the top two, which a heap can give in O(log n)."

Step 3: Pitch the Heap Optimization

"A max-heap gives O(log n) for both extraction and insertion. I'll use Python's heapq with negated values to simulate a max-heap. Heapify once in O(n), then loop: pop y, pop x, conditionally push remainder. Total: O(n log n)."

Step 4: Handle the Negation Carefully

The negation trick is the only gotcha here. Explain it explicitly: "heapq is min-heap; I negate all values so the smallest negative corresponds to the largest original weight. When reading back, I negate again."

🗣️ What to Say to the Interviewer

"This problem repeatedly needs the two largest elements with dynamic updates — that's a max-heap.
Python's heapq is min-heap, so I'll negate all values: -8 is smaller than -7, so popping gives -8 → weight 8.
Step 1: heap = [-s for s in stones]; heapq.heapify(heap) → O(n).
Step 2: while len(heap) > 1: pop y, pop x (both un-negated).
Step 3: if y > x: push -(y-x) back. If equal: push nothing.
Step 4: return -heap[0] if heap else 0.
Time: O(n log n). Space: O(n) for the heap."
🔮 Likely Follow-up Questions
  • "What if stones have float weights?" → Heap still works; Python heapq handles floats fine.
  • "Can you do it in-place (O(1) extra space)?" → Negate and heapify the original array in-place: no new list needed.
  • "What if you need the k heaviest at each step?" → Pop k times instead of 2; push back the cumulative remainder.
  • "Last Stone Weight II" → Harder variant — partition stones into two groups to minimize |sum1 - sum2|; uses 0/1 knapsack DP (#1049).

🎯 When to Recognize the Max Heap Simulation Pattern
  • "Repeatedly pick the largest (or smallest) element" → priority queue
  • "Elements are added and removed dynamically across rounds" → heap over sorting
  • "Merge, smash, or combine the two heaviest/lightest each step" → this exact pattern
  • "Top-K elements" with updates → min-heap of size K for K largest; max-heap for K smallest
