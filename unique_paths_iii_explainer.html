Unique Paths III

Count all paths from Start to End that walk over every non-obstacle cell exactly once.

Hard Backtracking Visit All Empty Cells Time: O(4^(m┬╖n)) Space: O(m┬╖n)
1 Understanding the Problem тЦ╛

You are given an m ├Ч n integer grid where each cell holds one of four values:

Find the number of 4-directional walks from Start to End that step on every non-obstacle, non-start square exactly once. You may move up, down, left, or right тАФ no diagonals, no cell revisits.

ЁЯТб Key Insight

This is a Hamiltonian Path problem on a grid. The path must be a complete "tour" тАФ every walkable cell visited exactly once, ending at the target. There is no efficient greedy or DP shortcut: we must try all routes, but we can prune dead branches the moment the path can no longer reach the required coverage.

2 The Algorithm: DFS Backtracking with Coverage Count тЦ╛

We use depth-first search (DFS) with backtracking. At each cell we try all 4 neighbors, mark the current cell as visited by overwriting it with -1, recurse, then restore the original value on the way back. A counter called remaining tracks how many non-obstacle cells still need to be visited.

1 Pre-scan: Count all non-obstacle cells тЖТ remaining. Locate the Start cell (sr, sc). This count includes Start, all empty cells, and End.
2 Launch DFS: Call dfs(sr, sc, remaining - 1) тАФ we subtract 1 because Start is already "on" when we begin.
3 Base case тАФ End reached: If grid[r][c] == 2, return 1 if remaining == 1 (End itself is the last cell), else 0.
4 Mark & recurse: Save original value, overwrite with -1 (visited). Try all 4 neighbors in bounds and not blocked. Pass remaining - 1 to each recursive call.
5 Backtrack: After all recursions, restore grid[r][c] to its saved original value. This "un-visits" the cell so other paths can use it.
ЁЯФС The In-Place Visited Trick

Instead of a separate boolean visited[r][c] array, we temporarily overwrite cells with -1 (the obstacle value). Any neighbor check that skips -1 cells automatically skips visited cells too. Restoring the original on backtrack costs O(1) and keeps the algorithm space-elegant.

тЪая╕П Off-by-One: Why remaining == 1 at End?

Our pre-scan includes the End cell in remaining. We call dfs(sr, sc, remaining-1) to account for Start. Each step into a non-End, non-obstacle cell decrements remaining. When we arrive at End, we haven't decremented for End yet тАФ so remaining == 1 means "End is the only cell left", i.e., we've visited everything else.

3 Why It Works: The Invariant тЦ╛
ЁЯФР The Invariant

At every point in the DFS, remaining equals the number of non-obstacle cells (including End) that the current path has not yet stepped on. The path is valid iff it reaches End with remaining == 1.

Why correctness is guaranteed: Initially, remaining = all walkable cells тИТ 1 (for Start). Each time we move to an empty cell or End, we decrement. When we arrive at End and remaining == 1, every other walkable cell must have been decremented тАФ they were all visited. This is exactly the Hamiltonian path condition.

Backtracking completeness: We try every possible path through the 4-directional grid. Any path that hits a dead end (no valid neighbors, or reaches End too early) returns 0 without counting. Because we restore the grid on backtrack, every other branch gets the correct unvisited grid state. No path is missed, and no path is double-counted.

Why we can't use DP here: Dynamic programming requires that subproblem answers can be reused. But here, the answer for "from cell X to End" depends on which cells are already visited, which changes for every path. To capture that with DP you'd need bitmask DP тАФ exponential state space. For small grids bitmask DP exists but DFS backtracking is simpler and equally fast for this problem size.

тП▒ Complexity

Time: O(4m┬╖n) worst case тАФ each cell can branch in 4 directions. In practice obstacles and coverage checks prune heavily; constraints keep m┬╖n тЙд 20. Space: O(m┬╖n) for the recursion call stack (max depth = number of walkable cells).

тЪая╕П Common Mistake: Decrementing for End

Do NOT decrement remaining for End before the base-case check. End is included in the initial remaining count, and the check remaining == 1 uses that fact. If you decrement at End too, you'd need to check remaining == 0, which works тАФ just be consistent. The code above uses the "decrement only for non-End" style implicitly.

5 ЁЯОд Interview Strategy: How to Arrive at This тЦ╛
Step 1: Pattern Recognition

Key signals: "count all paths", "visit every cell exactly once", 4-directional grid movement. The phrase "exactly once" kills DP shortcuts тАФ you'd need exponential state. This is a Hamiltonian Path count тЖТ backtracking is the canonical approach.

Step 2: Define the State

State = (current position, which cells remain to be visited). We represent "which remain" as a single integer counter remaining. No need for a bitmask since we can use in-place marking.

Step 3: In-Place Visited Trick

Rather than a separate visited matrix, overwrite visited cells with -1 (the obstacle sentinel). Restore on backtrack. This is a standard grid-backtracking pattern тАФ mention it explicitly to show awareness of space optimization.

Step 4: Handle the Off-by-One on remaining

Pre-scan counts all walkable cells including Start and End. Subtract 1 before first DFS call (for Start). At End, check remaining == 1 (End itself). This is the trickiest part тАФ trace through on a small example with the interviewer to verify.

ЁЯЧгя╕П What to Say to the Interviewer

"We need to count paths that cover every non-obstacle cell тАФ that's a Hamiltonian path problem. Since the grid can be at most 4├Ч5 = 20 cells, backtracking is feasible.

My plan: pre-scan to count walkable cells and find Start. Then DFS with 4 directions, marking visited in-place by overwriting with -1 and restoring on backtrack. The base case is reaching End when remaining == 1.

Complexity is O(4^(m*n)) but constraints keep this under 4^20 тЙИ 10^12... hmm, that's a lot. In practice, obstacles and dead-end pruning cut this drastically. ЁЯТб Potential Follow-ups
  • Can we add pruning? Yes тАФ check if the remaining unvisited cells are still all reachable (connectivity check). If splitting into disconnected components, prune immediately.
  • Bitmask DP alternative? State = (position, visited bitmask). O(m┬╖n┬╖2^(m┬╖n)) time and space тАФ same exponential order but different constant. Backtracking is simpler to code.
  • What if the grid is large? Then this problem becomes NP-complete (Hamiltonian Path) and no polynomial solution exists.

ЁЯза Analogy: Painting Every Tile

Imagine you're a floor painter who must paint every tile in a room exactly once before leaving through the exit. You paint as you walk, mark each tile (so you don't repaint it), and if you get stuck in a corner without reaching unpainted tiles, you step back (backtrack) and try a different route. You count how many total routes let you paint every tile and exit cleanly.
