LeetCode #202

Happy Number

Determine if a number n is "happy": repeatedly replace it with the sum of squares of its digits until it equals 1 (happy) or loops endlessly in a cycle that never includes 1 (not happy).

Easy Linked List Detect Cycle Time: O(log n) Space: O(1)
1 Understanding the Problem ▾

We apply a simple transformation repeatedly: replace the number with the sum of the squares of its digits. Two things can happen — the number eventually reaches 1 (happy!), or it falls into an infinite loop that never reaches 1 (not happy).

✓ Happy: n = 19

19 → 1²+9² = 1+81 = 82
82 → 8²+2² = 64+4 = 68
68 → 6²+8² = 36+64 = 100
100 → 1²+0²+0² = 1 ✓

✗ Not Happy: n = 2

2→4→16→37→58→89
→145→42→20→4
→16→37→... (cycle!)

💡 Key Insight: It's a Cycle Detection Problem

Think of each number as a node in a linked list, where each node's "next pointer" leads to its digit-square-sum. A happy number's path eventually reaches 1 (then stays at 1). A non-happy number's path enters a cycle that never touches 1. Detecting this cycle is the problem.

Mathematical fact: It is proven that all non-happy numbers eventually enter the cycle: 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4. This is the only possible non-1 cycle.

Edge cases: n=1 is immediately happy (1²=1). n=7 reaches 1 after several steps. All inputs are positive integers.

2 The Algorithm: Floyd's Fast-Slow Pointer (Tortoise and Hare) ▾

Imagine two runners on a circular track: a slow tortoise taking one step at a time, and a fast hare taking two steps at a time. If the track has a loop, the hare will eventually lap the tortoise — they'll be at the same position. If the track ends (reaches 1), both will arrive there.

1 Define the helper digit_sq_sum(x): extract each digit via divmod(x, 10), square it, and sum. This is the "next" function of the implicit linked list.
2 Initialize: slow = n (tortoise, 1 step/iter) and fast = digit_sq_sum(n) (hare, starts 1 step ahead and takes 2 steps/iter).
3 Loop while slow ≠ fast: advance slow = f(slow) one step, fast = f(f(fast)) two steps.
4 Meeting point: When slow == fast, check the value. If it equals 1, the number is happy. Otherwise, they've met inside the non-1 cycle.

🔑 Why Floyd's — Not Just a Hash Set?

A hash set (track all seen values, return False on repeat) is the intuitive solution and is O(log n) time — but uses O(log n) space for the set. Floyd's algorithm detects cycles in O(1) space using only two integer variables. In an interview, propose the hash set first, then say "I can optimize space to O(1) with Floyd's cycle detection."

3 Why It Works: The Invariant ▾
🔐 Floyd's Algorithm — Origin and Invariant

Origin: Robert W. Floyd, 1967. Also called the "tortoise and hare" algorithm. Designed to detect cycles in sequences in O(1) space.

Core Invariant: If a cycle of length L exists, fast gains exactly 1 step on slow per iteration. Within at most L more iterations after both enter the cycle, fast laps slow and they meet. The meeting is guaranteed.

Why the sequence must be bounded: Under digit-square-sum, large numbers shrink rapidly. A number with d digits maps to a value at most 81d — so any 4+ digit number maps to a 3-digit range. The sequence can only visit finitely many distinct values, so it must eventually repeat. Once it repeats, it loops forever.

Proof that two outcomes exist:

  • Happy numbers: sequence reaches 1. Then 1 → 1 → 1 ... (1 is a fixed point). Both pointers reach 1. Loop exits with slow == fast == 1.
  • Non-happy numbers: sequence enters the 4→16→...→20→4 cycle. Once both pointers are in the cycle, fast laps slow in at most 8 steps (cycle length = 8). They meet at some value ≠ 1.
⚠️ Common Misconception: "What if the meeting point isn't 1 or 4?"

The loop condition while slow != fast exits whenever they meet — at any value. After the loop, we simply check return slow == 1. Whether they met at 4, 89, 145, or anything else, if it's not 1, the answer is False. We don't need to know the specific cycle values.

Complexity: The "tail" before entering the cycle or reaching 1 is O(log n) steps (numbers shrink). Inside the cycle, meeting takes at most 8 more steps (constant). Total: O(log n) time, O(1) space.

4 The Code (Annotated) ▾

Solution 1 — Floyd's Cycle Detection [Interview Pick] — O(log n) / O(1)

def isHappy(n: int) -> bool:
def digit_sq_sum(x): ← "next pointer": compute digit-square-sum
total = 0
while x: ← process digits one at a time right-to-left
x, d = divmod(x, 10) ← d = last digit; x = remaining digits
total += d * d ← add square of this digit
return total
slow = n ← tortoise starts at n
fast = digit_sq_sum(n) ← hare starts one step ahead
while slow != fast: ← loop until they meet (cycle detected)
slow = digit_sq_sum(slow) ← tortoise: 1 step
fast = digit_sq_sum(digit_sq_sum(fast)) ← hare: 2 steps
return slow == 1 ← met at 1 → happy; met at cycle → not happy
⚠️ Why slow != fast on exit (loop condition) rather than slow == 1?

If we loop while slow != 1, we'd loop forever on non-happy numbers since the sequence never reaches 1. Floyd's approach exits on ANY meeting of slow and fast — then we check where they met. Both techniques are valid; Floyd's is smarter because it handles both outcomes with the same loop exit condition.

Solution 2 — Hash Set — O(log n) / O(log n)

def isHappy(n: int) -> bool:
seen = set() ← track all values visited so far
while n != 1 and n not in seen:← stop if happy OR if we see a repeat (cycle)
seen.add(n) ← mark this value visited
n = sum(int(d)**2 for d in str(n))← Pythonic digit-square-sum
return n == 1 ← stopped at 1 → happy; stopped at repeat → not

Solution 3 — Hardcode the Known Cycle — O(log n) / O(1)

def isHappy(n: int) -> bool:
CYCLE = {4, 16, 37, 58, 89, 145, 42, 20}← the only non-1 cycle
while n != 1 and n not in CYCLE:
n = sum(int(d)**2 for d in str(n))
return n == 1
💡 When to Prefer Each Approach

Interview: Start with hash set (clearest intent), then upgrade to Floyd's for O(1) space.
Production: Hash set is most readable and maintainable.
Trivia/brain teaser: Hardcode the cycle — but requires memorizing a math fact.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Understand What "Forever" Means

"If the sequence never reaches 1, it must eventually revisit a number — otherwise it'd need to visit infinitely many values, but numbers are bounded by the transformation. So 'loops forever' means 'enters a cycle.' I need cycle detection."

Step 2: Propose Hash Set First (Brute Force)

"Simplest: store every value I compute. If I see a repeat, return False. If I reach 1, return True. Time O(log n), Space O(log n) for the set. This is clear and correct — let me code it."

Step 3: Identify the Linked List Analogy

"Each number has exactly one 'next' value: its digit-square-sum. This is an implicit linked list. Cycle detection in a linked list with O(1) space = Floyd's algorithm — two pointers, slow (1 step) and fast (2 steps)."

Step 4: Implement Floyd's and Verify Termination

"Loop exits when slow == fast. If they meet at 1, happy. Otherwise they met in a cycle. Either way terminates — happy numbers reach 1 (fixed point), non-happy enter the known 8-element cycle. Fast always catches slow within cycle-length iterations."

🗣️ What to Say to the Interviewer

"The sequence is bounded, so it must eventually cycle or reach 1.
I'll model it as an implicit linked list — each number points to its digit-square-sum.
Brute force: hash set to track visited values — O(log n) time, O(log n) space.
Optimization: Floyd's fast-slow pointers — same time but O(1) space.
Slow moves 1 step, fast moves 2 steps; they meet when a cycle exists.
If meeting value is 1: happy. Otherwise: trapped in non-1 cycle.
Time: O(log n) — large numbers shrink fast. Space: O(1)."
🔮 Likely Follow-up Questions
  • "Why O(log n) not O(n)?" → Large n quickly reduces to 3-digit range (max 81×d digits). Tail before cycle is O(log n).
  • "Can you find where the cycle starts?" → Yes — Floyd's phase 2 (reset slow to n, advance both at same speed). But not needed here.
  • "Find the Duplicate Number (#287)" → Same trick: array as implicit linked list, Floyd's finds duplicate.
  • "What's the cycle for non-happy numbers?" → Always 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4 (proven mathematically).

7 Summary & Key Takeaways ▾
Happy Number is a cycle detection problem disguised as a math puzzle — the transformation creates an implicit linked list
Floyd's fast-slow pointer detects cycles in O(1) space: fast laps slow inside any cycle
Meeting at 1 means happy; meeting at any other value means trapped in the non-1 cycle
Large numbers shrink rapidly under digit-square-sum, so the sequence is bounded and terminates
In interviews: propose hash set first, then upgrade to Floyd's when asked to optimize space
