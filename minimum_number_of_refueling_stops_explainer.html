LeetCode #871

Minimum Number of Refueling Stops

A car starts at position 0 with startFuel litres. Find the minimum refueling stops to reach the target, or return -1 if impossible.

Hard Greedy Max Heap Time: O(n log n) Space: O(n)
1 Understanding the Problem ▾

A car starts at position 0 with startFuel litres of fuel. Each unit of fuel moves 1 unit of distance. You want to reach target. Along the way are gas stations: stations[i] = [position, fuelAvailable]. You may stop at any station you physically reach — but you want to stop as few times as possible.

💡 Key Insight

You don't have to decide at each station whether to stop. Instead, defer all refueling decisions. As you drive past stations, record their fuel as "available options." When you run dry, retroactively pick the richest station you've passed. This "lazy greedy" always uses the minimum number of stops.

Edge cases: If startFuel >= target, return 0 immediately. If the heap empties while fuel is still negative, return -1. Stations are already sorted by position — no need to sort.

⚠️ Common Misconception

"Stop whenever fuel is low" or "stop at every station" — both are suboptimal. The optimal greedy is: never stop unless forced, and when forced, pick the richest passed station retroactively.

2 The Algorithm: Greedy with Max-Heap of Passed Stations ▾

Drive forward through stations in order. At each stop, push its fuel onto a max-heap (but don't actually stop yet). When fuel goes negative — car is stuck — greedily pop the largest fuel value from the heap and add it to the tank. Count each pop as one stop. Repeat until fuel is non-negative or heap is empty (impossible).

1 Init: heap=[], fuel=startFuel, stops=0, prev=0. Append [target, 0] to stations as a virtual endpoint.
2 Travel: fuel -= (pos - prev). Subtract the distance from the last position to current.
3 Stuck? While fuel < 0 and heap is non-empty, pop the largest value and add it to fuel. Increment stops each pop.
4 Impossible? If fuel < 0 and heap is empty, return -1.
5 Pass station: Push current station's fuel onto the heap (-cap for max-heap). Update prev = pos.
6 Return: After processing the virtual target station, return stops.
🔬 Deep Dive: Why Retroactive Greedy Works

Core invariant: At every step, fuel represents the maximum fuel we could have using at most stops refueling choices from the stations passed so far. The heap holds unchosen "deferred" options.


Greedy-choice property: When stuck, choosing the station with the most fuel is always at least as good as any other choice. More fuel per stop = fewer stops total. This is provable by exchange argument: any solution that uses a different stop at this point can be swapped to use our max-fuel stop without increasing the stop count.


Relation to DP: The O(n²) DP approach sets dp[i] = maximum distance reachable with exactly i stops. The heap approach implicitly finds the same optimal answer in O(n log n) by ordering choices greedily.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After each refuel decision, fuel = the maximum possible fuel achievable using exactly stops refueling choices from all stations passed so far. No other selection of stops stations gives more remaining fuel.

Why max-fuel picks are globally optimal: Suppose the real optimal solution uses stops at stations A, B, C. Our greedy will also make exactly 3 stops. Each time we're forced to stop, we pick the maximum-fuel option, which can only be ≥ any option the optimal solution picks. So our fuel at every point is at least as high — meaning we need at most as many stops.

Empty heap = truly impossible: If the heap empties while fuel is still negative, it means the sum of all available fuel from all passed stations is insufficient to bridge the gap. Even using every possible station wouldn't help — return -1.

Virtual target station trick: Appending [target, 0] as a station ensures the loop processes the final leg (last real station → target) using the same logic. The 0 fuel prevents any spurious refueling at the destination.

4 The Code (Annotated) ▾
import heapq← Python's min-heap; we'll negate values to simulate max-heap
def minRefuelStops(target, startFuel, stations):
heap = [] ← stores negated fuel values of passed stations
fuel = startFuel ← current tank level
stops = 0 ← refueling stop count
prev = 0 ← last visited position (for distance calculation)
for pos, cap in stations + [[target, 0]]:← virtual target station unifies final-leg logic
fuel -= pos - prev ← deduct fuel for distance traveled
while fuel < 0 and heap: ← stuck! retroactively refuel with best passed station
fuel -= heapq.heappop(heap) ← pop negated max → subtract negative = add fuel
stops += 1 ← each pop counts as one stop
if fuel < 0: ← heap empty, still negative → impossible
return -1
heapq.heappush(heap, -cap) ← store negated cap; this station's fuel is now "available"
prev = pos ← advance position marker
return stops← minimum stops needed
⚠️ Max-Heap Trick: Why Negate?

Python's heapq always gives the smallest element first (min-heap). We want the largest fuel station. By storing -fuel, the richest station has the most-negative value — and thus pops first. When we pop -60, we do fuel -= (-60) which equals fuel += 60. Elegant negation trick!

⚠️ Virtual Target Station

We append [target, 0] to stations so the loop handles the final leg (last station → target) identically to all other legs. The cap=0 means heappush(heap, 0) adds nothing useful — it's harmless and avoids a separate post-loop check.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize Minimum-Count Optimization

"Minimum number of X to satisfy Y" → think Greedy first. If greedy doesn't work, fall back to DP. Here the problem has a greedy-choice property (provable), so greedy wins.

Step 2: Sketch the DP (Show Depth)

Briefly mention: dp[i] = max distance reachable with exactly i stops → O(n²). Then say: "Can we speed this up?" Yes — greedy with a heap eliminates the inner loop.

Step 3: Discover the Retroactive Greedy

Key question: "Do I need to decide at each station, or can I defer?" Answer: defer. Pass all stations, record their fuel, and only "stop" when forced. At that point, retroactively pick the richest one passed.

Step 4: Choose Data Structure

"I need fast access to the maximum fuel among passed stations." → Max-heap (priority queue). O(log n) push/pop. Total: O(n log n).

Step 5: Handle Edge Cases Out Loud

Mention: startFuel ≥ target (return 0), heap exhaustion (return -1), virtual target station trick to simplify the final-leg check.

if fuel < 0:
return -1

prev = pos
return stops

7 Summary & Key Takeaways ▾
Defer all refueling decisions until you run out — then pick the richest passed station
Use a max-heap (negated min-heap in Python) of passed stations' fuel values
Each heap pop = one retroactive refueling stop; increment stops counter each time
Append target as a virtual station with 0 fuel to unify the final-leg logic
If heap empties while fuel < 0: return -1 (truly unreachable)
🎯 Recognize This Pattern When...
  • Minimizing the count of discrete "actions" (stops, jumps, swaps) to meet a threshold
  • Resources are encountered sequentially and each has a value you can greedily maximize
  • The decision of which resource to use can be deferred until you're forced to choose
  • Optimal choice at each forced decision = maximum available value (greedy-choice property holds)
