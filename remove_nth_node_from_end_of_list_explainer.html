LeetCode #19

Remove Nth Node From End of List

Given the head of a linked list, remove the n-th node from the end and return its head — in one pass.

Medium Linked List Two Pointers (N Gap) Time: O(L) Space: O(1)
1 Understanding the Problem ▾

We receive a singly linked list and must remove a node identified by its position counting backwards from the tail. "Remove the 2nd from the end" in [1→2→3→4→5] means removing node 4 — the result is [1→2→3→5].

The core difficulty: linked lists have no random access. We can only follow .next pointers forward. To know where "n from the end" is, we'd normally need to first reach the end — forcing two passes. The challenge is doing it in one pass.

💡 Key Insight

To remove node X we need its predecessor (the node just before X), so we can do predecessor.next = X.next. The predecessor of the n-th-from-end is the (n+1)-th-from-end. If we can find that predecessor in one pass, we're done.

n=2 on [1→2→3→4→5]

2nd from end = node 4
predecessor = node 3
3.next = 3.next.next (= 5)
Result: [1→2→3→5] ✓

n=5 (remove head)

5th from end = node 1 (head)
predecessor = none ← tricky!
Use a dummy node before head
Result: [2→3→4→5] ✓

Edge cases: n equals list length → remove the head (need a dummy node). Single-node list with n=1 → return None. The problem guarantees 1 ≤ n ≤ length, so n is always valid.

2 The Algorithm: One-Pass Two-Pointer Gap ▾

The clever observation: if two pointers maintain a gap of exactly n+1 nodes between them, then when the leading pointer (fast) reaches None, the trailing pointer (slow) will be exactly at the predecessor of the node to remove.

Why n+1 and not n? A gap of n would land slow on the target itself — but we need its predecessor. Adding 1 to the gap puts slow one step earlier, which is exactly the predecessor position.

1 Create a dummy sentinel node with dummy.next = head. Set both fast and slow to dummy. The dummy makes head-removal uniform — no special-case needed.
2 Advance fast by n+1 steps. After this, the gap from slow (at dummy) to fast is exactly n+1. This "pre-loads" the spacing we need.
3 Move both pointers in lockstep (one step at a time) until fast becomes None. The gap of n+1 is preserved throughout. When fast hits None, slow is at the predecessor of the target.
4 Unlink the target: slow.next = slow.next.next. This bypasses the target node. Return dummy.next as the new head (not head, since the original head might have been removed).
🔑 Proof of Positioning

List has L real nodes. After advancing fast n+1 steps from dummy, fast is at 1-indexed position n in the list. Lockstep runs L - n more steps until fast is None. Slow has moved L - n steps from dummy → slow is at 1-indexed position L - n. The target (nth from end) is at position L - n + 1 = slow.next. Exactly right.

3 Why It Works: The Invariant ▾
🔐 The Invariant

Throughout the lockstep phase, the distance (in nodes) from slow to fast is always exactly n+1. This gap is created once during the advance phase and preserved step-by-step since both pointers move together.

Why the dummy node eliminates edge cases: When n = L (remove the head), we need the predecessor of the head, which doesn't exist in the original list. The dummy node acts as a virtual predecessor at position 0. Fast advances n+1 = L+1 steps from dummy and immediately reaches None (since the list only has L nodes). Slow stays at dummy. Then slow.next = slow.next.next sets dummy.next = head.next, effectively removing the head. No branching needed.

⚠️ Common Mistake: Return head Instead of dummy.next

Always return dummy.next, not the original head variable. If n == L (head was the target), head is now a dangling reference to the deleted node. dummy.next always points to the correct new head, whether the original head was removed or not.

Single-element list, n=1: fast advances 2 steps from dummy → immediately None. Slow stays at dummy. dummy.next = dummy.next.next = None. Returns None. Correct.

4 The Code (Annotated) ▾

Solution 1 — One-Pass Two-Pointer Gap (Optimal, Interview Pick)

def removeNthFromEnd(head, n):
dummy = ListNode(0) ← Sentinel node; makes head-deletion uniform
dummy.next = head ← Attach real list to dummy
fast = slow = dummy ← Both start at dummy (before index 0)
for _ in range(n + 1): ← Advance fast n+1 steps to create gap
fast = fast.next ← Each step moves fast one node forward
while fast is not None: ← Lockstep: run until fast falls off end
fast = fast.next ← Fast advances
slow = slow.next ← Slow follows; gap stays n+1
slow.next = slow.next.next ← Unlink target; slow.next is the target
return dummy.next ← Always return dummy.next (never bare head)
⚠️ Why n+1 Steps and Not n?

If fast advances only n steps, when fast reaches None, slow would be sitting on the target node itself. You cannot unlink a node without its predecessor. The extra step ensures slow is one position earlier — at the predecessor — when fast exhausts the list.

Solution 2 — Two-Pass (Count Length First)

def removeNthFromEnd(head, n):
dummy = ListNode(0, head) ← Dummy with next=head
L, cur = 0, head
while cur: ← Pass 1: count list length
L += 1; cur = cur.next
cur = dummy ← Pass 2: walk to predecessor
for _ in range(L - n): ← L-n steps from dummy → predecessor
cur = cur.next
cur.next = cur.next.next ← Unlink
return dummy.next
💡 Complexity Summary

One-Pass Gap: Time O(L), Space O(1) — single traversal, constant extra memory
Two-Pass: Time O(L), Space O(1) — traverses list twice, simpler reasoning
Same asymptotic complexity; one-pass has better constant factor and is more impressive in interviews.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Draw an Example and Identify What We Need

Draw [1→2→3→4→5], n=2 on the board. Mark the target (node 4) and its predecessor (node 3). Articulate: "I need to find node 3, the predecessor of the nth-from-end, and do predecessor.next = predecessor.next.next."

Step 2: Propose the Two-Pass Solution First

"The simple approach: one pass to count length L, then walk L-n steps from dummy to find the predecessor. Clean, O(L) time, O(1) space. Two traversals though — can I do it in one pass?"

Step 3: Derive the Gap Trick

"Instead of counting, I want to measure 'n from the end' dynamically. If fast is always n+1 ahead of slow, when fast hits None, slow is exactly at the predecessor. I pre-load the gap, then march both together."

Step 4: Explain the Dummy Node

"To remove the head without a special case, I add a dummy node before it. Both pointers start at dummy. When fast advances n+1 steps past dummy and n equals the list length, fast immediately hits None — slow stays at dummy, which is the predecessor of the old head. Uniform code path."

🗣️ What to Say to the Interviewer

"I'll use a dummy head and two pointers with a fixed gap.
Dummy.next = head. Both fast and slow start at dummy.
Advance fast n+1 times — that creates a gap of n+1 between them.
Then move both one step at a time until fast is None.
At that point slow is at the predecessor of the target — unlink with slow.next = slow.next.next.
Return dummy.next, not head, since head might have been the deleted node.
Time O(L), Space O(1), single pass, no edge-case branches."

7 Summary & Key Takeaways ▾
A gap of n+1 between fast and slow ensures slow lands at the predecessor of the nth-from-end node when fast hits None
Dummy head makes head-removal uniform — no separate branch needed; always return dummy.next
The gap invariant (n+1 throughout lockstep) is the mathematical core of this algorithm
Two-pass (count length, then walk L-n steps) is equally valid but visits the list twice
The gap pointer pattern generalises: vary the initial gap to position slow at any fixed offset from the end
ApproachTimeSpacePasses
Two-Pass (count + walk) O(L) O(1) 2
One-Pass Gap (Two Pointers) ✓ O(L) O(1) 1
🎯 When to Recognise the Two-Pointer Gap Pattern
  • "k-th node from the end of a linked list" → gap of k or k+1 between two pointers
  • "Find the middle node" → fast at 2×, slow at 1× speed (a gap that doubles)
  • "Detect/find cycle entry" → Floyd's algorithm, same fast/slow pointer family
  • Any problem where you need to measure distance from the tail without a second pass
