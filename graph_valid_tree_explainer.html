LeetCode #261

Graph Valid Tree

Given n nodes (0 to n-1) and a list of undirected edges, return true if these edges form a valid tree.

Medium Graph Union-Find Time: O(n · α(n)) Space: O(n)
1 Understanding the Problem ▾

A tree is a special undirected graph with two properties that must both hold simultaneously:

  1. Connected: Every node can reach every other node via some path.
  2. Acyclic: There are no cycles — no way to start at a node and return to it by following different edges.
💡 Key Insight: The n−1 Edge Formula

A connected undirected graph on n nodes is a tree if and only if it has exactly n − 1 edges. This single constraint captures both conditions: fewer edges means disconnected; more edges forces a cycle. So we only need to verify: len(edges) == n - 1 AND the graph contains no cycle.

✓ Valid Tree: n=5, edges=[[0,1],[0,2],[0,3],[1,4]]

4 edges == n−1 = 4 ✓
No cycle — star+chain
All 5 nodes reachable ✓
Result: True

✗ Not a Tree: n=5, edges=[[0,1],[1,2],[2,3],[1,3],[1,4]]

5 edges ≠ n−1 = 4 ✗
Cycle: 1→2→3→1
Too many edges
Result: False

Edge Cases: n=1 with no edges is a valid single-node tree. Duplicate edges create a cycle. Self-loops (edge from a node to itself) are always a cycle. A disconnected graph with exactly n−1 edges is impossible — the edge count check and no-cycle check together guarantee connectivity.

2 The Algorithm: Union-Find (Disjoint Set Union) ▾

Union-Find maintains a collection of disjoint sets, each representing a connected component. We process each edge by attempting to union its two endpoints. If they're already in the same set, adding this edge creates a cycle.

1 Quick-fail on edge count: If len(edges) != n - 1, return False immediately. A tree on n nodes must have exactly n−1 edges — this O(1) check eliminates both disconnected and over-connected graphs.
2 Initialize: Create parent[i] = i (each node is its own root, representing n isolated components) and rank[i] = 0 for union-by-rank optimization.
3 Find with path compression: find(x) returns the root of x's component. Path compression flattens the tree so future finds are faster — every node on the path points directly to the root.
4 Union and cycle check: For each edge (u, v), compute ru = find(u) and rv = find(v). If ru == rv — same root, same component — this edge creates a cycle: return False. Otherwise, merge the smaller component under the larger (union by rank).
5 Return True: After processing all n−1 edges without a cycle, all nodes are in one connected component — a valid spanning tree.
🔑 Algorithm Deep-Dive: Union-Find

Origin: Tarjan (1975). Solves the dynamic connectivity problem — does a path exist between two nodes as edges are added?

Core invariant: All nodes sharing the same root are in the same connected component. find(x) returns the canonical representative of x's component.

Path compression: After find(x) locates the root, every intermediate node is made to point directly to the root, flattening chains for future lookups.

Union by rank: Attach the root of the smaller-rank tree under the root of the larger-rank tree, keeping trees shallow and find() efficient.

Combined complexity: O(α(n)) amortized per operation, where α is the inverse Ackermann function — effectively constant (≤5 for n ≤ 10^600).

Recognize when: "Are these nodes in the same group?", "Detect cycle in undirected graph", "Number of connected components", "Merge groups dynamically".

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point during edge processing, each disjoint set in the Union-Find structure represents exactly one connected component of the partial graph. When we call union(u, v), we merge two components. If they're already the same component (same root), a cycle would form — we detect and reject it immediately.

Why n−1 edges guarantees full connectivity (no disconnection): Start with n isolated nodes = n components. Every successful union(u, v) reduces the component count by exactly 1 (merging two distinct components). After n−1 successful unions, component count = n − (n−1) = 1. One component = fully connected.

Why same root means cycle: find(u) == find(v) means there already exists a path u ↔ v through previously processed edges. Adding edge (u,v) creates a second path — completing a cycle. The Union-Find detects this in near O(1) without traversing the graph.

⚠️ Common Mistake: Checking Only One Condition

Only checking no-cycle: A disconnected graph can be acyclic but not a tree (e.g., n=3, edges=[[0,1]] has no cycle but is disconnected). Only checking n−1 edges: [[0,1],[1,2],[2,0]] has n−1=2 edges for n=3 but wait — actually 3 edges for n=3, so it fails. But duplicate-edge tricks can fool a naive edge count. Use BOTH checks together.

Proof that n−1 + no-cycle implies connected: By induction. A single node (n=1, 0 edges) is trivially connected. For n>1 with n−1 edges and no cycle, consider any node v. The component containing v must connect to remaining nodes via the n−1 edges without forming cycles — this forces a tree structure connecting all n nodes.

4 The Code (Annotated) ▾

Solution 1 — Union-Find (Optimal, Interview Pick)

def validTree(n: int, edges: list) -> bool:
if len(edges) != n - 1: ← n-1 edge check: O(1) early exit
return False
parent = list(range(n)) ← parent[i]=i: each node is its own root
rank = [0] * n ← rank[i]=0: used for union-by-rank
def find(x): ← Find root with path compression
if parent[x] != x: ← x is not its own root
parent[x] = find(parent[x]) ← Compress: point x directly to root
return parent[x]
def union(x, y) -> bool: ← Returns False if cycle detected
rx, ry = find(x), find(y) ← Find roots of both nodes
if rx == ry: ← Same root = same component = CYCLE!
return False
if rank[rx] < rank[ry]: ← Attach smaller tree under larger
parent[rx] = ry
elif rank[rx] > rank[ry]:
parent[ry] = rx
else: ← Equal rank: pick one, bump its rank
parent[ry] = rx
rank[rx] += 1
return True
return all(union(u, v) for u, v in edges) ← Process all; any False = cycle
⚠️ Why not just return all(union(u,v) for u,v in edges)?

Without the len(edges) != n-1 check first, a disconnected graph with fewer than n−1 edges would pass the union loop (no cycle detected!) and incorrectly return True. The edge count check is essential for correctness, not just optimization.

Solution 2 — BFS (Conceptually transparent)

from collections import defaultdict, deque
def validTree(n: int, edges: list) -> bool:
if len(edges) != n - 1: return False ← Same edge count guard
adj = defaultdict(list) ← Build undirected adjacency list
for u, v in edges: ← Both directions for undirected graph
adj[u].append(v); adj[v].append(u)
visited = {0} ← BFS from node 0
queue = deque([0])
while queue:
node = queue.popleft()
for nei in adj[node]:
if nei not in visited: ← Only visit unvisited neighbors
visited.add(nei)
queue.append(nei)
return len(visited) == n ← All n nodes reached = connected
💡 Complexity Summary

Union-Find: Time O(n · α(n)) ≈ O(n), Space O(n)
BFS/DFS: Time O(n + e) = O(n) since e = n−1, Space O(n) for adjacency list
Union-Find wins on elegance, generalizability, and interview impressiveness. BFS wins on code readability for beginners.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: State the Tree Definition

"A valid tree requires two things: connected (all nodes reachable from any start) and acyclic (no cycles). I'll verify both."

Step 2: Derive the n−1 Edge Shortcut

"I know a tree on n nodes always has exactly n−1 edges. If the edge count doesn't match, return False immediately. This handles disconnected graphs (too few edges) and guaranteed-cycle graphs (too many) in O(1)."

Step 3: Propose Union-Find for Cycle Detection

"With the edge count verified, I still need to confirm no cycle. Union-Find is ideal: for each edge (u,v), if find(u) == find(v), we're adding an edge within an existing component — that's a cycle. Return False. Otherwise, union them. After all n−1 edges process without cycle, return True."

Step 4: Mention BFS Alternative if Asked

"Alternatively, BFS/DFS from node 0 counting visited nodes: if visited.size() == n after traversal, the graph is connected (and acyclic since we already ruled that out via edge count). Same O(n) complexity, more code."

🗣️ What to Say to the Interviewer

"A valid tree needs exactly n−1 edges and no cycles.
First, O(1) check: if len(edges) != n-1, return False.
Then Union-Find: initialize parent[i]=i for all i.
For each edge (u, v): find roots. If same root, cycle detected — False.
Otherwise union them. After all edges, return True.
Time: O(n · α(n)) ≈ O(n). Space: O(n). The α function is inverse Ackermann — practically O(1) per op."
🔮 Likely Follow-up Questions
  • "How would you find connected components?" → Count distinct roots after all unions
  • "What if edges arrive as a stream?" → Union-Find handles online updates perfectly, O(α(n)) per query vs BFS needing full graph
  • "Explain path compression" → When find(x) traverses to root, each ancestor is made to point directly to root, flattening chains
  • "What's inverse Ackermann / why near-constant?" → α(n) ≤ 4 for any input that could exist in the physical universe
