LeetCode #963

Minimum Area Rectangle II

Find the minimum area of any rectangle formed by 4 points — the rectangle need not be axis-aligned
🟡 Medium Math Center + Diagonal Length O(n²) Time · O(n²) Space
1
Understanding the Problem
▼

You are given n points in the 2D plane. A rectangle is any quadrilateral with 4 right angles — it does not have to be axis-aligned. Return the minimum area of any rectangle that can be formed using 4 of these points as its corners, or 0 if no rectangle can be formed.

🔑 Reframe: What Makes a Rectangle?

A quadrilateral ABCD is a rectangle if and only if its two diagonals (AC and BD) satisfy:

  • Same midpoint — diagonals of any parallelogram bisect each other
  • Same length — diagonals of a parallelogram are equal iff it is a rectangle

This gives us a powerful shortcut: instead of checking 4-point combinations for right angles, check 2-segment "diagonal" pairs for equal center and equal length.

Concrete Example

Points: (0,0), (2,0), (2,2), (0,2) — a 2×2 axis-aligned square.

  • Diagonal 1: (0,0)↔(2,2) → midpoint proxy=(2,2), sq_len=8
  • Diagonal 2: (2,0)↔(0,2) → midpoint proxy=(2,2), sq_len=8
  • Both diagonals share the same key ((2,2), 8) → rectangle confirmed!
  • Area: sides have length √2, so area = √2 × √2 × 2 = ... we just compute |p1p2| × |p1p3| directly.
💡 The "Aha!" Moment

We don't look for 4-corner combinations. Instead, we look for 2 segments that could be the two diagonals of a rectangle. This is the key reduction from O(n⁴) to O(n²).

⚠️ The Catch: Dot Product Check

Two segments sharing midpoint and length form a parallelogram — but we specifically need a rectangle. The perpendicularity of adjacent sides is verified with a dot product = 0 check. In practice this is always satisfied when the midpoint + length conditions hold for a true rectangle, but it catches degenerate inputs.

The Algorithm: Center + Diagonal Length Grouping
▼

The core insight: treat each pair of points as a potential rectangle diagonal. Group pairs by their "diagonal signature" — (midpoint, squared length). Any two pairs in the same group form a candidate rectangle.

1 Enumerate all C(n,2) pairs — O(n²) total. For each pair (i, j), compute a diagonal signature.
2 Compute signature — mid = (x₁+x₂, y₁+y₂) (proxy, avoids /2), sq = (x₂-x₁)²+(y₂-y₁)². Key = (mid, sq).
3 Group into hash map — diags[key].append((i, j)). Groups with ≥2 pairs contain potential rectangle diagonals.
4 Verify perpendicularity — For each pair-of-pairs in a group, take anchor p1, compute edge vectors p1→p2 and p1→p3. Dot product must equal 0.
5 Compute area and track minimum — Area = |p1→p2| × |p1→p3|. Update global minimum.
📐 Key Formulas
key = ((x₁+x₂, y₁+y₂), (x₂-x₁)²+(y₂-y₁)²)
dot = (p2.x-p1.x)·(p3.x-p1.x) + (p2.y-p1.y)·(p3.y-p1.y)
area = √(dx12²+dy12²) × √(dx13²+dy13²) if dot = 0

Why It Works: The Geometric Invariant
▼
🔐 The Invariant

In our algorithm, every group in the hash map corresponds to a set of segments that all share the same center (midpoint) and the same squared length. Any two such segments with 4 distinct endpoints form a parallelogram with equal diagonals — which is precisely a rectangle.

Formal Proof Sketch

Claim 1 — Same midpoint implies parallelogram. If segments AB and CD share midpoint M, then A+B = 2M = C+D as 2D vectors. This means A+B = C+D, which is the defining condition of a parallelogram ACBD (opposite vertices sum to the same point).

Claim 2 — Equal diagonal lengths in parallelogram implies rectangle. In a parallelogram with sides a, b and included angle θ, the diagonal lengths are d₁ = √(a²+b²+2ab·cosθ) and d₂ = √(a²+b²−2ab·cosθ). For d₁ = d₂, we need cosθ = 0, i.e., θ = 90°. A parallelogram with a 90° angle is a rectangle.

Why the dot product check? The dot product p1p2⃗·p1p3⃗ = 0 directly verifies perpendicularity of adjacent sides from anchor p1. This is a numerical confirmation of the geometric argument above, and guards against degenerate inputs.

💡 Why Use sq_len, Not len?

Storing the squared length keeps everything in integers, avoiding floating-point precision issues during hash grouping. We only need sqrt() when computing the final area — the very last step.

Edge Cases Handled

  • n < 4 — Cannot form a rectangle; return 0 (no group will have ≥2 pairs with 4 distinct points)
  • All collinear — All pairs will have the same direction, no group will yield dot product = 0
  • Duplicate points — Pairs (i,j) and (i,k) sharing endpoint i1 won't form a valid rectangle; dot product check catches this (degenerate vectors)
  • Axis-aligned rectangles — Handled automatically; the algorithm is fully general

Solution 1 — Optimal: Diagonal Hash Map (O(n²) · Interview Pick)

# Time: O(n²)  Space: O(n²)
def minAreaFreeRect(points):
    n = len(points)
    # --- Phase 1: Build diagonal signature map ---
    diags = {}                              # (midpoint_proxy, sq_len) → [(i,j)]
    for i in range(n):
        for j in range(i + 1, n):          # enumerate all pairs once
            x1, y1 = points[i]
            x2, y2 = points[j]
            mid = (x1 + x2, y1 + y2)      # proxy for midpoint; no /2 needed
            sq  = (x2-x1)**2 + (y2-y1)**2 # squared length; integers only
            key = (mid, sq)
            diags.setdefault(key, []).append((i, j))

    result = float('inf')

    # --- Phase 2: Find rectangles within each group ---
    for lst in diags.values():
        if len(lst) < 2:
            continue                        # need at least 2 diagonal candidates
        for a in range(len(lst)):
            for b in range(a + 1, len(lst)):
                i1, j1 = lst[a]            # Diagonal 1: points[i1] ↔ points[j1]
                i2, j2 = lst[b]            # Diagonal 2: points[i2] ↔ points[j2]

                p1 = points[i1]            # Anchor vertex
                p2 = points[j1]            # Opposite on diag 1
                p3 = points[i2]            # Adjacent (one end of diag 2)

                dx12 = p2[0] - p1[0]      # Edge vector p1→p2
                dy12 = p2[1] - p1[1]
                dx13 = p3[0] - p1[0]      # Edge vector p1→p3
                dy13 = p3[1] - p1[1]

                # Dot product = 0 → perpendicular → rectangle
                dot = abs(dx12 * dx13 + dy12 * dy13)
                if dot == 0:
                    d12 = (dx12**2 + dy12**2) ** 0.5
                    d13 = (dx13**2 + dy13**2) ** 0.5
                    result = min(result, d12 * d13)

    return result if result != float('inf') else 0
⚠️ Common Mistake: Using Midpoint Directly

Many implementations compute the true midpoint as ((x1+x2)/2, (y1+y2)/2). Dividing by 2 introduces floats, which can cause two identical midpoints to be treated as different keys due to floating-point representation. The proxy (x1+x2, y1+y2) stays in integers and is safe.

⚠️ Why p3 = points[i2], Not points[j2]?

We need the dot product of two adjacent sides from anchor p1. p2 is the diagonal partner of p1 (they're on diagonal 1). p3 is one end of diagonal 2 — one of the two adjacent corners. Either end works because in a rectangle all corners have 90° angles; p3 = points[j2] would also give dot = 0.

💡 Line-by-Line Summary
  • diags = {} — Initialize empty hash map
  • for i in range(n): for j in range(i+1, n): — Enumerate all C(n,2) pairs
  • mid = (x1+x2, y1+y2) — Integer midpoint proxy (avoids division)
  • sq = (x2-x1)²+(y2-y1)² — Squared diagonal length (avoids sqrt)
  • diags[key].append((i,j)) — Group pairs sharing same diagonal signature
  • dot == 0 — Perpendicularity check: this is a rectangle, not just a parallelogram
  • d12 * d13 — Area = product of adjacent side lengths (only sqrt here)

Solution 2 — Brute Force O(n⁴) (for comparison)

# Try every 4-point combination. O(n⁴) time. For small n or as a sanity check.
from itertools import combinations
def minAreaFreeRect_brute(points):
    def dist2(a,b): return (a[0]-b[0])**2+(a[1]-b[1])**2
    result = float('inf')
    for p1,p2,p3,p4 in combinations(points,4):
        # Try all 3 diagonal pairings of 4 points
        for a,b,c,d in [(p1,p3,p2,p4),(p1,p2,p3,p4),(p1,p4,p2,p3)]:
            if (a[0]+b[0],a[1]+b[1])==(c[0]+d[0],c[1]+d[1]) and dist2(a,b)==dist2(c,d):
                # Compute area via cross product magnitude
                dx1,dy1 = c[0]-a[0],c[1]-a[1]
                dx2,dy2 = d[0]-a[0],d[1]-a[1]
                dot = dx1*dx2+dy1*dy2
                if dot==0:
                    result=min(result,(dist2(a,c)*dist2(a,d))**0.5)
    return result if result!=float('inf') else 0

🎤 Interview Strategy: How to Arrive at This
▼
Step 1: State Brute Force First

"I can try every combination of 4 points — C(n,4) combinations — and check if they form a rectangle. That's O(n⁴), which is too slow. Let me think about what mathematical properties define a rectangle."

Step 2: Think About Diagonals

"A rectangle has two diagonals that bisect each other and are equal length. So instead of looking at 4 corners, I should look at 2 diagonal segments. This drops the search from 4-tuples to 2-tuples — O(n²) pairs instead of O(n⁴) quadruples."

Step 3: Design the Hash Key

"I'll group pairs by (midpoint, squared_length). I use the sum (x1+x2, y1+y2) as a midpoint proxy — same for equal midpoints, avoids division. Squared length avoids sqrt for grouping."

Step 4: Area Computation

"For any two pairs in the same group, verify dot product = 0 (right angle), then area = |side1| × |side2|. I only need sqrt at the final area step."

Step 5: State Complexity

"O(n²) time for building the map. Within each group, pairs-of-pairs add at most O(n²) more work overall. Space is O(n²) for the hash map. Total: O(n²) time, O(n²) space."

🗣️ What to Say to the Interviewer

"The key observation is that any rectangle's diagonals have two properties: they bisect each other (same midpoint) and they have equal length. So I'll enumerate all O(n²) point pairs, group them by (midpoint_proxy, squared_length). Any group with 2+ pairs contains potential diagonal candidates. For each pair of pairs in a group, I verify the sides are perpendicular via dot product, then compute area = |side1| × |side2|. This runs in O(n²) time and O(n²) space — much better than the O(n⁴) brute force of trying all 4-point combinations."
🎯 Follow-up Questions to Expect
  • "What if we want squares only?" — Additionally check that all sides are equal: |p1→p2| = |p1→p3| after the dot product check.
  • "How does this differ from Minimum Area Rectangle I?" — Problem I is axis-aligned, solvable with a point hash set in O(n²) time and O(n) space; much simpler.
  • "Can you reduce space below O(n²)?" — Difficult without losing the grouping advantage. Acknowledge the trade-off; the O(n²) space is an inherent cost of the diagonal grouping approach.
  • "What happens with duplicate points?" — Two pairs sharing an endpoint will have dx·dy products that don't satisfy the perpendicularity condition, so they're safely rejected.

Summary & Key Takeaways
▼
✓ Two diagonals of a rectangle: same midpoint AND same length — this is the key invariant
✓ Group O(n²) pairs by (midpoint proxy, squared_length) → avoids O(n⁴) brute force
✓ Midpoint proxy (x1+x2, y1+y2) stays in integers — prevents floating-point grouping bugs
✓ Perpendicularity dot product = 0 confirms rectangle (not just parallelogram)
✓ Area = |side1| × |side2|; only use sqrt at the very last step
✓ Return 0 if no rectangle found (result remains ∞)
💡 When to Recognize This Pattern

  • Brute force would be O(n⁴) — you need to reduce to O(n²)
  • Keywords: "rectangle", "any orientation", "points in plane"
