LeetCode #909

Snakes and Ladders

Find the minimum number of dice rolls to reach the last square of an n×n Snakes and Ladders board, or −1 if impossible.

Medium Graph BFS Shortest Path Time: O(n²) Space: O(n²)
1 Understanding the Problem ▾

You start at cell 1 of an n×n board and want to reach cell n². On each turn you roll a die (value 1–6) and advance that many cells. If the destination has a snake or ladder, you are instantly transported — at no extra roll cost. Find the minimum rolls to reach the last cell.

💡 Key Insight

This is a shortest-path problem on an unweighted implicit graph. Every cell is a node. Every dice roll is a unit-cost edge. Snakes and ladders are free "teleport" edges baked into the dice-roll transitions. Because all edges cost exactly 1 roll, BFS finds the minimum automatically — no Dijkstra or DP needed.

The boustrophedon numbering (hardest part): Cells are numbered in a zigzag from the bottom row upward. Row 1 (bottom) goes left→right, row 2 goes right→left, and so on, alternating. The board array is stored top-to-bottom (board[0] = top row). Converting a cell number to a (row, col) index in the array is where almost all bugs live.

Edge Cases: n=1 means you're already at the end (0 rolls if n²=1). Snakes can go forward and ladders can go backward (the problem doesn't restrict this). The start cell 1 itself is never teleported from — only cells landed on by dice. Return −1 if all reachable cells are exhausted without reaching n².

2 The Algorithm: BFS on Implicit Graph ▾

Model the board as a graph: node = cell number, edge = (cell, cell+die) after applying any teleport. BFS processes nodes in order of increasing number of rolls — so the first time we reach cell n², that roll count is minimal.

1 Initialise: Queue = [(1, 0)]. Visited = {1}. We start at cell 1 with 0 rolls made.
2 Dequeue front: (cell, rolls). Try all six dice outcomes: nxt = cell+1, cell+2, …, cell+6. Stop if nxt > n².
3 Convert nxt to (row, col): Use boustrophedon formula. r_from_bottom = (nxt−1)//n; c = (nxt−1)%n; if r_from_bottom is odd, c = n−1−c; array_row = n−1−r_from_bottom.
4 Apply teleport if present: if board[row][col] ≠ −1, override nxt with that value. One roll, instant teleport — no extra step.
5 Check goal: if nxt == n², return rolls+1. BFS guarantees this is optimal.
6 Enqueue unvisited cells: if nxt not in visited, add nxt to visited and enqueue (nxt, rolls+1). Mark visited at enqueue time — not dequeue.
7 Return −1 if the queue empties without reaching n². The board is disconnected or blocked.

🔑 Boustrophedon Conversion (Derive Once, Use Always)
def cell_to_rc(s, n): r = (s - 1) // n # 0-indexed row from bottom c = (s - 1) % n # col in that row (left-to-right) if r % 2 == 1: # odd rows from bottom go RIGHT-to-LEFT c = n - 1 - c # reverse column return n - 1 - r, c # convert to 0-indexed from top (array order)
3 Why It Works: The BFS Invariant ▾
🔐 The Invariant

At BFS level k, every cell currently in the queue was first discovered using exactly k dice rolls. When a cell is added to visited at level k, it cannot be reached in fewer than k rolls — otherwise it would have been visited at an earlier level. Therefore, the first time any cell appears in our BFS, we have the minimum roll count to reach it.

Why mark visited at enqueue (not dequeue)? Consider cells A and B, both reachable from cell X in 1 roll, and both able to reach cell C. If we mark C visited only at dequeue, C could be enqueued twice — once from A, once from B — wasting work. Marking visited at enqueue time prevents any cell from ever entering the queue more than once.

Why teleportation doesn't add a roll: A snake or ladder is not a decision — it's automatic. "Rolling die → landing on cell 3 → ladder takes you to 15" is one action: one roll. So we apply the teleport immediately after computing nxt, before enqueueing. The enqueued cell is already the post-teleport destination.

⚠️ The Classic Bug: Teleporting Again After a Teleport

Once you've applied the snake/ladder teleport to nxt, you must NOT apply another teleport to that new nxt. The game rules specify: roll once → land → if snake/ladder, take it (once). You do not chain teleports. The code applies if board[r][c] != -1: nxt = board[r][c] exactly once per dice roll.

💡 BFS vs Dijkstra for this problem

Dijkstra is the general shortest-path algorithm for weighted graphs. Here, every dice roll costs exactly 1 regardless of value. This uniform weight means BFS is provably optimal and simpler: O(n²) time vs O(n² log n) for Dijkstra. Whenever edge weights are uniform, always prefer BFS.

4 The Code (Annotated) ▾

Solution 1 — BFS (Optimal, O(n²) — Interview Pick)

from collections import deque
def snakesAndLadders(board: list[list[int]]) -> int:
n = len(board) ← board is n×n; cells numbered 1 to n²
def cell_to_rc(s): ← convert 1-indexed cell s → (row, col) in board array
r = (s - 1) // n ← 0-indexed row from bottom (0 = bottom row)
c = (s - 1) % n ← 0-indexed column within that row (left-to-right by default)
if r % 2 == 1: c = n - 1 - c ← odd rows from bottom go right-to-left — reverse the column
return n - 1 - r, c ← convert row-from-bottom to 0-indexed row-from-top (array order)
visited = {1} ← mark start as visited immediately; avoids re-enqueueing cell 1
q = deque([(1, 0)]) ← BFS queue: (cell_number, number_of_rolls_to_reach_it)
while q: ← process until queue empty (all reachable cells explored)
cell, rolls = q.popleft() ← FIFO dequeue — ensures level-by-level BFS ordering
for die in range(1, 7): ← try each dice value 1 through 6
nxt = cell + die ← candidate destination after this roll
if nxt > n * n: break ← can't go past last cell; break (larger die values also exceed — break not continue)
r, c = cell_to_rc(nxt) ← find board array position for this cell number
if board[r][c] != -1: ← snake or ladder present? (-1 means none)
nxt = board[r][c] ← teleport to snake/ladder destination (free, no extra roll)
if nxt == n * n: return rolls+1 ← reached the goal! BFS guarantees this is minimum rolls
if nxt not in visited: ← only process each cell once
visited.add(nxt) ← mark at enqueue time (prevents duplicate queue entries)
q.append((nxt, rolls + 1)) ← enqueue with roll count incremented by 1
return -1 ← queue exhausted; n² unreachable
⚠️ Why break instead of continue at nxt > n*n?

Dice values are tried in ascending order (1, 2, 3, …). Once cell + die > n*n, every larger die value will also exceed n*n. So break skips the remaining dice — correct and slightly more efficient. Using continue would still check die=4, 5, 6 after die=3 already fails — wasteful and easy to avoid.

Solution 2 — BFS with Pre-flattened Board (Cleaner for Interviews)

def snakesAndLadders(board): ← same BFS idea; pre-compute a flat[cell] = teleport_dest array
n = len(board)
flat = [-1] * (n * n + 1) ← flat[s] = board teleport value for cell s (-1 if none)
for s in range(1, n * n + 1): ← precompute for all cells once
r = (s-1) // n; c = (s-1) % n
if r % 2 == 1: c = n - 1 - c
flat[s] = board[n-1-r][c] ← O(1) teleport lookup during BFS
from collections import deque
visited = [False] * (n*n + 1); visited[1] = True
q = deque([(1, 0)])
while q:
cell, rolls = q.popleft()
for die in range(1, 7):
nxt = cell + die; if nxt > n*n: break
if flat[nxt] != -1: nxt = flat[nxt] ← flat lookup replaces cell_to_rc
if nxt == n*n: return rolls + 1
if not visited[nxt]:
visited[nxt] = True; q.append((nxt, rolls + 1))
return -1
💡 Complexity Summary

Both BFS solutions: Time O(n²) — each of the n² cells is enqueued and dequeued at most once. Space O(n²) — visited set plus queue.
Pre-flattened: Extra O(n²) space for flat[] but cleaner inner loop — often preferred in interviews for readability.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognise the Pattern

"Minimum number of moves to reach a goal — this signals shortest-path. Each dice roll costs 1 regardless of value, so uniform-weight graph. Uniform weight + shortest path = BFS, not Dijkstra."

Step 2: Define the Graph Explicitly

"Nodes are cell numbers 1..n². From node s, there are edges to s+1, s+2, …, s+6 (capped at n²). If a destination has a snake/ladder, the edge actually goes to the teleport target. This is an implicit graph — I don't build it explicitly, just generate neighbors on the fly."

Step 3: Solve the Coordinate Puzzle First

"Before BFS, I'll derive and unit-test cell_to_rc. Row from bottom = (s-1)//n. Col = (s-1)%n. Reverse col for odd rows from bottom. Convert row to array index. I'll verify: cell 1 → board[n-1][0], cell n → board[n-1][n-1], cell n+1 → board[n-2][n-1]."

Step 4: Code BFS, Double-check Visited Timing

"Standard BFS. Key invariant: mark cells visited when enqueuing, not dequeuing. Apply teleport before checking goal. Return rolls+1 at goal (not rolls)."

🗣️ What to Say to the Interviewer

"Minimum rolls to reach the end — that's shortest path with unit edge weights — BFS.
Each cell is a node; die rolls 1..6 create edges to up to 6 neighbours.
Landing on a snake or ladder teleports you — it's built into the edge destination.
Before coding BFS, let me derive the boustrophedon coordinate conversion:
r_from_bottom = (s-1) // n; c = (s-1) % n
if r_from_bottom is odd: c = n-1-c (right-to-left rows)
array_row = n-1-r_from_bottom
BFS from cell 1: dequeue, try 6 dice, convert, apply teleport, check n², enqueue unvisited.
Mark visited at enqueue. Time O(n²), Space O(n²)."
🔮 Likely Follow-up Questions
  • "Print the actual path?" → Track parent pointers: parent[nxt] = cell during BFS; reconstruct by walking backwards
  • "Dice rolls 1..k instead of 1..6?" → Change range(1,7) to range(1, k+1) — same algorithm
  • "Weighted moves (cost per roll value)?" → Use Dijkstra with a min-heap instead of BFS deque
  • "What if you can choose NOT to take a ladder?" → Add a branch: enqueue both nxt (no teleport) and board[r][c] (teleport)

7 Summary & Key Takeaways ▾
Model as implicit graph: cells = nodes, dice rolls = unit-weight edges, teleports applied at edge destination
BFS gives minimum rolls because all edges have the same cost (1 roll)
Mark cells visited at enqueue time to prevent duplicate queue entries and cycles
Boustrophedon conversion is the hardest part — derive and verify it independently
Apply snake/ladder teleport before enqueueing; it does not cost an extra roll
Return −1 when queue empties — some boards are impossible
ApproachTimeSpaceNotes
BFS (set-based visited) ✓ O(n²) O(n²) Interview pick; provably optimal
BFS (pre-flattened board) O(n²) O(n²) Cleaner code; extra flat[] array
DFS + Memoization O(n²) O(n²) Correct but harder; BFS simpler for shortest path
🎯 When to Recognise BFS Shortest Path
  • "Minimum steps / moves / rolls?" + all moves cost 1 → BFS
  • Finite state space (cells, positions, board configs) → BFS terminates
  • Possible cycles in the state graph → Need visited set!
  • Teleports, portals, instant transitions → Apply them at edge time, not as separate steps
