LeetCode #498

Diagonal Traverse

Given an m × n matrix, return all elements in diagonal order — alternating up-right and down-left through each anti-diagonal.

Medium Matrix Diagonal Groups Alternate Direction Time: O(m·n) Space: O(min(m,n))
1 Understanding the Problem ▾

Given a matrix, return all its elements in a "zigzag diagonal" order. The traversal visits every anti-diagonal (elements where row + col is constant), but alternates direction on each one: first diagonal goes up-right, second down-left, third up-right, and so on.

💡 Key Insight

Every cell (r, c) belongs to exactly one anti-diagonal, identified by r + c. This sum ranges from 0 (the top-left corner) to m+n-2 (the bottom-right corner), giving exactly m + n - 1 diagonals. Even-indexed diagonals go up-right; odd-indexed go down-left.

Why is this Medium? The diagonal grouping concept is elegant, but correctly clamping row/column bounds for non-square (rectangular) matrices is where most attempts fail. Getting r_start = max(0, d-(n-1)) exactly right takes careful reasoning.

2 The Algorithm: Diagonal Groups + Alternate Direction ▾

We iterate over all m + n - 1 anti-diagonals by index d. For each, we collect cells in down-left order (r increases, c decreases), then reverse if d is even to get the up-right output direction. This avoids complex mid-step direction switching.

1 Loop over diagonals: d ranges from 0 to m+n-2 inclusive. Each value of d is an anti-diagonal index (d = r+c for all cells on that diagonal).
2 Clamp starting row: r_start = max(0, d - (n-1)). We need c = d - r to stay ≤ n-1, so r ≥ d-(n-1). We also can't go above row 0. Similarly, r_end = min(d, m-1) ensures r ≤ m-1 and c = d-r ≥ 0.
3 Collect the diagonal: Walk r from r_start to r_end, reading mat[r][d-r] at each step. This traverses the diagonal in top-to-bottom (down-left) order.
4 Apply direction: If d is even, reverse the collected list (output should go up-right = bottom-to-top). If d is odd, use as-is (down-left is already the correct output direction). Append to result.
🔑 Alternative: Pure Simulation

Start at (0,0), track a direction flag going_up. Each step, move diagonally. On hitting a boundary, make a turn (up→right or left→down depending on which wall). This uses truly O(1) extra space but requires handling 4 boundary types with correct priority order when hitting corners — more error-prone to code under pressure.

⚠️ The Rectangular Matrix Trap

For a 2×4 matrix (2 rows, 4 cols), diagonal d=3 starts at r=max(0,3-3)=0, not r=d=3 (which doesn't exist). Without the max(0, d-(n-1)) clamp, your starting row is wrong for diagonals beyond the first n rows. Always sketch a 2×4 and 4×2 example to verify your bounds.

3 Why It Works: The Invariant ▾
🔐 The Invariant

All cells with the same value of r + c lie on the same anti-diagonal. Within diagonal d, moving from (r, c) to (r+1, c-1) preserves the sum r+c=d. So collecting with r increasing always gives us cells in down-left order. Reversing for even d gives us up-right order. The parity of d is the only state we need.

Why m+n-1 diagonals? The minimum sum is 0+0=0 (top-left corner), the maximum is (m-1)+(n-1) = m+n-2 (bottom-right). From 0 to m+n-2 inclusive is exactly m+n-1 values.

Why collect down-left and reverse for up-right? Down-left collection (r_start to r_end, increasing r) is mechanically simple and uniform — it doesn't depend on the direction. The reversal for even diagonals is a single list.reverse() call. This separates "collection logic" from "direction logic" cleanly.

Proof every cell appears exactly once: Each cell (r,c) has a unique (r,c) pair, and exactly one value d = r+c. It is collected exactly once during the loop iteration for diagonal d. The while loop is bounded by the valid row range for that diagonal, and those ranges partition all m×n cells.

⚠️ Common Mistake: Off-by-one on r_end

The loop condition r <= min(d, m-1) can become r < min(d+1, m) equivalently. But writing r < m and c >= 0 as the condition is fragile — it works but is harder to prove correct. The formula min(d, m-1) makes the bound explicit.

4 The Code (Annotated) ▾

Solution 1 — Diagonal Groups with Reverse (Interview Pick)

def findDiagonalOrder(mat):
m, n = len(mat), len(mat[0]) ← Dimensions: m rows, n columns
result = [] ← Output array
for d in range(m + n - 1): ← d is the diagonal index (= r+c for each cell)
diag = [] ← Collect this diagonal's cells
r = max(0, d - (n - 1)) ← Starting row: clamp so c=d-r stays ≤ n-1
while r <= min(d, m - 1): ← End when r exceeds bottom row or when c goes negative
diag.append(mat[r][d - r]) ← c = d - r (since r+c=d)
r += 1 ← Walk down-left: r↑, c↓
if d % 2 == 0: ← Even diagonal: output goes up-right
diag.reverse() ← Flip bottom-to-top for up-right output
result.extend(diag) ← Append all elements of this diagonal to result
return result ← All m×n elements in correct diagonal order
⚠️ Why max(0, d - (n-1)) for the starting row?

Column c = d − r must satisfy 0 ≤ c ≤ n-1. The constraint c ≤ n-1 means r ≥ d-(n-1). But r can't be negative either. So r_start = max(0, d-(n-1)). When d < n, r_start = 0 (we start from the top row). When d ≥ n, r_start = d-(n-1) (we start from a lower row to keep c in bounds).

Solution 2 — Pure Simulation (O(1) extra space)

def findDiagonalOrder(mat):
m, n = len(mat), len(mat[0])
r, c, going_up = 0, 0, True ← Start top-left, first diagonal goes up-right
result = []
for _ in range(m * n): ← Visit every cell exactly once
result.append(mat[r][c])
if going_up: ← Moving ↗ (r-1, c+1)
if c == n - 1: ← Hit right wall: slide down, flip
r += 1; going_up = False
elif r == 0: ← Hit top wall: slide right, flip
c += 1; going_up = False
else: r -= 1; c += 1 ← Normal diagonal step up-right
else: ← Moving ↙ (r+1, c-1)
if r == m - 1: ← Hit bottom wall: slide right, flip
c += 1; going_up = True
elif c == 0: ← Hit left wall: slide down, flip
r += 1; going_up = True
else: r += 1; c -= 1 ← Normal diagonal step down-left
return result
⚠️ Wall Priority in Simulation (Corner Trap!)

When going up and hitting the top-right corner (r=0 AND c=n-1), which boundary do we apply? Right wall takes priority over top wall when going up. Similarly for going down at corners. The if-elif ordering above encodes this: right wall (c==n-1) is checked before top wall (r==0) in the going_up branch.

💡 Complexity Summary

Diagonal Groups: Time O(m·n) — each cell visited once. Space O(min(m,n)) — the diag buffer holds at most min(m,n) elements (the longest anti-diagonal).
Simulation: Time O(m·n), Space O(1) extra — no temporary buffer needed beyond the output array.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Notice the r+c Pattern

Look at the first few outputs: 1 (0,0), 2 (0,1), 4 (1,0). Observe: for 2 and 4, r+c=1 for both. For 7 (2,0), 5 (1,1), 3 (0,2): r+c=2 for all three. This is the key observation — same anti-diagonal = same r+c value.

Step 2: Count and Name the Diagonals

r+c ranges from 0 (top-left) to (m-1)+(n-1) = m+n-2 (bottom-right). That's m+n-1 diagonals. Tell the interviewer: "I'll iterate over m+n-1 diagonals by index d = r+c."

Step 3: Work Out the Direction Rule

Even diagonals go up-right in the output; odd go down-left. Collect all cells in down-left order (easiest to bound-check). Even d: reverse. Odd d: as-is. Say: "I collect down-left, then conditionally reverse — that's just one if-statement."

Step 4: Derive the Bounds Carefully

For diagonal d: c = d-r must be in [0, n-1]. From c ≤ n-1 → r ≥ d-(n-1) → r_start = max(0, d-(n-1)). From c ≥ 0 → r ≤ d → r_end = min(d, m-1). Verify with a 2×4 example at the board.

🗣️ What to Say to the Interviewer

"I notice that all cells with the same r+c value form one anti-diagonal. r+c ranges from 0 to m+n-2,
so there are m+n-1 diagonals total. I'll loop d from 0 to m+n-2.
For each d, I collect cells from r=max(0,d-(n-1)) to r=min(d,m-1), reading mat[r][d-r].
That collects in down-left order. Even diagonals should go up-right, so I reverse those.
Time O(m·n) since every cell is visited once. Space O(min(m,n)) for the temporary diagonal buffer."

7 Summary & Key Takeaways ▾
r + c is constant along every anti-diagonal — this is the grouping key
There are exactly m + n - 1 anti-diagonals, indexed d = 0 to m+n-2
Even diagonals go up-right → collect down-left, then reverse
Bounds: r_start = max(0, d-(n-1)), r_end = min(d, m-1) — handles all rectangle shapes
Simulation approach is O(1) extra space but has trickier corner-case logic
