Ransom Note

LeetCode #383 – Can you build the ransom note from magazine letters?

Easy Hash Table Frequency Subtraction O(n+m) Time O(1) Space
1 Understanding the Problem ▼

Imagine you're a kidnapper (for educational purposes only! 😄) writing a ransom note by cutting letters from a magazine. You need to check: do you have enough of each letter?

Given:

  • ransomNote — the message you want to create
  • magazine — the source of available letters

Return: true if you can construct the ransom note, false otherwise.

✅ Can Build
Note: "aa"
Magazine: "aab"
✓ Has 2× 'a'
❌ Cannot Build
Note: "aa"
Magazine: "ab"
✗ Only 1× 'a'
💡 Key Insight

This is an inventory problem: count what you have (magazine), then check if you have enough to fulfill the order (ransom note). Each letter is "consumed" once used.

2 The Algorithm: Frequency Subtraction ▼

We use the Frequency Subtraction technique: build an inventory, then subtract as we consume.

1 Build inventory: Count every character in magazine → hash map or array[26]
2 Process ransom note: For each character needed, decrement its count
3 Check stock: If any count goes negative → return false (not enough)
4 Success: If we process all characters without running out → return true
🎯 Why "Subtraction" Works

Instead of building two maps and comparing, we use one map: add from magazine, subtract for ransom note. If we ever go negative, we're trying to use a letter we don't have!

3 Why It Works: The Invariant ▼
🔐 The Invariant

At any point, count[c] represents how many more times we can use character c. If it goes negative, we've overdrawn our account.

Proof of Correctness:

  1. We start with exact counts from magazine (inventory is accurate)
  2. Each character in ransom note decrements its count by 1
  3. If count[c] ≥ 0 after all decrements → we had enough of character c
  4. If any count[c] < 0 → we needed more c's than magazine provides
⚠️ Edge Cases
  • Empty ransom note: Always true (need nothing)
  • Empty magazine: False unless ransom note is also empty
  • Ransom note longer than magazine: Might still work if letters overlap enough
4 The Code (Annotated) ▼
# Frequency Subtraction approach
def canConstruct(ransomNote, magazine):
# Build inventory from magazine
count = {}
for c in magazine:
count[c] = count.get(c, 0) + 1
# Try to construct ransom note
for c in ransomNote:
if count.get(c, 0) == 0:
return False # Out of stock!
count[c] -= 1
return True # Successfully built

Alternative: Using Counter

from collections import Counter
def canConstruct(ransomNote, magazine):
return not (Counter(ransomNote) - Counter(magazine))
💡 Counter Subtraction Magic

Counter(a) - Counter(b) keeps only positive counts. If result is empty, all characters in a were covered by b!

Complexity:

  • Time: O(n + m) — One pass through each string
  • Space: O(1) — At most 26 lowercase letters
5 🎤 Interview Strategy ▼
Step 1: Recognize the Pattern

Keywords: "construct from", "each used once", "enough letters" → Frequency counting problem. Same family as Valid Anagram, but asymmetric (ransom ≤ magazine).

Step 2: Choose the Right Data Structure

Hash map for O(1) lookups. Mention array[26] alternative for lowercase-only constraint (slightly faster in practice).

Step 3: Decide Count Direction

Count magazine (the source), then subtract for ransom note. Alternatively: count both and compare, but subtraction is cleaner.

🗣️ What to Say to the Interviewer

"This is a frequency counting problem. I'll build a character count from the magazine — that's my inventory. Then I'll iterate through the ransom note, decrementing counts as I 'use' each letter. If I ever try to use a letter with count 0, I return false. Otherwise, true. This gives O(n+m) time and O(1) space since we have at most 26 letters."
🎯 Follow-up Questions
  • What if case-insensitive? — Convert both to lowercase first
  • What if Unicode? — Hash map still works, just more keys
  • Return which chars are missing? — Track negative counts

7 Summary & Key Takeaways ▼
✓ Frequency Subtraction: count source, decrement for each use
✓ Early exit when count hits 0 (out of stock)
✓ O(1) space with bounded alphabet (26 letters)
✓ Can use Counter subtraction for one-liner solution
🎯 When to Use This Pattern
  • Checking if one sequence can be built from another
  • "Each element used at most once" constraints
  • Inventory/resource allocation problems
  • Asymmetric frequency comparison (A ⊆ B, not A == B)
