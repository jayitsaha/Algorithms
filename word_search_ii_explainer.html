LeetCode #212

Word Search II

Given an m×n board of characters and a list of strings words, return all words in the list that can be found in the board by moving through adjacent cells (no cell reused).

Hard Tries DFS Backtracking Time: O(M·N·4·3^L) Space: O(W·L)
1 Understanding the Problem ▾

You have a grid of letters (the "board") and a list of target words. A word is "findable" if you can trace its letters on the board by moving through adjacent cells (up, down, left, right — no diagonals) without reusing any cell within a single word's path.

💡 Key Insight

Word Search II is Word Search I with many words at once. Running a separate DFS for every word is too slow. The fix: build a Trie from all words, then do one DFS sweep of the board. At each step, the Trie tells you instantly whether the current path prefix could lead to any target word — if not, prune immediately.

Edge cases: duplicate words in input (return unique), the same board path spelling a word multiple times (report once), empty words list, a 1×1 board, words longer than M×N (impossible to find), and words sharing long prefixes (where Trie shines).

🔑 Why This Is "Hard"

The difficulty is not any single concept — it's composing them correctly: build Trie from words → DFS from every board cell → use Trie node as state → mark cells visited in-place → restore after backtracking → prune found words from Trie → prevent duplicate results. Each piece is simple; the composition is the challenge.

2 The Algorithm: Trie + DFS Backtracking ▾

Two phases: Build the Trie from the word list, then Search via DFS from every board cell using the Trie as a guide.

1 Build the Trie: Insert every word into a Trie. Each TrieNode has a children dict (letter → TrieNode) and a word field (stores the complete word at terminal nodes, None elsewhere). Building takes O(W × L) time.
2 DFS from every cell: For each cell (r, c), call dfs(r, c, root). The Trie root acts as the starting state — no prefix built yet. If the board letter is not a child of the current node, return immediately (no word starts with this prefix from this path).
3 Descend the Trie in sync with the board: At each cell, we move from the current TrieNode to node.children[board[r][c]]. The Trie node and the board path always stay synchronized — one level of Trie per one letter of the board path.
4 Record words: When the current TrieNode's word is not None, we found a complete word. Append it to results, then set node.word = None to prevent adding duplicates if the same word is reachable via another board path.
5 Backtrack: Before exploring the 4 neighbors, set board[r][c] = '#' to mark visited. After all neighbors are explored, restore board[r][c] to its original letter. This is the core backtracking invariant.
6 Trie pruning: After a word is found and its node becomes a leaf (no children), delete it from its parent. This aggressively shrinks the Trie so future DFS calls skip these dead branches entirely.

🔑 Why Trie Over Checking Each Word Separately?

A hash set of words only answers "is this a complete word?" A Trie answers "is this a valid prefix of any word?" When the DFS reaches "ZZ" on the board and no word starts with "ZZ", the Trie immediately returns — no further exploration. For words sharing prefixes like "EAT", "EACH", "EARTH", the Trie explores E→A exactly once and branches into all three simultaneously.

3 Why It Works: The Invariant ▾
🔐 The DFS Invariant

At any recursive call dfs(r, c, node), node represents the Trie state after processing all board cells in the current path from the starting cell to the parent of (r, c). If board[r][c] is not in node.children, the current path cannot lead to any target word — safe to prune. If it is in node.children, we've extended a valid prefix and must explore.

Completeness — we won't miss words: We start DFS from every board cell, trying all M×N starting positions. Within each DFS, we explore all 4 directions (within bounds, not revisiting). The Trie contains every word. Any valid board path spelling a word will eventually hit a terminal Trie node and be recorded.

Soundness — no false positives: A word is only added when node.word is not None, which only happens for words explicitly inserted during Trie construction. The board[r][c] = '#' guard prevents the same cell appearing twice in one path.

⚠️ The '#' Marking Trick — Why It's Safe

We overwrite the board in-place with '#' before recursing, then restore it after. This is safe because: (1) we always restore before returning from any DFS call, so the board is correct for all parallel searches from other starting cells; (2) '#' will never appear in a real word (not a letter), so the check ch == '#' at the start of DFS catches it immediately.

Why Trie pruning is correct: When we delete a Trie leaf after finding its word, we're saying "no more words live under this branch." Any future DFS that would have descended into this branch now prunes at the parent check (ch not in node.children). Since the word was already found and recorded, we lose nothing.

💡 Complexity

Trie build: O(W × L) where W = number of words, L = average word length
DFS search: O(M × N × 4 × 3L-1) — at each of M×N cells, DFS explores up to 4 × 3L-1 paths (4 first moves, then 3 choices since we can't go back)
Space: O(W × L) for Trie nodes + O(L) recursion stack depth per path

4 The Code (Annotated) ▾

Solution 1 — Trie + DFS Backtracking (Optimal, Interview Pick)

class TrieNode: ← Simple node with two fields
def __init__(self):
self.children = {} ← letter → child TrieNode
self.word = None ← stores word string at terminal; None otherwise
def findWords(board, words):
root = TrieNode()
for word in words: ← Build Trie: O(W × L)
node = root
for ch in word:
if ch not in node.children:
node.children[ch] = TrieNode() ← create node for new prefix
node = node.children[ch]
node.word = word ← mark terminal with full word (easy retrieval)
ROWS, COLS = len(board), len(board[0])
result = []
def dfs(r, c, node):
ch = board[r][c]
if ch == '#' or ch not in node.children:← visited OR not in Trie → prune
return
next_node = node.children[ch] ← descend one Trie level
if next_node.word: ← complete word found!
result.append(next_node.word)
next_node.word = None ← clear to prevent duplicates
board[r][c] = '#' ← mark visited in-place
for dr, dc in [(-1,0),(1,0),(0,-1),(0,1)]:← 4 adjacent directions
nr, nc = r + dr, c + dc
if 0 <= nr < ROWS and 0 <= nc < COLS:
dfs(nr, nc, next_node) ← recurse with updated Trie node
board[r][c] = ch ← RESTORE — critical backtrack step
if not next_node.children: ← if now a leaf, prune from Trie
del node.children[ch]
for r in range(ROWS): ← try every board cell as a start
for c in range(COLS):
dfs(r, c, root)
return result
⚠️ Most Common Mistake: Forgetting to Restore board[r][c]

If you set board[r][c] = '#' but don't restore it, DFS calls from other starting cells or via other paths will see '#' instead of the actual letter. They'll incorrectly skip valid paths. The restore line board[r][c] = ch must come after all recursive calls — no exceptions.

⚠️ Why Clear node.word After Finding?

If two different board paths can spell the same word (e.g., two 'O's both leading to 'OAT'), without next_node.word = None after the first find, the second path would add 'OAT' again. The problem says return unique words. Clearing the flag after the first find is the simplest deduplication.

Solution 2 — Brute Force: DFS Per Word (for reference)

def findWords_brute(board, words): ← O(W × M × N × 4 × 3^L): one DFS per word
ROWS, COLS = len(board), len(board[0])
def exist(word):
def dfs(r, c, i): ← i = current index in word
if i == len(word): return True
if r<0 or r>=ROWS or c<0 or c>=COLS: return False
if board[r][c] != word[i]: return False
tmp, board[r][c] = board[r][c], '#'
found = any(dfs(r+dr, c+dc, i+1) for dr,dc in [(-1,0),(1,0),(0,-1),(0,1)])
board[r][c] = tmp; return found
return any(dfs(r,c,0) for r in range(ROWS) for c in range(COLS))
return [w for w in words if exist(w)] ← no prefix sharing; W × search cost
💡 The Core Trade-off

Brute force: W separate DFS passes, each of cost O(M×N×4×3^L). Trie approach: one DFS pass of the same cost, but guided by the Trie so invalid prefixes are pruned immediately. The Trie approach wins most when: (1) W is large, (2) words share many prefixes, (3) the board has few letters matching word prefixes.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Extension

"This is Word Search I extended to multiple words. I know how to find one word with DFS + backtracking from every cell. The question is how to efficiently handle many words without W separate passes."

Step 2: Identify the Bottleneck

"The brute force runs the same DFS W times with no shared work. The expensive part is exploring board paths that turn out to lead nowhere. I need a way to reject invalid prefixes immediately, before exploring further."

Step 3: Propose the Trie

"A Trie encodes all words such that I can check prefix validity in O(1) per step. During DFS, instead of a word index I carry a Trie node. If the next board letter is not in the current node's children, I prune — no word could continue from here."

Step 4: Mention the Pruning Optimization

"Once a word is found, I clear its terminal flag to prevent duplicates. If its node becomes a leaf, I delete it from the Trie. This progressively shrinks the Trie, making later DFS calls faster. In the best case, the entire Trie empties after all words are found."

🗣️ What to Say to the Interviewer

"Step 1: Build a Trie from all words — TrieNode has children dict and word field (non-None at terminal).
Step 2: DFS from every board cell (r, c) with the Trie root as initial state.
Step 3: At each cell, if board[r][c] is not in current node's children, return (prefix pruned).
Step 4: If node.children[board[r][c]].word exists, record and clear it (dedup).
Step 5: Mark cell '#', recurse 4 neighbors, restore cell (backtracking invariant).
Step 6: If the child node is now childless, delete it (Trie pruning).
Time: O(M·N·4·3^(L-1)) search, O(W·L) build. Space: O(W·L) Trie."
🔮 Likely Follow-up Questions
  • "How does Trie pruning affect worst-case complexity?" → It doesn't change the worst case but dramatically improves average case when words share prefixes
  • "What if we allow diagonal moves?" → Add 4 diagonal directions to the direction list
  • "What if words can reuse cells?" → Remove the '#' guard
  • "How would you handle 10,000 words?" → Trie still scales; build cost is O(W·L) which is manageable
  • "Is there a way to avoid backtracking the Trie?" → The Trie is traversed top-down during DFS; no backtracking needed on the Trie itself (only on the board)

7 Summary & Key Takeaways ▾
Build a Trie from all words first — this is what enables searching for all words simultaneously
The Trie node descends in lockstep with the board path — one node per letter explored
board[r][c] = '#' before recursing, board[r][c] = ch after — the backtracking invariant, always
Set node.word = None after recording to prevent duplicate results across paths
Delete childless Trie nodes after finding words — progressively shrinks the search space
Words sharing prefixes (EAT, EACH, EARTH) are explored together in one DFS pass
