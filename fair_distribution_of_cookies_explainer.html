LeetCode #2305

Fair Distribution of Cookies

Distribute cookie bags among k children to minimize the maximum cookies any child receives.

Medium Dynamic Programming Backtracking Time: O(k^n) Space: O(n + k)
1 Understanding the Problem ▾

You have an array cookies where cookies[i] is the number of cookies in bag i, and an integer k — the number of children. Each bag must be given to exactly one child (you cannot split a bag). The unfairness of a distribution is the maximum cookies any one child receives. Return the minimum possible unfairness.

💡 Key Insight

We must try every possible assignment of bags to children. With n bags and k children, this is a partition problem: minimizing the maximum partition sum. The search space is k^n, but smart pruning collapses it dramatically. We use backtracking — assign each bag one by one, track totals, and prune whenever we already exceed the best known answer.

Example: cookies=[8,15,10,20,8], k=2

Child 0 gets: [15, 10, 8] → 33
Child 1 gets: [8, 20] → 28
Unfairness = max(33, 28) = 33

Bad split: child 0 gets [20, 15, 10]

Child 0 gets: [20, 15, 10] → 45
Child 1 gets: [8, 8] → 16
Unfairness = max(45, 16) = 45 (worse)

Constraints: 2 ≤ n ≤ 8, 2 ≤ k ≤ n. The small n (≤ 8) is a hint — exhaustive search with pruning is feasible. If n were 1000, we'd need a completely different strategy (approximation or DP over subsets).

🔑 Why Can't We Be Greedy?

Greedy ideas like "give the largest bag to the child with the fewest cookies" don't work — local optimal choices don't guarantee global optimal. Example: cookies=[1, 1, 6, 6], k=2. Greedy puts 6 with 1 and 6 with 1, giving max=7. But the optimal is [6,6] and [1,1], giving max=7 too — in this case equal. But more subtle examples break greedy. Backtracking with pruning is the right tool.

2 The Algorithm: Backtracking with Pruning ▾

Backtracking is systematic exhaustive search. We try to assign each bag to each child, recursively. When a partial assignment already exceeds our best answer, we prune that branch — skip the subtree entirely. This transforms an exponential problem into something tractable for small n.

1 Initialize: Create a bags[k] array of zeros tracking each child's current cookie total. Initialize ans = infinity.
2 Recurse on bag index i: For i = 0 to n-1, try assigning cookies[i] to each child j. Add cookies[i] to bags[j], recurse on i+1, then remove it (backtrack).
3 Prune early: Before recursing, check if bags[j] >= ans. If a child already holds as much or more than our best answer, adding more can't help — skip this branch.
4 Base case: When i == n (all bags assigned), compute max(bags) and update ans = min(ans, max(bags)).
5 Symmetry pruning: If two children have the same bag total, giving bag i to either produces the same future — skip duplicates to halve (or better) the search space.

💡 The Pruning Power

Without pruning: k^n = 2^3 = 8 leaf nodes. With symmetry pruning (skip duplicate bag totals): effectively k!/(k-used)! fewer branches. For k=3, n=8: from 6561 leaves down to hundreds. The key observation is children are interchangeable — only the multiset of bag totals matters, not which child holds which total.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point in the recursion, bags[j] holds the sum of bags assigned to child j so far. The recursion explores every valid complete assignment, and the prune condition (bags[j] >= ans) never discards a potentially optimal solution — because if bags[j] already equals or exceeds our best answer, no matter how we assign the remaining bags, the unfairness can only stay the same or get worse.

Why the prune is safe: We prune when bags[j] >= ans before adding cookies[i]. After adding, bags[j] would be >= ans + cookies[i] > ans. The max of any distribution containing this child would be at least bags[j] + cookies[i] > ans — provably suboptimal. We can skip the entire subtree without loss.

Why symmetry pruning is correct: Children are unlabeled — only the set of totals matters for unfairness. If two children j1 and j2 have the same bags[j1] == bags[j2] == t, giving bag i to j1 vs j2 produces the exact same future state (a different labeling of the same multiset). We only need to explore one of them.

⚠️ Common Mistake: Modifying bags Without Backtracking

Always undo: bags[j] += v → recurse → bags[j] -= v. If you forget the subtraction, subsequent iterations see corrupted state and the algorithm gives wrong answers. Backtracking requires exact restoration of state after each recursive call.

Edge cases: If k == n, each child gets exactly one bag — the answer is simply max(cookies). If all bags have the same value, any distribution is equally fair. Both cases fall out naturally from the backtracking without special handling.

4 The Code (Annotated) ▾

Solution 1 — Backtracking with Pruning (Interview Pick)

def distributeCookies(cookies, k):
bags = [0] * k ← One slot per child; track each child's cookie total
self.ans = float('inf') ← Best (minimum) unfairness found so far
def backtrack(i):
if i == len(cookies): ← Base case: all bags assigned
self.ans = min(self.ans, max(bags)) ← Record unfairness of this complete assignment
return
seen = set() ← Symmetry: skip children with same total (avoid duplicate work)
for j in range(k): ← Try giving bag i to each child j
if bags[j] in seen: continue ← Skip if another child already has same total (symmetry prune)
if bags[j] >= self.ans: continue ← Skip if already at/above best (value prune)
seen.add(bags[j]) ← Mark this total as explored
bags[j] += cookies[i] ← Assign bag i to child j
backtrack(i + 1) ← Recurse to assign next bag
bags[j] -= cookies[i] ← BACKTRACK: undo to try next child
backtrack(0) ← Start assigning from bag 0
return self.ans
💡 Complexity

Time: O(k^n) worst case — k choices per bag, n bags. With pruning, far fewer paths are explored in practice. For n=8, k=8: 8^8 = 16M, but pruning reduces this enormously.
Space: O(n + k) for recursion stack (depth n) and bags array (size k).

Solution 2 — Bitmask DP (Alternative, polynomial for small n)

For each possible subset of bags, precompute the sum. Then dp[j][mask] = minimum unfairness when assigning the bags in mask among j children. For the j-th child (1-indexed), try all submasks as that child's portion.

def distributeCookies(cookies, k):
n = len(cookies)
total = [0] * (1 << n) ← Precompute sum of cookies for every subset mask
for mask in range(1 << n):
for i in range(n):
if mask >> i & 1: ← If bit i is set, bag i is in this subset
total[mask] += cookies[i]
INF = float('inf')
dp = [[INF] * (1 << n) for _ in range(k+1)] ← dp[j][mask]: min unfairness distributing mask among j kids
dp[0][0] = 0 ← 0 children, 0 bags → unfairness 0
for j in range(1, k+1):
for mask in range(1 << n):
sub = mask ← Enumerate all submasks of mask
while sub:
if dp[j-1][mask ^ sub] < INF: ← Remaining bags (mask XOR sub) covered by j-1 kids
dp[j][mask] = min(dp[j][mask],
max(dp[j-1][mask ^ sub], total[sub])) ← Give sub to kid j; unfairness = max of both halves
sub = (sub - 1) & mask ← Next submask iteration trick
return dp[k][(1 << n) - 1] ← All bags (full mask) assigned to k children
Recurrence Relation (Bitmask DP)
dp[j][mask] = min over all submasks sub of mask:
  max( dp[j-1][mask XOR sub], total[sub] )

Give subset sub to child j, and optimally distribute mask\sub among j-1 children.
Unfairness is max of the two parts.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Note the Small Constraint

n ≤ 8 is the critical signal. When n is tiny, exhaustive search becomes feasible. Ask: "Can I try all possibilities?" With k^n at most 8^8 ≈ 16M but with heavy pruning much less, yes. This signals backtracking or bitmask DP.

Step 2: Frame as Partition Problem

Reframe: "partition n items into k groups to minimize the maximum group sum." This is a classic NP-hard problem in general, but feasible for small n. Say: "I'll use backtracking — assign each item to a group, prune branches that can't beat the current best."

Step 3: Articulate Both Pruning Rules

Say explicitly: "I'll apply two pruning optimizations. First, if a child's current total already matches or exceeds my best answer, I skip adding to that child. Second, if two children have the same total, giving the current bag to either produces identical futures — I explore only one."

Step 4: Mention the Bitmask DP Alternative

"There's also a bitmask DP approach: dp[j][mask] = min unfairness distributing bags in mask among j children. It's O(3^n × k) time, which is polynomial but has higher memory overhead. For this problem, backtracking with pruning is cleaner and equally fast in practice."

🗣️ What to Say to the Interviewer

"I see n ≤ 8 — that's the key. This looks like a partition problem: assign n bags to k children, minimize the max sum.
I'll use backtracking. I'll maintain a bags[k] array. For each cookie bag in order, I try giving it to each child.
Two pruning rules make this fast: (1) skip if bags[j] ≥ current best answer; (2) skip if bags[j] equals another child's total — symmetry.
Base case: when all bags are assigned, record max(bags) as a candidate answer.
Time is O(k^n) worst-case, but pruning makes it much better. Space is O(n+k) for the stack and bags array."
🔮 Likely Follow-up Questions
  • "What if n were 1000?" → Exact solution becomes NP-hard. Use approximation algorithms or heuristics (e.g., LPT greedy, simulated annealing).
  • "Can you solve it with bitmask DP?" → Yes! dp[j][mask] with submask enumeration. O(3^n × k) time, O(k × 2^n) space.
  • "What if bags can be split?" → Becomes a fractional partition problem — solvable greedily (sort descending, assign to least-loaded child).
  • "Explain the symmetry pruning more." → If bags[j1] == bags[j2], the entire subtree rooted at "give bag i to j1" is identical to "give bag i to j2" — just with j1 and j2 swapped. Since unfairness only looks at max(bags), this swap doesn't matter.

7 Summary & Key Takeaways ▾
Small n (≤ 8–15) signals exhaustive search — backtracking or bitmask DP are viable
Backtracking = try + recurse + undo. The "undo" step is critical and often forgotten
Value pruning: if a partial state already exceeds the best answer, skip the subtree
Symmetry pruning: skip duplicate states (same bag total means same subtree outcomes)
For bitmask DP: dp[j][mask] with submask enumeration runs in O(3^n × k)
The "minimize the maximum" objective is the hallmark of minimax / load-balancing problems

🎯 When to Recognize This Pattern
  • n is very small (≤ 15) — exhaustive search is implied
  • "Partition items into k groups" — backtracking with group totals
  • "Minimize the maximum" or "maximize the minimum" — minimax objective
  • Items cannot be split — rules out greedy, suggests combinatorial search
