LeetCode #2305

Fair Distribution of Cookies

Assign cookie bags to children — minimize the maximum cookies any child receives
Medium Dynamic Programming Backtracking or Bitmask DP O(k^n) Backtracking · O(2^n · k) Bitmask
1
Understanding the Problem
▼

You have n bags of cookies (each bag has some count of cookies) and k children. Every bag must be assigned to exactly one child — you cannot split a bag. Your goal is to find an assignment that minimizes the maximum cookies any single child receives.

This is sometimes called a min-max partition problem: we are partitioning n items into k groups and trying to make the heaviest group as light as possible.

💡 Key Insight

We do not need to find the optimal assignment analytically. Because n ≤ 8 and k ≤ 8, the entire search space is tiny enough for brute-force backtracking with smart pruning (or bitmask DP). At each step, we pick the next unassigned bag and try giving it to each child — then backtrack if no improvement is possible.

Worked example: cookies = [8, 15, 10, 20, 8], k = 2

One possible split: child 0 gets [8, 15, 8] = 31; child 1 gets [10, 20] = 30 → max = 31.

Another split: child 0 gets [15, 10, 8] = 33; child 1 gets [8, 20] = 28 → max = 33. Worse.

Optimal: child 0 gets [20, 10] = 30; child 1 gets [15, 8, 8] = 31 → max = 31.

📐 Why Brute Force Fits Here

Constraints: 2 ≤ cookies.length ≤ 8, 2 ≤ k ≤ cookies.length. So at most 8^8 = 16,777,216 assignments before pruning — completely feasible. Without these small constraints, we would need the Bitmask DP approach.

2
The Algorithm: Backtracking with Pruning
▼

The primary approach is recursive backtracking: we process bags one by one (left to right) and try assigning each to every child, tracking the best minimum-maximum found so far.

1
Initialize: Create an array children[k] all zeros (current cookie totals). Set ans = ∞.
2
Base case: When all bags are assigned (i == n), update ans = min(ans, max(children)).
3
Pruning (key!): Before assigning bag i to child j, check if children[j] + cookies[i] >= ans. If so, skip — this path can never beat our current best.
4
Symmetry pruning: Skip if another child already has the same total — assigning to either produces symmetric (duplicate) subtrees. Track seen totals per level.
5
Recurse and backtrack: Add bag to child, recurse to next bag, then remove bag (undo). This explores all valid assignments.
⚠️ Pruning is Essential

Without the children[j] + cookies[i] >= ans guard, the backtracking explores far more branches. In practice, with pruning, the effective branching factor is much smaller than k.

Alternative — Bitmask DP: Enumerate all 2^n subsets. For each child, compute the sum of each subset. Then use DP: dp[mask] = min unfairness when the cookies in mask have been distributed among the first few children optimally. This is O(2^n · n · k) time.

📐 Bitmask DP Recurrence
dp[0][0] = 0 (0 children, no cookies distributed)
dp[j][mask] = min over all submask S of mask:
                  max(dp[j-1][mask ^ S], sum(S))

Answer = dp[k][full_mask]
3
Why It Works: Correctness & Invariants
▼
🔐 The Backtracking Invariant

At every recursive call for bag index i, children[j] holds the exact total cookies child j has received from bags 0..i-1. When we try assigning bag i to child j, we temporarily update and restore — guaranteeing we explore all complete assignments without redundancy (modulo the symmetry pruning).

Completeness: Because every bag must be assigned to one of k children, and the recursion tries all k options for each bag, the algorithm is exhaustive over the partition space. No valid assignment is missed.

Soundness of pruning: If children[j] + cookies[i] >= ans, then even assigning the remaining bags optimally cannot reduce the maximum below ans (since child j alone already has ≥ ans). So cutting this branch is safe.

Symmetry pruning: Children are interchangeable — if children A and B both have total 0, assigning bag to A vs B produces identical sub-trees. Tracking a seen set of encountered totals at each recursive call avoids this duplication.

💡 Why the Problem Has Optimal Substructure

The min-max objective decomposes: once bags 0..i-1 are distributed, the optimal way to distribute bags i..n-1 depends only on the current totals vector, not on which specific bags produced those totals. This is why both backtracking and DP work correctly.

Edge cases: If k == n, each child gets exactly one bag — return max(cookies). If k == 1, one child gets everything — return sum(cookies). The algorithm handles both correctly.

The Code (Annotated)

Solution 1 — Backtracking with Pruning (Interview Pick)

def distributeCookies(cookies, k):
children = [0] * k ← current cookie totals for each child
ans = [float('inf')] ← best (minimum-maximum) found so far; list for mutation in closure
def backtrack(i):
if i == len(cookies): ← base case: all bags assigned
ans[0] = min(ans[0], max(children)) ← update best if this assignment is better
return
seen = set() ← symmetry pruning: skip duplicate child totals at this level
for j in range(k):
if children[j] in seen: ← another child already has same total: skip (symmetric branch)
continue
if children[j] + cookies[i] >= ans[0]: ← pruning: can't beat current best
continue
seen.add(children[j]) ← mark this total as seen for this level
children[j] += cookies[i] ← assign bag i to child j
backtrack(i + 1) ← recurse to next bag
children[j] -= cookies[i] ← backtrack: undo assignment
backtrack(0) ← start with first bag
return ans[0]
⚠️ Common Mistake: Forgetting to Backtrack

If you do children[j] += cookies[i] and forget children[j] -= cookies[i] after the recursive call, subsequent iterations will use a corrupted state. Backtracking MUST undo every change it makes.

Solution 2 — Bitmask DP

def distributeCookies(cookies, k):
n = len(cookies)
total = [0] * (1 << n) ← precompute sum of each subset
for mask in range(1 << n):
for i in range(n):
if mask >> i & 1: ← if bit i is set in mask, bag i is in this subset
total[mask] += cookies[i]
INF = float('inf')
dp = [[INF] * (1 << n) for _ in range(k + 1)]
dp[0][0] = 0 ← 0 children, 0 bags assigned → max = 0
for j in range(1, k + 1): ← process each child
for mask in range(1 << n): ← for each possible "assigned so far" mask
if dp[j-1][mask] == INF: continue
sub = mask
while True: ← enumerate all subsets of remaining bags
rest = ((1 << n) - 1) ^ mask ← bits not yet assigned
sub = rest
while sub:
new_mask = mask | sub
dp[j][new_mask] = min(dp[j][new_mask], max(dp[j-1][mask], total[sub]))
sub = (sub - 1) & rest ← classic subset enumeration trick
break
return dp[k][(1 << n) - 1] ← k children, all bags assigned
💡 Bitmask Subset Enumeration

The pattern sub = (sub - 1) & rest iterates through all non-empty subsets of rest in O(2^|rest|) time — a classic bitmask trick. Each subset represents the bags given to the next child.

Interview Strategy: How to Arrive at This
▼
Step 1: Recognize the Pattern

Key signals: small constraints (n ≤ 8), "distribute items into groups", "minimize the maximum". These point to either backtracking or bitmask DP. With n ≤ 8 explicitly, bitmask (2^8 = 256 states) is a strong signal.

Step 2: Start with Brute Force

State: "I'll try all k^n assignments." That's at most 8^8 ≈ 16M. With pruning, far fewer. Propose backtracking first — it's simpler to code and explain.

Step 3: Define Pruning Conditions

Two prunings: (1) Bound pruning — skip if current child's total + this bag already hits or exceeds best. (2) Symmetry pruning — if two children have the same total, trying the bag on either gives identical results; use a seen set to avoid.

Step 4: Mention Bitmask DP as Upgrade

If the interviewer asks for a more formal DP solution: encode "which bags have been distributed" as a bitmask. dp[j][mask] = minimum max-cookies when j children have been assigned subsets totaling mask.

What to Say to the Interviewer

"I notice n and k are both at most 8, which is very small.
This looks like a partition-into-k-groups problem where I want to minimize the max group sum.
My first instinct is backtracking: assign each bag to one of k children, track totals, and prune.
I'll add two prunings: if a child's total already hits our best answer, skip; and skip symmetric branches where two children have identical totals.
The complexity is O(k^n) worst case but much less with pruning.
If you'd prefer a DP formulation, I can also do bitmask DP: dp[j][mask] where mask encodes which bags have been distributed among j children."
⚠️ Common Follow-Up Questions
  • What if n were 30? (Answer: Bitmask DP is 2^30 — too large. Need heuristics or approximation.)
  • How does sorting cookies help? (Descending order explores expensive assignments first, enabling earlier pruning.)
  • Can this be solved greedily? (No — greedy assignment doesn't guarantee global minimum.)

Complexity Analysis
▼
ApproachTimeSpaceNotes
Backtracking + Pruning O(k^n) worst case O(n + k) Much faster in practice due to pruning
Bitmask DP O(3^n · k) O(2^n · k) 3^n from enumerating subsets of subsets

For n ≤ 8, k ≤ 8: backtracking is preferred for its simplicity and better constant factors. Bitmask DP has guaranteed worst-case and is better if asked for a DP formulation.

Summary & Related Problems
▼
  • Recognize min-max partition problems → try backtracking or bitmask DP when n ≤ 20
  • Bound pruning: if current assignment already exceeds best, cut the branch immediately
  • Symmetry pruning with a seen set dramatically reduces duplicate sub-trees
  • Bitmask DP: encode "which items are distributed" as bits; iterate over all subsets
🎯 When to Use This Pattern
  • Distribute n items into k groups (n ≤ ~20)
  • Minimize the maximum group sum (or maximize the minimum)
  • "Fair" or "balanced" distribution problems
  • Constraints are tiny (strong hint for bitmask or backtracking)
