LeetCode #1411

Number of Ways to Paint N × 3 Grid

You have a grid of n rows and 3 columns. Color each cell with one of 3 colors such that no two adjacent cells (sharing a side) have the same color. Return the number of valid colorings modulo 10⁹ + 7.

Hard Dynamic Programming Two-Color & Three-Color Patterns Time: O(n) Space: O(1)
1 Understanding the Problem ▾

You have a grid of n rows × 3 columns. Each cell must be painted with one of 3 colors (call them R, G, B) such that no two cells sharing a side have the same color. Return the total count of valid colorings, modulo 10⁹ + 7.

The adjacency constraint applies in two directions: horizontally (within the same row, left-right neighbors differ) and vertically (across rows, cells in the same column differ row-to-row). Diagonal neighbors may share a color.

💡 Key Insight

A valid row of 3 cells with 3 colors, where adjacent cells differ, has exactly 12 distinct color patterns. Of those 12, some use only 2 distinct colors (two-color patterns, e.g., R-G-R), and some use all 3 distinct colors (three-color patterns, e.g., R-G-B). These two types transition differently to valid next rows — and that's the only information we need per row!

2 The Algorithm: Two-State Row DP ▾

Instead of tracking which exact color pattern each row uses (which would require 12 states), we can abstract each row into just two states: whether it used 2 distinct colors or 3 distinct colors. The transition counts between these two types are fixed constants, independent of which specific colors are used.

1 Count valid patterns for row 1: There are 6 two-color patterns and 6 three-color patterns. Initialize t2 = 6, t3 = 6.
2 Derive transition rules (key math): By exhaustive enumeration (done once), a two-color row generates 3 compatible two-color rows and 2 compatible three-color rows for the next row. A three-color row generates 2 compatible two-color rows and 2 compatible three-color rows.
3 Iterate n−1 times: For each subsequent row, compute new totals: new_t2 = 3*t2 + 2*t3 and new_t3 = 2*t2 + 2*t3. Apply modulo 10⁹+7 to prevent overflow.
4 Return total: (t2 + t3) % MOD is the answer — the total valid colorings across all pattern types for all n rows.
🔑 Why Fixed Transitions? (The Core Insight)

Consider any two-color row, say R-G-R. The valid next rows must differ in each column: col 0 ≠ R, col 1 ≠ G, col 2 ≠ R, AND col 0 ≠ col 1, col 1 ≠ col 2. Enumerating gives exactly 3 two-color and 2 three-color options. By symmetry of the 3 colors, every two-color row has exactly the same counts. So we can collapse 6 individual counts into one aggregate t2. Same logic for three-color rows.

3 Why It Works: DP, Invariant & Correctness ▾

Why is this Dynamic Programming?

Optimal substructure: The number of valid colorings for an n-row grid depends only on the number of valid colorings for an (n−1)-row grid in each pattern class. Each row's valid transitions depend only on the current row's type (two-color or three-color), not on any of the rows before it.

Overlapping subproblems: A naive recursion would recompute the count for "how many valid 4-row grids end in a two-color row" many times from different branches. DP memoizes this as a single value t2[4].

🔐 The Invariant

After processing row i, t2 counts the total valid colorings of the first i rows where row i is any two-color pattern, and t3 counts the same for any three-color pattern. Together, t2 + t3 = total valid colorings of a grid with i rows.

Proving the transition constants (enumerate for R-G-R):

Given row pattern R-G-R, the next row must satisfy: cell[0] ≠ R, cell[1] ≠ G, cell[2] ≠ R, cell[0] ≠ cell[1], cell[1] ≠ cell[2]. Let's enumerate:

Next rows compatible with R-G-R:
G-R-G ← two-color (ABA)
G-B-G ← two-color (ABA)
B-R-B ← two-color (ABA)
G-R-B ← three-color (ABC)
B-R-G ← three-color (ABC)
Total: 3 two-color + 2 three-color (same for every two-color row by symmetry)
⚠️ Common Mistake: Thinking Transitions are 3+3

It's tempting to think a two-color row gives 3 two-color and 3 three-color options. But the third column of a two-color row (which equals column 0) severely constrains what column 2 of the next row can be. Work through the enumeration above carefully — the answer is 3 two-color and only 2 three-color.

Why can we use just t2 and t3? By symmetry of the 3 colors, every two-color pattern (all 6 of them) generates exactly the same distribution of next-row types. We never need to track which specific color was used — just whether 2 or 3 colors were used. This collapses 12 state variables into 2.

4 The Code (Annotated) ▾

Solution 1 — Two-State DP, Bottom-Up / Tabulation (Optimal, O(n) time, O(1) space)

def numOfWays(n: int) -> int:
MOD = 10**9 + 7 ← Required modulus to prevent overflow
t2, t3 = 6, 6 ← Base case: row 1 has 6 two-color + 6 three-color patterns
for _ in range(n - 1): ← Process each additional row (n-1 transitions)
t2, t3 = (3*t2 + 2*t3) % MOD, ← New two-color: 3 per old t2 + 2 per old t3
(2*t2 + 2*t3) % MOD ← New three-color: 2 per old t2 + 2 per old t3
return (t2 + t3) % MOD ← Total valid colorings for n rows
⚠️ The Simultaneous Update Trick

Notice t2, t3 = expr1, expr2 on one line. Python evaluates both right-hand sides using the OLD values of t2 and t3 before assigning. If you wrote t2 = ...; t3 = ... on two separate lines, the second line would use the newly updated t2 in computing t3 — giving a wrong answer. Always update both simultaneously.

Solution 2 — Memoization / Top-Down (same complexity, illustrates DP structure)

from functools import lru_cache
def numOfWays(n: int) -> int:
MOD = 10**9 + 7
@lru_cache(maxsize=None) ← Cache results; state is (row, type)
def dp(row, kind): ← row: which row; kind: 2=two-color, 3=three-color
if row == 1: ← Base: 6 patterns of each type for first row
return 6
if kind == 2: ← Two-color row at this level
return (3*dp(row-1, 2) + 2*dp(row-1, 3)) % MOD
else: ← Three-color row at this level
return (2*dp(row-1, 2) + 2*dp(row-1, 3)) % MOD
return (dp(n, 2) + dp(n, 3)) % MOD ← Sum both types for the final row
💡 Complexity Summary

Tabulation (optimal): Time O(n), Space O(1) — two variables, updated in-place.
Memoization: Time O(n), Space O(n) — cache stores 2 results per row: dp(row, 2) and dp(row, 3).
Both are correct. The tabulation version is interview-preferred due to constant space.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Count Valid Single-Row Patterns

Start by asking: how many valid colorings exist for a single 3-cell row? With 3 colors and no adjacent same color: cell[0] has 3 choices, cell[1] has 2, cell[2] has 2 → 3×2×2 = 12. Show you can enumerate them systematically: 6 use exactly 2 colors (ABA), 6 use all 3 colors (ABC).

Step 2: Identify the State (Key Reduction)

Ask: "Do I need to track exactly which colors were used, or just how many?" By the symmetry of 3 colors, every two-color row generates the same number of compatible next rows. So the type (2-color vs 3-color) is the sufficient state — not the specific colors. This reduces 12 states to 2.

Step 3: Work Out Transitions on Paper

Pick one two-color pattern (e.g., R-G-R) and enumerate all valid next rows. You get 3 two-color and 2 three-color options. Pick one three-color (e.g., R-G-B) — you get 2 two-color and 2 three-color. These are the transition constants.

Step 4: Write the Recurrence, Code it Up

Base case: t2 = t3 = 6. Each iteration: new_t2 = 3×t2 + 2×t3; new_t3 = 2×t2 + 2×t3. Run for n−1 iterations. Return (t2 + t3) % MOD. Verify with n=1 → 12, n=2 → 3×6+2×6 + 2×6+2×6 = 30+24 = 54. (Actual answer for n=2 is 54 ✓)

🗣️ What to Say to the Interviewer

"A 3-cell row with 3 colors gives 12 valid patterns — 6 using 2 colors, 6 using all 3.
By symmetry of colors, every two-color row generates the same count of valid next rows.
So I can track just two numbers: t2 and t3, the counts of valid grids ending in each type.
By enumeration, transitions are: new_t2 = 3*t2 + 2*t3; new_t3 = 2*t2 + 2*t3.
I run this for n-1 steps starting from t2=t3=6, taking mod 10^9+7 throughout.
Time O(n), Space O(1)."
🔮 Likely Follow-up Questions
  • "What if it's N × 4 grid?" → More pattern types needed; profile count grows but DP structure is identical
  • "What about K colors instead of 3?" → More complex counting, same DP idea
  • "Can you solve it with matrix exponentiation?" → Yes — encode transitions as 2×2 matrix, use fast exponentiation for O(log n)
  • "Why must we update t2 and t3 simultaneously?" → Because the recurrence uses old values of both; sequential update corrupts results

7 Summary & Key Takeaways ▾
A 3-cell row with 3 colors yields exactly 12 valid patterns: 6 two-color (ABA) and 6 three-color (ABC)
By color symmetry, all patterns of the same type are interchangeable — track counts t2, t3, not specific colors
Fixed transition constants: T2→T2: ×3, T2→T3: ×2, T3→T2: ×2, T3→T3: ×2
Always update t2 and t3 simultaneously (Python tuple assignment) to avoid using stale values
Space O(1) — only two variables needed regardless of n; modulo applied every step
Can extend to O(log n) with 2×2 matrix exponentiation for the transition matrix
ApproachTimeSpaceNotes
Naive Backtracking O(12ⁿ) O(n) TLE for large n; correct but exponential

Two-State DP (t2/t3) ✓ O(n) O(1) Optimal; uses color symmetry to collapse states
Matrix Exponentiation O(log n) O(1) Best for very large n; advanced technique
🎯 When to Recognize the Two-Color / Three-Color DP Pattern
  • Grid coloring with adjacency constraints and limited colors
  • The number of possible patterns per row is small and fixed
  • Symmetry between colors means you only care about the structure (how many distinct colors), not which specific colors
  • Answer depends only on the previous row's "type", not its full history
