LeetCode #3

Longest Substring Without Repeating Characters

Find the length of the longest contiguous substring that contains no duplicate characters.

Medium Sliding Window Hash Set Time: O(n) Space: O(k)
1 Understanding the Problem ▾

Given a string s, find the length of the longest substring that contains no repeated characters. A substring is a contiguous portion of the string — you cannot skip characters, unlike a subsequence.

💡 Key Insight

Think of a sliding window — a moving frame [left, right] over the string. We want the longest frame that contains all unique characters. Expand the right edge freely. The moment a duplicate slips in, shrink the left edge until the duplicate is evicted. At every valid state, record the window size as a candidate answer.

✓ Example: s = "abcabcbb"

"abc" → length 3 ✓ (best)
"bca" → length 3 ✓
"cab" → length 3 ✓
All 3-char windows — answer is 3

✗ Why not try all substrings?

O(n²) pairs × O(n) check
= O(n³) total work.
For n=10⁵: 10¹⁵ ops.
Completely infeasible.

Edge Cases to consider: empty string → return 0; single character → return 1; all-same like "aaaa" → return 1 (window never grows beyond 1); all-unique like "abcdef" → return n; strings with spaces, digits, mixed case (all valid characters).

2 The Algorithm: Variable Sliding Window + Hash Set ▾

We maintain a variable-size window [left, right] over the string. A hash set mirrors which characters are currently inside the window, giving us O(1) membership tests. When right encounters a character already in the set, we advance left — removing the leftmost character each step — until the duplicate is gone. Then we safely add the new character and record the window length.

1 Initialize: left = 0, max_len = 0, char_set = set(). The window starts empty — right hasn't moved yet.
2 Expand right: For each position right in range(len(s)), check if s[right] is already in char_set.
3 Shrink on duplicate: While s[right] is in char_set, remove s[left] from the set and advance left += 1. Repeat until the duplicate is gone.
4 Add and record: Add s[right] to char_set (now safe — no duplicate). Update max_len = max(max_len, right - left + 1).
5 Return max_len after the full scan completes.
🔑 Why a Hash Set (not just a counter)?

We only need to answer "is this character currently in the window?" — a yes/no question. A set gives O(1) membership check and O(1) removal. A hash map (char → most recent index) allows an even faster "jump" variant of shrinking — but the set version is the cleanest to reason about and code correctly in an interview.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every moment after the while loop exits and before the next iteration begins, the substring s[left..right] contains only unique characters. The hash set char_set is always in perfect sync with this window — every character in the window is in the set, and nothing else is.

Why shrinking from the left is always sufficient: When s[right] is a duplicate, the offending earlier copy is somewhere inside the current window at index k where left ≤ k < right. Since the window already has all-unique characters (by the invariant), there is exactly one copy of this character inside. By removing characters from the left one at a time, we will eventually reach and remove s[k], restoring uniqueness. At that point the while loop exits.

Why we never miss the optimal window: The window only shrinks when forced (a duplicate appeared — and only shrinks the minimum needed). Every other step it expands. This means every position of right gets paired with the leftmost possible left — i.e., the longest possible valid window ending at right. We observe and compare every such window against max_len.

O(n) despite the nested while loop: Each character index is added via right exactly once and removed via left at most once. Total set operations: at most 2n. Total time: O(n).

⚠️ Common Mistake: Updating max_len Inside the While Loop

During the while loop, the window is temporarily invalid (it still contains the duplicate). Recording max_len mid-shrink gives wrong answers. Always record max_len after the while loop exits and after adding s[right] to the set — only then is the window definitively valid and sized correctly.

Edge case verification — "aaaa": right=0: 'a' not in set → add → set={'a'}, max_len=1. right=1: 'a' in set → enter while loop → remove s[0]='a', left=1 → set={} → exit loop → add s[1]='a', set={'a'}, max_len=max(1,1)=1. Same at right=2,3. Final answer: 1. Correct.

4 The Code (Annotated) ▾

Solution 1 (Optimal, Interview Pick) — Sliding Window + Hash Set, O(n)

def lengthOfLongestSubstring(s: str) -> int:
char_set = set() ← Mirrors current window contents; O(1) add/remove/lookup
left = 0 ← Left boundary of the sliding window
max_len = 0 ← Best (longest valid window) seen so far
for right in range(len(s)): ← Expand right edge one character at a time
while s[right] in char_set: ← Duplicate detected — must shrink until it's gone
char_set.remove(s[left]) ← Evict the leftmost char from both window and set
left += 1 ← Advance left boundary
char_set.add(s[right]) ← Duplicate cleared; safe to include s[right]
max_len = max(max_len, right - left + 1) ← Current window length is a candidate
return max_len
⚠️ Why right - left + 1 (not right - left)?

The window [left, right] is inclusive on both ends. If left=2 and right=4, the characters at indices 2, 3, 4 are in the window — that's 3 characters, but 4-2=2. The +1 corrects for inclusive counting. Sanity check: when left == right (window of 1), right-left+1=1. Correct.

Solution 2 — Hash Map (char → last index): O(1) jump instead of step-shrink

def lengthOfLongestSubstring(s: str) -> int:
char_map = {} ← Stores char → most recently seen index
left = 0
max_len = 0
for right, ch in enumerate(s):
if ch in char_map and char_map[ch] >= left: ← Seen AND within current window
left = char_map[ch] + 1 ← Jump left past the previous occurrence (O(1) shrink!)
char_map[ch] = right ← Always update to most recent index
max_len = max(max_len, right - left + 1)
return max_len
💡 Map vs Set: Trade-offs

The hash map approach is strictly faster in practice (O(1) jump vs while-loop step-shrink), but has a subtle bug risk: char_map[ch] >= left is crucial. Without it, you might jump left backwards if ch was seen before the current window. The hash set approach is simpler to reason about and harder to get wrong — great for interviews where clarity matters.

Solution 3 — Brute Force, O(n²): mention briefly then discard

def lengthOfLongestSubstring(s: str) -> int:
max_len = 0
for i in range(len(s)): ← Start each possible substring
seen = set()
for j in range(i, len(s)): ← Extend until duplicate or end
if s[j] in seen: break
seen.add(s[j])
max_len = max(max_len, j - i + 1)
return max_len ← O(n²) — TLE for n = 10⁵
5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Pattern

The problem asks for the longest contiguous substring satisfying a constraint (all unique). This is the canonical signal for variable sliding window. Any time you see "longest/shortest subarray/substring where property holds" — reach for a two-pointer window that expands and shrinks based on validity.

Step 2: Propose Brute Force First

"I could check all O(n²) substrings with a set, giving O(n²) total. Too slow for n=10⁵. Key observation: when I extend right and find a duplicate, I don't need to restart from zero — I only need to shrink left until the duplicate leaves. This amortizes to O(n)."

Step 3: Derive the Sliding Window

"Two pointers: left and right. A hash set tracks what's in the window. Move right forward. If s[right] is in the set — duplicate — advance left, removing s[left] from the set each time, until s[right] is no longer in the set. Then add s[right] and update max_len."

Step 4: Offer the Map Optimization

"An optimization: use char → last seen index. When a duplicate is found at position j inside the window, jump left directly to j+1. Same O(n) time complexity but fewer iterations in practice. I need to guard char_map[ch] >= left to avoid jumping backwards."

🗣️ What to Say to the Interviewer

"I'll use a sliding window with two pointers, left and right, and a hash set.
Move right across the string. When s[right] is already in the set — we have a duplicate —
shrink from left: remove s[left] from set, advance left. Repeat until no duplicate.
Then add s[right], update max_len = max(max_len, right - left + 1).
Each character enters and leaves the set at most once → O(n) time.
Space is O(k) where k is the character set size — O(1) for ASCII."
🔮 Likely Follow-up Questions
  • "At most k repeated characters?" → Use a counter; shrink when any count exceeds k (#395)
  • "At most 2 distinct characters?" → Map + shrink when map size exceeds 2 (#159/#340)
  • "Find all anagram substrings?" → Fixed-size window + frequency count (#438)
  • "Minimum window containing all chars of t?" → Variable window, shrink when over-satisfied (#76)

7 Summary & Key Takeaways ▾
Variable sliding window: expand right freely, shrink left only when a duplicate appears
Hash set mirrors window contents — O(1) membership check and removal
Invariant: s[left..right] always contains only unique characters (maintained by the while loop)
O(n) time: each character enters the set once (via right) and leaves at most once (via left)
Hash map variant jumps left directly — same complexity, fewer operations, but subtle guard needed
Record max_len AFTER the while loop exits and AFTER adding s[right] — not during shrinking

Brute Force (all substrings) O(n²) O(k) TLE for n = 10⁵
Sliding Window + Set ✓ O(n) O(k) Interview pick — clear invariant
Sliding Window + Map O(n) O(k) Fastest in practice; guard required
🎯 When to Recognize the Variable Sliding Window Pattern
  • "Longest/shortest subarray/substring satisfying a condition" → variable sliding window
  • Constraint involves character or element uniqueness or bounded frequency
  • You need O(n), two pointers avoids restarting from scratch each time
  • The window validity can be checked and restored by shrinking from left
