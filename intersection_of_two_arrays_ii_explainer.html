Intersection of Two Arrays II

LeetCode #350 – Find common elements with frequency matching

Easy Hash Table Frequency Map O(n+m) Time O(min(n,m)) Space
1 Understanding the Problem ▼

Find all elements that appear in both arrays, where each element appears as many times as it appears in both.

Example: nums1 = [1,2,2,1], nums2 = [2,2]

  • 1 appears 2× in nums1, 0× in nums2 → min(2,0) = 0 in result
  • 2 appears 2× in nums1, 2× in nums2 → min(2,2) = 2 in result
  • Output: [2, 2]
💡 Key Insight

For each unique value, count how many times it appears in both arrays. The result includes that value min(count1, count2) times.

🎯 Difference from Intersection I

Intersection I: Return unique common elements (use Set)
Intersection II: Return with multiplicity (use frequency Map)

2 The Algorithm: Frequency Map Intersection ▼

Build a frequency map for one array, then iterate the other array and "consume" matches.

1 Build frequency map from the smaller array (for optimal space)
2 Iterate through the larger array
3 For each element: if in map with count > 0, add to result and decrement count
4 Return the result array
⚠️ Why Decrement?

Decrementing ensures each occurrence in nums1 can only match one occurrence in nums2. Without decrementing, we'd count duplicates multiple times.

3 Why It Works: The Invariant ▼
🔐 The Invariant

count[v] represents the number of unmatched occurrences of value v from nums1. When we find v in nums2, we "use up" one occurrence by decrementing.

Why this gives correct multiplicity:

  • If v appears 3× in nums1 and 2× in nums2, we add v twice (count goes 3→2→1)
  • If v appears 2× in nums1 and 3× in nums2, we add v twice (count goes 2→1→0, third occurrence sees count=0)
  • This naturally computes min(count1, count2) for each value
💡 Space Optimization

Build the map from the smaller array to minimize space usage. Worst case is O(min(n, m)).

4 The Code (Annotated) ▼
# Solution 1: Hash Map (Optimal for unsorted)
def intersect(nums1, nums2):
# Build map from smaller array
if len(nums1) > len(nums2):
return intersect(nums2, nums1)
count = Counter(nums1) # Frequency map
result = []
for num in nums2:
if count[num] > 0:
result.append(num)
count[num] -= 1 # Consume one match
return result

Solution 2: Two Pointers (If arrays are sorted)

def intersect_sorted(nums1, nums2):
nums1.sort() # Skip if already sorted
nums2.sort()
i = j = 0
result = []
while i < len(nums1) and j < len(nums2):
if nums1[i] < nums2[j]:
i += 1
elif nums1[i] > nums2[j]:
j += 1
else: # Match found
result.append(nums1[i])
i += 1
j += 1
return result

5 🎤 Interview Strategy ▼
Step 1: Clarify Requirements

Ask: "Do duplicates matter?" (Yes → multiplicity). "Are arrays sorted?" (Affects approach choice).

Step 2: Choose Approach

Unsorted → Hash Map O(n+m). Sorted → Two Pointers O(n+m) with O(1) space. Very large data on disk → Two Pointers with external sort.

Step 3: Optimize Space

Build hash map from the smaller array to minimize memory usage.

🗣️ What to Say to the Interviewer

"I'll use a hash map to count frequencies of elements in the smaller array. Then I iterate through the larger array, and for each element found in the map with count > 0, I add it to the result and decrement the count. This handles multiplicity correctly and runs in O(n+m) time."
🎯 Common Follow-ups
  • What if nums1 is much smaller than nums2? — Hash map approach is ideal
  • What if both are sorted? — Two pointers saves space
  • What if nums2 is stored on disk? — Stream through nums2, keep only the map in memory
