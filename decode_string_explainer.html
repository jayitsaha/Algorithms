LeetCode #394

Decode String

Given an encoded string, return its decoded string. The encoding rule is k[encoded_string], where encoded_string inside brackets is repeated exactly k times.

Medium Recursion Recursive Parsing Time: O(n·k) Space: O(n)
1 Understanding the Problem ▾

The input is a compressed string using the format k[s], meaning "repeat string s exactly k times." These patterns can be nested — a bracket group can contain another bracket group inside it.

💡 Key Insight

The brackets define a natural recursive structure: every [...] group is a self-contained sub-problem whose result feeds into its parent. Solve innermost groups first, then work outward — exactly what recursion (or a stack simulating it) does automatically.

Example 1: Simple

Input: "3[a]2[bc]"
→ "a" × 3 = "aaa"
→ "bc" × 2 = "bcbc"
Output: "aaabcbc"

Example 2: Nested

Input: "2[a2[b]]"
→ inner: "b" × 2 = "bb"
→ outer: ("a"+"bb") × 2
Output: "abbabb"

Edge cases to consider: multi-digit k values like 12[ab] (accumulate digits), plain letters with no brackets (copied verbatim), nested at any depth, and strings where the outer count is 1 (no visible repetition).

⚠️ Multi-digit k values

The repeat count k before [ can be more than one digit: "10[ab]" means repeat "ab" ten times. You must accumulate digits using current_k = current_k * 10 + int(ch) and only use the final value when you see [.

2 The Algorithm: Stack-Based Iterative Parsing ▾

We scan the input left-to-right, maintaining two stacks: one for saved repeat-counts (k_stack) and one for saved partial strings (str_stack). When we enter a bracket, we push our current state. When we leave a bracket, we pop and expand.

1 Initialize: current_string = "", current_k = 0, both stacks empty. We're building the result in current_string at all times.
2 Digit seen: Accumulate into current_k via current_k = current_k * 10 + int(ch). This correctly handles multi-digit numbers like 12.
3 '[' seen: Push current_k onto k_stack and current_string onto str_stack. Then reset both: current_k = 0, current_string = "". We're now building inside the bracket.
4 Letter seen: Append to current_string. These are literal characters that will eventually be repeated.
5 ']' seen: Pop k from k_stack and prev from str_stack. Set current_string = prev + k * current_string. This collapses the innermost bracket group and merges the expanded result back into its parent context.
6 Return current_string: After all characters are processed, current_string holds the fully decoded result.
🔑 Analogy: Nested Save States

Think of it like a video game save system. When you enter a dungeon ([), you save your current progress (push to stacks) and start fresh inside. When you exit (]), you restore your save and add the dungeon's loot (repeated content) to your inventory.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point in the scan, current_string contains the fully decoded output for the current bracket level. Each entry in str_stack holds the already-decoded prefix at one outer nesting depth. When we close a bracket, all inner content is guaranteed to already be decoded in current_string, so the expansion is always correct.

Why nesting resolves correctly: Consider "2[a2[b]]". The inner 2[b] closes first — at that moment current_string = "bb" and str_stack holds the outer level's prefix "a". Closing the outer ] gives us "a" + "bb" = "abb" repeated × 2 = "abbabb". The inner bracket always resolves before the outer one needs its result.

💡 Stack = Simulated Recursion

The two-stack approach is functionally identical to a recursive solution. Each [ is like a recursive call (push context), each ] is a return (pop and combine). The explicit stack avoids relying on the system call stack, making it safe for deeply nested inputs that might overflow the Python recursion limit.

4 The Code (Annotated) ▾

Solution 1 — Stack (Optimal, Interview Pick)

def decodeString(s: str) -> str:
k_stack = [] ← Saved repeat-counts from outer bracket levels
str_stack = [] ← Saved partial strings from outer bracket levels
current_string = "" ← Decoded output being built at the current depth
current_k = 0 ← Digit accumulator (handles multi-digit like "12")
for ch in s: ← Scan every character once — O(n) pass
if ch.isdigit():
current_k = current_k * 10 + int(ch) ← Shift left (×10) and add digit
elif ch == '[': ← Enter bracket: save state, reset for inner level
k_stack.append(current_k) ← Save the k that belongs to this '[...]' group
str_stack.append(current_string) ← Save everything we'd built before this bracket
current_k = 0 ← Reset — ready to parse digits for next nested bracket
current_string = "" ← Reset — start building content for this bracket
elif ch == ']': ← Close bracket: expand and merge back into parent
k = k_stack.pop() ← Retrieve repeat count for this bracket
prev = str_stack.pop() ← Retrieve prefix that was building before this bracket
current_string = prev + k * current_string ← Prefix + repeated inner content
else: ← Plain letter: append to current level's buffer
current_string += ch
return current_string ← Fully decoded result
⚠️ Easy Bug: Forgetting to reset after '['

After pushing to both stacks at [, you MUST reset current_k = 0 AND current_string = "". Forgetting either means the inner bracket level inherits outer state — the output will be silently wrong, with no error thrown.

Solution 2 — Recursive Parser (Elegant, equally valid)

def decodeString(s: str) -> str:
def parse(i): ← Recursive helper: i = current scan index
result = "" ← Decoded output for this call's scope
while i < len(s) and s[i] != ']': ← Run until end or matching ']'
if s[i].isdigit(): ← Parse multi-digit number
k = 0
while i < len(s) and s[i].isdigit():
k = k * 10 + int(s[i]); i += 1
i += 1 ← Skip the '[' that follows the digits
inner, i = parse(i) ← Recurse: parse inside the '[...]'
i += 1 ← Skip the ']' that parse stopped at
result += k * inner ← Expand and append the repeated inner string
else: ← Plain letter
result += s[i]; i += 1
return result, i ← Return decoded string AND the position we stopped at
return parse(0)[0] ← Start from index 0, return only the string
💡 Complexity Summary

Stack approach: Time O(n·k_max) where n = output length, Space O(d) where d = max nesting depth
Recursive approach: Same time; depth-bounded by call stack — may hit Python recursion limit on pathological inputs
Both are optimal. Stack approach is safer for production; recursive is cleaner to derive on a whiteboard.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Recursive Structure

Signal: nested balanced brackets. Any time you see balanced nesting (parentheses, brackets, braces, XML tags), immediately think recursion or stack. Say out loud: "The brackets create a recursive structure where inner groups must be decoded before their parent can use them."

Step 2: Identify What State to Save at '['

Ask yourself: "What do I need to remember when I re-enter the outer scope after the bracket closes?" Answer: (1) the repeat count for this bracket, and (2) the string prefix we'd already built at this level. Those two pieces are exactly your two stacks.

Step 3: Trace a Nested Example Live

Trace "3[a2[c]]" by hand with the stacks. Show: first '[' pushes (k=3, prefix=""), second '[' pushes (k=2, prefix="a"). First ']' produces "acc", second ']' produces "acc"×3 = "accaccacc". Live tracing impresses interviewers and catches bugs early.

Step 4: Mention Multi-digit k Proactively

Before writing code, say: "I'll accumulate consecutive digits with current_k = current_k × 10 + int(ch) to handle values like '12' before I reset at '['. This shows attention to edge cases without being asked."

🗣️ What to Say to the Interviewer

"I see nested balanced brackets — this calls for a stack.
I'll use two stacks: one for repeat-counts, one for partial strings.
When I see '[': push current state and reset for the inner level.
When I see ']': pop the saved count and prefix, expand current_string, merge back.
Digits accumulate into current_k — handles multi-digit. Letters append directly.
Time O(output length), Space O(nesting depth). No edge cases missed."
🔮 Likely Follow-up Questions
  • "Can you solve it recursively?" → Yes — each '[' is a recursive call, ']' is a return.
  • "What if k could be 0?" → 0 * "abc" = "" — Python handles this naturally.
  • "What's the maximum output size?" → Exponential with nesting: 9[9[9[...]]]
  • "Handle incorrectly nested brackets?" → Stack would be non-empty at end — easy validation.
