LeetCode #286

Walls and Gates

Fill each empty room in a grid with the minimum distance to its nearest gate. Walls block passage.

Medium Graph Multi-Source BFS Time: O(m·n) Space: O(m·n)
1 Understanding the Problem ▾

You receive an m × n integer grid with exactly three possible cell values:

  • -1 — a Wall. Impassable. Ignore it completely.
  • 0 — a Gate. Leave it as 0. This is a destination.
  • INF = 2,147,483,647 — an Empty Room. Replace with the minimum number of steps to reach any gate, moving only up/down/left/right. If no gate is reachable, leave it as INF.

The problem is modified in-place — no return value; update the grid directly.

💡 The Core Insight — Flip the Direction of Search

A naive approach asks: "For each empty room, how far is the nearest gate?" — requiring one BFS per room: O((m·n)²). The key insight is to reverse the search direction: run BFS from all gates simultaneously. Because BFS visits cells in non-decreasing distance order, the first time any empty room is reached is guaranteed to be from the nearest gate. One BFS pass fills the entire grid in O(m·n).

Edge cases: No gates (all rooms stay INF). No rooms (grid is all walls/gates). A room completely enclosed by walls (stays INF — BFS never reaches it). A room adjacent to two gates (gets distance 1 from the nearest one automatically).

2 The Algorithm: Multi-Source BFS ▾

Standard BFS from a single source finds the shortest path to all reachable nodes in O(V+E). Multi-source BFS extends this to multiple sources by treating all sources as already enqueued at distance 0. Conceptually, it is equivalent to adding a virtual "super-source" node at distance -1 connected to all gates — then running a single BFS from that super-source.

1 Scan and seed: Iterate every cell. Whenever you find a gate (rooms[r][c] == 0), add (r, c) to the BFS queue. These are all at distance 0.
2 BFS loop: While the queue is non-empty, pop (r, c) from the front. Its current value rooms[r][c] is already the minimum distance from any gate.
3 Explore 4 neighbours: For each of up/down/left/right, check if the neighbour is in-bounds and equals INF. If so, set its value to rooms[r][c] + 1 and enqueue it.
4 Why the INF check works as a visited guard: Walls are -1 (not INF), gates are 0 (not INF), and already-visited rooms are ≤ their true distance (not INF). Only unvisited empty rooms pass — no separate visited set needed.

3 Why It Works: The BFS Distance Invariant ▾
The Invariant

When a cell (r, c) is dequeued, rooms[r][c] already holds the minimum number of steps from any gate to (r, c). This value will never be improved by any future BFS step — any later path would be at least as long.

Why BFS guarantees shortest paths in unweighted grids: BFS explores cells in strictly non-decreasing order of distance. All cells at distance 1 are processed before any at distance 2, and so on. This means the first time we arrive at a room, we have found the shortest path. Any subsequent arrival (which we prevent by the INF check) would be via an equal or longer path.

Why seeding with all gates simultaneously is correct: The minimum distance from any gate to room X is min over all gates G of: dist(G, X). By starting BFS from every gate at the same time, we are running the "super-source" BFS. The BFS wave that first reaches X arrives from the closest gate — exactly the minimum. This is not an approximation; it is mathematically equivalent to running independent BFS from each gate and taking the minimum, but 1000× faster.

Why Unreachable Rooms Are Handled Automatically

If a room is completely surrounded by walls with no path to any gate, BFS simply never reaches it. Its INF value is never overwritten — which is precisely the correct answer. No post-processing or special casing is needed.

m, n = len(rooms), len(rooms[0])← grid dimensions
INF = 2**31 - 1 # 2,147,483,647← the sentinel value for empty rooms
queue = deque()← BFS frontier, starts empty
# ── Phase 1: Seed queue with every gate ──
for r in range(m):
for c in range(n):
if rooms[r][c] == 0:← gate found
queue.append((r, c))← enqueue at distance 0
# ── Phase 2: BFS expansion ──
dirs = [(0,1),(0,-1),(1,0),(-1,0)]← right, left, down, up
while queue:← stops when all reachable rooms are filled
r, c = queue.popleft()← dequeue; rooms[r][c] is already final
for dr, dc in dirs:
nr, nc = r + dr, c + dc← compute neighbour
if (0 <= nr < m and 0 <= nc < n # in-bounds
and rooms[nr][nc] == INF): # unvisited empty room
rooms[nr][nc] = rooms[r][c] + 1← one step further from a gate
queue.append((nr, nc))← enqueue for further expansion
⚠️ Why We Check rooms[nr][nc] == INF (not a visited set)

The cell's value itself encodes whether it has been visited: walls are -1 (≠ INF), gates are 0 (≠ INF), and filled rooms have a finite positive distance (≠ INF). Only unvisited empty rooms hold INF. This saves memory (no extra boolean array) and slightly simplifies the code.

⚠️ Common Mistake: Using rooms[nr][nc] > rooms[r][c] + 1 instead of == INF

In a multi-source BFS with all sources at equal distance, the condition == INF is equivalent and simpler. Using > would also work here, but implies a Dijkstra-style relaxation which is unnecessary — BFS already ensures we reach each room optimally on the first visit.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognise "Shortest Path in Unweighted Grid" → BFS

Keywords: grid, distance, nearest, movement only horizontally/vertically, all moves cost the same (1). This immediately signals BFS, not DFS. DFS finds A path, not the SHORTEST path.

Step 2: Name the Brute Force and its Cost

For each of O(m·n) empty rooms, run a BFS to find the nearest gate: O(m·n) per room → O((mn)²) total. For a 1000×1000 grid that's 10¹² operations. Unacceptable.

Step 3: Ask "Can I Search from Targets Instead?"

Distance is symmetric: dist(room, gate) = dist(gate, room). What if we reverse direction? Run BFS from gates to rooms. With multiple gates, use multi-source BFS — seed all gates at once. Cost: O(m·n). Every room is enqueued at most once.

Step 4: State the "Super-Source" Mental Model

Add a virtual node S connected to all gates with edge weight 0. BFS from S is exactly multi-source BFS. This framing immediately shows correctness: standard BFS from S finds shortest paths to all rooms, and the edges S→gates have weight 0, so distances equal dist(gate, room) for the nearest gate.

🗣️ What to Say to the Interviewer

"This is a shortest-path problem on an unweighted grid, which suggests BFS.

My first instinct: BFS from each empty room to find its nearest gate. But that's O((mn)²) — too slow for large grids.

Better approach: flip the direction. Run BFS from all gates simultaneously. Since BFS visits cells in order of increasing distance, the first time any empty room is reached is guaranteed to be from the nearest gate. I'll seed the queue with every gate, then expand outward.

No separate visited set — I use the cell's value: rooms[r][c] == INF means 'not yet visited'. Walls are -1, gates are 0, filled rooms have a positive integer — none of them match INF."
Follow-up Questions to Prepare
  • No gates in the grid? — Queue starts empty, loop never runs, all rooms stay INF. Correct.
  • Grid with diagonal movement? — Add 4 more direction pairs: (±1,±1).
  • Weighted edges (different terrain costs)? — BFS no longer works; use Dijkstra with a min-heap.
  • What if we need the actual path, not just distance? — Store parent pointers during BFS.
  • Thread safety / distributed grid? — Discuss partitioning the grid and merging BFS frontiers.

7 Summary & Key Takeaways ▾
Multi-source BFS seeds the queue with ALL sources (gates) simultaneously at distance 0
BFS visits cells in non-decreasing distance order — first visit equals minimum distance
The cell value itself (==INF) serves as the "unvisited" marker — no extra set needed
Walls (-1) and settled rooms (positive integer) are automatically skipped by the INF check
Unreachable rooms (walled off) stay INF — BFS simply never reaches them
Time O(m·n) — every cell is enqueued and dequeued at most once
🎯 When to Reach for Multi-Source BFS
  • Grid/graph problem: "fill each node with distance to nearest X"
  • Multiple sources, all equally valid starting points
  • Keywords: "nearest", "closest", "minimum steps to any X"
  • Variants: "rotten oranges", "01 matrix", "distance to nearest 0"
