LeetCode #834

Sum of Distances in Tree

There is an undirected connected tree with n nodes labelled 0 to n-1 and n-1 edges. Return an array ans where ans[i] is the sum of distances between node i and every other node.

Hard Dynamic Programming Rerooting DP Time: O(n) Space: O(n)
1 Understanding the Problem ▾

We have a tree with n nodes (0-indexed) and n-1 edges. For every node i, we want to know: if I stood at node i, how far would I have to travel to reach every other node, in total? Return that total for every node.

💡 Key Insight

A brute-force BFS/DFS from every node costs O(n²). The key insight is that once you know the answer for the root, you can derive the answer for every other node in O(1) per node using a rerooting identity. Two DFS passes give you all n answers in O(n) total.

Naive: BFS from every node — O(n²)

For n=6, run 6 BFS/DFS.
Each DFS scans 6 nodes.
Total: 36 operations → too slow for n=3×10⁴

Rerooting DP — O(n)

DFS 1: get ans[0] (answer for root).
DFS 2: shift root to each neighbor in O(1).
Total: 2 DFS passes = O(n) always.

Concrete example (n=6, root=0): Tree looks like a star from 0 to {1,2,3,4,5} but with 3 attached to nodes further down. From node 0, distances to all others sum up to some number. From node 1 (adjacent to 0), the "subtree under 1" gets 1 closer to every node, but the rest of the tree gets 1 farther.

Edge cases: n=1 → answer is [0]. A linear chain (path graph) → rerooting propagates perfectly. Star graph → leaf nodes each see n-2 distance-1 neighbors + 2 jumps to other leaves.

2 The Algorithm: Two-Pass Rerooting DP ▾

We maintain two auxiliary arrays alongside the answer array:

  • count[node] — number of nodes in the subtree rooted at node (when tree is rooted at 0). This includes the node itself.
  • ans[node] — sum of distances from node to all nodes in its subtree (after DFS 1), then updated to the global answer after DFS 2.
1 Build adjacency list. Represent the undirected tree as a dictionary of sets. Each edge (u, v) goes both ways.
2 DFS 1 (post-order, bottom-up): Root the tree at node 0. For each node in post-order: count[node] = 1 + sum(count[child]) and ans[node] = sum(ans[child] + count[child]) for all children. After this, ans[0] = sum of all distances from node 0.
3 DFS 2 (pre-order, top-down — the rerooting): For each parent-child edge, shift the virtual root from parent to child using the rerooting formula. The subtree under child gains 1 step closer; the rest of the tree (n − count[child] nodes) gains 1 step farther.
4 Rerooting formula: ans[child] = ans[parent] − count[child] + (n − count[child]). Simplified: ans[child] = ans[parent] + n − 2 * count[child].
📐 The Rerooting Recurrence
DFS 1 (bottom-up):
count[node] = 1 + Σ count[child]
ans[node] = Σ (ans[child] + count[child])

DFS 2 (rerooting, for each child of parent):
ans[child] = ans[parent] − count[child] + (n − count[child])
                = ans[parent] + n − 2 × count[child]
3 Why It Works: The Rerooting Identity ▾
🔐 The Core Identity

When we "move" the root from a parent to a child across the edge (parent → child), exactly two groups of nodes are affected:

  • count[child] nodes in the child's subtree each get 1 step closer to the new root.
  • (n − count[child]) nodes outside the child's subtree each get 1 step farther from the new root.

Net change: +1 × (n − count[child]) − 1 × count[child] = n − 2 × count[child].

Why DFS 1 gives correct ans[0]: The post-order accumulation is a classic tree DP. In the recurrence ans[node] = Σ (ans[child] + count[child]), each +count[child] term accounts for the extra "one edge" hop from node to every node inside child's subtree. Adding children's subtree answers propagates all distances up correctly.

⚠️ Common Mistake: Wrong Count Update

After DFS 2, do not update count[child]. The count values are fixed with respect to the original rooting at 0. The rerooting formula uses the subtree sizes from DFS 1 — they are constant throughout DFS 2.

Edge case: n=1. All arrays have size 1. ans[0] = 0 by definition. The DFS loops simply don't execute any children steps.

Why DFS not BFS? We need post-order (children computed before parent) for DFS 1, and pre-order (parent computed before children) for DFS 2. DFS naturally gives these orderings; BFS with level-by-level processing also works but requires more bookkeeping.

4 The Code (Annotated) ▾

Solution 1 — Rerooting Tree DP (Optimal, O(n) time + space) — Interview Pick

def sumOfDistancesInTree(n, edges):
from collections import defaultdict
graph = defaultdict(set) ← Adjacency list (undirected tree)
for u, v in edges: ← Each edge stored in both directions
graph[u].add(v); graph[v].add(u)
count = [1] * n ← count[i] = subtree size (initialized to 1: the node itself)
ans = [0] * n ← ans[i] = sum of distances from i (built up in two passes)
def dfs1(node, parent): ← Post-order DFS: children computed before parent
for child in graph[node]: ← Iterate over all neighbors
if child == parent: continue← Skip the edge back to parent (avoid cycle in undirected graph)
dfs1(child, node) ← Recurse: compute child's subtree first
count[node] += count[child] ← Add child's subtree size to this node's count
ans[node] += ans[child] + count[child] ← +ans[child]: distances within child's subtree; +count[child]: one extra hop for each node in it
def dfs2(node, parent): ← Pre-order DFS: parent answer computed before children
for child in graph[node]: ← For each child of current node
if child == parent: continue
ans[child] = ans[node] + n - 2 * count[child] ← Rerooting: shift virtual root from node to child
dfs2(child, node) ← Recurse with the updated ans[child]
dfs1(0, -1) ← Root at node 0; -1 means no parent
dfs2(0, -1) ← Propagate root's answer to all other nodes
return ans ← ans[i] now holds the global sum of distances from i
⚠️ Python Recursion Depth

For large n (up to 3×10⁴) on a path graph (linear chain), the DFS recursion depth is O(n). Python's default recursion limit is ~1000. Add import sys; sys.setrecursionlimit(50000) at the top, or convert the DFS to iterative using an explicit stack.

Solution 2 — BFS from Every Node (Brute Force, O(n²))

def sumOfDistancesInTree_BF(n, edges): ← Brute force: BFS from each node
from collections import deque, defaultdict
graph = defaultdict(list)
for u, v in edges: graph[u].append(v); graph[v].append(u)
def bfs(src): ← BFS computes distances from src
dist, q = [-1]*n, deque([src])
dist[src] = 0
while q:
u = q.popleft()
for v in graph[u]:
if dist[v] == -1: dist[v] = dist[u]+1; q.append(v)
return sum(dist)
return [bfs(i) for i in range(n)] ← O(n) per node × n nodes = O(n²) total
💡 Complexity Summary

Rerooting DP: Time O(n) (two DFS passes), Space O(n) (graph + count + ans arrays)
Brute Force BFS: Time O(n²), Space O(n) — times out for n=3×10⁴

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the "for every root" pattern

The problem asks for a result from the perspective of every single node. Whenever you see "compute for all nodes", think: can I compute for one node (usually the root) and then efficiently propagate to the others? This is the hallmark of rerooting DP.

Step 2: Establish what DFS 1 computes

Root the tree arbitrarily at node 0. DFS 1 (post-order) fills two arrays: count[v] = subtree size and ans[0] = sum of all distances from the root. This is a standard tree DP — tell the interviewer you're doing a "subtree aggregation".

Step 3: Derive the rerooting formula on the whiteboard

Say: "If I move my imaginary root from node u to its neighbor v, every node in v's subtree gets 1 closer. Every node outside gets 1 farther. Subtree of v has count[v] nodes; the rest have n − count[v]. So ans[v] = ans[u] + (n − count[v]) − count[v] = ans[u] + n − 2·count[v]." Write this derivation out — it's the heart of the solution.

Step 4: Code both DFS passes cleanly

DFS 1 is post-order (recurse first, then aggregate). DFS 2 is pre-order (update child, then recurse). Both need to avoid re-visiting the parent — pass parent as a parameter (no visited set needed for trees).

🗣️ What to Say to the Interviewer

"The brute force is O(n²) — BFS/DFS from every node. We need better.
Key observation: once I know the answer for the root (node 0), I can derive the answer for its neighbor v using the rerooting identity: ans[v] = ans[0] + n − 2·count[v].
So: DFS 1 (post-order) builds count[] and fills ans[0]. DFS 2 (pre-order) propagates to all nodes. Two DFS passes = O(n).
The rerooting insight: moving the root across an edge (u→v), v's subtree of size count[v] gets 1 closer, the rest get 1 farther."
🔮 Likely Follow-up Questions
  • "What if edges have weights?" → Same approach; DFS 1 accumulates weighted distances; rerooting uses the edge weight instead of 1.
  • "Find the node with minimum sum of distances?" → Just return min(enumerate(ans), key=lambda x: x[1]) after computing ans.
  • "What if it's not a tree, but a general graph?" → Rerooting DP only works on trees. Use Dijkstra for graphs.
  • "Can you avoid Python recursion limit?" → Iterative DFS with explicit stack, using topological order (BFS with in-degree tracking).

7 Summary & Key Takeaways ▾
Root the tree anywhere (node 0 is conventional). DFS 1 computes subtree sizes and the answer for the root.
The rerooting identity ans[child] = ans[parent] + n − 2×count[child] is the entire trick. Derive it on paper first.
DFS 1 is post-order (children before parent). DFS 2 is pre-order (parent before children). Order matters.
No visited set needed for trees — just pass parent as a parameter to avoid backtracking to the edge you came from.
Final complexity: O(n) time, O(n) space. Optimal — you must read all edges at least once.

  • Value has a natural subtree aggregation form (DFS 1 is easy)
