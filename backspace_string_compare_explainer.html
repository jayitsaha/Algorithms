Backspace String Compare

LeetCode #844 — Two Pointers: Reverse (Skip Backspaces)
Easy Two Pointers Reverse Two Pointers O(n) Time · O(1) Space
1
Understanding the Problem
▼

Two strings typed in a text editor where # means a backspace. Determine if the resulting strings are equal.

💡 Key Insight

Naive: simulate typing into a stack (O(n) space). Optimal: scan right-to-left. A # erases the character immediately to its left. Scanning backwards, we see the # first and count pending erasures before encountering the character to erase.

Example: s = "ab#c" → "ac", t = "ad#c" → "ac" → true

Edge cases: Multiple consecutive #, leading # on empty buffer, one string entirely erased.

2
Two Solutions
▼

Solution 1 — Stack Simulation O(n) space:

def backspaceCompare_stack(s, t): def process(st): stack = [] for ch in st: if ch == '#': if stack: stack.pop() else: stack.append(ch) return stack return process(s) == process(t)

Solution 2 — Reverse Two Pointers O(1) space:

1i = end of s, j = end of t; skip_s = skip_t = 0
2Scan s left: if s[i]=='#' → skip_s++, i--. Elif skip_s>0 → skip_s--, i--. Else break.
3Do the same for t with j
4Both exhausted together → equal. Only one → unequal. s[i] != t[j] → false. Else i--, j--.
🔑 Why Right-to-Left Works

Left-to-right: we don't know how many '#' follow a character. Right-to-left: we see '#' first and accumulate credits, spending them to skip preceding characters before ever needing to compare them.

3
The Code (Annotated)
▼
def backspaceCompare(s: str, t: str) -> bool: i, j = len(s)-1, len(t)-1 # start from the end of each string skip_s = skip_t = 0 # pending backspace credits while i >= 0 or j >= 0: while i >= 0: # find next real char in s if s[i] == '#': skip_s += 1; i -= 1 # earn skip elif skip_s > 0: skip_s -= 1; i -= 1 # spend skip else: break # real char found while j >= 0: # same for t if t[j] == '#': skip_t += 1; j -= 1 elif skip_t > 0: skip_t -= 1; j -= 1 else: break if i < 0 and j < 0: return True # both exhausted if i < 0 or j < 0: return False # different lengths if s[i] != t[j]: return False # mismatch i -= 1; j -= 1 # matched, continue return True
⚠️ Both vs One Exhausted

Always distinguish i < 0 AND j < 0 (equal) from i < 0 OR j < 0 (unequal lengths). Getting this wrong is the #1 bug in this solution.

🎤 Interview Strategy
▼
1Start with stack: Easy O(n) space baseline.
2Improve: "Can we do O(1) space? Scan right-to-left, use a counter for pending backspaces."
3Invariant: skip_s = number of '#' seen but not yet applied. Each non-'#' either gets skipped (if skip>0) or becomes the next real character.
4State all 4 outcomes: both at -1 → true; one at -1 → false; s[i]!=t[j] → false; match → continue.
