LeetCode #844

Backspace String Compare

Given two strings s and t, return true if they are equal when both are typed into empty text editors — '#' means a backspace character.

Easy Two Pointers Reverse Two Pointers Time: O(n+m) Space: O(1)
1 Understanding the Problem ▾

Imagine you're typing into a text editor. Every time you type a normal letter, it appears on screen. Every time you type a #, the cursor goes back and erases the last character — just like pressing the Backspace key on your keyboard. This problem asks: after "typing" both strings s and t into their own empty editors, are the resulting screens identical?

💡 Key Insight

A # doesn't mean "the string contains a hash character" — it means "delete the previous character." If there is no previous character, the # is a no-op (backspace on an empty editor does nothing). The real string is whatever survives after all deletions are processed.

✓ Equal: s="ab#c", t="ad#c"

s: a → b → # (del b) → c → "ac"
t: a → d → # (del d) → c → "ac"
Both produce "ac" → True

✗ Not equal: s="ab##", t="c#d"

s: a → b → # (del b) → # (del a) → ""
t: c → # (del c) → d → "d"
"" ≠ "d" → False

Edge cases to consider: a # at the very start of a string (no-op — nothing to delete), consecutive ## sequences (each deletes one more character leftward), strings that consist entirely of backspaces (result is an empty string), and strings where the final results have different lengths despite the original strings having the same length.

2 The Algorithm: Reverse Two Pointers ▾

The optimal approach scans both strings from right to left simultaneously without using any extra space. The key observation: a backspace affects the character immediately to its left. If we scan right-to-left, we encounter the # before its victim character — so we carry a skip counter that accumulates pending deletions instead of needing a stack.

🔑 Why Scan Backward?

When scanning left-to-right, encountering # means we must retroactively undo a character already processed. But scanning right-to-left, we see # before the character it would delete — so we increment a skip counter and simply ignore the next real characters. This converts a stack-based O(n) space operation into an O(1) counter trick.

1 Initialize: Set i = len(s) - 1, j = len(t) - 1 — both at the rightmost index. Set skip_s = skip_t = 0 to track pending backspaces for each string independently.
2 Advance i to next surviving character in s: Inner while loop — if s[i] is '#', increment skip_s and move i left. If s[i] is a letter but skip_s > 0, decrement skip_s (character erased) and move i left. If a letter with skip_s == 0, break — this character survives.
3 Advance j to next surviving character in t: Same logic — inner while loop with skip_t. Stop when you find a surviving character in t or exhaust the string.
4 Compare the surviving characters: If both i and j are valid, they must match. If one is exhausted but the other isn't → strings differ in length → False. If characters differ → False. Otherwise match; decrement both and repeat.
⚠️ Stack Approach (O(n+m) Space) — Mention First in Interviews

The simpler approach simulates both strings using stacks. For each character, push letters and pop for #. Then compare the two resulting stacks. This is O(n+m) time and O(n+m) space. It is correct and very easy to explain — always propose this first, then offer to optimize to O(1) space with the reverse pointer trick.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After each inner while loop completes, the current pointer position (i for s, j for t) sits on the next character that would actually appear on screen when reading from the right end. This is maintained because the skip counter precisely tracks how many real characters still need to be consumed by previously-seen backspaces. When skip reaches 0 and we land on a letter, that letter is guaranteed to survive.

Why the skip counter is equivalent to a stack: Normally, a stack processes characters left-to-right — push letters, pop for #. Reading right-to-left, every # seen is a "future pop" (it will delete something to the left). The skip counter accumulates those future pops. When we encounter a real letter, we either decrement skip (it gets consumed by a pending pop) or break (it survives). This is functionally identical to running a stack simulation, but uses O(1) space instead of O(n).

Edge case — # at position 0: When the pointer reaches position 0 with skip > 0, the inner while loop condition fails (i < 0) before it can over-decrement. The skip counter is simply left positive, which is fine — the string is now exhausted, correctly treated as empty.

⚠️ Common Mistake: Not handling "no-op" backspaces

If a string starts with #a, the first # does nothing (nothing to delete), and the result is just "a". The inner while loop handles this correctly because it checks i >= 0 before consuming: when the pointer reaches -1, the loop exits. Never let skip go negative — always check elif skip_s > 0 before decrementing.

Termination guarantee: Both i and j decrease monotonically. Inner loops also only move pointers left. The outer while exits when both pointers fall below 0. The algorithm always terminates in O(n+m) steps total.

4 The Code (Annotated) ▾

Solution 1 — Stack Simulation (Brute Force, O(n+m) space)

def backspaceCompare(s: str, t: str) -> bool:
def process(string): ← simulate typing into an editor
stack = []
for c in string: ← left-to-right scan
if c != '#': ← normal letter: add to screen
stack.append(c)
elif stack: ← backspace + non-empty: erase last char
stack.pop() ← if stack is empty, '#' is a no-op
return stack
return process(s) == process(t) ← compare resulting character lists

Solution 2 — Reverse Two Pointers (Optimal, O(1) space) ✓ Interview Pick

def backspaceCompare(s: str, t: str) -> bool:
i, j = len(s) - 1, len(t) - 1 ← start at rightmost index of each string
skip_s = skip_t = 0 ← pending backspace count per string
while i >= 0 or j >= 0: ← loop while either string has characters left
while i >= 0: ← advance i to next surviving char in s
if s[i] == '#': ← backspace seen: accumulate a pending deletion
skip_s += 1; i -= 1
elif skip_s > 0: ← real char erased by pending backspace
skip_s -= 1; i -= 1
else: break ← real surviving char found; stop inner loop
while j >= 0: ← same logic for t using skip_t
if t[j] == '#':
skip_t += 1; j -= 1
elif skip_t > 0:
skip_t -= 1; j -= 1
else: break
if i >= 0 and j >= 0: ← both have surviving characters: must match
if s[i] != t[j]: return False
elif i >= 0 or j >= 0: ← one exhausted, other not: lengths differ
return False
i -= 1; j -= 1 ← both matched, advance both pointers left
return True ← all pairs matched, both exhausted together
💡 Complexity Summary

Stack Simulation: Time O(n+m), Space O(n+m) — each string creates a stack up to its original length
Reverse Two Pointers: Time O(n+m), Space O(1) — no extra data structures; each position visited at most once
Both are optimal in time. The two-pointer approach is the space-optimal solution.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify the '#' Behavior

Ask: "What happens when # appears at the beginning — no-op or error?" (No-op, per the problem.) "Can there be consecutive ##?" (Yes — each one deletes one more character to the left.) This shows you read the edge cases carefully.

Step 2: Propose the Stack Approach First

"My first idea: simulate each string with a stack. Push letters, pop for #, no-op if stack is empty. Compare the resulting stacks. This is O(n+m) time and O(n+m) space. Clean and easy to verify."

Step 3: Drive Toward O(1) Space

"Can I do O(1) space? Key observation: a # deletes the character to its left. If I scan right-to-left, I see the backspace before its victim. A skip counter replaces the stack — when I see #, increment skip; when I see a letter with skip > 0, it's been erased — decrement skip. When skip is 0, the letter survives."

Step 4: Describe the Comparison Protocol

"Two pointers, both starting at the rightmost index. Advance each past erased positions. When both land on surviving characters, compare. Mismatch → False. One exhausted before the other → False. Both exhausted at the same time → True."

🗣️ What to Say to the Interviewer

"I'll start with a stack per string: push letters, pop for '#', no-op if empty. Compare the stacks. O(n+m) space.
To optimize to O(1) space: scan right-to-left. '#' means skip the next real character on the left.
Track skip_s and skip_t. Inner while loops advance each pointer past erased characters.
Then compare surviving characters. Three outcomes: both exhausted → True;
one leftover → False; characters differ → False. Time O(n+m), Space O(1)."
🔮 Likely Follow-up Questions
  • "What if '#' appears at the start of the string?" → No-op; the inner while exits when i < 0 before consuming a non-existent character.
  • "What about '###' three in a row?" → skip accumulates to 3, then consumes the next three real characters scanning left.
  • "What if the strings are given as streams (no random access)?" → Must use the stack approach; the reverse pointer trick requires reading from the end, needing indexable input.
  • "How would you handle a 'forward delete' key too?" → That would require forward scanning with a deque or two-pass approach — an interesting extension.
