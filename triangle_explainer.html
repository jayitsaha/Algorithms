LeetCode #120

Triangle

Given a triangle array, return the minimum path sum from top to bottom. Each step you may move to an adjacent number in the row below.

Medium Dynamic Programming Bottom-up DP Time: O(n²) Space: O(n)
1 Understanding the Problem ▾

Picture a pyramid of numbers: one number at the top, two in the next row, three in the next, and so on. You must travel from the apex to any cell in the bottom row. At each step, you can only move directly below (same column index) or diagonally right-below (column index + 1). You want the path with the minimum total sum.

💡 Key Insight

Every cell triangle[i][j] is reachable from exactly one or two cells in the row above. This bounded dependency is the hallmark of bottom-up DP: we build the answer row-by-row in reverse, only ever needing the row just below the current one. A single rolling 1D array captures everything.

Example: [[2],[3,4],[6,5,7],[4,1,8,3]]

3 4
6 5 7

Best path: 2+3+5+1 = 11

Adjacency rule: From [i][j], you can go to [i+1][j] (directly below) or [i+1][j+1] (right-below). Edge cells at j=0 and j=i have only one parent from above.

Edge cases: single-element triangle (return that value), all-negative values (min still well-defined), deep triangles where the optimal path zigzags non-intuitively.

2 The Algorithm: Bottom-Up Tabulation with Rolling Array ▾

Instead of "which path should I take from here?" (exponential top-down), we answer "what is the best cost from here to the bottom?" for every cell, building upward so each answer reuses already-computed sub-answers below it.

1 Base case — copy the last row: dp = triangle[-1][:]. Each dp[j] = the min-cost path starting at column j in the bottom row (which is just the cell's own value — no further steps to take).
2 Sweep upward: For each row i from n-2 down to 0, for each column j: dp[j] = triangle[i][j] + min(dp[j], dp[j+1]). This says: "Best path from (i,j) to the bottom = this cell's value + the lesser of the two paths from its children below."
3 Return dp[0]: After the apex (row 0, exactly one cell), dp[0] holds the minimum total path sum from top to bottom.
📐 The Recurrence
dp[j] = triangle[i][j] + min(dp[j], dp[j+1])
Base: dp[j] = triangle[n-1][j] for each j in last row
Answer: dp[0] after processing row 0
🔑 Why Bottom-Up (not Top-Down)?

In top-down we'd need to explore every possible downward path — that's 2ⁿ paths for n rows. Bottom-up reuses results: when we compute dp[j] for row i, both dp[j] and dp[j+1] from the row below are already final. We never recompute anything. Total work = n² cells, one operation each.

3 Why It Works: Optimal Substructure & Overlapping Subproblems ▾
🔐 The Invariant

After we process row i in the bottom-up sweep, dp[j] equals the minimum total cost of any path that starts at cell (i, j) and reaches some cell in the bottom row. This holds for every j at every processed row, and gives us dp[0] at the apex as our answer.

Optimal Substructure: The minimum-cost path from (i,j) to the bottom is triangle[i][j] + min(best_from(i+1,j), best_from(i+1,j+1)). The globally optimal path through (i,j) contains globally optimal sub-paths from its children. This recursive structure is exactly optimal substructure — the reason DP works here.

Overlapping Subproblems: In naive recursion, cell (3,1) in a triangle might be reached via (2,0)→(3,1) and (2,1)→(3,1). Both calls would independently recurse all sub-paths from (3,1). For n=20 rows, this blows up to O(2²⁰) calls. Memoization or tabulation computes each (i,j) exactly once.

⚠️ Why Greedy Fails in General

Greedy picks the locally smaller child at each step without look-ahead. On [[1],[2,3],[20,4,1]]: Greedy picks 2 (smaller than 3) then 4. Sum = 7. But optimal picks 3 then 1. Sum = 5. Greedy missed it because 3's subtree contains the best value. DP considers all subtrees before deciding.

O(n) space suffices: At any moment, we only need dp values from the row directly below the current one. We overwrite dp[j] in-place as we sweep left-to-right within each row — the overwritten dp[j] (which was the "below-j" child) has already been consumed. The key ordering: process j=0,1,...,i (left to right within row i) so dp[j+1] is still the old "below-right" value when we compute dp[j].

4 The Code (Annotated) ▾

Solution 1 — Bottom-Up Tabulation (Optimal, Interview Pick ✓)

def minimumTotal(triangle: list[list[int]]) -> int:
dp = triangle[-1][:] ← Slice-copy last row (base case; avoids mutating input)
for i in range(len(triangle)-2, -1, -1):← From second-to-last row, walk up to row 0
for j in range(len(triangle[i])): ← Row i has exactly i+1 elements
dp[j] = triangle[i][j] + min(dp[j], dp[j+1]) ← Best path from (i,j) = cell + best child
return dp[0] ← Apex processed: dp[0] = global min path sum
⚠️ Why triangle[-1][:] not triangle[-1]?

dp = triangle[-1] would make dp a reference to the same list object. Modifying dp[j] would corrupt the original input triangle. The slice [:] creates a shallow copy. In interviews: always copy input data when you plan to modify it in-place.

Solution 2 — Top-Down Memoization (same O(n²) time, easier to derive)

from functools import lru_cache
def minimumTotal(triangle: list[list[int]]) -> int:
n = len(triangle)
@lru_cache(maxsize=None) ← Cache (row, col) → min path cost from this cell
def dp(i, j): ← Returns min cost path from (i,j) to bottom
if i == n - 1: ← Base: bottom row, path cost = cell value
return triangle[i][j]
return triangle[i][j] + min(dp(i+1,j), dp(i+1,j+1)) ← Recurse to children
return dp(0, 0) ← Query from apex

Solution 3 — O(1) Extra Space (in-place, for follow-up)

def minimumTotal(triangle: list[list[int]]) -> int:
for i in range(len(triangle)-2, -1, -1): ← Modify triangle itself — no extra array
for j in range(len(triangle[i])):
triangle[i][j] += min(triangle[i+1][j], triangle[i+1][j+1]) ← Update in-place
return triangle[0][0] ← Apex now holds the answer; trade-off: mutates input
💡 Complexity Summary

Brute Force: Time O(2ⁿ), Space O(n) — recomputes shared subproblems
Top-Down Memo: Time O(n²), Space O(n²) — cache + recursion stack
Bottom-Up (✓ Interview Pick): Time O(n²), Space O(n) — rolling 1D array
In-Place: Time O(n²), Space O(1) extra — mutates input, O(1) beyond triangle itself

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the DP Signal

The problem asks for a minimum over all paths. "Optimize over all choices" is the strongest signal for DP. Confirm: can the minimum path from (i,j) be expressed in terms of smaller subproblems? Yes — min_path(i,j) = triangle[i][j] + min(min_path(i+1,j), min_path(i+1,j+1)). Optimal substructure: check. Do subproblems overlap? Yes, multiple paths merge at the same cell. DP is justified.

Step 2: Define State and Direction

State: dp[j] = min cost from cell (current_row, j) to the bottom row. Direction: bottom-up, because we need the sub-answers from the row below to compute each row above. This is the natural direction — base cases (last row) are trivial.

Step 3: Write the Recurrence and Code

Start with the memoized recursive form (top-down) — it directly mirrors the recurrence and is easy to explain. Then offer the bottom-up rolling array as the space-optimized version. Mention O(1) in-place as a final follow-up.

Step 4: Verify with Example

Trace through [[2],[3,4],[6,5,7],[4,1,8,3]]: Start dp=[4,1,8,3]. Row 2: dp=[7,6,10,3]. Row 1: dp=[9,10,10,3]. Row 0: dp=[11,...]. Return 11. Walk the interviewer through at least two rows to show you understand the sweep direction.

🗣️ What to Say to the Interviewer

"I see we need minimum over all root-to-leaf paths — strong DP signal.
Let dp[j] = min path cost from column j in the current row to the bottom.
Base case: last row — dp[j] = triangle[-1][j].
Recurrence: sweep upward, dp[j] = triangle[i][j] + min(dp[j], dp[j+1]).
At the end, dp[0] = answer. Time O(n²), Space O(n).
If you want O(1) extra space, I can modify the triangle in-place — same logic."
🔮 Likely Follow-up Questions
  • "O(1) extra space?" → Modify triangle in-place: triangle[i][j] += min(triangle[i+1][j], triangle[i+1][j+1])
  • "Print the actual path, not just the sum?" → Track parent indices alongside dp; backtrack from dp[0]
  • "Maximum path sum?" → Replace min with max everywhere — same structure
  • "K minimum paths?" → Priority queue variant of the DP; significantly harder

7 Summary & Key Takeaways ▾
Optimal substructure: min path from (i,j) = cell + min(best from child-left, best from child-right)
Bottom-up sweep (last row → apex) processes each cell exactly once — O(n²) total, no recomputation
Rolling 1D dp array suffices because we only ever look one row down — O(n) space
Copy the last row with slice [:] to avoid mutating the original input
O(1) extra space variant: modify triangle in-place, return triangle[0][0]
