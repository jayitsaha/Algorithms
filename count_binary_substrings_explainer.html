LeetCode #696

Count Binary Substrings

Count substrings with equal numbers of consecutive 0s and 1s — all 0s together, all 1s together.

Easy Two Pointers Group Count Comparison Time: O(n) Space: O(1)
1 Understanding the Problem ▾

Given a binary string s (containing only '0' and '1'), count how many substrings contain an equal number of 0s and 1s, with all the 0s grouped together and all the 1s grouped together. The groups must be consecutive — "0011" and "10" and "1100" qualify; "0101" does not.

💡 Key Insight

Think of the string as a sequence of runs — maximal blocks of the same character. "00110011" breaks into runs: [00][11][00][11] with lengths [2, 2, 2, 2]. Every valid substring straddles exactly one boundary between adjacent runs. At a boundary between runs of lengths prev and cur, there are exactly min(prev, cur) valid substrings.

✓ Valid substrings in "00110011"

"01" — indices [1,2]
"0011" — indices [0,3]
"10" — indices [3,4]
"1100" — indices [2,5]
"01" — indices [5,6]
"0011" — indices [4,7]
Total: 6

✗ Does NOT qualify

"0101" — 0s and 1s alternate;
not in a single block each

"0110" — has equal 0s and 1s
but the 0s are split around 1s
→ invalid structure

Edge cases: All-same-character string (e.g., "0000") → 0 valid substrings. Length-1 string → 0. Two adjacent different characters "01" → exactly 1 valid substring. Longer runs produce more valid substrings at that one boundary.

2 The Algorithm: Group Count Comparison ▾

Instead of examining every substring (brute force: O(n²) to O(n³)), we scan the string once, tracking the length of the current run (cur) and the immediately preceding run (prev). Every time a run ends, the boundary contributes min(prev, cur) to the answer.

min(2, 3) = 2 valid substrings

Why min(prev, cur)? At a boundary between k zeros and m ones, you can form substrings "01", "0011", "000111", … up to length 2 × min(k, m). That is exactly min(k, m) distinct substrings, one per "step size".

1 Initialize: prev = 0 (no prior run), cur = 1 (the first character starts the first run), count = 0.
2 Scan from index 1: At each position i, compare s[i] with s[i-1]. If they are the same, the current run continues — increment cur.
3 On boundary: If s[i] != s[i-1], the run ended. Add min(prev, cur) to count. Then shift: set prev = cur, reset cur = 1.
4 After the loop: The final run pair never triggers the "else" branch. Add min(prev, cur) one more time to capture the last boundary.
5 Return count.
🔑 Brute Force Alternative (O(n²))

For each starting index, expand right as long as the substring stays valid (checks if it has consecutive equal-length blocks). Correct but quadratic. Propose this first in an interview, then immediately improve to the O(n)/O(1) group-count approach.

3 Why It Works: The Invariant ▾
🔐 The Invariant

Every valid substring straddles exactly one boundary between two adjacent same-character runs. At a boundary between a run of length prev and a run of length cur, there are exactly min(prev, cur) valid substrings. Summing this over all boundaries yields the complete answer.

Why exactly one boundary? A valid substring is [block of X][block of Y], where X ≠ Y. This construction crosses one boundary. If a substring crossed two boundaries, it would have three runs — say "00110" — and its leftmost and rightmost characters would be the same ('0'), meaning the first and third blocks are the same character, making the substring invalid for our criteria.

Proof that min(prev, cur) is the count: Say we have k zeros followed by m ones at a boundary (k = prev, m = cur). The valid substrings at this boundary are:

  • The rightmost 1 zero + 1 one from the boundary → length 2 substring
  • The rightmost 2 zeros + 2 ones → length 4 substring
  • The rightmost min(k, m) zeros + min(k, m) ones → length 2×min(k,m)

That gives exactly min(k, m) distinct valid substrings per boundary. No more (the shorter run runs out), no fewer (each nested pair is distinct).

⚠️ The "Off-by-One" Boundary Trap

The loop counts a boundary when s[i] != s[i-1] — that is, at the start of a new run. This means the last boundary (where the last run begins) is counted in the loop, but the boundary after the last run is never triggered. That's why you must add min(prev, cur) once more after the loop. Forgetting this always gives an answer that is too small by exactly min(prev_last, cur_last).

Edge: all-same-character string. Only one run exists; the "else" branch never fires, so prev stays 0. After the loop, min(0, n) = 0. Correct.

4 The Code (Annotated) ▾

Solution 1 — Group Count Two Pointers (O(n) time, O(1) space) — Interview Pick ✓

def countBinarySubstrings(s: str) -> int:
prev, cur, count = 0, 1, 0 ← prev=prev run length (0 initially), cur=current run length, count=answer
for i in range(1, len(s)): ← compare each character to its predecessor
if s[i] == s[i - 1]: ← same character? we are still in the same run
cur += 1 ← extend the current run
else: ← character changed → we hit a run boundary
count += min(prev, cur)← valid substrings at this boundary = min(prev, cur)
prev = cur ← the run that just ended becomes the "previous" run
cur = 1 ← start counting the new run (first character already seen)
count += min(prev, cur) ← the final run pair needs one more boundary count
return count ← total number of valid substrings
⚠️ Common Mistake: Missing the Final Boundary

The loop only counts a boundary when s[i] != s[i-1] — meaning at the start of each new run. The loop never "fires" on behalf of the last run's end. Always write count += min(prev, cur) after the loop. Verify with "0011": boundary at index 2 fires inside loop (min(0,2)=0), then after loop min(2,2)=2. Correct total: 2.

Solution 2 — Run-Length Array (O(n) time, O(n) space, clearer conceptually)

def countBinarySubstrings(s: str) -> int:
groups, cur_len = [], 1 ← groups: list of run lengths
for i in range(1, len(s)):
if s[i] == s[i - 1]:
cur_len += 1
else:
groups.append(cur_len)← save completed run length
cur_len = 1
groups.append(cur_len) ← don't forget the last run
return sum(min(a, b) for a, b in zip(groups, groups[1:]))← sum min at each adjacent pair
💡 Complexity Summary

Group Count (Optimal, Solution 1): Time O(n) — one linear scan. Space O(1) — only 3 integers.
Run-Length Array (Solution 2): Time O(n), Space O(n) — stores all run lengths. Easier to reason about but wastes memory.
Brute Force: Time O(n²–n³), Space O(1) — enumerate all substrings. Correct but too slow for competitive inputs.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: List Examples — Find the Structure

Write out the valid substrings of "00110011": "01", "0011", "10", "1100", "01", "0011". Notice they all cluster around the transitions between 0-blocks and 1-blocks. Say: "Every valid substring must straddle a transition between two consecutive character runs."

Step 2: Reduce to Run-Lengths

Articulate: "I can break the string into run lengths. For '00110011', the runs are [2, 2, 2, 2]. At each boundary between adjacent runs, I can form min(left, right) valid substrings." Verify: min(2,2) + min(2,2) + min(2,2) = 6. Correct.

Step 3: Propose O(n) Space, Then Optimize to O(1)

Start with the run-length array approach: "I'll collect all run lengths, then zip adjacent pairs and sum mins — O(n) time and space." Then say: "I can optimize space to O(1) by keeping only the previous and current run lengths as I scan, updating whenever the character changes."

Step 4: Code It and Narrate the Edge Case

Write the loop. Explicitly mention: "After the loop I need to add min(prev, cur) one more time — the loop never processes the final boundary." Check edge case: all same character → prev stays 0, final min(0, n) = 0. Correct.

🗣️ What to Say to the Interviewer

"I notice that every valid substring must lie at the boundary between two consecutive runs.
If I know the run lengths, the count at each boundary is min(left_run, right_run).
I can compute this in O(n) time by tracking just two values: the length of the previous run and the length of the current run.
When the character changes, I add min(prev, cur) to the answer and slide the window.
After the loop, I add one final min(prev, cur) for the last boundary.
Time O(n), space O(1)."
🔮 Likely Follow-up Questions
  • "Return the actual substrings, not just the count?" → Track boundary indices; reconstruct substrings. Output-size space.
  • "What if we allow up to k mismatches?" → Sliding window with a counter — entirely different algorithm.
  • "Generalize to k equal consecutive groups?" → Compare run groups that are k apart: sum(min(groups[i], groups[i+k]) for i in range(len(groups)-k)).
  • "What if the string is not binary?" → Same algorithm — runs of any repeated character; boundaries still yield min(prev, cur).

🎯 When to Recognize the Group Count Pattern
  • Problem involves counting substrings over consecutive same-character blocks
  • Valid substring structure requires adjacent groups — think run-length decomposition
  • Counting by "boundary contribution" rather than by brute enumeration
  • Any time the answer at a transition point depends only on the two adjacent run sizes
