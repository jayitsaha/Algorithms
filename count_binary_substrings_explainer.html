Count Binary Substrings

LeetCode #696 — Two Pointers: Group Count Comparison
Easy Two Pointers Group Count Comparison O(n) Time · O(1) Space
1
Understanding the Problem
▼

Count substrings that have an equal number of consecutive 0s and 1s, where all 0s and 1s within the substring are grouped (no mixing). Examples: "01", "10", "0011", "1100", "000111".

💡 Key Insight

Valid substrings only occur at group boundaries. "000111" gives min(3,3)=3 valid substrings. Adjacent groups of sizes [a, b] contribute min(a, b) valid substrings. So reduce the problem to: count group sizes, sum min(adjacent pairs).

Example: s = "00110011" → groups [2,2,2,2] → min(2,2)+min(2,2)+min(2,2) = 6

2
The Algorithm: Sliding Group Counters
▼

Track curr (current group length) and prev (previous group length). On each group boundary, update prev=curr, reset curr=1. At every step: if curr ≤ prev, a new valid substring of length 2*curr ends here.

1prev=0, curr=1, result=0
2For i=1 to len-1: if s[i]==s[i-1] → curr++ else prev=curr, curr=1
3After updating: if curr ≤ prev → result++
🔑 Why curr ≤ prev?

When curr reaches some value k with prev≥k, there are k chars of group A and at least k chars of group B adjacent to them, forming a valid "BBB...AAA..." substring of length 2k ending right here. Each increment of curr discovers exactly one new valid substring (if prev is large enough).

The Code (Annotated)
▼
def countBinarySubstrings(s: str) -> int: prev, curr = 0, 1 # prev=previous group size, curr=current group size result = 0 for i in range(1, len(s)): if s[i] == s[i-1]: # same char as previous → extend group curr += 1 else: # group boundary crossed prev = curr # save current group as previous curr = 1 # start counting new group if curr <= prev: # current run fits within previous result += 1 # new valid substring of length 2*curr ends here return result
