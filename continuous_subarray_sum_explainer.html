LeetCode #523

Continuous Subarray Sum

Given an integer array nums and an integer k, return true if nums has a continuous subarray of size at least two whose elements sum up to a multiple of k.

Medium Prefix Sum Prefix Sum + Modulo + Hash Map Time: O(n) Space: O(k)
1 Understanding the Problem ▾

We have an array of non-negative integers and a divisor k. We need to find whether any contiguous subarray of length ≥ 2 has a sum that is a multiple of k (i.e., divisible by k, including 0).

💡 Key Insight

Sum of nums[i+1..j] is divisible by k if and only if prefix[j] % k == prefix[i] % k. So instead of checking every subarray's sum, we just track the remainders of prefix sums modulo k and look for a repeat at distance ≥ 2.

Edge cases to consider: k=0 (any two elements work only if their partial sum is 0 — Leetcode guarantees k≥1), arrays where the sum itself equals k or 0, subarrays starting from index 0 (need the sentinel seen={0: -1}), and arrays with all zeros (prefix sum is always 0, and any 2 elements form a valid subarray when k divides 0).

2 The Algorithm: Prefix Sum + Modulo + Hash Map ▾

The core technique exploits the modular arithmetic property: if two prefix sums share the same remainder when divided by k, the subarray between them has a sum divisible by k. We hash the first occurrence of each remainder to later check the subarray length.

1 Initialize: seen = {0: -1} — the remainder 0 is "seen" at a virtual index −1. This sentinel allows subarrays starting at index 0 to count as having length ≥ 2 when remainder 0 reappears at index ≥ 1.
2 Running sum & remainder: For each index i, add nums[i] to curr_sum, then compute rem = curr_sum % k. This is the remainder of the prefix sum prefix[0..i].
3 Lookup in hash map: If rem is in seen, check if i - seen[rem] >= 2. If yes, there exists a subarray from seen[rem]+1 to i with sum divisible by k and length ≥ 2 — return True.
4 Store only the first occurrence: If rem is NOT in seen, record it: seen[rem] = i. We must never overwrite an existing entry — the earlier the first occurrence, the larger the potential subarray, giving more chances to satisfy length ≥ 2.
5 Return False: If we exhaust the array without finding a valid pair, return False.
🔑 Why the Brute-Force Fails

A brute-force approach checks every pair (i, j) to see if sum(nums[i..j]) % k == 0. With a prefix sum array this is O(n²) time — too slow for n up to 10⁵. The modular remainder trick collapses the problem to a single O(n) pass because two equal remainders at distance ≥ 2 guarantee the answer is True.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every index i, seen[rem] holds the earliest index where the running prefix sum had remainder rem when divided by k. This means if we see rem again at index j, the subarray nums[seen[rem]+1 .. j] has sum divisible by k.

Proof of correctness using modular arithmetic:

Let P[i] = prefix sum up to index i. We want: P[j] - P[i] ≡ 0 (mod k) for some j - i ≥ 2. This is equivalent to P[j] ≡ P[i] (mod k). So we just need two indices with the same prefix sum remainder, separated by at least 2.

Why seen = {0: -1}? If the entire prefix sum up to index j is divisible by k, then P[j] % k == 0. We need a prior index i where P[i] % k == 0. The sentinel sets that prior index to −1, so the length is j - (-1) = j + 1 ≥ 2 as long as j ≥ 1.

⚠️ Critical: Never Overwrite seen[rem]

If you update seen[rem] = i every time you see remainder rem, you risk shrinking the potential gap. For example, if remainder 3 was first seen at index 1 and seen again at index 2, then at index 3, you must check against index 1 (gap = 2 ✓), not index 2 (gap = 1 ✗). Overwriting would lose the earliest occurrence and give a wrong answer.

Why length ≥ 2 specifically? The problem requires at least two elements. A single element's sum could trivially be divisible by k, but that doesn't count. By requiring j - seen[rem] ≥ 2, we ensure the subarray spans at least two indices.

4 The Code (Annotated) ▾

Solution 1 — Prefix Sum + Modulo + Hash Map (Optimal)

def checkSubarraySum(nums: list[int], k: int) -> bool:
seen = {0: -1} ← sentinel: remainder 0 seen at virtual index −1
curr_sum = 0 ← running prefix sum (we never build the full array)
for i, num in enumerate(nums):
curr_sum += num ← extend prefix sum to index i
rem = curr_sum % k ← remainder of this prefix sum mod k
if rem in seen: ← have we seen this remainder before?
if i - seen[rem] >= 2: ← yes — is the subarray long enough?
return True ← subarray nums[seen[rem]+1..i] is valid!
else:
seen[rem] = i ← first time seeing this remainder — record it
return False ← no valid subarray found
⚠️ Common Mistake: Updating seen[rem] unconditionally

Writing seen[rem] = i after the check (without the else) overwrites earlier occurrences and can both miss valid subarrays and produce wrong length checks. Always use else: seen[rem] = i — update only if not already stored.

Solution 2 — Brute Force O(n²) (for comparison)

def checkSubarraySum_brute(nums, k):
n = len(nums)
for i in range(n - 1): ← outer: start of subarray
total = nums[i]
for j in range(i + 1, n): ← inner: end of subarray (at least one step)
total += nums[j]
if total % k == 0: ← check every subarray sum
return True
return False ← O(n²) — too slow for n=10⁵
💡 Complexity Summary

Brute Force: Time O(n²), Space O(1) — checks every pair
Prefix Sum + Modulo + Hash Map: Time O(n), Space O(min(n, k)) — one pass, hash map has at most k distinct remainders
Space note: The hash map stores at most k distinct remainder values (0..k-1), so space is O(k) ≤ O(n).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

"Contiguous subarray with sum divisible by k" — this is a prefix sum + divisibility problem. Any time you see "subarray sum" combined with a divisibility constraint, think prefix sums and modular arithmetic. This is a twin of Subarray Sums Divisible by K (#974), but with the added length ≥ 2 constraint.

Step 2: Propose Brute Force

"The naive approach tries every pair (i, j) with j > i+1 and checks if the subarray sum is divisible by k. Using a prefix sum array this is O(n²). For n = 10⁵ this is too slow."

Step 3: State the Modular Key Observation

"If two prefix sums have the same remainder modulo k, the subarray between them has a sum divisible by k. So I only need to track the first occurrence of each remainder and check the gap. This runs in O(n)."

Step 4: Handle the Sentinel and the "Only Store First" Rule

"I initialize seen = {0: -1} to handle subarrays starting at index 0. I only write seen[rem] = i when rem is NOT already in seen — this preserves the earliest occurrence, which maximizes potential gap and avoids missing valid answers."

🗣️ What to Say to the Interviewer

"This is a prefix sum + modular arithmetic problem.
Key insight: prefix[j] % k == prefix[i] % k implies sum(i+1..j) is divisible by k.
I'll track remainders in a hash map with their FIRST occurrence index.
Initialize seen = {0: -1} as a sentinel for subarrays starting at index 0.
For each i: accumulate sum, compute rem = sum % k.
If rem in seen and i - seen[rem] >= 2: return True.
Else if rem not in seen: seen[rem] = i.
O(n) time, O(k) space — one pass, hash map has at most k entries."
