#28 · Find the Index of the First Occurrence in a String

Return the start index of needle inside haystack, or −1 if not found
Easy String Processing KMP Algorithm Time O(n+m) Space O(m)
1 Understanding the Problem ▼

We have a haystack (the big string, e.g. "sadbutsad") and a needle (the pattern we're searching for, e.g. "sad"). Return the index where the needle first appears in the haystack, or −1 if it doesn't appear at all.

💡 Key Insight

The naive approach slides a window of size m across the haystack and compares character by character. It works, but when we get a mismatch mid-comparison, we throw away all the information we gathered. KMP reuses that information by precomputing how far to jump after a mismatch — it never re-reads any character in the haystack.

Worked Example by Hand

haystack = "aabxaab", needle = "aab"

  • Try at index 0: compare "aab" vs "aab" → match! Return 0

haystack = "mississippi", needle = "issip"

  • Try at index 1: "issip" vs "issis" → mismatch at position 4 ('p' ≠ 's')
  • KMP knows that after seeing "issi" before the mismatch, we can jump smartly instead of sliding just one position

Key Variables

  • n = len(haystack), m = len(needle)
  • i = pointer in haystack (never goes backwards in KMP!)
  • j = pointer in needle (can jump back using the failure function)
  • lps[] = "longest proper prefix which is also suffix" — the precomputed jump table
2 The Algorithm: KMP Pattern Matching ▼

KMP runs in two phases: Phase 1 — build the LPS (failure function) array from the needle alone. Phase 2 — search the haystack using the LPS to skip redundant comparisons.

1 Build the LPS table — For each position k in the needle, lps[k] = length of the longest prefix of needle[0..k] that is also a suffix (but not the entire string). This tells us how far to reset j on a mismatch.
2 Initialize search pointers — i = 0 (haystack), j = 0 (needle). Both start at the beginning.
3 Compare characters — If haystack[i] == needle[j]: advance both i and j. If j == m: found! Return i − m.
4 On mismatch — If j > 0: set j = lps[j−1] (jump back, but don't advance i). If j == 0: just advance i. We never move i backwards.
5 End of haystack — If we exhaust the haystack without j == m, return −1.
🔑 What is the LPS Array?

For needle = "ababc":

Index k01234
needle[k]ababc
lps[k]00120

At index 3 (char 'b'), lps[3] = 2: the prefix "ab" == suffix "ab" within "abab". So on a mismatch after matching "abab", we didn't waste — we already matched "ab" from the front!

3 Why It Works: KMP Algorithm Deep-Dive ▼
🔐 The KMP Invariant

At every point during the search, j characters of the needle have been matched from position i − j in the haystack. The pointer i never moves backwards. When a mismatch occurs, we know the last lps[j−1] characters we matched are a valid prefix of the needle — so we restart matching from there instead of from scratch.

KMP Algorithm — Origin and Class

  • Origin: Knuth, Morris, and Pratt (1977). The canonical linear-time string matching algorithm.
  • Core invariant: i only moves forward; j resets to lps[j−1] on mismatch, retaining valid prefix matches
  • Why O(n+m): i advances at most n times. j can only increase as many times as i advances, so it can decrease at most n times total. Building LPS is O(m) by the same argument. Total: O(n+m).
  • Generalization: Multiple pattern search → Aho-Corasick (build a trie of patterns + failure links)
  • Recognize when: "Find pattern in text" + O(n) time or "avoid O(n·m)" → KMP

Why Brute Force is O(n·m)

The naive approach tries every starting position (n positions) and compares up to m characters for each. Worst case: "aaaa...a" with needle "aaa...ab" — matches m−1 chars then fails at every position. KMP avoids this by not resetting to position 0 blindly.

Edge Cases

  • needle = "" — Return 0 (empty needle always found at index 0)
  • needle longer than haystack — Return −1 immediately
  • Single character needle — LPS = [0]; pure character scan
  • Repeated pattern — LPS shines here: "aaaa" needle → large LPS values, O(n) with minimal resets

Solution 1 — Brute Force  (Good starting point)

def strStr(haystack: str, needle: str) -> int: n, m = len(haystack), len(needle) for i in range(n - m + 1): # only start where needle can still fit if haystack[i:i+m] == needle: # O(m) slice comparison return i return -1

Solution 2 — KMP  (Interview Pick)

def strStr(haystack: str, needle: str) -> int: n, m = len(haystack), len(needle) # ── Phase 1: Build LPS (failure function) ───────────────── lps = [0] * m length, k = 0, 1 # length = current prefix-suffix length while k < m: if needle[k] == needle[length]: length += 1 lps[k] = length k += 1 elif length > 0: length = lps[length - 1] # fall back, don't advance k else: lps[k] = 0 k += 1 # ── Phase 2: Search haystack ─────────────────────────────── i, j = 0, 0 # i = haystack pointer, j = needle pointer while i < n: if haystack[i] == needle[j]: i += 1; j += 1 if j == m: return i - m # full needle matched; start index elif i < n and haystack[i] != needle[j]: if j > 0: j = lps[j - 1] # jump j, NOT i else: i += 1 # no prefix match; just advance i return -1 # Solution 3 — Python built-in (know but don't rely on in interviews) # return haystack.find(needle)
⚠️ Common Mistake: Advancing i on mismatch when j > 0

When there's a mismatch and j > 0, you must set j = lps[j−1] but NOT advance i. The characters from i−lps[j−1] to i already match the beginning of the needle — moving i would skip an overlap.

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Clarify and State Brute Force

Start with the sliding window O(n·m) brute force. It's easy to implement and correct. Ask: "Can we do better?" — the interviewer will usually say yes, which opens KMP discussion.

Step 2: Identify the Waste in Brute Force

Draw on the whiteboard: "aaaxaaax" needle in "aaaxaaaxaaaaxaaax" haystack. After matching "aaax" at position 0 and failing, brute force throws away the "aaa" information. KMP doesn't — it reuses the prefix-suffix overlap in the needle.

Step 3: Introduce the LPS Table

Explain: "I'll preprocess the needle to compute lps[k] = length of longest proper prefix of needle[0..k] that is also a suffix. This is O(m) space and time, computed by running KMP on the needle itself." Build the table on the whiteboard for the needle.

Step 4: Execute the Search

Two pointers i (haystack) and j (needle). Match → advance both. Full match → return i−m. Mismatch with j>0 → j = lps[j−1] (don't move i). Mismatch with j=0 → advance i. Code is ~20 lines; very implementable under pressure.

🗣️ What to Say to the Interviewer

"The brute force is O(n*m) — for each position in haystack we compare up to m chars. The inefficiency is that after a partial match then mismatch, we discard work. KMP precomputes a 'failure function' on the needle: lps[k] = the length of the longest prefix of needle[0..k] that also appears as a suffix. This tells us, after a mismatch at needle[j], how far back to reset j without retreating in haystack. Phase 1 builds lps in O(m). Phase 2 scans haystack once in O(n). i never goes backwards — so total is O(n+m), space O(m) for the lps array."
🎯 Potential Follow-ups
  • "Find all occurrences, not just the first" — Continue after a match by setting j = lps[m−1] instead of returning
  • "Multiple patterns" — Aho-Corasick algorithm (trie + failure links); extends KMP naturally
  • "Case insensitive" — Lowercase both strings before KMP
  • "Rabin-Karp approach" — Rolling hash, O(n+m) average with O(nm) worst case; simpler to implement than KMP

7 Summary & Key Takeaways ▼
✓ Brute force O(n·m) is acceptable for small inputs; KMP is optimal at O(n+m)
✓ KMP's key idea: precompute the LPS array from the needle to avoid redundant comparisons
✓ On mismatch: jump j to lps[j−1] — do NOT move i backwards
✓ lps[k] = longest proper prefix of needle[0..k] that is also a suffix of that substring
✓ i only ever moves forward → O(n) for the search phase; building lps is O(m)
🎯 When to Use KMP
  • "Find pattern in text" with O(n) requirement
  • Text contains many repeated characters (naive fails badly here)
  • Must handle multiple searches with the same needle → build lps once, reuse
  • Keywords: "first occurrence", "index of substring", "strStr", "pattern search"
