LeetCode #15

3Sum

Given an integer array nums, return all triplets [nums[i], nums[j], nums[k]] such that i ≠ j, i ≠ k, j ≠ k, and nums[i] + nums[j] + nums[k] == 0.

Medium Two Pointers Sort + Fix One Time: O(n²) Space: O(1)
1 Understanding the Problem ▾

We need to find all unique triplets in the array that sum to zero. The two critical words here are "all" (there may be many valid triplets) and "unique" (no duplicate triplets in the output, even if duplicate numbers appear in the input).

💡 Key Insight

If we sort the array first, the two-sum problem on the remaining elements becomes trivially solvable with two opposing pointers moving inward. Sorting costs O(n log n) but the total complexity is still dominated by the O(n²) scan — and it gives us duplicate-skipping for free.

The challenge: This is not just finding one triplet — we must find ALL of them, and return no duplicates. Given nums = [-1, 0, 1, 2, -1, -4], the valid triplets are [-1, -1, 2] and [-1, 0, 1]. Note that although there are two -1s in the input, we only report the triplet [-1, 0, 1] once.

✓ Valid triplets for [-1, 0, 1, 2, -1, -4]

[-1, -1, 2] → sum = 0 ✓
[-1, 0, 1] → sum = 0 ✓
Output: [[-1,-1,2], [-1,0,1]]

✗ Common mistakes

[-1, 0, 1] reported twice → DUPLICATE
[0, 0, 0] when only one 0 → WRONG
Same index used twice → INVALID

Edge Cases: Array shorter than 3 (return []), all zeros (only [0,0,0] if three exist), no valid triplets, duplicate numbers scattered throughout the array.

2 The Algorithm: Sort + Fix One + Two Pointers ▾

The key idea is to reduce 3Sum to 2Sum. Fix one element nums[i], then find two more that sum to -nums[i] using the classic two-pointer convergence technique on the sorted remainder.

1 Sort the array. This enables two-pointer search and makes it easy to skip duplicate values with a simple while nums[i] == nums[i-1] guard. Cost: O(n log n).
2 Fix the first element (pivot): Iterate i from 0 to n−3. If nums[i] > 0, stop — a positive pivot means the sum of three non-decreasing values can never be zero. Skip duplicate pivots: if i > 0 and nums[i] == nums[i-1], continue.
3 Two-pointer scan: Set L = i+1 and R = n-1. While L < R: compute s = nums[i] + nums[L] + nums[R]. If s == 0, record the triplet. If s < 0, advance L (need larger sum). If s > 0, retreat R (need smaller sum).
4 Skip duplicates after a match: After finding a triplet, advance L while nums[L] == nums[L-1], and retreat R while nums[R] == nums[R+1]. Then move both pointers inward. This prevents re-reporting the same triplet with equal values.
🔑 Why Sorting Enables Two Pointers

In a sorted array, if nums[L] + nums[R] is too small, we know the only way to increase the sum is to move L right (to a larger value). If too large, move R left. This monotone property — that moving L right always increases and moving R left always decreases — is what makes O(n) per pivot possible, instead of O(n²) brute force inner loop.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any moment during the two-pointer phase, every valid triplet that uses nums[i] as the pivot and has both other elements in the range [L..R] has not yet been missed. As L and R converge, they cover all possible pairs without revisiting any.

Completeness: Every pair (j, k) with j < k satisfying nums[j] + nums[k] == -nums[i] will be found because the two pointers systematically cover all pairs in the window. When the sum is too small, L can only move right (we've already verified all smaller-j options with the current R). When too large, R moves left.

Uniqueness via sorting + skip: After sorting, all equal values are adjacent. The skip logic — if i > 0 and nums[i] == nums[i-1]: continue — ensures no two pivots with the same value are processed. Similarly, after recording a triplet, skipping equal L and R values ensures we don't record the same value combination twice.

⚠️ The Duplicate Pivot Trap

A subtle bug: the pivot skip guard is if i > 0 and nums[i] == nums[i-1], NOT if nums[i] == nums[i+1]. The latter would skip a pivot we haven't processed yet — for example, with [-1, -1, 2], both -1s must be tried as pivots to find the triplet [-1, -1, 2]. Only skip i if we've already processed an equal previous pivot.

Early termination: Since the array is sorted, once nums[i] > 0, the minimum possible sum (with i, L=i+1, R=i+2) is already positive. No further triplets summing to zero are possible — we can break the outer loop.

4 The Code (Annotated) ▾

Solution 1 — Sort + Two Pointers (Interview Pick, O(n²) time)

def threeSum(nums: List[int]) -> List[List[int]]:
nums.sort() ← Sort for two-pointer + duplicate skipping
res = []
for i in range(len(nums) - 2): ← Fix the first element (pivot)
if nums[i] > 0: break ← Sorted: if pivot > 0, sum can't be 0
if i > 0 and nums[i] == nums[i-1]: continue ← Skip duplicate pivots
L, R = i + 1, len(nums) - 1 ← Two pointers from both ends of remainder
while L < R: ← Converge until pointers meet
s = nums[i] + nums[L] + nums[R]
if s == 0: ← Found a valid triplet!
res.append([nums[i], nums[L], nums[R]])
while L < R and nums[L] == nums[L+1]: L += 1 ← Skip dup Ls
while L < R and nums[R] == nums[R-1]: R -= 1 ← Skip dup Rs
L += 1; R -= 1 ← Move both inward after recording
elif s < 0: L += 1 ← Sum too small: need larger left value
else: R -= 1 ← Sum too large: need smaller right value
return res
⚠️ Off-by-One in Skip Loop

The inner skip while L < R and nums[L] == nums[L+1]: L += 1 advances L past all equal values and stops at the last occurrence. Then L += 1 moves to the next distinct value. If you forget the final L += 1; R -= 1, you'll infinite-loop on a triplet found at L, R.

Solution 2 — Brute Force (O(n³), for comparison only)

def threeSum_brute(nums): ← Only for small inputs or interview clarity
n, res = len(nums), set()
for i in range(n): ← Try every triple
for j in range(i+1, n):
for k in range(j+1, n):
if nums[i]+nums[j]+nums[k] == 0:
res.add(tuple(sorted([nums[i],nums[j],nums[k]])))
return [list(t) for t in res] ← Set handles dedup but at cost of hashing
💡 Complexity Summary

Brute Force: Time O(n³), Space O(n) for the set — too slow for n=3000
Two Pointers: Time O(n²), Space O(1) extra — the standard interview answer
Hash Set (Two-Sum variant): Time O(n²), Space O(n) — same time, more space, harder to deduplicate

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

"We need triplets summing to a target. I know Two Sum with a hash map. Can I reduce 3Sum to Two Sum? Yes — fix one element, then solve Two Sum on the rest. But hash map deduplication is tricky. If I sort, Two Sum becomes two-pointer, and deduplication is clean."

Step 2: Propose Brute Force First

"Brute force: three nested loops, O(n³). Use a set to deduplicate. For n=3000, that's 27 billion operations — too slow. Let me optimize."

Step 3: Derive the Optimal

"Sorting takes O(n log n). After sorting, I fix nums[i] and run two-pointer on nums[i+1..n-1]. The two pointers converge in O(n). With n pivots, total is O(n²). Deduplication is just skipping adjacent equal values."

Step 4: Walk through the Dedup Logic Explicitly

"I must be careful about three levels of deduplication: (1) skip pivot if nums[i] == nums[i-1] and i > 0; (2) after a match, skip duplicate Ls; (3) after a match, skip duplicate Rs."

🗣️ What to Say to the Interviewer

"3Sum reduces to: fix one element, find two more summing to its negation.
I'll sort first — O(n log n) — which lets me use two pointers for the inner two-sum in O(n).
Outer loop iterates n times, inner two-pointer is O(n), so total is O(n²).
Deduplication: skip equal pivots, and skip equal L/R values after a match.
Space: O(1) extra (ignoring output). The sort is in-place."

  • "3Sum Closest?" → Same structure; track minimum delta instead of exact match (#16)
  • "4Sum?" → Add one more outer loop; same two-pointer inner (#18)
  • "What if no sorting allowed?" → Hash set approach; dedup via canonical sorted tuple

7 Summary & Key Takeaways ▾
Sort first — it enables two-pointer convergence and free deduplication
Reduce 3Sum → 2Sum: fix one element, two-pointer on the rest
Three places to skip duplicates: pivot, left pointer, right pointer
Early break when pivot > 0 (sorted array; sum can only grow)
O(n²) time is optimal for a problem that may output O(n²) triplets
