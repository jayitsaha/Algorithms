Reorganize String

LeetCode #767 – Rearrange so no two adjacent chars are the same

Medium Heap Greedy O(n log k) Time O(k) Space
1 Understanding the Problem ▼

Given a string, rearrange its characters so that no two adjacent characters are the same. Return any valid arrangement, or "" if impossible.

Example: "aab" → "aba" ✓  |  "aaab" → "" ✗

🎯 When Is It Impossible?

If any character appears more than ⌈n/2⌉ times (i.e., more than (n+1)/2), it's impossible. Why? In a string of length n, we have at most ⌈n/2⌉ "slots" at odd positions (0, 2, 4...). If one char needs more slots than that, we can't avoid adjacency.

Feasibility Check
max_freq ≤ (n + 1) / 2

For "aab" (n=3): max_freq('a')=2 ≤ (3+1)/2=2 ✓

For "aaab" (n=4): max_freq('a')=3 > (4+1)/2=2 ✗

💡 Key Insight

Greedily place the most frequent character that's different from the last placed one. A max-heap lets us always pick the highest-frequency character available.

2 The Algorithm: Max Heap + Alternating Place ▼

Use a max-heap ordered by frequency. At each step, pop the most frequent char, place it, then re-insert if count > 0. To avoid adjacency, keep a "cooldown" of the previously placed char.

1 Count frequencies of each character
2 Check feasibility: if max_freq > (n+1)/2, return ""
3 Build max-heap with (-count, char) pairs
4 While heap not empty: pop top char, append to result
5 Push back the previous char (if any count left) after placing the new one
⚠️ The "Cooldown" Trick

After placing char X, don't immediately re-insert it. Wait until after placing the next char. This ensures we never place X twice in a row.

3 Why It Works: The Greedy Invariant ▼
🔐 The Invariant

At each step, we place the highest-frequency available character. If the solution exists (max_freq ≤ (n+1)/2), there's always at least one character different from the last placed one that we can use.

Why does greedy work here?

  • We always reduce the count of high-frequency chars first
  • The cooldown mechanism ensures no immediate repeats
  • Since max_freq ≤ (n+1)/2, we never "run out" of alternative chars

Edge Case: When only one char type remains, it must have count=1 (otherwise we'd have failed the feasibility check).

4 The Code (Annotated) ▼
# Max Heap + Alternating Place
import heapq
from collections import Counter
def reorganizeString(s: str) -> str:
freq = Counter(s)
# Check feasibility
if max(freq.values()) > (len(s) + 1) // 2:
return ""
# Build max-heap (negate for max behavior)
heap = [(-cnt, ch) for ch, cnt in freq.items()]
heapq.heapify(heap)
result = []
prev = None # (count, char) on cooldown
while heap:
cnt, ch = heapq.heappop(heap)
result.append(ch)
# Push back previous if it still has count
if prev:
heapq.heappush(heap, prev)
prev = None
# Set current as next cooldown if count > 0
if cnt + 1 < 0: # Remember: negated
prev = (cnt + 1, ch)
return "".join(result)

5 🎤 Interview Strategy ▼
Step 1: Identify the Constraint

"No two adjacent same" → we need to alternate between different chars. High-frequency chars are the bottleneck.

Step 2: Determine Feasibility

State the condition: max_freq ≤ (n+1)/2. Explain why with the "slots" intuition.

Step 3: Choose Greedy + Heap

Greedily pick most frequent available char. Max-heap gives O(1) access to max. Cooldown prevents repeats.

🗣️ What to Say to the Interviewer

"First, I'll check feasibility: if any char appears more than (n+1)/2 times, it's impossible. Then I'll use a max-heap to greedily pick the most frequent character that's not the same as the last one placed. After placing a char, I delay re-inserting it until the next iteration to avoid adjacency. This gives O(n log k) time where k ≤ 26."
🎯 Follow-up Questions
  • Can we do O(n)? Yes, interleaving: fill even indices first with most frequent, then odd.
  • What about distance k apart? Use a queue of size k as cooldown instead of single prev.
