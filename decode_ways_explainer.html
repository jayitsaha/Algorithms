LeetCode #91

Decode Ways

A message is encoded using the mapping 'A'=1, 'B'=2, ..., 'Z'=26. Given a digit string, return the number of ways to decode it.

Medium Dynamic Programming Based on 1 or 2 Digits Time: O(n) Space: O(1)
1 Understanding the Problem ▾

Imagine a message encoded as digits: each letter 'A'–'Z' maps to 1–26. A string like "12" is ambiguous — it could be 1,2 → "AB", or 12 → "L". We want to count all valid interpretations.

💡 Key Insight

At each position, you make a binary decision: consume 1 digit or consume 2 digits. Both are valid if the resulting number maps to a letter. The total count is the sum of valid choices — classic 1D counting DP with Fibonacci structure.

✓ "226" → 3 ways

(2)(2)(6) → B,B,F
(22)(6) → V,F
(2)(26) → B,Z
Answer: 3

✗ "06" → 0 ways

'0' alone: no letter maps to 0
'06' as pair: 6, not 10–26
Leading '0' always kills decode
Answer: 0

Validity Rules:

  • Single digit valid: digit is 1–9 (non-zero). Maps to A–I.
  • Two-digit pair valid: forms a number 10–26. Maps to J–Z.
  • A standalone '0' is always invalid — there is no letter at position 0.
  • '0' can only appear as the second digit of a valid pair: "10" (J) or "20" (T).

Edge Cases: empty string (0), leading '0' (0), "10" (1 way — only as pair), "30" (0 ways — 30 > 26).

2 The Algorithm: DP on 1 or 2 Digits ▾

Define dp[i] = number of valid decodings of the first i characters. At each position we try consuming 1 digit or 2 digits, inheriting counts from valid predecessor states. This has the same structure as Fibonacci — each answer depends on at most 2 previous values.

📐 Recurrence Relations
dp[0] = 1 # empty prefix: 1 way (seed)
dp[1] = 0 if s[0]=='0' else 1

For i = 2..n:
  one = int(s[i-1])
  two = int(s[i-2:i])
  dp[i] = 0
  if 1 <= one <= 9: dp[i] += dp[i-1] # single digit
  if 10 <= two <= 26: dp[i] += dp[i-2] # two-digit pair
1 Guard clause: If s is empty or starts with '0', immediately return 0 — no valid decode possible.
2 Seed base cases: prev2 = dp[0] = 1 (empty decode), prev1 = dp[1] = 1 (first digit is valid, checked by guard).
3 Single digit check: If the current character s[i-1] is 1–9, add prev1 (dp[i-1]) to current count.
4 Two-digit check: If the two-character substring s[i-2:i] forms a number 10–26, add prev2 (dp[i-2]) to current count.
5 Slide window: Advance prev2 = prev1, prev1 = curr. After the loop, prev1 holds dp[n] — the answer.
⚠️ Why dp[0] = 1 and not 0?

dp[0] is a counting seed for the empty prefix. Without it, no two-digit pair can ever contribute: dp[2] += dp[0] would add 0 instead of 1. Think of it as: "there is exactly one way to decode the empty string — by decoding nothing."

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing position i, dp[i] exactly equals the number of valid decodings of s[0..i-1]. This invariant holds for all i from 0 to n, established by the base cases and maintained by the recurrence.

Why the recurrence is correct: Every valid decoding of s[0..i-1] ends with either a single-character letter or a two-character letter. These two cases are mutually exclusive (the last letter consumed 1 or 2 digits, not both). So:

  • Decodings ending with single digit s[i-1] (valid if 1–9): exactly dp[i-1] of them
  • Decodings ending with pair s[i-2..i-1] (valid if 10–26): exactly dp[i-2] of them

Summing disjoint sets gives the exact total. No overcount, no undercount.

Zero propagation — how dead ends are handled: If position i has neither a valid single digit nor a valid pair, dp[i] = 0. This zero propagates forward: once dp[i] = 0, any position that only has dp[i] as a predecessor also gets 0. The entire subtree of dead-end decodings collapses to zero automatically.

Example: "10" → 1 way

dp[0]=1, dp[1]=1
i=2: one='0' → invalid
two="10" → valid, +dp[0]=1
dp[2]=1 ✓ → only "J"

Example: "30" → 0 ways

dp[0]=1, dp[1]=1
i=2: one='0' → invalid
two="30"=30 > 26 → invalid
dp[2]=0 ✓ → no decode

4 The Code (Annotated) ▾

Solution 1 (Interview Pick) — Space-Optimized Bottom-Up DP

def numDecodings(s: str) -> int:
if not s or s[0] == '0': ← Guard: empty string or leading '0' = 0 ways
return 0
prev2, prev1 = 1, 1 ← dp[0]=1 (empty seed), dp[1]=1 (first char valid per guard)
for i in range(2, len(s) + 1): ← Process positions 2..n (1-indexed in dp)
curr = 0 ← Start fresh; any valid path will add to this
one = int(s[i-1]) ← The single digit at position i (0-indexed: s[i-1])
two = int(s[i-2:i]) ← The two-char number: s[i-2] and s[i-1] together
if 1 <= one <= 9: ← Non-zero single digit maps to A–I
curr += prev1 ← Add all decodings of the first (i-1) chars
if 10 <= two <= 26: ← Two-digit number maps to J (10) through Z (26)
curr += prev2 ← Add all decodings of the first (i-2) chars
prev2, prev1 = prev1, curr ← Slide the rolling window one step forward
return prev1 ← dp[n] = total decodings of the full string
⚠️ Common Mistake: "27" looks like it should be valid

27 is NOT a valid letter (only 1–26 are). The check two <= 26 is critical. Similarly, "07" has two=7, which is less than 10 — the condition 10 <= two rejects it. Both boundary checks together are necessary.

Solution 2 — Top-Down Memoization (Natural Recursion)

def numDecodings(s: str) -> int:
from functools import lru_cache
@lru_cache(maxsize=None)
def dp(i): ← Returns # ways to decode s[i:]
if i == len(s): return 1← All digits consumed: valid complete decode
if s[i] == '0': return 0← Leading '0' in this suffix: dead end
ways = dp(i + 1)← Always try single-digit (valid since s[i]!='0')
if i+1 < len(s) and int(s[i:i+2]) <= 26:
ways += dp(i + 2)← Add two-digit decode if pair is 10–26
return ways
return dp(0)
📊 Complexity

Bottom-Up DP: O(n) time, O(1) space (rolling two variables).
Memoization: O(n) time, O(n) space (memo cache + call stack).
Both visit each position exactly once. Bottom-up is preferred in interviews for its constant space.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize "count the ways" → DP

The problem asks how many ways — not which way or best way. Counting problems almost always use DP. The moment you see "number of ways to decode/parse/split," reach for 1D counting DP.

Step 2: Identify the Fibonacci structure

At each position you consume either 1 or 2 digits. So the answer at position i depends on i-1 and i-2. This is Fibonacci DP — the same structural pattern as Climbing Stairs (#70), just with validity conditions on each step.

Step 3: Define state and recurrence explicitly

State: dp[i] = decodings of first i chars. Recurrence: add dp[i-1] if single digit valid, add dp[i-2] if pair valid. Base: dp[0]=1, dp[1]=0 or 1. Answer: dp[n].

Step 4: Optimize space to O(1)

Since only two previous values are needed, replace the array with prev2, prev1. Mention this proactively — it shows you think beyond correctness to efficiency.

🗣️ What to Say to the Interviewer

"This is a counting problem — I need the total number of valid decodings, not a specific one. Since at each position I can consume 1 or 2 digits, the count at position i depends on i-1 and i-2. That's a Fibonacci-shaped DP.

I'll define dp[i] as the number of ways to decode the first i characters. A single digit adds dp[i-1] if it's 1–9. A two-digit pair adds dp[i-2] if it's between 10 and 26. Base cases are dp[0]=1 (empty) and dp[1]=1 if the first char isn't zero.

Since I only need two previous values, I'll use O(1) space with two rolling variables. Edge cases: leading zero returns 0 immediately. Any '0' digit kills single-digit decode; only '10' or '20' can contain a '0' as part of a valid pair."
🎯 Pattern Recognition Signals
  • "How many ways to decode/parse a string?" → 1D counting DP
  • "At each step, consume 1 or 2 elements" → Fibonacci DP (dp[i] from dp[i-1] + dp[i-2])
  • Digit string with validity constraints per position → check boundaries before adding
  • Follow-up: "Wildcard '*' digit" → extension to Decode Ways II (#639)

7 Summary & Key Takeaways ▾
dp[i] = # decodings of first i characters — the invariant at every position
Two-branch recurrence: single digit (1–9) → add dp[i-1]; pair (10–26) → add dp[i-2]
dp[0] = 1 is the indispensable seed — the "empty decode" that lets two-digit paths start
Only prev2 and prev1 are needed → O(1) space via rolling variables
Zero propagates naturally: a '0' that can't be paired kills that branch automatically
