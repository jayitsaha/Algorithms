LeetCode #104

Maximum Depth of Binary Tree

Given the root of a binary tree, return its maximum depth — the number of nodes along the longest root-to-leaf path.

Easy Trees DFS Postorder Time: O(n) Space: O(h)
1 Understanding the Problem ▾

The depth of a binary tree is the length of the longest root-to-leaf path, counted in number of nodes (not edges). The root itself is at depth 1. A leaf is a node with no children. An empty tree (null root) has depth 0.

💡 Key Insight

The depth of a tree rooted at any node N is exactly 1 + max(depth of left subtree, depth of right subtree). This recursive definition perfectly mirrors the structure of the tree: each node trusts its children to report their depths, then adds 1 for itself.

Example: [3, 9, 20, null, null, 15, 7] → Depth = 3
Answer = 3

Path 1: 3 → 9 (depth 2)
Path 2: 3 → 20 → 15 (depth 3)
Path 3: 3 → 20 → 7 (depth 3)
Maximum = 3

Edge Cases

root = null → return 0
Single node → return 1
Left-only chain [1,2,3] → 3
All on same level → 1

The problem is elegant once you spot the recursive structure. Every node asks the same question: "What is the longest path from me to a leaf?" — and the answer is always 1 + max(left_depth, right_depth). This recurrence IS the solution.

2 The Algorithm: Recursive DFS (Postorder) ▾

We use a postorder DFS traversal — Left subtree first, then Right subtree, then the current node. Each node first recurses into both children to collect their depths, then takes the max and adds 1. The computation flows bottom-up: leaves report their depth first, and results bubble up to the root.

1 Base case — null node: If root is None, return 0. An empty subtree has no nodes and therefore depth 0. This also handles the edge case of an empty tree.
2 Recurse left: Call maxDepth(root.left). This recursively computes the maximum depth of the entire left subtree and returns it as an integer.
3 Recurse right: Call maxDepth(root.right). Same idea — compute the maximum depth of the right subtree.
4 Combine: Return 1 + max(left_depth, right_depth). The 1 counts the current node. max() picks whichever child subtree is deeper — that determines the longest path through us.
Why Postorder?

Postorder processes children before the parent. This is the correct traversal order whenever a parent's answer depends on its children — depth, size, subtree sum, balance factor. Compare to preorder (parent first), which is used for top-down problems like "find all root-to-leaf paths." The traversal order determines whether data flows top-down or bottom-up.

There is also an iterative BFS approach: perform a level-order traversal and count the number of levels. When the queue is empty, the level count equals the maximum depth. BFS uses O(width) space, DFS uses O(height) stack space.

3 Why It Works: The Invariant ▾
The Invariant

At every node N, the function maxDepth(N) returns the exact maximum number of nodes on any root-to-leaf path within the subtree rooted at N.

Proof by structural induction on the tree:

BASE Null node: An empty subtree has zero nodes, so depth = 0. Correct — the function returns 0.
IND. Non-null node: Assume maxDepth(L) and maxDepth(R) are correct for the left child L and right child R. The longest path from node N to any leaf either goes through L (length = 1 + maxDepth(L)) or through R (length = 1 + maxDepth(R)). The maximum of these two is 1 + max(maxDepth(L), maxDepth(R)). This is what the function returns. Correct.
Common Pitfall: Edges vs Nodes

LeetCode #104 counts depth in nodes. A single-node tree has depth 1. If asked for depth in edges instead, the base case for a leaf returns 0 (not 1), and null returns -1. Always confirm the definition before writing the base case.

Minimum Depth Is Different!

Problem #111 asks for minimum depth. You cannot just swap max for min here. A node with only a left child has minimum depth 1 + minDepth(left), NOT min(1+0, 1+min(right)) — because a null branch is not a valid leaf path. The minimum depth must go to an actual leaf node.

4 The Code (Annotated) ▾

Solution 1 — Recursive DFS Postorder (Interview Pick)

def maxDepth(root: Optional[TreeNode]) -> int:
if root is None: ← base case: empty subtree = depth 0
return 0 ← also handles empty tree input
left = maxDepth(root.left) ← recurse: depth of left subtree
right = maxDepth(root.right) ← recurse: depth of right subtree
return 1 + max(left, right) ← +1 for this node; take deeper side

Solution 2 — Iterative BFS Level-Order

from collections import deque
def maxDepth(root: Optional[TreeNode]) -> int:
if not root: return 0 ← guard: empty tree
queue, depth = deque([root]), 0 ← init queue with root; depth counter starts at 0
while queue: ← process one level per outer loop iteration
for _ in range(len(queue)): ← snapshot current level size before adding children
node = queue.popleft()
if node.left: queue.append(node.left) ← enqueue left child if it exists
if node.right: queue.append(node.right) ← enqueue right child if it exists
depth += 1 ← entire level processed: depth goes up by 1
return depth ← levels counted = maximum depth
Complexity Summary

Time O(n): Both approaches visit every node exactly once.
Space O(h) — DFS: Recursion stack grows to the tree height. Balanced tree: O(log n). Skewed (degenerate) tree: O(n).
Space O(w) — BFS: Queue holds at most one level at a time. For a complete binary tree, the bottom level has ~n/2 nodes → O(n). BFS uses more space for wide, shallow trees; DFS uses more for deep, narrow ones.

5 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Tree + Bottom-Up Pattern

Any tree problem where a node's answer depends on its children → postorder DFS. Signals: "depth", "height", "maximum path", "size of subtree", "is this tree balanced?". You'll compute answers at leaves and bubble results up to the root.

Step 2: Write the Recurrence First

Say it out loud before coding: "The depth of a tree is 1 plus the depth of the deeper subtree. The depth of an empty tree is 0." That English sentence maps 1:1 to the code. Writing the recurrence first makes the base case obvious.

Step 3: Handle Null Base Case Explicitly

Every recursive tree function starts with if not root: return <identity>. For depth the identity is 0. For sum it's 0. For product it's 1. For boolean existence checks it's False. State this out loud — it shows you understand termination.

Step 4: Mention the BFS Alternative

Proactively mention: "An iterative BFS counts levels as nodes are dequeued — this avoids Python's recursion depth limit for very deep trees. The trade-off is O(width) space vs O(height) for DFS." This shows depth of knowledge beyond the obvious solution.

What to Say to the Interviewer

"Depth is a property that depends on subtrees, so I'll use postorder DFS. Base case: null node returns 0 — empty subtree has no depth. For each non-null node, I recursively get left and right subtree depths, then return 1 plus the max of those two. The 1 accounts for the current node; max picks the deeper branch. This is O(n) time — every node visited once — and O(h) space for the call stack, where h is the height. For a balanced tree that's O(log n); degenerate case is O(n). If we're worried about recursion limits, I can switch to iterative BFS and count levels. Want me to show that version too?"
Follow-Up Questions to Anticipate
  • "N-ary tree?" → Same logic: 1 + max(maxDepth(c) for c in node.children)
  • "Minimum depth?" → Different! Must reach an actual leaf. Need to handle single-child nodes specially
  • "Is it balanced?" → At every node: if |left - right| > 1, not balanced. Return -1 as a sentinel to short-circuit
  • "Diameter of tree?" → At each node: candidate = left_depth + right_depth. Track global max
  • "Maximum path sum?" → Same shape but with values; tricky because negatives can make it suboptimal to extend

7 Summary & Key Takeaways ▾
The recurrence 1 + max(left_depth, right_depth) IS the solution — writing it in English first makes the code obvious.
Postorder DFS (children before parent) is correct for any bottom-up tree aggregation.
Base case: null node returns 0. Without this, the recursion has no termination condition.
Space is O(h) — tree height. Balanced: O(log n). Degenerate (one big chain): O(n).
BFS alternative counts levels; useful when recursion depth is a concern but uses O(width) space.
Minimum depth (#111) requires different logic — must reach an actual leaf, not just a null branch.
When to Use DFS Postorder on Trees
  • Computing bottom-up aggregates: height, subtree size, sum of descendants
  • Structural checks: is it balanced? is it symmetric? is it a valid BST?
  • Path-through problems: diameter (left_h + right_h at node), max path sum
  • Transformation: invert tree (swap children after recursing), flatten to list
  • Pattern signal: "parent's value depends on children's values"
