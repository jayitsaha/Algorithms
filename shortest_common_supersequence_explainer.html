LeetCode #1092

Shortest Common Supersequence

Given two strings str1 and str2, return the shortest string that has both str1 and str2 as subsequences. If there are multiple valid strings, return any of them.

Hard Dynamic Programming LCS + Reconstruction Time: O(m·n) Space: O(m·n)
1 Understanding the Problem ▾

A supersequence of two strings is any string from which both originals can be obtained by deleting some characters (preserving relative order). We want the shortest possible one.

💡 Key Insight

Characters that appear in both strings can serve double duty — written once in the supersequence, they satisfy both str1 and str2 simultaneously. Characters unique to one string must appear separately. Therefore: shortest SCS length = len(str1) + len(str2) − LCS_length. But returning the actual string (not just its length) requires careful reconstruction.

Example: str1="abac", str2="cab"

LCS("abac","cab") = "ab"
LCS length = 2
SCS length = 4+3−2 = 5
Answer: "cabac"
"abac" ⊆ "cabac" ✓
"cab" ⊆ "cabac" ✓

Why LCS chars get reused

str1: a b a c
str2: c a b
LCS: a b ← shared!

SCS merges them:
c a b a c
↑ "c" only in str2
↑ "ab" shared
↑ "ac" only in str1

Edge Cases: Empty str1 → SCS = str2. Empty str2 → SCS = str1. Identical strings → SCS = either. No common chars (LCS length 0) → SCS = str1 + str2.

2 The Algorithm: LCS DP + Backtracking Reconstruction ▾

The solution has two distinct phases. Phase 1: fill a 2D DP table to find the Longest Common Subsequence. Phase 2: backtrack through that table to reconstruct the actual supersequence character by character.

1 Build LCS table: Create a 2D array dp[i][j] = length of LCS of str1[:i] and str2[:j]. Dimensions: (m+1) × (n+1). Row 0 and column 0 are all zeros (base cases).
2 Fill recurrence: For each cell (i,j): if str1[i-1] == str2[j-1] → dp[i][j] = dp[i-1][j-1] + 1. Else → dp[i][j] = max(dp[i-1][j], dp[i][j-1]).
3 Set up reconstruction: Start pointers i = m, j = n (bottom-right of table). Build result list in reverse — we walk backward and flip at the end.
4 Match → shared character: If str1[i-1] == str2[j-1], this is part of the LCS — write it once. Advance both i and j backward.
5 Mismatch → follow origin: If dp[i-1][j] ≥ dp[i][j-1], we "came from above" — include str1[i-1] and decrement i. Otherwise include str2[j-1] and decrement j.
6 Drain leftovers: When one pointer hits 0, append remaining characters from the other string. Reverse the entire list for the final answer.
📐 The Recurrences
dp[i][j] = dp[i-1][j-1] + 1 if str1[i-1] == str2[j-1]
dp[i][j] = max(dp[i-1][j], dp[i][j-1]) otherwise

SCS_length = m + n − dp[m][n]
3 Why It Works: The DP Invariant ▾
🔐 The Invariant

dp[i][j] always holds the exact length of the longest common subsequence of the first i characters of str1 and the first j characters of str2. This is established row by row — each cell depends only on already-computed cells, so induction over (i, j) guarantees correctness.

Why DP is appropriate here — two hallmarks:

  • Optimal Substructure: LCS(str1[:i], str2[:j]) derives from at most three smaller subproblems: LCS(str1[:i-1], str2[:j-1]), LCS(str1[:i-1], str2[:j]), LCS(str1[:i], str2[:j-1]).
  • Overlapping Subproblems: A naive recursive LCS would recompute the same (i,j) pairs exponentially. The DP table caches each in O(1) time after it is first computed.

Why reconstruction works: During backtracking, at every cell we know exactly which of the three transitions was used to produce it. A match (diagonal) means that character is in the LCS — include once for both. A mismatch means one string's pointer is "ahead" in the LCS sense — that character must appear in the SCS from that string alone.

⚠️ Common Confusion: LCS vs SCS

LCS maximizes what can be shared. SCS must include everything not in the LCS, too. Every character in str1 and str2 must appear in the SCS — the LCS just tells us which ones we can "merge" (write once) vs. which we must "keep separate" (write once per original string).

Why build in reverse? Backtracking starts at (m, n) and moves toward (0, 0), which means we encounter the final characters first. Prepending to a string is O(n) each time; instead we append to a list and reverse once at the end — standard O(n) trick for reconstruction problems.

4 The Code (Annotated) ▾

Solution 1 — LCS DP + Backtracking Reconstruction (Interview Pick)

def shortestCommonSupersequence(str1: str, str2: str) -> str:
m, n = len(str1), len(str2) ← store lengths once
# ─── Phase 1: Build LCS DP table ───
dp = [[0]*(n+1) for _ in range(m+1)] ← (m+1)×(n+1); row 0 / col 0 = 0 base cases
for i in range(1, m+1): ← iterate str1 prefixes
for j in range(1, n+1): ← iterate str2 prefixes
if str1[i-1] == str2[j-1]: ← characters match
dp[i][j] = dp[i-1][j-1] + 1 ← extend LCS diagonally
else: ← characters differ
dp[i][j] = max(dp[i-1][j], dp[i][j-1]) ← skip one char from either string
# ─── Phase 2: Reconstruct SCS by backtracking ───
result = []
i, j = m, n ← start at bottom-right (full strings)
while i > 0 and j > 0:
if str1[i-1] == str2[j-1]: ← LCS character: write once for both
result.append(str1[i-1])
i -= 1; j -= 1 ← both pointers advance
elif dp[i-1][j] >= dp[i][j-1]: ← came from above → str1's char is unique here
result.append(str1[i-1])
i -= 1
else: ← came from left → str2's char is unique here
result.append(str2[j-1])
j -= 1
result.extend(str1[:i]) ← drain remaining str1 chars (if j hit 0 first)
result.extend(str2[:j]) ← drain remaining str2 chars (if i hit 0 first)
return ''.join(reversed(result)) ← we built it backward; flip for final answer
⚠️ Why >= not > in the mismatch branch?

When dp[i-1][j] == dp[i][j-1] (a tie), both directions lead to equally long LCS paths. Choosing >= breaks ties by preferring str1; choosing > would prefer str2. Either is correct — the resulting SCS will be the same length. Just be consistent.

💡 Complexity Summary

Time: O(m·n) for Phase 1 + O(m+n) for Phase 2 = O(m·n) overall
Space: O(m·n) for the DP table. Reconstruction requires the full table — you cannot space-optimize with rolling rows and still backtrack.

Solution 2 — Naive Concatenation (Brute Force Baseline)

def shortestCommonSupersequence_naive(str1, str2):
return str1 + str2 ← trivially valid (str1 is prefix, str2 is suffix) but length m+n — never optimal
5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the DP Pattern

"Two strings, asking for shortest/longest something involving subsequences of both" — this is the canonical signal for 2D DP on string prefixes. Ask: length only, or actual string? This asks for the actual string, which requires reconstruction.

Step 2: Reduce to LCS

"To minimize the supersequence, I want to maximize what I can share between the two strings. That's precisely the LCS. So the length is m + n − LCS_length. I'll build the LCS DP table first."

Step 3: Explain the LCS Table

"dp[i][j] = LCS length of str1[:i] and str2[:j]. Base cases: row 0 and col 0 are zero. Recurrence: match → diagonal+1; mismatch → max(up, left). Fill in O(m·n)."

Step 4: Describe Reconstruction

"To build the actual SCS, I backtrack from dp[m][n]. If characters match, include once (it's from the LCS). If they don't, I follow the direction of the larger neighbor — that character belongs to just one string. Drain leftovers and reverse."

🗣️ What to Say to the Interviewer

"Two strings, asking for shortest supersequence — classic 2D DP.
Key insight: SCS_length = m + n − LCS_length. LCS characters can be shared.
Phase 1: Fill the LCS DP table in O(m·n).
Phase 2: Backtrack from dp[m][n] → match means 'write once for both',
mismatch means 'write from the string we came from'.
Drain leftovers, reverse. Total: O(m·n) time, O(m·n) space."
🔮 Likely Follow-up Questions
  • "What's the minimum length?" → m + n − dp[m][n]. Can compute in O(min(m,n)) space with rolling rows (but lose reconstruction path).
  • "How does this relate to Edit Distance?" → Edit Distance = (m − LCS) + (n − LCS) = m + n − 2·LCS. Same table, different formula.
  • "Can you do it for 3 strings?" → 3D DP with O(m·n·p) time/space. General k strings is NP-hard.
  • "Are there multiple valid answers?" → Yes, whenever there are ties in the mismatch branch. Any is acceptable.

dp = [[0]*(n+1) for _ in range(m+1)]
for i in range(1, m+1):
for j in range(1, n+1):
if str1[i-1] == str2[j-1]:
dp[i][j] = dp[i-1][j-1] + 1
else:
dp[i][j] = max(dp[i-1][j], dp[i][j-1])
result = []; i, j = m, n
while i > 0 and j > 0:

7 Summary & Key Takeaways ▾
SCS length = m + n − LCS_length: minimize by maximizing shared characters
LCS DP table: match → diagonal + 1, mismatch → max(up, left)
Reconstruction backtracks: match → write once (LCS char), mismatch → take from the "origin" string
Build result reversed (append to list), flip once at the end for O(m+n) reconstruction
Reconstruction requires full O(m·n) table — cannot space-optimize and still trace the path
