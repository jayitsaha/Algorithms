LeetCode #1091

Shortest Path in Binary Matrix

Given an n×n binary grid, return the length of the shortest clear path from top-left (0,0) to bottom-right (n-1,n-1) using 8-directional movement through 0-cells.

Medium Graph · BFS 8 Directions Time: O(n²) Space: O(n²)
1 Understanding the Problem ▾

You're given an n × n binary matrix where 0 = clear cell and 1 = obstacle. Starting from (0, 0) (top-left), reach (n-1, n-1) (bottom-right) moving only through clear (0) cells. Unlike typical grid problems, you can move in all 8 directions — including diagonals.

The path length counts every cell you step on, including both start and end. Return the minimum such length, or -1 if no path exists.

💡 Key Insight

Every move costs exactly 1, so this is an unweighted shortest path problem. BFS is tailor-made for unweighted shortest paths: it expands outward in rings of equal distance. The first time BFS reaches the goal is provably the shortest route — no path of fewer steps was missed.

  • Fully blocked corridor: BFS queue empties → return -1
2 The Algorithm: BFS with 8-Directional Expansion ▾

BFS (Breadth-First Search) processes cells level by level. Every cell at distance d is fully explored before any cell at distance d+1. This layer-by-layer guarantee makes BFS the natural choice for shortest path in unweighted graphs.

1 Guard clause: If grid[0][0] == 1 or grid[n-1][n-1] == 1, return -1 — start or end is blocked. No BFS needed.
2 Initialize: Add (row=0, col=0, dist=1) to a deque. Mark grid[0][0] = 1 to indicate "visited." Handle 1×1 grid as a special case (return 1).
3 BFS loop: While queue is non-empty, dequeue the front cell (r, c, dist). Try all 8 neighbor directions (r+dr, c+dc).
4 Validate neighbor: Check it's in bounds (0 ≤ nr < n, 0 ≤ nc < n) and unvisited (grid[nr][nc] == 0). If valid: check for goal, then mark visited and enqueue.
5 Goal check: If a valid neighbor is (n-1, n-1), return dist + 1. If the queue empties without finding the goal, return -1.
🔑 Smart Visited Marking — Reuse the Grid

Instead of maintaining a separate visited set (which costs O(n²) extra memory), we flip visited cells from 0 to 1. Since blocked cells are already 1, this naturally merges "blocked" and "visited" — the BFS condition grid[nr][nc] == 0 filters both. This is a standard space optimization for binary grid BFS.

The 8 direction delta pairs (all combinations of dr, dc ∈ {-1, 0, 1} except (0,0)):

DIRS = [(-1,-1),(-1,0),(-1,1),← NW, N, NE
( 0,-1), ( 0,1),← W, E (no (0,0))
( 1,-1),( 1,0),( 1,1)]← SW, S, SE
3 Why It Works: The BFS Shortest-Path Invariant ▾
🔐 The Invariant

When BFS dequeues cell X, every cell reachable in fewer steps than X has already been fully processed. Consequently: the first time BFS reaches the destination, it has found the shortest path. No alternative route of equal or lesser length was skipped — BFS guarantees complete coverage at each distance level before advancing.

Why not DFS? DFS might traverse a long winding path before trying the short diagonal. It finds a path but cannot guarantee it is the shortest without exhaustively checking all paths — exponential time. BFS avoids this because its queue enforces distance ordering.

Why mark at enqueue, not at dequeue? If we marked "visited" only when dequeuing, the same cell could be enqueued multiple times (once from each neighbor that discovered it). This doesn't break correctness but inflates the queue from O(n²) to O(n² × 8) entries and wastes significant time. Marking at enqueue ensures each cell enters the queue exactly once.

⚠️ Common Mistake: Checking Goal When Dequeuing vs Enqueuing

If you check if r == n-1 and c == n-1: return dist when dequeuing, you must initialize dist=1 and return dist. If you check when enqueuing a neighbor, return dist+1. Either approach works — just be consistent. The walkthrough below uses the "check at enqueue" approach. The 1×1 edge case must be handled separately in both approaches.

Time Complexity: O(n²) — Each of the n² cells is enqueued and dequeued at most once. For each we check 8 neighbors → O(8n²) = O(n²).

Space Complexity: O(n²) — The queue in the worst case holds O(n²) cells (e.g., the entire open frontier). The visited marking is O(1) extra since we reuse the grid.

4 The Code (Annotated) ▾

Solution 1 — BFS (Optimal · Interview Pick) · O(n²) time, O(n²) space

from collections import deque← O(1) popleft; list.pop(0) is O(n)
def shortestPathBinaryMatrix(grid):
n = len(grid) ← grid is n×n
if grid[0][0] == 1 or grid[n-1][n-1]==1: ← start or end blocked
return -1
if n == 1: return 1 ← 1×1: start IS the goal
DIRS = [(-1,-1),(-1,0),(-1,1),(0,-1), ← 8 direction deltas
(0,1),(1,-1),(1,0),(1,1)]
queue = deque([(0, 0, 1)]) ← (row, col, distance=1)
grid[0][0] = 1 ← mark start visited
while queue: ← BFS main loop
r, c, dist = queue.popleft() ← FIFO: nearest first
for dr, dc in DIRS: ← try all 8 directions
nr, nc = r + dr, c + dc
if 0 <= nr < n and 0 <= nc < n and grid[nr][nc] == 0:
if nr == n-1 and nc == n-1: ← goal reached!
return dist + 1 ← +1 for this final step
grid[nr][nc] = 1 ← mark visited at enqueue
queue.append((nr, nc, dist+1))← enqueue with updated dist
return -1 ← queue empty, no path
⚠️ Why deque, not list?

Python's list.pop(0) is O(n) — it shifts every remaining element left. collections.deque.popleft() is O(1) with a doubly-linked list. For BFS on an n×n grid, you may enqueue up to n² cells. Using list gives O(n³) total for BFS — a silent performance killer. Always use deque for Python BFS.

Solution 2 — BFS (Alternative: check goal at dequeue)

def shortestPathBinaryMatrix(grid):
n = len(grid)
if grid[0][0] == 1 or grid[n-1][n-1] == 1: return -1
q = deque([(0, 0, 1)]); grid[0][0] = 1
while q:
r, c, dist = q.popleft()
if r == n-1 and c == n-1: return dist ← goal check at dequeue
for dr in (-1,0,1):
for dc in (-1,0,1):
if dr==0 and dc==0: continue
nr, nc = r+dr, c+dc
if 0<=nr<n and 0<=nc<n and grid[nr][nc]==0:
grid[nr][nc]=1; q.append((nr,nc,dist+1))
return -1
💡 Complexity Summary

BFS (both variants): Time O(n²), Space O(n²) — visit each cell once
DFS (brute force all paths): O(8^(n²)) — exponential, infeasible
A* search: O(n² log n) with heap — identical worst case, overkill for equal-cost moves

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Pattern Recognition

The signals: "grid" + "shortest path" + "unweighted moves" = BFS. The twist "8 directions" vs. the standard 4 is trivial — just add 4 more entries to the DIRS list. Naming this out loud shows the interviewer you have a systematic pattern vocabulary.

Step 2: Defensive Edge Cases First

Before writing BFS, check start and end cells. This is O(1) and prevents the BFS from running pointlessly. Also call out the 1×1 edge case — interviewers appreciate catching boundary conditions proactively.

Step 3: Encode 8 Directions Explicitly

Write the DIRS constant as a flat list of 8 tuples. Some candidates try to generate it with nested loops — fine, but a flat list is clearer in an interview. Explicitly listing the 8 directions demonstrates you thought through all cases including the 4 diagonals.

Step 4: Mention the Grid-as-Visited Optimization

Proactively say: "I'll reuse the grid as a visited array by flipping 0s to 1s when I enqueue. This avoids O(n²) extra space for a separate visited set." This shows you think about trade-offs and memory efficiency.

🗣️ What to Say to the Interviewer

"This is a shortest path on an unweighted grid — BFS is the right tool.
First, guard: if start or end is blocked, return -1 immediately.
I'll define 8 direction vectors covering horizontal, vertical, and diagonal moves.
BFS from (0,0) with initial distance 1. For each dequeued cell, try all 8 neighbors.
If a neighbor is in-bounds and equals 0 (open), mark it 1 (visited) and enqueue with dist+1.
If the neighbor is the goal (n-1,n-1), return dist+1 — that's the shortest path.
If the queue empties without reaching the goal, return -1.
Time: O(n²) since we visit each cell once. Space: O(n²) for the queue."
🔮 Likely Follow-up Questions
  • "What if cells have different traversal costs?" → Replace BFS with Dijkstra (min-heap) — O(n² log n)
  • "What if you can break up to k walls?" → BFS with 3D state (r, c, k_remaining) — see LC #1293
  • "Can we improve worst-case beyond O(n²)?" → No — every cell must potentially be checked; O(n²) is tight
  • "Bidirectional BFS?" → BFS from both start and end simultaneously; meets in the middle, halves the search radius in practice

7 Summary & Key Takeaways ▾
BFS guarantees shortest path in unweighted graphs — DFS cannot
8-directional BFS is identical to 4-directional — just use 8 direction tuples
Mark cells visited at enqueue time to prevent duplicate queue entries
Reuse the grid as a visited array — avoids an extra O(n²) set
Guard start/end before BFS — saves work on impossible inputs
Use deque for O(1) popleft — list.pop(0) silently makes BFS O(n³)
ApproachTimeSpaceNotes
DFS (all paths) O(8^(n²)) O(n²) Finds a path, not the shortest
BFS ✓ (Optimal) O(n²) O(n²) Shortest path guaranteed; simplest correct approach
A* (with heuristic) O(n² log n) O(n²) Faster in practice on large sparse grids; overkill here
🎯 When to Recognize the BFS 8-Directions Pattern
  • "Shortest path" + "grid" + uniform move cost → BFS
  • "8 directions" / "any adjacent cell including diagonals" → extend DIRS to 8 tuples
  • "Binary matrix (0/1)" where 0 = open → BFS with grid[nr][nc]==0 condition
  • Multiple starting points → multi-source BFS (enqueue all sources at dist=0)
  • State has extra dimension (obstacles remaining, keys collected) → BFS with tuple state
