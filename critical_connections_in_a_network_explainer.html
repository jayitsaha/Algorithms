LeetCode #1192

Critical Connections in a Network

Find all edges whose removal would increase the number of connected components — known as bridges in graph theory.

Hard Graph Tarjan's Bridge Finding Time: O(V+E) Space: O(V+E)
1 Understanding the Problem ▾

You are given n servers (0 to n−1) connected by undirected network cables. A critical connection is a cable that, if cut, would leave at least one server unable to communicate with the rest.

In graph theory these are called bridges: edges whose removal increases the number of connected components. Your task is to find all of them efficiently — not one by one by brute force, but in a single DFS pass.

Not a Bridge

Edge (0–1): even if removed, servers 0 and 1 can still communicate through server 2 via path 0–2–1. It's part of a cycle.

Bridge!

Edge (1–3): server 3 is a leaf — its only connection is through node 1. Remove this cable and server 3 is isolated. No alternative path.

💡 Key Insight

An edge (u, v) is a bridge if and only if node v's entire subtree (in the DFS tree) cannot "reach back" to node u or any of u's ancestors through a back-edge. Tarjan's algorithm detects this in one DFS pass using two timestamps per node: disc (discovery time) and low (lowest reachable ancestor).

Example: n=4, connections=[[0,1],[1,2],[2,0],[1,3]]. Nodes 0,1,2 form a triangle (a cycle), so none of those three edges are bridges. Node 3 connects only through node 1, making (1,3) the single bridge. Answer: [[1,3]].

2 The Algorithm: Tarjan's Bridge Finding ▾

Robert Tarjan (1974) devised a linear-time algorithm that detects bridges during a single DFS. Every node gets two values:

  • disc[v] — the discovery time: a global counter that increments each time we first visit a node. Think of it as a timestamp.
  • low[v] — the low-link value: the minimum disc value reachable from node v or any descendant of v, possibly via one back-edge (an edge to an ancestor in the DFS tree).
low[v] > disc[u]

1 Build adjacency list — store each undirected edge in both directions.
2 Initialize disc[] = low[] = −1 (−1 means unvisited). Set global timer = 0.
3 DFS from node 0. On first visit to node v: set disc[v] = low[v] = timer, then increment timer.
4 For each neighbor w of v:
— If w is the parent: skip (avoid trivial back-tracking on undirected edge).
— If w is unvisited: recurse dfs(w, v); after returning, update low[v] = min(low[v], low[w]).
— If w is already visited (back-edge): update low[v] = min(low[v], disc[w]).
5 Bridge check: after returning from dfs(w, v), if low[w] > disc[v], then edge (v, w) is a bridge. Collect it.
🔐 Why disc vs low?

disc[v] is fixed at first visit. low[v] starts equal to disc[v] and can only decrease: it "pulls up" the minimum ancestor reachable from v's subtree. If the lowest reachable is still below v (low[v] ≤ disc[parent]), then a cycle exists through the tree edge and it is safe to remove. If not, the edge is the only connection — a bridge.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After the DFS fully processes node v: low[v] = the minimum disc[x] for any node x reachable from v's subtree via at most one non-tree (back) edge.

Why this guarantees bridge detection: Consider tree edge (u → v). Node v's subtree remains connected to the rest of the graph even if (u, v) is removed, only if some descendant of v has a back-edge to u or an ancestor of u. Such a back-edge would give low[v] ≤ disc[u]. Contrapositive: if low[v] > disc[u], no such back-edge exists, so removing (u, v) disconnects v's subtree.

Propagation: low values propagate bottom-up during DFS. When we return from dfs(w), we do low[v] = min(low[v], low[w]). This correctly "bubbles up" the earliest reachable ancestor from any node in v's subtree.

⚠️ The Parent-Edge Trap

In an undirected graph, each edge (u, v) is stored as both u→v and v→u. When visiting neighbors of v with parent u, we must skip u — otherwise the edge u→v (stored as v's neighbor) would be treated as a back-edge, giving low[v] = disc[u], making it look like v can always reach back to u. This would prevent any bridge from ever being detected.

Edge cases: Isolated nodes — problem guarantees connectivity. Parallel edges — to handle multiple edges between the same pair correctly, track by edge index (not parent node): skip the edge we came from by its index, not all edges to parent. Single node — no edges, return empty list.

4 The Code (Annotated) ▾
from collections import defaultdict
def criticalConnections(n, connections):
graph = defaultdict(list) adjacency list for undirected graph
for u, v in connections: build bidirectional adjacency
graph[u].append(v)
graph[v].append(u)
disc = [-1] * n -1 = unvisited; set to timer at first visit
low = [-1] * n lowest disc reachable from node's subtree
timer = [0] mutable int (list trick for closure capture)
bridges = [] collect answers here
def dfs(node, parent):
disc[node] = low[node] = timer[0] stamp discovery time; low starts equal
timer[0] += 1 advance global timer
for nei in graph[node]: iterate all neighbors
if nei == parent: skip the edge we came from (parent)
continue
if disc[nei] == -1: neighbor is unvisited → tree edge
dfs(nei, node) recurse into neighbor
low[node] = min(low[node], low[nei]) pull up low from child subtree
if low[nei] > disc[node]: bridge condition: child can't reach back to us
bridges.append([node, nei]) found a bridge!
else: already visited → back-edge
low[node] = min(low[node], disc[nei]) update with ancestor's disc time (not low!)
dfs(0, -1) start DFS from node 0, no parent
return bridges
🚫 Bug Alert: Use disc[nei], NOT low[nei] for back-edges

For back-edges (already-visited neighbor), always update: low[node] = min(low[node], disc[nei]). Do NOT use low[nei]. Using low[nei] could propagate incorrect low values across back-edges, causing non-bridges to be falsely reported as bridges. The disc value of an already-visited ancestor is the correct bound.

💡 Complexity

Time: O(V + E) — each node visited once, each edge traversed twice (both directions). Space: O(V + E) — adjacency list + disc/low arrays O(V) + recursion stack O(V) worst case (linear chain).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

Keywords: "critical edge", "single point of failure", "remove edge disconnects graph", "bridge". These signal a classic graph bridge-detection problem. The graph is undirected and connected (guaranteed), so one DFS from any node visits everything.

Step 2: Propose Brute Force

For each edge, remove it, then run BFS/DFS to check if the graph remains connected. O(E × (V+E)). For n=100,000 this is way too slow. Show the interviewer you understand the problem before jumping to the optimal.

Step 3: The Core Observation

An edge is a bridge if and only if it is NOT part of a cycle. So the question becomes: how do we know if a DFS tree edge is in a cycle? If some descendant has a back-edge to an ancestor at or above the parent, we have an alternative path — not a bridge. We need to track the "earliest reachable ancestor" from any subtree: that's low[v].

Step 4: Introduce Tarjan's Algorithm

disc[v] = when first visited (monotonically increasing). low[v] = min disc reachable via subtree + at most one back-edge. Bridge condition: low[child] > disc[parent]. One DFS, O(V+E). This is Tarjan's bridge-finding algorithm.

🗣️ What to Say to the Interviewer

"First, brute force: remove each edge, BFS to check connectivity. O(E × (V+E)) — too slow." "Better insight: an edge is critical only if it's not in any cycle. If I can detect cycles during DFS, I'm done." "I'll use Tarjan's bridge-finding algorithm. Each node gets disc (discovery time) and low (earliest ancestor reachable from its subtree). After DFS on child v from parent u: if low[v] > disc[u], v's subtree has no back-edge to u or above — removing (u,v) disconnects it." "One DFS, O(V+E) time and space. One subtle point: skip the edge back to the parent node, otherwise the undirected graph would always show a back-edge that prevents bridge detection."
📋 Common Follow-Ups

Articulation Points: Same DFS, different condition — a node u is an articulation point if low[child] ≥ disc[u] (for non-root; root is special). Parallel edges: Track by edge index not parent node to correctly handle multiple edges between same pair. Disconnected graph: Run DFS for each unvisited component (loop over all nodes).

7 Summary & Key Takeaways ▾
A bridge is an edge that is NOT part of any cycle — Tarjan identifies these in one DFS.
disc[v] = discovery timestamp (set once, never changes). low[v] = lowest disc reachable from v's subtree (decreases as back-edges are found).
Bridge condition: low[child] > disc[parent] — no back-edge from child's subtree can reach parent or above.
Always skip the parent edge when traversing neighbors in an undirected graph.
For back-edges use disc[nei] (not low[nei]) to update low values correctly.
Time O(V+E), Space O(V+E) — optimal. Brute force is O(E × (V+E)).
