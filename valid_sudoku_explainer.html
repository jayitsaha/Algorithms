LeetCode #36

Valid Sudoku

Determine if a 9×9 Sudoku board is valid. Only the filled cells need to be validated.

Medium Matrix / Grid Hash Sets Time: O(1) Space: O(1)
1 Understanding the Problem ▾

We have a partially-filled 9×9 grid. For every filled number, verify it does not conflict with any other filled number in its row, its column, and its 3×3 box. We do not need to check whether the board is solvable — only that existing numbers don’t violate the three uniqueness rules.

💡 Key Insight

We need to track what’s seen in 3 independent dimensions simultaneously: row, column, and 3×3 box. Any digit appearing twice in the same dimension is invalid. Empty cells ('.') have no constraint — skip them entirely.

Row 0: 5 3 . . 7 . . . .

Row 2: . 9 8 . . . . 6 .
No digit repeats in any row, col, or box → Valid

✗ Invalid board (duplicate in row 0)

Row 0: 5 3 . . 7 . . . 5
Row 1: 6 . . 1 9 5 . . .
Row 2: . 9 8 . . . . 6 .
'5' appears twice in row 0 → Invalid

Edge cases to consider: An all-'.' board is valid (no filled cells to conflict). A board with a single filled cell is valid. A duplicate in any one of the 27 units (rows, columns, or boxes) is enough to return False immediately.

2 The Algorithm: Single Pass with Hash Sets ▾

We scan every cell exactly once. For each filled cell we check and update three sets — one per row, one per column, one per box. A duplicate detection in any set triggers an immediate False.

1 Create 27 sets: rows[0..8], cols[0..8], boxes[0..8] — one set per “unit”. Each set tracks which values have appeared in that unit so far.
2 Scan every cell (r, c): Outer loop r in 0–8, inner loop c in 0–8. This visits all 81 cells in row-major (left-to-right, top-to-bottom) order.
3 Skip empty cells: If board[r][c] == '.', execute continue. Empty cells have no constraints — do not add them to any set.
4 Compute box_idx: box_idx = (r // 3) * 3 + (c // 3). This maps the cell’s row-group (0/1/2) and column-group (0/1/2) to a unique box number 0–8.
5 Check for duplicates: If val is already in rows[r], cols[c], or boxes[box_idx] → return False immediately (fail-fast). Each set lookup is O(1).
6 Register the value: Add val to rows[r], cols[c], and boxes[box_idx]. Future cells in the same units will detect this value as a conflict.
7 Return True: If we process all 81 cells without finding a conflict, the board is valid.

🔑 The Box Index Formula

box_idx = (r // 3) * 3 + (c // 3) — integer-divide row by 3 to get row-group (0, 1, or 2); integer-divide col by 3 to get col-group (0, 1, or 2). Multiply row-group by 3, add col-group → unique index 0–8. This is the “aha moment” of this problem. Example: cell (4,7) → (4//3)*3+(7//3) = 1*3+2 = 5 (middle-right box).

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing cell (r, c), rows[r] contains exactly the distinct values seen so far in row r; cols[c] contains exactly the distinct values seen so far in column c; boxes[box_idx] contains exactly the distinct values seen so far in that 3×3 box. If we try to add a value already in any of these sets, we have found a constraint violation.

Why sets work here: A set’s in operator is O(1) average. Adding a value we already have signals a duplicate. Since digits 1–9 are at most 9 values per unit, each set has at most 9 elements — truly O(1) space per set.

Box formula derivation: Rows 0–2 → r//3 = 0; rows 3–5 → r//3 = 1; rows 6–8 → r//3 = 2. Cols similarly produce 0, 1, or 2. Multiplying row-group by 3 and adding col-group yields values 0*3+0=0 through 2*3+2=8. Each (row-group, col-group) pair is unique, so the formula produces a unique box index for each of the nine 3×3 sub-grids.

Why a single pass suffices: We scan left-to-right, top-to-bottom. The constraint requires that no two filled cells in the same unit share the same value. We check the current cell against all previously seen cells in its units. Since we process every cell exactly once, any duplicate pair will be caught when we reach the second occurrence.

⚠️ Why O(1) time and space?

The board is always 9×9 = 81 cells, fixed. The sets hold at most 9 values each (digits 1–9). We never scale with input size. Despite having loops, the iterations are bounded by a constant (81). We have 27 sets of at most 9 elements each = 243 total stored values max — all O(1).

4 The Code (Annotated) ▾

Solution 1 — Single Pass Hash Sets (Optimal, Interview Pick)

def isValidSudoku(board):
rows = [set() for _ in range(9)] ← 9 independent sets, one per row
cols = [set() for _ in range(9)] ← 9 sets, one per column
boxes = [set() for _ in range(9)] ← 9 sets, one per 3×3 sub-box
for r in range(9): ← outer: iterate all 9 rows
for c in range(9): ← inner: iterate all 9 cols
val = board[r][c] ← read current cell value
if val == '.': ← empty cell has no constraints
continue ← skip — never add '.' to sets
box_idx = (r // 3) * 3 + (c // 3) ← maps (r,c) to one of 9 boxes
if val in rows[r] or \ ← duplicate in this row?
val in cols[c] or \ ← duplicate in this column?
val in boxes[box_idx]: ← duplicate in this 3×3 box?
return False ← fail-fast on first conflict found
rows[r].add(val) ← register in row tracker
cols[c].add(val) ← register in column tracker
boxes[box_idx].add(val) ← register in box tracker
return True ← all 81 cells passed, board is valid
⚠️ Don’t Forget to Skip ‘.’

'.' is not a digit — it represents an empty cell with no constraints. If you don’t skip it, you would add '.' to the sets, and the second empty cell in any unit would be falsely flagged as a duplicate. Always continue when val == '.'.

⚠️ Memorize the Box Index Formula

(r // 3) * 3 + (c // 3) is the critical insight. The //3 operation collapses 9 rows into 3 row-groups, and 9 cols into 3 col-groups. Multiplying row-group by 3 and adding col-group yields a unique number 0–8 for each of the 9 boxes. Practice deriving it from scratch on paper.

Solution 2 — Three-Pass Brute Force (simpler to reason about)

def isValidSudoku(board):
def has_dup(unit): ← helper: does this unit have a duplicate?
nums = [x for x in unit if x != '.'] ← filter out empty cells
return len(nums) != len(set(nums)) ← set removes dups; shorter means dup existed
for i in range(9): ← check all 9 rows, cols, and boxes
if has_dup(board[i]): ← row i = board[i] (list of 9 values)
return False
if has_dup(board[r][i] for r in range(9)): ← col i: element i from each row
return False
br, bc = (i // 3) * 3, (i % 3) * 3 ← top-left corner of box i
box = [board[br+dr][bc+dc] ← build flat list of 9 cells in this box
for dr in range(3) for dc in range(3)]
if has_dup(box): return False
return True ← all 27 units are clean
5 🎤 Interview Strategy: How to Arrive at This ▾
🔎 Pattern Recognition Signal

Grid problem + uniqueness constraint = hash sets for each unit. When you see “each row/column/region must contain unique values,” your reflex should be: one set per dimension, O(1) membership check per cell.

Step 1: Identify the Constraints

Three independent uniqueness constraints: rows, columns, 3×3 boxes. Each is separate — a digit can repeat across different rows, different columns, and different boxes, but never within the same unit.

Step 2: Propose Brute Force First

“For each of the 27 units (9 rows + 9 cols + 9 boxes), extract the digits and check for duplicates using a set. Three separate passes. Let me think if we can do it in one pass.”

Step 3: Optimize to Single Pass

“I can maintain 27 sets simultaneously. At each cell, check all three constraints before adding. The key insight is mapping (r, c) to a box index: (r//3)*3 + (c//3). Let me derive that formula.”

Step 4: Derive the Box Formula Live

“Rows 0–2 share box-row 0, rows 3–5 share box-row 1, rows 6–8 share box-row 2. So box-row = r//3. Same idea for columns. Box index = box-row * 3 + box-col = (r//3)*3 + (c//3).”

🗣️ What to Say to the Interviewer

"The board has three types of units to validate: rows, columns, and 3x3 boxes.
I'll use 27 hash sets — 9 per type — and do a single left-to-right, top-to-bottom scan.
At each filled cell, I check if its value already appears in its row set, column set, or box set.
If it does, return False immediately — fail-fast. Otherwise add it to all three sets and continue.
The non-obvious part: box index = (r//3)*3 + (c//3) maps any (r,c) to one of 9 boxes.
Empty cells ('.' ) are skipped entirely. Time O(81)=O(1), Space O(27)=O(1)."
🔮 Likely Follow-up Questions
  • “Now solve the Sudoku.” → Use this validator inside a backtracking solver (LeetCode #37)
  • “Why 27 sets and not just 3?” → Each of the 27 units is independent. A digit can appear in multiple rows, columns, or boxes — just not twice in the same one. You need separate tracking per unit.
  • “Validate a different board size?” → For an N×N board with √N-sized boxes: (r // sqrt_n) * sqrt_n + (c // sqrt_n)
  • “Can you do it with less space?” → Yes, bitmasks: a 9-bit integer per unit, set bit k for digit k. Same O(1) space but lower constant than hash sets.

7 Summary & Key Takeaways ▾
Three independent constraints (row, col, box) are checked simultaneously using 27 hash sets — one per unit
Always skip empty cells ('.') — only filled cells participate in uniqueness constraints
box_idx = (r//3)*3 + (c//3) is the critical formula — maps any (r,c) to a unique box 0–8
Fail fast: return False the moment a duplicate is detected in any set, no need to scan the whole board
O(1) time and space because the board size is fixed (9×9 = 81 cells, 27 sets of at most 9 elements each)
