LeetCode #133

Clone Graph

Given a reference to a node in a connected undirected graph, return a deep copy (clone) of the graph.

Medium Graph DFS + Hash Map Clone Time: O(V+E) Space: O(V)
1 Understanding the Problem ▾

You receive a reference to any single node in a connected, undirected graph. Each node contains a val (integer 1 to n) and a neighbors list (references to adjacent nodes). You must return a deep copy — an entirely new set of node objects with the same values and edge structure, but no shared references with the original graph.

💡 Key Insight

The challenge is not the traversal — it's handling cycles without infinite recursion, and ensuring each original node maps to exactly one new clone. A hash map old_node → clone_node solves both: it acts as a visited guard (prevents re-processing) and as a lookup table (reconnects cloned neighbors to the right clone objects).

Consider this 4-node cycle graph: 1—2—3—4—1 with a diagonal 1—3. A naive recursive clone of node 1 would try to clone its neighbor node 2, which would try to clone its neighbor node 1 again — an infinite loop. We must track what we've already cloned.

✗ Naive: Infinite Recursion on Cycles

clone(1) → clone neighbors
→ clone(2) → clone neighbors
→ clone(1) → clone neighbors
→ clone(2) → ... ∞

✓ Hash Map Guard: One Clone Per Node

visited[node1] = clone1
clone(2) hits neighbor 1
visited[node1] exists!
→ return clone1 directly ✓

Edge Cases: node = None → return None. Single isolated node (no neighbors) → return a clone with empty neighbors list. Self-loops → the hash map handles them the same way as cross edges.

2 The Algorithm: DFS with Hash Map Memoization ▾

We use Depth-First Search starting from the given node, visiting every reachable node exactly once. At each node, we first register its clone in a hash map, then recurse into its neighbors. The hash map doubles as our visited set and our "node identity" store.

1 Null guard: If node is None, return None. Empty graph — nothing to clone.
2 Check visited: If node is already in the visited hash map, return visited[node] immediately. This is the existing clone for this node — returning it prevents infinite loops and ensures pointer identity.
3 Create and register clone: Create clone = Node(node.val) (empty neighbors list). Store visited[node] = clone immediately — before recursing. This is the cycle-breaker step.
4 Clone all neighbors: For each neighbor in node.neighbors, call dfs(neighbor) and append the result to clone.neighbors. Each recursive call either returns a fresh clone (first visit) or an already-registered clone (revisit).
5 Return clone: Return clone up the call stack. The initial call returns the clone of the starting node, which is your entry point into the fully copied graph.
🔑 Why Register BEFORE Recursing?

Storing visited[node] = clone before the neighbor loop is the critical ordering. If you stored it after, any back-edge (a neighbor pointing to an ancestor) would not find the ancestor in visited, trigger a second call, create a second clone, and break the "one clone per node" invariant. Always: create → register → then recurse.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point during DFS, visited[node] exists if and only if node has been encountered before — and the corresponding clone object is permanently fixed. Any future reference to this original node (from any path through the graph) returns this exact same clone object. This guarantees structural fidelity of the copy.

Every node is cloned exactly once. When dfs(node) is called: the first action checks visited. The very next action (before any recursion) writes visited[node] = clone. So even if ten different neighbors all eventually recurse into the same node, only the first call creates a clone — all subsequent calls return the cached result.

Every edge is reproduced correctly. For each original node, we iterate all neighbors and append dfs(neighbor) to the clone's neighbor list. Whether the neighbor is fresh or already cloned, we get its clone back and link it. Since the graph is undirected and both directions are stored in each node's adjacency list, all edges appear in the clone's adjacency structure.

⚠️ Termination Guarantee

The graph is connected with at most n ≤ 100 nodes. Each node enters visited exactly once — the DFS calls that do real work are bounded by n. Every other call hits the visited check at O(1) cost and returns immediately. Total work: O(n + m) where m is edges.

⚠️ The Order Bug That Causes Infinite Loops

Moving visited[n] = clone to after the neighbor loop breaks everything. Example: graph 1—2—1. dfs(1) starts, clone not yet in visited. Recurse into dfs(2). Node 2 recurses into dfs(1). Node 1 is still not in visited (we haven't finished yet) → creates another clone → infinite loop. Register first, always.

Solution 2 — BFS + Hash Map (Iterative, avoids deep recursion)

from collections import deque
def cloneGraph(node):
if not node: return None
visited = {node: Node(node.val)} ← Pre-clone the starting node, seed visited
queue = deque([node]) ← BFS queue of original nodes to expand
while queue:
curr = queue.popleft() ← Original node we're processing
for nb in curr.neighbors: ← Each original neighbor
if nb not in visited: ← First time? Create clone + enqueue
visited[nb] = Node(nb.val)
queue.append(nb)
visited[curr].neighbors.append(
visited[nb]) ← Wire clone's neighbors (always, even if nb existed)
return visited[node] ← Return clone of the starting node
💡 Complexity Summary

DFS (recursive): Time O(V+E), Space O(V) — visited map V entries, call stack up to V deep
BFS (iterative): Time O(V+E), Space O(V) — queue up to V entries; no stack overflow risk
Both visit each node exactly once and each edge twice (from each endpoint).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize Graph Traversal

You need to visit every node exactly once (to copy it) and reproduce all connections. That's a graph traversal — DFS or BFS. Signals: arbitrary graph (not a tree), adjacency-list representation, "make a copy of all reachable nodes."

Step 2: Identify the Cycle Hazard

"This is an undirected graph, so node 1's adjacency list contains node 2, and node 2's list contains node 1. Naive recursion creates an infinite loop. I need to track which nodes I've already cloned — a visited set."

Step 3: Upgrade Visited Set → Hash Map

"A plain visited set tells me what I've seen, but I also need to reconnect clone objects as neighbors. I'll change visited from a set to a dict: original_node → clone_node. Then when wiring neighbors, I look up the clone directly in O(1)."

Step 4: State the Ordering Invariant Explicitly

"I'll register visited[node] = clone immediately after creating the clone, before the neighbor loop. This way, if any cycle brings recursion back to this node, it finds the clone in the map and returns it without re-creating."

🗣️ What to Say to the Interviewer

"I'll use DFS with a hash map from original nodes to their clones.
Base case: if node is null, return null.
Visited check: if the node is already in the map, return its clone — handles cycles.
Otherwise: create a new Node with the same val, register it in the map immediately,
then recurse on each neighbor and append results to the clone's neighbors list.
Time O(V+E): visit each node and edge once. Space O(V): hash map and call stack.
I can also do this iteratively with BFS if recursion depth is a concern."
🔮 Likely Follow-up Questions
  • "Can you do it without recursion?" → BFS with a deque; same hash map, same pattern.
  • "What if the graph has disconnected components?" → Problem guarantees connected, but if not: iterate all nodes; run DFS from each unvisited one.
  • "How is this different from copying a binary tree?" → Trees are DAGs — no cycles, no back-edges. No visited map needed. Graphs can have cycles → need the map.
  • "What's the worst-case recursion depth?" → O(V) — a path graph degenerates to a linear chain, DFS depth equals V.

7 Summary & Key Takeaways ▾
Graph deep copy = DFS/BFS + hash map: original_node → clone_node
Register the clone in the map BEFORE recursing into neighbors — this is the cycle-breaker
The hash map serves dual purpose: visited guard (prevents re-processing) + clone lookup (reconnects edges)
BFS version avoids Python's recursion limit on very deep graphs (up to ~1000 nodes)
Time O(V+E): each node visited once, each edge processed twice. Space O(V) for the map.

🎯 When to Recognize the DFS + Hash Map Clone Pattern
  • "Deep copy" any graph or cyclic structure (linked list with random pointers, etc.)
  • Any graph problem needing O(1) "have I seen this node?" lookup
  • Building a new graph from an old one: transformation, serialization, mapping
  • Whenever you see "node objects with references to other nodes" → hash map clone
