LeetCode #433

Minimum Genetic Mutation

Find the minimum number of single-character mutations needed to transform a gene string from start to end, where every intermediate gene must be valid (in the bank).

Medium Graph BFS Word Ladder Time: O(N·32) Space: O(N)
1 Understanding the Problem ▾

A gene string is exactly 8 characters, each drawn from the nucleotide alphabet 'A', 'C', 'G', 'T'. We want to transform the start gene into the end gene. Each mutation changes exactly one character at any position, and the resulting string must appear in the given bank of valid intermediates. We want the fewest total mutations — or return -1 if it is impossible.

💡 Key Insight

This is structurally identical to Word Ladder (LeetCode #127). Each gene is a node. Two nodes share an edge when they differ by exactly one character AND both are in the bank (or one is the start). The problem asks for the shortest path from start to end in this unweighted graph — which is exactly what BFS guarantees in one pass.

Edge cases: start equals end → return 0. Bank empty or end not in bank → return -1. Valid genes exist in bank but none bridge to end → return -1. Bank may contain start (that is fine; just skip if already visited).

🔑 Why BFS and Not DFS?

BFS explores genes in order of increasing mutation distance. The very first time BFS discovers the end gene as a neighbour, the current step count is provably minimal — no shorter route could exist because BFS would have found it earlier. DFS can find a path but cannot guarantee it is the shortest without exhaustive search.

2 The Algorithm: BFS on an Implicit Gene Graph ▾

We never explicitly build the graph. Instead, for each gene we pop from the BFS queue, we generate all possible single-character mutations on the fly — 8 positions × 3 alternative bases = up to 24 candidates — and check each against the bank (a set for O(1) lookups). This is the hallmark of the "Word Ladder BFS" pattern: the graph is implicit and explored lazily.

1 Quick exits: If start == end, return 0. Convert bank to a set. If end not in bank_set, return -1 immediately — no valid path can exist.
2 Initialize BFS: Create a deque with (start, 0) — the start gene at step count 0. Create a visited set containing start to prevent cycling back to it.
3 Expand each node: Pop (gene, steps). For each position i in 0..7 and each character c in 'ACGT', form candidate gene[:i] + c + gene[i+1:]. Skip if c == gene[i].
4 Validate & enqueue: If candidate is not in bank_set, skip. If it equals end, return steps + 1 immediately — guaranteed shortest. Otherwise, if not in visited, mark visited and enqueue with steps + 1.
5 Failure exit: If the queue drains without ever generating end, return -1 — no valid mutation path exists through the bank.

⚠️ Why the End Check Matters Early

If end is not in the bank, it can never be a valid mutation result — even if it is one character away from a bank gene. This upfront O(1) check avoids running the entire BFS for nothing. Always validate this before creating the queue.

3 Why It Works: The Invariant ▾
🔐 The BFS Distance Invariant

At any moment, every gene in the queue at BFS level d was reached via exactly d valid mutations. Because BFS uses a FIFO queue and processes all level-d genes before any level-(d+1) gene, the first time the algorithm generates end as a neighbour, it is at the minimum possible distance. No shortcut can exist, because if one did, BFS would have found it at an earlier level.

Why the visited set is essential: Without it, BFS might re-enqueue a gene already discovered at distance 1 while processing a level-2 node — producing a longer, redundant path through the queue and potentially an incorrect (larger) answer if the gene is the end. Mark a gene visited the moment it is enqueued, not when it is dequeued, to prevent duplicates.

Complexity breakdown: Bank has at most N genes; gene length = 8; alphabet = 4 chars. Per dequeued gene: 8 × 3 = 24 mutation candidates, each a set lookup O(1). Total: O(N × 24) = O(N). Space for visited + queue: O(N).

💡 Bidirectional BFS — the Expert Upgrade

Run BFS simultaneously from start and from end. Each step, expand the smaller frontier. When the two frontiers meet, the sum of their depths is the answer. This reduces the worst-case search space from O(4d) to O(4d/2), a massive speedup for large banks. Mention this in interviews as a follow-up if time permits.

⚠️ Common Mistake: Removing from Bank Instead of Using visited

Some solutions bank.remove(mutant) to avoid revisiting — which works but modifies the input. The cleaner, non-destructive approach is a separate visited set. Never mutate input data unless the problem explicitly allows it.

4 The Code (Annotated) ▾

Solution 1 — BFS (Optimal, Interview Pick)

from collections import deque
def minMutation(start: str, end: str, bank: list) -> int:
if start == end: return 0 ← trivial case: already at destination
bank_set = set(bank) ← O(1) membership lookup vs O(N) for list
if end not in bank_set: return -1 ← end can never be reached if not a valid gene
queue = deque([(start, 0)]) ← (current_gene, mutations_used_so_far)
visited = {start} ← mark on enqueue, not dequeue
while queue: ← BFS main loop
gene, steps = queue.popleft() ← FIFO guarantees shortest-first expansion
for i in range(len(gene)): ← try mutating each of the 8 positions
for c in 'ACGT': ← try each valid nucleotide at position i
if c == gene[i]: continue ← skip no-op mutations (same character)
mutant = gene[:i]+c+gene[i+1:] ← form the mutated string
if mutant not in bank_set: continue ← invalid intermediate → discard
if mutant == end: return steps+1 ← FOUND! BFS guarantees this is shortest
if mutant not in visited: ← only explore unvisited valid genes
visited.add(mutant) ← mark NOW to prevent duplicate enqueuing
queue.append((mutant, steps+1)) ← enqueue for future exploration
return -1 ← queue exhausted; no path to end
⚠️ Why Check end Separately Instead of "if gene == end" After Dequeue?

We check if mutant == end: return steps+1 before enqueuing. This is slightly more efficient — we avoid enqueuing end only to check it on the next dequeue. Both variants are correct; the early check is the cleaner idiom.

Solution 2 — DFS with Backtracking (finds a path, not guaranteed shortest)

def minMutation_dfs(start, end, bank): ← educational only — O(N!) worst case
bank_set, result = set(bank), [float('inf')]
def dfs(gene, visited, steps):
if gene == end:
result[0] = min(result[0], steps); return
for i in range(len(gene)):
for c in 'ACGT':
m = gene[:i]+c+gene[i+1:]
if m in bank_set and m not in visited:
visited.add(m)
dfs(m, visited, steps+1)
visited.remove(m) ← backtrack
dfs(start, {start}, 0)
return result[0] if result[0] != float('inf') else -1
💡 Complexity Summary

BFS (optimal): Time O(N × 32) ≈ O(N), Space O(N) for visited + queue
DFS (exhaustive): Time O(N!) pathological, Space O(N) recursion depth — never for minimum queries

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Graph Structure

"The problem asks for minimum steps between two states, where each step is a constrained local change. This screams graph — states are nodes, valid single-mutation transitions are edges. The graph is unweighted, so BFS is optimal."

Step 2: Recognise the Word Ladder Pattern

"I've seen this pattern before — Word Ladder (#127). Same structure: BFS from start word to end word, each step changes one character, intermediate must be valid. Here the alphabet is ACGT and words are length 8."

Step 3: Design Neighbour Generation

"I'll generate neighbours on the fly: 8 positions × 3 non-identical chars = 24 candidates. Check each against the bank set in O(1). No need to pre-build the adjacency list — lazy generation is both simpler and efficient."

Step 4: Handle Edge Cases Upfront

"Before touching the queue, handle: start == end (return 0), end not in bank (return -1). These O(1) checks prevent wasted BFS on trivially impossible inputs."

🗣️ What to Say to the Interviewer

"This is a shortest-path problem on an unweighted implicit graph. States = genes. Transitions = single valid mutations. BFS guarantees shortest path.
First, convert bank to a set for O(1) lookup. Quick-exit if start==end or end not in bank.
BFS: start with (start_gene, 0). Per gene, try all 8 × 3 = 24 mutations. If mutant is in bank:
— If mutant == end, return steps+1 (done, this is the minimum).
— Else if not visited, mark visited and enqueue with steps+1.
If queue empties, return -1.
Time O(N × 32), Space O(N). Follow-up: bidirectional BFS halves the exponent."
🔮 Likely Follow-up Questions
  • "Bidirectional BFS?" → Expand from both start and end simultaneously; meet in the middle; search space from O(4d) → O(4d/2)
  • "What if gene length varies?" → Same BFS; time becomes O(N × L × sigma) where L = length, sigma = alphabet size
  • "What if mutations have costs?" → Replace BFS with Dijkstra's algorithm
  • "What if you need all shortest paths?" → BFS to record the BFS tree, then DFS/backtrack on the tree (Word Ladder II #126)

7 Summary & Key Takeaways ▾
Minimum mutations = shortest path in an implicit graph → BFS
Convert bank to a set first for O(1) membership checks
Early exit: if end not in bank, return -1 before touching the queue
Generate neighbours on the fly: 8 positions × 3 alternatives = 24 candidates per gene
Mark visited on enqueue (not dequeue) to prevent duplicate queue entries
The first time end appears as a BFS neighbour, the current depth is provably minimal

🎯 When to Recognise the BFS Word Ladder Pattern
  • "Minimum number of steps / mutations / transformations"
  • Each step changes exactly one unit (character, digit, bit, move)
  • Each intermediate state must satisfy a validity constraint (in a bank, in a dictionary, no deadend)
  • Start and end states are given explicitly
  • → Model as states (nodes) + valid single-step changes (edges) → BFS shortest path
