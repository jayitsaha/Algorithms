LeetCode #17

Letter Combinations of a Phone Number

Given a string of digits 2–9, return all possible letter combinations the digits could represent (like a T9 phone keyboard).

Medium Backtracking Digit to Letters Mapping Time: O(4ⁿ · n) Space: O(n)
1 Understanding the Problem ▾

Remember old T9 phones? Each digit key maps to a few letters. Press 2 → could be 'a', 'b', or 'c'. Press 3 → could be 'd', 'e', or 'f'. Given a sequence of digit characters, return every string you could type if each digit maps to its set of letters independently.

💡 Key Insight

For digits "23", digit 2 gives {a,b,c} and digit 3 gives {d,e,f}. The answer is every possible pairing of one letter from each group — the Cartesian product: ad, ae, af, bd, be, bf, cd, ce, cf. With 2 digits of 3 letters each, that is 3² = 9 combinations. Digits 7 and 9 have 4 letters ("pqrs", "wxyz"), so the worst case with n digits is 4ⁿ.

2 The Algorithm: Recursive Backtracking ▾

Think of building each combination one character at a time. At position index, we pick one letter for digits[index], then recurse for index + 1. When we have chosen a letter for every digit, we record the complete combination. This is the classic backtracking template: make a choice → recurse → (undo choice — implicit here because strings are immutable in Python).

1 Guard empty input: If digits == "", return [] immediately — no digits means no combinations.
2 Build the mapping: A dictionary from digit character to its letter string. This is a constant lookup table, not computed on the fly.
3 Define backtrack(index, current): index says which digit we are choosing for next. current is the string built so far (length = index).
4 Base case — collect result: When index == len(digits), every digit has a letter. Append current to the result list and return.
5 Branch on each letter: For each letter in phone[digits[index]], call backtrack(index + 1, current + letter). String concatenation creates a new object — no explicit undo needed.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every call to backtrack(index, current), the string current contains exactly one valid letter for each of digits[0..index-1]. When index reaches len(digits), we have one valid letter for every digit — a complete combination. Because we exhaustively loop through every letter for each digit, we visit every possible combination exactly once with no duplicates.

Why all combinations are covered: At each recursive level we loop through all letters for the current digit, then recurse with all letters for the next digit. This constructs every path from root to leaf in the recursion tree. The set of all root-to-leaf paths equals the Cartesian product of all digit letter sets.

Why no duplicates: The tree is traversed depth-first with strict left-to-right order. Each unique path through the tree corresponds to a unique choice of one letter per digit position. No two paths are identical, so no combination appears twice.

⚠️ Why Not Pop and Undo Like Other Backtracking?

In permutation problems you often append to a list then pop. Here, current + letter creates a brand-new string in Python — the parent's current string is untouched. "Undo" is free. This elegance comes at a cost: each recursive call allocates a new string of length index + 1, giving O(n) allocation per leaf path.

Complexity: With n digits, the recursion tree has at most 4ⁿ leaf nodes. Each leaf is reached via a path of length n (allocating one string per call). Total work is O(n · 4ⁿ). For n ≤ 4 this is at most 1024 string operations — trivially fast in any language.

4 The Code (Annotated) ▾

Solution 1 — Recursive Backtracking (Optimal, Interview Pick)

def letterCombinations(digits: str) -> list[str]:
if not digits: return [] ← Edge case: empty input → no combinations possible
phone = { ← Static lookup: digit char → string of letters
'2':'abc','3':'def','4':'ghi',
'5':'jkl','6':'mno','7':'pqrs',
'8':'tuv','9':'wxyz'
}
result = [] ← Shared accumulator — all complete combos go here
def backtrack(index, current): ← Closure captures result, phone, digits
if index == len(digits): ← Base case: every digit has been assigned a letter
result.append(current) ← current is a complete n-char combination
return
for letter in phone[digits[index]]: ← Try each mapped letter for the current digit
backtrack(index+1,current+letter) ← Recurse: advance index, extend string (no undo needed)
backtrack(0, "") ← Kick off: start at digit 0 with empty combo
return result
⚠️ Common Mistake: Returning Inside the Loop

Beginners sometimes write return backtrack(...) inside the for loop. This short-circuits after the very first letter, returning None instead of all combinations. The loop must complete for all letters to be explored. The return belongs only in the base case.

Solution 2 — Iterative BFS (Queue-based)

from collections import deque
def letterCombinations(digits: str) -> list[str]:
if not digits: return []
phone = {'2':'abc','3':'def','4':'ghi',
'5':'jkl','6':'mno','7':'pqrs','8':'tuv','9':'wxyz'}
queue = deque([""]) ← Start with one empty partial combo
for digit in digits: ← Process one digit at a time left to right
for _ in range(len(queue)): ← Expand exactly the current layer (not new additions)
combo = queue.popleft() ← Take a partial combo of length k
for letter in phone[digit]: ← Extend with each possible letter
queue.append(combo+letter) ← Now queue has (k+1)-length combos for this digit
return list(queue) ← All n-length combos remain in queue
💡 Complexity Summary

Backtracking: Time O(4ⁿ · n), Working Space O(n) call-stack depth
Iterative BFS: Time O(4ⁿ · n), Working Space O(4ⁿ · n) peak queue size
Both produce O(4ⁿ · n) total output. Backtracking is better on working memory.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

"Return all possible combinations" is the key signal for backtracking. Any time you must enumerate all arrangements, permutations, or combination sets, think backtracking first. The independent per-position choice (one letter per digit) makes this a clean Cartesian product — no pruning needed.

Step 2: Define the Recursion Signature

Ask: "What decision am I making at each step?" → Picking a letter for the current digit. "How do I know I'm done?" → When index reaches len(digits). "What changes each call?" → index increments by 1. This directly gives backtrack(index, current).

Step 3: Write Base Case First

Two guards: (1) empty input → return [] at the top level. (2) index == len(digits) → append current and return. Write these before the recursive loop — you'll never forget them if you reason from the smallest valid input first.

Step 4: Confirm Complexity

O(4ⁿ · n) time. For n ≤ 4, that is at most 4⁴ · 4 = 1024 operations. Completely fine. Call stack depth is O(n) = O(4). Total output size is O(4ⁿ · n).

🗣️ What to Say to the Interviewer

"This is a combination generation problem, so I'll use backtracking.
At each digit position I pick one of its mapped letters and recurse to the next position.
Base case: when the index equals len(digits), I have a complete combination — append it.
Since Python strings are immutable, current + letter creates a new string, so no explicit undo.
Time: O(4^n · n), Space: O(n) for the call stack. For n ≤ 4 this is effectively O(1)."
🔮 Likely Follow-up Questions
  • "Can you do it without recursion?" → Yes — BFS with a deque, expanding one digit at a time
  • "What if digits include 0 or 1?" → Add entries to the map or skip those digits
  • "Return results in lexicographic order?" → The map letters are already sorted; DFS left-to-right produces sorted output
  • "Filter only valid English words?" → Cross-reference a Trie or set — prune branches when no valid word starts with the current prefix
