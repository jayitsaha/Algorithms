Flatten a Multilevel Doubly Linked List

Given a doubly linked list where nodes may have a child pointer to another sub-list, flatten all levels into a single-level doubly linked list in DFS (depth-first) order.

Medium Linked List DFS / Stack Time: O(n) Space: O(d)
1 Understanding the Problem ▾

Each node has three pointers: prev, next, and child. When child is not null, it points to the head of a completely separate doubly linked list. That child list can itself have nodes with children — creating arbitrary nesting depth.

Your task: reorganize all nodes into a single flat doubly linked list with no remaining child pointers. The output order is depth-first: when you hit a node with a child, fully exhaust the child chain (and all of its descendants) before continuing with the node's original next.

💡 Key Insight

Think of each child list as an interruption. The moment you encounter a child, you must fully "insert" that child's entire chain between the current node and its next. This insertion-before-continuing is exactly DFS preorder: go deeper before going wider. A stack lets us save the "right side" so we can rejoin it after exhausting the child.

2 The Algorithm: Iterative DFS with Explicit Stack ▾

We walk curr through the list. Whenever we find a node with a child, we perform a splice: redirect curr.next to the child head, push the original curr.next onto a stack, and clear the child pointer. When we reach the tail of a sub-list (curr.next is None) and the stack is non-empty, we pop the saved continuation and reattach it.

1 Initialize: curr = head, stack = []. The stack will save "right-side" continuations whenever we dive into a child list.
2 Walk the list: Loop while curr is not None. At each iteration, first check for a child, then check if we need to pop.
3 Child found — splice in: If curr.next exists, push it onto the stack. Set curr.next = curr.child and curr.child.prev = curr. Clear curr.child = None.
4 Sub-list end — rejoin: If curr.next is None and the stack is non-empty, pop the saved node. Set curr.next = popped and popped.prev = curr.
5 Advance: curr = curr.next. Repeat until curr is None. Return head.
🔑 Why This is DFS

When we dive into the child list, we push the "future" (original next) onto the stack and process the entire child chain before returning to it. Popping happens only when the child chain ends — exactly like DFS backtracking. The stack depth equals the nesting depth d of the structure.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point in the traversal, every node from head up to (but not including) curr is already correctly spliced into the flat list with valid prev/next pointers and no child pointers remaining. The stack holds the "right boundary" continuations for any open sub-lists we are currently inside.

Splice correctness: When we find curr.child, we relink curr → child_head in place of curr → curr.next. We also set child_head.prev = curr. The old curr.next is pushed to the stack so it can be reattached when the child chain ends. At each step, the list remains locally consistent.

Stack semantics: Each entry on the stack represents "we interrupted processing at this node to go deeper." Pops happen when a sub-list reaches its tail (next is None). Since we process children before siblings (depth-first), the flattened order exactly matches a DFS preorder traversal of the multi-level structure.

⚠️ Most Common Bug: Missing prev Pointer Updates

This is a doubly linked list. Every splice must update pointers in both directions. Two critical spots: (1) When attaching child head: child_head.prev = curr. (2) When reattaching the saved next after a sub-list ends: saved_next.prev = curr where curr is the tail of the sub-chain. Missing either breaks the doubly-linked invariant and causes incorrect reverse traversal.

Complexity: Each node is visited exactly once — O(n) time. The stack holds at most d entries (max nesting depth), so O(d) extra space. In the worst case (a chain of children at depth d = n), space is O(n). For a fully flat list, O(1) extra space.

4 The Code (Annotated) ▾

Solution 1 — Iterative Stack (Interview Pick)

def flatten(head): ← takes head: Node | None
if not head: ← Edge case: empty list → return None
return None
curr = head ← Walking pointer starts at head
stack = [] ← Explicit stack: saves paused "next" continuations
while curr: ← Visit every node exactly once
if curr.child: ← Does this node have a child sub-list?
if curr.next: ← Only push if there is a "right side" to save
stack.append(curr.next) ← Save continuation: resume here after child chain ends
curr.next = curr.child ← Splice: redirect curr → child head
curr.child.prev = curr ← Fix backward pointer (doubly linked!)
curr.child = None ← Problem requires clearing the child pointer
if not curr.next and stack: ← At tail of a sub-list with saved continuations
top = stack.pop() ← Retrieve the saved next node from the parent level
curr.next = top ← Attach saved node after this tail
top.prev = curr ← Fix backward pointer for the rejoined node
curr = curr.next ← Advance to the next node
return head ← Head unchanged; list is now fully flat
⚠️ Why Guard "if curr.next" Before Pushing?

We only push curr.next if it is non-null. If curr.next is already None when we find a child, there is nothing to save — the child chain will simply become the remainder of the list. Pushing None would cause a bug when we later try to set pointers on a null object.

Solution 2 — Recursive DFS (elegant, same complexity)

def flatten(head): ← Call-stack acts as the explicit stack
if not head: return None
curr = head
while curr:
if curr.child:
child_flat = curr.child ← Head of child (already will be flattened by recursion)
# Find the tail of the child chain
tail = child_flat
while tail.next: tail = tail.next
nxt = curr.next ← Save original next
curr.next = child_flat ← Splice child chain in
child_flat.prev = curr
curr.child = None
tail.next = nxt ← Reconnect after the child tail
if nxt: nxt.prev = tail
curr = curr.next
return head
💡 Complexity Summary

Iterative Stack: Time O(n), Space O(d) explicit stack (d = max nesting depth)
Recursive: Same O(n) time, O(d) call-stack space — but risks stack overflow for very deep nesting
Best case (already flat): O(1) extra space. Worst case (every node is a child): O(n) space.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify the Traversal Order

Ask: "Is the output depth-first (child subtree before siblings) or breadth-first?" The answer is depth-first. This immediately suggests DFS — either a recursive approach (call stack) or an iterative approach with an explicit stack.

Step 2: Identify the Core Sub-Problem

"When I see a node with a child, I need to insert the entire child chain between this node and its next. I need to: (1) redirect curr.next to child.head, (2) save the original next to reattach later after the child ends." This is the key operation — everything else is bookkeeping.

Step 3: Choose Iterative over Recursive

Recursive is concise but may overflow on deeply nested inputs. Iterative with an explicit stack is equally clear, safer, and shows you know the mechanics. Present the iterative version as your main solution, mentioning recursion as an alternative.

Step 4: Explicitly Call Out the prev Pointer

"This is a doubly linked list — I must update both directions at every splice. Let me be systematic: when I splice the child head in, I set child_head.prev = curr. When I reattach the saved next at the tail, I set saved_next.prev = tail." Making this explicit impresses interviewers.

🗣️ What to Say to the Interviewer

"I'll walk curr through the list. When I find a node with a child:
- Push curr.next (if it exists) onto a stack to save the continuation
- Redirect curr.next = curr.child (splice child chain in)
- Set child.prev = curr, clear child pointer
When curr.next is None and stack is non-empty:
- Pop and reattach: curr.next = top, top.prev = curr
This is iterative DFS. Time O(n), Space O(d) for max nesting depth."
🔮 Likely Follow-up Questions
  • "What if there were 3 or more levels of nesting?" → No change needed — the stack handles arbitrary depth automatically.
  • "Can you do it in O(1) extra space?" → Not straightforwardly; you need O(d) for the stack. If we use Morris-style traversal you can reduce space but it becomes very complex.
  • "Related problem: Flatten Binary Tree to Linked List (#114)" → Same DFS splice idea, but on a binary tree where right-child plays the role of next.
  • "How would you undo the flattening?" → Not possible without extra metadata; we lost information about where children were originally attached.

7 Summary & Key Takeaways ▾
Flatten order is depth-first: child subtrees come fully before siblings
Use an explicit stack to save "right-side" continuations when diving into a child
Always update BOTH prev and next at every splice — this is doubly linked
Clear curr.child = None after splicing (the problem requires no remaining child pointers)
Only push curr.next when it is non-null; no point saving None
Space O(d) depth: O(1) for flat inputs, up to O(n) for a degenerate nesting chain

🎯 When to Use DFS / Stack on Linked Structures
  • "Process children before continuing" → DFS order → explicit stack or recursion
  • "Insert a sub-chain into the middle of a list" → save tail, splice, reattach
  • Multiple levels of nesting in any data structure → DFS + stack
