Longest Consecutive Sequence

LeetCode #128 – Find the longest run of consecutive integers

Medium Hash Set Sequence Start Check O(n) Time O(n) Space
1 Understanding the Problem ▼

Given an unsorted array of integers, find the length of the longest sequence of consecutive integers (e.g., 1,2,3,4).

Example: [100,4,200,1,3,2] → longest is 1,2,3,4 → length 4

💡 Key Insight

Only start counting from a sequence's beginning — a number n is a sequence start if n-1 is NOT in the set. This avoids redundant counting and keeps time O(n).

⚠️ Why not sort?

Sorting gives O(n log n). The problem asks for O(n). Hash set lookup is O(1), enabling a linear solution.

2 The Algorithm: Hash Set + Sequence Start Check ▼
1Insert all numbers into a hash set
2For each number, check: is num-1 in the set? If yes → skip (not a sequence start)
3If no → this is a sequence start. Count up: how far does num, num+1, num+2… go?
4Update best answer
🔐 Why Each Number Is Visited At Most Twice

Each number is touched once when we check if it's a start, and at most once when it's counted as part of a sequence starting from a smaller number. Total work = O(n).

3 Why It Works: The Invariant ▼
🔐 The Invariant

We only expand sequences that start from their true beginning. A number is a start iff num-1 ∉ set. This means each sequence is counted exactly once — from its leftmost element.

This prevents O(n²) redundancy: if we start counting from every element, we'd recount subsequences. By skipping non-starts, each consecutive chain is traversed exactly once.

4 The Code (Annotated) ▼
def longestConsecutive(nums):
num_set = set(nums) # O(n) lookup
best = 0
for num in num_set:
# Only start from sequence beginnings
if num - 1 not in num_set:
length = 1
while num + length in num_set:
length += 1
best = max(best, length)
return best

Time: O(n) — each element touched at most twice. Space: O(n) for the set.

5 🎤 Interview Strategy ▼
Step 1: Clarify the constraint

Interviewer wants O(n). Sorting is O(n log n) — mention it, then explain why we can do better with a hash set.

Step 2: Key observation

Only need to start counting from sequence beginnings. A number starts a sequence iff num-1 is absent.

🗣️ What to Say

"I'll insert all numbers into a set. For each number, I only start counting if num-1 isn't in the set — that means it's a sequence start. Then I extend upward while num+length is in the set. This is O(n) because each number is visited at most twice."
