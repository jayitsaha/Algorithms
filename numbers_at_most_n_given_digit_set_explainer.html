LeetCode #902

Numbers At Most N Given Digit Set

Given an array of digit strings and integer N, return the count of all positive integers that can be formed using those digits that are less than or equal to N.

Hard Dynamic Programming Digit DP Time: O(log N · |D|) Space: O(log N)
1 Understanding the Problem ▾

You have a set of allowed digits, say D = ["1","3","5","7"]. You can build any positive integer by choosing digits from D with repetition (so "13", "333", "71" are all valid constructions). Given an upper bound N, count how many such integers are ≤ N.

Walk-through example: D = ["1","3","5","7"], N = 100. Valid numbers: 1, 3, 5, 7 (1-digit), 11, 13, 15, 17, 31, 33, 35, 37, 51, 53, 55, 57, 71, 73, 75, 77 (2-digit). No 3-digit number from D is ≤ 100 (smallest is 111 > 100). Answer = 4 + 16 = 20.

💡 Key Insight: Split by Length

Every valid number either has fewer digits than N (automatically ≤ N — trivial to count as |D|1 + |D|2 + ... per length), or has exactly as many digits as N (needs a careful digit-by-digit comparison to stay ≤ N). These two groups are disjoint and exhaustive — count them separately and add.

✓ Group 1: Shorter numbers (easy)

N = 100 has 3 digits
1-digit: |D|1 = 4 numbers
2-digit: |D|2 = 16 numbers
All are < 100 by definition
Subtotal: 20

✗ Group 2: Same-length (harder)

Must be exactly 3 digits AND ≤ 100
N = "1","0","0"
pos 0: need digit ≤ "1" from D
Only "1" matches → stay tight
pos 1: need "0" from D → "0" ∉ D
0 valid 3-digit numbers

Edge cases: All digits in D are larger than N's leading digit (same-length count = 0); D = ["9"], N = 9 (answer = 1, N itself); N has only 1 digit (no shorter-number group).

2 The Algorithm: Digit DP — Position-by-Position with TIGHT State ▾

Let k = number of digits in N (= len(str(N))), d = |D|. The algorithm processes N's digits left to right, maintaining a crucial distinction:

TIGHT
Every digit placed so far
exactly matches N's prefix
→
place digit < N[pos]
FREE
Already strictly less than N
any remaining digit is OK

Once FREE, you stay FREE — the TIGHT → FREE transition is one-way.

1 Sort D and convert N to a string s. Sort D so we can easily compare digits. k = len(s).
2 Count shorter-length numbers: For each length i = 1 to k−1, add di to the answer. Every i-digit number built from D is automatically < N (since N has k > i digits and we disallow leading zeros).
3 Walk N digit by digit (TIGHT phase): At position pos (0-indexed), count how many D-digits are strictly less than N[pos]. Call this less. Each such choice makes us FREE for the remaining k−1−pos positions, contributing less × dk−1−pos to the answer.
4 Stay TIGHT or stop: If N[pos] itself is in D, we can continue placing N[pos] and remain TIGHT. If N[pos] is NOT in D, no TIGHT number can be formed from here — stop the loop immediately.
5 If we reach the end still TIGHT: We successfully placed every digit of N from D. N itself is a valid number — add 1.
🔑 Why Is This Called "Digit DP"?

We're making decisions position by position, and our state is (current position, tight or free). This "position × constraint" structure is the hallmark of Digit DP. Instead of enumerating every integer from 1 to N (up to 109!), we exploit the structure of N's digit representation to count in O(k × d) time.

3 Why It Works: The Invariant ▾
🔐 The Invariant

During the TIGHT scan, at every position pos we maintain: the k-digit prefix we've built so far is exactly equal to N's first pos digits. This invariant means: (a) placing a digit < N[pos] breaks us free — the remaining positions can be anything from D; (b) placing N[pos] preserves the invariant for pos+1; (c) placing anything > N[pos] would exceed N — we never do this.

Why FREE contributions = less × dk−1−pos: When we go FREE at position pos (by placing a digit < N[pos]), there are k−1−pos remaining positions, each independently choosable from all d digits. Multiplication principle gives dk−1−pos completions per FREE choice. We have less such choices, so the contribution is less × dk−1−pos.

Why shorter numbers are always fully counted: Any number with fewer than k digits is a k′-digit number (k′ < k). Since N has k digits and positive numbers don't have leading zeros, any k′-digit number is ≤ 10k′ − 1 < 10k−1 ≤ N. So all d1 + d2 + ... + dk−1 are valid.

📐 The Complete Formula
answer = Σ(i=1 to k-1) d^i ← shorter numbers
+ Σ(pos=0 to k-1) count(x in D: x < N[pos]) × d^(k-1-pos) ← TIGHT scan FREE
+ (1 if every digit of N is in D, else 0) ← N itself

where the TIGHT scan stops early if N[pos] ∉ D
⚠️ Common Mistake: Off-by-one on the exponent

At position pos (0-indexed), there are k − 1 − pos positions after pos (not including pos itself). Many beginners use k − pos and double-count the current position. At the very last position (pos = k−1), the exponent is 0, meaning d0 = 1 — exactly one way to complete a number with no remaining positions.

Correctness: Every number counted in "shorter" has exactly i < k digits (can't be N). Every number from the FREE contributions is strictly less than N by construction (digit at pos is strictly less than N[pos]). N itself is counted only when explicitly validated. No number > N is ever counted. The sets are disjoint. QED.

4 The Code (Annotated) ▾

Solution 1 — Math Digit DP (Optimal, Interview Pick)

def atMostNGivenDigitSet(digits: list[str], n: int) -> int:
s = str(n) ← Convert N to string; each char is one digit of N
k = len(s) ← k = number of digits in N (= log₁₀(N) + 1)
d = len(digits) ← |D| = size of allowed digit set
result = 0 ← Accumulates the count
# ── Part 1: All numbers with fewer digits than N ──
power = d ← Starts as d^1 for 1-digit numbers
for i in range(1, k): ← i = 1, 2, ..., k-1 (lengths shorter than N)
result += power ← d^i numbers of length i are all valid
power *= d ← power = d^(i+1) for next iteration
# ── Part 2: k-digit numbers ≤ N, digit by digit ──
for i, ch in enumerate(s): ← ch = N's digit at position i
# Count D-digits strictly less than N's digit here
less = sum(1 for x in digits if x < ch) ← How many choices go FREE
result += less * (d ** (k - 1 - i)) ← Each FREE choice: d^(remaining) completions
if ch not in digits: ← N's digit is not in D → can't stay TIGHT
return result ← No TIGHT path possible; return early
# ch IS in D: stay TIGHT, continue to next position
# Survived all positions in TIGHT state: N itself is valid
result += 1 ← N itself counts as a valid constructable number
return result
⚠️ Why d ** (k - 1 - i) and not d ** (k - i)?

At position i (0-indexed in a k-digit number), there are k − 1 − i positions remaining after position i. For the last digit (i = k−1): k−1−i = 0, so d0 = 1 — each FREE choice at the last position adds exactly 1 valid number. Using k−i instead would give d1 = d at the last position, wrongly counting d times.

Solution 2 — Recursive Digit DP with Memoization (more general pattern)

def atMostNGivenDigitSet(digits: list[str], n: int) -> int:
s = str(n); D = sorted(digits)
from functools import lru_cache
@lru_cache(None)
def dp(pos, tight): ← State: (position, is_tight?)
if pos == len(s): return 1 ← Built a complete k-digit number ≤ N
limit = s[pos] if tight else '9' ← Upper bound digit at this pos
count = 0
for dig in D: ← Try each digit; D is sorted → early break
if dig > limit: break
count += dp(pos+1, tight and dig == limit) ← Stay tight iff dig == N[pos]
return count
# Same-length numbers (k digits)
result = dp(0, True)
# Add all shorter-length numbers
pw = len(D)
for _ in range(len(s) - 1):
result += pw; pw *= len(D)
return result
💡 Complexity Summary

Solution 1 (Math): Time O(k × |D|) — one pass per digit of N, one scan of D per digit. Space O(k) for the string. k = ⌊log₁₀(N)⌋ + 1 ≤ 10 for N ≤ 109.
Solution 2 (Memoized DP): States = (pos, tight) — 2k states × |D| transitions = O(k × |D|). Space O(k) for call stack + cache. Same asymptotically, but more general (easy to add extra state dimensions like digit sum).
Brute Force: O(N × k) — TLE for N up to 109.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize "Count integers ≤ N with digit constraint"

The problem says "how many numbers ≤ N can be formed using digits D." N can be up to 109, so brute force (check each 1..N) is O(N) — too slow. The constraint is on individual digits (which digits are allowed). These two signals — upper bound N, digit-based constraint — are the canonical indicators of Digit DP.

Step 2: Decompose by number of digits

Numbers with fewer digits than N are trivially ≤ N (no comparison needed). Count them as power sums of |D|. Numbers with exactly len(N) digits need careful handling — this is where the DP comes in.

Step 3: Introduce the TIGHT constraint

For same-length numbers, process digit by digit. At each position, you either go FREE (by placing a digit smaller than N's digit — adds |D|^remaining options) or stay TIGHT (by placing exactly N's digit, if it's in D). If you can't do either, return early. This is the "tight DP" skeleton that generalizes to many digit constraint problems.

Step 4: Handle the edge case — N itself

If you complete all positions still TIGHT (every digit of N was in D), then N itself is a valid constructable number. Add 1. Beginners often forget this.

🗣️ What to Say to the Interviewer

"The key observation is that N can be up to 10^9, so we can't check each number.
I'll split valid numbers into two groups: those shorter than N (easy power sums), and those with exactly len(N) digits.
For the second group, I walk through N's digits left to right with a 'tight' flag.
At each position: count how many digits in D are strictly less than N's digit here — each goes 'free' with |D|^remaining completions.
If N's digit itself is in D, stay tight and continue. If not, no tight path remains — return.
If I finish all digits still tight, N itself is valid — add 1.
Total time: O(log N × |D|). For N up to 10^9, log N ≤ 10, so this is essentially constant."
🔮 Likely Follow-up Questions
  • "Count integers in range [L, R]?" → f(R) − f(L−1), where f(x) uses this algorithm
  • "What if digits can appear at most once?" → Track used-digit bitmask in DP state
  • "Count numbers with digit sum = k up to N?" → Add current sum as DP state dimension
  • "What if N is given as a string (very large)?" → Same algorithm; it already works on the string representation
  • "Non-adjacent digits constraint?" → Add last-placed-digit to DP state

7 Summary & Key Takeaways ▾
Split numbers into: shorter-than-N (count as power sums) + same-length-as-N (digit DP)
The TIGHT → FREE state transition is the core of every Digit DP problem
At each TIGHT position: count(D < N[pos]) × |D|^(remaining) = FREE contribution
Stop TIGHT scan early if N[pos] is not in D (no valid TIGHT path from here)
If all digits of N are in D, N itself is a valid number — add 1
Time O(k × |D|) ≈ O(log N × |D|) — handles N up to 109 trivially (k ≤ 10)

🎯 When to Recognize Digit DP
  • "Count integers ≤ N satisfying some digit-based property P" → Digit DP
  • N can be very large (up to 1018) → brute force is O(N), too slow
  • Property P depends on individual digits (their values, sums, uniqueness, ordering)
  • State = (position, tight_flag, [other per-digit constraint]) with DP over positions
