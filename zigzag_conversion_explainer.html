LeetCode #6

Zigzag Conversion

Medium String Processing Row Index Oscillation O(n) Time · O(n) Space
1 Understanding the Problem

Write the string s diagonally in a zigzag pattern across numRows rows — like a snake moving down then up, down then up. Then read every row left-to-right and concatenate them to form the output.

For s = "PAYPALISHIRING" and numRows = 3:

P··· A··· H··· N·
·A·P ·L·S ·I·I ·G
··Y· ··I· ··R· ··
Read → PAHN + APLSIIG + YIR = "PAHNAPLSIIGYIR"
💡 Key Insight: We Don't Need a 2D Grid

We never actually write characters into a 2D matrix. Instead, we keep one string buffer per row and simply append each character to its responsible row as we scan s from left to right. The zigzag path is simulated by a bouncing row index that oscillates between 0 and numRows-1.

⚠️ Edge Cases to Handle First

numRows = 1: every character stays in row 0 — output equals input.
numRows ≥ len(s): each char gets its own row — output also equals input.
In both cases, return s immediately and skip all the logic.

2 The Algorithm: Row Buffer Simulation

Instead of simulating the 2D grid, simulate only the row index. Maintain a current row number and a direction (+1 for going down, −1 for going up). Each character gets appended to the buffer for its current row.

1 Edge-case guard: if numRows == 1 or numRows ≥ len(s), return s immediately.
2 Create buffers: rows = [''] * numRows — one empty string per row.
3 Initialize state: cur = 0 (start at row 0), direction = +1 (going DOWN).
4 Walk the string: for each character, append it to rows[cur]. Then check boundaries: if cur == 0, set direction = +1; if cur == numRows-1, set direction = −1. Finally advance: cur += direction.
5 Join and return: ''.join(rows) concatenates all row buffers in order, row 0 first.
📐 The Period Formula
period = 2 × (numRows − 1)
The zigzag pattern repeats every period characters. For numRows=3: period=4. Row 0 chars are at indices 0, 4, 8, 12 (every period). Row 2 chars are at 2, 6, 10 (period/2 offset). Middle rows have two chars per period. This formula is the basis of the mathematical O(1)-space approach.
3 Why It Works: The Invariant
🔐 The Invariant

At every step, rows[0..numRows-1] collectively contain exactly the characters s[0..i-1] in their left-to-right reading order. No character is ever skipped, duplicated, or placed in the wrong row.

Why append-then-advance is correct: We append s[i] to rows[cur] and then check boundaries. This means the boundary check sees the row after the write and correctly sets the direction for the next character — never skipping a row.

Why the join produces the right string: Row 0 holds exactly the characters that appear first when reading left-to-right across the zigzag grid. Row 1 holds the next characters, and so on. Concatenating rows in order preserves this reading sequence exactly.

❌ If You Flip Direction Before Appending

When cur hits row 2, flip direction first,
then append → char lands on row 1 instead!
Result: wrong assignment for boundary chars.

✓ Append First, Then Check Boundary

Append char at cur (e.g. row 2),
then flip direction = -1,
then advance → next char at row 1. ✓

4 The Code (Annotated)
# O(n) time, O(n) space — optimal for this problem def convert(s: str, numRows: int) -> str: # Short-circuit: no zigzag possible in 1 row or oversized numRows if numRows == 1 or numRows >= len(s): return s rows = [''] * numRows # One string buffer per row cur, direction = 0, 1 # Start at row 0, going DOWN (+1) for char in s: rows[cur] += char # Append to the current row if cur == 0: # Hit top → must go DOWN direction = 1 elif cur == numRows - 1: # Hit bottom → must go UP direction = -1 cur += direction # Move to next row return ''.join(rows) # Concat rows 0..numRows-1
⚠️ Why Not Check cur == 0 with an elif?

The if/elif here is purposeful: cur == 0 and cur == numRows-1 are mutually exclusive (a string with numRows ≥ 2 can't have row 0 == row numRows-1). Using if/elif is correct and slightly faster since we avoid checking both conditions when one is true. If numRows == 1, we already returned early, so this edge case never reaches the loop.

⏱ Complexity

Time: O(n) — single pass through s.
Space: O(n) — total chars across all row buffers equals len(s). This is optimal since the output itself is O(n).

5 🎤 Interview Strategy: How to Arrive at This
Step 1: State Edge Cases Upfront

Immediately say: "If numRows is 1 or ≥ len(s), the output equals the input — no transformation needed." This shows you're thinking about constraints before diving into code.

Step 2: Reject the 2D Matrix Approach

Interviewers expect you to see through the "grid" framing. Say: "I could allocate a 2D matrix, but that wastes space. Instead, I only need one buffer per row — the column index never matters for reading the output."

Step 3: Define the Bouncing State

Explain: "I'll track a current row index that oscillates between 0 and numRows-1. A direction variable (+1 / -1) controls whether we move down or up. Flip it when we hit either boundary."

Step 4: Offer the Mathematical Extension

Bonus: "There's also an O(1) space approach using math: period = 2*(numRows-1). For each row, I can directly calculate which source indices map to it. But it adds code complexity for the same O(n) time, so the row-simulation is usually preferred."

🗣️ What to Say to the Interviewer

"The key realization is that we don't need columns — just rows. I'll walk through the string once, maintaining a current row index that bounces between 0 and numRows-1. I append each char to its row's buffer and flip the direction when hitting a boundary. Then I join all buffers. Edge cases first: numRows==1 or numRows≥n both return s unchanged."

def convert(s, numRows):
if numRows == 1 or numRows >= len(s): return s
rows = [''] * numRows
cur, direction = 0, 1
for char in s:
rows[cur] += char
if cur == 0: direction = 1
elif cur == numRows - 1: direction = -1
cur += direction
return ''.join(rows)

7 Summary & Key Takeaways
The "2D grid" framing is a red herring — one string buffer per row is all you need
A single direction variable (+1/−1) controls the zigzag oscillation cleanly
Always append to rows[cur] before checking boundaries — not after
Edge cases first: numRows==1 and numRows≥len(s) both return s
The period formula 2*(numRows-1) enables an O(1) space math-only solution
🎯 When to Use This Pattern (Row Index Oscillation)
  • Any problem that involves writing/reading characters in a "wave" or "snake" pattern
  • Problems where a 2D layout is described but the column index never matters
  • Interview hint: the phrase "write in a zigzag/wave pattern, read row by row"
  • Variants: different step sizes, diagonal patterns, spiral reading
