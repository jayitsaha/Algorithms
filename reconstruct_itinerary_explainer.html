LeetCode #332

Reconstruct Itinerary

Given a list of airline tickets [from, to], find an itinerary that uses all tickets starting from "JFK". If multiple valid orderings exist, return the lexicographically smallest one.

Hard Graph Hierholzer's Algorithm Time: O(E log E) Space: O(E)
1 Understanding the Problem ‚ñæ

You hold a stack of airline tickets, each as a [from, to] pair. You must use every single ticket exactly once, starting at "JFK". Return the travel order ‚Äî and if multiple valid orderings exist, pick the lexicographically smallest one (treat it like dictionary order on airport codes).

üí° Key Insight

This is an Eulerian path problem on a directed graph. Every ticket is a directed edge. We need a trail through the graph that uses each edge exactly once ‚Äî starting from JFK. The classic algorithm for this is Hierholzer's, which uses a post-order DFS trick to handle cycles correctly.

‚úì Valid Example

tickets = [
 ["JFK","ATL"],
 ["ATL","JFK"],
 ["JFK","SFO"],
 ["SFO","SJC"]
]
Output: ["JFK","ATL","JFK","SFO","SJC"]

Edge Cases: Cycles (JFK‚ÜíATL‚ÜíJFK), multiple edges between same airports, airports that are dead ends (must be visited last), lexicographic tie-breaking at every fork.

üîë Eulerian Path Condition

A directed graph has an Eulerian path if: exactly one vertex has out_degree ‚àí in_degree = 1 (start), at most one has in_degree ‚àí out_degree = 1 (end), and all others have equal in/out degree. The problem guarantees a valid itinerary exists, so these conditions are always met.

2 The Algorithm: Hierholzer's via Post-Order DFS ‚ñæ

The key idea is post-order insertion: we add an airport to the result only after exhausting all of its outgoing edges. Then we reverse. This means dead-end airports end up in the result first ‚Äî which, after reversal, places them correctly at the end of the itinerary.

1 Build sorted adjacency list: For each ticket [src, dst], add dst to graph[src]. Sort tickets before insertion so that each airport's destinations are in lexicographic order. This ensures we always try the smallest destination first.
2 DFS with stack: Push "JFK" onto a stack. While the top of the stack has outgoing edges, pop the smallest destination (the first in the sorted list) and push it onto the stack ‚Äî marking that edge as used by removing it from the adjacency list.
3 Post-order pop: When the current top of stack has no more outgoing edges (it's a dead end, or the path ends here), pop it from the DFS stack and append it to the result list. This is the Hierholzer trick.
4 Reverse at the end: The result list accumulates airports in post-order ‚Äî deepest nodes first, JFK last. Reversing it gives the correct itinerary in forward order: JFK first, final destination last.
üîë Why Post-Order Handles Cycles

Consider JFK‚ÜíATL, ATL‚ÜíJFK, JFK‚ÜíSFO. A simple DFS adds JFK when visited (pre-order), enters ATL, returns to JFK via the cycle, goes to SFO ‚Äî and JFK appears in the middle! Post-order adds JFK only after ALL its edges are used: SFO, JFK (from cycle), ATL, JFK (start) ‚Üí reversed = JFK,ATL,JFK,SFO. Perfect.

3 Why It Works: The Invariant ‚ñæ
üîê The Hierholzer Invariant

When a node is appended to the result (the post-order pop moment), every edge reachable from that node has already been fully explored, and every airport reachable via those edges is already in the result. Reversing the result then places this node before all its destinations ‚Äî which is exactly the correct order for the itinerary.

Why dead ends go first (before reversal): If airport B is a dead end ‚Äî no outgoing edges ‚Äî it gets popped to result immediately when visited. Since B must appear at the end of any path that reaches it, placing it first in the result (and reversing) puts it last. Correct!

Why cycles are handled correctly: In the example JFK‚ÜíATL‚ÜíJFK‚ÜíSFO‚ÜíSJC: DFS goes JFK, ATL, JFK, SFO, SJC. SJC (dead end) pops first, then SFO, then JFK (second), then ATL, then JFK (first). Result before reverse: [SJC, SFO, JFK, ATL, JFK]. Reversed: [JFK, ATL, JFK, SFO, SJC]. Every ticket used, lexicographically smallest, correct order.

‚öÝÔ∏è Common Mistake: Pre-Order Insertion

If you append an airport when you first visit it (pre-order), cycles cause incorrect results. JFK gets added at the start, ATL next, then JFK again (from the cycle) ‚Äî putting JFK in two wrong positions. Always append post-order (when no more edges remain from that node).

Lexicographic guarantee: By sorting all destinations before the DFS and always consuming the smallest first (pop(0) or heapq.heappop), the post-order reversal naturally produces the lexicographically smallest valid itinerary ‚Äî no need to compare multiple complete paths.

4 The Code (Annotated) ‚ñæ

Solution 1 ‚Äî Hierholzer's Iterative (Interview Pick, O(E log E))

from collections import defaultdict
def findItinerary(tickets): ‚Üê tickets = list of [from, to] pairs
graph = defaultdict(list) ‚Üê adjacency list: airport ‚Üí sorted list of destinations
for src, dst in sorted(tickets): ‚Üê sort tickets so destinations are in lex order
graph[src].append(dst) ‚Üê each ticket adds one directed edge
result = [] ‚Üê will hold itinerary in reverse (post-order)
stack = ["JFK"] ‚Üê DFS stack; always start from JFK
while stack: ‚Üê keep going until all airports processed
top = stack[-1] ‚Üê peek: current airport (do NOT pop yet)
if graph[top]: ‚Üê if this airport still has outgoing edges
stack.append(graph[top].pop(0)) ‚Üê push smallest dest, consume edge
else: ‚Üê no edges left: dead end or path complete
result.append(stack.pop()) ‚Üê POST-ORDER: add to result after all edges used
return result[::-1] ‚Üê reverse: post-order built backwards ‚Üí flip
‚öÝÔ∏è pop(0) is O(n) ‚Äî Use heapq for large inputs

List pop(0) shifts all remaining elements ‚Äî O(n) per pop, giving O(E¬≤) total. For large E, build adjacency with heapq.heappush(graph[src], dst) then replace pop(0) with heapq.heappop(graph[top]). This gives the guaranteed O(E log E) time complexity.

Solution 2 ‚Äî Recursive DFS (same complexity, cleaner)

from collections import defaultdict, deque
def findItinerary(tickets):
graph = defaultdict(deque) ‚Üê deque for O(1) popleft()
for src, dst in sorted(tickets):
graph[src].append(dst)
result = []
def dfs(airport):
while graph[airport]: ‚Üê exhaust all outgoing edges first
dfs(graph[airport].popleft()) ‚Üê recurse into smallest destination
result.append(airport) ‚Üê POST-ORDER: add only after all edges gone
dfs("JFK")
return result[::-1] ‚Üê same reversal as iterative version
üî¨ Algorithm Deep-Dive: Hierholzer's Algorithm

Origin: Carl Hierholzer, 1873. Finds Eulerian circuits/paths in O(E) time. The DFS + post-order approach is the modern equivalent of his original two-phase algorithm.

Core invariant: When a vertex is appended (post-order), all edges reachable from it are already committed to the result. Reversal then places the vertex before all its successors ‚Äî the correct edge direction.

Why it generalizes: For Eulerian circuits, every vertex has equal in/out degree ‚Äî the DFS returns to start. For Eulerian paths, the start has one extra out-edge ‚Äî the DFS terminates at the end vertex. Hierholzer's handles both identically.

Recognize when: "Use every edge exactly once" ¬∑ "Traverse all connections" ¬∑ Eulerian path/circuit ¬∑ "Reconstruct a sequence from pairs."

üí° Complexity Summary

Brute-force backtracking: O(E!) ‚Äî tries all permutations of edge use
Hierholzer's (with heapq): O(E log E) ‚Äî sort once, each edge visited once
Space: O(E) for graph storage + O(E) stack depth in worst case

5 üé§ Interview Strategy: How to Arrive at This ‚ñæ
Step 1: Model as a Directed Graph

"Each ticket is a directed edge: from ‚Üí to. Airports are nodes. I need to traverse all edges from JFK." Frame it as graph traversal immediately ‚Äî this shows pattern recognition.

Step 2: Explain Why Greedy Fails

"Greedy 'always pick smallest next airport' can get trapped in a sub-graph, leaving unreachable edges." Draw a small counter-example ‚Äî e.g., two paths where picking lexicographically smallest first closes off other edges too early.

Step 3: Name and Explain Hierholzer's

"I'll use Hierholzer's algorithm. Sort adjacency lists. DFS: push next smallest destination until dead end. On dead end: pop to result. Reverse at end. The post-order ensures cycles are handled." Naming the algorithm shows depth.

Step 4: Discuss Data Structure Trade-offs

"list.pop(0) is O(n) ‚Äî for large inputs I'd use heapq for O(log n) pops. Or sort the full list upfront and reverse each adjacency list to use pop() from the end instead." This shows you think about constants.
