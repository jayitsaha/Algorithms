LeetCode #115

Distinct Subsequences

Count the number of distinct ways to form string t as a subsequence of string s.

Hard Dynamic Programming 2D String DP Time: O(m·n) Space: O(n)
1 Understanding the Problem ▾

Given two strings s and t, count the number of distinct ways you can pick characters from s (in order, without reordering) to spell out t. Each selection must preserve the original left-to-right order of characters in s.

💡 Key Insight

A subsequence keeps characters in their original order but does not require them to be contiguous. "Distinct" means we count by which index positions we chose from s, not just whether the characters matched. Two selections using different index sets are always distinct — even if they produce the same characters.

✓ s="rabbbit", t="rabbit" → 3

r·a·b·b·b·i·t → skip 3rd b
r·a·b·b·b·i·t → skip 2nd b
r·a·b·b·b·i·t → skip 1st b
Answer: 3

Edge Cases

t = "" → 1 (pick nothing)
s = "" → 0 (can't form t)
len(s) < len(t) → 0
s = t → exactly 1
s = "aaa", t = "a" → 3

Why is this Hard? The same character can appear many times in s. Each occurrence is a branching choice: use this one for the current t-position, or skip it and hope for another later. Brute-force recursion explores exponentially many branches. DP collapses them in O(m·n).

🔑 The Mental Model: Filling Slots

Imagine t has n "slots" to fill, left-to-right. Walk through s from left to right. When you see a character that matches the current unfilled slot, you have a choice: fill this slot now, or wait for a later occurrence. DP tracks all possible slot-filling combinations simultaneously.

2 The Algorithm: 2D Bottom-Up DP ▾

We build a 2D table where each cell encodes a count of ways, then read off the answer from the bottom-right corner. No fancy data structures — just a table and a recurrence.

1 Define state: dp[i][j] = number of distinct ways to form t[0..j-1] using only s[0..i-1]. Rows index into s (1-indexed), columns index into t (1-indexed), so row 0 and column 0 represent the empty string.
2 Base cases: dp[i][0] = 1 for all i (one way to form the empty string: select nothing). dp[0][j] = 0 for j > 0 (impossible to form any non-empty t from empty s).
3 No match — s[i-1] ≠ t[j-1]: We cannot use s[i-1] for this t-slot. The count is exactly what it was without s[i-1]: dp[i][j] = dp[i-1][j].
4 Match — s[i-1] == t[j-1]: Two mutually exclusive choices. Use s[i-1] (adds dp[i-1][j-1] ways — we now also need to form t[:j-1] from s[:i-1]). Or skip s[i-1] (adds dp[i-1][j] ways — ignore this char). Sum both: dp[i][j] = dp[i-1][j-1] + dp[i-1][j].
5 Answer: dp[m][n] — ways to form all of t using all of s.
📐 Recurrence Relations
dp[i][0] = 1     for all i               (base: empty t)
dp[0][j] = 0     for j > 0            (base: empty s)

if s[i-1] ≠ t[j-1]: dp[i][j] = dp[i-1][j]
if s[i-1] == t[j-1]: dp[i][j] = dp[i-1][j-1] + dp[i-1][j]

3 Why It Works: The Invariant ▾
🔐 The DP Invariant

At all times, dp[i][j] holds the exact count of distinct index-sets from s[0..i-1] that spell out t[0..j-1]. Every cell we write is computed from already-finalized cells (smaller i, same j; or same i, smaller j — but we only read from row i-1), so no cell is computed before its dependencies.

Why the two cases are exhaustive:

  • No match: s[i-1] cannot contribute to t[j-1]'s slot. Every valid selection must exclude index i-1. Those selections are exactly dp[i-1][j] — same t-slots filled, from a shorter s.
  • Match: Two disjoint scenarios. (A) We include index i-1 to fill t[j-1]. We then need t[0..j-2] filled from s[0..i-2] → dp[i-1][j-1] ways. (B) We exclude index i-1 even though it matches → dp[i-1][j] ways. The sets are disjoint (one includes position i-1, the other doesn't), so we add.

Optimal Substructure: dp[i][j] depends only on dp[i-1][j] and dp[i-1][j-1] — strictly smaller subproblems. No circular dependency.

Overlapping Subproblems: A recursive approach on s="aaa…", t="aa…" would recompute the same (i,j) pair exponentially many times. DP computes each pair exactly once → O(m·n).

⚠️ Common Mistake: Forgetting the Skip Term on Match

Beginners write dp[i][j] = dp[i-1][j-1] when s[i-1]==t[j-1], omitting +dp[i-1][j]. This misses all selections that skip the current matching character and use a later occurrence instead. For s="rabbbit" → "rabbit" this gives 1 instead of 3.

4 The Code (Annotated) ▾

Solution 1 — 2D DP Tabulation (Interview Pick, O(m·n) time, O(m·n) space)

def numDistinct(s: str, t: str) -> int:
m, n = len(s), len(t) ← m rows for s, n cols for t
dp = [[0]*(n+1) for _ in range(m+1)]← (m+1)×(n+1) table, all zeros
for i in range(m+1): ← Base case: 1 way to form "" from any prefix of s
dp[i][0] = 1 ← Pick nothing; always valid
for i in range(1, m+1): ← For each character of s (1-indexed)
for j in range(1, n+1): ← For each character of t (1-indexed)
dp[i][j] = dp[i-1][j] ← Always: skip s[i-1] (carry forward count without it)
if s[i-1] == t[j-1]: ← If characters match, we may ALSO use s[i-1]
dp[i][j] += dp[i-1][j-1]← Add: ways where we DO use this match
return dp[m][n] ← Full s vs full t → the answer
⚠️ Why dp[i][j] starts as dp[i-1][j]?

We ALWAYS have the option to skip s[i-1]. Every way we could form t[:j] from s[:i-1] is also a valid way from s[:i] (just don't use the new character). This is the skip-baseline. The match term only adds additional options on top.

Solution 2 — Space-Optimized 1D DP (O(m·n) time, O(n) space)

def numDistinct(s: str, t: str) -> int:
n = len(t)
dp = [0] * (n + 1) ← dp[j] = ways to form t[:j] so far
dp[0] = 1 ← 1 way to form empty string
for c in s: ← Process each char of s
for j in range(n, 0, -1): ← RIGHT-TO-LEFT: avoid reusing same s-char
if c == t[j-1]:
dp[j] += dp[j-1] ← dp[j-1] still holds OLD value (before this s-char)
return dp[n]
💡 Why Traverse j Right-to-Left?

We need the pre-update value of dp[j-1] (equivalent to dp[i-1][j-1] in 2D). Going left-to-right would overwrite dp[j-1] before we read it for dp[j], effectively reusing the same s-character multiple times (like unbounded knapsack). Right-to-left ensures we always read from the "previous row".

Solution 3 — Top-Down Memoization (for building intuition)

from functools import lru_cache
def numDistinct(s: str, t: str) -> int:
@lru_cache(maxsize=None)
def dp(i, j): ← Ways to form t[j:] using s[i:]
if j == len(t): return 1 ← All t matched → success
if i == len(s): return 0 ← Exhausted s, t remains → fail
res = dp(i+1, j) ← Always: skip s[i]
if s[i] == t[j]:
res += dp(i+1, j+1) ← Also: use s[i] to match t[j]
return res
return dp(0, 0)
5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the DP Signal

"Count the number of distinct ways" — this exact phrase is the loudest DP signal in interviews. Combine it with "subsequence" (order-preserving selection) and you immediately have a 2D string DP problem. No greedy exists here because each matching character gives a binary choice (use it now or skip it).

Step 2: Define the State

What changes at each step? How much of s we've seen (i) and how much of t we've filled (j). State = (i, j). Value = count of distinct ways. This gives a 2D table indexed [0..m][0..n].

Step 3: Write the Recurrence from Scratch

For each cell (i,j): "Can I use s[i-1] for t[j-1]?" If YES → use it (adds dp[i-1][j-1]) OR skip it (adds dp[i-1][j]). If NO → must skip (adds dp[i-1][j]). The skip term appears in BOTH cases — default to it, then conditionally add the match term.

Step 4: Nail the Base Cases

dp[i][0] = 1 for all i (one way to form empty string). dp[0][j] = 0 for j > 0 (can't form non-empty t from empty s). Getting these wrong corrupts every downstream cell.

🗣️ What to Say to the Interviewer

"This is a counting DP — 'number of distinct ways' is the signal.
I'll define dp[i][j] = ways to form t[:j] using s[:i].
Base: dp[i][0]=1 for all i (empty t), dp[0][j]=0 for j>0 (empty s).
Transition: dp[i][j] = dp[i-1][j] always (skip s[i-1]).
If s[i-1]==t[j-1], also add dp[i-1][j-1] (use the match).
Answer = dp[m][n]. Time O(m·n), reducible to O(n) space."
🔮 Likely Follow-up Questions
  • "Reduce space to O(n)?" → 1D DP, scan j right-to-left
  • "What if we can reuse characters in s?" → Scan left-to-right (unbounded)
  • "Count distinct subsequences of s itself?" → Different DP: dp[i] = 2·dp[i-1] − dp[last_seen[s[i]]] (#940)
  • "Is t a subsequence of s (yes/no)?" → Two-pointer greedy, no DP needed (#392)

7 Summary & Key Takeaways ▾
"Count distinct ways" + "subsequence" = 2D counting DP. Recognize this immediately.
The skip term dp[i-1][j] is ALWAYS present — even when characters match.
The match term dp[i-1][j-1] adds on top of the skip baseline only when s[i-1]==t[j-1].
Base case: empty t can always be formed in 1 way (select nothing).
Space reduces to O(n) by scanning j right-to-left in a 1D array (same as 0/1 knapsack).
Each repeated character in s multiplies possibilities — each occurrence is a distinct selection.

🎯 When to Recognize the Count-Ways 2D DP Pattern
  • "Count distinct ways / selections / sequences" → counting DP (not optimization DP)
  • Two strings, order-preserving alignment → 2D dp[i][j] table
  • Each character gives binary choice (use or skip) → sum both branches
  • Space: if dp[i] only depends on dp[i-1], reduce to 1D with right-to-left scan
