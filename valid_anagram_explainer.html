LeetCode #242

Valid Anagram

Given two strings s and t, return true if t is an anagram of s, and false otherwise.

Easy String Processing Frequency Count Time: O(n) Space: O(k)
1 Understanding the Problem ▾

An anagram is a word formed by rearranging all the letters of another word. "listen" → "silent" is an anagram. "rat" → "car" is not — 'r', 'a', 't' ≠ 'c', 'a', 'r' (one has 't', the other has 'c').

💡 Key Insight

Two strings are anagrams if and only if every character appears the same number of times in both strings. Position is irrelevant — only character frequency matters.

✓ Anagram: s="anagram", t="nagaram"

s: a×3, n×1, g×1, r×1, m×1
t: a×3, n×1, g×1, r×1, m×1
Same frequencies → True

✗ Not Anagram: s="rat", t="car"

s: r×1, a×1, t×1
t: c×1, a×1, r×1
s has 't', t has 'c' → False

Edge Cases to consider: strings of different lengths (always False), empty strings (both empty → True), strings of all-same characters like "aaa"/"aaa", and inputs containing only one character.

2 The Algorithm: Two-Pass Frequency Count ▾

We maintain a single frequency map (hash map) as a tally sheet. Increment for each letter in s, then decrement for each letter in t. If all tallies return to zero — they're anagrams.

1 Quick-fail on length: If len(s) != len(t), return False immediately. An anagram must use all original letters exactly once, so lengths must match.
2 Count s (increment pass): Iterate through s. For each character c, do freq[c] += 1. After this pass, freq maps each character to how many times it appears in s.
3 Verify t (decrement pass): Iterate through t. For each c, do freq[c] -= 1. If freq[c] drops below 0, t has used that character more times than s had it — return False immediately (fail fast).
4 Return True: If we reach the end without any count going negative (and lengths matched), every character was used exactly the right number of times — t is a valid anagram of s.
🔑 Sorting Alternative (mention first in interviews)

return sorted(s) == sorted(t) — if both strings sort to the same sequence, they're anagrams. Simple and correct, but O(n log n) time and O(n) space for the sorted copies. Propose this first, then offer to optimize to O(n).

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point during the t scan, freq[c] equals (occurrences of c in s) − (occurrences of c in t processed so far). It represents "how many more times c must still appear in the remaining t." If this goes negative, t has over-consumed c.

Why negative means failure: After counting s, freq[c] = times c appears in s. As we scan t, each use decrements the remaining budget. A negative means t demanded c more times than s provided — that's impossible for a rearrangement.

Why no final "all-zeros" check is needed: Since we already verified len(s) == len(t), and no count went negative during the t pass, the total number of decrements equals the total increments. With equal totals and no overage, all counts must be exactly 0 at the end.

⚠️ Common Trap: Checking Zeros at the End

Many beginners write return all(v == 0 for v in freq.values()) after the loops. This is correct but unnecessary — the length equality check combined with "no negatives" already guarantees all zeros. The negative check during the loop also gives us early exit on bad input, which is more efficient.

Counter-example for intuition (s="rat", t="car"): After counting s: freq = {r:1, a:1, t:1}. Scanning t: 'c' → freq.get('c',0)−1 = −1 < 0 → return False immediately. Clean and correct.

4 The Code (Annotated) ▾

Solution 1 — Frequency Count (Optimal, O(n) time)

def isAnagram(s: str, t: str) -> bool:
if len(s) != len(t): ← Quick fail: different lengths can't be anagrams
return False
freq = {} ← Tally sheet: char → (count in s) − (count in t so far)
for c in s: ← Pass 1: count each character in s
freq[c] = freq.get(c, 0) + 1 ← get(c, 0) handles unseen chars gracefully
for c in t: ← Pass 2: subtract each character in t
freq[c] = freq.get(c, 0) - 1 ← If c not in s, get returns 0, becomes -1
if freq[c] < 0: ← t used this char more than s had it → fail fast
return False
return True ← All counts balanced → valid anagram
⚠️ Why freq.get(c, 0) in the t loop instead of freq[c]?

If t contains a character that was never in s, freq[c] raises a KeyError. Using freq.get(c, 0) defaults to 0. Subtracting 1 gives −1, triggering the < 0 check. This handles the "character not in s" case cleanly without any if-guards.

Solution 2 — Sort and Compare (O(n log n), simpler)

def isAnagram(s: str, t: str) -> bool:
return sorted(s) == sorted(t) ← Rearrange both alphabetically, then compare

Pythonic one-liner (interview bonus)

from collections import Counter
def isAnagram(s: str, t: str) -> bool:
return Counter(s) == Counter(t) ← Same O(n) logic, but no early exit on negatives
💡 Complexity Summary

Frequency Count: Time O(n), Space O(k) where k = distinct chars (≤ 26 for a–z → O(1))
Sort & Compare: Time O(n log n), Space O(n) for sorted copies
Fixed Array (a–z only): count = [0]*26, index by ord(c) - ord('a') — same O(n)/O(1) with better cache locality

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify Constraints

Ask: "Can the strings contain Unicode characters or only lowercase a–z?" If only lowercase, a fixed 26-element integer array works. If Unicode, use a hash map (handles any character). This shows you think about generality.

Step 2: Propose Brute Force First

"The intuitive approach: sort both strings and compare — sorted(s) == sorted(t). O(n log n) time, O(n) space. Correct, but we can do better."

Step 3: Optimize with Frequency Count

"Instead of sorting, I can count character frequencies in one pass over s, then verify in one pass over t. If any count goes negative, t is using a character more than s had — return False. O(n) time, O(k) space."

Step 4: Mention the Fixed Array Optimization

"If we're limited to lowercase English, I'd use count = [0]*26 with index ord(c) - ord('a') instead of a dict — same complexity but better cache performance and no hashing overhead."

🗣️ What to Say to the Interviewer

"First, I'll check if lengths differ — if so, immediate False, since anagrams use all letters exactly once.
Then I need to verify both strings have the same character counts. I'll use a frequency map.
Pass 1: count up each char in s. Pass 2: count down each char in t.
If any count goes below zero during Pass 2, t has over-used that character — return False.
Otherwise return True. Time O(n), Space O(1) for lowercase English (26 possible keys)."
🔮 Likely Follow-up Questions
  • "What if the input is Unicode?" → Use a hash map (dict). Already handles it!
