LeetCode #1071

Greatest Common Divisor of Strings

For strings s and t, "t divides s" means s = t + t + ... + t (some repetitions). Return the largest string x that divides both str1 and str2.

Easy Math GCD of Lengths Time: O(n+m) Space: O(n+m)
1 Understanding the Problem ▾

The problem defines a new kind of "divisibility" for strings: t divides s if repeating t some number of times gives you exactly s. For example, "AB" divides "ABABAB" (repeat 3 times). But "AB" does NOT divide "ABABA" because 5 / 2 is not an integer — you can't tile it exactly.

💡 Key Insight

This is the string analogue of numeric GCD. Just as gcd(12, 8) = 4 is the largest number dividing both, the "GCD string" is the longest string that divides both str1 and str2. Crucially, its length must equal gcd(len(str1), len(str2)) — the same Euclidean algorithm we use for numbers works here too.

✓ Has GCD: str1="ABABAB", str2="ABAB"

"AB" × 3 = "ABABAB" ✓
"AB" × 2 = "ABAB" ✓
gcd(6, 4) = 2 → answer = "AB"

✗ No GCD: str1="LEET", str2="CODE"

"LEET"+"CODE" = "LEETCODE"
"CODE"+"LEET" = "CODELEET"
Not equal → return ""

Edge cases: If no common divisor exists (like "LEET" and "CODE"), return "". If one string fully divides the other (str2 = "ABC", str1 = "ABCABC"), the GCD is str2. If both are identical (str1 = str2 = "ABC"), the GCD is the whole string. Single repeated character strings like "AAAA" and "AA" trivially share "A" or "AA".

2 The Algorithm: Concatenation Test + GCD of Lengths ▾

The optimal approach elegantly maps the string problem to number theory. It has just two steps: first verify that a common base exists, then find how long it must be.

1 Concatenation test (existence check): If str1 + str2 != str2 + str1, return "". This tests whether the two strings are "commutative" under concatenation. It is both necessary AND sufficient: a common base exists if and only if the concatenations are equal. This is a proven mathematical result (related to the Fine and Wilf theorem).
2 GCD of lengths: Compute g = math.gcd(len(str1), len(str2)). Since the common base must have a length that divides both string lengths, and we want the longest such base, g is exactly the answer length. The Euclidean algorithm gives this in O(log n) time.
3 Extract the answer: Return str1[:g]. Because the concatenation test passed, the first g characters of str1 (which equals str2's first g characters) is guaranteed to be the common base pattern. No extra validation needed.
Euclidean GCD Trace: gcd(6, 4)
gcd(6, 4) → 6 mod 4 = 2 → gcd(4, 2)
gcd(4, 2) → 4 mod 2 = 0 → gcd(2, 0) → 2
str1[:2] = "AB" → Answer!
🔑 Why does len(answer) = gcd(len(str1), len(str2))?

If str1 repeats base k1 times and str2 repeats it k2 times, then len(str1) = k1 × len(base) and len(str2) = k2 × len(base). So len(base) is a common divisor of both lengths. To get the greatest common divisor string, we want the maximum possible len(base), which is exactly gcd(len(str1), len(str2)).

3 Why It Works: The Invariant ▾
🔐 The Key Theorem (Proof Sketch)

Forward direction (easy): If str1 and str2 are both built from base b, then str1 = b × k1, str2 = b × k2. So str1 + str2 = b × (k1 + k2) = str2 + str1. Concatenation is commutative.

Backward direction (harder, but true): If str1 + str2 == str2 + str1, then the two strings must share a common "period" — they are both repetitions of some base string. This follows from a theorem in combinatorics on words: if uv = vu for strings u and v, then both are powers of a common string.

Why no extra verification is needed: Once we know the concatenation test passes (base exists) and compute g = gcd(len(str1), len(str2)), the prefix str1[:g] is guaranteed correct. We don't need to explicitly check that this prefix divides both strings — the math guarantees it.

⚠️ Common Mistake: Skipping the Concatenation Test

If you skip the concat test and just return str1[:gcd_len], you'll get wrong answers. For str1="LEET", str2="CODE": gcd(4,4)=4, str1[:4]="LEET" — but "LEET" doesn't divide "CODE"! The test is not optional. Always check str1+str2 == str2+str1 first.

Intuition via small example: str1="AAB", str2="AB". gcd(3,2)=1, so we'd try "A" as the base. But "A" × 3 = "AAA" ≠ "AAB". And indeed, the concatenation test fails: "AAB"+"AB" = "AABAB" ≠ "AB"+"AAB" = "ABAAB". The test catches this instantly.

Identity strings edge case: str1 = str2 = "ABCABC". str1+str2 = str2+str1 trivially. gcd(6,6) = 6. Answer = "ABCABC" — the whole string divides itself (1 repetition).

4 The Code (Annotated) ▾

Solution 1 — Optimal: Concatenation Check + GCD (Interview Pick)

from math import gcd ← Python's built-in Euclidean GCD, O(log n)
def gcdOfStrings(str1: str, str2: str) -> str:
if str1 + str2 != str2 + str1: ← Concat test: no common base → early return
return "" ← No GCD string exists between these two
g = gcd(len(str1), len(str2)) ← GCD of lengths = length of the answer
return str1[:g] ← Prefix of length g is the base pattern
⚠️ Why can we return str1[:g] without verifying it?

The concatenation test already proves a common base exists. Since the base must have length gcd(len(str1), len(str2)) = g, and str1 starts with that base, str1[:g] IS the base. We get this for free — no loop needed to double-check. This is the elegance of this approach.

Solution 2 — Alternative: Brute Force Prefix Search

def gcdOfStrings(str1: str, str2: str) -> str:
def divides(cand, s): ← Check if cand tiles s exactly
k, r = divmod(len(s), len(cand)) ← How many times? Any remainder?
return r == 0 and cand * k == s ← Length fits AND content matches
for l in range(min(len(str1),len(str2)), 0, -1):← Try from longest prefix downward
cand = str1[:l] ← Candidate = prefix of length l
if divides(cand, str1) and divides(cand, str2):
return cand ← First found is the longest (going down)
return "" ← No divisor found
💡 Complexity Summary

Optimal (Concat + GCD): Time O(n+m) for string concatenation comparison. Space O(n+m) for the two concatenated strings.

Brute Force: Time O(min(n,m) × (n+m)) — trying up to min(n,m) prefixes, each comparison O(n+m). Space O(n) for candidate × k.

In an interview, propose brute force first, then offer the O(n+m) solution with the "commutativity insight."

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Analogy to Numeric GCD

The problem explicitly uses the word "divisor" for strings. This is a strong signal — map the string problem to the numeric GCD you already know. The "greatest common divisor string" must have a length equal to gcd(lengths). Say this insight out loud to the interviewer early.

Step 2: Propose Brute Force

"First approach: try each prefix of the shorter string as a candidate, check if it tiles both strings. Return the longest one that works. Time O(n² ) in worst case." This shows correctness reasoning and sets up the optimization.

Step 3: Discover the Concatenation Observation

"To optimize, I ask: when does a common base exist at all? If str1 = b × k1 and str2 = b × k2, then str1 + str2 = b × (k1+k2) = str2 + str1. So commutativity of concatenation is a necessary condition. It turns out it's also sufficient — one check replaces iterating over all prefix lengths."

Step 4: Apply gcd(lengths) and Extract

"Once we know the base exists, its length divides both len(str1) and len(str2). The greatest divisor of two numbers is math.gcd of them. So g = gcd(len(str1), len(str2)), and the answer is str1[:g]."

🗣️ What to Say to the Interviewer

"I notice the problem uses 'divisor' language for strings, similar to numeric GCD.
First I'll check if a common base even exists: str1 + str2 must equal str2 + str1.
If not, return empty string immediately.
If yes, the base length must be gcd(len(str1), len(str2)) — same Euclidean algorithm.
Return str1 up to that length. Total time O(n+m), space O(n+m) for the concat comparison."
🔮 Likely Follow-up Questions
  • "Can you prove the concat test is sufficient?" → Theorem: if uv = vu for non-empty strings, both are powers of a common string.
  • "What about 3 strings?" → Check pairwise commutativity and use gcd of all three lengths: gcd(gcd(n1,n2), n3).
  • "What if we need the LCM string (smallest common multiple)?" → Use lcm(len(str1),len(str2)) as the target length.
  • "What if str1 or str2 is empty?" → Empty string divides everything; empty + str2 == str2 + empty always, gcd(0, n) = n, answer is the non-empty string.
