LeetCode #10

Regular Expression Matching

Given string s and pattern p, implement regex matching supporting . (any single char) and * (zero or more of preceding element). The match must cover the entire string.

Hard Dynamic Programming 2D DP · Dot and Star Time: O(m·n) Space: O(m·n)
1 Understanding the Problem ▾

We are implementing a strict subset of regular expression matching. The pattern p can contain ordinary letters, . (matches any single character), and * (zero or more of the immediately preceding element). The match must cover the entire string s — not a prefix, not a substring, but all of it.

💡 Key Insight

The * is always paired with its predecessor. The unit a* means "zero or more a's". The unit .* means "zero or more of any character". When you see a *, look one step back in p to know what it multiplies. This two-character grouping is the entire difficulty of this problem.

✓ Matches

s="aa", p="a*" → True
s="ab", p=".*" → True
s="aab", p="c*a*b" → True
s="", p="a*b*" → True

✗ Does Not Match

s="aa", p="a" → False
s="ab", p=".*c" → False
s="aaa", p="ab*a" → False
s="a", p="b*" → False

Edge cases to consider: empty pattern with non-empty string (False), empty pattern with empty string (True), patterns like a*b*c* that can match empty string via zero repetitions, and .* which matches any string including empty.

⚠️ This is NOT Wildcard Matching (LeetCode #44)

In Wildcard Matching, * alone matches any sequence. Here, * modifies its predecessor. So * by itself is meaningless — it must follow a character or .. The constraint guarantees p is a valid pattern (no leading *).

2 The Algorithm: 2D Bottom-Up DP ▾

Define dp[i][j] = True if pattern prefix p[0..j-1] fully matches string prefix s[0..i-1]. Build a table of size (m+1) × (n+1) where m = len(s), n = len(p). Fill row by row, left to right. Answer is dp[m][n].

1 Base case: dp[0][0] = True. Also fill first row: for patterns like c*a*b* that can match empty string by zero-repeating each x* unit, dp[0][j] = dp[0][j-2] when p[j-1] == '*'.
2 Literal or dot — single character match: If p[j-1] == '.' or p[j-1] == s[i-1], then dp[i][j] = dp[i-1][j-1]. Both the string character and pattern character are consumed together — we look diagonally.
3 Star — zero occurrences: dp[i][j] = dp[i][j-2]. We discard the entire x* unit (two pattern positions) and check if the shorter pattern still matches.
4 Star — one-more occurrence: If the predecessor p[j-2] matches s[i-1] (either literal match or p[j-2] == '.'), then dp[i][j] |= dp[i-1][j]. We consume one character of s but stay at the same pattern position — the * can fire again.
5 Answer: dp[m][n] — does the full pattern match the full string?
📐 The Recurrence Relations
dp[0][0] = True
dp[0][j] = dp[0][j-2] if p[j-1] == '*' (empty string vs x* unit)

For i ≥ 1, j ≥ 1:
if p[j-1] in (s[i-1], '.'):
  dp[i][j] = dp[i-1][j-1] # single char, diagonal

elif p[j-1] == '*':
  dp[i][j] = dp[i][j-2] # zero occurrences
  if p[j-2] in (s[i-1], '.'):
    dp[i][j] |= dp[i-1][j] # one-more, stay at j
🔑 Why dp[i-1][j] and not dp[i-1][j-2] for "one-more"?

When x* matches one more character of s, we advance in s (i→i-1) but we do not advance in the pattern. The x* unit is still "active" — it can fire again on the next character of s. That is why we look at dp[i-1][j] (same column, one row up) rather than dp[i-1][j-2].

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any cell dp[i][j] (computed in row-major order), the value is True if and only if p[0..j-1] can exactly cover s[0..i-1]. Every cell depends only on previously computed cells — dp[i-1][j-1] (diagonal), dp[i][j-2] (left-two), and dp[i-1][j] (above) — so filling left-to-right, top-to-bottom guarantees correctness.

Why DP? Optimal Substructure: Whether the full match succeeds depends only on shorter prefixes. If you know whether p[0..j-3] matches s[0..i-1] (for the zero-occurrence case), you can derive whether p[0..j-1] matches without re-examining any earlier characters.

Why DP? Overlapping Subproblems: A naive recursive solution recomputes match(s[i:], p[j:]) for the same (i, j) pair many times. With patterns like .*.*a, the number of recursive paths is exponential. Memoization collapses all duplicate calls to a single computation per cell.

⚠️ The first-row fill is easy to miss

Row 0 represents matching against the empty string. Most literals cannot match empty string, so dp[0][j] = False by default. But a pattern like c*a* can match empty (zero c's, zero a's). The first row fill propagates: dp[0][2] = dp[0][0], dp[0][4] = dp[0][2], etc. Forgetting this row causes wrong answers on empty-string inputs.

Why |= for the star case: Two independent choices both yield True → the cell is True. We first set it for zero-occurrence, then OR in the one-more result. Using |= prevents overwriting a True from the zero case with a False from the one-more case (or vice versa).

4 The Code (Annotated) ▾

Solution 1 — 2D Tabulation (Interview Pick)

def isMatch(s: str, p: str) -> bool:
m, n = len(s), len(p) ← string length m, pattern length n
dp = [[False]*(n+1) for _ in range(m+1)] ← (m+1)×(n+1) table, all False
dp[0][0] = True ← empty pattern matches empty string
for j in range(1, n+1): ← first row: which patterns match ""?
if p[j-1] == '*': ← only x* units can match empty
dp[0][j] = dp[0][j-2] ← skip x* (j-2), inherit its result
for i in range(1, m+1): ← iterate through each char of s
for j in range(1, n+1): ← iterate through each char of p
if p[j-1] == '*': ← star case: look at predecessor p[j-2]
dp[i][j] = dp[i][j-2] ← option 1: zero occurrences of x*
if p[j-2] in (s[i-1], '.'): ← predecessor matches current s char?
dp[i][j] |= dp[i-1][j] ← option 2: use one-more, stay at j
elif p[j-1]=='.' or p[j-1]==s[i-1]:← dot or literal match
dp[i][j] = dp[i-1][j-1] ← single char consumed, look diagonal
return dp[m][n] ← full pattern matched full string?
⚠️ Why p[j-2] not p[j-1] when checking predecessor?

At column j, p[j-1] is the * itself. The character it modifies is at index j-2, i.e., p[j-2]. We check whether that predecessor character (or .) matches the current string character s[i-1] before deciding if "one-more" is applicable.

Solution 2 — Top-Down Memoization (with @cache)

from functools import cache
def isMatch(s: str, p: str) -> bool:
@cache ← auto-memoize on (i, j) pairs
def dp(i, j): ← match s[i:] against p[j:]
if j == len(p): return i == len(s) ← pattern exhausted: s must also end
first = i < len(s) and p[j] in (s[i], '.') ← can current chars match?
if j+1 < len(p) and p[j+1] == '*': ← next char is '*'?
return dp(i, j+2) or (first and dp(i+1, j))← zero OR one-more
return first and dp(i+1, j+1) ← single char: advance both
return dp(0, 0)
💡 Complexity Summary

2D Tabulation (Optimal): Time O(m·n), Space O(m·n). Rolling array → O(n) space.
Top-Down Memoization: Time O(m·n), Space O(m·n) for the cache + O(m+n) stack.
Brute Force Recursion: Exponential O((m+n)·2^(m+n/2)) — TLE on large inputs.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the DP Signal

There are decisions at each character: literal match, dot match, or * with two sub-choices. The same sub-problem match(s[i:], p[j:]) can repeat many times (especially with patterns like .*.*). Repeated subproblems + optimal substructure = DP.

Step 2: Define the State

dp[i][j] = "can p[0..j-1] match s[0..i-1] entirely?" Two integer indices fully describe the state. The table has (m+1)(n+1) cells — polynomial in input size.

Step 3: Handle Star in Two Cases

Always split the star case: (1) Zero uses of x* → check dp[i][j-2]. (2) One-more use if predecessor matches → check dp[i-1][j]. Write these as two separate OR branches. Never try to express it in one formula.

Step 4: Don't Forget the First Row

The first row (matching against empty string) is easy to forget. Walk through j = 1..n and propagate dp[0][j] = dp[0][j-2] when p[j-1] == '*'. Patterns like a*b* can match empty string.

🗣️ What to Say to the Interviewer

"I'll use 2D DP. dp[i][j] = True if p[0..j-1] fully matches s[0..i-1].
Base: dp[0][0]=True. First row: dp[0][j] = dp[0][j-2] if p[j-1]='*'.
For star: two choices. Zero occurrences: dp[i][j] = dp[i][j-2].
One-or-more: if p[j-2] matches s[i-1], dp[i][j] |= dp[i-1][j].
For literal or dot: dp[i][j] = dp[i-1][j-1] if chars match.
Return dp[m][n]. Time O(m*n), Space O(m*n). Can optimize space to O(n) with two rows."
🔮 Likely Follow-up Questions
  • "Optimize space to O(n)?" → Use two 1D arrays (previous row and current row).
  • "Wildcard Matching (#44)?" → Similar DP but * matches any sequence — predecessor not needed.
  • "What if '+' was added (one or more)?" → Like * but zero-case is removed.
  • "Trace s='aab', p='c*a*b'" → Be ready to walk the DP table cell by cell (that is what this visualizer shows).
  • "What about '?' (exactly one of any)?" → Behaves like . in this problem; trivial extension.

7 Summary & Key Takeaways ▾
* always pairs with predecessor — think of x* as a two-character unit
Star gives two choices: zero occurrences (skip, look at j-2) or one-more (same j, consume s)
dp[i-1][j] for "one-more" keeps pattern fixed — * can fire again
First row (i=0) must be filled: patterns like a*b* can match empty string
Use |= for star case to OR both options without losing either True result
Full-match requirement: answer is dp[m][n], not any substring cell
