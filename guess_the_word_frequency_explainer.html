843. Guess the Word

Solution 4 — Frequency Uniqueness + Adaptive End
Hard Frequency Scoring Adaptive Heuristic O(n·L) sort O(n·L) space
1 Understanding This Approach ▼

Unlike Minimax (which picks the word minimising worst-case bucket), this solution uses positional character frequency to score each word, then adaptively picks from either end of the sorted list depending on zero-match feedback.

🔑 Core Idea

place_counts[str(i) + char] = how many words have character char at position i. A word's uniqueness score = sum of its per-position frequencies. High score → common letters at common positions. Low score → rare/unique letter+position combos.

💡 The Adaptive Trick

The code alternates which end of the sorted word list it picks from:

  • end = -1 → pick the most common word (highest score, end of sorted list)
  • end = 0 → pick the most unique word (lowest score, start of sorted list)

After getting 0 matches: if we were picking unique (end=0), switch to common (end=-1). The idea: a unique word that scores 0 matches tells us little — switch strategy.

After getting 1-5 matches: if we were picking common (end=-1), switch to unique (end=0). The idea: partial matches from a common word are weak signals — a unique word gives sharper signal.

⚠️ Why This Works (& Its Limitations)

This heuristic doesn't guarantee optimal worst-case like Minimax, but it's simple to code and good enough in practice. The sort is O(n·L) vs Minimax's O(n²·L) per round. However, the adaptive end-switching is a heuristic — it can get unlucky on adversarial inputs.

2 How Frequency Scoring Works ▼

For every word at every position, we count how many other words share that same (position, character) pair. A word's score is the sum of these counts across all 6 positions.

1Build place_counts: For each word, for each position i, increment place_counts[str(i) + char]. The key "0a" means "character 'a' at position 0".
2Score each word: word_uniqueness(w) = sum of place_counts[str(i)+c] for each position. High score = many words share these characters. Low score = unique.
3Sort by score: words.sort(key=word_uniqueness). Index 0 = most unique. Index -1 = most common.
4Guess loop: Pick words[end]. If matches=6 → done. If matches=0 → aggressive filter (remove any word sharing ANY position). If matches=1-5 → normal filter. Adapt end based on result.

3 The Adaptive End Strategy ▼

This is the most creative part of the solution. Instead of always picking the same "best" word, the code switches strategy based on feedback:

Most Common

High overlap with many words

⟷
end = 0
Most Unique

🔑 Decision Logic

Start: end = -1 (pick most common first)

Got 0 matches? → Eliminate words sharing ANY letter-position with guess. If currently end=0, flip to end=-1. (A unique word with 0 matches is a wasted guess — switch to common.)

Got 1-5 matches? → Keep only words matching exactly that count. If currently end=-1, flip to end=0. (Partial match from common word is noisy — switch to unique for sharper signal.)

⚠️ Important Subtle Detail: word_is_possible

The word_is_possible function returns False if guess_word == word (we already guessed it — don't keep it). Otherwise returns True only if the number of exact position matches equals the returned match count. This is the core filtering invariant: the secret word must match the guess by exactly that many positions.

4 The Code (Annotated) ▼
class Solution:
def findSecretWord(self, words, master):
# Step 1: Count (position, char) frequencies across all words
place_counts = collections.defaultdict(int)
for word in words:
for i, char in enumerate(word):
place_counts[str(i) + char] += 1 # key="0a" means char 'a' at pos 0
# Step 2: Scoring function — sum of position-frequencies
def word_uniqueness(word):
score = 0
for i, char in enumerate(word):
score += place_counts[str(i) + char]
return score # high = common, low = unique
# Step 3: Sort — index 0 = most unique, index -1 = most common
words.sort(key=word_uniqueness)
# Step 4: Filtering helper
def word_is_possible(guess_word, word, matches):
if guess_word == word: # already guessed — skip
return False
match_count = sum(a == b for a, b in zip(guess_word, word))
return match_count == matches # keep only if matches agree
# Step 5: Adaptive guess loop
end = -1 # start by picking most common
for i in range(30): # at most 30 iterations (safety net)
guess_word = words[end] # pick from sorted end
matches = master.guess(guess_word)
if matches == 6: # found!
break
elif matches == 0: # zero matches — aggressive filter
words = [w for w in words
if not any(a==b for a,b in zip(guess_word, w))]
if end == 0: end = -1 # swap: unique→common
else: # 1-5 matches — normal filter
words = [w for w in words
if word_is_possible(guess_word, w, matches)]
if end == -1: end = 0 # swap: common→unique
5 🎤 Interview Strategy ▼
Step 1: Recognize the Problem Type

Interactive/oracle problem — we can't see the answer, only get match counts. Signal: Master.guess() returns a number, limited guesses.

Step 2: Propose Baseline First

Start with "random pick + filter". Then improve: "What if we pick words that give the most information?" This motivates frequency scoring.

Step 3: Explain Frequency Scoring

"I'll count how common each (position, character) pair is. Words with rare combos are more distinguishing. Words with common combos overlap with many candidates."

Step 4: Mention the Adaptive Twist

"After 0 matches, a unique guess was uninformative — I'll switch to common. After partial matches, a common guess was noisy — I'll switch to unique." This shows you're thinking about information theory.

🗣️ What to Say to the Interviewer

"I'll score each word by positional character frequency — high-frequency words share letters with many candidates, low-frequency words are unique. I sort by score, then adaptively pick from either end: if a guess returns 0 matches, I switch to picking common words (they overlap more, so 0-match eliminates more). If I get partial matches, I switch to unique words for sharper signal."
💡 Comparison with Minimax

Minimax (S1): O(n²) per round, optimal worst-case, harder to code live. Frequency (S4): O(n·L) sort, heuristic but simple to code. In a 45-min interview, this is faster to implement and explain. Mention Minimax as a follow-up for bonus points.

7 Summary & Key Takeaways ▼
Positional frequency captures how "common" a word is — shared (position, char) pairs mean high overlap
Sorting once gives O(1) access to both most unique and most common words
Adaptive end-switching adjusts strategy based on feedback: 0-match → try common, partial → try unique
Zero-match filter is aggressive: if ANY position matches, eliminate that word
Partial-match filter is exact: keep only words with exactly that number of position matches
word_is_possible also excludes the guess itself, preventing re-guessing
🎯 When to Use This Approach

When you need a quick-to-code heuristic for an interactive problem. Frequency scoring is a common technique in word games (Wordle solvers use it too!). The adaptive switching is a neat trick to mention in interviews for extra credit.
