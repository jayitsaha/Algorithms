LeetCode #698

Partition to K Equal Sum Subsets

Given an integer array nums and an integer k, return true if it is possible to divide nums into k non-empty subsets whose sums are all equal.

Medium Dynamic Programming Bitmask DP Time: O(2^n · n) Space: O(2^n)
1 Understanding the Problem ▾

You have a collection of numbers and need to split them into exactly k groups where every group sums to the same value. Think of it like distributing tasks fairly among k workers — everyone must do exactly the same total amount of work, and each task goes to exactly one worker.

💡 Key Insight

The target sum per bucket is fixed: target = sum(nums) / k. If the total isn't divisible by k, or if any single element exceeds the target, it's immediately impossible. The hard part is determining which numbers go into which bucket — this is an NP-hard assignment problem best solved with bitmask DP for small n.

✓ Feasible: nums=[4,3,2,3,5,2,1], k=4

sum = 20, target = 20/4 = 5

Bucket 2: [1, 4] = 5 ✓
Bucket 3: [2, 3] = 5 ✓

→ True

✗ Not Feasible: nums=[1,2,3,4], k=3

sum = 10, 10/3 = 3.33...
Not a whole number →
Immediate False

Why is this hard? The number of ways to partition n elements into k groups grows exponentially. With n=15, there are billions of possible assignments. Brute force is infeasible. The key observation is that we only need to know which elements have been placed, not which exact bucket they're in — because all buckets have identical targets. This symmetry lets us encode all used elements in a single bitmask integer.

Edge cases: k=1 (always True — one group holds everything), k=n (each element must exactly equal target), any element > target (impossible), sum % k ≠ 0 (impossible).

2 The Algorithm: Bitmask DP (Bottom-Up) ▾

A bitmask is an integer where each bit i encodes whether element at index i has been placed into a bucket. For 4 elements, mask 0110 (decimal 6) means elements 1 and 2 are used; elements 0 and 3 are free. There are exactly 2n possible masks.

1 Prune early: If sum(nums) % k ≠ 0 or max(nums) > target, return False immediately. No valid partition exists.
2 State definition: dp[mask] = the current partial bucket sum when exactly the elements indicated by mask have been distributed. Value -1 = unreachable.
3 Base case: dp[0] = 0. No elements used, bucket sum is 0.
4 Transition: For each reachable mask, try adding each unused element i. If dp[mask] + nums[i] ≤ target, set dp[mask | (1<<i)] = (dp[mask] + nums[i]) % target. The modulo resets the bucket to 0 when it completes.
5 Answer: Return dp[(1<<n) - 1] != -1. If all n elements are used and reachable, the partition exists.

For each mask in [0, 2^n - 1] where dp[mask] != -1:
  For each unused bit i (mask & (1<<i) == 0):
    new_sum = dp[mask] + nums[i]
    if new_sum <= target:
      dp[mask | (1<<i)] = new_sum % target
🔑 The "Mod Target" Bucket Reset Trick

When we add an element that fills the bucket to exactly target, the expression new_sum % target evaluates to 0. This signals "bucket is complete, start a fresh one." The dp value never stores which bucket is current — it only stores how full the current bucket is. The number of completed buckets is implicitly encoded in the popcount of the mask.

3 Why It Works: The Invariant ▾
🔐 The Invariant

If dp[mask] = v (v ≠ -1), then the elements marked in mask can be distributed into some number of complete buckets (each summing to target) plus one partial bucket currently summing to v. The value v is always in range [0, target-1] — it never exceeds the target.

Why this guarantees correctness:

  • Each transition only fires when new_sum ≤ target — we never overfill a bucket.
  • When new_sum == target, new_sum % target = 0 — the bucket is closed and a fresh one opens. This implicitly counts completed buckets.
  • At the final mask (all 1s), sum(nums) = k * target guarantees exactly k complete buckets have been formed (since every mod-target cycle is one complete bucket).
  • Iterating masks in increasing order works because adding a bit always increases the mask value — every sub-state is guaranteed to be processed before any super-state needs it.
⚠️ Why Not Pure Backtracking?

Pure backtracking can revisit the same subset of used elements through different orderings. If we try placing element 0 then element 1, vs element 1 then element 0 — the resulting state is identical, but backtracking would explore both branches. Bitmask DP memoizes by subset: once dp[mask] is computed, we never recompute it. The bitmask is the perfect key because order within a bucket doesn't matter — only membership does.

4 The Code (Annotated) ▾

Solution 1 — Bitmask DP Bottom-Up (Interview Pick)

def canPartitionKSubsets(nums, k):
total = sum(nums)← Sum all elements
if total % k != 0: ← If not evenly divisible, no valid partition
return False
target = total // k ← Each of k buckets must sum to this
if max(nums) > target: ← A single element can never fit in any bucket
return False
n = len(nums)
dp = [-1] * (1 << n) ← 2^n states; -1 = unreachable
dp[0] = 0 ← Base: no elements used, partial bucket sum = 0
for mask in range(1 << n):← Process all subsets in increasing order
if dp[mask] == -1: ← Skip states we can never reach
continue
for i in range(n): ← Try adding each element to the current state
if mask & (1 << i):← Element i already used in this subset
continue
new_sum = dp[mask] + nums[i]← What would new partial bucket sum be?
if new_sum <= target: ← Only place if it doesn't overflow bucket
next_mask = mask | (1 << i)← Mark bit i as used
dp[next_mask] = new_sum % target← % target: reset to 0 when bucket fills
return dp[(1 << n) - 1] != -1← All elements placed and reachable → True
⚠️ Why must masks be iterated in increasing order 0 → 2^n-1?

Each transition goes from mask to mask | (1<<i). OR-ing a bit always produces a strictly larger integer. So when we process mask m, every mask that could have produced m has a smaller numeric value and was already processed. The bottom-up order is valid because it respects the dependency direction: smaller masks → larger masks.

Solution 2 — Backtracking with Pruning (Conceptually Intuitive)

def canPartitionKSubsets(nums, k):
total = sum(nums)
if total % k != 0 or max(nums) > total // k:
return False
target = total // k
nums.sort(reverse=True) ← Largest first: fills buckets faster, prunes early
buckets = [0] * k ← k running bucket sums
def bt(i): ← Try placing nums[i] into some bucket
if i == len(nums): return True← All placed → success
seen = set() ← Prune: don't try identical bucket sums twice
for j in range(k):
if buckets[j] + nums[i] <= target and buckets[j] not in seen:
seen.add(buckets[j])
buckets[j] += nums[i]
if bt(i + 1): return True
buckets[j] -= nums[i]← Backtrack: undo and try next bucket
return False← No placement worked for nums[i]
return bt(0)
💡 Complexity Comparison

Bitmask DP: Time O(2n · n), Space O(2n). Deterministic. Guaranteed no recomputation.
Backtracking: Worst case O(kn) but much faster in practice with pruning and sorting.
For n ≤ 16 (LeetCode constraint): both are acceptable; bitmask DP preferred for tight time limits.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

The signals: "partition array into k groups", "equal sum subsets", small n (≤16). Whenever you need to track which elements have been assigned and n is small, think bitmask DP. The subset structure is the giveaway.

Step 2: State the Two Pruning Conditions First

"Before any DP, I'll check: (1) sum % k must be 0, otherwise immediate False. (2) no element can exceed target = sum/k." This shows methodical thinking and handles 50% of test cases with O(n) work.

Step 3: Propose Backtracking as the Stepping Stone

"Start with recursive backtracking: try placing each element into each bucket, backtrack on overflow. The issue: same subset explored multiple times from different orderings — O(k^n) worst case."

Step 4: Introduce Bitmask DP as the Optimization

"To avoid recomputing the same subsets, encode used elements as a bitmask. dp[mask] = current partial bucket sum. Key: dp[mask | (1<<i)] = (dp[mask]+nums[i]) % target. The modulo elegantly handles bucket completion. O(2^n · n) time."

🗣️ What to Say to the Interviewer

"First, compute target = sum(nums) / k. If sum % k != 0 or max > target, return False.
The key state: which elements have we placed so far? Encode this as a bitmask.
dp[mask] = partial sum of the current bucket when mask elements are placed.
Transition: for each unused element i, if dp[mask] + nums[i] <= target,
set dp[mask|(1<<i)] = (dp[mask]+nums[i]) % target.
The %target trick: when a bucket fills to exactly target, the mod resets to 0 — new bucket!
Iterate masks in increasing order (0 to 2^n-1) — sub-states always computed first.
Answer: dp[(1<<n)-1] != -1. Time O(2^n * n), Space O(2^n)."
🔮 Likely Follow-up Questions
  • "What if k=2?" → Simpler! 0/1 Knapsack: does any subset sum to total/2? (#416, O(n * total/2))
  • "Why does n ≤ 16 in constraints?" → 2^16 = 65,536 states. At n=30, 2^30 ≈ 1 billion — too slow.
  • "Can you sort to help backtracking?" → Yes! Sort descending. Large elements prune quickly.
  • "What if we want all valid partitions?" → Track parents in DP table; exponential output though.

7 Summary & Key Takeaways ▾
Bitmask DP encodes "which elements are used" as bits in an integer — perfect for subset tracking when n ≤ 20
Always prune first: sum % k ≠ 0 or max(nums) > target → immediate False
The mod target trick resets the bucket sum to 0 when a bucket completes — one elegant line handles all k buckets
Iterate masks in increasing order: since adding a bit increases the mask value, dependencies always resolve before use
If dp[(1<<n)-1] ≠ -1, all n elements are partitioned into exactly k complete buckets
ApproachTimeSpaceNotes
Brute Force (all partitions) O(k^n) O(n) Infeasible for n > 10
