LeetCode #74

Search a 2D Matrix

Write an efficient algorithm to search for a value in an m×n matrix with sorted rows and globally ascending order across rows.

Medium Binary Search Treat as 1D Array Time: O(log mn) Space: O(1)
1 Understanding the Problem ▾

You're given an m × n integer matrix with two guarantees: each row is sorted in ascending order, and the first integer of each row is greater than the last integer of the previous row. You need to find whether a target value exists — efficiently.

💡 Key Insight

The two properties together mean that if you read the matrix row-by-row from top-left to bottom-right, you get a single globally sorted sequence of length m×n. This transforms the 2D problem into a 1D binary search — you just need a way to convert a flat index back into (row, col) coordinates.

Examine this 3×4 example — reading left-to-right, top-to-bottom yields 1, 3, 5, 7, 10, 11, 16, 20, 23, 30, 34, 60 — perfectly sorted:

The index mapping trick: For a matrix with n columns, virtual flat index mid maps to row = mid // n and col = mid % n. This is exactly the base-n number system — quotient gives row, remainder gives column.

2 The Algorithm: Virtual 1D Binary Search ▾

Because the 2D matrix is logically a sorted 1D array, we can run a single standard binary search. The only adaptation: instead of indexing into a real array, we convert each midpoint to (row, col) coordinates on the fly using integer division and modulo.

1 Set virtual bounds: lo = 0, hi = m * n - 1. These are the virtual indices of the first (top-left) and last (bottom-right) elements.
2 Pick midpoint: mid = (lo + hi) // 2. Convert to 2D: row = mid // n, col = mid % n. Read matrix[row][col].
3 Three-way compare: If equal → return True. If matrix[row][col] < target → the target is in the right half, so lo = mid + 1. If greater → target is in the left half, so hi = mid - 1.
4 Terminate: Loop exits when lo > hi. At that point every element has been ruled out — return False.

🔑 Why This Beats All Other Approaches

Scanning every cell is O(mn). Scanning rows to find the candidate row, then binary-searching within it, is O(m + log n). Virtual 1D binary search is O(log mn) = O(log m + log n) — a single pass that discards half the search space each iteration. For a 1000×1000 matrix that is 20 comparisons vs up to 1,000,000.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every iteration: if the target exists in the matrix, it lies at some virtual index within [lo, hi]. When matrix[row][col] < target, every element at virtual index ≤ mid is also < target (because the sequence is globally sorted), so we safely raise lo to mid+1. The symmetric argument applies for the hi case.

Why the index mapping is a bijection: For a matrix of shape m×n, the function idx → (idx // n, idx % n) maps each integer in [0, mn−1] to exactly one cell, and every cell maps back to exactly one index. So binary-searching virtual indices is mathematically identical to binary-searching the sorted values in reading order.

Why the sequence is globally sorted: By the problem's second guarantee, each row's last element is strictly less than the next row's first element. Combined with each row being internally sorted, this chain of inequalities makes the whole flat sequence monotonically increasing — no gaps, no inversions.

⚠️ Off-By-One: hi Should Be m*n-1, Not m*n

The last valid virtual index is m*n - 1. Setting hi = m*n creates an out-of-bounds access when mid lands on the ghost element. In Python this silently gives wrong results (no IndexError from // and % on an integer), so it's a silent correctness bug — easy to miss in testing.

⚠️ Division by n, Not m

The mapping is row = mid // n and col = mid % n, where n is the number of columns. Dividing by m (rows) is the most common mistake — it produces wrong row/column values for non-square matrices. Remember: columns determine how many elements fit in a row, so that's your divisor.

4 The Code (Annotated) ▾

Solution 1 — Virtual 1D Binary Search (Optimal · Interview Pick)

def searchMatrix(matrix, target):
if not matrix or not matrix[0]: ← guard: empty matrix or empty row
return False
m, n = len(matrix), len(matrix[0]) ← m rows, n columns
lo, hi = 0, m * n - 1 ← virtual index range: [0 .. m*n-1]
while lo <= hi: ← standard binary search loop
mid = (lo + hi) // 2 ← midpoint of remaining range
row, col = mid // n, mid % n ← project to 2D (divide by COLUMNS)
val = matrix[row][col] ← read actual value
if val == target: ← found it!
return True
elif val < target: ← mid too small → look right
lo = mid + 1
else: ← mid too large → look left
hi = mid - 1
return False ← exhausted all candidates

Solution 2 — Two-Phase Binary Search (pedagogically useful, same complexity)

def searchMatrix(matrix, target):
m, n = len(matrix), len(matrix[0])
# Phase 1: find the row that could contain target
top, bot = 0, m - 1
while top <= bot:
row = (top + bot) // 2
if matrix[row][0] <= target <= matrix[row][-1]:
break ← target falls in range of this row
elif target < matrix[row][0]: bot = row - 1
else: top = row + 1
else: return False ← Python for-else: loop ended without break
# Phase 2: binary search within that row
lo, hi = 0, n - 1
while lo <= hi:
mid = (lo + hi) // 2
if matrix[row][mid] == target: return True
elif matrix[row][mid] < target: lo = mid + 1
else: hi = mid - 1
return False
💡 Complexity Summary

Virtual 1D (Solution 1): Time O(log mn), Space O(1) — one binary search over entire matrix
Two-phase (Solution 2): Time O(log m + log n) = O(log mn), Space O(1) — two separate searches
Brute force: Time O(m × n), Space O(1) — never used in interviews
Both optimal solutions have identical asymptotic complexity. Solution 1 is cleaner to code.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Binary Search Signal

"Sorted structure + search for value" always signals binary search. The 2D aspect is a distraction. Ask yourself: "Is this data globally sorted if I read it in some order?" Here, reading row-by-row gives a globally sorted sequence, so yes.

Step 2: State the Reframing Aloud

"Each row starts after the previous one ends, so the matrix is equivalent to a 1D sorted array of length m×n. I'll binary search over virtual indices 0 to m*n-1, converting each mid back to (row, col) using integer division and modulo." This immediately impresses interviewers.

Step 3: Derive the Formula with a Small Example

Quickly verify: "For a 4-column matrix, virtual index 5: row = 5//4 = 1, col = 5%4 = 1. That's the second row, second column." Tracing one conversion shows you understand why n (not m) is the divisor.

Step 4: Mention the #240 Follow-Up Proactively

"Note: if the matrix only guarantees row-sorted and column-sorted (like LeetCode #240), this trick doesn't work — you'd use the staircase approach starting from top-right: O(m+n). The key property here is that rows are globally chained."

🗣️ What to Say to the Interviewer

"Because each row starts after the previous row ends, reading the matrix left-to-right,
top-to-bottom gives a sorted sequence. So I'll treat it as a 1D sorted array of m*n elements
and run standard binary search: lo=0, hi=m*n-1. Each step: mid=(lo+hi)//2,
row=mid//n, col=mid%n. If matrix[row][col] equals target → True.
If less → lo=mid+1. If greater → hi=mid-1. Return False if loop ends.
Time: O(log mn). Space: O(1)."
🔮 Likely Follow-up Questions
  • "What if I relax the cross-row constraint?" → Use staircase from top-right: O(m+n). Binary search no longer applies because the 1D equivalence breaks.
  • "What about integer overflow computing mid?" → Python: no overflow. Java/C++: use lo + (hi - lo) / 2.
  • "Can you return the index, not just True/False?" → Return (row, col) from the found branch instead.
  • "How would you handle duplicates?" → Works correctly — returns True on any occurrence of target.

7 Summary & Key Takeaways ▾
When a 2D matrix has sorted rows AND each row starts after the previous ends, read it as a single sorted 1D array
Virtual-index binary search: lo=0, hi=m*n-1; map mid to (mid//n, mid%n) — pure O(1) coordinate math
O(log mn) time, O(1) space — optimal. Far better than any 2D scan or row-first approach
The critical property is the strict cross-row ordering; without it you must use the staircase algorithm (LeetCode #240)
Common bug: dividing by m instead of n in the index mapping — always divide by the number of COLUMNS
ApproachTimeSpaceNotes
Brute force scan O(mn) O(1) Scan every cell
Linear row scan + row binary search O(m + log n) O(1) Better, but not optimal
Virtual 1D binary search ✓ O(log mn) O(1) Interview pick
Staircase (for LC #240) O(m + n) O(1) Use when cross-row guarantee is absent
🎯 When to Recognize This Pattern
  • "Sorted 2D matrix + search" → check if globally sorted → virtual 1D binary search
  • Any multi-dimensional structure that can be flattened into a sorted sequence can use this trick
  • The key question: "Does reading in some order give a globally sorted sequence?"
  • Index mapping generalizes to higher dimensions: for 3D, mid → (mid//(n*p), (mid//p)%n, mid%p)
