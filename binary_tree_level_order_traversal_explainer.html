LeetCode #102

Binary Tree Level Order Traversal

Given the root of a binary tree, return the level order traversal of its node values — left to right, level by level — as a list of lists.

Medium Trees BFS · Queue Time: O(n) Space: O(n)
1 Understanding the Problem ▾

A binary tree has a natural depth structure: the root sits at level 0, its children at level 1, their children at level 2, and so on. Level Order Traversal asks us to visit nodes one level at a time, collecting all values at depth d before moving to depth d+1.

💡 Key Insight

A queue (FIFO) processes elements in the order they were added — which naturally corresponds to visiting siblings before children. The challenge is knowing when one level ends and the next begins. The trick: freeze len(queue) at the start of each level — that's the exact node count for that level.

Concrete example — tree with 5 nodes:

        3          ← Level 0: [3]
       / \
      9  20         ← Level 1: [9, 20]
        /  \
       15    7      ← Level 2: [15, 7]

Expected output: [[3], [9, 20], [15, 7]]

Edge Cases to consider:

  • Empty tree (root=None) → return []
  • Single node → return [[root.val]]
  • Skewed tree (like a linked list) → n levels, each with one node
  • Perfect binary tree of height h → 2^h nodes at the last level
  • Nodes with only one child — handled naturally by checking left and right separately
2 The Algorithm: BFS with Level Snapshots ▾

We use a queue (FIFO) to do BFS. At the start of each level, we snapshot level_size = len(queue) — this is the count of nodes at the current level. We dequeue exactly that many nodes, collecting their values, and enqueue their children. The children populate the queue for the next level.

1 Guard clause: If root is None, immediately return [].
2 Seed the queue with the root node. Initialize result = [].
3 Outer while loop — continues while the queue is non-empty.
4 Freeze the level size: level_size = len(queue). This is the exact count of nodes at the current level.
5 Inner for loop — runs level_size times. Each iteration: dequeue a node, record its value, enqueue its existing children.
6 Append level: After the inner loop, append level_vals to result. Repeat outer loop for next level.
🔑 The Level Boundary Mechanism

When we start an outer-loop iteration, the queue holds exactly the nodes at the next unprocessed level — because we enqueued them while processing the previous level. Snapshotting len(queue) before the inner loop gives us a stable count: even as we enqueue new children inside the loop, we only dequeue the original batch.

3 Why It Works: The Invariant ▾
🔐 The Queue Invariant

At the start of each outer-while-loop iteration, the queue contains exactly and only the nodes at the next unprocessed level. This invariant holds throughout the algorithm.

Proof by induction:

Base case (level 0): We seed the queue with only the root. The root is the only node at level 0. Invariant holds. ✓

Inductive step: Assume before iteration k, the queue contains exactly the nodes at level k. We dequeue all k-level nodes (we know exactly how many via the frozen level_size). For each, we enqueue its left and right children. Children of level-k nodes are by definition at level k+1. After the inner loop, the queue contains exactly the level-(k+1) nodes. ✓

Completeness (no node missed): The root is seeded. Every non-root node has a parent, which gets processed and enqueues it. By induction, every node enters the queue exactly once.

No double-counting: Each node is enqueued once and dequeued once. So each node's value appears in exactly one level's sublist.

⚠️ The Subtle Bug: Don't Use len(queue) in the for loop header

Writing for _ in range(len(queue)): directly is dangerous in languages where len() re-evaluates each iteration, or if you confuse yourself. In Python, range(len(queue)) does freeze the count at the time it's called — but the safe, explicit, and interview-readable approach is level_size = len(queue) then for _ in range(level_size). Never use while queue and i < initial_len — that's error-prone.

4 The Code (Annotated) ▾

Solution 1 — BFS with Level Snapshot (Interview Pick)

from collections import deque— O(1) popleft; list.pop(0) is O(n)
def levelOrder(root):
if not root:— guard: null tree
return []
queue = deque([root])— seed BFS with root
result = []— accumulates level sublists
while queue:— loop until all nodes processed
level_size = len(queue)— FREEZE: nodes at this level
level_vals = []— collect values for current level
for _ in range(level_size):— process exactly level_size nodes
node = queue.popleft()— O(1) dequeue
level_vals.append(node.val)— record value
if node.left:— enqueue left child (if exists)
queue.append(node.left)
if node.right:— enqueue right child (if exists)
queue.append(node.right)
result.append(level_vals)— level complete, store it
return result— list of lists
💡 Why deque and not list?

list.pop(0) is O(n) because every remaining element shifts left. deque.popleft() is O(1). With n nodes, the total dequeue cost with a list would be O(1 + 2 + ... + n) = O(n²) vs O(n) with deque. Always use collections.deque for BFS queues in Python.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

Signal words: "level by level", "layer by layer", "row by row", "breadth-first". Whenever a tree problem groups nodes by depth, BFS with a queue is the canonical tool. DFS naturally goes deep; BFS naturally goes wide.

Step 2: Identify the Data Structure

Queue (FIFO). First in, first out means we process the root before its children, and process sibling nodes before their children. That's exactly level order.

Step 3: Solve the Level Boundary Problem

A plain BFS gives us all nodes in level order, but doesn't group them. Ask: "How do I know when level k ends and level k+1 begins?" Answer: snapshot len(queue) at the start of each outer loop — that count is exactly the number of nodes at the current level.

Step 4: Write the Code

Use collections.deque (say why: O(1) popleft). Write outer while loop, freeze level_size, write inner for loop, collect level_vals, append to result. Handle null root first.

Step 5: Know the Variations

This exact BFS skeleton powers at least 6 related LeetCode problems — zigzag, right side view, level averages, max per row, minimum depth, maximum width. Knowing the skeleton makes each a 5-minute adaptation.

🗣️ What to Say to the Interviewer

"This is a level order traversal — the phrase 'level by level' signals BFS. I'll use a queue seeded with the root. The key insight is determining level boundaries: I snapshot len(queue) at the start of each iteration — that's the exact count of nodes at the current level. I dequeue that many nodes, collect their values, and enqueue their children. After the inner loop I append the level's values to result. Time: O(n) — every node enqueued and dequeued exactly once. Space: O(n) — queue holds at most the widest level, which can be n/2 leaves. I'll use collections.deque for O(1) popleft."

7 Summary & Key Takeaways ▾
BFS (queue) naturally visits nodes in level order — FIFO ensures siblings before children.
Snapshotting level_size = len(queue) before the inner loop is the key to separating levels cleanly.
Use collections.deque for O(1) popleft — this is the Pythonic BFS queue.
Every node is enqueued once and dequeued once: Time O(n), Space O(n).
