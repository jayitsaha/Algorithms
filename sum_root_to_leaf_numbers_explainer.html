LeetCode #129

Sum Root to Leaf Numbers

Given the root of a binary tree containing digits 0–9, return the total sum of all numbers formed by every root-to-leaf path, reading node values top-to-bottom as decimal digits.

Medium Trees DFS Preorder Time: O(n) Space: O(h)
1 Understanding the Problem ▾

Think of the binary tree as a number-writing machine. Every path from the root down to a leaf node spells out a number by reading the node values top-to-bottom — the root's digit is the most significant, each deeper node shifts everything left one decimal place and appends its own digit. We want the total sum of all these leaf-spelled numbers.

💡 Key Insight

As you walk down one step in the tree, the number formed so far transforms as: new_number = parent_number × 10 + current_node_value. This is exactly how a human reads a multi-digit number digit by digit. You don't need to store the full path — just carry the running number as you recurse.

✗ Common Misconceptions

"Sum all digits" → 1+2+1+3 = 7 ✗
"Sum leaf values" → 2+3 = 5 ✗

The path spells a decimal number,
not an arithmetic sum of digits.

Edge Cases: A single-node tree (root is both root and leaf — the answer is just root.val). A node with value 0 mid-path (still multiply by 10, add 0 — perfectly valid). A very deep, linear tree (large numbers accumulate but no precision issue in standard integer arithmetic). An empty tree (return 0).

2 The Algorithm: DFS with Running Number (Pass Curr Down) ▾

We use a preorder DFS — process the current node before its children. The key parameter we pass downward is curr: the decimal integer formed by reading digits from the root to the current node. When we hit a leaf, curr is the complete root-to-leaf number. We return it upward. Internal nodes return the sum of their subtrees.

1 Base case — null node: dfs(None, curr) returns 0. A null pointer is not a valid node, so it contributes nothing.
2 Extend the running number: curr = curr * 10 + node.val. Now curr represents the decimal number formed by the path from root down to this node inclusive.
3 Leaf base case — return curr: If not node.left and not node.right, this is a complete path. Return curr — it is one root-to-leaf number.
4 Internal node — recurse both children: Return dfs(node.left, curr) + dfs(node.right, curr). Missing children return 0 automatically via the null base case.
5 Entry point: return dfs(root, 0). Start with accumulator 0 so that the root's first operation gives 0 × 10 + root.val = root.val.
🔑 "Pass Current Number Down" — the Sub-Pattern

This is top-down parameter passing. Instead of computing from children upward (post-order), we pass an accumulating value downward (pre-order). The parameter curr encodes everything a call needs from its ancestors — no global state, no backtracking path list. Recognise this pattern whenever: the answer at a leaf depends on ALL ancestors' values, not just the immediate parent.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any call dfs(node, curr), curr is the decimal integer formed by reading node values from the root down to (but not including) this node. After computing curr = curr * 10 + node.val, it becomes the integer formed by the path root → … → this node inclusive. When we reach a leaf, this is exactly one complete root-to-leaf number.

Why × 10 is the right operation: Decimal notation constructs the number d₁d₂d₃ as ((0×10 + d₁)×10 + d₂)×10 + d₃. Each level of the tree adds one digit, so multiplying by 10 and adding the node value is this incremental decimal construction — exact, with no approximation.

Why summing left + right subtree results is correct: Every root-to-leaf path passes through exactly one of: the left subtree or the right subtree. The recursive call on each child independently sums all paths through that subtree. Adding the two totals covers all paths — no path is counted twice, and no path is missed.

⚠️ A Node with One Child Is NOT a Leaf

A leaf requires not node.left and not node.right — both children null. If a node has only a left child (right is null), dfs(None, curr) is called for the right and returns 0 — correctly contributing nothing. Mistakenly treating one-child nodes as leaves would generate spurious path numbers.

Single-node tree correctness: Root is a leaf. dfs(root, 0) → curr = 0×10 + root.val = root.val → no children → return root.val. Correct.

4 The Code (Annotated) ▾

Solution 1 — DFS with Running Number (Optimal, Interview Pick)

def sumNumbers(root): ← outer function; clients call this
def dfs(node, curr): ← inner DFS; curr = number built so far (excluding this node)
if not node: ← null child: not a valid path endpoint, contribute 0
return 0
curr = curr * 10 + node.val ← shift digits left, append this node's digit
if not node.left and not node.right: ← leaf: no children → complete root-to-leaf number
return curr ← return the full path number (e.g., 12 for path 1→2)
return dfs(node.left, curr) \ ← sum of all paths through left subtree
+ dfs(node.right, curr) ← plus sum of all paths through right subtree
return dfs(root, 0) ← kick off with accumulator 0 (first op: 0×10 + root.val)
⚠️ Why pass curr=0, not curr=root.val?

Passing curr=0 lets the first thing dfs does be 0 * 10 + root.val = root.val. This keeps the update logic identical at every node — no special handling for the root. Pre-seeding curr=root.val would require either skipping the update at the root, or applying it twice.

Solution 2 — Iterative DFS with Stack (avoids recursion limit)

def sumNumbers(root): ← iterative version using explicit stack
if not root: return 0 ← empty tree has no paths
stack = [(root, 0)] ← each entry: (node, number_so_far_before_this_node)
total = 0 ← accumulates sum of all leaf numbers
while stack: ← process nodes until nothing remains
node, curr = stack.pop() ← pop next (node, accumulated_number) pair
curr = curr * 10 + node.val ← same update as recursive version
if not node.left and not node.right: ← leaf: add complete path number to total
total += curr
if node.right: ← push right first (it will be popped after left)
stack.append((node.right, curr))
if node.left: ← push left last → popped first → preorder left-to-right
stack.append((node.left, curr))
return total ← total sum of all root-to-leaf numbers
💡 Complexity Summary

Both solutions — Time O(n): every node is visited exactly once.
Space O(h): recursion call stack or explicit stack depth equals the height h of the tree. O(log n) for balanced trees, O(n) for skewed (worst case).
No better than O(n) — every node must be visited at least once to discover all leaves.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Root-to-Leaf Path Pattern

The problem asks about paths from root to leaf — any time you hear "root to leaf," think DFS. BFS is possible but requires carrying path state in the queue, which is messier. DFS naturally recurses down each path.

Step 2: Ask What to Carry Downward

The key question: "What does a recursive call need to know about its ancestors?" Here, it needs the number formed by the path so far. Just one integer. That means we can pass it as a parameter — no list, no global, no backtracking.

Step 3: State the Update Formula Explicitly

Always say: curr = curr * 10 + node.val. Verify on an example: root=1 gives 0*10+1=1, then left child 2 gives 1*10+2=12. This is the heart of the solution — make sure the interviewer sees it clearly.

Step 4: Define Leaf Condition Precisely

State explicitly: "A leaf is a node where both children are null — not node.left and not node.right." This avoids the common mistake of treating one-child nodes as leaves.

🗣️ What to Say to the Interviewer

"This is a root-to-leaf path problem, so I'll use preorder DFS — I need to process the parent before I can compute the child's number.
The key idea: instead of storing the path as a list, I carry a running integer downward.
At each node: curr = curr * 10 + node.val — this appends the current digit to the decimal number.
At a leaf (both children null): return curr — that IS the root-to-leaf number.
At an internal node: return dfs(left, curr) + dfs(right, curr).
Kick off with dfs(root, 0) so the first update gives 0*10 + root.val = root.val.
Time O(n), Space O(h). For very deep trees, I'd use the iterative stack version."
🔮 Likely Follow-up Questions
  • "Same problem but binary digits (0/1)?" → Replace × 10 with × 2. LeetCode #1022.

  • "What if nodes have negative values?" → The × 10 formula still works; negative digits would form negative numbers — problem constraints (digits 0–9) prevent this for #129.

7 Summary & Key Takeaways ▾
Root-to-leaf paths → preorder DFS (process node before recursing into children)
Pass running number down: curr = curr * 10 + node.val at every node
No path list needed — the integer curr IS the path encoding
Leaf condition: not node.left and not node.right (both children null)
