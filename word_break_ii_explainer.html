LeetCode #140

Word Break II

Given a string s and a dictionary wordDict, return all sentences you can form by inserting spaces so that every word is in the dictionary.

Hard Dynamic Programming DP + Backtracking Time: O(n² + n·2ⁿ) Space: O(n·2ⁿ)
1 Understanding the Problem ▾

You have a string like "catsanddog" and a dictionary ["cat","cats","and","sand","dog"]. The task: insert spaces to split the string into valid dictionary words — and return every valid sentence.

The tricky part: a prefix can be cut multiple ways. "cats" can be cut as cat|s or kept as cats. Both lead to valid sentences, so we must explore every branching path.

💡 Key Insight

This is two problems in one. First, determine which positions are reachable by ending a valid word there (DP). Second, enumerate all paths to the end using only reachable positions (backtracking). Without step one, backtracking wastes time on dead branches. Without step two, DP only gives True/False.

🔑 Why Two Phases?

Phase 1 (DP): In O(n²), build dp[i] = "is the first i characters fully segmentable?" Positions where dp is False are dead ends — no sentence can pass through them.
Phase 2 (Backtrack/DFS): Traverse only positions where dp[i] is True. This bounds the search to live branches only, making the algorithm correct and as fast as possible given the output size.

2 The Algorithm: DP + Memoized Backtracking ▾

Why is This Dynamic Programming?

DP applies when two conditions hold:

  1. Optimal Substructure: Whether position i is reachable depends entirely on shorter prefixes. dp[i] is True iff there exists some j < i with dp[j]=True and s[j:i] in the dictionary.
  2. Overlapping Subproblems: In a naive DFS, we'd recompute "what sentences can I make from s[7:]?" every time any prefix leads to position 7 — potentially many times. DP (or memoization) stores each suffix result exactly once.
📐 The Recurrence
dp[0] = True                       (empty prefix trivially OK)
dp[i] = any(dp[j] and s[j:i] in W, for j < i)
dfs(i) = all sentences from s[i:], memoized
answer = dfs(0)

Step-by-Step Algorithm

1Convert wordDict to a set for O(1) membership tests.
2Define dfs(i): returns list of all sentences from s[i:]. Check memo first.
3Base case: i == len(s) → return [""] (empty string sentinel, not empty list).
4For each j from i+1 to n: if s[i:j] in dict, recurse dfs(j). Prepend word to each result.
5Store result in memo[i] before returning. Call dfs(0).
⚠️ Why Not Pure Backtracking?

Without memoization, if "aaaaaa..." has words ["a","aa","aaa",...], pure DFS re-explores the same suffix from exponentially many different prefixes. Memoization collapses all those re-explorations into one stored result.

Brute-Force DFS

No caching. Re-computes each suffix many times. Can TLE on adversarial inputs. O(2ⁿ · n) time.

Memoized DFS (Optimal)

Each suffix computed exactly once. Exponential only in actual output size. O(n² + n·2ⁿ) overall.

3 Why It Works: The Invariant ▾
🔐 The DP Invariant

dp[i] is True if and only if there exists at least one way to partition s[0..i-1] into words all present in the dictionary. This is established left-to-right; once True, it never changes. Later entries depend on earlier ones only.

Why Backtracking is Correct Given Memoization

The memoized DFS computes dfs(i) = all sentences from s[i:]. When we try word s[i:j] and recurse on dfs(j), we combine: word + " " + each_tail. This covers every valid sentence because:

  1. We try all valid word prefixes from each position (completeness).
  2. We recurse on every valid continuation (no pruning of live branches).
  3. The base case [""] correctly terminates the last word (no over- or under-counting).

Edge Cases

  • No valid segmentation: dfs(0) returns [] — the outer loop finds no valid words, so result stays empty.
  • Single-character string: If "a" is in dict, dfs(0) tries j=1, finds "a", recurses dfs(1)=[""], builds "a". Correct.
  • Word longer than s: The slice s[i:j] for j > n is naturally empty or smaller — simply won't match any word.
  • Duplicate words in dict: Set handles deduplication; no correctness impact.
⚠️ Critical Detail: Return [""] not []

At the base case (i == len(s)), returning [] instead of [""] makes the caller's for tail in dfs(j): loop execute zero iterations — silently swallowing the last valid word. The [""] sentinel propagates the signal "we've consumed everything successfully."

Solution 2 — Explicit DP + Backtrack (Two-Phase)

n, ws = len(s), set(wordDict)
dp = [False]*(n+1); dp[0]=True ← Phase 1: fill dp[0..n]
for i in range(1, n+1):
for j in range(i):
if dp[j] and s[j:i] in ws:← j reachable + word valid?
dp[i]=True; break ← mark reachable; skip rest
if not dp[n]: return [] ← early exit: no segmentation
res = []
def bt(i, path): ← Phase 2: backtrack using dp
if i==n: res.append(" ".join(path)); return
for j in range(i+1, n+1):
if dp[j] and s[i:j] in ws:← only recurse where dp is True
path.append(s[i:j]); bt(j,path); path.pop()← backtrack step
bt(0, []); return res
💡 Optimization: Limit j by max word length

If the dictionary's longest word has length L, then s[i:j] for j > i+L can never be in the dictionary. Replace range(i+1, n+1) with range(i+1, min(n,i+L)+1) in both loops. This reduces the inner loop from O(n) to O(L) steps — significant when L is small.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize "All Ways" = Backtracking + Pruning

Whenever a problem says "return all valid X," think backtracking. Whenever it also says "partition/segment a string using a dictionary," think DP for feasibility pruning.

Step 2: Define the Recursive Subproblem

Ask: "If I know all sentences from s[j:], how do I build sentences from s[i:]?" Answer: try every valid word s[i:j], prepend it to each result from dfs(j). This is the recurrence.

Step 3: Identify Memoization Opportunity

The same suffix s[j:] can be reached from many different prefixes. Without memo, dfs(j) runs many times returning the same list. Add memo = {} and cache the result.

Step 4: Handle the Base Case Carefully

When i == len(s), return [""] not []. This is a deliberate sentinel — the empty string propagates correctly through the string-building logic without creating a spurious space.

🗣️ What to Say to the Interviewer

"This asks for all valid sentences — a signal for DFS + memoization.
My subproblem: dfs(i) = all sentences from s[i:] onward.
For each valid word s[i:j], I recurse dfs(j) and prepend the word.
The key twist: memoize dfs to avoid recomputing the same suffix.
Base case: i == len(s) returns [''] so the last word appends cleanly.
Time: O(n² inner loop + n·2^n output construction). Space: O(n·2^n) for the results.
Shall I also walk through the explicit DP + backtrack two-phase version?"
💡 Likely Follow-Ups
  • Word Break I (LC 139): Return True/False only — just run the DP, skip backtracking.
  • Huge dictionary? Use a Trie so each character of the window is matched incrementally, not by hashing the whole slice.
  • Count sentences instead of listing? Change result = [] to result = 0 and add counts instead of appending strings.

7 Summary & Key Takeaways ▾
When a problem asks for "all valid segmentations," use DFS + memoization — not pure DP alone.
The base case must return [""] (not []) so the final word gets appended correctly.
Memoizing per starting index ensures each suffix is computed exactly once, regardless of how many prefixes lead to it.
The explicit DP pre-check (dp[n]) allows an O(n²) early exit when no segmentation is possible at all.
Output is unavoidably O(n·2ⁿ) when many sentences exist — exponential complexity here is irreducible.

🎯 When to Use This Pattern
  • Problem says "return all ways to partition/segment" a string or array.
  • Tokens come from a finite set (dictionary, alphabet, numbers).
  • Reachability of position i depends only on earlier positions (DP-amenable).
  • Subproblem f(i) (all solutions from position i) recurs via smaller indices.
