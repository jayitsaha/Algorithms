LeetCode #1749

Maximum Absolute Sum of Any Subarray

You are given an integer array nums. Return the maximum absolute sum of any (possibly empty) subarray of nums.

Medium Array Manipulation Kadane's Algorithm Time: O(n) Space: O(1)
1 Understanding the Problem ▾

A subarray is a contiguous slice of an array. Every single element is a subarray. The empty subarray has sum 0. The absolute sum of a subarray is the absolute value of its total — so a subarray summing to −15 scores 15, just as a subarray summing to +15 scores 15.

We want the subarray whose absolute sum is the largest possible. This is different from "maximum subarray sum" — here, a deeply negative subarray might win.

💡 Key Insight

The winner must be either the subarray with the largest positive sum (caught by standard Kadane's) or the subarray with the most negative sum (caught by Kadane's running min). We never need to search all O(n²) subarrays — just run Kadane's twice in a single pass and take the larger absolute value.

Example A — positive wins

nums = [1, −3, 2, 3, −4]
Max subarray [2, 3] → sum = 5
Min subarray [−4] → sum = −4
Answer: max(|5|, |−4|) = 5

Example B — negative wins

nums = [2, −5, 1, −4, 3, −2]
Max subarray [2] → sum = 2
Min subarray [2,−5,1,−4] → sum = −6
Answer: max(|2|, |−6|) = 6

Edge Cases to consider: a single negative number (answer is its absolute value), an all-positive array (answer is the total sum), an all-negative array (answer is the magnitude of the most-negative element), and the empty array (allowed, sum = 0, so the result is always ≥ 0).

2 The Algorithm: Dual Kadane's Pass ▾

The core trick is recognising that max(max_subarray_sum, |min_subarray_sum|) gives us the answer. Both are instances of Kadane's algorithm — one tracking a running maximum, one tracking a running minimum.

Pass 1 — Maximum Kadane

At each element, decide: "Is it better to extend the current positive run, or restart fresh from here?" Track the globally best result.

curr_max = max(num, curr_max + num)
max_sum = max(max_sum, curr_max)

Pass 2 — Minimum Kadane

Symmetrically: "Is it better to extend the current negative run, or restart here?" Track the globally most-negative result.

curr_min = min(num, curr_min + num)
min_sum = min(min_sum, curr_min)

Both passes happen in the same loop — we do four updates per element. Total cost: O(n) time, O(1) space.

1 Initialize: All four variables start at 0. The baseline is the empty subarray (sum = 0), which is always a valid candidate.
2 Single scan: For each num, update curr_max (extend or restart positive), then max_sum; then update curr_min (extend or restart negative), then min_sum.
3 Final answer: max(max_sum, abs(min_sum)). Whichever absolute value is larger is the answer.
🔑 Prefix Sum Alternative (same complexity)

Any subarray sum equals prefix[j] − prefix[i]. Maximising |prefix[j] − prefix[i]| means maximising the spread between the largest and smallest prefix values. So the answer equals max(prefix_sums) − min(prefix_sums) — a one-pass O(n)/O(1) solution that is often easier to derive on a whiteboard.

3 Why It Works: The Invariant ▾
🔐 Kadane's Invariant

At each index i, curr_max holds the maximum sum of any subarray ending exactly at i. If curr_max + nums[i] < nums[i], the previous running sum was negative — it drags us down, so we restart at nums[i] alone. By taking the global max over all ending positions, we guarantee we find the optimal subarray among all O(n²) candidates.

Why max + |min| covers all cases: For any subarray, its sum is either non-negative (caught by max Kadane) or negative (caught by min Kadane). An absolute value collapses these two cases. The overall maximum absolute value is therefore max(max_sum, abs(min_sum)). No third case exists.

Worked trace for nums = [2, −5, 1, −4, 3, −2]:

start:max=0, cmax=0min=0, cmin=0
num=2:cmax=max(2,0+2)=2 → max=2cmin=min(2,0+2)=2 → min=0
num=−5:cmax=max(−5,2−5)=−3 → max=2cmin=min(−5,2−5)=−5 → min=−5
num=1:cmax=max(1,−3+1)=1 → max=2cmin=min(1,−5+1)=−4 → min=−5
num=−4:cmax=max(−4,1−4)=−3 → max=2cmin=min(−4,−4−4)=−8 → min=−8
num=3:cmax=max(3,−3+3)=3 → max=3cmin=min(3,−8+3)=−5 → min=−8
num=−2:cmax=max(−2,3−2)=1 → max=3cmin=min(−2,−5−2)=−7 → min=−8
answer:max(3, |−8|)= max(3, 8) = 8 ← [2,−5,1,−4] sums to −8
⚠️ Why Not Just abs(max_sum)?

In the example above, max_sum = 3 but the answer is 8. Taking abs(max_sum) would give 3 — catastrophically wrong. The most-negative subarray [2, −5, 1, −4] = −8 has a larger absolute value than any positive subarray. This is why we must run both Kadane passes independently.

4 The Code (Annotated) ▾

Solution 1 — Dual Kadane's (Optimal, O(n) time, O(1) space) — Interview Pick

def maxAbsoluteSum(nums: list[int]) -> int:
max_sum = min_sum = 0 ← Global bests; start at 0 (empty subarray baseline)
curr_max = curr_min = 0 ← Running Kadane accumulators for the current window
for num in nums: ← Single O(n) pass — do both passes simultaneously
curr_max = max(num, curr_max + num) ← Extend positive run or restart fresh
max_sum = max(max_sum, curr_max) ← Update global maximum
curr_min = min(num, curr_min + num) ← Extend negative run or restart fresh
min_sum = min(min_sum, curr_min) ← Update global minimum (most negative)
return max(max_sum, abs(min_sum)) ← Larger of the two magnitudes
⚠️ Order of Updates Within the Loop

Update curr_max before max_sum, and curr_min before min_sum. Within each pair the order matters — we must compute the new running value before comparing it to the global best. The two pairs (max pair, min pair) can be interleaved in any order relative to each other.

Solution 2 — Prefix Sum Spread (equally optimal, alternative derivation)

def maxAbsoluteSum(nums: list[int]) -> int:
prefix = max_p = min_p = 0 ← Running prefix sum and its extremes
for num in nums:
prefix += num ← Extend prefix sum by one element
max_p = max(max_p, prefix) ← Track largest prefix sum seen
min_p = min(min_p, prefix) ← Track smallest prefix sum seen
return max_p - min_p ← Max spread of prefix sums = max |subarray sum|
💡 Why max_prefix − min_prefix Equals the Answer

Any subarray sum = prefix[j] − prefix[i−1]. To maximise |prefix[j] − prefix[i−1]|, we want the largest possible difference between two prefix values. Since prefix[0] = 0 is always included (representing the empty prefix), max_p − min_p correctly handles the empty subarray and finds the optimal pair in O(n).

Solution 3 — Brute Force O(n²) (for reference only)

def maxAbsoluteSum(nums: list[int]) -> int:
result = 0
for i in range(len(nums)): ← Try every start index
total = 0
for j in range(i, len(nums)): ← Try every end index
total += nums[j] ← Extend subarray to include nums[j]
result = max(result, abs(total)) ← Track best absolute sum
return result ← Correct but O(n²) — use this to verify, not to submit
5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognise the Absolute Value Split

When you see "maximum absolute sum", immediately split it: max |x| = max(x_positive, |x_negative|). This converts one hard problem into two classic Kadane problems. State this insight explicitly — it shows strong pattern recognition.

Step 2: Propose Brute Force First

"I can try every pair (i, j) and compute the subarray sum — O(n²) time, O(1) space. It's correct but too slow for large inputs. Let me optimize."

Step 3: Derive Kadane's from First Principles

"Let me define curr_max[i] = best sum of any subarray ending at i. The recurrence is: curr_max[i] = max(nums[i], curr_max[i−1] + nums[i]). If the previous best was negative, restart. I track the global max. I run the same logic for the minimum with min instead of max."

Step 4: Offer the Prefix Sum Alternative

"Alternatively: the answer equals max(prefix_sums) − min(prefix_sums). This is sometimes easier to explain — we want the largest spread between any two prefix values. Same O(n)/O(1) complexity. Both work; I'll code whichever you prefer."

🗣️ What to Say to the Interviewer

"The absolute value means I need the most extreme subarray — either the largest positive sum or the most negative sum, whichever has greater magnitude.
I'll run Kadane's algorithm twice in a single pass: curr_max tracks the best positive running sum, curr_min tracks the most negative running sum.
At each element, I choose: extend the current run or restart from this element alone.
Final answer: max(max_sum, abs(min_sum)). Time O(n), Space O(1)."
🔮 Likely Follow-up Questions
  • "What if the array can wrap around (circular)?" → Max circular = max(linear max, total − linear min). (#918)
  • "Can you return the actual subarray, not just the sum?" → Track start/end indices when global best updates.
  • "What if nums can have up to 10⁵ elements?" → O(n) is required; O(n²) TLEs at that scale.
  • "What's the minimum absolute sum?" → Always 0 (empty subarray), since absolute values are non-negative.

7 Summary & Key Takeaways ▾
max |subarray sum| = max(max subarray sum, |min subarray sum|)
Run Kadane's algorithm twice in a single pass — four updates per element
Initialise all accumulators to 0: the empty subarray is always valid
Prefix spread (max_prefix − min_prefix) is an equivalent O(n)/O(1) derivation
Never take abs(max_sum) alone — the most-negative subarray can dominate
ApproachTimeSpaceNotes
Brute Force (nested loops) O(n²) O(1) Correct but TLEs at scale

Dual Kadane's ✓ O(n) O(1) Interview pick — single pass, minimal state

🎯 When to Recognise Kadane's Algorithm
  • Problem says "subarray" + "sum" + "maximum/minimum" — Kadane's is the default O(n) tool
  • "Absolute" or "magnitude" variant → run both max and min Kadane simultaneously
  • "Circular" variant → Kadane + (total − min subarray)
  • "At most k deletions" variant → DP building on Kadane's recurrence
