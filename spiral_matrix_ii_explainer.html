LeetCode #59

Spiral Matrix II

Given a positive integer n, generate an n×n matrix filled with elements from 1 to n² in spiral order.

Medium Matrix Generate by Spiral Order Time: O(n²) Space: O(n²)
1 Understanding the Problem ▾

We need to generate an n×n grid and fill it with the integers 1 through n² by tracing a clockwise spiral path. The path always begins at the top-left corner, moves right → down → left → up, and spirals inward layer by layer until every cell holds a unique value.

💡 Key Insight

A spiral is nothing but four repeated linear sweeps: right across the top, down the right edge, left across the bottom, up the left edge — then shrink all four boundaries inward and repeat. We fill cells sequentially and never revisit.

Concrete example — n = 3: Fill positions as the spiral visits them:

→ Pass 1 (top row): [0,0]=1, [0,1]=2, [0,2]=3
↓ Pass 2 (right col): [1,2]=4, [2,2]=5
← Pass 3 (bottom row): [2,1]=6, [2,0]=7
↑ Pass 4 (left col): [1,0]=8
→ Pass 5 (center): [1,1]=9
Result: [[1,2,3],[8,9,4],[7,6,5]]
✓ n = 1 (edge case)

Output: [[1]]
Single cell — top row pass fills [0,0]=1 and we're done.

✗ Forgetting to shrink bounds

Without shrinking top after filling the top row, the next iteration would overwrite cells in row 0 again. Each boundary must tighten immediately after its direction pass completes.

Key observations: n² total cells; each cell written exactly once; for odd n there is one center cell (value n²); for even n no center cell exists. The algorithm visits cells in strict ascending order 1, 2, … n².

2 The Algorithm: Four-Boundary Simulation ▾

We track four shrinking boundary pointers: top, bottom, left, right. They define the rectangle of cells still to be filled. Each direction pass fills one edge of that rectangle and then pulls the corresponding boundary one step inward.

1 Initialize: Build an n×n zero matrix. Set top=0, bottom=n−1, left=0, right=n−1. Set counter num=1.
2 → Fill top row: For c in [left .. right]: matrix[top][c] = num++. Then top += 1 (row consumed).
3 ↓ Fill right column: For r in [top .. bottom]: matrix[r][right] = num++. Then right -= 1 (column consumed).
4 ← Fill bottom row: For c in [right .. left] descending: matrix[bottom][c] = num++. Then bottom -= 1 (row consumed).
5 ↑ Fill left column: For r in [bottom .. top] descending: matrix[r][left] = num++. Then left += 1 (column consumed).
6 Repeat steps 2–5 as long as num ≤ n². The boundaries converge inward until all cells are filled.
🔑 Intuition: How to Discover This

Reframe: A spiral is a sequence of straight-line segments that turn clockwise. Each segment is bounded by a shrinking rectangle. Key observation: after finishing one side of the rectangle, that side will never be visited again — shrink it and move on. This transforms an intimidating 2D problem into a simple loop of four bounded for-loops.

🔑 Direction Vector Alternative

Another approach: keep a dirs = [(0,1),(1,0),(0,-1),(-1,0)] cycle (right/down/left/up). Place num, advance one step in the current direction; if the next cell is out-of-bounds or already nonzero, rotate 90° clockwise and try again. Same O(n²) complexity — more compact but the boundary approach is clearer in interviews.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the start of every while loop iteration, the rectangle defined by [top..bottom] × [left..right] contains exactly the unfilled cells. Every cell outside this rectangle has already been assigned its final value. After each direction pass, the corresponding boundary shrinks by 1, maintaining this invariant for the next iteration.

Why boundary shrinking prevents overwrites: After filling the top row, every cell in row top has been written. Advancing top permanently excludes that row from all future passes — we cannot accidentally revisit it.

Why the termination guard is just num <= n*n: We increment num exactly once per cell written. Since num starts at 1 and the last cell gets n², the condition num <= n² holds for exactly n² writes, then becomes false. No mid-loop boundary checks are needed for a square matrix (unlike Spiral Matrix I which handles rectangular grids).

⚠️ Key Difference from Spiral Matrix I

Spiral Matrix I (#54) reads an existing m×n rectangular matrix and needs guards like if top > bottom: break between passes because a single-row remaining strip could be traversed twice (once going right, once going left). Here we generate a square matrix and fill exactly n² cells total — the outer while condition handles termination safely without internal checks.

Odd vs even n: For odd n, the boundaries eventually collapse to top == bottom == left == right, giving a 1×1 ring that the top-row pass fills in a single step (value n²). For even n, the innermost ring is always 2×2 or larger — all four direction passes execute on the final iteration.

4 The Code (Annotated) ▾

Solution 1 — Four-Boundary Simulation (Optimal, O(n²) time, O(1) auxiliary)

def generateMatrix(n: int) -> List[List[int]]:
matrix = [[0]*n for _ in range(n)] ← n×n grid initialized to 0
top, bottom, left, right = 0, n-1, 0, n-1 ← four boundary pointers
num = 1 ← next integer to place (1 → n²)
while num <= n * n: ← loop while unfilled cells remain
for c in range(left, right+1): ← → fill top row left→right
matrix[top][c] = num; num += 1
top += 1 ← top row consumed; shrink boundary down
for r in range(top, bottom+1): ← ↓ fill right column top→bottom
matrix[r][right] = num; num += 1
right -= 1 ← right col consumed; shrink boundary left
for c in range(right, left-1, -1): ← ← fill bottom row right→left
matrix[bottom][c] = num; num += 1
bottom -= 1 ← bottom row consumed; shrink boundary up
for r in range(bottom, top-1, -1): ← ↑ fill left column bottom→top
matrix[r][left] = num; num += 1
left += 1 ← left col consumed; shrink boundary right
return matrix
⚠️ Why no guard between the bottom-row and left-column passes?

When the boundaries converge to a single cell (odd n, last step), num reaches n² during the top-row pass, and the outer while loop exits before reaching the bottom-row pass. So the extra for-loops are simply never executed — we don't need if top > bottom: break in square generation mode.

Solution 2 — Direction Vector (Compact Alternative, same complexity)

def generateMatrix(n: int) -> List[List[int]]:
matrix = [[0]*n for _ in range(n)]
dirs = [(0,1),(1,0),(0,-1),(-1,0)] ← right, down, left, up
d, r, c = 0, 0, 0 ← direction index, current row, current col
for num in range(1, n*n+1): ← place values 1..n² in order
matrix[r][c] = num
nr = r + dirs[d][0] ← look one step ahead
nc = c + dirs[d][1]
if not (0<=nr<n and 0<=nc<n and matrix[nr][nc]==0):← blocked?
d = (d + 1) % 4 ← turn clockwise
nr = r + dirs[d][0]
nc = c + dirs[d][1]
r, c = nr, nc
return matrix
💡 Complexity Summary

Both approaches: Time O(n²) — each of the n² cells is written exactly once.
Space: O(n²) for the output matrix. Auxiliary space is O(1) — only a handful of integer variables are used beyond the matrix itself.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern Signal

The moment you see "spiral" + "fill/generate" + matrix, the canonical approach is four-boundary simulation. Signals: fill every cell once, fixed clockwise direction cycle, shrink the active region after each sweep.

Step 2: Trace a Small Example (n=3) by Hand

Before writing code, draw a 3×3 grid on the whiteboard and label each sweep. After filling row 0 (→), circle it and say "row 0 is done, top boundary moves down." After filling column 2 (↓), circle it and say "col 2 is done, right boundary moves left." This live tracing reveals the boundary structure naturally — it impresses interviewers and catches off-by-one errors before they become bugs.

Step 3: Code with Named Boundary Variables

Use four explicitly named variables: top, bottom, left, right. This is far clearer than a single "layer" counter or magic arithmetic. Write each direction as its own for-loop followed by the corresponding boundary increment/decrement. Guard the outer loop with num <= n*n.

Step 4: Discuss the Alternative and Trade-offs

Mention: "An alternative is a direction vector array that cycles right→down→left→up and turns clockwise when the next cell is out-of-bounds or already written. It's more compact but the boundary approach makes the algorithm's intent self-documenting." Both are O(n²)/O(1) auxiliary.

🗣️ What to Say to the Interviewer

"I'll maintain four boundary pointers: top, bottom, left, right — forming a shrinking rectangle.
Each iteration: fill the top row left-to-right, then advance top. Fill the right column top-to-bottom, then retreat right.
Fill the bottom row right-to-left, then retreat bottom. Fill the left column bottom-to-top, then advance left.
Loop while our counter num hasn't exceeded n-squared.
Each cell is written exactly once, so time is O(n²). Auxiliary space is O(1) — just four boundary integers and a counter."
🔮 Likely Follow-up Questions
  • "Read back spiral order of an existing matrix?" → Spiral Matrix I (#54) — same boundaries but read instead of write; add mid-loop guards for non-square matrices.
  • "Spiral starting from center outward?" → Reverse the counter: start from n² and decrement, or build the matrix inside-out.
  • "What about a non-square m×n matrix?" → Add guards if top > bottom: break and if left > right: break between the last two direction passes.
  • "Can we avoid initializing to zero?" → Use the direction-vector approach: check matrix[nr][nc] == 0 as the visited sentinel.

7 Summary & Key Takeaways ▾
Four shrinking boundaries model each spiral ring — pull inward after every direction sweep
Guard the outer loop with num <= n*n — no mid-loop boundary checks needed for square generation
Each cell written exactly once → O(n²) time, O(1) auxiliary space
For odd n, the last value n² lands in the single center cell during a degenerate 1-cell top-row pass
Direction vector alternative cycles (0,1)→(1,0)→(0,-1)→(-1,0) and turns when the next cell is blocked
