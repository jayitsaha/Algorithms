LeetCode #723

Candy Crush

Simulate the Candy Crush game: repeatedly find groups of 3+ identical adjacent candies, crush them, apply gravity, and repeat until stable.

Medium Matrix Mark + Gravity Simulation Time: O((R·C)²) Space: O(1)
1 Understanding the Problem ▾

You are given a 2D integer grid (the board). Non-zero values represent candy types. Zero means an empty cell. Your task is to simulate the Candy Crush game until no further moves are possible.

One round consists of exactly three sub-steps:

  1. Identify all groups of 3 or more identical candies that are horizontally or vertically adjacent.
  2. Crush them simultaneously — set those cells to 0. A candy that belongs to both a horizontal and a vertical group is still only crushed once.
  3. Gravity — in each column independently, non-zero candies fall to the bottom. Zeros float to the top. Relative order of surviving candies is preserved.

Repeat until no group of 3+ identical adjacent candies exists. Return the final board.

💡 Key Insight

The crucial subtlety is simultaneous crushing. You must not remove a candy mid-scan, because doing so could destroy a neighbour's group eligibility. The solution: use a mark phase (flag all candidates) followed by a separate crush phase (zero them all). The in-place negation trick lets you mark without using any extra space.

✗ Wrong: Remove mid-scan

Crush [1,1,1] in row 0 immediately → that row changes → when you check columns, the zero disrupts vertical group detection. The board is in an inconsistent state.

✓ Correct: Mark, then crush

Negate all matches in one full pass. Then zero all negatives. A cell already marked (negative) still participates in neighbour checks via abs() — no inconsistency.

Edge cases: A 0-valued cell can never start a group (checked with != 0 condition). Groups of exactly 3 qualify; groups of 4 or 5 also qualify (all members get crushed). A candy at the intersection of a horizontal and vertical group is correctly negated twice — -abs(x) keeps it negative regardless.

2 The Algorithm: Mark + Gravity Simulation ▾

We run a loop that terminates when an entire mark pass produces no new marks. Each iteration has four distinct phases:

① MARK H
negate horiz groups
→
② MARK V
negate vert groups
→
③ CRUSH
zero negatives
→
④ GRAVITY
sink non-zeros
1 Mark horizontal groups: For each cell (r, c) where c ≤ C−3, check if abs(board[r][c]) == abs(board[r][c+1]) == abs(board[r][c+2]) != 0. If true, negate all three using -abs(val). Set crushed = True.
2 Mark vertical groups: Same scan but checking board[r][c], board[r+1][c], board[r+2][c] for r ≤ R−3. This completes marking for the full round.
3 Check + Crush: If crushed is False, the board is stable — break and return. Otherwise, zero out every cell with a negative value. This is the actual removal.
4 Gravity: For each column, collect all non-zero values (top to bottom). The number of zeros added at the top equals R − len(non_zeros). Write these back into the column. This simulates candies falling due to gravity.
🔑 Why Use -abs() as the Mark Sentinel?

We need to flag a cell for removal without losing its candy type (value). Negating the value serves as a flag. But a cell might be marked twice — once horizontal, once vertical. If it's already -3 and we apply -board[r][c], we'd get +3 and accidentally un-mark it! Using -abs(board[r][c]) always produces a negative, regardless of current sign — safely idempotent.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After every complete round (both mark passes + crush + gravity), the board is in a valid gravity-settled state: in every column, all non-zero values appear below all zeros. The loop halts when an entire round's mark passes produce no negatives — meaning no valid group of 3+ exists anywhere on the board.

Correctness of simultaneous crush: Using abs() for comparisons means marked (negative) cells still appear as their original candy type to their neighbours. A group that straddles a previously-marked cell is still correctly detected. This faithfully implements the "all eligible groups crush at the same time" game rule.

Correctness of gravity: Collecting non-zeros top-to-bottom and placing them at the bottom of the column preserves relative order (stable partition). Columns are independent — no horizontal shuffle occurs — so each column can be processed separately.

Termination: Every round that does not immediately break must crush at least 3 cells (a minimum group is 3 candies). Since the total number of non-zero cells is finite and strictly decreasing per active round, the loop must terminate in at most ⌊R·C/3⌋ rounds, giving O((R·C)²) total work.

⚠️ Off-by-One Loop Bounds

Horizontal scan: for c in range(C - 2) — the last window starts at C−3, so c+2 = C−1 is in-bounds. Vertical scan: for r in range(R - 2) — same logic for rows. Using range(C - 1) would cause an IndexError on the right boundary.

4 The Code (Annotated) ▾

Solution 1 — Mark + Gravity, In-Place Negation (Optimal · Interview Pick)

def candyCrush(board): ← accepts List[List[int]], mutates in-place
R, C = len(board), len(board[0]) ← cache dimensions for repeated use
while True: ← loop until board is stable
crushed = False ← did we mark anything this round?
for r in range(R): ← Phase 1: scan all rows for horizontal groups
for c in range(C - 2): ← window needs c, c+1, c+2 all in-bounds
a = abs(board[r][c]) ← abs() handles already-negated cells
if a and a==abs(board[r][c+1])==abs(board[r][c+2]):
board[r][c] = -abs(board[r][c]) ← mark (idempotent negate)
board[r][c+1] = -abs(board[r][c+1])
board[r][c+2] = -abs(board[r][c+2])
crushed = True ← note that a change occurred
for r in range(R - 2): ← Phase 2: scan columns for vertical groups
for c in range(C):
a = abs(board[r][c]) ← same abs() pattern
if a and a==abs(board[r+1][c])==abs(board[r+2][c]):
board[r][c] = -abs(board[r][c])
board[r+1][c] = -abs(board[r+1][c])
board[r+2][c] = -abs(board[r+2][c])
crushed = True
if not crushed: break ← no marks → board is already stable
for r in range(R): ← Phase 3: crush — zero out all marked cells
for c in range(C):
if board[r][c] < 0: board[r][c] = 0 ← negative = was marked = crush it
for c in range(C): ← Phase 4: gravity, column by column
nz = [board[r][c] for r in range(R) if board[r][c]]← surviving candies, top→bottom
gap = R - len(nz) ← how many empty slots appear at the top
for r in range(R): ← rewrite column: zeros top, candies bottom
board[r][c] = 0 if r < gap else nz[r - gap]
return board
⚠️ Why not just write board[r][c] = -board[r][c]?

If cell (r,c) is already marked negative (say -3) from the horizontal pass, and the vertical pass marks it again with -board[r][c] = -(-3) = 3, we've accidentally un-marked it! The cell would survive the crush phase. Using -abs(board[r][c]) guarantees the result is always negative, making marking idempotent — safe to apply multiple times.

Solution 2 — Separate boolean matrix (clearer, O(R·C) space)

def candyCrush(board): ← alternative with explicit crush set
R, C = len(board), len(board[0])
changed = True
while changed:
crush = set() ← set of (r,c) to zero out
for r in range(R): ← horizontal pass
for c in range(C - 2):
if board[r][c] and board[r][c]==board[r][c+1]==board[r][c+2]:
crush |= {(r,c),(r,c+1),(r,c+2)}
for r in range(R - 2): ← vertical pass
for c in range(C):
if board[r][c] and board[r][c]==board[r+1][c]==board[r+2][c]:
crush |= {(r,c),(r+1,c),(r+2,c)}
changed = bool(crush) ← any matches found?
for r,c in crush: board[r][c] = 0 ← crush all at once
for c in range(C): ← gravity (same as before)
nz=[board[r][c] for r in range(R) if board[r][c]]
gap=R-len(nz)
for r in range(R): board[r][c]=0 if r<gap else nz[r-gap]
return board
💡 Which solution to use in interviews?

The negation trick (Solution 1) is the interview-preferred approach. It shows mastery of in-place sentinel tricks and avoids extra data structures. Solution 2 is perfectly correct and easier to reason about — mention it as the "cleaner but O(R·C) space" alternative, then code Solution 1.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Decompose the Game Loop

The problem says "repeat until stable." This is always a simulation loop. Ask: what is one round? The problem defines it for you: mark → crush → gravity. Name the phases explicitly — it shows structure.

Step 2: Spot the Simultaneous-Crush Subtlety

The word "simultaneously" is a red flag. It means you cannot remove a candy mid-scan. Explicitly state this: "I need to find ALL groups first, then remove them all at once." This is what separates a correct solution from an incorrect one.

Step 3: Choose Your Marking Strategy

Start with the set-based approach (Solution 2) for clarity. Then improve to the negation trick: "Instead of a separate set, I can mark in-place by negating. I use abs() for comparisons so already-marked cells still match. This saves O(R·C) space."

Step 4: Explain Gravity Precisely

Say "for each column independently, I collect the non-zero values (preserving top-to-bottom order), then write them back starting at the bottom. Zeros fill the top." This is a stable in-place partition on a 1D slice.

🗣️ What to Say to the Interviewer

"This is a simulation problem. Each round: mark groups of 3+ identical adjacent candies, crush them, apply gravity.
The key subtlety: crushing is simultaneous, so I need to mark first and remove after a full scan.
I'll use in-place negation as my mark sentinel — negate the candy value. I compare using abs() so already-marked cells still register as their candy type.
After marking, if nothing was marked, I break — board is stable. Otherwise zero out negatives, then apply gravity column-by-column: collect non-zeros, write back at bottom.
Time: O((R·C)²). Space: O(1). Each round removes at least 3 candies, so we loop at most O(R·C) times."
🔮 Likely Follow-up Questions
  • "What if groups of 2 count?" → Change window to c+1 / r+1 checks (window size 2).
  • "What if candies fall sideways too?" → Add a horizontal gravity pass after vertical gravity.
  • "Count the score (sum of crushed candy values)?" → Accumulate abs(board[r][c]) before zeroing.
  • "Can you do it without mutation?" → Return a copy; cost is O(R·C) per round for copies.
  • "What if groups of 4 give a special power candy?" → Detect group size in the mark pass and record it.

7 Summary & Key Takeaways ▾
Two-phase per round: mark first (negate), then crush — never modify mid-scan
Use -abs(x) to mark — idempotent, safe to apply multiple times to same cell
Use abs() when comparing — lets marked (negative) cells still match neighbours
Gravity is a per-column stable partition: non-zeros sink, zeros float to top
Exit condition: a full mark pass with crushed=False means the board is stable
Time O((R·C)²), Space O(1) — the in-place negation trick buys constant space
ApproachTimeSpaceNotes
Set-based crush O((R·C)²) O(R·C) Cleaner code; extra set per round
In-place Negation ✓ O((R·C)²) O(1) Interview pick; sentinel trick
🎯 When to Recognize Mark + Gravity Simulation
  • "Repeat until stable / no more moves" → outer simulation loop
  • "Remove groups simultaneously" → mark phase before remove phase
  • "Elements fall after removal" → gravity = column-wise stable partition
  • "In-place, space O(1)" → negation as sentinel (only works for non-negative integer grids)
