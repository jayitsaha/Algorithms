LeetCode #116

Populating Next Right Pointers in Each Node

Given a perfect binary tree, populate each node's next pointer to point to its next right node at the same level, or null if there is no next right node.

Medium Trees Level Order + Link Siblings Time: O(n) Space: O(1)
1 Understanding the Problem ▾

We have a perfect binary tree: every interior node has exactly two children, and all leaves are at the same depth. Each node starts with next = null. Our job: set each node's next to the node immediately to its right on the same level. The rightmost node on each level keeps next = null.

Key Insight

This is really a linked-list construction problem applied to each tree level. If we can visit nodes level by level (left to right), we can wire next pointers as we go. The perfect-tree guarantee means the structure is perfectly symmetric — we can even use the already-linked current level to traverse it like a linked list and wire up the next level, with zero extra storage.

Edge Cases: Single-node tree (root.next stays null, return root). Empty tree (return null). The perfect-tree constraint means we never need to handle a node with one child, which simplifies the cross-parent linking logic greatly.

2 The Algorithm: Level-by-Level Pointer Threading ▾

Two approaches exist. Start with BFS (intuitive), then optimize to O(1) space (impressive). Both are O(n) time.

Approach A — BFS Queue (Easy to Derive, Interview Starting Point)

Standard level-order traversal with a queue. Each outer iteration processes one level. Inside, snapshot the level size, then dequeue exactly that many nodes, linking each to the queue's front (the next peer), except the last node which stays null.

1 Seed queue with root. Start outer while loop.
2 Snapshot level size: size = len(q) before the inner loop so we know which nodes belong to this level.
3 Inner loop: for i in range(size): dequeue node. If i < size-1: node.next = q[0] (front of queue is the next peer). Enqueue left and right children.
4 Repeat until queue empty. Return root.
Approach B — O(1) Space: Use What You've Already Linked

Once level i's next pointers are set, we can traverse level i for free using those pointers (like a linked list). From each node we can reach its children (level i+1). Strategy: maintain a leftmost anchor for the start of the level being wired. Walk curr across via .next, wire children of curr, advance leftmost down when done.

1 leftmost = root. Outer loop runs while leftmost.left exists (leaf level has no children to link).
2 curr = leftmost. Inner loop walks across the current level via curr.next.
3 Same-parent link: curr.left.next = curr.right. Left child points to right child — always valid since every node has two children.
4 Cross-parent link (if curr.next): curr.right.next = curr.next.left. Right child points to the left child of curr's right neighbor. This is the crucial inter-family link.
5 Advance: curr = curr.next. When curr becomes null (end of level): leftmost = leftmost.left, repeat outer loop.
3 Why It Works: The Invariant ▾
The Invariant (O(1) Approach)

Before we start wiring level i+1: all next pointers at level i are already correctly set. This lets us traverse level i as a linked list via node.next and use each node to find and wire its children (level i+1).

Why the invariant holds by induction:

  • Base case: Root (level 0) has no peers. next = null by default. Invariant holds trivially.
  • Inductive step: Assuming level i is fully linked — walk curr across it. For each curr: curr.left.next = curr.right (same-parent). If curr.next: curr.right.next = curr.next.left (cross-parent). After processing all of level i, level i+1 is fully linked. QED.

Why This ONLY Works for Perfect Binary Trees

This O(1) approach depends on every node having exactly two children. If a node had only a left child (no right), curr.left.next = curr.right would assign null. For LeetCode #117 (general binary tree), you need a dummy-head prev-pointer technique that handles sparse levels.

4 The Code (Annotated) ▾

Solution 1 — BFS Queue (O(n) space)

from collections import deque
def connect(root): entry point
if not root: return None empty tree guard
q = deque([root]) seed with root
while q: process level by level
size = len(q) snapshot: how many nodes at THIS level
for i in range(size): process exactly this level
node = q.popleft() dequeue leftmost node of level
if i < size - 1: not the last node in level?
node.next = q[0] wire to next node (currently front of queue)
if node.left: q.append(node.left) enqueue left child for next level
if node.right: q.append(node.right) enqueue right child for next level
return root modified in-place; return root
Why snapshot size BEFORE the loop?

We need to know how many nodes are at the current level before we start adding children of the next level. Taking size = len(q) before the for-loop captures only the current level's count. Without this, we'd mix levels together.

Solution 2 — O(1) Space Iteration (Optimal — Interview Pick)

def connect(root): optimal: no auxiliary storage
if not root: return None
leftmost = root anchor: leftmost node of the level we're about to wire
while leftmost.left: stop at leaf level (no children to link)
curr = leftmost walk across current level using .next pointers
while curr: traverse via already-set next pointers
curr.left.next = curr.right same-parent link: left child -> right child
if curr.next: is there a sibling to curr on this level?
curr.right.next = curr.next.leftcross-parent link: curr's right -> sibling's left
curr = curr.next advance along level
leftmost = leftmost.left drop to next level's leftmost node
return root
Complexity Summary

BFS Queue: Time O(n) — every node visited once. Space O(n) — queue holds up to n/2 nodes (full leaf level).
O(1) Space: Time O(n) — every node visited once. Space O(1) — only leftmost and curr pointers.

5 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Level-Order Signal

The phrase "next right node on the same level" is a direct BFS trigger. Nodes at the same level = same BFS depth. Propose BFS first — it's O(n) time, O(n) space, correct, and easy to explain. This immediately earns credibility.

Step 2: Code the Queue BFS

Seed queue with root. Outer while loop: snapshot size. Inner for i in range(size): dequeue, if not last: node.next = q[0], enqueue children. Clean ~10-line solution. Run through the example mentally.

Step 3: Observe the Space and Optimize

After BFS works, say: "The queue uses O(n) space. Since this is a perfect binary tree, once level i is linked, I can traverse it for free via .next. I can wire the next level using only a few pointers." Explain the leftmost + curr strategy and the two link types.

Step 4: Emphasize the Two Link Types

The O(1) solution has exactly two pointer assignments: (1) curr.left.next = curr.right (same parent, always valid in a perfect tree), and (2) curr.right.next = curr.next.left (cross-parent, only when curr.next exists). Being explicit about this shows mastery.

What to Say to the Interviewer

"Since we need to link nodes at the same level, I'll use BFS — process one level at a time.
Seed the queue with root. Each outer iteration: snapshot size, inner loop dequeues size nodes,
links each to q[0] (the next peer in queue) unless it's the last node. Enqueue children.
That's O(n) time, O(n) space for the queue.

For O(1) space: since it's a perfect tree, after linking level i its next pointers form a free
linked list. Walk leftmost across using curr = leftmost, then curr = curr.next.
Two wires per curr: curr.left.next = curr.right, and if curr.next: curr.right.next = curr.next.left.
Then leftmost = leftmost.left for the next level. Pure O(1) space."
Likely Follow-up Questions
  • "What if the tree is not perfect?" → LeetCode #117: use a dummy-head prev-pointer per level to handle missing children.
  • "Can you solve recursively?" → Yes: recurse left first; then link left.right.next = right.left, and if node.next: link right.right.next = node.next.left; then recurse right.
  • "How much memory does the leaf level use in the queue?" → n/2 nodes = O(n) — this is the dominant term.
