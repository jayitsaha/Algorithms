LeetCode #71

Simplify Path

Given a Unix-style absolute file path, convert it to its simplified canonical form.

Medium Stacks Stack for Directory Components Time: O(n) Space: O(n)
1 Understanding the Problem ▾

A Unix filesystem path can be written in many equivalent ways: trailing slashes, double slashes, single-dot "current directory" references, and double-dot "go up" references. Your job is to normalize any messy path into the one clean canonical form.

Only three kinds of special tokens exist. Everything else is a real directory name:

Real name — Push

home, docs, notes, ... — any non-special token. Descend into this directory. Push it onto the stack.

. or empty — Skip

Single dot = "stay here." Empty string comes from double-slashes or leading/trailing slashes. In both cases: do nothing and move on.

.. — Pop

Double dot = "go up one level." Pop the stack. If already at root (stack empty), stay at root — you can't go above /.

💡 Key Insight

The current path from root is exactly a stack of directory names. "Descend" = push. "Go up" = pop. Ignore noise (dots and empty segments). At the end, join the stack with / separators and prefix / for the absolute path.

Messy inputs

/home//foo/
/../
/a/./b/../../c/
/a//b///c

Canonical outputs

/home/foo
/
/c
/a/b/c

Edge cases to consider: / alone returns /; /.. at root returns / (can't go above root); ... (three dots) is a valid directory name, not a special token; only .. exactly means "go up."

2 The Algorithm: Split + Stack ▾

The cleanest approach is to split the path on '/' first, converting the problem from "parse a string" into "process a list of tokens." Splitting automatically handles all slash variations — consecutive slashes produce empty string tokens, which we simply ignore.

1 Split on '/': path.split('/') gives a list of tokens. Leading / → empty first token. Trailing / → empty last token. Double // → extra empty token in the middle. All benign.
2 Initialize an empty stack: stack = []. The stack will hold the exact sequence of directory names on the current canonical path from root.
3 Process each token — 3-way branch: If token == '..' and stack non-empty → stack.pop(). Else if token is truthy and not '.' → stack.append(token). Else (empty string, '.', or '..' at root) → skip silently.
4 Assemble the canonical path: '/' + '/'.join(stack). The leading '/' is always present (this is an absolute path). An empty stack means we're at root, so the result is just '/'.
🔑 Why Split, Not Scan?

Scanning character-by-character requires you to accumulate characters into a buffer and detect token boundaries yourself — more code, more bugs. path.split('/') hands you clean tokens in one line. You only need to classify tokens, not parse them. Always prefer letting the language's string primitives do the heavy lifting.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing each token, stack contains exactly the sequence of directory names on the canonical absolute path at that point. Equivalently, '/' + '/'.join(stack) is always the correct simplified path to the current location in the filesystem.

Proof sketch: We start with an empty stack (at root — correct). For each token:

  • A real directory name: we descend one level. The invariant holds because we appended the new directory.
  • ..: we ascend one level. The invariant holds because we removed the last directory (we're now at the parent). If already at root, root has no parent, so we stay — still correct.
  • . or empty: no movement. Invariant trivially holds — stack unchanged.

Why splitting handles double slashes: "/a//b".split('/') yields ['', 'a', '', 'b']. The empty string fails the token truthiness check → skip. Whether there are one, two, or ten consecutive slashes, we always get empty tokens that are all silently discarded.

⚠️ Critical Distinction: "..." vs ".."

Three dots (...) is a perfectly valid Unix directory name. Only the string ".." exactly means "go up." Your condition must use token == '..' — a strict equality check. Never use '.' in token, token.startswith('..'), or len(token) == 2 alone.

⚠️ Pop Only When Stack is Non-Empty

At the root level, .. is a no-op. If you call stack.pop() on an empty list, Python raises IndexError. Always guard with if stack: stack.pop().

4 The Code (Annotated) ▾

Solution 1 — Stack with Split (Optimal, Interview Pick)

def simplifyPath(path: str) -> str:
stack = [] ← Stack holds confirmed directory components on current path
for token in path.split('/'): ← split('/') handles all slash variants; empty tokens come from //, leading /, trailing /
if token == '..': ← Exact check — '..' means "go up"
if stack: ← Guard: can't go above root; empty stack = already at root
stack.pop() ← Remove last directory (ascend to parent)
elif token and token != '.': ← Real name: truthy (not empty) AND not single-dot (not "stay here")
stack.append(token) ← Descend into this directory
return '/' + '/'.join(stack) ← Always absolute; empty stack → '/' + '' = '/'
⚠️ Why not just elif token not in ('', '.', '..')?

That would also work! elif token and token != '.' is equivalent because the outer if token == '..' already handled the .. case. The elif branch only runs when the token is NOT '..', so checking token != '.' and token (truthiness) is sufficient. Both formulations are correct — pick whichever reads more naturally to you.

Solution 2 — Brute Force Regex (Illustrative, avoid in interviews)

import re
def simplifyPath_regex(path: str) -> str:
p = path
while True: ← Repeat until no more simplifications possible
prev = p
p = re.sub(r'/+', '/', p) ← Collapse multiple slashes
p = re.sub(r'/\.(?=/|$)', '', p) ← Remove /. (current dir references)
p = re.sub(r'/[^/]+/\.\.(?=/|$)', '', p) ← Collapse /name/..
p = re.sub(r'^/\.\.', '', p) ← Remove .. at root
if p == prev: break ← Fixed point = done
return p or '/'
Why Regex is Worse Here

The regex approach is O(n²) in the worst case (e.g., /a/../a/../a/../ needs O(n) passes each of O(n) work). It's also fragile — missing one edge case in the regex pattern causes wrong answers. The stack solution does everything in O(n) and is trivially correct once you understand the invariant.

💡 Complexity Summary

Stack (optimal): Time O(n) — single split + single pass. Space O(n) for the stack (at most n/2 components in pathological input).
Regex loop: Time O(n²) worst case. Space O(n). Instructive to know but avoid in interviews.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Enumerate the Special Tokens

Before writing any code, list what's special: empty string (from slashes), single dot (current dir), double dot (parent dir). Everything else is a real name. Confirm with the interviewer: "Is ... a valid directory name?" The answer is yes.

Step 2: Model as a Stack

Say: "The path from root to my current location is naturally a stack. Entering a directory pushes, going up pops. That maps perfectly to a list in Python." This is the key insight — name it explicitly.

Step 3: Explain Why Split First

Say: "I'll split on '/' to get tokens. This naturally collapses all slash variations — consecutive slashes just produce empty tokens, which I'll ignore. Simpler than parsing characters."

Step 4: Walk Through an Example

Trace /a/b/../c on the whiteboard: split → ['', 'a', 'b', '..', 'c'], push 'a', push 'b', pop (back to ['a']), push 'c' → ['a', 'c'] → /a/c. This takes 30 seconds and eliminates all doubt.

🗣️ What to Say to the Interviewer

"The key observation: my current location is a stack of directory names.
I'll split the path on '/' to tokenize — empty strings from double slashes are then trivially ignored.
For each token: if it's '..' and I'm not at root, pop. If it's a real name, push. Otherwise skip.
At the end, '/' + '/'.join(stack) gives the canonical path. Empty stack is just '/'.
Time O(n), Space O(n). No edge-case handling needed — the stack model covers them all."
🔮 Likely Follow-up Questions
  • "Handle relative paths too?" → Same algorithm; just don't force-prefix /. Start from a known base directory.
  • "What if you're given a current directory and a relative path?" → Prepend current directory to the input, then simplify.
  • "Can you do it without splitting?" → Yes: scan char-by-char, accumulate into a buffer, flush on each /. Same O(n) complexity, more code.
  • "Design an in-memory filesystem?" → LeetCode #588 — uses a trie (not a stack), but path simplification is a sub-problem.

🎯 When to Recognize the Stack for Directory Components Pattern
  • Problem involves navigating or normalizing a hierarchical path
  • Tokens that "undo" previous tokens appear (go up = undo go down)
