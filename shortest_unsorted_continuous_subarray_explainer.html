LeetCode #581

Shortest Unsorted Continuous Subarray

Find the length of the shortest contiguous subarray that, when sorted, makes the entire array sorted in ascending order.

Medium Monotonic Stack Find Left/Right Boundaries Time: O(n) Space: O(1)
1 Understanding the Problem ▾

Given an integer array nums, we want to find the shortest contiguous subarray such that sorting only that subarray — leaving everything else unchanged — would make the whole array sorted in ascending order.

Think of it like this: you're looking at a shelf of books that's almost in order. There's one chaotic section in the middle. What's the narrowest possible span you'd need to pull out and rearrange to fix the whole shelf?

💡 Key Insight

An element belongs to the "unsorted middle" if it is smaller than the maximum element seen to its left (it's in the wrong place relative to the ascending prefix), OR it is larger than the minimum element seen to its right (it's in the wrong place relative to the descending suffix). The disorder window spans from the leftmost such element to the rightmost.

Input: [2, 6, 4, 8, 10, 9, 15]

6 > 4 → 4 is too small (disorder starts)
10 > 9 → 9 is too small (disorder extends)
Window: indices 1–5 = [6,4,8,10,9]
Length = 5

After sorting subarray [1..5]

[2, 4, 6, 8, 9, 10, 15]
Entire array is now sorted.
Answer = length of window = 5

Edge cases: Already-sorted array → 0; single element → 0; two-element reverse [3,1] → 2; all identical elements → 0; single out-of-place element (e.g., [1,3,2,4]) → window from index 1 to 2, length 2.

2 The Algorithm: Two-Pass Boundary Scan ▾

The O(n) solution uses two complementary linear scans — one from each direction — that together precisely locate both edges of the disorder window. No sorting required.

→ Left → Right Track running max.
If nums[i] < max → update right boundary.
← Right → Left Track running min.
If nums[i] > min → update left boundary.
1 Initialize sentinels: left = -1 and right = -1. These stay at −1 if no disorder is found (meaning the array is already sorted). Set max_so_far = -∞ and min_so_far = +∞.
2 Forward scan — find the RIGHT boundary: Scan left to right, maintaining max_so_far. Whenever nums[i] < max_so_far, element at index i is smaller than something before it (a violation). Update right = i. Keep updating — we want the last (rightmost) violation.
3 Backward scan — find the LEFT boundary: Scan right to left, maintaining min_so_far. Whenever nums[i] > min_so_far, element at index i is larger than something after it (a violation). Update left = i. Keep updating — we want the first (leftmost) violation.
4 Return the answer: If right == -1, no violations were found — array is sorted; return 0. Otherwise return right − left + 1 (the inclusive window length).
🔑 Why Forward Scan Finds the RIGHT Boundary

In a sorted array, every element is ≥ the running max. If nums[i] < max_so_far, then some larger element exists to the left of i. In sorted order, i should have come BEFORE that larger element — so it must move leftward, meaning it's definitely in the disorder window. We keep taking the rightmost such violation.

🔑 Why Backward Scan Finds the LEFT Boundary

Scanning right to left, min_so_far is the smallest element in the suffix seen so far. If nums[i] > min_so_far, then some smaller element exists to the right of i. In sorted order, i should come AFTER that smaller element — so it must move rightward. We keep taking the leftmost such violation.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After both passes: right is the index of the rightmost element that is out of place relative to the prefix (smaller than the max before it). left is the index of the leftmost element that is out of place relative to the suffix (larger than the min after it). The subarray nums[left..right] is exactly the minimal window that contains all disorder.

Why this span is both sufficient and necessary:

  • Sufficient: After sorting nums[left..right], every element to the left of left was already ≤ its neighbors (no violations), and every element to the right of right was already ≥ its neighbors. Sorting the window slots in the minimum and maximum of the window at the right spots.
  • Necessary: We cannot shrink the window further. left is the leftmost violation (elements before it need no movement). right is the rightmost violation (elements after it need no movement). Leaving either one out of the sorted window would leave a disorder unsolved.

Proof for sorted array: If nums is sorted, then for every i, nums[i] ≥ max_so_far (since max_so_far = nums[i-1] in a sorted sequence). So the forward scan never triggers right. Similarly, the backward scan never finds nums[i] > min_so_far. Both stay at −1; return 0. Correct.

⚠️ Common Mistakes

1. Off-by-one: The window is [left, right] inclusive. Length is right − left + 1, not right − left.

2. Swapping left and right: Right boundary comes from the FORWARD scan (looking for elements smaller than max). Left boundary comes from the BACKWARD scan (looking for elements larger than min). It's counterintuitive but correct — the forward scan tells you how far RIGHT the disorder extends.

3. Forgetting to update max/min in the right order: Update right first, THEN update max_so_far. Otherwise you'd compare the element against itself.

4 The Code (Annotated) ▾

Solution 1 — Two-Pass Boundary Scan (Optimal · O(n) time, O(1) space)

def findUnsortedSubarray(nums: list[int]) -> int:
n = len(nums) ← cache length
left, right = -1, -1 ← -1 = "not yet found"; if both stay -1, array is sorted
max_so_far = float('-inf') ← running max from the left; -inf so first element always updates it
min_so_far = float('inf') ← running min from the right; +inf so first element always updates it
for i in range(n): ← forward pass: find right boundary
if nums[i] < max_so_far: ← this element is smaller than something before it → out of place
right = i ← mark as rightmost disorder; keep overwriting to find LAST violation
max_so_far = max(max_so_far, nums[i])← update running max after the check
for i in range(n - 1, -1, -1): ← backward pass: find left boundary
if nums[i] > min_so_far: ← this element is larger than something after it → out of place
left = i ← mark as leftmost disorder; keep overwriting to find FIRST violation
min_so_far = min(min_so_far, nums[i])← update running min after the check
return 0 if right == -1 else right - left + 1← sorted? 0. Otherwise inclusive window size.
⚠️ Why check right == -1 and not left == -1?

If the array is sorted, neither scan ever fires. Both stay at −1. We only need to check one — any disorder visible in the forward scan is also visible in the backward scan (they're complementary views of the same disorder). Checking right == -1 is conventional; checking left == -1 works equally well.

Solution 2 — Sort and Compare (O(n log n), simpler to derive)

def findUnsortedSubarray(nums: list[int]) -> int:
sorted_nums = sorted(nums) ← O(n log n); creates a sorted copy for comparison
left, right = 0, len(nums) - 1 ← start from both ends
while left < len(nums) and nums[left] == sorted_nums[left]:
left += 1 ← skip prefix that already matches sorted order
if left == len(nums): return 0 ← every element matched → already sorted
while nums[right] == sorted_nums[right]:
right -= 1 ← skip suffix that already matches sorted order
return right - left + 1 ← window between first and last mismatch
💡 Complexity Summary

Two-Pass Boundary Scan: Time O(n) — two linear passes. Space O(1) — three variables only.
Sort & Compare: Time O(n log n) — one sort. Space O(n) — sorted copy.
Interview pick: Two-Pass Boundary Scan — optimal in both dimensions.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

Signal: "shortest subarray" + "sorted" + "contiguous." This is a boundary-finding problem. Ask yourself: "What property must an element satisfy to be in the disorder window?" The answer: it's either smaller than something to its left OR larger than something to its right.

Step 2: Propose the Sort Approach First

"Simplest approach: sort a copy, find first and last position where original and sorted differ. O(n log n) time, O(n) space. Correct but suboptimal. Can I do O(n)?"

Step 3: Derive the O(n) Insight

"For the right boundary: scan left-to-right, track max. Any element smaller than max is a disorder; the LAST such element is right. For the left boundary: scan right-to-left, track min. Any element larger than min is a disorder; the LAST such element (scanning backwards, so it's the FIRST in the array) is left."

Step 4: Code It and Handle Edge Case

Initialize left = right = -1. Run both passes. If right == -1, return 0. Otherwise return right − left + 1. Mention the "+1" for inclusive bounds.

🗣️ What to Say to the Interviewer

"I'll start with sorting: sort a copy and find where original and sorted first diverge — O(n log n). Can we do better?

Yes. I need to find two boundaries without sorting.
For the right boundary: scan left→right tracking running max. If an element is less than max, it must move left in sorted order — it's in the disorder zone. The LAST such element is my right boundary.
For the left boundary: scan right→left tracking running min. If an element is greater than min, it must move right in sorted order. The LAST such element (scanning backwards) is my left boundary.

Initialize left = right = -1. Two passes. Return 0 if right never updated, else right - left + 1.
Time O(n), Space O(1)."
🔮 Likely Follow-up Questions
  • "Can you do it in one pass?" → Not simply — the two scans solve complementary halves of the problem.
  • "What if there are duplicates?" → Handled correctly. Duplicates don't change max/min tracking logic.
  • "What's the actual unsorted subarray?" → Return nums[left:right+1] — boundaries already computed.
  • "How does this relate to a monotonic stack?" → The running max/min mimics the top of a monotone stack (descending for max, ascending for min). An explicit stack gives the same result in O(n) time and O(n) space.

7 Summary & Key Takeaways ▾
Forward scan (left→right) with running max finds the RIGHT boundary — the last element smaller than a preceding maximum
Backward scan (right→left) with running min finds the LEFT boundary — the last element larger than a following minimum
Both boundaries found in O(n) total time with O(1) space — two passes, three tracking variables
If right stays at −1 after the forward pass, the array is already sorted — return 0
Final length = right − left + 1 (inclusive bounds — the +1 is essential!)
