LeetCode #404

Sum of Left Leaves

Given the root of a binary tree, return the sum of all left leaves.

Easy Trees DFS Postorder Time: O(n) Space: O(h)
1 Understanding the Problem ▾

You are given the root of a binary tree. Your task is to find the sum of all left leaves. A leaf is a node with no children. A left leaf is a leaf that is the left child of its parent.

Walk through this concrete example by hand:

        3
       / \
      9  20
        /  \
       15    7

The leaves are: 9, 15, and 7. Which are left leaves?

  • Node 9: it's a leaf AND the left child of 3. ✓ Count it.
  • Node 15: it's a leaf AND the left child of 20. ✓ Count it.
  • Node 7: it's a leaf, but the right child of 20. ✗ Skip it.

Answer = 9 + 15 = 24.

💡 Key Insight

A node has no pointer to its own parent — it cannot know if it's a left or right child. Only the parent knows that. The trick is to carry a boolean flag is_left from parent to child: when we recurse into the left child, we pass True; when we recurse into the right child, we pass False. At a leaf node, we check this flag to decide whether to add its value to the sum.

Not a Left Leaf — Skip

Node 7: no children (is a leaf) but arrived via a right edge. Its is_left flag is False. We return 0.

Is a Left Leaf — Count!

Node 9: no children (is a leaf) AND arrived via a left edge. Its is_left flag is True. We return 9.

2 The Algorithm: DFS with Left-Child Flag Propagation ▾

We traverse the tree with a recursive DFS. At each call, we pass down a boolean that encodes "was this node reached via a left edge?" When a leaf sees that flag is True, it adds its value to the sum.

1 Entry point: Call dfs(root, is_left=False). Root has no parent, so it is neither left nor right — False is correct.
2 Base case — null: If node is None, return 0. No contribution.
3 Leaf check: If not node.left and not node.right (it's a leaf), return node.val if is_left else 0.
4 Recurse left: Call dfs(node.left, True) — left child always has is_left=True.
5 Recurse right: Call dfs(node.right, False) — right child always has is_left=False.
6 Combine: Return the sum of both recursive calls. The postorder accumulation bubbles the totals up to the root.

🔑 The "Track Left Child Flag" Sub-Pattern

This is a generalization of a common tree DFS technique: when a node needs information about how it was reached (its position relative to its parent), the parent passes that context as a parameter. Other examples: passing the current max value along a path, passing depth, passing accumulated sum. The node itself cannot look "up" — context must flow downward.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every recursive call dfs(node, is_left), the value of is_left is exactly correct: it is True if and only if node was the left child of its parent, and False otherwise (including the root, which has no parent).

Why does this invariant hold?

  • The flag is set at the call site, not by the node itself. The parent is the authoritative source of that information.
  • When node 3 calls dfs(node.left, True), the left child will always receive True regardless of what it is — correct by construction.
  • When node 3 calls dfs(node.right, False), the right child always receives False — also correct.

Why does this guarantee correctness?

  • Every node is visited exactly once (DFS visits all nodes).
  • A leaf adds its value iff is_left=True — which is True iff it is a left child.
  • Non-leaf nodes only accumulate and return child sums — they never add their own value.
  • The final sum is the correct total of all left-leaf values.
⚠️ Edge Case: Single-Node Tree

If the tree has only a root and no children, the root is a leaf but starts with is_left=False. It is NOT counted. Correct — a lone root is not anyone's left child.

⚠️ Edge Case: Right-Skewed Tree

If every node only has a right child, no leaf will ever be reached with is_left=True. The function returns 0 — correct, since there are no left leaves at all.

4 The Code (Annotated) ▾

Solution 1 — DFS with is_left flag (Interview Pick)

def sumOfLeftLeaves(root):
def dfs(node, is_left): ← inner helper; is_left tracks whether node is a left child
if not node: ← null node: contributes 0 to the sum
return 0
if not node.left and not node.right: ← leaf: no children
return node.val if is_left else 0 ← only count if arrived via left edge
return (dfs(node.left, True) ← left child is always a left child → True
+ dfs(node.right, False)) ← right child is never a left child → False
return dfs(root, False) ← root has no parent → is_left=False
⚠️ Common Mistake: Peeking at Children from the Parent

Many beginners write: if node.left and not node.left.left and not node.left.right: total += node.left.val. This works but means every node must peek at its grandchildren. The flag approach is cleaner: each node asks "was I a left child?" without any peeking.

Solution 2 — Iterative DFS with explicit stack (same complexity, avoids recursion)

def sumOfLeftLeaves(root):
if not root: return 0 ← empty tree guard
stack, total = [(root, False)], 0 ← stack of (node, is_left) pairs
while stack:
node, is_left = stack.pop() ← pull next (node, flag) from stack
if not node.left and not node.right: ← leaf?
if is_left: total += node.val ← count only if arrived via left edge
else:
if node.right: stack.append((node.right, False)) ← right child: flag=False
if node.left: stack.append((node.left, True)) ← left child: flag=True
return total

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize Context-Propagation in Trees

The signal is: "distinguish left leaves from right leaves." Since nodes don't know their own position, you need the parent to pass that context. Any time you read "left child," "right child," "depth," or "path from root" in a tree problem, think: pass a parameter down.

Step 2: Define the Recursive State

The state at each DFS call is: current node + boolean flag. Write the signature: dfs(node, is_left). This immediately frames the structure of the solution.

Step 3: Write Base Cases First

Null node → 0. Leaf + is_left=True → node.val. Leaf + is_left=False → 0. These three cases handle all terminal conditions. The recursive case is just sum of children — almost trivial once base cases are correct.

Step 4: Trace through the Example

dfs(3, False) → not leaf → dfs(9, True) + dfs(20, False). dfs(9, True) → leaf and is_left → return 9. dfs(20, False) → not leaf → dfs(15, True) + dfs(7, False). dfs(15, True) → 15. dfs(7, False) → 0. Total: 9 + 15 = 24. ✓

🗣️ What to Say to the Interviewer

"The challenge is that a node can't tell whether it's a left or right child. So I'll do DFS and pass an is_left boolean from parent to child. At each node: if null, return 0. If it's a leaf and is_left is True, return its value. Otherwise, recurse — left child gets is_left=True, right child gets is_left=False. Return the sum of both recursive calls. Time is O(n) — we visit every node once. Space is O(h) for the call stack — O(log n) balanced, O(n) worst case. I can also do it iteratively with an explicit stack of (node, is_left) pairs, same complexity."
💡 Likely Follow-Up Questions
  • Can you solve it iteratively? (Yes — stack of (node, is_left) pairs)
  • What about the sum of all right leaves? (Change check to not is_left)
  • What about the sum of all leaves (both sides)? (Remove the is_left condition)
  • Can you solve this without a helper function? (Yes — check if left child is a leaf directly from parent)

7 Summary & Key Takeaways ▾
A node cannot know its own side (left/right) — only the parent knows when making the recursive call.
Solution: pass a boolean is_left flag down the call stack from parent to child.
Pattern: Track Left Child Flag — a form of context propagation in tree DFS.
Time O(n) — every node visited once. Space O(h) — recursion depth equals tree height.
Same structure works iteratively: stack of (node, is_left) pairs eliminates call-stack risk for very deep trees.
🎯 When to Use This Pattern
  • Problem distinguishes left vs right children or subtrees
  • You need information about how a node was reached (side, depth, path)
  • The parent has context that the node itself cannot derive
  • Any time you think "I need to pass state top-down through the tree"
