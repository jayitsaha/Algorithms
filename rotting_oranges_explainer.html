LeetCode #994

Rotting Oranges

Given an m×n grid of fresh oranges (1), rotten oranges (2), and empty cells (0), return the minimum minutes until all fresh oranges rot — or −1 if impossible.

Medium Graph Multi-Source BFS Time: O(m·n) Space: O(m·n)
1 Understanding the Problem ▾

You have a grid. Each cell is either empty (0), has a fresh orange (1), or has a rotten orange (2). Every minute, every rotten orange simultaneously spreads rot to all 4-directionally adjacent fresh oranges. Find the minimum number of minutes until no fresh oranges remain — or −1 if it's impossible.

💡 Key Insight

This is a shortest-distance-from-multiple-sources problem on a grid. Each rotten orange is a "source." BFS naturally finds shortest distances — and because rot spreads simultaneously from all sources, we seed the BFS queue with every rotten orange at once. The answer is the BFS level at which the last fresh orange is first reached.

Edge cases to consider: No fresh oranges at all → return 0. All cells empty → return 0. Fresh oranges exist but no rotten ones → return −1. A fresh orange surrounded entirely by empty cells → −1 even if other fresh oranges do rot successfully.

2 The Algorithm: Multi-Source BFS ▾

Standard BFS finds the shortest path from one source. Here, multiple rotten oranges activate simultaneously at time 0. We solve this with a single BFS that starts from all rotten oranges at once — they are all "distance 0." BFS then expands outward level by level, each level representing one minute.

1 Scan the grid: Enqueue every rotten orange (value = 2). Count every fresh orange (value = 1). This single scan is the "multi-source" setup — all rotten oranges enter the queue simultaneously, making them all level 0.
2 Early exit: If fresh == 0, no fresh oranges exist → return 0 immediately.
3 BFS level by level: Each BFS level = 1 minute. Snapshot the current queue size. Process every cell at the current level; for each, check all 4 neighbors — if a neighbor is fresh (1), mark it rotten (2), decrement fresh, enqueue it for the next level. After the full level, increment minutes.
4 Return the answer: After BFS finishes, if fresh == 0 → return minutes. If fresh > 0 → some orange was unreachable → return −1.

🔑 Why BFS, Not DFS?

BFS processes cells level by level, so the first time a fresh orange is reached, it is reached via the shortest path from the nearest rotten orange. DFS would dive deep along one path before exploring another — it cannot simulate simultaneous, equidistant spreading. Use BFS whenever the question is "minimum steps" in an unweighted graph.

3 Why It Works: The Invariant ▾
🔐 The BFS Distance Invariant

At the start of BFS level t, every cell in the queue has the property: "its shortest distance from any initial rotten orange is exactly t." Processing level t produces all cells at distance t+1. This is the fundamental BFS correctness guarantee — applied simultaneously across multiple sources.

Why multi-source is correct: Imagine a virtual "super-source" node S with zero-cost edges to every initial rotten orange. Single-source BFS from S gives each cell its distance from the nearest rotten orange. Because the S→rotten cost is 0, distance(S, fresh) = distance(nearest-rotten, fresh). Multi-source BFS with all rotten oranges in the initial queue is exactly this.

Why marking visited immediately matters: We change a fresh orange to rotten as soon as we discover it (when enqueuing), not when we dequeue it. This prevents the same cell from being enqueued multiple times by different rotten neighbors — each cell is processed at most once, giving us O(m·n) total time.

⚠️ Common Trap: Off-by-One on Minutes

A typical level-by-level BFS loop increments minutes after each full level. But after the last real spreading level, the queue still holds the newly-rotted oranges from that level. The loop runs one more time (no new spreading occurs), then increments minutes again — an extra +1. Fix: return minutes - 1. Alternative: only increment if the queue was non-empty at the start of the inner loop.

Why -1 is correct when fresh > 0: BFS visits every cell reachable from any initial rotten orange. If a fresh orange is never visited, no path of adjacent non-empty cells connects it to any rotten orange. The grid topology itself makes it unreachable — no amount of waiting will change that.

4 The Code (Annotated) ▾

Solution 1 — Multi-Source BFS (Optimal, Interview Pick)

from collections import deque
def orangesRotting(grid): ← O(m·n) time, O(m·n) space
rows, cols = len(grid), len(grid[0])
queue = deque() ← BFS queue; holds (row, col) of rotten oranges
fresh = 0 ← countdown of remaining fresh oranges
for r in range(rows): ← scan every cell once — O(m·n)
for c in range(cols):
if grid[r][c] == 2: ← rotten → seed the BFS (multi-source!)
queue.append((r, c))
elif grid[r][c] == 1: ← fresh → count it
fresh += 1
if fresh == 0: return 0 ← no fresh oranges → already done
minutes = 0
dirs = [(0,1),(0,-1),(1,0),(-1,0)] ← 4 cardinal directions
while queue: ← process one BFS level per outer iteration
for _ in range(len(queue)): ← snapshot level size before expanding
r, c = queue.popleft()
for dr, dc in dirs:
nr, nc = r+dr, c+dc
if 0<=nr
grid[nr][nc] = 2 ← mark visited immediately (prevent re-enqueue)
fresh -= 1 ← one fewer fresh orange
queue.append((nr, nc)) ← enqueue for next BFS level
minutes += 1 ← finished one wave = one minute
return minutes - 1 if fresh == 0 else -1 ← -1 corrects the final empty-wave increment
⚠️ Why "minutes - 1" at the return?

The inner for loop processes the last wave of spreading. Afterward, queue contains the newly-rotted oranges from that wave. The while queue condition is still True, so the loop runs again. The inner loop processes those newly-rotted cells, finds no fresh neighbors, adds nothing new, and minutes is incremented one extra time. Subtracting 1 removes that spurious increment. An equivalent pattern: if queue: minutes += 1 only when the queue was non-empty before the inner loop.

Solution 2 — BFS with Time in Queue Tuple (alternate style)

def orangesRotting_v2(grid): ← same O(m·n) complexity; slightly different bookkeeping
rows, cols = len(grid), len(grid[0])
queue = deque(); fresh = 0; max_time = 0
for r in range(rows):
for c in range(cols):
if grid[r][c] == 2: queue.append((r, c, 0)) ← store time=0 for initial rotten
elif grid[r][c] == 1: fresh += 1
while queue:
r, c, t = queue.popleft() ← t = time this cell became rotten
for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
nr, nc = r+dr, c+dc
if 0<=nr
grid[nr][nc] = 2; fresh -= 1; max_time = t + 1
queue.append((nr, nc, t+1)) ← spread time = parent time + 1
return max_time if fresh == 0 else -1 ← max_time = latest time any fresh orange was reached
💡 Complexity Summary

Multi-Source BFS: Time O(m·n) — every cell enqueued and dequeued at most once; Space O(m·n) — queue can hold at most all cells.
Brute Force (simulate each minute): Time O((m·n)²) — rescans entire grid each minute; Space O(m·n) for the copy.
The BFS solution is optimal — you cannot do better than O(m·n) since you must at least read the input.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Pattern Recognition

Signals for BFS: "minimum time/steps," "spreading/propagating simultaneously," "grid with 4-directional neighbors," "can it reach all X?" The word simultaneously is the strongest signal — it means all sources activate at once, which is exactly multi-source BFS.

Step 2: State the Brute Force

"I could simulate minute by minute: scan the whole grid, find all rotten oranges, spread to fresh neighbors, repeat. Each simulation step is O(m·n), and there can be up to O(m·n) steps → O((m·n)²) total. Too slow for large grids."

Step 3: Identify the BFS Optimization

"BFS naturally processes cells in levels — each level is one minute. Instead of rescanning the full grid each minute, BFS only visits cells at the current frontier. We start with ALL rotten oranges in the queue simultaneously — multi-source BFS — so they expand in parallel."

Step 4: Handle the −1 Case Explicitly

"After BFS completes, I check if any fresh orange remains. If so, it was never reachable from any rotten orange — return −1. Otherwise return the number of BFS levels."

🗣️ What to Say to the Interviewer

"This is a multi-source BFS problem — I'll seed the queue with all initially rotten oranges simultaneously.
I also count fresh oranges. If fresh == 0, return 0 immediately.
Then run BFS level by level — each level = 1 minute.
For each rotten cell, spread to fresh 4-directional neighbors, mark them rotten, decrement fresh, enqueue.
After BFS: if fresh == 0, return minutes. Else return -1 (unreachable oranges).
Time: O(m·n). Space: O(m·n)."
🔮 Likely Follow-up Questions
  • "What if spread is 8-directional?" → Add 4 diagonal pairs to the dirs list
  • "What if different rotten oranges spread at different speeds?" → Use Dijkstra's instead (weighted BFS with a min-heap)
  • "Can you avoid modifying the grid?" → Use a visited set initialized to all rotten positions
  • "What if fresh oranges can spontaneously become rotten after k minutes?" → Add all fresh oranges at time k to the queue during setup

7 Summary & Key Takeaways ▾
Multi-source BFS = seed the queue with ALL sources at level 0, then run standard BFS
BFS finds shortest distances — "minimum minutes" is exactly the distance from the nearest rotten orange
Mark cells rotten immediately when enqueuing (not when dequeuing) to avoid duplicate processing
Count fresh oranges upfront; after BFS, if fresh > 0 → unreachable → −1
Off-by-one fix: the last BFS iteration adds no new cells, so return minutes − 1
