LeetCode #269

Alien Dictionary

Given a sorted list of words in an alien language, determine the character ordering of the alien alphabet. Return any valid ordering, or an empty string if the ordering is invalid (i.e., contains a cycle).

Hard Graph Topological Sort Time: O(C) Space: O(U)
1 Understanding the Problem ▾

Think of this like cracking a code. You are given a dictionary — a list of words sorted in some alien alphabetical order you don't know. Your task is to figure out what that ordering must be, based purely on the sorted word list.

💡 Key Insight

When two adjacent words in the sorted list differ for the first time at position i, the character in the earlier word must come before the character in the later word in the alien alphabet. That single pair gives you one directed ordering constraint — one edge in a graph. Collect all such edges, then topologically sort the graph to get the alphabet order.

Concrete example: words = ["wrt","wrf","er","ett","rftt"]

✓ Edges Derived

"wrt" vs "wrf" → t before f
"wrf" vs "er" → w before e
"er" vs "ett" → r before t
"ett" vs "rftt"→ e before r
Result: "wertf"

✗ Invalid: Prefix Conflict

words = ["abc", "ab"]
"ab" is a prefix of "abc"
But "abc" appears first!
Impossible → return ""

Edge cases to consider:

  • Cycle in ordering constraints (e.g., a→b and b→a) → return ""
  • Prefix conflict (longer word before its shorter prefix) → return ""
  • Single word → return its unique characters in any order
  • Multiple valid orderings → return any one
  • All identical adjacent words → no constraint from that pair
2 The Algorithm: Build Graph + Kahn's BFS Topological Sort ▾

This problem splits cleanly into two phases: build the constraint graph, then topologically sort it. The graph has characters as nodes and "must-come-before" relationships as directed edges.

1 Initialize all characters — scan every word and every character. Add each character to the graph with an empty adjacency set and indegree 0. This ensures even characters with no constraints appear in the output.
2 Extract ordering edges — for each adjacent word pair, zip their characters and find the first position where they differ. That gives one edge: c1 → c2. Stop after the first diff — further positions give no ordering information. Check for prefix conflict before processing.
3 Seed the BFS queue — add all characters with indegree 0 to the queue. These have no prerequisites and can appear first in the alien alphabet.
4 BFS (Kahn's Algorithm) — repeatedly dequeue a character, append it to the result, and for each of its neighbors, decrement their indegree. If a neighbor's indegree drops to 0, enqueue it.
5 Cycle check — if len(result) != len(all unique chars), some characters were never dequeued (their indegree never reached 0 due to a cycle). Return "".
🔑 Kahn's vs DFS Topological Sort

Both produce valid topological orderings. Kahn's BFS is preferred here: cycle detection is trivial (compare output length to unique char count), and the implementation is iterative — no recursion stack overflow risk. DFS topo sort requires a three-color visited state (unvisited / in-stack / done) which is slightly harder to implement correctly under interview pressure.

⚠️ Only the FIRST Differing Position Matters

When comparing "wrt" and "wrf", the fact that positions 0 and 1 match (w=w, r=r) means nothing. Only position 2 (t vs f) tells us something: t comes before f. Positions after the first mismatch are irrelevant — the ordering is already decided. Always break after the first diff.

3 Why It Works: The Invariant ▾
🔐 The Invariant (Kahn's Algorithm)

At every step of the BFS, every character in the queue has indegree 0 — meaning all of its prerequisite characters are already in the result. Appending it to the output never violates any ordering constraint. This invariant is maintained after every dequeue: we only enqueue neighbors whose indegree just dropped to 0.

Why edges are correct: In a sorted dictionary, if word A comes before word B and they first differ at position i, then A[i] must precede B[i] in the alien alphabet — by definition of the sort. This is the only information that specific adjacent pair gives. One pair → one edge → one ordering constraint.

Why cycle = invalid: A directed cycle means "X must come before Y AND Y must come before X" — a logical contradiction. No linear ordering satisfies this. In Kahn's, cyclic nodes can never reach indegree 0 (they're perpetually waiting on each other), so they never enter the queue and never appear in the output. The length check catches this.

Why deduplicate edges? Multiple word pairs may yield the same constraint (e.g., both pair 1 and pair 3 say t→f). Without deduplication, you'd increment indegree[f] twice for one logical edge — it would need two decrements to reach 0, but only one ever happens. Result: f gets stuck and is falsely flagged as a cycle. Use a set for adjacency.

⚠️ The Prefix Edge Case

If words = ["abc", "ab"], "ab" is a prefix of "abc" but appears later. In any valid sorted order, a prefix always comes before its extensions (just as "app" before "apple"). So this ordering is impossible — return "" immediately without processing further.

4 The Code (Annotated) ▾

Solution 1 — BFS Topological Sort / Kahn's Algorithm (Interview Pick)

from collections import defaultdict, deque
def alienOrder(words): ← takes list of sorted alien words
adj = {c: set() for w in words for c in w} ← every char seen → empty neighbor set
indegree = {c: 0 for c in adj} ← all chars start with 0 prerequisites
for i in range(len(words) - 1): ← compare each consecutive pair
w1, w2 = words[i], words[i + 1]
min_len = min(len(w1), len(w2))
if len(w1) > len(w2) and w1[:min_len] == w2: ← prefix conflict check
return "" ← "abc" before "ab" is impossible
for c1, c2 in zip(w1, w2): ← scan both words in lockstep
if c1 != c2: ← first mismatch = constraint
if c2 not in adj[c1]: ← dedup: don't add same edge twice
adj[c1].add(c2) ← c1 → c2: c1 must come before c2
indegree[c2] += 1 ← c2 has one more prerequisite
break ← STOP — only first diff matters
queue = deque([c for c in indegree if indegree[c] == 0]) ← chars with no prereqs
result = []
while queue: ← BFS: process frontier
c = queue.popleft() ← safe to place: all prereqs satisfied
result.append(c) ← add to alien alphabet ordering
for neighbor in adj[c]: ← unlock chars that depended on c
indegree[neighbor] -= 1 ← one fewer prerequisite
if indegree[neighbor] == 0: ← all prereqs met?
queue.append(neighbor) ← yes: ready to place
if len(result) != len(adj): ← output shorter than total chars?
return "" ← cycle: some chars stuck, never freed
return "".join(result) ← valid alien alphabet ordering
⚠️ Why use set() for adjacency, not list()?

If two different word pairs both yield the same edge (e.g., both "wr*" pairs give t→f), using a list would add the edge twice and double-increment indegree[f]. Then when we process t and decrement f's indegree once, f would still have indegree 1 — stuck, falsely treated as part of a cycle. A set automatically deduplicates, ensuring each edge is counted exactly once.

Solution 2 — DFS Topological Sort (Alternative)

def alienOrder_dfs(words):
adj = {c: set() for w in words for c in w}
# ... (same edge-building logic) ...
visited = {} # False = done, True = in current DFS path (cycle check)
result = []
def dfs(c):
if c in visited: return visited[c] ← True = cycle found, False = already done
visited[c] = True ← mark as on current path
for nb in adj[c]:
if dfs(nb): return True ← propagate cycle detection
visited[c] = False ← done: mark so others can skip
result.append(c) ← post-order = reverse topological order
for c in adj:
if c not in visited and dfs(c): return ""
return "".join(reversed(result)) ← reverse post-order = correct topo order
💡 Complexity

Let C = total characters across all words, U = unique characters.
BFS (Kahn's): Time O(C), Space O(U) — edge count ≤ U, each char processed once.
DFS: Same O(C) time, O(U) space — but needs extra call stack depth O(U).
Both are optimal — you cannot avoid reading the entire input.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize "Ordering from Comparisons" = Topological Sort

The moment you see "derive a ranking or ordering from pairwise comparisons," think directed graph + topological sort. Here, adjacent word pairs are your pairwise comparisons, characters are nodes, and ordering constraints are edges.

Step 2: State the Graph Construction Clearly

"I'll compare adjacent word pairs. For each pair, I zip characters and take only the first position where they differ — that gives me one directed edge. I collect all characters seen in all words as graph nodes, so isolated characters (no constraints) still appear in output."

Step 3: Choose Kahn's BFS and Explain Why

"I'll use Kahn's BFS topological sort. It's iterative, clean, and cycle detection is trivial: if the output length is less than the number of unique characters, a cycle exists. DFS also works but needs three-color visited state."

Step 4: Name Edge Cases Before Coding

Say proactively: "I'll handle the prefix conflict case — if a longer word appears before its shorter prefix, that's immediately invalid. And I'll deduplicate edges using sets to avoid double-counting indegrees."

🗣️ What to Say to the Interviewer

"This is a graph problem. I need to extract a topological ordering of characters from the constraint edges I can read off from adjacent sorted words.
Step 1: Initialize every character as a graph node with indegree 0.
Step 2: For each adjacent pair, find the first differing character — that gives one directed edge c1→c2.
Edge case: if w2 is a prefix of w1, return empty string immediately.
Step 3: Run Kahn's BFS — seed with zero-indegree nodes, process greedily, decrement neighbors' indegrees.
Step 4: If output length equals unique char count, return it. Otherwise a cycle exists, return empty string.
Time O(C), Space O(U) where C = total chars, U = unique chars."
🔮 Likely Follow-up Questions
  • "Multiple valid orderings?" → Return any. Use a min-heap instead of deque to get lexicographically smallest.
  • "Can you use DFS?" → Yes. Post-order DFS + reverse gives topo order. Cycle = node re-visited while on current stack.
  • "How does this relate to Course Schedule?" → Identical structure. Both are topo sort + cycle detection. The only difference is we must first extract the graph from the word list here.
  • "What if characters repeat across many words?" → Same logic. The deduplication with sets handles this — each edge is added at most once regardless of how many word pairs imply it.

7 Summary & Key Takeaways ▾
Sorted alien words → directed ordering edges: only the FIRST differing character between adjacent words tells you anything
Build directed graph (char→char edges), then run Kahn's BFS topological sort
Use sets for adjacency to prevent duplicate edges from skewing indegrees
Cycle detection is trivially len(result) != len(unique chars) in Kahn's
Prefix conflict (longer word before shorter prefix) = immediate return ""
Time O(C) where C = total chars across all words; Space O(U) unique chars
