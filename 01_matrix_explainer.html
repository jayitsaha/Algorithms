LeetCode #542

01 Matrix

Given an m×n binary matrix, return a new matrix where each cell contains the distance to its nearest 0.

Medium Graph Multi-source BFS Time: O(m·n) Space: O(m·n)
1 Understanding the Problem ▾

You're given an m×n binary matrix filled with 0s and 1s. Your task: return a new matrix of the same shape where every cell holds the Manhattan distance (number of up/down/left/right steps) to the nearest 0 in the original.

💡 Key Insight

Cells that are 0 trivially have distance 0 to themselves. Every 1-cell needs the shortest path to any zero — not one specific zero. This is a multi-source shortest-path problem on an unweighted grid, and BFS is the exact tool for it.

Input matrix (0s and 1s)

0  0  0
0  1  0
1  1  1

Output: distance to nearest 0

0  0  0
0  1  0
1  2  1

The center cell (1,1) is 1 step from (0,1) above. The bottom-center (2,1) is 2 steps away — it must go through (1,1) and then to a 0. Edge cases: entire matrix is 0s (output = all 0s), a single cell with 0, large grids where interior 1-cells have large distances, guaranteed at least one 0 in input.

2 The Algorithm: Multi-Source BFS from All 0s ▾

The key move is reversing the search direction. Instead of "from each 1, find the nearest 0" (slow — up to O(mn) per cell), we ask "from all 0s at once, how far can BFS waves reach?" BFS guarantees the first visit to any cell uses the shortest path, because it expands level by level.

1 Initialize: Create a dist matrix filled with infinity (∞) for 1-cells, and 0 for every 0-cell. Enqueue all 0-cells into the BFS queue simultaneously — they are all sources at distance 0.
2 BFS expansion: Dequeue a cell. For each of its 4 neighbors, check if the neighbor's current distance is greater than dist[cell] + 1. If so, update and enqueue the neighbor. This condition doubles as the "visited" check.
3 Wave propagation: BFS naturally expands in rings — all distance-1 neighbors of 0s get processed before distance-2 neighbors, etc. The first time we reach a cell is always via the shortest path from any 0.
4 Termination: When the queue empties, every cell has been settled with its true minimum distance. Return dist.

🔑 Why Not BFS from Each 1-Cell Individually?

If you BFS from each 1-cell separately, you touch all m·n cells for each of up to m·n ones → O(m²n²). Multi-source BFS visits each cell exactly once across the entire algorithm → O(m·n). The trick: enqueueing all 0-cells simultaneously is equivalent to adding a "virtual super-source" with zero-weight edges to every 0-cell, then running single-source BFS from it.

3 Why It Works: The Invariant ▾
🔐 The BFS Invariant

When a cell is dequeued from the BFS queue, its distance value is final and optimal. It will never be updated again. This holds because BFS processes cells in non-decreasing order of distance — the first time we reach any cell is always via the shortest path.

Why BFS gives shortest paths on unweighted grids: Each move between adjacent cells costs exactly 1. BFS explores all paths of length 1 before any path of length 2. So the first arrival at any cell is provably shortest. This is Dijkstra's algorithm with all edge weights = 1, simplified to a plain queue.

Why multi-source is equivalent to single-source with a super-node: Imagine a virtual node S connected to every 0-cell with weight-0 edges. BFS from S gives dist[S → cell] = dist[nearest_0 → cell] for every cell, because the zero-weight edges make all 0-cells equidistant from S. Our algorithm skips S and seeds all 0-cells directly — mathematically identical.

⚠️ Common Mistake: Separate "visited" Set

Many beginners maintain a visited = set() and check if (nr, nc) in visited. With the dist[nr][nc] > dist[r][c]+1 update condition, this is redundant. A cell with a finite distance ≤ the proposed new distance already has its best path — the comparison rejects it. Saves memory and a hash lookup per step.

Guaranteed coverage: The problem guarantees at least one 0 exists and the grid is 4-connected (you can reach any cell from any other by steps). So BFS from the 0-cells reaches every 1-cell. No cell stays at infinity in the output.

4 The Code (Annotated) ▾

Solution 1 — Multi-Source BFS (Optimal, Interview Pick) ✓

from collections import deque ← O(1) append + popleft (list.pop(0) is O(n))
def updateMatrix(mat):
m, n = len(mat), len(mat[0]) ← grid dimensions
dist = [[float('inf')]*n for _ in range(m)] ← all 1-cells start at ∞; overwrite 0-cells below
q = deque()
for r in range(m): ← seed phase: find all 0-cells
for c in range(n):
if mat[r][c] == 0: ← this is a source cell
dist[r][c] = 0 ← distance from a 0 to itself = 0
q.append((r, c)) ← enqueue ALL sources at once
dirs = [(0,1),(0,-1),(1,0),(-1,0)] ← 4-directional movement
while q: ← BFS until queue is empty
r, c = q.popleft() ← FIFO: processes closer cells first
for dr, dc in dirs:
nr, nc = r+dr, c+dc ← neighbor coordinates
if 0 <= nr < m and 0 <= nc < n: ← bounds check
if dist[nr][nc] > dist[r][c]+1: ← shorter path found (also acts as visited check)
dist[nr][nc] = dist[r][c]+1 ← update distance
q.append((nr, nc)) ← enqueue to propagate further
return dist ← every cell has its true min distance to nearest 0
⚠️ Why float('inf') and not -1 for unvisited?

Using -1 as "unvisited" requires a separate check: if dist[nr][nc] == -1. Using float('inf') allows the single condition dist[nr][nc] > dist[r][c]+1 to handle both "never visited" (∞ > anything) and "already has a worse distance" in one check. Cleaner and correct.

Solution 2 — Dynamic Programming Two-Pass (O(1) extra space)

def updateMatrix_dp(mat):
m, n = len(mat), len(mat[0])
INF = m + n ← max distance in an m×n grid (can't exceed m+n steps)
dist = [[0 if mat[r][c]==0 else INF ← 0-cells = 0, 1-cells = INF
for c in range(n)] for r in range(m)]
for r in range(m): ← Pass 1: top→bottom, left→right
for c in range(n): ← relax from top and left neighbors
if r > 0: dist[r][c] = min(dist[r][c], dist[r-1][c]+1)
if c > 0: dist[r][c] = min(dist[r][c], dist[r][c-1]+1)
for r in range(m-1,-1,-1): ← Pass 2: bottom→top, right→left
for c in range(n-1,-1,-1): ← relax from bottom and right neighbors
if r<m-1: dist[r][c]=min(dist[r][c], dist[r+1][c]+1)
if c<n-1: dist[r][c]=min(dist[r][c], dist[r][c+1]+1)
return dist ← two passes cover all 4 directions combined
💡 Complexity Summary

Multi-Source BFS: Time O(m·n), Space O(m·n) — queue holds up to m·n cells
DP Two-Pass: Time O(m·n), Space O(1) extra — modifies dist in-place
Naive (BFS per 1-cell): Time O(m²n²) — TLE on large grids, never use in interviews

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

"Shortest distance from each cell to the nearest X on a grid" → this is BFS territory. The signals: binary grid, 4-directional movement, distance queries. Every move costs 1 step, so BFS (not Dijkstra) is sufficient and optimal.

Step 2: Propose the Naive Approach

"I could BFS from every 1-cell independently to find its nearest 0. That's correct but O(m²n²) — too slow for large grids. Can I share computation across cells?"

Step 3: The "Aha!" — Reverse the Direction

"Instead of searching FROM 1s TO 0s, I'll push FROM all 0s simultaneously. BFS naturally expands outward, and the first time it reaches any 1-cell gives that cell's minimum distance. One BFS pass covers the whole grid."

Step 4: Code the Seeding Step Clearly

"Scan once to seed: set dist=0 for all 0-cells, enqueue them. Init dist=∞ for 1-cells. Then standard BFS: dequeue, check 4 neighbors, update if dist[nbr] > dist[curr]+1, enqueue if updated."

🗣️ What to Say to the Interviewer

"This is a shortest-distance problem on an unweighted grid, so BFS is the right tool.
The naive approach — BFS from each 1-cell — is O(m²n²) and too slow.
The key insight: instead of 'from 1s, find nearest 0', I flip it: 'from all 0s, spread outward'.
Multi-source BFS: seed all 0-cells at distance 0, enqueue them simultaneously.
BFS expands level by level — first arrival at any cell is guaranteed optimal.
Total work: O(m·n) time (each cell enqueued/dequeued at most once), O(m·n) space for the queue."
🔮 Likely Follow-up Questions
  • "Can you do O(1) extra space?" → DP two-pass (shown in Section 4)
  • "What if diagonal moves are allowed?" → Expand dirs to all 8 directions
  • "What if some cells are walls (blocked)?" → Skip blocked cells during BFS expansion
  • "What if we need distance to nearest 1 instead?" → Swap which cells seed the queue
  • "What about weighted grids (not uniform cost)?" → Use Dijkstra with a priority queue

7 Summary & Key Takeaways ▾
Multi-source BFS: seed ALL 0-cells at once — one BFS pass computes every distance in O(m·n)
Reversing search direction (start from 0s, not 1s) is the critical optimization insight
BFS on unweighted grids always gives shortest paths — level-by-level ordering is the proof
Update condition dist[nr][nc] > dist[r][c]+1 replaces a separate visited set
DP two-pass is an equally correct O(mn) alternative needing only O(1) extra space

🎯 When to Recognize Multi-Source BFS

  • "Infection/fire/rot spreads from multiple starting points simultaneously" → multi-source BFS
  • Time complexity should be O(m·n) — one BFS pass across the whole grid
