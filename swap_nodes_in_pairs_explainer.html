LeetCode #24

Swap Nodes in Pairs

Given a linked list, swap every two adjacent nodes and return its head. You must not modify node values — only node pointers.

Medium Linked List Iterative Pair Swap Time: O(n) Space: O(1)
1 Understanding the Problem ▼

You are given a singly linked list. The goal is to swap every pair of adjacent nodes — the 1st with the 2nd, the 3rd with the 4th, and so on — and return the new head. If the list has an odd number of nodes, the last unpaired node stays in place. The critical constraint: you may only rewire pointers; changing node values is explicitly forbidden.

Before swapping
1 → 2 → 3 → 4 → NULL
After swapping
2 → 1 → 4 → 3 → NULL

Walk through it by hand: take nodes 1 and 2. After swapping, node 2 leads, pointing back to node 1. Node 1 must then point to the result of swapping the remaining list — specifically node 4, the new head of the second pair. This "stitching across pairs" is the subtlety that catches people off guard.

💡 Key Insight

A dummy head node placed before the real list lets you treat every pair — including the very first one — with identical logic. Without it, you need a special branch for the head pair. With it, every pair has a well-defined prev node, and the three rewires are always the same.

🔑 Why Pointer-Only?

In production, nodes can carry large payloads — entire structs, strings, or references. Swapping values would require deep-copying all that data. Rewiring pointers is O(1) regardless of payload size. This constraint forces you to think in pointer graph terms, which is the real skill being tested.

2 The Algorithm: Iterative Pair Swap with Dummy Head ▼

Maintain four named pointers at each iteration: prev (node before the pair), a (first of pair), b (second of pair), and nxt (everything after the pair). Rewire in exactly three steps, then advance prev by two positions.

0 Create dummy. Allocate dummy = ListNode(0, head). Set prev = dummy. The dummy node acts as the sentinel predecessor for the first pair.
1 Check pair exists. While prev.next and prev.next.next are both non-null, there is a pair. Set a = prev.next, b = a.next, nxt = b.next (save tail before rewiring!).
2 Rewire three pointers. ① prev.next = b — splice b before a in the chain. ② b.next = a — b now points back to a, completing the swap. ③ a.next = nxt — a's tail reconnects to the rest of the list.
3 Advance. Set prev = a. After swapping, a is the second (trailing) node of the pair and sits right before the next pair. This keeps the invariant intact.
4 Return. return dummy.next. Because the first pair may have a new head (b), we return through dummy rather than the original head variable.

⚠️ Order Matters — Always Save nxt First

If you write b.next = a before saving nxt = b.next, you permanently lose the only pointer into the rest of the list. Snapshot nxt as your very first action inside the loop, before touching any .next pointers.

3 Why It Works: The Invariant ▼
🔐 Loop Invariant

At the start of every loop iteration, prev points to the last correctly-placed node in the already-swapped prefix. All nodes at or after prev.next are unprocessed and still in their original relative order.

Why does this guarantee a correct result?

  • Dummy satisfies the invariant trivially before the first pair: no nodes have been processed, and prev.next = head, so prev sits before all unprocessed nodes.
  • Each iteration preserves the invariant: after swapping pair (a, b) into (b, a) and stitching a.next = nxt, node a is the rightmost placed node. Setting prev = a restores the invariant.
  • Termination: every iteration advances prev by exactly two positions. The list is finite, so the loop terminates in O(n/2) iterations.
  • Odd-length lists: the guard prev.next and prev.next.next ensures we only swap when a full pair exists. A lone trailing node is left in place — exactly what the problem requires.

Edge cases: empty list → dummy.next = NULL, returned immediately. Single node → loop guard fails at once, node returned as-is. Two nodes → one pair swap, done correctly.

4 The Code (Annotated) ▼

Solution 1 — Iterative with Dummy Head Interview Pick ✓

def swapPairs(head: ListNode) -> ListNode:
dummy = ListNode(0, head)// sentinel before list — removes need for head special-case
prev = dummy// prev always trails the current pair
while prev.next and prev.next.next:// need two nodes to form a pair
a = prev.next// first node of pair
b = a.next// second node of pair
nxt = b.next// save rest of list BEFORE any rewire
prev.next = b// ① splice b into lead position
b.next = a// ② b points back to a (pair reversed)
a.next = nxt// ③ a's tail reconnects to remaining list
prev = a// advance: a is now the rightmost placed node
return dummy.next// new head (may differ from original head)
⚠️ Common Mistake: Forgetting to Save nxt

Writing b.next = a before nxt = b.next overwrites the only pointer to the rest of the list. The tail is permanently lost. Save nxt as step zero, every time.

Solution 2 — Recursive

def swapPairs(head: ListNode) -> ListNode:
if not head or not head.next:// base: 0 or 1 node — nothing to swap
return head
a, b = head, head.next// current pair
a.next = swapPairs(b.next)// a tails into result of swapping remaining list
b.next = a// b leads the pair
return b// b is new head of this sub-list
📐 Complexity Summary

Iterative: Time O(n) — each node visited once. Space O(1) — only a handful of pointer variables.
Recursive: Time O(n). Space O(n/2) — one stack frame per pair. Can overflow on lists with millions of nodes.
In interviews: always lead with the iterative solution.

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Recognise the Pattern

Signals in problem: linked list + local structural rearrangement + "no value modification." That combination screams pointer rewiring. Pair-wise operation → each iteration handles exactly two nodes → iterative with a trailing prev pointer is the go-to structure.

Step 2: Draw It Before Coding

On the whiteboard, draw four boxes: prev, a, b, and the rest. Draw arrows. Physically trace each of the three rewires. Counting arrows before and after confirms correctness and prevents the "lost pointer" bug.

Step 3: Propose the Dummy Head Proactively

Say: "I'll use a dummy head so the first pair is handled identically to all others — no special case." This signals pattern fluency. Interviewers notice when you reach for dummy head without being prompted.

Step 4: Narrate the Three Rewires

Name each: "First, prev points to b. Second, b points to a. Third, a points to nxt." Narrating keeps you from misordering them under pressure. Save nxt before all three.

Step 5: Trace Example + Mention Edge Cases

Trace [1→2→3→4] step by step. Then mention without being asked: empty list, single node, two nodes, odd length. All handled by the same code — no extra branches. This demonstrates thoroughness.

🗣️ What to Say to the Interviewer

"I see we need to swap adjacent pairs without modifying values, so this is a pointer rewiring problem on a linked list. I'll add a dummy head so the first pair is handled the same as all others — that keeps the logic clean and avoids special-casing the head. At each step I'll track: prev (the node before the pair), a (first of pair), b (second of pair), and nxt (everything after). I'll save nxt before any rewiring — that's the key to not losing the tail. Three rewires per pair: prev→b, then b→a, then a→nxt. Advance prev to a. Total: O(n) time, O(1) space. Want me to also sketch the recursive version?"
💡 Likely Follow-up Questions
  • Reverse Nodes in k-Group (LC 25) — generalise pair swap to groups of k
  • Can you do it without a dummy head? — Yes, but you need an if-branch for the first pair. Messier and error-prone.
  • What if the list is doubly-linked? — Update both .next and .prev pointers per node.
  • Can you do this in-place with no extra pointer variables? — No; you need at least prev and the saved nxt to avoid losing nodes.

7 Summary & Key Takeaways ▼
A dummy head eliminates special-casing the first pair — every pair uses the same three-rewire logic
Save nxt = b.next before any rewiring — this is the single most common bug in this problem
Three pointer rewires in order: prev→b, b→a, a→nxt — memorise this sequence
Advance prev = a after each swap — a is the rightmost placed node and the predecessor of the next pair
Loop guard checks both prev.next AND prev.next.next — handles odd-length lists transparently
Iterative: O(1) space (use this in interviews). Recursive: elegant but O(n/2) call-stack depth.
🎯 When to Use This Pattern
  • Linked list structural rearrangement (not value swaps)
  • Group-wise operations: swap pairs, reverse k-groups, rotate segments
  • Any linked list problem where "prev" and "saved tail" are necessary
