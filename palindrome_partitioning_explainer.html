LeetCode #131

Palindrome Partitioning

Given a string s, partition s such that every substring of the partition is a palindrome. Return all possible palindrome partitions.

Medium Backtracking Partition + Check Palindrome Time: O(n · 2ⁿ) Space: O(n)
1 Understanding the Problem ▾

We're given a string like "aab" and need to slice it into pieces where every piece is a palindrome. A palindrome reads identically forwards and backwards — "a", "aa", "aba", "racecar" are all palindromes.

The critical word is every possible partition. We don't want just one valid cut — we need all of them. For "aab":

✓ Valid Partitions of "aab"

["a", "a", "b"]
["aa", "b"]

✗ Invalid Partitions

["aab"] ← "aab" is not a palindrome
["a","ab"] ← "ab" is not a palindrome

💡 Key Insight

Think of it as placing cut-points between characters. For a string of length n, there are n−1 potential cut points. Each is either used or skipped — giving up to 2^(n−1) possible ways to cut. We output every way where each resulting piece is a palindrome.

Edge Cases: Single character (trivially valid — every char is a palindrome). All-same characters like "aaa" — every possible split is valid: ["a","a","a"], ["aa","a"], ["a","aa"], ["aaa"]. Empty string returns [[]]. Long strings produce exponential output — the constraint is usually small (≤16).

2 The Algorithm: Backtracking with Palindrome Pruning ▾

This is a classic generate-all-solutions backtracking problem. We try every possible prefix starting from our current position. If that prefix is a palindrome, we accept it, advance our position, and recurse. When we reach the end of the string, our current collection of pieces is one complete valid partition — record it.

1 Start at index 0 with an empty path. This is the "root" of our decision tree — no pieces chosen yet.
2 Try every prefix ending at index end (from start to n-1). The candidate piece is s[start:end+1].
3 Palindrome check (pruning): If the candidate is NOT a palindrome, skip it — don't recurse. Only palindromes can be pieces.
4 Choose and Recurse: Append the palindrome piece to path, then call backtrack(end+1) to handle the remaining suffix.
5 Unchoose (Backtrack): After returning from recursion, path.pop() removes the piece. Now try the next longer prefix from the same start.
6 Base case — record: When start == len(s), we've covered the entire string with palindrome pieces. result.append(path[:]) records this partition.

🔑 Palindrome Check Options

Simple (interview): Two-pointer — expand from both ends inward, compare characters. O(k) per check, O(1) space.
Optimized: Precompute pal[i][j] = True/False using DP in O(n²) time/space. Then each check is O(1). Mention this as a follow-up optimization.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point during backtracking, every string in path is a confirmed palindrome, and together they cover s[0:start] exactly — contiguous, no overlaps, no gaps. Recursion only adds palindromes; non-palindromes are pruned before entering path.

Why base case guarantees a valid partition: The base case fires when start == len(s), meaning the palindrome pieces in path cover the entire string. Each call to backtrack(end+1) advances start by exactly the length of the chosen piece — so there are no gaps and no re-covering.

Why all solutions are found: The loop for end in range(start, n) tries every possible end position for the next piece. Combined with full recursion at each step, the algorithm exhaustively explores the entire decision tree. Every branching choice is considered, so no valid partition is ever missed.

⚠️ The path.pop() Trap — Forgetting to Backtrack

The most common bug is forgetting path.pop() after the recursive call. Without it, pieces from one branch "contaminate" later branches — the path keeps growing with stale pieces. Always pair path.append(piece) with path.pop() immediately after the recursive call. This append/pop sandwich is the mechanical heart of backtracking.

⚠️ Copy the Path — Not a Reference

result.append(path[:]) — NOT result.append(path). Python lists are objects; storing path would store a reference. After backtracking completes, path ends up empty — and every "stored" partition would silently be empty too. The slice path[:] creates a snapshot of the current state.

4 The Code (Annotated) ▾

Solution 1 — Backtracking + Two-Pointer Palindrome Check (Interview Pick)

def partition(s: str) -> List[List[str]]:
result, path = [], [] ← result: all valid partitions; path: current partition being built
n = len(s)
def is_palindrome(l, r): ← Check s[l..r] with two pointers: O(r-l+1) time, O(1) space
while l < r: ← Converge from both ends toward center
if s[l] != s[r]: return False ← Mismatch → not a palindrome; fail early
l += 1; r -= 1 ← Both pointers move inward
return True ← All chars matched → palindrome confirmed
def backtrack(start): ← start: index of the first char of the NEXT piece to place
if start == n: ← Base case: used entire string → valid partition found!
result.append(path[:]) ← Snapshot of path; DO NOT write result.append(path)
return
for end in range(start, n): ← Try every end index: piece is s[start..end] (inclusive)
if is_palindrome(start, end): ← Pruning: only recurse on palindrome pieces
path.append(s[start:end+1]) ← Choose: add this palindrome piece to current partition
backtrack(end + 1) ← Explore: recurse for the remaining suffix
path.pop() ← Unchoose: remove piece to try the next longer one
backtrack(0) ← Kick off from the very start of the string
return result
⚠️ path.pop() must come right after the recursive call

The loop tries end = start, start+1, start+2, .... For each palindrome found, we: append → recurse → pop. If pop were missing, the second iteration's path.append would be on top of the previous piece, corrupting the path for all future branches.

Solution 2 — Backtracking + DP Precomputed Palindrome Table

def partition(s: str) -> List[List[str]]:
n = len(s)
# Build pal[i][j]: True iff s[i..j] is a palindrome
pal = [[False]*n for _ in range(n)] ← O(n²) space; pal[i][j] answers in O(1) later
for i in range(n): pal[i][i] = True ← All single chars are palindromes (base case)
for length in range(2, n+1): ← Fill by increasing substring length (bottom-up)
for i in range(n - length + 1): ← All starting positions
j = i + length - 1
if length == 2:
pal[i][j] = (s[i] == s[j]) ← Two chars: palindrome iff they match
else:
pal[i][j] = (s[i]==s[j] and pal[i+1][j-1]) ← Outer chars match AND inner substring is palindrome
result, path = [], []
def backtrack(start):
if start == n:
result.append(path[:]); return
for end in range(start, n):
if pal[start][end]: ← O(1) lookup instead of O(k) scan — the payoff!
path.append(s[start:end+1])
backtrack(end + 1)
path.pop()
backtrack(0)
return result
💡 Complexity Summary

Solution 1: Time O(n · 2ⁿ) — at most 2^(n−1) partitions, O(n) for each palindrome check/copy. Space O(n) recursion depth + path.
Solution 2: Time O(n²) precompute + O(n · 2ⁿ) backtracking = O(n · 2ⁿ) total. Extra Space O(n²) for pal table.
In practice Solution 2 is significantly faster for large n since palindrome checks are O(1) — no re-scanning of the same substrings repeatedly.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: "All possible" → Backtracking

The phrase "return all possible partitions" is the canonical backtracking signal. Any time you need to enumerate ALL valid configurations of a combinatorial structure, backtracking is the default framework.

Step 2: Identify the Choice at Each Step

At position start, the choice is: where does the next piece end? Every end index start..n-1 is a candidate. This is the branching factor at each level of the recursion tree.

Step 3: State = One Integer

The only state is start. This simplicity is a sign we're on the right track. Base case: start == n (full string consumed). No need for a "used" array or set.

Step 4: Mention DP Optimization

After the baseline solution, say: "We can precompute a 2D DP table pal[i][j] in O(n²) to make each palindrome lookup O(1) — important for large n where the same substrings get checked many times."

🗣️ What to Say to the Interviewer

"'Return all possible' → backtracking. At each position I choose where the next piece ends.
If that piece is a palindrome, I commit to it and recurse on the remainder.
When I've consumed the full string, I record the current partition.
After each recursive call, I pop the piece to explore other branch options.
Time: O(n · 2ⁿ). We could precompute a pal[i][j] DP table to make
palindrome checks O(1) instead of O(k), adding O(n²) space."
🔮 Likely Follow-up Questions

  • "What's the worst-case input?" → All same chars "aaaa" — all 2^(n-1) partitions valid
