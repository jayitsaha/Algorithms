LeetCode #139

Word Break

Given a string s and a dictionary wordDict, return true if s can be segmented into a space-separated sequence of one or more dictionary words.

Medium Dynamic Programming dp[i] = any dp[j] + word match Time: O(n² · L) Space: O(n)
1 Understanding the Problem ▾

You have a string like "leetcode" and a dictionary like ["leet","code"]. The question: can you slice the string into pieces where every piece is a word in the dictionary? You can reuse words, and you don't need to use all of them — you just need to cover the entire string.

✓ Valid Segmentation

"leetcode" → "leet" + "code"
Both in dict → return true

✗ No Valid Segmentation

"catsandog" → "cats"+"and"+"og"
"og" not in dict → return false

💡 Key Insight: Reachable Positions

Think of each position in the string as a stepping stone. You start at position 0. You can hop from position j to position i if the substring s[j:i] is in the dictionary. The question becomes: can you reach position n?

This naturally maps to DP: dp[i] = "is position i reachable?" Answer = dp[n].

Why does this have optimal substructure? If s[0:i] can be segmented, there must be a last word ending at i. Say it starts at j. Then s[0:j] must also be segmentable. So the answer at i depends only on strictly smaller subproblems — classic DP territory.

Why are there overlapping subproblems? In a naive recursive solution, canBreak("leetcode") calls canBreak("etcode") and canBreak("code"). But canBreak("code") might also be called from a different split path. We'd recompute the same suffixes many times without memoization.

2 The Algorithm: Bottom-Up DP (Tabulation) ▾

Build a boolean array dp[0..n] from left to right. Each dp[i] answers: "can the first i characters of s be segmented?" We compute small subproblems first and use their answers to build up to the full string.

📐 Recurrence Relation
dp[0] = True   # base: empty string segments trivially
dp[i] = OR { dp[j] AND s[j:i] in word_set }   for 0 ≤ j < i
answer = dp[n]
1 Convert wordDict to a set for O(1) average substring lookups instead of O(k) list search.
2 Initialize: dp = [False] * (n+1), then dp[0] = True. Size n+1 so index n represents the full string.
3 Outer loop i (1 to n): For each end position i, we're asking "can we reach here?"
4 Inner loop j (0 to i-1): Try every possible start of the last word ending at i. The last word would be s[j:i].
5 If dp[j] is True and s[j:i] is in word_set: set dp[i] = True and break (no need to check other j values).
6 Return dp[n]: True if we can reach the end of the full string.
🔑 Why Both Conditions Are Necessary

dp[j] AND s[j:i] in word_set — both are required. dp[j] = True means we can validly reach position j. s[j:i] in word_set means there's a valid word to jump from j to i. We need both a valid path to j and a valid word from j to i.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any point in the computation, dp[i] = True if and only if there exists at least one valid segmentation of s[0:i] using words from wordDict.

Correctness proof sketch:

  • Base: dp[0] = True because the empty string requires zero words — trivially segmented.
  • Inductive step: Assume all dp[0..i-1] are correct. For dp[i]: if any valid segmentation of s[0:i] exists, it has a last word s[j:i]. By induction, dp[j] is correct, so the algorithm will find this j and correctly set dp[i] = True. Conversely, if dp[i] is set to True, it's because a valid dp[j] was found with s[j:i] in the dict — a genuine segmentation.
⚠️ Edge Cases
  • dp[0] = True is non-negotiable: Without it, no chain ever starts — every dp[i] stays False.
  • Word reuse: Handled automatically. "aaaa" with ["a","aa"] works: dp[1]=T (j=0,"a"), dp[2]=T (j=0,"aa" or j=1,"a"), etc.
  • Whole-string is one word: j=0, dp[0]=True, s[0:n] in dict → dp[n]=True. Works.
  • No possible segmentation: No j ever satisfies both conditions → dp[n] stays False → returns False.
4 The Code (Annotated) ▾

Solution 1 — Bottom-Up Tabulation (Interview Pick) — O(n²·L) time, O(n) space

def wordBreak(s, wordDict): define function
word_set = set(wordDict) O(1) lookup per check; list would be O(k)
n = len(s) string length; dp will be size n+1
dp = [False] * (n + 1) dp[i] = can s[0:i] be segmented?
dp[0] = True base case: empty string is always valid
for i in range(1, n + 1): build up from left: position 1 to n
for j in range(i): try every possible last-word start
if dp[j] and s[j:i] in word_set:j reachable AND last word is valid?
dp[i] = True yes — position i is reachable
break found one valid split; no need to try more
return dp[n] can we segment the entire string?
⚠️ Off-By-One Trap

The dp array must be size n+1. Index i represents "the first i characters" — so dp[n] covers all n characters. If you use size n, you can't index dp[n] and lose the answer entirely.

Solution 2 — Top-Down Memoization — Same complexity, recursive

def wordBreak(s, wordDict): define function
word_set = set(wordDict) O(1) lookup
memo = {} cache: start_index -> bool
def dp(start): can s[start:] be segmented?
if start == len(s): base case: used entire string
return True success — no characters remain
if start in memo: already computed?
return memo[start] return cached answer
for end in range(start+1, len(s)+1):try all word endings from start
if s[start:end] in word_set and dp(end):word valid AND rest segmentable?
memo[start] = True cache success
return True early exit
memo[start] = False no valid segmentation from start
return False
return dp(0) start from beginning
💡 Bottom-Up vs Top-Down: Which to Use in an Interview?

Both have identical time/space complexity. Bottom-up (tabulation) is generally preferred in interviews: no recursion stack risk, straightforward to trace, and the DP table is immediately visible to the interviewer. Top-down (memoization) is sometimes cleaner to derive — you write the recursion naturally, then add caching.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

Three signals point to DP: (1) it's a feasibility question over a linear sequence, (2) the answer at position i depends on answers at earlier positions, (3) the same substrings can appear from multiple paths. The "can we partition?" framing is classic 1D DP.

Step 2: Define the State Clearly

Say aloud: "dp[i] = True means the first i characters can be segmented into dictionary words." State space is O(n), transitions need O(n) per state.

Step 3: Write the Recurrence

To compute dp[i]: for each j from 0 to i-1, if dp[j] is True AND s[j:i] is in the dictionary, then dp[i] = True. Base case: dp[0] = True (empty string).

Step 4: Implement and Check Complexity

Two nested loops = O(n²). Each iteration does a substring hash (O(L) where L is word length). Total: O(n²·L). Space: O(n) for dp + O(|dict| * avg_len) for word_set. Mention the set optimization upfront.

🗣️ What to Say to the Interviewer

"I see a classic DP setup. Let me define dp[i] as whether s[0:i] can be segmented using dictionary words. Base case: dp[0] = True — the empty string is trivially valid. Transition: for each i, I try every split point j from 0 to i-1. If dp[j] is true AND s[j:i] is in the word set, then dp[i] = True. I'll convert wordDict to a set first for O(1) lookups. Final answer is dp[n]. Time: O(n^2 * avg_word_len). Space: O(n) for the dp array."
💡 Follow-Up Questions to Expect

  • BFS variant: Treat positions 0..n as graph nodes; BFS from 0, enqueue i if s[j:i] in dict from reachable j

7 Summary & Key Takeaways ▾
Model positions as stepping stones: dp[i] = "can I reach position i?"
Recurrence: dp[i] = any(dp[j] == True and s[j:i] in dict) for j < i
Base case dp[0] = True is the anchor — without it nothing propagates
Convert wordDict to a set first for O(1) substring checks
Both tabulation and memoization give O(n²·L) time and O(n) space
The early break optimization avoids redundant j checks once dp[i] is set

🎯 When to Use This DP Pattern
  • Feasibility question: "can the string / array be partitioned into valid pieces?"
  • Pieces come from a dictionary, set, or rule — not all substrings qualify
  • The same subproblems (positions) appear from multiple split paths
  • Boolean output ("yes/no") → single bool per state; count output → int per state
