Add Bold Tag in String

LeetCode #616 — Boolean Array + Interval Merge
Medium String Transformation Interval Merging O(n·m·k) O(n)
1 Understanding the Problem ▾

Given a string s and an array words, we need to wrap every substring of s that matches any word in words with <b> and </b> tags. The twist: overlapping or adjacent bold regions must be merged into a single pair of tags.

Example: s = "abcxyz123", words = ["abc", "123"]

→ "abc" matches at [0,3), "123" matches at [6,9) → Result: "<b>abc</b>xyz<b>123</b>"

Trickier example: s = "aaabbcc", words = ["aaa", "aab", "bc"]

→ "aaa" at [0,3), "aab" at [1,4), "bc" at [4,6) — all overlap or are adjacent → One merged region [0,6) → "<b>aaabbc</b>c"

💡 Key Insight

Don't think about inserting tags directly. Instead, mark which characters should be bold (boolean array), then scan for contiguous bold regions to place tags. This naturally handles overlaps and adjacency.

2 The Algorithm: Boolean Array + Merge ▾

The approach has two clear phases:

1 Mark bold characters: Create a boolean array bold[i] of length n. For every word, find every occurrence in s and mark bold[start..start+len-1] = True.
2 Build result string: Scan bold[]. When we enter a True region, insert <b>. When we leave it, insert </b>. Copy characters in between.

The boolean array is the magic: it automatically merges overlapping and adjacent matches because multiple words can set the same positions to True.

For each word w in words
↓
Find all occurrences of w in s
↓
Mark bold[start .. start+len(w)-1] = True
↓
After all words processed, scan bold[]
↓
Insert <b> at True-start, </b> at True-end
3 Why It Works: The Invariant ▾
🔐 The Invariant

After Phase 1, bold[i] = True if and only if character s[i] is covered by at least one matching word occurrence. This means contiguous True regions exactly correspond to merged bold intervals — overlapping/adjacent matches are automatically unified.

Why merging is free: If word "aab" covers indices [1,3] and word "bc" covers [3,5], both set bold[3]=True. The resulting contiguous run [1,5] naturally produces one <b> tag pair.

Edge cases handled:

  • No matches: bold[] stays all False → return s unchanged
  • Entire string bold: bold[] all True → one <b>s</b> wrapper
  • Adjacent matches: Two words covering [0,2] and [3,5] → bold[0..5] all True → single tag pair
  • Single character words: Each occurrence marks exactly one position
⚠️ Adjacent vs Gap

"Adjacent" means two substrings share a boundary (end of one = start of next). The boolean array handles this correctly: [0,2] and [3,5] leave contiguous True values. But if there's a gap ([0,2] and [4,5]), index 3 stays False, creating two separate tag pairs.

4 The Code (Annotated) ▾
# Solution 1: Boolean Array (Interview Pick)
def addBoldTag(s, words):
n = len(s)
bold = [False] * n # bold[i]: should char i be bold?
for word in words: # try every word
start = s.find(word) # find first occurrence
while start != -1: # while we keep finding it
for i in range(start, start + len(word)):
bold[i] = True # mark these chars bold
start = s.find(word, start + 1) # find next
res = []
for i in range(n):
if bold[i] and (i == 0 or not bold[i-1]):
res.append('<b>') # start of bold region
res.append(s[i])
if bold[i] and (i == n-1 or not bold[i+1]):
res.append('</b>') # end of bold region
return ''.join(res)
⚠️ Common Mistake

Don't try to insert <b> tags as you find matches — indices shift when you insert text! The boolean array approach avoids this entirely by separating marking from building.

Time: O(n × m × k) — n=len(s), m=len(words), k=avg word length. For each word, find() scans O(n×k), and we mark O(k) positions per match.

Space: O(n) for the bold array.

5 🎤 Interview Strategy ▾
Step 1: Recognize the Pattern

"Merge overlapping regions" + "string matching" → Think interval merging. But instead of sorting intervals, a boolean array is simpler — it's a sweep-line over characters.

Step 2: Separate Marking from Building

The key design decision: mark ALL bold positions first, THEN build the string. This two-pass approach avoids the headache of shifting indices.

Step 3: Handle Boundaries

A bold region starts when bold[i]=True and either i=0 or bold[i-1]=False. It ends when bold[i]=True and either i=n-1 or bold[i+1]=False.

🗣️ What to Say to the Interviewer

"I'll use a boolean array approach. First, for each word, I'll find all occurrences in s and mark those positions as bold. The boolean array automatically merges overlaps. Then I'll scan the array — wherever a True region starts, I insert <b>, and where it ends, </b>. This is O(n·m·k) time, O(n) space. For a follow-up, I could optimize matching with a Trie for O(n·L) where L is the max word length."

7 Summary & Key Takeaways ▾
✓ Boolean array as sweep-line — marks which chars need bold, automatically merging overlaps.
✓ Two-pass design: Phase 1 marks positions, Phase 2 builds the result — no shifting indices.
✓ Boundary detection: A bold region starts when bold[i] and !bold[i-1], ends when bold[i] and !bold[i+1].
✓ Trie optimization: For large word lists or long max word length, build a Trie for O(n·L) matching.
🎯 When to Use This Pattern
  • Wrapping/tagging substrings that may overlap or be adjacent
  • "Merge overlapping intervals" on a 1D character array
  • Any problem where multiple matches need to be unified into contiguous regions
