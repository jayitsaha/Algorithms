LeetCode #427

Construct Quad Tree

Given an n×n grid of 0s and 1s, build a Quad Tree by recursively dividing mixed regions into four quadrants until every region is uniform.

Medium Divide and Conquer Recursive Quadrant Division Time: O(n² log n) Space: O(log n)
1 Understanding the Problem ▾

A Quad Tree encodes a 2D binary grid hierarchically. Every node covers a rectangular region of the grid. If the region is uniform (all 0s or all 1s), it becomes a leaf node — no children, just a value. If the region is mixed, it becomes an internal node with exactly four children, each covering one quadrant.

Each Node has two key fields:

  • isLeaf — True if all cells in this region are the same value
  • val — The value (0 or 1 for leaves; 1 by convention for internal nodes)
💡 Key Insight

A Quad Tree is a spatial compression scheme. Uniform regions collapse to a single leaf, saving space. Mixed regions are split until they become uniform. The recursion stops when a region is either all-0 or all-1 — or when it's a single cell (always uniform).

Each non-uniform region splits into four equal quadrants

✓ Leaf Node (uniform)

All cells = 1 → isLeaf=True, val=1
All cells = 0 → isLeaf=True, val=0
Single cell → always a leaf

✗ Internal Node (mixed)

Has both 0s and 1s → isLeaf=False
val = 1 (convention, ignored)
Has exactly 4 children

Edge Cases: 1×1 grid (root is always a leaf), all-uniform grid (root is a leaf — no splitting), checkerboard pattern (recurses all the way to 1×1 cells at max depth).

2 The Algorithm: Recursive Quadrant Division ▾

The core function is solve(r, c, size) — it builds the quad tree node for the size × size region whose top-left corner is at row r, column c.

1 Uniformity check: Count the number of 1s in the region. If count equals 0 → all zeros. If count equals size² → all ones. Either way, return a leaf node with the appropriate value.
2 Divide: If the region is mixed, compute half = size // 2. The four quadrants are: TL at (r, c), TR at (r, c+half), BL at (r+half, c), BR at (r+half, c+half). Each covers exactly half × half cells.
3 Conquer: Recurse on each of the four quadrants: tl = solve(r, c, half), tr = solve(r, c+half, half), bl = solve(r+half, c, half), br = solve(r+half, c+half, half).
4 Combine: Return an internal node: Node(val=1, isLeaf=False, topLeft=tl, topRight=tr, bottomLeft=bl, bottomRight=br). The four subtrees are attached as children.
🚀 Optimization: 2D Prefix Sum

Without optimization, the uniformity check scans all size² cells — O(n²) per call. With a 2D prefix sum array built in O(n²) upfront, any subgrid sum is O(1). This transforms the total complexity from O(n⁴) worst case to O(n² log n). The formula: sum(r,c,size) = pre[r+size][c+size] - pre[r][c+size] - pre[r+size][c] + pre[r][c].

3 Why It Works: The Invariant ▾
🔐 The Invariant

Every call to solve(r, c, size) returns a correct quad tree node for exactly the size × size region at (r, c): either a leaf (if uniform) or an internal node whose four children together tile the region exactly — no overlap, no gap.

Why the recursion terminates: Each recursive call handles a subgrid of size half = size // 2. Since the problem guarantees n is a power of 2, each halving produces integers: 4 → 2 → 1. At size 1, the region is a single cell — always uniform → always a leaf → no further recursion. Maximum depth is log₂(n).

Why the four quadrants partition the parent exactly: The top-left corner of the four sub-regions are at offsets (0,0), (0,half), (half,0), (half,half) from the parent's top-left. Each covers half × half cells. Since half = size/2, two halves in each dimension reconstruct the full size × size square with no gaps and no overlaps.

⚠️ Common Mistake: Dividing Uniform Regions

Always check uniformity before dividing. If you skip the check and always divide, a uniform region would produce four identical leaf children instead of one leaf — the tree would be deeper than necessary and technically incorrect for the quad tree spec.

⚠️ Why val=1 for Internal Nodes?

The problem spec says internal node val is irrelevant — only leaf node val carries meaning. Setting it to 1 is a common convention. Don't try to set it to anything meaningful; it just needs to be a valid integer.

Solution 2 — Prefix Sum Optimization (Optimal, Interview Pick)

def construct(grid):
n = len(grid)
# Build 2D prefix sum (1-indexed for easier boundary math)
pre = [[0]*(n+1) for _ in range(n+1)] ← (n+1)×(n+1) with sentinel border of zeros
for i in range(1, n+1):
for j in range(1, n+1):
pre[i][j] = (grid[i-1][j-1] ← current cell (0-indexed grid)
+ pre[i-1][j] + pre[i][j-1] ← add row above and col to left
- pre[i-1][j-1]) ← subtract overlap (inclusion-exclusion)
def rsum(r, c, sz): ← O(1) sum of [r..r+sz) × [c..c+sz)
r2, c2 = r+sz, c+sz
return pre[r2][c2]-pre[r][c2]-pre[r2][c]+pre[r][c] ← classic 2D prefix formula
def solve(r, c, sz):
s = rsum(r, c, sz) ← count of 1s in this subgrid in O(1)
if s == 0: return Node(0, True) ← all zeros → leaf val=0
if s == sz*sz: return Node(1, True) ← all ones → leaf val=1
h = sz // 2 ← mixed → divide into four halves
return Node(1, False,
solve(r, c, h), solve(r, c+h, h),
solve(r+h, c, h), solve(r+h, c+h, h))
return solve(0, 0, n)
💡 Complexity Summary

Naive: Time O(n⁴) worst case (O(n²) scan per call × O(n²) nodes). Space O(log n) stack.
Prefix Sum: Time O(n² log n) (O(n²) prefix build + O(1) per node × O(n²·log n) nodes). Space O(n²) prefix + O(log n) stack.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Divide and Conquer Signal

The problem literally defines the structure: "divide into four quadrants and recurse." Any time you see "split a 2D region into four equal sub-regions and process each independently," that's Recursive Quadrant Division. Related signals: spatial indexing, image compression, hierarchical grid encoding.

Step 2: Define the Recursive State Clearly

State: solve(r, c, size) — top-left row, top-left col, and current region size. These three values uniquely define any subgrid. State the invariant: "This function returns the quad tree node for the size × size region starting at (r, c)."

Step 3: Identify Both Termination Conditions

There are two leaf conditions, not one: (1) all zeros (sum=0), (2) all ones (sum=size²). Handle both before dividing. The implicit third: size=1 (always uniform, always a leaf — covered by conditions 1 and 2).

Step 4: Offer the Prefix Sum Optimization Proactively

Start with the naive O(n⁴) solution, then immediately say: "I can reduce the uniformity check from O(n²) to O(1) using a 2D prefix sum precomputed in O(n²). This gives total O(n² log n) time." This demonstrates awareness of standard 2D range query techniques.

🗣️ What to Say to the Interviewer

"I'll use divide and conquer. Define solve(r, c, size) for any subgrid.
First, check if the region is uniform: count 1s. If count=0, return leaf(0). If count=size², return leaf(1).
Otherwise, split into four equal quadrants and recurse on each.
Naive: O(n²) scan per call → O(n⁴) worst case.
Optimization: precompute a 2D prefix sum in O(n²). Then uniformity check is O(1).
Total: O(n² log n) time, O(log n) stack space (plus O(n²) for prefix sum)."

7 Summary & Key Takeaways ▾
A region is a leaf when all values are identical — check sum = 0 or sum = size²
Check uniformity BEFORE dividing — don't recurse into uniform regions
2D prefix sum enables O(1) uniformity checks, cutting total time to O(n² log n)
Recursion depth is log₂(n) — the problem guarantees n is a power of 2
Internal node val is conventional (1); only leaf val encodes real grid data
