LeetCode #198

House Robber

You are a professional robber planning to rob houses along a street. Adjacent houses have connected alarms — robbing two directly-adjacent houses triggers the alarm. Given an array nums, return the maximum amount you can rob tonight.

Medium Dynamic Programming Max Skip or Take Time: O(n) Space: O(1)
1 Understanding the Problem ▾

Imagine a row of houses, each holding a certain amount of money. You can rob any subset of them, but the constraint is strict: no two directly adjacent houses can both be robbed. Robbing houses 0 and 2 is fine; robbing houses 1 and 2 triggers the alarm.

💡 Key Insight

At every house, you face a binary decision: rob it (gain nums[i], but you must have skipped house i-1) or skip it (keep whatever you've accumulated up to house i-1). This two-option structure at every step, with each choice affecting future options, is the hallmark of 1D dynamic programming.

✓ Optimal: [2, 7, 9, 3, 1]

Rob house 0: +2
Skip house 1
Rob house 2: +9
Skip house 3
Rob house 4: +1
Total = 12 ✓

✗ Greedy (pick largest each time)

Rob house 1: +7 (largest)
Must skip 0 and 2
Rob house 3: +3
Must skip 4
Total = 10 ✗

Why greedy fails: Choosing the locally largest house (7) forces you to skip two neighbors (2 and 9) that together are worth 11. Greedy can't account for this because it doesn't look ahead. DP considers all possible choices and keeps track of the best outcome from every prefix.

Edge Cases: Empty array → return 0. Single house → return nums[0]. Two houses → return max(nums[0], nums[1]). All houses same value → rob every other one. Very large values → no overflow concerns in Python.

2 The Algorithm: 1D "Max Skip or Take" DP ▾

We define dp[i] = the maximum money we can rob from houses 0 through i. At each house i, we have exactly two options:

1 Rob house i: Collect nums[i]. Since we can't touch house i-1, the best we could have done before is dp[i-2]. Contribution: nums[i] + dp[i-2].
2 Skip house i: Don't rob it. The best we have is whatever we achieved up through house i-1: dp[i-1].
3 Take the max: dp[i] = max(dp[i-1], nums[i] + dp[i-2]). We record whichever choice yields more money.
4 Optimize space: dp[i] only looks back 2 positions. Replace the entire array with two rolling variables prev2 (old dp[i-2]) and prev1 (old dp[i-1]) for O(1) space.
📐 The Recurrence Relation
dp[0] = nums[0]
dp[1] = max(nums[0], nums[1])
dp[i] = max(dp[i-1], nums[i] + dp[i-2])  for i ≥ 2

dp[i-1] = best profit if we SKIP house i
nums[i] + dp[i-2] = best profit if we ROB house i

Full trace through nums = [2, 7, 9, 3, 1]:

dp[i] = max 2 7 11 11 12 ✓
3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing house i, dp[i] always equals the maximum loot achievable from houses 0 through i with no two adjacent houses robbed. This invariant is established at the base cases and preserved by the recurrence at every step.

Optimal Substructure: The best answer for houses 0…i depends only on the answers for 0…i-1 (skip case) and 0…i-2 (rob case). No other history is needed — only the maximum dollar amount at those two lookback positions. This means sub-answers compose correctly into the full answer.

Overlapping Subproblems: A naive recursive solution for house 4 calls dp(3) and dp(2). dp(3) then calls dp(2) and dp(1) — computing dp(2) again. This exponential redundancy is exactly what memoization (top-down) or tabulation (bottom-up) eliminates.

Why the two-variable optimization is correct: Since dp[i] depends only on dp[i-1] and dp[i-2], we only ever need the two most recent values. Rolling two variables is equivalent to maintaining the full table — with O(1) space.

⚠️ Critical: Update prev2 Before prev1

In the loop body, you must do prev2 = prev1 before prev1 = curr. If you swap the order, prev2 gets the new prev1 value instead of the old one, corrupting the lookback. One safe pattern: use simultaneous assignment prev2, prev1 = prev1, curr.

Base case verification:

  • 1 house [5]: prev1=0, prev2=0. Loop: curr=max(0, 5+0)=5. prev1=5. Return 5. ✓
  • 2 houses [2,7]: After i=0: prev1=2,prev2=0. After i=1: curr=max(2,7+0)=7, prev1=7. Return 7. ✓
4 The Code (Annotated) ▾

Solution 1 — Space-Optimized DP (Interview Pick, O(n) time / O(1) space)

def rob(nums: list[int]) -> int:
prev2, prev1 = 0, 0 ← prev2=dp[i-2], prev1=dp[i-1]; 0 before any house
for num in nums: ← Process each house left to right
curr = max(prev1, ← Option A: skip this house, carry forward prev1
num + prev2)← Option B: rob this house + best from 2 steps back
prev2 = prev1 ← Slide: old prev1 becomes new prev2
prev1 = curr ← Slide: curr becomes new prev1 for next iteration
return prev1 ← prev1 = dp[n-1] = maximum loot over all houses
⚠️ Why prev2, prev1 start at 0 (not nums[0])

We initialize both to 0 and let the loop handle all houses uniformly — including index 0. For house 0: curr = max(prev1=0, nums[0]+prev2=0) = nums[0]. For house 1: curr = max(prev1=nums[0], nums[1]+prev2=0) = max(nums[0], nums[1]). Both base cases emerge naturally without special handling.

Solution 2 — Tabulation with Full DP Array (O(n) / O(n)) — easier to read

nums[i] + dp[i-2]) ← Rob house i + best from 2 back
return dp[n-1] ← Last entry = max over all n houses

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the DP Signal

"We're maximizing a sum over a sequence where each choice (rob or skip) constrains future choices." This is the textbook 1D DP signal. The constraint "no two adjacent" makes greedy fail and forces you to track the best outcome for each prefix.

Step 2: Define the State

Say explicitly: "Let dp[i] = maximum amount we can rob from houses 0 through i." A precise state definition is the foundation. Note that dp[i] doesn't track which houses were robbed — just the max dollar value. That's sufficient because of optimal substructure.

Step 3: Write the Recurrence

Walk through the two choices: "At house i, I either skip it (→ dp[i-1]) or rob it (→ nums[i] + dp[i-2]). I take the max." Write dp[i] = max(dp[i-1], nums[i] + dp[i-2]). Handle base cases: dp[0]=nums[0], dp[1]=max(nums[0], nums[1]).

Step 4: Optimize Space

"Since dp[i] only looks back two steps, I can replace the full array with two rolling variables prev2 and prev1, achieving O(1) space." This is a strong signal to your interviewer that you know DP optimization techniques.

🗣️ What to Say to the Interviewer

"This is a 1D DP problem. At each house I can rob it or skip it.
Robbing house i means I can't have robbed house i-1, so I add nums[i] to dp[i-2].
Skipping house i means I keep whatever dp[i-1] was.
Recurrence: dp[i] = max(dp[i-1], nums[i] + dp[i-2]).
Since we only look back 2 steps, I'll use two rolling vars: prev2 and prev1.
Time O(n), Space O(1)."
🔮 Likely Follow-up Questions
  • "What if the houses form a circle?" → House Robber II (#213): run DP twice — once excluding first house [1..n-1], once excluding last [0..n-2]; take the max.
  • "What if the houses form a binary tree?" → House Robber III (#337): tree DFS returning (rob_root_profit, skip_root_profit) pair from leaves up.
  • "Which houses were actually robbed?" → Trace back the dp array: if dp[i] == dp[i-1], house i was skipped; otherwise it was robbed.
  • "Delete and Earn problem?" → #740: transform to House Robber by creating a value array where value[v] = v * count(v), then run this algorithm on it.

7 Summary & Key Takeaways ▾
At each house: rob it (nums[i] + dp[i-2]) or skip it (dp[i-1]); always take the max
Recurrence: dp[i] = max(dp[i-1], nums[i] + dp[i-2]) — the entire algorithm in one line
Only two values needed at once → O(1) space with rolling prev2/prev1 variables
Greedy fails because local greed can block access to globally superior non-adjacent choices
This exact pattern generalizes to circular (HR II), tree (HR III), and transformed variants (Delete and Earn)
ApproachTimeSpaceNotes
Naive Recursion O(2ⁿ) O(n) stack Exponential — TLE on large inputs

Space-Optimized ✓ O(n) O(1) Interview pick — two rolling variables
🎯 When to Recognize the "Max Skip or Take" Pattern
  • "Cannot pick adjacent / neighboring elements" → skip-or-take DP
  • Maximizing a sum over a sequence with a local exclusion constraint
  • Greedy (pick largest) produces wrong answer — need to look at two lookback states
  • Only last 2 states matter → O(1) space optimization always possible
