LeetCode #48

Rotate Image

You are given an n×n 2D matrix representing an image. Rotate the matrix 90 degrees clockwise in-place — do not allocate another 2D matrix.

Medium Matrix Transpose + Reverse Rows Time: O(n²) Space: O(1)
1 Understanding the Problem ▾

Rotating an image 90° clockwise means: what was the top row becomes the right column; what was the left column becomes the top row. Every pixel slides to a new position determined by the rotation geometry.

💡 Key Insight

A 90° clockwise rotation maps every cell matrix[i][j] to position matrix[j][n-1-i]. The challenge is performing this in-place without allocating a second matrix. The Transpose + Reverse decomposition achieves exactly that in two clean O(n²) passes, each touching cells in-place.

Constraints: Modify in-place — no copy(), no second matrix. Always square (n×n). Edge cases: 1×1 is a no-op; 2×2 cycles four cells. The rotation direction (clockwise vs counter-clockwise) matters — always clarify before coding.

2 The Algorithm: Transpose then Reverse Each Row ▾

The Transpose + Reverse method decomposes 90° CW rotation into two simpler operations, each trivially in-place:

1 Transpose: Reflect across the main diagonal — swap matrix[i][j] with matrix[j][i] for all pairs where j > i (upper triangle only, to avoid double-swapping). After this pass, rows become columns.
2 Reverse each row: After transposing, each row has the right values but reversed order. Call row.reverse() on each row — Python's built-in does this in O(n) with a two-pointer swap from both ends.
3 Done! Transpose maps (i,j) → (j,i); row-reverse maps (j,i) → (j, n-1-i). Composing: (i,j) → (j, n-1-i) — the exact 90° CW formula. No extra array required.

🔑 Alternative: Layer-by-Layer Cyclic Swap

Process each concentric ring from outside in. For each layer at depth d, perform four-cell cyclic rotations: top→right→bottom→left→top, using one temporary variable. More index-math to write but reveals geometric intuition directly. Both are O(n²) time, O(1) space — transpose+reverse is shorter to code and explain.

3 Why It Works: The Mathematical Proof ▾
🔐 Coordinate Composition

90° CW rotation formula: (i, j) → (j, n-1-i)
Transpose: (i, j) → (j, i)
Reverse row j: column i → (n-1-i), so (j, i) → (j, n-1-i)
Composing: (i,j) → (j,i) → (j, n-1-i) ✓ QED.

Concrete proof for cell (i=0, j=2) in a 3×3 matrix:

  1. Start: cell (0,2) holds value 3. Target after 90° CW: (j, n-1-i) = (2, 2-0) = (2,2).
  2. After transpose: cell moves from (0,2) to (2,0). Value 3 is now at (2,0).
  3. After reversing row 2: column 0 → column (n-1-0) = column 2. Value 3 is now at (2,2). ✓
⚠️ Counter-Clockwise Rotation

CCW formula: (i,j) → (n-1-j, i). Decomposition reverses: first reverse each row, then transpose. Equivalently: transpose first, then reverse each column. Knowing both variants demonstrates you understand the geometry rather than a memorized recipe.

Why upper-triangle only? Iterating all (i,j) and swapping (i,j)↔(j,i) visits each off-diagonal pair twice — the second swap undoes the first. Restricting to j > i visits each pair exactly once. The diagonal (i==j) is self-symmetric; swapping a cell with itself is a no-op we skip by using strict inequality.

4 The Code (Annotated) ▾

Solution 1 — Transpose + Reverse (Optimal · Interview Pick)

def rotate(matrix: list[list[int]]) -> None:
n = len(matrix) ← size of the n×n grid
# Step 1: Transpose (reflect across main diagonal)
for i in range(n): ← iterate each row index
for j in range(i + 1, n): ← UPPER TRIANGLE ONLY (j > i); avoids double-swap
matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]← swap symmetric pair
# Step 2: Reverse each row in-place
for row in matrix: ← iterate all n rows
row.reverse() ← Python built-in; O(n) two-pointer swap in-place
⚠️ Common Mistake: Full Range in Inner Loop

Writing for j in range(n) instead of range(i+1, n) swaps each symmetric pair twice, undoing both swaps and leaving the matrix unchanged. Always use strictly j > i for the upper triangle. This is a subtle bug that passes simple tests but fails non-symmetric inputs.

Solution 2 — Extra Space, Direct Formula (O(n²) time and space)

def rotate_extra(matrix): ← Not in-place; useful for understanding the formula
n = len(matrix)
copy = [row[:] for row in matrix] ← shallow-copy each row into a new list
for i in range(n):
for j in range(n):
matrix[j][n-1-i] = copy[i][j] ← direct formula: (i,j) → (j, n-1-i)

Solution 3 — Layer-by-Layer Cyclic Swap (O(n²) time · O(1) space)

def rotate_layer(matrix): ← process outer ring then inner rings
n = len(matrix)
for d in range(n // 2): ← d = layer depth (0 = outermost ring)
for k in range(d, n - 1 - d): ← k = position along the layer edge
tmp = matrix[d][k] ← save top-side value
matrix[d][k] = matrix[n-1-k][d] ← left → top
matrix[n-1-k][d] = matrix[n-1-d][n-1-k] ← bottom → left
matrix[n-1-d][n-1-k] = matrix[k][n-1-d] ← right → bottom
matrix[k][n-1-d] = tmp ← saved top → right
💡 Complexity Summary

Transpose + Reverse: O(n²) time — visits every cell twice. O(1) space — only swap variables.
Extra Space: O(n²) time, O(n²) space — requires full matrix copy.
Layer Cyclic: O(n²) time, O(1) space — one tmp variable for four-cell cycles.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify Before Coding

Confirm clockwise vs counter-clockwise. Ask: "Must I do this in-place?" (Yes.) Write the target formula (i,j) → (j, n-1-i) on the whiteboard — it proves you understand the geometric transformation before touching implementation details.

Step 2: Propose Brute Force

"Trivial: make a copy, apply the formula directly — O(n²) time and O(n²) space. Correct but violates the in-place constraint. I want to eliminate the extra matrix."

Step 3: Discover the Decomposition

"I can decompose the rotation: transpose (reflect across main diagonal) maps (i,j)→(j,i). Then reversing each row maps (j,i)→(j,n-1-i). Composing gives exactly the rotation formula. Both steps are in-place."

Step 4: Code and Verify

Write the transpose loop (inner from i+1), then row reversal. Trace the 3×3 example: original top row [1,2,3] → after transpose row 0 = [1,4,7] → after reverse row 0 = [7,4,1] → becomes the top row of the rotated image. Correct!

🗣️ What to Say to the Interviewer

"The 90° CW rotation formula is: (i,j) maps to (j, n-1-i).
To do this in-place, I decompose into two operations:
First transpose: swap matrix[i][j] with matrix[j][i] for all j greater than i.
This maps (i,j) to (j,i). Then reverse each row: (j,i) maps to (j, n-1-i).
Composing those: (i,j) to (j, n-1-i) — exactly 90° clockwise.
Time O(n-squared). Space O(1). Truly in-place."
🔮 Likely Follow-up Questions
  • "Counter-clockwise?" → Reverse each row first, then transpose (formula: (i,j)→(n-1-j, i))
  • "180° rotation?" → Reverse row order, then reverse each row (or apply 90° CW twice)
  • "Non-square matrix?" → In-place impossible — dimensions change; must allocate new matrix
  • "Very large image on disk?" → Block rotation with cache-friendly access patterns; tiling approach
