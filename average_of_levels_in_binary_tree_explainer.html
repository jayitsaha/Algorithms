LeetCode #637

Average of Levels in Binary Tree

Given the root of a binary tree, return the average value of the nodes on each level in the form of an array.

Easy Trees BFS Level Order Time: O(n) Space: O(w) max width
1 Understanding the Problem ▾

A binary tree can be read level by level: the root sits at level 0, its children at level 1, their children at level 2, and so on. This problem asks: for each level, what is the mean (average) value of all nodes that live there? Return these averages as a list, one per level from top to bottom.

💡 Key Insight

BFS (Breadth-First Search) with a queue naturally visits nodes level by level. The crucial trick is: capture level_size = len(queue) before the inner loop starts. That snapshot tells you exactly how many nodes are on the current level — children pushed during the inner loop belong to the next level and are never counted in this snapshot.

Example Tree

    3
  /   \
 9    20
     /  \
    15   7

Expected Output

Level 0: [3]     → 3/1  = 3.0
Level 1: [9, 20] → 29/2 = 14.5
Level 2: [15, 7] → 22/2 = 11.0
Result: [3.0, 14.5, 11.0]

Edge cases to keep in mind:

  • Single node: result = [root.val] — only one level, average is the root itself.
  • Left-skewed tree: Each level has exactly one node; the average equals that node's value.
  • Perfect binary tree: Level k has 2^k nodes — the widest level determines queue space usage.
  • Large values: If node values are up to 10^4 and a level can have 32768 nodes (2^15), max sum ≈ 3.2×10^8. In Python this is fine; in Java/C++ use long for the sum.
🔑 What is a "Level"?

All nodes at the same depth from the root form a level. Root is depth 0. A node's depth equals its parent's depth + 1. BFS explores all depth-d nodes before any depth-(d+1) node — that's the property we exploit.

2 The Algorithm: BFS Level-Order Traversal ▾

We use a queue (FIFO) and process the tree one level at a time. The trick that separates level-order BFS from a plain BFS is: we snapshot the queue size at the start of each level to know exactly how many nodes to dequeue before computing the average.

1 Handle empty tree: If root is None, return [] immediately — no levels exist.
2 Seed the queue: Initialize a deque with just the root. It represents the sole node at level 0.
3 Outer loop — while queue not empty: Each iteration of this outer loop processes exactly one complete level of the tree.
4 Snapshot level size: level_size = len(queue) — lock in how many nodes are at this level before any children are enqueued.
5 Inner loop — exactly level_size iterations: Dequeue each node. Add its value to level_sum. Enqueue its non-null left and right children for the next level.
6 Compute and store average: level_sum / level_size (float division). Append to result. Then the outer loop continues — queue now holds the next level.
🔑 Why Snapshot Before the Inner Loop?

During the inner loop we add children to the queue. If we checked len(queue) dynamically, the inner loop would keep running as new children arrive, causing us to bleed into the next level. By capturing the count first, we guarantee the inner loop processes exactly this level's nodes — no more, no less. This is the core trick of BFS level-order traversal.

Alternative — DFS with depth parameter: A recursive preorder DFS can maintain a list of (sum, count) pairs indexed by depth. When visiting depth d for the first time, append a new entry; otherwise, update the existing one. Both BFS and DFS are O(n) time, but BFS is the canonical approach for level-order questions and should be your first instinct in an interview.

3 Why It Works: The Invariant ▾
🔐 The BFS Level Invariant

At the start of each outer-loop iteration, every node in the queue belongs to exactly the same depth. This invariant holds because:

  • We initialize with only the root (depth 0).
  • In each inner loop, we dequeue a depth-D node and enqueue only its children (depth D+1). The two depths never mix.
  • We process exactly level_size nodes — the snapshot count — ensuring we stop before touching depth-(D+1) nodes.

Why this guarantees correct averages: If all nodes in the queue at the start of an outer iteration are at depth D, then after processing those level_size nodes, the queue holds exactly all nodes at depth D+1. We computed level_sum and level_size for depth D precisely, so level_sum / level_size is correct. By induction on D, every level's average is computed correctly.

⚠️ Common Mistake: Integer Division

In Python 3, / always produces a float: 29 / 2 = 14.5. But in Python 2, Java, or C++, integer division truncates: 29 / 2 = 14. Always ensure float division when computing averages. In Python, avoid //. In Java/C++, cast the sum or size to double before dividing.

⚠️ Common Mistake: Using list.pop(0) Instead of deque.popleft()

In Python, list.pop(0) is O(n) — all elements shift left after the removal. Over n dequeue operations this gives O(n²). Use collections.deque with popleft() for O(1) removal from the front.

Space analysis: The queue never holds more than two full levels simultaneously (one being dequeued, one being enqueued). In the worst case (a complete binary tree), the bottom level has ⌈n/2⌉ nodes, so queue space is O(n/2) = O(n). In practice for a balanced tree, this is O(w) where w is the tree's maximum width.

Solution 1 — BFS Level Order (Interview Pick, O(n) time)

def averageOfLevels(root): ← root: TreeNode | None
if not root: ← Empty tree → no levels → empty list
return []
result = [] ← Collect one float per level
queue = deque([root]) ← Seed with root; deque for O(1) popleft
while queue: ← Loop until all levels processed
level_size = len(queue) ← CRITICAL: snapshot before inner loop starts
level_sum = 0 ← Fresh accumulator for this level
for _ in range(level_size): ← Process exactly this level's nodes
node = queue.popleft() ← O(1) dequeue; 'node' is current level-D node
level_sum += node.val ← Accumulate value
if node.left: ← Enqueue left child (level D+1) if exists
queue.append(node.left)
if node.right: ← Enqueue right child (level D+1) if exists
queue.append(node.right)
result.append(level_sum / level_size) ← Float division → average for level D
return result ← [avg_level_0, avg_level_1, ..., avg_level_k]

Solution 2 — DFS with depth tracking (O(n) time, O(h) space)

def averageOfLevels(root):
sums = [] ← sums[d] = running total of node values at depth d
counts = [] ← counts[d] = number of nodes at depth d
def dfs(node, depth):
if not node: return ← Base case: null node, nothing to do
if depth == len(sums): ← First visit to this depth: expand arrays
sums.append(0); counts.append(0)
sums[depth] += node.val ← Add value to depth's running total
counts[depth] += 1 ← Count this node at this depth
dfs(node.left, depth + 1) ← Recurse left, one depth deeper
dfs(node.right, depth + 1) ← Recurse right, one depth deeper
dfs(root, 0)
return [s / c for s, c in zip(sums, counts)] ← Average per depth
💡 Complexity Summary

BFS: Time O(n) — every node enqueued and dequeued once. Space O(w) — queue holds at most one full level (max width w).
DFS: Time O(n) — every node visited once. Space O(h) — call stack depth equals tree height h (O(log n) balanced, O(n) skewed).
For wide, shallow trees BFS uses more space. For tall, narrow trees DFS uses more space. BFS is the expected answer for level-order problems.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

"Average value of nodes on each level" — any per-level aggregation signals BFS level-order traversal. Other signals: "level by level", "depth by depth", "rightmost node at each level". When you see these phrases, immediately reach for BFS + queue with the level-size snapshot.

Step 2: Verbalize the BFS Skeleton

Say out loud: "I'll use a queue. Seed it with the root. Each outer loop iteration = one level. Snapshot level_size = len(queue) before the inner loop. Dequeue exactly level_size nodes, summing values and enqueuing non-null children. Then compute sum / size for this level's average."

Step 3: State the Non-Obvious Insight

Explicitly say: "The reason I capture len(queue) before the inner loop is that children added during the inner loop belong to the next level — I don't want them counted in this level's size. That snapshot is the boundary-defining trick."

Step 4: Note Implementation Details

In Python: use collections.deque for O(1) popleft(). Use / not // for float division. In Java/C++: use LinkedList or ArrayDeque as queue; cast sum to double before division.

7 Summary & Key Takeaways ▾
BFS naturally processes trees level by level — use it for any per-level aggregation: average, max, min, count, rightmost value.
The key mechanic: level_size = len(queue) before the inner loop. This snapshot defines the level boundary even as children are being enqueued.
Always use collections.deque in Python for O(1) popleft(). Plain list.pop(0) is O(n) per call — catastrophic for large trees.
Use float division: level_sum / level_size (not //). In Java/C++ cast to double before dividing.
DFS alternative: track (sum, count) per depth index. O(h) space vs BFS's O(w). Choose based on tree shape — but default to BFS in interviews.
This exact BFS skeleton (seed → while queue → snapshot size → inner loop → compute) solves LeetCode #102, #107, #199, #515, #429, #116/117 with minimal modification.
