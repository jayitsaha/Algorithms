LeetCode #113

Path Sum II

Find all root-to-leaf paths in a binary tree where the sum of node values equals a target.

Medium Trees Backtracking Time: O(n²) Space: O(n)
1 Understanding the Problem ▾

Given a binary tree root and an integer targetSum, return all root-to-leaf paths where the path node values sum to targetSum. A leaf is a node with no children. Return the paths as a list of lists — each inner list contains the node values in traversal order.

This is the "collect all" version of Path Sum I (LeetCode #112), which only needs to return true/false. Here we must gather every qualifying path — and since there can be multiple, we explore the entire tree.

💡 Key Insight

Think of the tree as a map of hiking trails. You walk every possible trail from the trailhead (root) to each dead end (leaf), keeping a running tally. When you reach a dead end, check the tally — if it equals the target, record the trail. When you turn back (backtrack), undo the last step so the tally is accurate for other branches.

2 The Algorithm: DFS with Path Backtracking ▾

We traverse the tree with depth-first search (DFS), maintaining a single current path list. As we descend, we append the node. As we ascend (backtrack), we pop the node. This append/pop discipline ensures the path always reflects exactly the nodes from root to the current position.

1 Enter the node: Append node.val to path. Subtract node.val from remain (the running countdown). This "remaining target" approach means we reach zero when the path sum equals the target — no need to carry both a running sum and the target separately.
2 Leaf check: Is this node a leaf? (not node.left and not node.right). If yes, check if remain == 0. If both conditions hold, this path is valid — record a copy of path into results.
3 Recurse into children: Call dfs(node.left, remain) and then dfs(node.right, remain). Each subtree gets the same remain value (local variable, not mutated between calls).
4 Backtrack: After both children return, call path.pop(). This removes the current node from the shared path list, restoring it to exactly the state it was in before we entered this node — ready for sibling branches.
🔑 The "Remaining" Countdown Trick

Instead of tracking current_sum and comparing it to targetSum, we start with remain = targetSum and subtract each node's value as we descend. A zero remainder at a leaf means "path sum equals target." This simplifies the interface: the helper only needs (node, remain), not three parameters.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At any moment during DFS, path contains exactly the node values from root to the current node (inclusive), and remain equals targetSum minus the sum of those values. The path.pop() after both recursive calls ensures this is restored on every return.

Why the copy is non-negotiable: Python lists are objects — when you do results.append(path), you store a reference to the same list that backtracking will continue to modify. By the time you return results, all stored "paths" point to the same empty list. The fix: results.append(list(path)) or path[:] creates a snapshot at that moment in time, independent of future mutations.

⚠️ The #1 Bug: Forgetting the Copy

If you write results.append(path) instead of results.append(list(path)), your final answer will be a list of references all pointing to the same (now-empty) list object. Every supposedly "recorded" path gets silently corrupted. Always snapshot with list(path).

Why every root-to-leaf path is visited exactly once: The DFS call tree mirrors the actual tree. Each node is entered once and exited once. The path list is a perfect shadow of the DFS call stack — it grows when we go deeper and shrinks when we return. This structural correspondence guarantees completeness (all paths visited) and soundness (no path is counted twice).

Time complexity — why O(n²) and not O(n): We visit each node once: O(n). But at each qualifying leaf, we copy the current path, which has up to O(h) elements where h is the tree height. With a balanced tree, h = O(log n), but in the pathological case (a fully left-skewed "bamboo" tree), h = O(n) and there is one leaf, giving O(n) copy for O(n) total. In a complete binary tree with O(n) leaves each requiring O(log n) copy, total is O(n log n). The bound O(n²) covers the worst case.

4 The Code (Annotated) ▾

Solution 1 — DFS Backtracking (Interview Pick)

def pathSum(root, targetSum): ← outer function; root can be None
results, path = [], [] ← results: all valid paths; path: shared scratch list
def dfs(node, remain): ← inner DFS; remain counts down from targetSum
if not node: ← base case: null child — nothing to do
return
path.append(node.val) ← ① ENTER: add this node's value to path
remain -= node.val ← ② count down: how much target remains?
if not node.left and not node.right: ← ③ are we at a leaf?
if remain == 0: ← ④ does the path sum equal target?
results.append(list(path)) ← ⑤ snapshot! list(path) copies — not a reference
dfs(node.left, remain) ← ⑥ explore left subtree
dfs(node.right, remain) ← ⑦ explore right subtree
path.pop() ← ⑧ BACKTRACK: undo this node before returning
dfs(root, targetSum) ← kick off DFS from root with full target
return results ← all valid root-to-leaf paths collected
💡 Complexity Summary

Time: O(n²) — O(n) nodes × O(n) copy per leaf in worst case (skewed tree).
Space: O(n) — recursion depth O(h) + path list O(h). Results not counted in auxiliary analysis.
Note: This IS the optimal approach. There's no way to do better than O(n²) when the output itself can be O(n²) characters long.

Solution 2 — BFS with Queued Paths (iterative, avoids deep recursion)

from collections import deque
def pathSum(root, targetSum):
if not root: return [] ← guard: empty tree
results = []
# Each queue entry: (node, path_list, remain)
q = deque([(root, [root.val], targetSum - root.val)]) ← seed with root
while q:
node, path, remain = q.popleft()
if not node.left and not node.right: ← leaf
if remain == 0:
results.append(path) ← path is already a fresh list per entry — no copy needed
if node.left:
q.append((node.left, path + [node.left.val], remain - node.left.val)) ← path + [...] creates new list
if node.right:
q.append((node.right, path + [node.right.val], remain - node.right.val))
return results
⚠️ BFS Trade-off: Much Higher Memory

The BFS version is iterative (no risk of recursion stack overflow on very deep trees), but each queue entry carries its own copy of the path accumulated so far. At any moment, the queue can hold O(n) entries each with an O(h) path list — total memory O(n · h), which is O(n²) worst case. Only use BFS if stack overflow is a real concern (e.g., very deep trees in languages without tail-call optimization).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the "All Paths" Signal

"The problem asks for all root-to-leaf paths — not existence, not count. Any time a tree problem says 'all paths,' DFS with backtracking is the canonical tool. The tree structure maps naturally to recursion; the call stack itself tracks which path we're on."

Step 2: Design the Recursion State

"I need: (1) current node, (2) path so far, (3) remaining target. I'll use a shared mutable list for the path — cheaper than creating new lists at each recursive call — and use append/pop to maintain the invariant."

Step 3: Announce the Copy Pitfall Pre-emptively

"One critical detail: when I record a valid path, I must copy it with list(path). If I append the list reference directly, all recorded paths will be corrupted as backtracking empties the shared list. This is the most common bug in this problem."

Step 4: State Complexity Clearly

"Time is O(n²) — O(n) nodes visited, O(n) path copy at each leaf in worst case. Space is O(n) auxiliary — recursion depth plus the path list, both bounded by tree height. The O(n²) time is tight; this is already optimal given the output can be that large."

🗣️ What to Say to the Interviewer

"Since we need all root-to-leaf paths, I'll DFS with backtracking.
I'll maintain a shared path list — append on entry, pop on exit.
At each leaf, if the remaining target is zero, I snapshot the path with list(path).
The snapshot is critical — appending the list reference would corrupt all results as backtracking modifies it.
Time: O(n²) — O(n) traversal plus O(n) copy per leaf. Space: O(n) auxiliary."
🔮 Likely Follow-up Questions
