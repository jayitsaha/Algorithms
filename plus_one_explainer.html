LeetCode #66

Plus One

Given a large integer represented as an array of digits (most significant digit first), increment the integer by one and return the resulting array.

Easy Math Handle Carry Time: O(n) Space: O(1)*
1 Understanding the Problem ▾

You're given a number that's too large for a standard 32-bit integer — so it's stored as a digit array. digits[0] is the most significant (leftmost) digit. Your job: add 1 to this number and return the updated array.

💡 Key Insight

This is grade-school addition with carry propagation. Start from the ones place (rightmost digit), add 1. If the result is 10, write 0 and carry 1 to the next digit left. Keep carrying until no overflow. The only truly tricky case is when every digit is 9 — the carry bubbles all the way through and you must prepend a new leading 1.

✓ Simple case: [1, 2, 3]

123 + 1 = 124
digits[2]: 3 + 1 = 4 (no carry)
→ Return [1, 2, 4]

⚡ Hard case: [9, 9, 9]

999 + 1 = 1000
Carry cascades through all digits
→ Return [1, 0, 0, 0] — grows!

Edge Cases to consider: Single digit [0] → [1]. Single nine [9] → [1, 0]. All nines like [9, 9, 9] → [1, 0, 0, 0] (array grows by 1). Constraints guarantee no leading zeros and all digits are 0–9.

2 The Algorithm: Right-to-Left Carry Propagation ▾

Simulate what you'd do by hand: start at the ones place and work left, handling carry as you go. Stop as soon as carry dies — there's no need to touch digits further left.

1 Loop right-to-left: Start at i = len(digits) - 1 (ones place) and decrement i each iteration.
2 If digits[i] < 9: Simply do digits[i] += 1 and return immediately. No carry will propagate. Digits to the left are unchanged.
3 If digits[i] == 9: Set digits[i] = 0 (because 9 + 1 = 10, ones place = 0) and let the loop continue left to handle the carry.
4 If loop finishes without returning: Every digit was 9. The array is now all zeros. Prepend 1 → return [1] + digits. This is the only case the array grows.
🔑 Why Early Return is Correct

The moment we find a digit < 9, incrementing it gives a result from 1–9. No carry is generated. All digits to its left are unaffected. The function can safely return the modified array right there.

⚠️ The All-9s Trap

If every digit is 9, every iteration sets that digit to 0. We exit the loop having never hit the early return. At this point digits is all zeros. We prepend 1 to represent the new most-significant digit. E.g., [9,9,9] → after loop: [0,0,0] → [1]+[0,0,0] = [1,0,0,0].

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the start of each iteration at position i: digits from i+1 to len-1 are already in their final correct state, and there is exactly carry = 1 that must be applied to position i. Once we apply the carry without producing overflow, all remaining positions are untouched.

Initialization: Before the loop, carry = 1 (our "+1") must be applied starting at the last index. No digits have been modified yet. Invariant holds.

Maintenance: At each position i with carry = 1: if digits[i] < 9, we increment and return — positions i through end are all final. If digits[i] == 9, we set it to 0 and carry = 1 persists to position i-1. In both cases, positions to the right of i are already final.

Termination: Either early return is hit (carry absorbed at some digit < 9), or loop exhausts the array with carry = 1 still active, requiring a new leading digit.

⚠️ Why Not Convert to int?

The naive approach: int("".join(map(str, digits))) + 1. This works in Python due to arbitrary-precision integers, but fails in languages with fixed-size integers (Java's int or long overflow for very large digit arrays). The carry-propagation approach works in any language in O(n) time with O(1) extra space.

Correctness on edge cases:

  • [0]: i=0, 0<9 → digits[0]=1, return [1] ✓
  • [9]: i=0, 9==9 → digits[0]=0, loop ends → [1]+[0]=[1,0] ✓
  • [1,9]: i=1: 9→0; i=0: 1<9 → 2, return [2,0] ✓
  • [9,9,9]: all become 0, prepend 1 → [1,0,0,0] ✓
4 The Code (Annotated) ▾

Solution 1 — Right-to-Left Carry Propagation (Optimal, Interview Pick)

def plusOne(digits: list[int]) -> list[int]:
for i in range(len(digits) - 1, -1, -1):← Walk right-to-left: ones → tens → hundreds …
if digits[i] < 9: ← This digit won't overflow when incremented
digits[i] += 1 ← Add 1; result is at most 9 — no carry
return digits ← Carry dies here; all digits left are unchanged
digits[i] = 0 ← 9 + 1 = 10 → write 0, carry 1 continues left
return [1] + digits ← Only reached if ALL digits were 9 → new leading 1
⚠️ Why [1] + digits at the end?

If we reach the final return, every digit in the array was 9 and has been set to 0. The carry is still active but there are no more digits. We create a new most-significant digit of 1 and prepend it. E.g. [9,9] → after loop: [0,0] → [1,0,0].

Solution 2 — Explicit Carry Variable (More Readable, Same Complexity)

def plusOne(digits: list[int]) -> list[int]:
carry = 1 ← Treat the "+1" as an initial carry of 1
for i in range(len(digits) - 1, -1, -1):← Right-to-left scan
total = digits[i] + carry ← Sum of current digit and incoming carry
digits[i] = total % 10 ← Ones digit of sum (e.g., 10 % 10 = 0)
carry = total // 10 ← Carry out (e.g., 10 // 10 = 1, 9 // 10 = 0)
if carry == 0: break ← Optimization: stop early if no carry
if carry: ← Carry survived entire array → new leading digit
return [carry] + digits
return digits
💡 Complexity Summary

Time O(n): In the worst case (all 9s), we traverse the entire array once.
Space O(1): Modified in-place. The all-9s edge case creates a new length-(n+1) array, but that's the output, not auxiliary space.
Average case O(1): Most real numbers don't end in 9; we return after just one iteration.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

"Array of digits, add 1" is a math-on-arrays problem. The keyword is carry propagation — the same mechanism that makes long addition work on paper. Don't over-think: simulate what you'd do by hand.

Step 2: Walk Through Examples Aloud

"For [1,2,3]: last digit 3 < 9, increment to 4, done — [1,2,4]. For [9,9,9]: every digit causes carry, all become 0, need a new leading 1 — [1,0,0,0]." Mentioning both cases shows you've thought about edge cases.

Step 3: Describe the Early Exit

"I'll walk right-to-left. If the current digit is less than 9, I just increment it and return — carry can't propagate further. If it's 9, I set it to 0 and continue. If the loop finishes, all digits were 9 → prepend 1."

Step 4: State Complexity Explicitly

"Worst case O(n) time for an all-9s array. Otherwise O(1) average. Space is O(1) in-place, except the all-9s edge case where output grows to n+1."

🗣️ What to Say to the Interviewer

"I'll simulate grade-school addition. Walk from the rightmost digit left.
If the digit is less than 9, increment by 1 and return immediately — carry stops there.
If it's a 9, set it to 0 (because 9+1=10, ones digit is 0) and carry 1 leftward.
Special case: if we finish the whole loop without returning, all digits were 9.
The array is now all zeros — prepend a 1.
Time: O(n) worst case. Space: O(1) in-place."

7 Summary & Key Takeaways ▾
Traverse right-to-left: ones place first, carry flows leftward
If digit < 9: increment by 1, return early — carry absorbed, done
If digit == 9: set to 0 and continue — carry 1 passes to the left
All-9s case: after the loop, digits is all zeros — prepend 1
O(n) worst case (all 9s), O(1) average, O(1) auxiliary space
