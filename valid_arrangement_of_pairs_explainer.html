LeetCode #2097

Valid Arrangement of Pairs

Arrange pairs [start, end] so each pair's end equals the next pair's start — find an Eulerian path in a directed graph.

Hard Graph Hierholzer's Algorithm Eulerian Path Time: O(V+E) Space: O(V+E)
1 Understanding the Problem ▾

You are given a list of pairs, for example [[5,1],[4,5],[11,9],[9,4]]. Your task is to reorder them so they form a chain: the second element of each pair must equal the first element of the next pair.

Think of it like a chain of dominoes. Each domino [a, b] must connect to the next domino that starts with b. You must use every domino exactly once.

💡 Key Insight — Model as a Directed Graph

Each pair [a, b] becomes a directed edge from node a to node b. The chaining constraint means we need a path that visits every edge exactly once. In graph theory, this is called an Eulerian Path. Hierholzer's Algorithm finds it in O(V + E) time.

2 The Algorithm: Hierholzer's Eulerian Path ▾

Hierholzer's Algorithm (1873) finds an Eulerian path in a directed graph — a path that traverses every edge exactly once. It works in O(V + E) by using iterative DFS with a key insight: append nodes to the result when you hit a dead end, then reverse.

🔑 Eulerian Path Existence Conditions

A directed graph has an Eulerian path if and only if: (1) At most one node has out-degree − in-degree = 1 (the start). (2) At most one node has in-degree − out-degree = 1 (the end). (3) All other nodes have equal in- and out-degree. The problem guarantees these hold.

1 Build adjacency list: For each pair [a, b], append b to graph[a]. Also track out-degree and in-degree for every node encountered.
2 Find the start node: Scan all nodes. If any has out-degree − in-degree = 1, that's the Eulerian path start. If none found (all balanced), start at any node with edges — it's an Eulerian circuit.
3 Iterative DFS (Hierholzer's core): Initialize stack = [start], path = []. While stack is non-empty: peek at the top node. If it has unused edges, follow the next one (pop from adj list → push neighbor). If stuck (no edges), pop node off stack → append to path.
4 Reverse and convert: path is built in reverse order (we append during backtracking). Reverse it. Then convert consecutive node pairs [v0,v1,v2,...] to output pairs [[v0,v1],[v1,v2],...] and return.
⚠️ Why Iterative Instead of Recursive?

With up to 2×10⁵ pairs, a recursive DFS can hit Python's recursion limit (default 1000). The iterative version uses an explicit stack and handles any input size safely. The logic is identical — we're just simulating the call stack manually.

3 Why It Works: The Invariant ▾
🔐 The Hierholzer Invariant

When we hit a dead end (a node with no remaining outgoing edges), this node occupies its correct final position in the Eulerian path relative to all paths currently on the DFS stack. Appending it to the result path during backtracking — and reversing at the end — assembles the complete path in correct order.

Why "append on stuck" gives the right answer: Consider a node X that sits in the middle of an Eulerian path. When we reach X and its sub-path is exhausted, the nodes we collected after X (during earlier recursion) are all "downstream" of X. They belong later in the path, so they were already appended before X. When we reverse, X ends up before them — exactly correct.

Why every edge is visited: Each edge [a, b] is stored in graph[a] and consumed exactly once via pop(). Hierholzer's is complete — it only stops when the stack is empty. Since the graph is connected (problem guarantee) and every node has balanced degrees (except the start/end), the algorithm cannot stop early; it must exhaust all edges.

💡 Intuition: Post-Order = "Record on the Way Back"

Think of exploring a maze. When you hit a dead end, you know you've completed a sub-path from the last junction. You record this dead-end node (it comes last from that junction), then backtrack. This is exactly the post-order traversal intuition: process a node after all paths from it have been explored.

Named Algorithm Deep-Dive — Hierholzer's Algorithm (1873):

Origin: Carl Hierholzer, German mathematician, published posthumously in 1873.
Class of problems: Eulerian path/circuit in directed or undirected graphs.
Core invariant: A node is added to the result when it has no more outgoing unused edges. This ensures we never leave a node "stranded" — every sub-circuit is properly merged.
Time complexity: O(V + E) — each edge is processed (followed and popped) exactly once.
Recognize when: "Arrange items so X's end equals Y's start" or "use every edge/transition exactly once."
4 The Code (Annotated) ▾

Solution 1 — Hierholzer's Algorithm (Optimal, O(V+E), Interview Pick)

from collections import defaultdict
def validArrangement(pairs):
graph = defaultdict(list) ← adj list: node → [neighbors still to visit]
indeg = defaultdict(int) ← in-degree per node
outdeg = defaultdict(int) ← out-degree per node
for s, e in pairs: ← build graph in O(n)
graph[s].append(e) ← directed edge s→e
outdeg[s] += 1 ← s emits one more edge
indeg[e] += 1 ← e receives one more edge
start = pairs[0][0] ← default start (overridden below if imbalanced node found)
for node in outdeg: ← search for node where out > in
if outdeg[node] - indeg[node] == 1: ← Eulerian path start condition
start = node; break ← found it; at most one such node exists
path, stack = [], [start] ← result (built reversed) + DFS stack
while stack: ← iterate until all edges consumed
node = stack[-1] ← peek top (don't pop yet)
if graph[node]: ← unused edges remain from this node
stack.append(graph[node].pop()) ← consume edge, push neighbor (O(1) pop)
else: ← dead end: no more outgoing edges
path.append(stack.pop()) ← backtrack: this node goes into result now
path.reverse() ← path built backwards; reverse for correct order
return [[path[i], path[i+1]] ← consecutive nodes form the output pairs
for i in range(len(path)-1)]
⚠️ Common Mistake: Wrong Start Node

If you always start from pairs[0][0] without checking degrees, you may start from a non-Eulerian-start node. This can create disconnected sub-paths. Always check for the node with out-degree − in-degree = 1. If none exists, any node with outgoing edges is valid (Eulerian circuit case).

Solution 2 — Backtracking / Brute Force (O(n!), TLE for large inputs)

def validArrangement_brute(pairs):
def backtrack(chain, remaining):
if not remaining: return chain ← all pairs placed successfully
cur_end = chain[-1][1] ← the tail value we need to match next
for i, (s, e) in enumerate(remaining):
if s == cur_end: ← this pair can extend the chain
res = backtrack(chain + [[s,e]], remaining[:i]+remaining[i+1:])
if res: return res
return None ← no valid extension found from here
for i in range(len(pairs)): ← try each pair as starting pair
res = backtrack([pairs[i]], pairs[:i]+pairs[i+1:])
if res: return res
💡 Complexity Summary

Hierholzer's (Optimal): Time O(V+E) ≈ O(n), Space O(V+E) — each edge processed once via pop().
Brute Force: Time O(n!), Space O(n) — tries all permutations; impractical beyond ~12 pairs.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Reframe as a Graph Problem

"Each pair [a, b] is a directed edge a→b. The constraint 'end of one pair = start of next' means we need a path through all edges — that's an Eulerian path. Recognizing this mapping from domain to graph problem is the hardest part."

Step 2: Verify Eulerian Path Conditions

"Check degrees. At most one node with out−in=1 (start), at most one with in−out=1 (end), all others balanced. The problem guarantees these hold, so I don't need to verify — but I should mention the check."

Step 3: Apply Hierholzer's Algorithm

"Build adjacency list. Find start. Iterative DFS: while stack non-empty — if top has edges, follow one (pop from adj list, push neighbor); if stuck, pop node to result. Reverse result. Convert to pairs."

Step 4: Justify Complexity

"Graph has V ≤ 2n nodes and E = n edges. Building graph: O(n). Finding start: O(V). DFS: O(V+E) since each edge is popped exactly once. Converting to pairs: O(n). Total: O(V+E) ≈ O(n). Space O(V+E) for graph and stack."

🗣️ What to Say to the Interviewer

"Let me model this as a graph: each pair [a,b] is a directed edge a→b.
The chaining requirement means I need an Eulerian path — visit every edge exactly once.
First I'll check degrees to find the start node: look for out−in=1.
Then I'll run Hierholzer's iterative DFS: follow edges until stuck, pop to result, reverse at end.
Time O(V+E), Space O(V+E). This is essentially the Reconstruct Itinerary algorithm."
🔮 Likely Follow-up Questions
  • "What if no valid arrangement exists?" → Check Eulerian conditions first. If violated, return [].
  • "Reconstruct Itinerary (#332)?" → Identical! But sort adjacency lists for lexicographic order.
  • "What if there are duplicate pairs?" → Fine — each pair is a separate edge; duplicates create multi-edges, handled correctly by the list-based adjacency.
  • "Undirected version?" → All degrees must be even for Eulerian circuit; odd-degree nodes are endpoints.
