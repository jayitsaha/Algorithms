LeetCode #867

Transpose Matrix

Given a 2D integer array matrix, return the transpose of matrix — the matrix flipped over its main diagonal, switching the row and column indices of each element.

Easy Matrix Index Mapping Time: O(m×n) Space: O(m×n)
1 Understanding the Problem ▾

The transpose of a matrix reflects it over its main diagonal — the diagonal that runs from the top-left to the bottom-right corner. Every element at row i, column j moves to row j, column i.

💡 Key Insight

Transpose is simply an index swap: position (i, j) in the input maps to position (j, i) in the output. The entire algorithm is this one rule applied to every cell. For an m×n input, the output is n×m — dimensions flip.

Original (3×2)

matrix[0][0]=1 matrix[0][1]=2
matrix[1][0]=3 matrix[1][1]=4
matrix[2][0]=5 matrix[2][1]=6
3 rows × 2 cols

Transposed (2×3)

result[0][0]=1 result[0][1]=3 result[0][2]=5
result[1][0]=2 result[1][1]=4 result[1][2]=6
2 rows × 3 cols
Dimensions flipped!

The exact mapping for every element of the 3×2 example:

Source (i, j)ValueDestination (j, i)
(0, 0)1(0, 0)
(0, 1)2(1, 0)
(1, 0)3(0, 1)
(1, 1)4(1, 1)
(2, 0)5(0, 2)
(2, 1)6(1, 2)

Edge cases: 1×1 matrix (result is same 1×1), single row input (result is a single column), single column input (result is a single row). All handled identically by the same loop.

2 The Algorithm: Direct Index Mapping ▾

We allocate a fresh output matrix with swapped dimensions, then iterate over every cell of the input and copy it to the swapped position. No tricky pointer logic — just the one-rule mapping.

1 Read dimensions: m = len(matrix) (rows), n = len(matrix[0]) (cols). The result will be n rows × m cols.
2 Allocate output: result = [[0] * m for _ in range(n)]. Creates an n×m grid filled with placeholder zeros.
3 Apply the mapping: Nested loops over i ∈ [0, m) and j ∈ [0, n). For each: result[j][i] = matrix[i][j]. Every element is written exactly once.
4 Return result. All m×n assignments complete. The output is the transpose.
🔑 Python Elegance: zip(*matrix)

zip(*matrix) transposes implicitly. The * unpacks rows as separate arguments; zip groups the first element of each row, then the second, etc. — exactly what transposing does. Wrap in [list(row) for row in zip(*matrix)] to get list-of-lists. Same O(m×n) time and space, but requires familiarity with Python's zip semantics to be safe in an interview.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing cell (i, j), result[j][i] holds the value matrix[i][j]. At any point in the nested loop, every cell that has been visited is correctly placed. No cell is visited twice, and no cell is skipped — the nested loop covers all m×n positions exactly once.

Mathematical guarantee: The transpose is defined as T[j][i] = A[i][j] for all valid i ∈ [0, m) and j ∈ [0, n). Our algorithm implements exactly this definition. Since the loop visits every (i, j) pair, every cell of result is set, and because each (i, j) is unique the destination (j, i) is also unique — no overwrite occurs.

Why a new matrix is necessary for non-square inputs: If the input is 3×2 and you tried to modify it in-place, you'd need to write to (0, 2) — but the original matrix only has 2 columns! The dimensions truly change. For a square n×n matrix (m==n), in-place swapping across the upper triangle works because the dimensions do not change.

⚠️ Common Mistake: Allocating the Wrong Shape

It's tempting to write [[0]*n for _ in range(m)] — same shape as input. The result must be n rows by m columns: [[0]*m for _ in range(n)]. The first index into result is j (which runs to n), and the second is i (which runs to m).

4 The Code (Annotated) ▾

Solution 1 — Direct Index Mapping (Optimal, Interview Pick)

def transpose(matrix: List[List[int]]) -> List[List[int]]:
m, n = len(matrix), len(matrix[0]) ← m = rows, n = cols of input
result = [[0] * m for _ in range(n)] ← n rows × m cols (dimensions flipped!)
for i in range(m): ← iterate each row of input
for j in range(n): ← iterate each col of input
result[j][i] = matrix[i][j] ← swap indices: source(i,j) → dest(j,i)
return result ← all n×m cells assigned exactly once
⚠️ Why result[j][i], not result[i][j]?

We are reading from matrix[i][j] and writing to result[j][i]. The indices are swapped. Think of it as: the row coordinate in the result is the column coordinate from the source, and vice versa. Writing to result[i][j] would just copy the matrix unchanged.

Solution 2 — zip one-liner (Pythonic)

def transpose(matrix: List[List[int]]) -> List[List[int]]:
return [list(row) for row in zip(*matrix)] ← zip groups by column; *matrix unpacks rows

Solution 3 — In-place (Square matrices only, O(1) extra space)

def transpose_square(matrix: List[List[int]]) -> List[List[int]]:
n = len(matrix) ← must be n×n square
for i in range(n): ← row index
for j in range(i + 1, n): ← j > i: upper triangle only, avoid re-swap
matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j] ← swap pair
return matrix
💡 Complexity Summary

Solutions 1 & 2: Time O(m×n) — visit every element once. Space O(m×n) — output matrix.
Solution 3 (square): Time O(n²) — upper triangle only. Space O(1) — no extra matrix. Not applicable for non-square.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify Whether the Matrix Is Square

Ask: "Can the input be non-square (m ≠ n)?" If yes, you need a new output array — dimensions change. If guaranteed square, you can do it in-place. Always handle the general case in an interview unless asked otherwise.

Step 2: State the Core Rule Explicitly

Before writing any code, say: "The transpose maps element (i, j) to position (j, i). I'll allocate an n×m result matrix and apply this mapping." This proves you understand the problem mathematically.

Step 3: Trace One Element by Hand

After writing the loop, say: "Let me verify: matrix[0][1] = 2, so I write to result[1][0] = 2. That's correct." This immediately catches dimension bugs before the interviewer spots them.

Step 4: Mention zip(*matrix) as a Python Bonus

"In Python, [list(row) for row in zip(*matrix)] does this in one line. zip groups by position across iterables — unpacking matrix rows gives column-wise grouping." Shows language fluency without hiding the core idea.

🗣️ What to Say to the Interviewer

"The core operation is an index swap: element at (i,j) goes to (j,i).
Since the matrix can be non-square, the result has n rows and m columns.
I'll allocate a new n×m matrix, then fill it: result[j][i] = matrix[i][j].
Time: O(m×n) — visit each element once. Space: O(m×n) — new output matrix.
For square matrices only, we can do O(1) space by swapping the upper triangle in-place."
🔮 Likely Follow-up Questions
  • "Rotate 90° clockwise?" → Transpose the matrix, then reverse each row (#48)
  • "Rotate 90° counter-clockwise?" → Transpose, then reverse each column
  • "Can you do it with O(1) space?" → Only for square matrices; upper triangle swap

🎯 When to Recognize the Matrix Index Swap Pattern
  • "Flip rows and columns" or "reflect over main diagonal" → transpose (i,j) ↔ (j,i)
  • "Rotate 90° clockwise" → transpose + reverse each row
  • "Rotate 90° counter-clockwise" → transpose + reverse each column
  • "Rotate 180°" → reverse each row + reverse each column
  • Any problem asking you to change which dimension is "outer" vs "inner"
