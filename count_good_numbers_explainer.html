LeetCode #1922

Count Good Numbers

Count digit strings of length n where even-indexed positions hold even digits and odd-indexed positions hold prime digits. Return the count modulo 10⁹+7.

Medium Math Fast Exponentiation Time: O(log n) Space: O(1)
1 Understanding the Problem ▾

A "good number" is a digit string of exactly n characters where every position must satisfy a rule based on its index:

  • Even positions (index 0, 2, 4, …): digit must be even → valid choices: 0, 2, 4, 6, 8 → 5 options
  • Odd positions (index 1, 3, 5, …): digit must be prime → valid choices: 2, 3, 5, 7 → 4 options
💡 Key Insight

Each position's digit choice is completely independent of every other position. When n independent choices exist, the total count is their product. So the answer is 5^(even positions) × 4^(odd positions). The only challenge is computing this for massive n (up to 10¹⁵) without looping n times.

✓ n = 4 example

Index 0 (even): 5 choices
Index 1 (odd): 4 choices
Index 2 (even): 5 choices
Index 3 (odd): 4 choices
Total = 5×4×5×4 = 400

✗ Why naive loop fails

n can be 10¹⁵
Loop 10¹⁵ times ≈ TLE
At 3×10⁸ ops/sec:
10¹⁵ / 3×10⁸ ≈ 3M seconds

How to count positions: In a 0-indexed string of length n, even indices are 0, 2, 4, …. The count of even positions is ⌈n/2⌉ = (n+1)//2. Odd positions 1, 3, 5, … total ⌊n/2⌋ = n//2. They sum to n ✓.

Edge cases: n=1 → only even position (index 0) → 5 good strings. The constraint "n up to 10¹⁵" is a direct signal that you need a logarithmic algorithm, not a linear one.

2 The Algorithm: Binary (Fast) Exponentiation ▾

Once we establish that the answer is 5^e × 4^o mod M, we need fast modular exponentiation to compute x^n mod M in O(log n) time. The key idea: instead of multiplying x by itself n times, we square the base repeatedly and only absorb a factor into the running result when the corresponding bit of the exponent is set.

📐 The Core Recurrence
pow(x, n) = 1 if n = 0 (base case)
pow(x, n) = pow(x², n//2) if n is even
pow(x, n) = x · pow(x², n//2) if n is odd
1 Count positions: even_count = (n+1)//2, odd_count = n//2. For n=4: e=2, o=2.
2 Initialize fast-exp: result=1, base=x, exp=n. The invariant result × base^exp = x^n_original holds from the start.
3 While exp > 0: if exp is odd, multiply result by base (absorb one factor). Then square base and halve exp regardless. Each iteration halves exp → O(log n) iterations total.
4 Return (pow(5, e, M) × pow(4, o, M)) % M. Python's built-in pow(x, n, mod) does exactly this in C — use it directly in interviews.

Visualising the bit trick (why it's O(log n)): The exponent's binary representation has O(log n) bits. We process one bit per iteration — squaring the base for each bit, multiplying result when that bit is 1.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every iteration of the while loop, result × base^exp = original x^n. We start with result=1, base=x, exp=n (so 1×x^n = x^n ✓). Each step preserves this product while moving factors from base^exp into result or into a larger base. When exp=0, base^0=1, so result × 1 = result = x^n.

Step-by-step preservation proof:

  • Before iteration: invariant is result × base^exp = x^n
  • If exp is odd: we do result' = result × base, exp' = exp−1 (even). Then: result' × base^(exp') = (result × base) × base^(exp−1) = result × base^exp = x^n ✓
  • Then we square: base' = base², exp' = exp'//2. So result' × (base')^(exp') = result' × (base²)^(exp/2) = result' × base^exp' ✓

Why modular arithmetic is safe at every step: The distributive law of modular arithmetic states (a × b) mod M = ((a mod M) × (b mod M)) mod M. So taking mod at every multiplication gives the same final answer as taking mod once at the very end — but keeps intermediate numbers bounded by M² instead of growing astronomically large.

⚠️ Critical: Mod the Base When Squaring

Write base = base * base % MOD — not base = base * base. Without mod, after 50 squarings, base becomes 5^(2⁵⁰), a number with roughly 34 trillion digits. Python handles big integers but this is catastrophically slow. Always mod at each multiplication.

Counting positions — proof: In 0-indexed string length n, even indices {0,2,4,…} have count ⌈n/2⌉ = (n+1)//2. Odd indices {1,3,5,…} have count ⌊n/2⌋ = n//2. Sum = (n+1)//2 + n//2 = n ✓ (handles both even and odd n correctly).

Solution 1 — Python built-in pow (Optimal, Interview Pick)

def countGoodNumbers(n: int) -> int:
MOD = 10**9 + 7 ← Standard LeetCode modulus; prevents huge integers
even_count = (n + 1) // 2← Number of even-indexed positions (0,2,4,…) = ceil(n/2)
odd_count = n // 2 ← Number of odd-indexed positions (1,3,5,…) = floor(n/2)
return ( ← Multiply two modular exponents, take mod of product
pow(5, even_count, MOD) * ← 5 choices per even position; O(log n) binary exp
pow(4, odd_count, MOD) ← 4 choices per odd position; O(log n) binary exp
) % MOD ← Final mod for the product of two values each < MOD
⚠️ Never call pow(5, even_count) without MOD argument

Python's pow(x, n) (two-arg form) computes the exact integer first. pow(5, 5×10¹⁴) is a number with ~3×10¹⁴ digits — Python handles it but it's astronomically slow. The three-argument pow(x, n, mod) applies mod at every squaring step, keeping numbers bounded.

Solution 2 — Manual Binary Exponentiation (know the internals)

def fast_pow(x: int, n: int, mod: int) -> int:
result = 1 ← Accumulator; starts at multiplicative identity
x %= mod ← Reduce base once upfront (safe since (a*b)%m = ((a%m)*(b%m))%m)
while n > 0: ← Process each bit of the exponent
if n % 2 == 1: ← If current LSB is 1, this base^(2^k) contributes to answer
result = result * x % mod ← Absorb this factor into result, take mod
x = x * x % mod ← Square base: x → x², x² → x⁴, x⁴ → x⁸, ... (next bit's contribution)
n //= 2 ← Shift right: process next higher bit; log₂(n) iterations total
return result ← Invariant: result × x^n = original x^n_orig; n=0 → result = answer
def countGoodNumbers(n: int) -> int:
MOD = 10**9 + 7
e, o = (n + 1) // 2, n // 2
return fast_pow(5, e, MOD) * fast_pow(4, o, MOD) % MOD
🔬 Algorithm Deep-Dive: Binary / Fast Exponentiation

Origin & Names: Also called "exponentiation by squaring," "binary exponentiation," or "fast power." Known since ancient Indian mathematics (Pingala, ~200 BCE) for computing powers via repeated doubling. Formalized for computer science in the context of modular arithmetic and RSA cryptography (1970s).

Core invariant: result × base^exp = original x^n at every iteration. We redistribute factors from the right side into result without changing the product's value.

Why O(log n): Each iteration halves the exponent. Starting at n, after k iterations we have n/2^k. When n/2^k < 1, k = log₂(n). For n=10¹⁵: log₂(10¹⁵) ≈ 50 iterations.

Generalization: Works for any associative operation with an identity. Matrix multiplication → compute Matrix^n in O(log n) matrix multiplications → Fibonacci in O(log n). Polynomial multiplication. Modular inverse computation (Fermat's little theorem).

Recognize it when: "Compute x^n mod M" with large n (n > 10⁶); "k-th Fibonacci" for large k; "number of ways to tile n×m grid" via matrix exponentiation; any problem combining "combinatorics" + "mod" + "large exponent."

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Apply the Multiplication Principle

Observe that each position is independent. Draw out a small example (n=3): index 0 has 5 options, index 1 has 4, index 2 has 5 → 5×4×5=100. Generalize: answer = 5^(even positions) × 4^(odd positions). This is just combinatorics — no algorithm needed yet.

Step 2: Spot the Constraint Signal

n can be up to 10¹⁵. You cannot loop n times. The constraint "n up to 10¹⁵" + "return mod 10⁹+7" is a direct signal: use modular fast exponentiation. This pattern appears in probably 80% of LeetCode "Math" hard/medium problems with "mod" in the description.

Step 3: Write or invoke fast pow

In Python: pow(x, n, mod) — done. In Java/C++: write the iterative loop (result=1, while n>0: if odd absorb, square, halve). Verify n=1: (n+1)//2=1, n//2=0 → pow(5,1,M)×pow(4,0,M) = 5×1 = 5 ✓. Verify n=4: pow(5,2,M)×pow(4,2,M) = 25×16 = 400 ✓.

Step 4: Verify edge cases aloud

n=1: only even position → 5. n=2: e=1,o=1 → 5×4=20. n=0 (if allowed): e=0,o=0 → 1 (empty string — technically valid). Confirm with interviewer whether n≥1 is guaranteed.
