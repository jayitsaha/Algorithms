LeetCode #2461

Maximum Sum of Distinct Subarrays With Length K

Given an integer array nums and an integer k, find the maximum subarray sum among all subarrays of length exactly k with all distinct elements. Return 0 if no such subarray exists.

Medium Sliding Window Sum + Distinct Check Time: O(n) Space: O(k)
1 Understanding the Problem ▾

You are given an array nums and an integer k. You need to look at every contiguous window of exactly k elements. Among those windows where every element is distinct (no repeated values within the window), find the one whose elements have the largest sum. Return that sum, or 0 if no valid window exists.

💡 Key Insight

Two conditions must hold simultaneously: the window must be exactly k wide, and every element in it must be unique. A fixed-size sliding window handles the first constraint for free. We just need a way to efficiently answer "does the current window contain any duplicates?" as the window slides. A frequency map (value → count) lets us maintain this in O(1) per slide.

✓ Valid window example

nums = [1, 5, 4, 2, 9, 9, 9], k=3
Window [4, 2, 9] → all distinct
Sum = 4+2+9 = 15 ← answer
Window [5, 4, 2] → sum=11, valid ✓

✗ Invalid window (duplicate)

Window [2, 9, 9] → 9 appears twice
Sum=20 but INVALID (duplicate)
Window [9, 9, 9] → triple duplicate
Cannot count any window with 9,9

Edge Cases: (1) If k = 1, every single element is valid — return the maximum. (2) If every window of size k has a duplicate, return 0. (3) If all elements are the same, only a window of k=1 is valid. (4) If nums has fewer than k elements, no valid window exists — return 0.

2 The Algorithm: Fixed-Size Sliding Window + Frequency Map ▾

We maintain a window of exactly size k and slide it across the array. At each slide, we add the incoming right element and remove the outgoing left element. We track the window sum in a running variable (O(1) update per slide) and maintain a frequency map of elements in the window to detect duplicates.

1 Initialize: freq = {}, current_sum = 0, max_sum = 0, left pointer l = 0.
2 Expand right (for each r): Add nums[r] to current_sum and increment freq[nums[r]]. The window is now nums[l..r].
3 Shrink if over-sized: When window size r − l + 1 > k, remove nums[l]: decrement its freq (delete key if it hits 0), subtract from current_sum, advance l.
4 Check validity: When window size = k exactly, check len(freq) == k. If true, every element in the window is distinct (each appears exactly once). Update max_sum.
5 Return: After the full scan, return max_sum (stays 0 if no valid window was found).
🔑 Why len(freq) == k detects all-distinct

After removing zero-count entries, each key in freq represents a value that is actually present in the window. The window has size k. So if len(freq) == k, there are k distinct values in a window of k elements — every element is unique. If any value appears more than once, it takes up one key but contributes more than one to the window size count, so len(freq) < k.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every point during the scan, current_sum equals the exact sum of nums[l..r], and freq[v] equals the number of times value v appears in that same window. After removing zero-count keys, len(freq) equals the number of distinct values in the window. These invariants are maintained by every add and remove operation.

Why the invariant is maintained: Every time r advances, we correctly add the new element. Every time the window grows past k, we precisely remove the oldest element. Deleting the key when its count hits 0 keeps len(freq) accurate as the count of currently-present distinct values.

Why no window is missed: The right pointer r visits every index exactly once. The left pointer l only advances (never retreats). Each element is added once (when r reaches it) and removed once (when it falls off the left). So every subarray of length k is checked exactly once. Total work: O(n).

⚠️ The Subtle Bug: Using a Plain Set Instead of a Freq Map

If you used a set and removed nums[l] when sliding, you'd get wrong answers. Example: nums=[3,3,5], k=2. Window at l=1, r=2 is [3,5] — valid! But if l=0 had removed 3 from the set while 3 still exists at index 1, the set would incorrectly say "3 is not in window". The frequency map tracks how many copies of each value are in the window, so removal only clears the key when truly gone.

4 The Code (Annotated) ▾

Solution 1 — Sliding Window + Frequency Map (Optimal — Interview Pick)

def maximumSubarraySum(nums, k): ← inputs: the array and the required window size
freq = {} ← freq map: value → how many times it's in current window
current_sum = 0 ← running sum of all elements in nums[l..r]
max_sum = 0 ← best valid-window sum seen (0 = "no valid window yet")
l = 0 ← left pointer: start index of current window
for r in range(len(nums)): ← r = right pointer; grows the window by 1 each iteration
freq[nums[r]] = freq.get(nums[r],0)+1 ← add nums[r] to freq (handles new key gracefully)
current_sum += nums[r] ← update running sum: add new element
if r - l + 1 > k: ← window grew past k → must remove leftmost element
freq[nums[l]] -= 1 ← decrement count of the element leaving the window
if freq[nums[l]] == 0: ← if count hits 0, element fully gone from window
del freq[nums[l]] ← delete key so len(freq) stays accurate (critical!)
current_sum -= nums[l] ← subtract the removed element from running sum
l += 1 ← slide left pointer one step right
if r-l+1 == k and len(freq) == k: ← window is exactly k AND all k elements are distinct
max_sum = max(max_sum, current_sum)← valid window! update best answer
return max_sum ← returns 0 if no valid window was ever found
⚠️ Why must we delete the key when freq hits 0?

We rely on len(freq) to count distinct elements. If we leave zero-count entries in the dict, they inflate len(freq) and cause us to incorrectly think windows are all-distinct when they're not. Example: window [3,5] with freq={3:0, 5:1} would give len(freq)=2=k even though 3 is gone. Deleting the key keeps the count exact.

Solution 2 — Brute Force (O(n·k)) — for comparison

def maximumSubarraySum(nums, k): ← O(n·k) — check each window independently
max_sum = 0
for i in range(len(nums) - k + 1): ← iterate over all possible start positions
window = nums[i:i+k] ← O(k) slice: extract window
if len(set(window)) == k: ← O(k) check: all elements distinct?
max_sum = max(max_sum, sum(window))← O(k) sum — recomputed from scratch each time
return max_sum ← correct but wastes work on overlapping prefixes

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern Signal

"Contiguous subarray of exactly length k" is the canonical signal for a fixed-size sliding window. The fixed size means we don't need to dynamically shrink — we just keep removing from the left whenever the window exceeds k. The distinctness constraint is an additional invariant we maintain alongside the sum.

Step 2: Propose Brute Force First

"A naive approach: for each starting index, extract the k-element window, check if it has all-distinct elements, and sum it. That's O(n·k). Inefficient because adjacent windows share k-1 elements — we're recomputing the sum and distinctness from scratch when we could update them in O(1)."

Step 3: Optimize — what to maintain as the window slides

Two things to maintain: (1) the current_sum — add right element, subtract left element, O(1) per slide. (2) a frequency map — add right element, remove left element (delete key at 0). Check distinctness via len(freq) == k in O(1).

Step 4: Address the freq map detail proactively

Interviewers love this detail: "I use a freq map instead of a plain set because when the same value appears at position l AND somewhere inside the window, removing it from a set would incorrectly mark it absent. The freq map tracks how many copies are present — I only delete the key when the count hits 0."

🗣️ What to Say to the Interviewer

"I see 'subarray of exactly length k' — that's a fixed sliding window.
I'll maintain two things as I slide: current_sum (add right, subtract left, O(1))
and a frequency map to track element counts in the window.
When the window is exactly size k, I check len(freq)==k: that means all k elements
are distinct. If so, I update max_sum.
Time O(n) — each element added and removed exactly once. Space O(k) for the freq map."

7 Summary & Key Takeaways ▾
Fixed-size sliding window: "contiguous subarray of exactly length k" → slide one step at a time, add right, remove left.
Use a frequency map (not a set) to handle correct removal: only mark a value absent when its count drops to 0.
Distinctness check is O(1): compare len(freq) == k. When all k elements are unique, each has count=1 and the dict has exactly k keys.
Initializing max_sum = 0 handles the "no valid window" case automatically — no special case needed.
Each element is added once and removed once → O(n) total time, O(k) space.
🎯 Recognize This Pattern When You See
  • "Contiguous subarray/substring of exactly length k" → fixed sliding window
  • "Maximum/minimum aggregate over all length-k windows" → maintain running aggregate, update O(1) per slide
  • "All elements distinct" / "no repeats" → freq map in window, check len(freq) == window_size
  • O(n·k) brute force → think "what overlaps between adjacent windows can I reuse?"
