LeetCode #474

Ones and Zeroes

Find the largest subset of binary strings where the total zeros ≤ m and total ones ≤ n.

Medium Dynamic Programming 2D Knapsack Time: O(L·m·n) Space: O(m·n)
1 Understanding the Problem ▾

You receive a list of binary strings such as ["10","0001","111001","1","0"] and two capacity limits: m (max zeros) and n (max ones). Your goal is to find the largest number of strings you can select from the list such that their combined zeros do not exceed m and their combined ones do not exceed n.

💡 Key Insight

This is a 2D 0/1 Knapsack problem. Classic knapsack has one resource (weight). Here you have two independent resources: a budget of zeros and a budget of ones. Each string "costs" some zeros and some ones. You want to fit the maximum number of items within both budgets simultaneously — you cannot reuse a string and you cannot exceed either limit.

✓ Example: m=1, n=1, strs=["10","0","1"]

"10" → 1 zero, 1 one
"0" → 1 zero, 0 ones
"1" → 0 zeros, 1 one

Pick "0"+"1" → 1 zero ✓, 1 one ✓
Answer: 2

✗ Cannot pick all three

"10"+"0"+"1" = 2 zeros, 2 ones
Exceeds m=1 zeros ✗
Exceeds n=1 ones ✗

"10"+"0" = 2 zeros ✗
Greedy won't find optimal.

Edge cases to consider: strings of all zeros (only spend zero-budget), strings of all ones, m=0 or n=0 (can only pick strings with zero cost in that dimension), a single string that already exceeds both budgets (pick 0), and the empty string (costs nothing, always pickable).

2 The Algorithm: 2D Bottom-Up Knapsack ▾

We maintain a 2D DP table where dp[i][j] answers: "Using at most i zeros and j ones from the strings seen so far, what is the maximum number of strings I can include?" We process strings one at a time and update the table in reverse order.

1 Initialize: Create a 2D array dp of shape (m+1) × (n+1) filled with zeros. With no strings considered, every budget yields 0 selected strings.
2 Count each string: For each string s, count its zeros (z = s.count('0')) and ones (o = s.count('1')). These are its costs in each dimension.
3 Update in reverse: For i from m down to z, and j from n down to o: set dp[i][j] = max(dp[i][j], dp[i-z][j-o] + 1). The reverse traversal prevents using this string more than once.
4 Read answer: After all strings are processed, dp[m][n] holds the maximum subset size within the full budget (m zeros, n ones).

🔑 Why Reverse Traversal Prevents Double-Counting

If we looped forward (i from z to m), then when computing dp[i][j], the cell dp[i-z][j-o] might already include the current string from an earlier update in this same round — effectively adding it twice. By scanning backward (i from m down to z), every dp[i-z][j-o] we read is still from the previous round (before this string), so each string can only be picked at most once.

3 Why This is DP: Recurrence, Invariant & Proof ▾

Two pillars of DP are satisfied here:

✓ Optimal Substructure

The maximum subset for budget (i, j) is built from the maximum subset for the smaller budget (i−z, j−o) if we take the current string. The optimal solution to the big problem contains optimal solutions to smaller sub-problems.

✓ Overlapping Subproblems

A naive recursion would solve "best count for budget (i, j) from strings 0..k" repeatedly for the same (i, j, k) triplets. With memoization or tabulation, each state is computed exactly once.

📐 The Recurrence Relation
Define dp[i][j] = max strings selectable using ≤ i zeros and ≤ j ones

Base case: dp[i][j] = 0 for all i, j   (no strings seen yet)

For each string with cost (z zeros, o ones), update (reverse i, j):
  dp[i][j] = max( dp[i][j],  dp[i−z][j−o] + 1 )   ∀ i ≥ z, j ≥ o

Answer: dp[m][n]
🔐 The Invariant

After processing the first k strings, dp[i][j] holds the maximum number of strings that can be selected from strings 1..k using at most i zeros and j ones. Each string either improves some dp cells (by being taken) or leaves them unchanged (by being skipped). No cell ever decreases. At the end (k = L), dp[m][n] gives the global answer.

Why greedy fails: A greedy approach — pick the cheapest string first (fewest zeros, or fewest total characters) — doesn't work. Consider m=2, n=2, strs=["00","11","01","10"]. Greedy by zeros first would pick "11" (0z, 2o), then "01" (1z, 1o), then "10" (1z, 1o) = 3 strings. But "01"+"10"+"11" = 2z+2o = 3 strings is also achievable. Optimal is 3 here. But slight variations where greedy picks the wrong item early can cut off better combinations — only DP exhaustively considers all options without repeated computation.

4 The Code (Annotated) ▾

Solution 1 — Bottom-Up Tabulation (Interview Pick)

def findMaxForm(strs, m, n):
dp = [[0]*(n+1) for _ in range(m+1)] ← (m+1)×(n+1) table, all zeros. dp[i][j] = max strings with ≤i zeros, ≤j ones
for s in strs: ← Process each string one at a time (each is either taken or skipped)
z = s.count('0') ← Zero-cost of this string
o = s.count('1') ← One-cost of this string
for i in range(m, z-1, -1): ← Reverse from m down to z; i<z can't afford this string anyway
for j in range(n, o-1, -1): ← Reverse from n down to o; same reason
dp[i][j] = max( ← Compare: skip this string…
dp[i][j], ← …(keep old value — did not take string)
dp[i-z][j-o] + 1) ← …vs. take it: best count with (i-z) zeros and (j-o) ones budget, plus 1
return dp[m][n] ← Max strings fitting within full budget (m zeros, n ones)
⚠️ Why range(m, z-1, -1) and NOT range(m, -1, -1)?

We stop at z (inclusive) rather than 0 because if i < z, we cannot afford to take this string (not enough zero-budget). There's no point updating those cells — they would read dp[i-z][j-o] with a negative index and produce garbage. Stopping at z is both a correctness guard and a constant-factor speedup.

Solution 2 — Top-Down Memoization (easier to derive from the recurrence)

from functools import lru_cache
def findMaxForm(strs, m, n):
counts = [(s.count('0'),s.count('1')) for s in strs] ← Precompute all (z, o) pairs once
@lru_cache(maxsize=None)
def dp(idx, rem_z, rem_o): ← State: which string we're at, remaining zero/one budgets
if idx == len(counts): return 0 ← Base: no strings left → count is 0
z, o = counts[idx]
skip = dp(idx+1, rem_z, rem_o) ← Option A: skip, budget unchanged
take = 0
if rem_z >= z and rem_o >= o: ← Can we afford this string?
take = 1 + dp(idx+1, rem_z-z, rem_o-o) ← Option B: take it, reduce both budgets
return max(skip, take) ← Best of skip vs take
return dp(0, m, n) ← Start from string 0, full budgets
💡 Complexity Summary

Tabulation: Time O(L × m × n) — L strings, each triggers O(m × n) table updates. Space O(m × n) — only the 2D table.
Memoization: Time O(L × m × n) — same unique states. Space O(L × m × n) — cache + recursion stack depth O(L).
Winner: Tabulation for space efficiency. L can be up to 600, m and n up to 100 each, so m×n = 10,000 cells — very manageable.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Knapsack Shape

The problem structure is: "for each item (string), choose to take or skip it. You have limited resources. Maximize count (or value)." This is exactly 0/1 Knapsack. The twist: two resource dimensions instead of one. That turns the 1D dp array into a 2D dp table.

Step 2: Define the State

State = "remaining zero-budget and remaining one-budget." With these two numbers, you can make the take-or-skip decision for any string independently of which strings you already chose. Two-dimensional state → 2D table dp[i][j].

Step 3: Write the Recurrence and Base Case

dp[i][j] = max(dp[i][j], dp[i-z][j-o] + 1). Base case: all zeros (no strings processed → 0 selected). Scan strings one at a time to convert this into an iterative loop.

Step 4: Justify Reverse Traversal

Say: "I traverse i from m down to z and j from n down to o. This guarantees dp[i-z][j-o] is from the previous iteration — before this string was processed — so each string is counted at most once." This is the core 0/1 knapsack trick.

🗣️ What to Say to the Interviewer

"This is a 2D 0/1 Knapsack. Each string is an item with two costs: its zero-count and its one-count.
I need the maximum number of items (strings) fitting within budget m zeros and n ones.
I define dp[i][j] = max strings selectable from strings seen so far using at most i zeros and j ones.
For each string (z zeros, o ones): dp[i][j] = max(dp[i][j], dp[i-z][j-o] + 1).
I update in reverse — i from m to z, j from n to o — so each string is counted once.
Final answer: dp[m][n]. Time O(L·m·n), Space O(m·n)."
🔮 Likely Follow-up Questions
  • "What if strings can be reused?" → Unbounded knapsack variant: iterate i forward (0→m), j forward (0→n) instead
  • "Prove the reverse traversal works." → Show dp[i-z][j-o] is unchanged in the current round when i iterates from m downward
  • "What if you want to maximize total string length instead of count?" → Change +1 to +len(s) in the recurrence
  • "Can you solve with recursion + cache?" → Yes — top-down memoization, same asymptotic complexity

7 Summary & Key Takeaways ▾
2D Knapsack: two resource budgets → 2D DP table instead of the classic 1D array
State dp[i][j] = max strings pickable from strings-seen-so-far within i zeros, j ones
Recurrence: dp[i][j] = max(dp[i][j], dp[i-z][j-o] + 1)
Reverse traversal (i: m→z, j: n→o) is the 0/1 trick — prevents double-counting each string
Space O(m·n) — the 2D table only; no per-item storage beyond z/o counts
ApproachTimeSpaceNotes
Brute force (all 2^L subsets) O(2^L · L) O(L) TLE for L=600
Memoization (top-down) O(L·m·n) O(L·m·n) Cache grows with L
Tabulation (bottom-up) ✓ O(L·m·n) O(m·n) Optimal — interview pick
🎯 When to Recognize 2D / Multi-Dimensional Knapsack
  • "Select a subset" + "stay within two independent budgets simultaneously"
  • Each item has two cost dimensions (here: zero-count, one-count)
  • You optimize a count or total value, not just feasibility
  • One budget → 1D dp. Two budgets → 2D dp. K budgets → K-dimensional dp.
