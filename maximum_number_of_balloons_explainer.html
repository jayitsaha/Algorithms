Maximum Number of Balloons

LeetCode #1189 · Form "balloon" as many times as possible
Easy Hash Table Frequency Count O(n) Time O(1) Space
1 Understanding the Problem ▼

Given a string text, we want to form the word "balloon" as many times as possible using the characters from text. Each character can only be used once.

First, let's examine "balloon":

b×1
a×1
l×2
o×2
n×1

Required characters: b(1), a(1), l(2), o(2), n(1)

💡 Key Insight

The answer is limited by the scarcest resource. If you have 100 'b's but only 2 'l's, you can only form 1 "balloon" (since 'l' appears twice in "balloon", 2÷2=1).

Example: For text = "nlaebolko":
Counts: b=1, a=1, l=2, o=2, n=1 → Can form 1 "balloon"

Example: For text = "loonbalxballpoon":
Counts: b=2, a=2, l=6, o=4, n=2 → Min ratio: l=6÷2=3, o=4÷2=2, others=2 → Answer: 2

2 The Algorithm: Frequency Count + Min Ratio ▼

This is a frequency counting problem with a twist: we need the minimum ratio across all required characters.

1 Count frequencies: Scan through text and count occurrences of only b, a, l, o, n (ignore other characters).
2 Calculate ratios: For each required character, compute: count[char] ÷ required[char] (integer division).
3 Return minimum: The answer is the smallest ratio — the bottleneck character.

🔑 The Formula

answer = min(b÷1, a÷1, l÷2, o÷2, n÷1)

Think of it like a recipe: if you need 2 eggs per cake but only have 5 eggs, you can make at most 2 cakes (5÷2=2), regardless of how much flour you have.

3 Why It Works: The Invariant ▼
🔐 The Invariant

Each "balloon" consumes exactly b(1), a(1), l(2), o(2), n(1). We can form at most count[c]/required[c] instances limited by character c. The minimum across ALL characters is the true maximum.

Proof sketch:

  • Upper bound: We can't form more than min(ratios) because at least one character would run out.
  • Achievability: We CAN form exactly min(ratios) because all characters have at least that many "servings".
⚠️ Don't Forget!

'l' and 'o' appear twice in "balloon"! A common mistake is treating all characters equally. Make sure to divide by 2 for these.

Edge cases:

  • Missing character: If any of b,a,l,o,n has count=0 → answer is 0
  • Empty string: No characters → answer is 0
  • No relevant chars: e.g., "xyz" → answer is 0
4 The Code (Annotated) ▼
# Frequency Count + Min Ratio — O(n) time, O(1) space
def maxNumberOfBalloons(text: str) -> int:
# Count only the characters we need
count = {'b': 0, 'a': 0, 'l': 0, 'o': 0, 'n': 0}
for c in text:
if c in count:
count[c] += 1
# Calculate minimum ratio (l and o need 2 each)
return min(
count['b'],
count['a'],
count['l'] // 2, # Need 2 l's per balloon
count['o'] // 2, # Need 2 o's per balloon
count['n']
)

Alternative using Counter:

from collections import Counter
def maxNumberOfBalloons(text: str) -> int:
count = Counter(text)
balloon = Counter("balloon") # {'b':1,'a':1,'l':2,'o':2,'n':1}
return min(count[c] // balloon[c] for c in balloon)
💡 Space Complexity

We only store counts for 5 specific characters, so space is O(1) regardless of input size. Even if using Counter, at most 26 letters are stored.

Step 1: Analyze the Target Word

"Let me first break down 'balloon': b(1), a(1), l(2), o(2), n(1). Notice l and o appear twice."

Step 2: Identify the Pattern

"This is a resource allocation problem. Each 'balloon' consumes a fixed set of resources. The answer is limited by whichever resource runs out first."

Step 3: Propose the Solution

"I'll count frequencies, then find the minimum ratio of (available / needed) for each character."

Step 4: Handle Edge Cases

"If any required character is missing, the answer is 0. Integer division handles the 'l' and 'o' doubling naturally."

🗣️ What to Say

"This is a frequency counting problem with a twist. I'll count the characters b, a, l, o, n in the input. Then the answer is the minimum of: count[b], count[a], count[l]÷2, count[o]÷2, count[n]. The division by 2 for 'l' and 'o' accounts for them appearing twice in 'balloon'. Time O(n), space O(1)."

Follow-up they might ask:

  • "What if the target word is given as input?" → Generalize with Counter("target")
  • "What if we have multiple target words?" → Count each separately, return total
