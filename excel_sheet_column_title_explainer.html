LeetCode #168

Excel Sheet Column Title

Given an integer columnNumber, return its corresponding column title as it appears in an Excel spreadsheet (A, B, C, …, Z, AA, AB, …).

Easy Math Base 26 Conversion Time: O(log₂₆ n) Space: O(log₂₆ n)
1 Understanding the Problem ▾

Excel names its columns with letters: A=1, B=2, …, Z=26, then AA=27, AB=28, …, AZ=52, BA=53, … This looks like base-26, but there is a crucial twist: there is no zero digit. Every position must hold a letter from A (1) to Z (26).

💡 Key Insight: Bijective Numeral System

In normal base-10, digits are 0–9 and '10' means "one ten, zero ones." In Excel's system, digits are 1–26 (A–Z) and there is no "zero ones" position. When you reach Z (26), the next value is AA (27), not "Z0". This is called a bijective base-26 system, and it requires a special decrement trick during conversion.

2 The Algorithm: Bijective Base Conversion with Decrement Trick ▾

Standard base conversion repeatedly divides by the base and collects remainders. For bijective base-26 we do the same, but fix the "no zero digit" problem by decrementing n by 1 before each division. This temporarily shifts the 1-indexed system (A=1) to 0-indexed (A=0) so that standard modulo works correctly, then we shift back by mapping 0 → 'A'.

🔑 The Core Trick: n -= 1 Before Modulo

Without the trick: 26 % 26 = 0 → no letter for 0 → wrong.
With the trick: (26-1) % 26 = 25 → chr(ord('A') + 25) = 'Z' → correct!
This single n -= 1 is the entire difference between a buggy and a correct solution.

1 While n > 0: Keep extracting digits as long as there are more to process. Each iteration extracts one letter (rightmost first).
2 Decrement: n -= 1. Shift to 0-indexed internally. Now A=0, B=1, …, Z=25. This makes modulo safe.
3 Extract digit: remainder = n % 26. Gives 0–25. Map to letter: chr(ord('A') + remainder). Append to result list (building right-to-left).
4 Advance: n = n // 26. Integer-divide away the digit we just extracted. Repeat from step 2 if n > 0.
5 Reverse and join. Since we collected letters from right-to-left (least significant first), reverse the list and join into a string.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the beginning of each loop iteration, n represents the number of Excel columns still to be encoded (in 1-indexed bijective form). After n -= 1, it temporarily becomes 0-indexed. The expression n % 26 then always yields a valid letter index 0–25 (never a meaningless 0 for the "Z" case). After n //= 26, the encoded digit is consumed and n once again represents the remaining higher-order columns in 1-indexed form, ready for the next iteration.

Mathematical proof sketch: In bijective base-26, the value of a k-letter string c₁c₂…cₖ (left to right, each letter value 1–26) is Σᵢ₌₁ᵏ value(cᵢ) × 26^(k-i). To reverse this, extract the rightmost letter: value(cₖ) = ((n-1) mod 26) + 1, then set n = (n - value(cₖ)) / 26 = (n-1) // 26. Our code computes exactly this — the chr(ord('A') + ...) maps ((n-1)%26)+1 to the letter in one step.

Why the loop terminates: Each iteration performs n //= 26 (after the decrement). Since we always reduce n by at least 1 before dividing, and divide by 26, n strictly decreases each iteration. Starting from any finite positive integer, we reach 0 in ⌈log₂₆(n)⌉ steps.

💡 Analogy: Bijective Odometer

Imagine an odometer where each wheel shows letters A–Z (not 0–9). After ZZZZ, it rolls over to AAAAA — there's no "0 wheel." When you "carry" from Z, instead of becoming 0 and carrying 1, you eliminate that wheel and the next wheel advances by 1. The n -= 1 trick simulates this bijective carry behavior using standard integer division.

4 The Code (Annotated) ▾

Solution 1 — Iterative (Interview Pick, O(log₂₆ n))

def convertToTitle(columnNumber: int) -> str:
result = [] ← Collect letters LSB-first; reverse at end
n = columnNumber ← Working copy; we will mutate this
while n > 0: ← Loop until all digits extracted
n -= 1 ← KEY TRICK: shift 1-indexed → 0-indexed for safe modulo
remainder = n % 26 ← Extract rightmost digit (0=A, 1=B, ..., 25=Z)
result.append(chr(ord('A') + remainder)) ← Convert digit to letter ('A'+0='A', 'A'+25='Z')
n //= 26 ← Discard the digit just extracted; process remaining
return ''.join(reversed(result)) ← Letters were built right-to-left; reverse for final order
⚠️ Why append + reverse, not prepend?

Prepending to a Python list (list.insert(0, x)) is O(n) per call because it shifts all elements. Appending is O(1) amortized. For a result of length k, append+reverse costs O(k) total vs O(k²) for repeated prepend. Alternative: accumulate as result = chr(...) + result string — correct but also O(k²) due to string immutability.

Solution 2 — Recursive (Elegant, same complexity)

def convertToTitle(n: int) -> str:
if n == 0: ← Base case: no more digits to add
return ""
return convertToTitle((n - 1) // 26) + chr(ord('A') + (n - 1) % 26)
# ↑ Higher-order part (recurse) ↑ Rightmost letter (current)

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

This is a base conversion problem — you're mapping integers to a fixed-size alphabet positionally, just like converting decimal to hexadecimal. The pattern signal: "repeat until exhausted, extracting one digit per iteration via modulo."

Step 2: Identify the Bijective Twist

Test with n=26: expected "Z". Standard base-26 would give "10" (one 26, zero ones). That's wrong. This reveals: no zero digit exists. After Z comes AA, not ZA or Z0. The smallest column number is 1 (not 0), and every digit position holds 1–26, never 0.

Step 3: Derive the Fix

For standard modulo to work, you need 0-indexed values. The bijective system is 1-indexed. Fix: shift it by 1 before each modulo. n -= 1 makes A=0, B=1, …, Z=25. Now n % 26 always gives a valid 0–25 index. After extraction, n //= 26 discards the digit. The decrement happens fresh at each iteration, so the invariant holds every time.

Step 4: Handle the Order

Remainders come out least-significant first (rightmost letter first). Collect them in a list with O(1) appends, then reverse once at the end. This is always O(k) vs O(k²) for repeated string prepend.

🗣️ What to Say to the Interviewer

"This is a base conversion, but Excel's system is bijective base-26: A=1 through Z=26 with no zero digit.
If I naively do columnNumber % 26, I get 0 when the column is 'Z', and there's no letter for 0.
The fix: before each modulo, I subtract 1 — this shifts the system to 0-indexed temporarily.
So: n -= 1, then remainder = n % 26 gives 0–25, map to A–Z with chr(ord('A') + remainder).
Then n //= 26 discards that digit. Repeat while n > 0.
Collect letters right-to-left (LSB first), reverse at the end.
Time: O(log₂₆ n), Space: O(log₂₆ n) for the result — at most 7 chars for 32-bit input."
🔮 Likely Follow-up Questions
  • "Reverse it — title to number?" → LeetCode #171: Horner's rule left-to-right, each letter contributes value × 26^position
  • "Maximum output length for 32-bit input?" → ⌈log₂₆(2³¹−1)⌉ = 7 characters (FXSHRXW is the largest)
  • "Can you do it recursively?" → Yes: convertToTitle((n-1)//26) + chr(ord('A')+(n-1)%26), base case n=0 returns ""
  • "What if digits started at 0?" → Drop the n-=1 trick; use standard modulo directly

7 Summary & Key Takeaways ▾
Excel columns use bijective base-26: digits are A–Z (1–26), no zero exists
The critical fix: n -= 1 before each n % 26 shifts the 1-indexed system to 0-indexed
Without n -= 1, n=26 gives remainder 0 (no letter) instead of 'Z' — the classic trap

🎯 When to Recognize the Bijective Base Conversion Pattern
  • Converting integers to alphabetical sequences where the smallest symbol maps to 1 (not 0)
  • Numbering systems where "there is no empty digit" — like after Z comes AA, not Z0
  • Any problem where repeated modulo gives 0 but 0 has no valid representation
  • Signal phrase: "A=1, B=2, …, Z=26, AA=27, …" — that's the bijective signature
