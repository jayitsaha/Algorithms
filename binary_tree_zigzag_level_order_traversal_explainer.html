LeetCode #103

Binary Tree Zigzag Level Order Traversal

Given the root of a binary tree, return the zigzag level order traversal of its nodes — alternating left-to-right then right-to-left at each successive level.

Medium Trees · BFS Alternate Direction Time: O(n) Space: O(n)
1 Understanding the Problem ▾

This is level order traversal (BFS) with a twist: every other level gets its values listed in reverse. Level 0 (root) goes left-to-right, Level 1 right-to-left, Level 2 left-to-right, and so on — forming a zigzag "snake" pattern down the tree.

💡 Key Insight

The nodes are visited in standard BFS order every single time. The only thing that changes level-by-level is how we store each node's value: either appending to the end or inserting at the front of the current level list. A single boolean toggle left_to_right is all we need to control this.

Level 1: [9, 20] L→R (wrong!)

[[3],[9,20],[15,7]]

Edge cases: empty tree → []; single node → [[val]]; skewed trees (all left or all right children); balanced complete trees where the bottom level has up to ⌈n/2⌉ nodes.

2 The Algorithm: BFS with Direction Toggle ▾

We run standard BFS using a deque. The outer while loop processes exactly one level per iteration. Within each level, we snapshot the queue size before enqueuing any children — this is the critical trick that separates levels cleanly. A boolean flag alternates the storage direction.

1 Handle empty input: if root is None, return [].
2 Initialize: result=[], queue=deque([root]), left_to_right=True.
3 Snapshot level size: level_size = len(queue). This freezes how many nodes are on the current level before any children enter the queue.
4 Process current level: Loop exactly level_size times. Each iteration: pop from the left (popleft()), append or prepend the value based on the flag, then enqueue the node's left and right children if they exist.
5 Commit and toggle: append the completed level list to result, then flip left_to_right = not left_to_right. Repeat from step 3.
🔑 Why deque, Not list?

list.pop(0) is O(n) — it shifts every element left. collections.deque.popleft() is O(1). For n nodes that's the difference between O(n²) and O(n) total work on queue operations. Always import deque for BFS.

🔑 insert(0, val) vs level.reverse()

Two valid ways to handle R→L levels: (1) level.insert(0, val) as each node is popped — streams values in final order without a second pass. (2) Collect all values normally, call level.reverse() once at the end of the level. Both are O(n) overall; option 1 is slightly more elegant in interviews.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At the start of each outer-loop iteration, the deque contains exactly and only the nodes of the next unprocessed level, in left-to-right tree order. After level_size pops and their associated child enqueues, the deque will contain exactly the nodes of the following level — the invariant is restored.

Why level_size is the critical safeguard: As we process level k, we enqueue level k+1 nodes into the same deque. Without snapshotting the count first, the inner loop would "fall through" into level k+1 nodes — mixing two levels in one result sublist. The snapshot creates a hard boundary.

Why the toggle guarantees correct ordering: Nodes are always enqueued in left-then-right order (by BFS traversal). For L→R levels, we append each popped value — preserving the left-to-right order. For R→L levels, we insert each popped value at position 0 — the first popped (leftmost) gets shifted rightward by subsequent inserts, ending up at the end. The last popped (rightmost) stays at position 0. This produces a right-to-left sequence automatically.

⚠️ Classic Bug: Inner While Instead of Snapshot

Using while queue: as the inner loop instead of for _ in range(level_size): will process ALL remaining nodes in one pass — producing [[3, 9, 20, 15, 7]] instead of three separate levels. The level_size snapshot is non-negotiable.

Null children: if node.left: queue.append(node.left) — we only enqueue non-null children. This correctly handles unbalanced trees with missing nodes.

4 The Code (Annotated) ▾

Solution 1 — BFS with Direction Toggle (Interview Pick, O(n) time)

from collections import deque ← O(1) popleft — required for efficient BFS
def zigzagLevelOrder(root):
if not root: return [] ← Empty tree edge case: immediately return []
result = [] ← Final output: list of level lists
queue = deque([root]) ← BFS queue seeded with the root node
left_to_right = True ← True → append; False → insert at 0
while queue: ← Outer loop: one full iteration = one tree level
level_size = len(queue) ← SNAPSHOT before enqueuing children
level = [] ← Accumulates this level's values
for _ in range(level_size): ← Inner loop: process exactly current level
node = queue.popleft() ← O(1) deque pop from front
if left_to_right:
level.append(node.val) ← L→R: add to end, preserving order
else:
level.insert(0, node.val)← R→L: prepend reverses order on the fly
if node.left: ← Enqueue children in L-then-R order always
queue.append(node.left)
if node.right:
queue.append(node.right)
result.append(level) ← Commit completed level to output
left_to_right = not left_to_right← Toggle for next level
return result
⚠️ insert(0, val) has O(k) cost per call

Each level.insert(0, val) shifts existing elements — O(k) where k is the current level list length. Over an entire level of size m, this is O(1)+O(2)+...+O(m) = O(m²) in theory. However, since all levels sum to n nodes total, the amortized cost stays O(n). For large trees, prefer deque for the level list and use appendleft() — or just reverse the list once at the end.

Solution 2 — BFS with End-of-Level Reverse (cleaner, same complexity)

def zigzagLevelOrder(root):
if not root: return []
result, queue, left_to_right = [], deque([root]), True
while queue:
level_size = len(queue)
level_nodes = [queue.popleft() for _ in range(level_size)]
level_vals = [n.val for n in level_nodes] ← always L→R
if not left_to_right: level_vals.reverse() ← single O(m) reverse for R→L levels
result.append(level_vals)
for node in level_nodes:
if node.left: queue.append(node.left)
if node.right: queue.append(node.right)
left_to_right = not left_to_right
return result
💡 Complexity Summary

Time O(n): Each of the n nodes is enqueued once, popped once, and its value stored once.
Space O(n): The deque holds at most one full level at a time. The widest level of a complete binary tree is the bottom level with ⌈n/2⌉ nodes → O(n). The result list is also O(n) total.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognise "Level Order" → BFS

"The problem says level order — that's BFS. I need to process the tree one level at a time. The twist is just the alternating direction, which I can control with a flag."

Step 2: Propose brute force first

"I could do standard BFS to collect all levels, then reverse every odd-indexed sublist. That's O(n) time and O(n) space, correct but two passes. Let me show a cleaner single-pass approach."

Step 3: Introduce the toggle flag

"Keep a left_to_right boolean. On True levels, append each node's value; on False levels, insert at position 0. Either way, children are always enqueued left-then-right. The flag flips after every level."

Step 4: Emphasize the level-size snapshot

"The subtle but critical point: I snapshot level_size = len(queue) before the inner loop. This separates the current level from the next level's children which I'm about to enqueue."

🗣️ What to Say to the Interviewer

"This is level-order BFS with a direction toggle.
I'll use collections.deque for O(1) popleft operations.
Each outer loop iteration handles one level: snapshot level_size first,
then pop that many nodes, then enqueue their children.
If left_to_right, append values; else insert at 0.
Flip the flag after each level. Time O(n), Space O(n)."
🔮 Likely Follow-up Questions
  • "Can you solve it with DFS?" → Yes: use recursion with a depth parameter; collect into result[depth] and reverse odd-depth lists at end.
  • "What's the maximum queue size?" → Bottom level of a complete tree: ⌈n/2⌉ nodes → O(n).
  • "Why not list.pop(0)?" → O(n) per pop vs O(1) for deque.popleft — would make overall O(n²).
  • "What if you have an n-ary tree?" → Same BFS pattern; just enqueue all children, not just two.

7 Summary & Key Takeaways ▾
Zigzag = standard BFS with a toggle: append on L→R levels, prepend (insert at 0) on R→L levels
Always snapshot level_size = len(queue) before the inner loop — this is what separates levels cleanly
Children are enqueued left-then-right every time; only the value storage order changes
Use collections.deque for O(1) popleft — do NOT use list.pop(0) which is O(n)
Space O(n): the bottom level of a complete tree holds ⌈n/2⌉ nodes — this is the max queue width
ApproachTimeSpaceNotes
BFS + reverse odd levels O(n) O(n) Two passes per level; simpler to explain
BFS + toggle (insert at 0) ✓ O(n) O(n) One pass, streaming order; interview pick
DFS with depth + reverse O(n) O(h) O(h) call stack; harder to reason about order
