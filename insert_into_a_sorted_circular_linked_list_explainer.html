LeetCode #708

Insert into a Sorted Circular Linked List

Given a node in a sorted circular linked list, insert a value so the list stays sorted — handling the wrap-around and edge cases.

Medium Linked List Find Correct Position Time: O(n) Space: O(1)
1 Understanding the Problem ▾

You are given a reference to any node in a sorted circular singly linked list. Insert a new node with value insertVal such that the list remains sorted in non-descending order. Return any node in the modified list.

💡 Key Insight

A sorted circular list has exactly one descending adjacent pair: the wrap-around from the maximum value back to the minimum. This single "seam" defines three distinct cases for where a new value can be inserted. Master these three cases and the problem is solved.

Think of the values arranged around a clock face in sorted order. You can start reading anywhere on the clock — the sorted property means values increase until they wrap around at the "seam" (from 12 back to 1). Inserting a new value means finding the correct position on the clock.

Case 1 — Normal Gap
curr.val ≤ insertVal
AND insertVal ≤ curr.next.val
The new value fits naturally between two adjacent nodes on the ascending side. Both curr and curr.next are on the same side of the seam.
Case 2 — Wrap-Around
curr.val > curr.next.val
AND (insertVal ≥ curr.val
OR insertVal ≤ curr.next.val)
curr is the maximum. The new value is either a new max (goes after curr) or a new min (goes before curr.next). Both extremes share this insertion point.
Case 3 — Full Circle
curr.next == head
(looped back to start)
All nodes have the same value, or insertVal equals all of them. No case matched after a full traversal. Insert anywhere — list stays sorted.
Before: Insert 4 into [1→3→5→1...]

List: 1 → 3 → 5 → (back to 1)
Wrap-around: 5 → 1 is the seam
Insert 4: fits between 3 and 5 (Case 1)

After: [1→3→4→5→1...]

List: 1 → 3 → 4 → 5 → (back to 1)
Seam still at: 5 → 1
Sorted order maintained ✓

Tricky Edge Cases: Empty list (create a self-looping single node), list with all equal values (e.g. [3,3,3] — insert anywhere), inserting a new maximum (7 into [1,3,5]), inserting a new minimum (0 into [1,3,5]), and a single-element list.

2 The Algorithm: Circular Traversal with Three Cases ▾

We traverse the circular list once with a curr pointer. At each step we check whether insertVal belongs between curr and curr.next. Three distinct conditions cover all possible insertion points. If none fires after a complete loop, we insert at the current position anyway (Case 3).

1 Handle empty list: If head is None, create a new node that points to itself (new_node.next = new_node) and return it. A single self-looping node is a valid sorted circular list.
2 Initialize curr = head. We will advance curr each iteration. We start at whatever node the caller gave us — the circular structure means all starting points are equivalent.
3 Case 1 — Normal ascending gap: If curr.val ≤ insertVal ≤ curr.next.val, the new value fits between these two adjacent nodes. Break and insert here.
4 Case 2 — Wrap-around seam: If curr.val > curr.next.val (curr is the maximum, curr.next is the minimum), then check if insertVal is a new extreme: insertVal ≥ curr.val (new max) OR insertVal ≤ curr.next.val (new min). Both go here. Break.
5 Case 3 — Full circle: If curr.next == head, we've traversed the entire list without finding a case. Insert after curr regardless — the list has all-equal values or insertVal equals all of them.
6 Wire the new node: new_node.next = curr.next, then curr.next = new_node. This splices the new node between curr and curr.next. Always set new_node.next FIRST to avoid losing the chain reference.
🔑 Why Exactly Three Cases Cover Everything

In a sorted circular list, every adjacent pair is either non-descending (the normal ascending side) or descending (the unique wrap-around seam). The new value must fall into one ascending pair (Case 1), or at the seam if it's a new extreme (Case 2), or anywhere if all values are equal (Case 3). These three categories are exhaustive and mutually exclusive — no fourth case exists.

3 Why It Works: The Invariant ▾
🔐 The Invariant

A sorted circular linked list has at most one descending adjacent pair (the seam from max to min). Every other adjacent pair is non-descending. Inserting between curr and curr.next preserves the sorted property if and only if insertVal is bounded by the range of that pair — or if that pair is the seam and insertVal is a new extreme.

Why Case 2 handles both new-max and new-min at the same seam: At the seam, curr.val is the list's current maximum and curr.next.val is the current minimum. A value larger than the maximum becomes the new maximum — it extends the ascending run just before the seam, so it goes after curr. A value smaller than the minimum becomes the new minimum — it goes just before curr.next. Both cases use the same insertion point: between curr and curr.next at the seam.

Proof that Case 3 is always safe: If no case matches after a full loop, then for every adjacent pair, neither Case 1 nor Case 2 fired. Case 2 fires whenever a descending pair is found. If no descending pair was found, the list is entirely non-descending — meaning all values are equal. Inserting any equal value anywhere keeps the list non-descending.

⚠️ Common Bug: Wrong Loop Termination

Many implementations use while curr.next != head, stopping just before checking the seam edge between the last node and head. This misses the descending pair! The correct approach is while True: with a break on curr.next == head as Case 3. This ensures the seam between the last node and head is always evaluated.

Example — insert 7 (new max) into [1→3→5→1...]: At node 5: curr.val=5 > curr.next.val=1 — seam detected. Is 7 ≥ curr.val=5? Yes. Insert between 5 and 1 → list becomes [1→3→5→7→1...]. The seam is now from 7 back to 1. Sorted order preserved.

Example — insert 0 (new min) into [1→3→5→1...]: At node 5: curr.val=5 > curr.next.val=1 — seam. Is 0 ≤ curr.next.val=1? Yes. Insert between 5 and 1 → list becomes [1→3→5→0→1...]. Wait — 0 is now before 1 in the circular direction, making it the new minimum. Sorted ✓.

4 The Code (Annotated) ▾

Solution 1 — Single-Pass Traversal, Three Cases (Optimal · Interview Pick)

def insert(head, insertVal: int):
new_node = Node(insertVal) ← create the new node upfront (wire it at the end)
if not head: ← edge case: empty list
new_node.next = new_node ← self-loop: a single-node circular list
return new_node ← return it as the new list
curr = head ← start traversal from the given node (any node is valid)
while True:
# Case 1: value fits in a normal ascending gap
if curr.val <= insertVal <= curr.next.val:
break ← found: insertVal belongs between curr and curr.next
# Case 2: curr is the max (wrap-around seam)
if curr.val > curr.next.val: ← descending edge = the seam from max to min
if insertVal >= curr.val or \
insertVal <= curr.next.val:
break ← new max (≥ curr.val) or new min (≤ curr.next.val)
# Case 3: completed full circle without match
if curr.next == head: ← back to start: all values equal, insert anywhere
break
curr = curr.next ← advance to the next node in the circle
# Wire the new node in between curr and curr.next
new_node.next = curr.next ← MUST do this first — saves the old curr.next reference
curr.next = new_node ← now redirect curr to the new node → splice complete
return head ← return any node; the list is still circular
⚠️ Wiring Order is Critical

Always new_node.next = curr.next BEFORE curr.next = new_node. If you do it in reverse order, curr.next now points to new_node, and new_node.next = curr.next creates a self-loop — you've permanently lost the reference to the rest of the list.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Understand the Circular Structure

Ask: "The list is circular and sorted — so there's no natural head or tail, and there's a wrap-around from the max back to the min. Can the list be empty? Can values repeat?" These questions surface the three edge cases before you even start coding.

Step 2: Enumerate the Three Cases Verbally

Say explicitly: "I see three cases for where the new value can go: a normal ascending gap, the seam between max and min (handling both new-max and new-min), and the all-same-value fallback." Naming the cases shows structured thinking and prevents missed edge cases.

Step 3: Handle Empty List First

Always handle the empty list edge case with an early return before the main loop. Creates the node, makes it self-loop, return. Two lines. Shows you read the problem statement carefully.

Step 4: Code the Wiring Last and Carefully

State the order out loud: "First I'll save curr.next into new_node.next, then redirect curr.next. The order matters to avoid losing the chain." If the interviewer allows, draw a small diagram showing the three pointers before and after.

🗣️ What to Say to the Interviewer

"I'll handle empty list first — create a self-looping node and return it.
Otherwise, I traverse with a curr pointer. At each adjacent pair (curr, curr.next), I check:
Case 1: curr.val ≤ insertVal ≤ curr.next.val → normal gap, insert here.
Case 2: curr.val > curr.next.val (seam) AND insertVal is a new max or min → insert here too.
Case 3: curr.next == head (full circle) → all values equal, insert anywhere.
After the break, wire: new_node.next = curr.next first, then curr.next = new_node.
Return head. Time O(n), Space O(1)."
🔮 Likely Follow-up Questions
  • "What if all nodes have the same value?" → Case 3 fires after a full loop. Insert after head. List stays sorted.
  • "What if insertVal equals the min or max exactly?" → Case 1 handles it (≤, not <). No special handling needed.
  • "How would you delete a value from a sorted circular list?" → Same circular traversal, but track prev so you can wire around the target node.
  • "Can you do it without a while True loop?" → Yes: use while curr.next != head but process the seam edge after the loop. Slightly more code but avoids the infinite-loop concern.

7 Summary & Key Takeaways ▾
A sorted circular list has exactly one descending adjacent pair — the wrap-around seam from max to min
Three cases cover all insertions: normal ascending gap, seam extremes, and all-same fallback
Wire new_node.next = curr.next BEFORE curr.next = new_node — order prevents losing the chain
Empty list is a special case: create a single node pointing to itself (self-loop)
At the seam, both new-max and new-min go between curr (max) and curr.next (min)
