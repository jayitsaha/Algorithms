LeetCode #336

Palindrome Pairs

Given a list of unique words, return all ordered pairs (i, j) such that the concatenation words[i] + words[j] is a palindrome.

Hard Tries Trie with Palindrome Check Time: O(n·k²) Space: O(n·k)
1 Understanding the Problem ▾

You have a list of unique words. Find every ordered pair (i, j) where i ≠ j and words[i] + words[j] (concatenation) is a palindrome — a string that reads identically forwards and backwards.

💡 Key Insight

For A + B to be a palindrome, the strings must "mirror" each other. Specifically, after splitting word A at every position, if the right part is already a palindrome, then we just need the reverse of the left part to exist as another word in the list — and that word goes before A. Similarly, if the left part is a palindrome, the reverse of the right part can go after A. This 2-case per split decomposition is exhaustive.

✓ Concrete Example

words = ["abcd","dcba","lls","s","sssll"]

"abcd"+"dcba" = "abcddcba" ✓
"dcba"+"abcd" = "dcbaabcd" ✓
"s"+"lls" = "slls" ✓
"lls"+"sssll" = "llssssll" ✓

→ answer: [[0,1],[1,0],[3,2],[2,4]]

The Three Structural Cases

Case 1 (equal length):
rev(B) = A exactly
"abcd" + "dcba"

Case 2 (B shorter):
A has palindrome suffix,
rev(B) = A's prefix
"s" + "lls": rev("s")="s" in "lls"

Case 3 (B longer):
B has palindrome prefix
"lls" + "sssll"

⚠️ Critical Distinctions
  • Ordered pairs: (i, j) and (j, i) are both valid if both concat palindromes — don't deduplicate between them
  • Empty string: "" is a palindrome, so any palindrome word P pairs with "" in both directions
  • i ≠ j required: A word cannot pair with itself (even if the word is itself a palindrome)
Empty string edge
words = ["aba",""]
"aba"+"" = "aba" ✓
""+"aba" = "aba" ✓
→ [[0,1],[1,0]]
Self-reverse pair
words = ["abc","cba"]
"abc"+"cba" = "abccba" ✓
"cba"+"abc" = "cbaabc" ✓
→ both directions valid
Length mismatch OK
words = ["s","lls"]
"s"+"lls" = "slls" ✓
"lls"+"s" = "llss" ✗
→ only one direction
2 The Algorithm: Hash Map + Split-and-Check ▾

Instead of checking all O(n²) pairs, we fix one word and derive what the other must be. For each word A, we try every possible split point (0 to len(A) inclusive), giving prefix and suffix. Two cases per split tell us what complement word to look up in a hash map.

1 Build word → index map: Precompute word_map = {word: index} for all words. This enables O(1) lookup: "does word X exist in the list, and at which index?"
2 For each word A at index i: Try all split points j from 0 to len(A). Each split gives left = A[:j] and right = A[j:].
3 Case A — suffix palindrome: If right is a palindrome, we need rev(left) to exist as a different word B. Then B + A = rev(left) + left + right = palindrome (since rev(left)+left mirrors and right is palindrome in the middle). Add pair [word_map[rev(left)], i].
4 Case B — prefix palindrome: If left is a palindrome AND j < len(A) (guard against duplicates), we need rev(right) to exist as a different word B. Then A + B = left + right + rev(right) = palindrome. Add pair [i, word_map[rev(right)]].
🔑 Why the j < len(A) guard in Case B?

When j = len(A): left = A (full word), right = "". Case B would fire if A is a palindrome and rev("") = "" is in word_map. But Case A with j = 0 also fires when left = "" (palindrome) and rev(right) = rev(A) — finding the same pair. The guard j != len(A) prevents this duplicate. Case A with j=len(A) (empty suffix) already covers the full-word-reverse case.

3 Why It Works: The Invariant ▾
🔐 The Core Invariant

For any two words A and B, A + B is a palindrome if and only if either: (1) len(A) ≥ len(B) and A starts with rev(B) and the remaining suffix of A is a palindrome — OR — (2) len(B) > len(A) and B ends with rev(A) and the remaining prefix of B is a palindrome. These two cases cover all possible length relationships.

Formal proof sketch: Let |A| = m, |B| = n. The concatenation C = A + B has length m+n. For C to be a palindrome, C[k] = C[m+n-1-k] for all k.

  • Positions 0..m-1 come from A; positions m..m+n-1 come from B.
  • The "outer rim" positions 0..n-1 (from A's start) must match with positions m+n-1..m (from B's end). This means A[0..n-1] = rev(B). So the first n chars of A must equal the reverse of B.
  • The remaining "inner" part A[n..m-1] is in the middle of the palindrome and must be a palindrome itself.
  • Symmetrically, if |B| > |A|: positions from B's start must match A reversed, and B's remaining middle must be a palindrome.

This is exactly what our split encodes: we split A at position n (=len(B)), check A[n:] (the suffix) is palindrome, and check A[:n] reversed = B.

⚠️ Completeness: All Cases Are Covered

The split-at-every-position approach is exhaustive: we enumerate every possible length relationship between A and B by trying all split sizes from 0 to len(A). Case A covers scenarios where B is shorter or equal (the suffix of A is the palindrome middle). Case B covers scenarios where B is longer (the prefix of A is the palindrome middle, and B extends beyond A). No valid pair can be missed because its corresponding split point is always in range 0..len(A).

Empty string: rev("") = "". The split at j=0 gives left="" (palindrome, trivially), suffix=A. Case A: is A a palindrome? If yes, rev("") = "" must exist in word_map. If "" is in the list, add pair [word_map[""], i]. This correctly handles the "" pairing with any palindrome word.

4 The Code (Annotated) ▾

Solution 1 — Hash Map + 3-Case Decomposition (Optimal, Interview Pick)

def palindromePairs(words: list[str]) -> list[list[int]]:
def is_palin(s: str) -> bool: ← O(k) helper
return s == s[::-1]
# Build word → index lookup (O(nk) space)
word_map = {w: i for i, w in enumerate(words)} ← O(1) lookups
result = []
for i, word in enumerate(words): ← O(n) outer loop
n = len(word)
for j in range(n + 1): ← O(k) splits: 0..n inclusive
left = word[:j] ← word[0..j-1]
right = word[j:] ← word[j..n-1]
# Case A: right side is palindrome
# Pattern: [rev(left)] + [left + right]← rev(left)+word is palindrome
if is_palin(right): ← O(k) check
rev_left = left[::-1]
if rev_left in word_map and word_map[rev_left] != i:← exists & not self
result.append([word_map[rev_left], i])← B before A
# Case B: left side is palindrome
# Pattern: [left + right] + [rev(right)]← word+rev(right) is palindrome
if j != n and is_palin(left): ← j!=n guards against Case A dup
rev_right = right[::-1]
if rev_right in word_map and word_map[rev_right] != i:
result.append([i, word_map[rev_right]])← A before B
return result
⚠️ Why doesn't j != n suffice — why not also j != 0?

At j=0: left="", right=word. Case A: is right (=word) a palindrome? Possibly. If so, we add pair [word_map[""], i] — this is a legitimate new pair (empty string before word). Case B: j=0 ≠ n, is left="" palindrome? Yes. We add [i, word_map[rev(word)]] — also legitimate. So j=0 is fine for both cases. The duplicate risk is only at j=n where Case A and Case B both want to fire on the full-word vs empty scenario.

Solution 2 — Brute Force O(n²·k), for reference

def palindromePairs_brute(words):
result = []
for i in range(len(words)): ← O(n) outer
for j in range(len(words)): ← O(n) inner → O(n²)
if i != j:
cat = words[i] + words[j] ← O(k) concat
if cat == cat[::-1]: ← O(k) palindrome check
result.append([i, j])
return result ← total O(n²k), TLE on large n
💡 Complexity Analysis

Hash Map solution: n words × (k+1) splits × O(k) per palindrome check = O(n·k²). Space O(n·k) for the map.
Brute Force: O(n²·k). If n=10,000 and k=10: hash map = 10⁶ ops, brute force = 10⁹ ops. The hash map wins by 1000x.
Note: k² factor exists because each of the k splits requires an O(k) palindrome check. A suffix array or hashing trick could reduce this to O(k) per word = O(nk) total.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Establish the Brute Force Baseline

"The naive solution: for each of the n² ordered pairs (i,j), concatenate and check if it's a palindrome. O(n²k) time. For n=10,000, k=10 that's 10⁹ operations — too slow. I need to reduce the inner n loop."

Step 2: Fix One Word, Derive the Other

"Instead of checking every pair, I'll fix word A and ask: what specific word B makes A+B a palindrome? If I can answer this in O(k) instead of O(n), the total becomes O(nk) — huge improvement."

Step 3: Discover the Split Insight

"I'll split word A at every position j. Left part = A[:j], right part = A[j:]. If the right part is already a palindrome, then word B must exactly equal rev(A[:j]). Check a hash map in O(1). If left part is a palindrome, word B must equal rev(A[j:]) and B goes after A."

Step 4: Prove Completeness

"Every valid pair (A,B) has some relationship: len(B) ≤ len(A) or len(B) > len(A). The split at j=len(B) handles the first case; the split at j=len(A)-len(B) handles the second. Since j ranges from 0 to len(A), we cover all possible B lengths."

Step 5: Handle the Duplicate Edge Case

"At j=len(A), both Case A (empty suffix, trivially palindrome) and Case B (full word as left) would add the pair where B=rev(A). Guard: only run Case B when j < len(A). Case A at j=len(A) already handles this."

🗣️ What to Say to the Interviewer

"Brute force is O(n²k) — try all pairs, check if palindrome. Too slow for large n.

Better: for each word A, I'll try all O(k) split points. At each split into left+right:
1. If right is a palindrome, I need rev(left) to exist as another word — that word goes before A.
2. If left is a palindrome (and split is not at end), I need rev(right) to exist — that word goes after A.

Precompute a hash map word→index. Total: O(nk) splits × O(k) palindrome check = O(nk²).
Space: O(nk) for the hash map.

Edge cases: empty string pairs with any palindrome word; ordered pairs mean both [i,j] and [j,i] can be valid; guard j≠n in Case B prevents duplicates."
⚠️ Most Common Interview Mistakes
  • Forgetting pairs are ordered — (i,j) and (j,i) are distinct valid answers
  • Missing the empty string case — "" in words creates pairs with all palindrome words
  • Off-by-one in split range — must go 0 to len(word) inclusive (n+1 values)
  • Generating duplicate pairs — forgetting the j != n guard for Case B
  • Using word_map[rev_left] == i to exclude self — must check index, not string equality

7 Summary & Key Takeaways ▾
Every valid palindrome pair (A,B) is captured by exactly one split point within A — the point that divides the "core match" from the "palindrome middle".
A hash map converts the "find the complement word" step from O(n) to O(1), driving the total from O(n²k) to O(nk²).
The j != n guard in Case B is essential — without it, pairs get added twice when j equals the word length.
Empty string "" is handled automatically by the algorithm without special-casing — its "palindrome" check always passes, and its reverse is itself.
Pairs are ordered: algorithm correctly produces both [i,j] and [j,i] when both concatenations are palindromes.
🎯 When to Recognize This Pattern
  • "Find all pairs from a list satisfying a structural property" → fix one element, derive the other
  • Palindrome + multiple words → split-and-check with hash map
  • O(n²) brute force is bottlenecked → use hash map to eliminate inner loop
  • Trie problems → if the complement lookup needs prefix structure, use Trie instead of hash map
