LeetCode #857

Minimum Cost to Hire K Workers

Find the minimum total cost to hire exactly k workers, paying each worker proportionally to their quality while never paying below their minimum wage.

Hard Greedy Sort by Ratio + Heap Time: O(n log n) Space: O(k)
1 Understanding the Problem ▾

There are n workers. Each worker i has a quality quality[i] (a measure of how productive they are) and a minimum wage wage[i] (the least they will accept). You must hire exactly k workers. The payment must satisfy two non-negotiable rules:

Rule 1 — Proportional Pay

All hired workers are paid proportional to their quality. If worker A has twice the quality of worker B, A earns exactly twice as much as B.

pay[i] = r × quality[i] for some shared rate r.

Rule 2 — Minimum Wage Guarantee

No hired worker may be paid less than their stated minimum wage.

r × quality[i] ≥ wage[i]
i.e., r ≥ wage[i] / quality[i]

💡 Key Insight: The Captain Sets the Rate

Since r ≥ wage[i]/quality[i] for every worker in the group, the required rate is the maximum ratio wage[i]/quality[i] across all hired workers. The worker achieving that maximum is the captain — they are paid exactly their minimum wage; everyone else earns a bonus above theirs. So:

Total cost = r × Σ quality[i] where r = max(wage[j]/quality[j]) for j in group

📐 The Math, Step by Step
For any valid group G of size k:
r_min = max{ wage[j]/quality[j] : j ∈ G } (smallest valid rate)
Total cost = r_min × Σ{ quality[j] : j ∈ G }
Goal: minimize this over all C(n,k) possible groups G.

Worked example — quality=[10,20,5], wage=[70,50,30], k=2:

  • Worker 0: ratio = 70/10 = 7.0
  • Worker 1: ratio = 50/20 = 2.5
  • Worker 2: ratio = 30/5 = 6.0
  • Group {W1, W2}: r = max(2.5, 6.0) = 6.0. Cost = 6.0 × (20+5) = 6.0 × 25 = 150.0
  • Group {W0, W2}: r = max(7.0, 6.0) = 7.0. Cost = 7.0 × (10+5) = 7.0 × 15 = 105.0 ← minimum
  • Group {W0, W1}: r = max(7.0, 2.5) = 7.0. Cost = 7.0 × (10+20) = 7.0 × 30 = 210.0

Answer: 105.0. The optimal group is {W0, W2} with W0 as captain.

Edge cases: k=1 (just pick worker with lowest wage/quality × quality = their exact minimum wage); all same ratios (any k workers work); very large quality values (use float arithmetic, no overflow risk in Python).

2 The Algorithm: Sort by Ratio + Max-Heap ▾

Brute force tries all C(n,k) groups — exponential. The greedy observation: sort workers by their ratio, then sweep left to right, declaring each worker as captain. At position i, all workers to the left have lower ratios and will accept the captain's rate. We want the k workers with smallest total quality from the prefix — a max-heap of size k gives us that dynamically.

1 Compute and sort by ratio: For each worker compute ratio = wage[i] / quality[i]. Sort all workers by ratio ascending. Now worker[0] has the cheapest-per-quality ratio; worker[n-1] has the most expensive.
2 Sweep left to right — each worker is the captain: When we process worker i, we declare them the captain (highest ratio in any group we form using only workers 0..i). All previous workers have ratio ≤ ratio[i], so they accept the captain's rate. The total cost for a valid k-group with captain i is ratio[i] × quality_sum.
3 Maintain a max-heap of size k on qualities: After adding worker i to the heap, if the heap exceeds k elements, evict the one with the largest quality (they cost the most at any fixed rate). The remaining k elements have the minimum possible quality sum.
4 Record cost when heap hits k: Once the heap contains exactly k workers, compute cost = ratio[i] × quality_sum and update the running minimum.
5 Return the minimum cost seen.
🔑 Why Sorting Unlocks the Greedy

In sorted order, when we reach worker i, every worker j < i satisfies ratio[j] ≤ ratio[i]. That means ratio[i] × quality[j] ≥ wage[j]/quality[j] × quality[j] = wage[j]. So every worker in the prefix is happy being paid at rate ratio[i] — we have a valid group. Sorting converts the expensive "check all combos" into a single sweep.

⚠️ Python max-heap trick: negate the values

Python's heapq only gives a min-heap. To evict the largest quality, push -quality. The root of the min-heap is then the most-negative value = the largest quality. When popping, the returned value is negative; adding it to quality_sum effectively subtracts the evicted quality.

3 Why It Works: The Invariant ▾
🔐 The Loop Invariant

After processing worker i (0-indexed in sorted order), the heap contains the min(i+1, k) workers from the prefix 0..i with the smallest total quality. The variable quality_sum equals the sum of qualities in the heap. When len(heap) == k, quality_sum × ratio[i] is the cost of the cheapest valid k-group where worker i is the captain.

Validity proof (both rules satisfied):

  • Every worker j in the heap came from the prefix, so ratio[j] ≤ ratio[i] (sorted order).
  • Each worker j is paid ratio[i] × quality[j]. Since ratio[i] ≥ ratio[j] = wage[j]/quality[j], we get pay[j] ≥ wage[j]. Rule 2 satisfied.
  • All pay values equal ratio[i] × quality[j], sharing the same rate. Rule 1 (proportional pay) satisfied.

Optimality proof: Consider any optimal group G*. Let c = its captain (highest ratio). When the sweep reaches c, the heap holds the k workers from prefix 0..c with minimum total quality. Since G* is also a valid k-subset of this prefix, our heap's quality sum ≤ G*'s quality sum. So our computed cost ≤ G*'s cost. Taking the minimum over all captains covers all cases.

💡 The Max-Heap Eviction is Greedy-Optimal

When the heap has k+1 elements, we must drop one. For a fixed rate, total cost = rate × quality_sum. Dropping the largest quality minimizes quality_sum, which minimizes cost. The max-heap root gives us the largest quality in O(log k). This greedy local choice leads to a globally optimal solution — a classic "keep the k-best" greedy.

4 The Code (Annotated) ▾

Solution 1 — Sort by Ratio + Max-Heap (Optimal, Interview Pick)

import heapq
def mincostToHireWorkers(quality, wage, k):
workers = sorted( ← Create (ratio, quality) pairs, sort by ratio ascending
(wage[i] / quality[i], quality[i]) ← ratio = wage/quality; lower ratio = cheaper per unit of quality
for i in range(len(quality))
)
heap = [] ← Max-heap (via negation) of qualities in current candidate group
quality_sum = 0 ← Running total quality of workers in the heap
ans = float('inf') ← Best (minimum) total cost found so far
for ratio, q in workers: ← Each iteration: current worker is the captain
heapq.heappush(heap, -q) ← Push negated quality: largest quality = most negative = heap root
quality_sum += q ← Add this worker's quality to running sum
if len(heap) > k: ← Overflowed: drop the most expensive (largest quality) worker
quality_sum += heapq.heappop(heap) ← Pop returns -largest_q; += subtracts it from quality_sum
if len(heap) == k: ← Full group of k: compute this configuration's cost
ans = min(ans, ratio * quality_sum) ← Captain's rate × sum of k smallest qualities
return ans
⚠️ Why quality_sum += heapq.heappop(heap) works

heappop returns the most negative item from our negated heap — that's -largest_quality. Adding this negative number to quality_sum is equivalent to subtracting largest_quality. So one elegant line handles both "pop from heap" and "update sum." This is the trickiest part of the solution — explain it explicitly in interviews.

💡 Complexity

Time: O(n log n) — Sorting dominates at O(n log n). The sweep is O(n) iterations × O(log k) per heap operation = O(n log k) ≤ O(n log n).
Space: O(n + k) — O(n) for the sorted workers array, O(k) for the heap. Optimizable to O(k) if we sort in-place.

Solution 2 — Brute Force (Exponential, for understanding only)

from itertools import combinations
def mincostToHireWorkers_brute(quality, wage, k):
ans = float('inf')
for group in combinations(range(len(quality)), k): ← Try every C(n,k) subset
rate = max(wage[i]/quality[i] for i in group) ← Captain sets the rate
cost = rate * sum(quality[i] for i in group) ← Total cost = rate × sum of qualities
ans = min(ans, cost)
return ans

O(C(n,k) × k) — exponential. Use only to verify the greedy solution on small inputs.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Decode the payment constraint mathematically

Say: "Proportional pay means everyone shares a rate r. The minimum-wage constraint is r ≥ wage[i]/quality[i] for every hired worker. So r must be the max ratio in the group." This turns a murky English rule into a clean inequality — the key first move.

Step 2: Name the captain

Continue: "To minimize cost = r × Σquality, we want r minimal. The minimum valid r is the max ratio. The worker with max ratio is the 'captain' — they set the rate. So: for each possible captain, find the k-1 cheapest (smallest quality) helpers. Minimize over all captains."

Step 3: Justify sorting by ratio

"If I sort by ratio, when I reach worker i, everyone before me has lower ratio — they're valid helpers for captain i. I just need to pick the k workers (from 0..i) with smallest total quality. Dynamic k-smallest = max-heap of size k."

Step 4: Code the elegant heap pattern

"Push -quality to simulate max-heap. When size exceeds k, pop the largest. Track quality_sum. When size == k, record cost. The quality_sum += heappop(heap) pattern handles eviction and sum-update in one line."

🗣️ What to Say to the Interviewer

"The proportional pay rule means everyone gets paid at the same rate r. The constraint is r must be ≥ every worker's wage/quality ratio — so r equals the max ratio in the group. Total cost is r × sum of qualities.
I'll sort workers by ratio. Sweeping left to right, I declare each worker as captain. All previous workers satisfy the constraint at the captain's rate. I keep a max-heap of size k to maintain the k smallest qualities from the prefix.
When the heap hits size k: cost = captain_ratio × quality_sum. Track the minimum. O(n log n) overall."
🔮 Likely Follow-up Questions
  • "What if k=n?" → Must hire everyone, cost = max_ratio × sum(all qualities).
  • "Can you handle ties in ratio?" → Yes, sorting is stable; the algorithm handles equal ratios naturally.
  • "Why not sort by quality instead?" → Sorting by quality doesn't guarantee minimum rate — a low-quality worker can still demand a high ratio.
  • "Similar problems?" → IPO (#502) — greedy with two heaps; Task Scheduler (#621) — greedy; Kth Largest Element — max-heap of size k.

7 Summary & Key Takeaways ▾
The proportional-pay rule means one shared rate r; r equals the max wage/quality ratio in the group (the captain's ratio)
Total cost = captain's ratio × sum of k qualities — minimize by choosing small qualities at a small ratio
Sorting by ratio lets us sweep and declare each worker captain; all previous workers are valid helpers
A max-heap of size k dynamically maintains the k smallest qualities from the prefix in O(log k) per step
quality_sum += heapq.heappop(heap) is the elegant negation trick: pop returns -max_quality, adding it subtracts max_quality
