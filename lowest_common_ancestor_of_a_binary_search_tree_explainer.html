LeetCode #235

Lowest Common Ancestor of a BST

Given a Binary Search Tree and two nodes p and q, return their lowest common ancestor.

Medium Trees BST Property Time: O(h) Space: O(1)
1 Understanding the Problem ▾

The Lowest Common Ancestor (LCA) of two nodes p and q in a binary tree is the deepest node that is an ancestor of both. "Ancestor" includes the node itself, so if p is an ancestor of q, then p is the LCA.

💡 Key Insight

In a BST, every node's value tells you exactly which subtree any other value belongs to. If both p and q are less than the current node, they're both in the left subtree. If both are greater, they're both in the right. If they go in different directions, the current node is the LCA — it's the first "fork" on their paths from root.

Concrete Example: BST with root 6 (left subtree: 2, 0, 4; right subtree: 8, 7, 9). Find LCA(p=2, q=8):

✓ Node 6 IS the LCA

p=2 lives in left subtree of 6
q=8 lives in right subtree of 6
They split at 6 → 6 is LCA

✗ Node 2 is NOT the LCA

2 is an ancestor of p=2 ✓
But q=8 is NOT in subtree of 2
Not a common ancestor ✗

Edge Cases:

  • p or q equals root: root is the LCA (a node is its own ancestor)
  • p is ancestor of q: p is the LCA — when we reach p, q is in p's subtree
  • p == q: return either (they're the same node)
  • Skewed BST: algorithm still works; O(n) in worst case
2 The Algorithm: Navigate by Value Comparison ▾

The BST property (left < node < right at every level) is a compass. At any node, comparing p and q to the node's value tells us where to go — without exploring both branches.

1 Start at root. Set node = root and begin traversal.
2 Compare both p and q against the current node. If both are smaller → go left. If both are larger → go right.
3 Stop when they diverge. If p is on one side and q on the other (or one matches the current node), you've found the LCA.
4 Return the current node. It is the deepest common ancestor.
🔑 Why This Is Efficient

At every step we eliminate an entire subtree from consideration. We follow exactly one path from root to LCA — never branching. Time is O(h), where h is the tree height. For a balanced BST that's O(log n). For a skewed one it degrades to O(n).

3 Why It Works: The Split Invariant ▾
🔐 The Invariant

At every node we visit, that node is a common ancestor of both p and q. We keep descending as long as both p and q are provably in the same subtree. The first node where we can no longer descend together is the deepest common ancestor — the LCA.

Proof sketch for each case:

  • p.val < node.val AND q.val < node.val: By BST property, both p and q are in the left subtree. The current node is a common ancestor, but there exists a deeper one. We move left.
  • p.val > node.val AND q.val > node.val: Same logic; move right.
  • Otherwise: Either p.val ≤ node.val ≤ q.val or p.val equals node.val or q.val equals node.val. In all these cases, node is a common ancestor AND there's no deeper one that's also a common ancestor — because p and q are now in opposite subtrees (or one IS the node). Return immediately.
⚠️ BST vs. General Binary Tree

This O(h) trick only works on BSTs. For a general binary tree (LC #236), you cannot infer subtree membership from values. You must recurse into both subtrees and check return values — that's O(n). Always confirm whether the input is a BST in an interview.

Special case — one node is the ancestor of the other: Suppose we're searching for p=5, q=3 and we reach node 5. The condition evaluates to "else" because p.val equals node.val. We return node 5. This is correct: 5 is an ancestor of itself, and q=3 must live in the left subtree of 5 by BST property, making 5 the LCA.

4 The Code (Annotated) ▾

Solution 1 — Iterative (Interview Pick): O(h) time, O(1) space

def lowestCommonAncestor(root, p, q):takes the root TreeNode and two target nodes
node = rootstart at root; no extra stack/queue needed
while node:LCA guaranteed to exist per problem constraints
if p.val < node.val and q.val < node.val:both targets in left subtree
node = node.leftdescend left — LCA is deeper
elif p.val > node.val and q.val > node.val:both targets in right subtree
node = node.rightdescend right — LCA is deeper

return nodethis IS the deepest common ancestor
💡 Why O(1) Space?

We move a single pointer down the tree — no recursion stack, no visited set, no queue. We follow exactly one root-to-LCA path and stop the moment we find it.

Solution 2 — Recursive: O(h) time, O(h) space (call stack)

def lowestCommonAncestor(root, p, q):recursive — elegant, same logic
if p.val < root.val and q.val < root.val:both in left
return lowestCommonAncestor(root.left, p, q)recurse left
if p.val > root.val and q.val > root.val:both in right
return lowestCommonAncestor(root.right, p, q)recurse right
return rootsplit or match — this is the LCA
⚠️ Common Mistake: Confusing with LC #236

LC #236 (LCA of a general binary tree) does NOT use value comparisons. You must recurse both subtrees. Mixing up these two problems in an interview is a red flag — always confirm: "Is this guaranteed to be a BST?"

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Confirm It's a BST

Ask: "Is this guaranteed to be a Binary Search Tree?" This single question separates a O(h) O(1) solution from an O(n) solution. The BST property is the entire trick.

Step 2: Think About "Where Are p and q?"

Reason aloud: "At any node in a BST, I can tell which subtree a value lives in just by comparing values. If both targets are on the same side, I go there. If they diverge — the current node is the LCA."

Step 3: Three-Case Decision

Enumerate cases explicitly: (1) both smaller → go left, (2) both larger → go right, (3) otherwise (split or match) → return current. This naturally translates to the if/elif/else structure.

Step 4: Start Iterative for O(1) Space

Write the iterative version first — it shows space awareness. Then mention the recursive alternative if asked. Both are O(h) time; iterative wins on space.

🗣️ What to Say to the Interviewer

"Since this is a BST, I can use the ordering property to navigate directly to the LCA. At any node, if both p and q are smaller, the LCA must be in the left subtree. If both are larger, it's in the right subtree. The first node where p and q go in different directions — or one of them matches the current node — is the LCA. I'll write this iteratively for O(h) time and O(1) space."
💡 Follow-Up Questions to Prepare For
  • "What if it's a general binary tree?" → LC #236 — recurse both sides, return non-null
  • "What if nodes might not be in the tree?" → Need to verify existence first
  • "Can you do it recursively?" → Yes — same logic, O(h) stack space (show Solution 2)
  • "Worst case time complexity?" → O(n) for a completely left/right skewed BST

7 Summary & Key Takeaways ▾
The BST ordering property lets you navigate directly to the LCA without exploring both subtrees
The LCA is the first node where p and q go in different directions (or one matches the node)
Iterative solution is O(h) time and O(1) space — no extra data structures needed
Always confirm "Is this a BST?" — it's the difference between O(h) O(1) and O(n) O(n)
A node is its own ancestor — if p.val equals node.val, that node is the LCA
🎯 When to Use This Pattern

  • Need to find the deepest node that is an ancestor of both p and q
  • Want to exploit BST ordering to get O(h) traversal without branching
