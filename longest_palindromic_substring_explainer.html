LeetCode #5

Longest Palindromic Substring

Given a string s, return the longest palindromic substring in s.

Medium Two Pointers Expand Around Center Time: O(n²) Space: O(1)
1 Understanding the Problem ▾

A palindrome reads the same forwards and backwards. "racecar", "madam", "aba" are palindromes. "abc" is not. We want the longest contiguous substring of s that happens to be a palindrome. If there are ties, any one is acceptable.

💡 Key Insight: Palindromes Grow Outward from a Center

Every palindrome has a center. For odd-length palindromes like "racecar", the center is a single character ('e'). For even-length palindromes like "abba", the center is between two identical characters. If we plant ourselves at every possible center and expand outward while characters match, we find every palindrome — and keep the longest.

Edge Cases to consider: empty string (return ""), single character (trivially a palindrome), all identical characters like "aaa" (whole string is the answer), and strings where the best answer is a single character.

⚠️ Substring vs. Subsequence — Don't Mix Them Up

We need a substring (characters must be contiguous), NOT a subsequence. In "babad", the answer is "bab" or "aba" (length 3). A subsequence could be "baab" by skipping characters — that's a different problem (#516 Longest Palindromic Subsequence, solved with DP).

2 The Algorithm: Expand Around Center ▾

For each of the n character positions in s, we try two expansion types: one for odd-length palindromes (center is a single character) and one for even-length palindromes (center is between two characters). We expand outward as long as characters match, recording the best result found so far.

1 Initialize best: Set best_start=0, best_len=1. Any single character is always a palindrome of length 1, so we never return an empty string from a non-empty input.
2 For each index i, try two center types: Call expand(i, i) for odd-length (left and right start at same position), and expand(i, i+1) for even-length (left and right start at adjacent positions).
3 Expand while matching: In expand(left, right), loop while left >= 0 and right < n and s[left] == s[right]. Each successful iteration: left -= 1, right += 1. This grows the palindrome by two characters.
4 Recover the palindrome after mismatch: When the loop exits, left and right have overstepped by 1. The valid palindrome is s[left+1 : right] with length right - left - 1. Update best if this is longer.
5 Return the answer: After all centers are tried, s[best_start : best_start + best_len] is the longest palindromic substring.
🔑 Why Not Brute Force O(n³)?

The naive approach checks every O(n²) substring and validates each palindrome in O(n) time — O(n³) total. Too slow for n=1000 (10⁹ ops). The Expand Around Center trick reframes the work: instead of validating substrings, we construct palindromes outward from guaranteed-valid centers. Most expansions terminate early on the very first character mismatch. The worst case is O(n²) but the average is much faster.

3 Why It Works: The Invariant ▾
🔐 The Core Invariant

At any moment during an expansion, the substring s[left+1 : right] is a valid palindrome. We only advance the pointers when s[left] == s[right], which means each step "wraps" the existing palindrome in a new matching pair. The invariant is maintained because we never proceed past a mismatch — we stop and record what we have.

Every palindrome is found exactly once: A palindrome of any length has exactly one "center" (or one inter-character gap for even-length). When we iterate over all n positions (and n-1 gaps), we guarantee we try the true center of every possible palindrome. No palindrome is double-counted, and none is missed.

Correctness of the index recovery: After the while loop exits (mismatch or boundary hit), the pointers are one step past the valid palindrome. If the last valid positions were [L, R], after the failed step we have left = L-1 and right = R+1. So the palindrome is s[(L-1)+1 : (R+1)] = s[L : R+1] in Python — i.e., s[left+1 : right]. Length = R - L + 1 = (right-1) - (left+1) + 1 = right - left - 1.

⚠️ The Even-Length Case Starts with a Non-Trivial Check

For odd expansion, expand(i, i) starts with s[i] == s[i] (always true), so the first iteration is free. For even expansion, expand(i, i+1) starts with s[i] == s[i+1], which might immediately fail. If it fails, the loop never runs, and length = (i+1) - i - 1 = 0 — safely ignored since 0 is never greater than our best (≥1).

4 The Code (Annotated) ▾

Solution 1 — Expand Around Center (Interview Pick, O(n²) time, O(1) space)

def longestPalindrome(s: str) -> str:
if not s: return "" ← edge case: empty string
best_start, best_len = 0, 1 ← single char is always a palindrome
def expand(left, right): ← inner helper for one center config
nonlocal best_start, best_len ← allow mutation of outer variables
while left >= 0 and right < len(s) \ ← stay in-bounds...
and s[left] == s[right]: ← ...and characters must mirror
left -= 1 ← grow left boundary outward
right += 1 ← grow right boundary outward
# loop exits: left/right overstepped by 1
length = right - left - 1 ← valid palindrome length (corrected)
if length > best_len: ← is this palindrome the new champion?
best_start = left + 1 ← undo the final left-- overstep
best_len = length
for i in range(len(s)): ← try every position as a center
expand(i, i) # odd-length center ← e.g. "aba": center at 'b'
expand(i, i + 1) # even-length center ← e.g. "abba": center between 'b','b'
return s[best_start : best_start + best_len]← slice the winning palindrome
⚠️ Off-by-One: The #1 Bug in This Problem

After the while loop, left and right have moved one step past the valid palindrome. The valid start is left + 1 (not left), and the valid end is right - 1 (not right). Python slice s[left+1 : right] correctly captures this range (Python slices exclude the end index). Length = right - left - 1.

Solution 2 — Dynamic Programming (O(n²) time, O(n²) space)

def longestPalindrome(s: str) -> str:
n = len(s)
dp = [[False]*n for _ in range(n)] ← dp[i][j] = True if s[i..j] is palindrome
start, max_len = 0, 1
for i in range(n): dp[i][i] = True ← base: length 1 is always palindrome
for i in range(n-1): ← base: check adjacent pairs (length 2)
if s[i] == s[i+1]:
dp[i][i+1] = True; start, max_len = i, 2
for length in range(3, n+1): ← build up: length 3, 4, ..., n
for i in range(n - length + 1): ← try every start position
j = i + length - 1 ← end index
if s[i] == s[j] and dp[i+1][j-1]: ← outer chars match AND inner is palindrome
dp[i][j] = True
if length > max_len: start, max_len = i, length
return s[start : start + max_len]
💡 Complexity Comparison

Expand Around Center: O(n²) time, O(1) space — the interview pick
DP Table: O(n²) time, O(n²) space — intuitive recurrence but memory-heavy
Manacher's Algorithm: O(n) time, O(n) space — optimal, but non-trivial to derive live

Expand Around Center wins on space: no DP table, just a few index variables.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: State the Brute Force First

"The brute force is O(n³): check all O(n²) substrings, validate each in O(n). Too slow. I need to reduce to O(n²) or better."

Step 2: Identify the Center Property

"Every palindrome has a center — one character for odd length, or a gap between two equal characters for even length. There are 2n−1 possible centers. If I expand outward from each center while characters match, I find all palindromes in O(n²) total."

Step 3: Write the Expand Helper

"I'll extract a helper expand(left, right) that handles the while loop and index recovery. This makes it easy to handle both odd and even center types cleanly. Call it twice per index: expand(i,i) and expand(i,i+1)."

Step 4: Double-Check the Off-by-One

"After the while loop, left and right have overstepped. Valid palindrome is s[left+1:right], length right - left - 1. I always verify this on a concrete example before submitting."

🗣️ What to Say to the Interviewer

"Every palindrome has a center. There are 2n-1 possible centers — n single-character
centers for odd-length, and n-1 inter-character gaps for even-length.
I'll iterate all centers, expand outward while s[left]==s[right], then record the best.
Key off-by-one: after the while loop, the valid palindrome is s[left+1:right],
with length = right - left - 1, since pointers overshot by one in each direction.
Time: O(n^2) — n centers, each expanding at most n/2 steps.
Space: O(1) — just index variables, no DP table."

7 Summary & Key Takeaways ▾
Every palindrome has exactly one center — this is the core insight enabling O(n²)
Two expansion types per index: odd (center at i) and even (center between i and i+1)
After loop exits, valid palindrome is s[left+1 : right], length = right − left − 1
O(1) space — just a few index variables, unlike DP which needs an O(n²) table
Manacher's O(n) algorithm is the theoretical optimum but rarely needed in interviews

🎯 When to Recognize the Expand Around Center Pattern
  • "Longest / all palindromic substrings" → Expand around center
  • Problem involves symmetry around a midpoint → Try center-based expansion
  • Two-pointer moving outward (diverging, not converging) — the hallmark of this pattern
  • O(1) space required with palindrome property → Prefer expand over DP table
