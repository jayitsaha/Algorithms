Longest Palindromic Substring

LeetCode #5 — Expand Around Center

Understanding the Problem
▼

Find the longest substring that reads the same forwards and backwards.

💡 Key Insight

Every palindrome has a center. Expand outward from each of 2n-1 centers (n odd, n-1 even) while outer chars match. O(n²) total — simpler than Manacher's O(n) and preferred in interviews.

2
The Algorithm: Expand Around Center
▼

Approach: Expand Around Center

1For each i from 0..n-1: expand(i, i) → odd-length palindromes
2For each i from 0..n-2: expand(i, i+1) → even-length palindromes
3expand(l,r): while l≥0 and r
4Track best (l,r) across all centers
3
Why It Works: The Invariant
▼
🔒 Invariant

s[l..r] is always a valid palindrome. Expanding maintains this: we only expand when outer chars match.

Every palindrome has exactly one center. Trying all 2n-1 centers covers every possible palindrome. expand() returns the maximal palindrome for each center.

✅ Complexity

Time: O(n²) | Space: O(1)

The Code (Annotated)
▼
def longestPalindrome(s: str) -> str:
    def expand(l: int, r: int) -> tuple:
        while l >= 0 and r < len(s) and s[l] == s[r]:
            l -= 1; r += 1
        return l+1, r-1   # last valid bounds (loop overshot by 1)

    best_l = best_r = 0
    for i in range(len(s)):
        for l, r in [expand(i, i), expand(i, i+1)]:   # odd & even
            if r - l > best_r - best_l:
                best_l, best_r = l, r
    return s[best_l : best_r + 1]
⚠ Common Mistake

expand() returns l+1, r-1 because the while loop overshoots by 1 on each side. Forgetting this returns an invalid palindrome.
