LeetCode #525

Contiguous Array

Find the maximum length of a contiguous subarray with an equal number of 0s and 1s.

Medium Prefix Sum Hash Map Time: O(n) Space: O(n)
1 Understanding the Problem ▾

Given a binary array (only 0s and 1s), we want the longest contiguous slice where the number of 0s equals the number of 1s exactly. For example, in [0, 1, 0, 1] the entire array has two 0s and two 1s, so the answer is 4. In [0, 0, 1] the best we can do is [0, 1], giving length 2.

💡 Key Insight — The 0 → −1 Transform

Replace every 0 with −1. Now "equal number of 0s and 1s" is equivalent to "subarray sum equals 0", because each −1 cancels a +1. A problem about character balance becomes a standard prefix-sum problem.

✓ Balanced: nums=[0,1,0,1]

Transform: [−1, +1, −1, +1]
Prefix sums: −1, 0, −1, 0
Sum reaches 0 at index 3
→ length 4 (entire array) ✓

✗ Unequal overall: nums=[0,0,1]

Transform: [−1, −1, +1]
Prefix sums: −1, −2, −1
Prefix −1 repeats at idx 0 and 2
→ subarray [0,1] (idx 1..2), length 2

Edge cases to handle: all zeros (answer 0), all ones (answer 0), length 1 (always 0 — can't have both), entire array balanced (answer n), alternating pattern like [0,1,0,1,…].

2 The Algorithm: Prefix Sum + Hash Map ▾

After the 0→−1 transform, the problem is: find the longest subarray with sum exactly 0. We use the prefix sum + hash map pattern — the same technique as "Subarray Sum Equals K," just with K = 0.

nums 0 1 0 0 1 1
transformed −1 +1 −1 −1 +1 +1
prefix sums −1 0 −1 −2 −1 0
best match prefix 0 at idx 5 matches sentinel at −1 → length = 5 − (−1) = 6
1 Initialize: prefix = 0, best = 0, and seen = {0: −1}. The entry 0: −1 is a sentinel meaning "the running sum was 0 before the array started." This allows balanced subarrays beginning at index 0.
2 Scan left to right: For each element at index i, update prefix += 1 if num == 1 else −1. (We apply the 0→−1 transform on the fly — no need to build a new array.)
3 Check the map: If prefix is already in seen, a balanced subarray exists from seen[prefix] + 1 to i. Its length is i − seen[prefix]. Update best if this is longer.
4 Store only the FIRST occurrence: If prefix is NOT in seen, record seen[prefix] = i. We never overwrite an existing entry — keeping the earliest index maximizes future subarray lengths.
5 Return best after scanning all elements.
🔑 Why "same prefix sum at two indices" means balanced?

If prefix[j] == prefix[k] with j < k, the sum of elements from j+1 to k is prefix[k] − prefix[j] = 0. After the transform, sum 0 means the count of −1s (original 0s) equals the count of +1s (original 1s) in that range.

3 Why It Works: The Invariant ▾
🔐 The Invariant

At every step, seen[p] holds the smallest index at which prefix sum p was observed. When the same prefix sum recurs at index i, the subarray from seen[p]+1 to i has sum 0 — the longest possible for this particular prefix sum value.

Why only the first occurrence? Suppose prefix sum p first appears at index j. If it appears again at index k, and then again at m > k, the lengths are k − j and m − j. Using the earliest j always gives the longest result. Overwriting j with k would permanently lose access to the longer subarray.

Why the {0: −1} sentinel? Consider nums=[0,1]. After element 1 (index 1), prefix = 0. Without the sentinel, prefix 0 isn't in the map yet and we'd store seen[0] = 1. But the whole array (indices 0..1) is balanced with length 2. The sentinel encodes "prefix 0 was seen at 'index −1' (before the array)" so the length computes as 1 − (−1) = 2. Correct!

⚠️ Common Mistake: Overwriting seen on every visit

Writing seen[prefix] = i unconditionally (not just on first visit) replaces the earliest index with a later one, permanently shrinking the best possible subarray length for that prefix sum. Always guard with if prefix not in seen.

Proof of optimality: For any balanced subarray A[i..j], let p = prefix sum at index i−1. Then prefix sum at j equals p too. Our map finds the smallest index where p occurred, so we always measure the longest possible span ending at j with that prefix value.

4 The Code (Annotated) ▾

Solution 1 — Prefix Sum + Hash Map (Optimal, Interview Pick)

def findMaxLength(nums):
seen = {0: -1} ← Sentinel: prefix 0 was "seen" before index 0
prefix = 0 ← Running sum (0→−1, 1→+1 applied each step)
best = 0 ← Track longest balanced subarray length
for i, num in enumerate(nums): ← Scan with index
prefix += 1 if num == 1 else -1 ← Apply transform inline: 1→+1, 0→−1
if prefix in seen: ← Seen this prefix before → balanced subarray exists!
best = max(best, i - seen[prefix]) ← Length = current index − first-seen index
else:
seen[prefix] = i ← First occurrence only — never overwrite
return best ← Maximum balanced subarray length
⚠️ Why i − seen[prefix] and not i − seen[prefix] + 1?

seen[prefix] is the index before the balanced subarray begins. The subarray occupies indices seen[prefix]+1 through i inclusive. Its length is i − seen[prefix] — you can verify: (i) − (seen[prefix]+1) + 1 = i − seen[prefix].

Solution 2 — Brute Force O(n²) (start here in interviews)

def findMaxLength_brute(nums):
n, best = len(nums), 0
for i in range(n): ← Try every starting index
zeros = ones = 0
for j in range(i, n): ← Extend right one step at a time
if nums[j] == 0: zeros += 1
else: ones += 1
if zeros == ones: ← Balanced!
best = max(best, j - i + 1)
return best
💡 Complexity Summary

Prefix Sum + Hash Map: Time O(n), Space O(n) — single pass, at most n+1 entries in the map
Brute Force: Time O(n²), Space O(1) — two nested loops, no extra data structure needed

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Balance = Sum pattern

"Equal 0s and 1s" screams transform. Say: "If I replace 0 with −1, the constraint becomes 'subarray sum equals 0.' That's a much more tractable form." This single reframe unlocks the whole problem.

Step 2: Propose brute force and its cost

"Try all pairs (i, j) and count 0s and 1s — O(n²) time, O(1) space. Correct, but too slow for large inputs. We need a way to skip redundant work."

Step 3: Recognize the "Subarray Sum = K" shape

"This is structurally identical to Subarray Sum Equals K with K = 0. I'll track a running prefix sum, and whenever I see the same prefix sum again, the slice between those two indices sums to zero."

Step 4: Explain the sentinel explicitly

"I initialize seen = {0: −1}. The key 0 with value −1 handles the edge case where the prefix sum returns to 0 at some index i — the balanced subarray spans from 0 to i, and its length is i − (−1) = i + 1."

🗣️ What to Say to the Interviewer

"Replace 0 with −1. Now equal 0s and 1s means subarray sum = 0.
I'll track a running prefix sum. Whenever I see the same prefix sum twice, the elements in between sum to zero — balanced!
I store only the FIRST index of each prefix sum in a hash map to maximize subarray length.
Sentinel {0: −1} handles the case where the prefix sum returns to zero, covering subarrays that start at index 0.
Result: O(n) time, O(n) space — one pass, one map."
🔮 Likely Follow-up Questions
  • "What if we want K more 1s than 0s?" → Same prefix sum trick, but look for prefix difference = K
  • "Count all balanced subarrays, not just the longest?" → Store count of each prefix sum, add count to answer each time prefix recurs
  • "Can this work in O(1) space?" → No. We need O(n) for the hash map since prefix sums range from −n to +n
  • "2D version?" → Fix top and bottom row, compress each column to 1D prefix sum, apply this technique
