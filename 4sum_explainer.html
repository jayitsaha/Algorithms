LeetCode #18

4Sum

Given an integer array nums and an integer target, return all unique quadruplets [a, b, c, d] such that a + b + c + d == target.

Medium Two Pointers Sort + Fix Two + Two Pointers Time: O(n³) Space: O(1)
1 Understanding the Problem ▾

We need to find every unique set of four numbers from nums that add up to exactly target. Order within a quadruplet does not matter — [-1,0,1,0] and [-1,0,0,1] are the same quadruplet. Duplicates in the input are allowed but duplicates in the output are not.

💡 Key Insight

4Sum is a direct generalisation of 3Sum and Two Sum II. The strategy: sort once, fix two elements with nested loops, then reduce the remaining problem to a classic Two Pointer search on the sorted suffix. This turns O(n⁴) brute force into O(n³) with O(1) extra space.

✓ Example: nums=[1,0,-1,0,-2,2], target=0

The subtlest part is duplicate elimination. When the same value appears multiple times in nums, we must not generate duplicate quadruplets. Sorting ensures equal values are adjacent, making duplicate-skipping a simple if nums[k] == nums[k-1]: continue check — but the condition is slightly different for i vs j (see Section 3).

2 The Algorithm: Sort + Fix Two + Two Pointers ▾

We reduce 4Sum to many instances of 2Sum II. Sort the array. Use two outer loops to "fix" the first and second elements. For each (i, j) pair, run classic Two Pointer (left and right converging inward) to find all pairs from the remaining suffix that complete the target sum.

i — Outer Fixed
Loops 0 → n-4. Skip duplicate nums[i]. First element of each quadruplet.
j — Inner Fixed
Loops i+1 → n-3. Skip duplicate nums[j] (guard: j > i+1). Second element.
left — Left Probe
Starts at j+1. Moves right when sum is too small. Third element.
right — Right Probe
Starts at n-1. Moves left when sum is too large. Fourth element.
1 Sort: nums.sort(). O(n log n). Now equal values are adjacent (easy dedup) and smaller values are on the left (enables two-pointer direction logic).
2 Fix i: for i in range(n - 3). Skip if i > 0 and nums[i] == nums[i-1]. Optionally prune: if nums[i] * 4 > target, break (minimum possible sum already exceeds target). If nums[i] + nums[-1]*3 < target, skip (even the maximum sum with this i is too small).
3 Fix j: for j in range(i+1, n - 2). Skip if j > i+1 and nums[j] == nums[j-1]. Apply symmetric pruning for the (i,j) pair. Initialise left = j+1, right = n-1.
4 Two Pointer search: Compute s = nums[i]+nums[j]+nums[left]+nums[right]. If s == target: record quadruplet, skip duplicates at left and right, advance both. If s < target: left += 1. If s > target: right -= 1.
5 Skip duplicates on hit: After appending, advance left while nums[left] == nums[left+1] and shrink right while nums[right] == nums[right-1]. Then do left += 1; right -= 1 once more to land on genuinely new values.
🔑 Why Sorting Is the Master Key

Without sorting, deduplication requires a hash set (O(n) space), and two-pointer doesn't work (no ordering guarantee). Sorting gives us both for free. The two-pointer sweep is correct because in a sorted subarray, moving left right always increases the sum, and moving right left always decreases it — so we can safely eliminate whole ranges of possibilities in O(1) per step.

3 Why It Works: The Invariant ▾
🔐 The Two-Pointer Invariant

For fixed i and j, define the sub-problem as finding pairs (L, R) in nums[j+1..n-1] such that nums[L] + nums[R] == target - nums[i] - nums[j]. Because this subarray is sorted, at every step: all pairs with second element index greater than right or first element index less than left have already been checked or are proven impossible. The invariant ensures no valid pair is ever skipped.

Why moving pointers is correct: If the current sum is too small, only increasing left can help (larger value on the left). Decreasing right would only make things worse. The converse holds when sum is too large. This exhausts every possibility in O(n) per (i, j) pair.

Why the j-dedup guard differs from i-dedup:

  • For i: if i > 0 and nums[i] == nums[i-1]: continue — We've finished with i-1, so a repeated value at i would generate identical quadruplets.
  • For j: if j > i+1 and nums[j] == nums[j-1]: continue — We guard with j > i+1 because j restarts at i+1 every outer iteration. Without this guard, if nums[i+1] == nums[i], we'd accidentally skip the very first valid j = i+1.
⚠️ The Most Common Bug: Wrong j Dedup Guard

A classic mistake is writing if j > 0 and nums[j] == nums[j-1]: continue. This causes incorrect skipping when nums[i] == nums[i+1] — you'd miss valid quadruplets entirely. Always use j > i+1 as the guard for the inner loop dedup.

Complexity justification: Sorting = O(n log n). Two nested loops = O(n²). Two-pointer per (i,j) = O(n). Total: O(n³). Space = O(1) extra (ignoring output). No better general complexity is achievable — the output itself can contain O(n²) quadruplets.

⚠️ The j Guard Asymmetry (Critical)

Use j > i+1 not j > 0. Since j resets to i+1 every outer iteration, checking j > 0 would incorrectly skip j = i+1 when nums[i] == nums[i+1], causing missed quadruplets. The condition j > i+1 ensures we only skip if this value of j has already been explored in the current i-iteration.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognise the kSum Family

"This is 4Sum — the natural extension of Two Sum and 3Sum. The pattern generalises: fix k-2 elements with nested loops, then run Two Sum II on the remaining sorted suffix. Time goes from O(n^k) brute force to O(n^(k-1)) with sorting."

Step 2: Start with Brute Force, Then Optimise

"Four nested loops give O(n⁴). We can use a set for dedup. But this is too slow for n=200. Key insight: if we sort, we get two-pointer on the inner two elements for free, dropping to O(n³)."

Step 3: Explain the Sorting Benefit

"Sorting enables: (1) two-pointer convergence — moving left right increases sum, moving right left decreases it; (2) dedup without a hash set — equal adjacent values are skipped in O(1)."

Step 4: Carefully Walk Through Dedup Logic

"For i: if i > 0 and nums[i] == nums[i-1]: continue. For j: if j > i+1 and nums[j] == nums[j-1]: continue. The asymmetry: j resets each outer iteration, so we guard with j > i+1 not j > 0."

Step 5: Mention Pruning (Bonus Points)

"We can prune: if nums[i]*4 > target, break (all further i will only be larger). If nums[i]+3*nums[-1] < target, continue (even maximum sum from this i is too small). This speeds up average case significantly."

🗣️ What to Say to the Interviewer

"I'll start by sorting the array. Then I fix two elements i and j with nested loops.
For each (i, j) pair, I use two pointers left and right on the suffix nums[j+1..n-1].
If the sum equals target, I record the quadruplet and advance both pointers, skipping dups.
For dedup: i skips when nums[i]==nums[i-1]; j skips when j>i+1 and nums[j]==nums[j-1].
The j guard differs from i because j resets each outer iteration — using j>0 would skip

🔮 Likely Follow-up Questions
  • "Generalise to kSum?" → Recursive: fix one element + call (k-1)Sum, base case is 2Sum II
  • "What if nums contains very large values and overflows?" → Cast to long in Java/C++; Python has arbitrary precision
  • "Can you beat O(n³)?" → No in general; output can be O(n²) quadruplets, which must all be read
  • "4Sum II (LC 454) — is it the same?" → No — four separate arrays; use hash map on pairs for O(n²)

7 Summary & Key Takeaways ▾
Sort first — it unlocks both two-pointer convergence and O(1) deduplication
Fix two elements (i, j) and reduce inner problem to 2Sum II with two pointers
Dedup i: i > 0 and nums[i] == nums[i-1]
Dedup j: j > i+1 and nums[j] == nums[j-1] — asymmetry is intentional!
After a hit: skip left/right dups, then advance both pointers once more
Pruning cuts average-case time without changing O(n³) worst-case
This pattern extends to kSum via recursion: fix one, call (k-1)Sum
🎯 When to Use This Pattern
  • Find k elements from an array summing to a given target (k ≥ 2)
  • Uniqueness required — no duplicate tuples in output
  • Array can be sorted without violating problem constraints
  • Negative numbers are allowed (unlike some counting problems)
  • Signal words: "unique quadruplets/triplets", "sum equals target", "no duplicates"
