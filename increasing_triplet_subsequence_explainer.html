Increasing Triplet Subsequence

O(1) Space Solution: Track Two Smallest Values

#334 Medium Greedy / Two Variables O(n) Time · O(1) Space
1 Understanding the Problem

Given an array nums, determine if there exist three indices i < j < k such that nums[i] < nums[j] < nums[k].

💡 Key Insight

We don't need to find the actual triplet — just prove one exists. This means we can track just enough information to know if a valid third element appears:

  • first — The smallest value seen so far
  • second — The smallest value that's greater than some earlier element

If any value exceeds second, that's our third element — triplet exists!

2 The Algorithm: Maintain Two Smallest
1 Initialize first = second = ∞
2 For each num in array, check in order:
Is num ≤ first?
✓ Update first = num
New smallest so far
→ Check next condition
Is num ≤ second?
✓ Update second = num
New second smallest
→ Check next condition
num > second > first?
🎉 Return True!
Triplet found
3 Why It Works: The Hidden Invariant
🔐 The Critical Invariant

If second has a finite value, it GUARANTEES that some smaller value appeared before it in the array.

We don't need to remember which value — just that one exists. This is what makes O(1) space possible.

⚠️ The Tricky Case: Updating first After second Is Set

This confuses many people! Watch what happens with input [2, 1, 5, 0, 4, 6]:

  • • After num=5: first=1, second=5 — pair (1,5) established
  • • After num=0: first=0, second=5 — first changes but second stays!
  • • After num=6: 6 > second(5) — Triplet found!

Why is this correct? Even though first=0 now, the old second=5 still "remembers" there was a value (the old first=1) before it. The triplet (1, 5, 6) still exists in the array!

📐 The Logic
first = smallest value seen so far
second = smallest value that is GREATER than some earlier value
If num > second → triplet (?, second_history, num) exists
4 The Code (Annotated)
# Track two smallest values with special invariant
def increasingTriplet(nums):
first = second = float('inf') # Start with infinity
for num in nums:
if num <= first:
first = num # Smallest so far
elif num <= second:
second = num # 2nd smallest (with history!)
else:
return True # num > second > (some past value)
return False # No triplet found

5 🎤 Interview Strategy: How to Arrive at This
Step 1: Start Simple

"The brute force is O(n³) — check all triplets. Can we do better? We only need to PROVE a triplet exists, not find it."

Step 2: Reduce the Problem

"For each element to be a valid 'middle', we need something smaller before and larger after. What's the minimum information to track?"

Step 3: The Two-Variable Insight

"If I track the smallest value (first) and smallest-greater-than-first (second), any value > second completes the triplet."

Step 4: Address the Tricky Part

"Even if first gets updated later, second still remembers there was something before it. The invariant holds!"

🗣️ What to Say to the Interviewer
"Instead of finding the actual triplet, I'll track enough information to prove one exists. I maintain two variables: 'first' is the smallest seen so far, 'second' is the smallest value that's greater than some previous value. The key insight: if second ever gets set, it GUARANTEES something smaller existed before it. So if any later value exceeds second, we have our triplet. Even if first gets updated after second is set, the invariant holds — second still 'remembers' its history."

7 Summary & Key Takeaways

🎯 When to Use This Pattern
  • Finding increasing/decreasing subsequences of fixed length
  • Problems where you need to prove existence, not find actual elements
  • When O(1) space is required
  • Tracking "has something smaller appeared before?" implicitly

💡 Alternative: Prefix Min / Suffix Max

There's also an O(n) time, O(n) space solution that precomputes:

  • prefix_min[i] = smallest value in nums[0..i]
  • suffix_max[i] = largest value in nums[i..n-1]

Then check each middle: if prefix_min[j-1] < nums[j] < suffix_max[j+1], triplet exists. Easier to reason about, but uses extra space.
