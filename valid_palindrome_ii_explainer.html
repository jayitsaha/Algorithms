Valid Palindrome II

Two converging pointers — on first mismatch, try skipping the left character OR the right character, check if either produces a palindrome

Easy String Processing Two Pointers + One Skip Allowed Time: O(n) Space: O(1)
1 Understanding the Problem ▲

Given a string s, return true if it can become a palindrome by deleting at most one character. The string only contains printable ASCII characters (no need to clean non-alnum like in Valid Palindrome I).

✗ Not a palindrome as-is
"abca"

a≠a✓, b≠c → mismatch

✓ Delete one → palindrome
"abc" (delete 'a') or "aca" (delete 'b') → both palindromes… wait:
delete 'b' → "aca" ✓

True — at most one deletion needed

💡 Key Insight

This is a direct extension of Valid Palindrome I. We run the same two-pointer check, but when we hit the first mismatch we get one free deletion. We try both options — skip the left character (move left inward) or skip the right character (move right inward) — and check if either remaining window is a palindrome. If yes → True.

🔑 Why the First Mismatch is the Only Branch Point

Before the first mismatch, all pairs matched perfectly so no deletion was needed for those characters. At the mismatch, we must delete one of the two mismatching characters to continue — and we've used up our one deletion. So both sub-checks (s[left+1..right] and s[left..right-1]) get no further deletions allowed: they must be perfect palindromes.

2 The Algorithm: Two Pointers + One Skip ▲

We need a helper function is_palindrome(s, l, r) that checks a window s[l..r] with no deletions allowed. The main function uses it after spending the skip.

1 Set left=0, right=len(s)-1. Run the standard two-pointer palindrome check.
2 While left < right: if s[left] == s[right], move both pointers inward — no deletion needed here.
3 First mismatch found at positions (left, right). We have one deletion left. Try two options:
4 Option A: Delete s[left] → check if s[left+1 .. right] is a palindrome (no more deletions).
5 Option B: Delete s[right] → check if s[left .. right-1] is a palindrome (no more deletions).
6 Return True if either option passes. If neither passes, the string cannot be made palindrome with one deletion.
7 If the outer loop completes without any mismatch, the string is already a palindrome → return True.

Brute Force Try deleting every character, check each for palindrome O(n²) O(n)
Two Pointers + One Skip ✓ Two pointers; on first mismatch branch into 2 sub-checks O(n) O(1)
3 Why It Works: The Invariant ▲
🔐 The Two Invariants

Outer loop invariant: At the start of each iteration, all characters outside the current window [left, right] have already been confirmed as matching mirror pairs — no deletion was needed for them.

Sub-check invariant: After spending the deletion at the first mismatch, the remaining sub-window must be a perfect palindrome. Since no further deletion is allowed, the standard two-pointer check is applied.

Why Try BOTH Directions at the Mismatch?

When s[left] ≠ s[right], we don't know which character is the "rogue" one that should be deleted. Consider:

Case: delete left char
s = "abca"
left='a', right='a' ✓
left='b', right='c' ✗
Skip 'b' → "ca" → c≠a ✗
Skip 'c' → "bc"[b..b] = "b" ✓ wait…
"abca" skip right char 'c'→ "aba" wait no:
left=1 'b', right=2 'c' mismatch.
Skip left → s[2..2]="c" ✓(single)
Skip right → s[1..1]="b" ✓(single)
Either works → True
Case: only one direction works
s = "aguokepatgbnvfqmgmlcupuufxoohdfpgjdxrvpxgpqtqs..."
One option leads to a palindrome,
the other does not.
Must try BOTH to be safe.

Edge Cases

•Already a palindrome (zero deletions needed): Outer loop completes without mismatch → True.
•Single character: left=right immediately → loop doesn't run → True.
•Two characters, different: left=0, right=1, mismatch → try s[1..1] (palindrome) and s[0..0] (palindrome) → both True → True.
•Mismatch at very first pair: Still only one deletion, same logic — try skipping either end character.
4 The Code (Annotated) ▲

Optimal: Two Pointers + One Skip  ← Interview Pick

def validPalindrome(s: str) -> bool:
def is_pal(l, r): # helper: strict palindrome check on window s[l..r]
while l < r:
if s[l] != s[r]: # no deletions allowed in sub-check
return False
l += 1; r -= 1
return True
left, right = 0, len(s) - 1
while left < right:
if s[left] != s[right]: # first mismatch — spend our one deletion
return is_pal(left + 1, right) or is_pal(left, right - 1)
left += 1 # matched — advance both inward
right -= 1
return True # no mismatch → already a palindrome
⚠️ Common Mistake: Not Using a Helper

A common error is to try to add a used_delete flag inside a single loop. This breaks because after spending the deletion you need a clean palindrome check from scratch on a sub-window. A recursive call or a helper function is the cleanest way to express "check this window with no deletions allowed."

💡 Why Return Early at First Mismatch?

We know all characters outside [left, right] already match in pairs. The only "bad" character must be at left or right — there's no other candidate. If neither removing s[left] nor s[right] fixes it, no single deletion can help.

5 🎤 Interview Strategy ▲
Step 1 — Recognise it as "Palindrome I + one twist"

Any time you see "palindrome with at most k deletions/changes", your starting point is the two-pointer palindrome check. Here k=1, which makes it especially clean.

Step 2 — Say "I'll run two pointers; at first mismatch I branch"

This immediately signals you understand the problem structure. Mention that the branch creates two sub-windows, each checked with a helper that has no remaining deletions.

Step 3 — Justify why it's O(n)

The outer pointer scan is O(n). The two sub-checks are each O(n) in the worst case but they never both run their full length — and crucially they only fire once. Total: O(n).

Step 4 — Walk through edge cases proactively

Mention: already a palindrome (zero deletions), two-character string with different chars, mismatch at position 0.

🗣️ What to Say

"This is Valid Palindrome I, but with one free deletion."
"I'll run two pointers from both ends — same as before."
"The moment I find a mismatch, I branch: try skipping left OR skip right."
"Each branch is just a strict palindrome check — no more deletions."
"Return true if either branch succeeds."
"Time O(n) — each character touched at most twice. Space O(1)."
💡 Follow-Up: What if k=2 deletions?

For k=2, you'd recurse/branch again at the second mismatch inside each sub-check. This generalises to recursion with a deletions_left counter — but it becomes O(n·k) time and O(k) stack space. For large k, a DP solution (check if len(s) - LPS(s) <= k) is preferred.

7 Summary & Key Takeaways ▲
Run standard two-pointer palindrome check from both ends
At the first (and only) mismatch, branch: try skipping left OR skipping right
Each sub-check is a strict palindrome verification (no more deletions)
Return True if either sub-check passes — you only need one to succeed
If loop ends without mismatch, the string is already a palindrome → True
🎯 When to Recognise This Pattern
  • "Check palindrome after at most one deletion/change"
  • "At most k edits" with k=1 → always try both sides at first mismatch
  • Any "tolerate one error" problem where you can reconstruct using two-pointer structure
