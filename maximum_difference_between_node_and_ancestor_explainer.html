LeetCode #1026

Maximum Difference Between Node and Ancestor

Given a binary tree, find the maximum value of |node.val − ancestor.val| over all (ancestor, node) pairs where ancestor lies on the root-to-node path.

Medium Trees Track Min/Max on Path Time: O(n) Space: O(h)
1 Understanding the Problem ▾

For every node in a binary tree, there is a unique path from the root down to that node. Every node on that path (excluding the node itself) is an ancestor. We want to find the largest absolute difference between any node and any of its ancestors — globally across the whole tree.

💡 Key Insight

On any root-to-node path, the maximum |ancestor − node| is always achieved by comparing with either the minimum or maximum value seen among all ancestors. We don't need every ancestor — just carry a running min and max as we recurse down.

Edge Cases: Single-node tree → return 0 (no ancestor exists). All nodes same value → 0. Skewed tree (linear chain) → still O(n); the path grows to length n but we only track 2 values. Very deep vs. shallow branches are handled naturally since DFS visits every root-to-leaf path.

2 The Algorithm: Pass Min/Max Down via DFS ▾

Instead of comparing each node against every ancestor (O(n²)), observe that the widest gap to any ancestor is determined by just two values: the smallest and largest ancestor seen so far on the current path. We carry those two values as parameters as we recurse down the tree.

1 Start DFS from root: Initialize cur_min = cur_max = root.val. The root is the first potential ancestor for all other nodes, so both extremes begin at the root's value.
2 At each node, update the running extremes: cur_min = min(cur_min, node.val) and cur_max = max(cur_max, node.val). This registers the current node as an ancestor for all nodes below it.
3 Base case — null child: We've passed a leaf. The answer for this complete root-to-leaf path is cur_max − cur_min. Since every node on the path is an ancestor of the next, and both extreme values come from this path, this captures the widest difference.
4 Recurse both subtrees and return max: Each child gets the updated cur_min and cur_max. Bubble up the maximum answer from both children.
🔑 Why Track Both Min AND Max?

We need |ancestor − node|. Absolute difference is maximized when the pair is as far apart as possible. Two cases: a large ancestor with a small descendant (cur_max − node), or a small ancestor with a large descendant (node − cur_min). Tracking both extremes covers both cases. At a null leaf, cur_max − cur_min = max(cur_max − cur_min, cur_min − cur_max) since it's always non-negative.

3 Why It Works: The Invariant ▾
🔐 The Invariant

When dfs(node, cur_min, cur_max) is called, cur_min and cur_max are the minimum and maximum values seen on the path from the root to node's parent (inclusive). Every value on this path is a valid ancestor of node — and after we update min/max with node.val, they include node itself for its children.

Why the null-leaf base case is correct: When we reach None, cur_min and cur_max already incorporate every node from root through the leaf (including the leaf's value after the update step on the previous call). So cur_max − cur_min is the widest possible |u − v| for any two values on this root-to-leaf path, where u and v have an ancestor-descendant relationship.

Mathematical justification: For any two nodes u (higher) and v (lower) on the same root-to-leaf path, |u.val − v.val| ≤ max(cur_max, cur_min) is not tight — what IS tight is cur_max − cur_min which equals max over all pairs on this path of |a − b|. This is because the minimum and maximum of a set always determine the widest range.

⚠️ Common Confusion: Evaluating at the Leaf vs. at None

Some solutions check if not node.left and not node.right at the leaf node itself and return immediately. This also works but requires special-casing single-child nodes and is more verbose. The null-child base case is cleaner: update cur_min/max first (so this node is counted as an ancestor for its children), then pass both recurse calls — both null children return the same value anyway. max(v, v) = v.

Solution 1 — Pass Min/Max Down (Optimal) — Interview Pick

def maxAncestorDiff(root):
def dfs(node, cur_min, cur_max): ← inner helper; carries path extremes downward
if not node: ← past a leaf: path is complete
return cur_max - cur_min ← widest range = best diff on this path
cur_min = min(cur_min, node.val) ← include this node as a potential future ancestor
cur_max = max(cur_max, node.val) ← track the maximum seen so far
left = dfs(node.left, cur_min, cur_max)← recurse: left branch gets updated extremes
right = dfs(node.right, cur_min, cur_max)← recurse: right branch gets same updated extremes
return max(left, right) ← bubble up the global best from both branches
return dfs(root, root.val, root.val) ← kick off: root.val is the first ancestor value
⚠️ Why initialize cur_min = cur_max = root.val?

The root has no ancestor above it. We could initialize with float('inf') and float('-inf'), but then the root's value would always be between them, which works but is conceptually sloppy. Starting with root.val is accurate: the root IS an ancestor for all other nodes, so it enters the running min/max from the start.

Solution 2 — Brute Force: Carry Full Ancestor List

def maxAncestorDiff(root):
res = 0
def dfs(node, path): ← path holds all ancestor values
nonlocal res
for anc_val in path: ← compare current node with EVERY ancestor — O(depth)
res = max(res, abs(node.val - anc_val))
if node.left: dfs(node.left, path + [node.val])
if node.right: dfs(node.right, path + [node.val])
dfs(root, [])
return res
💡 Complexity Summary

Pass Min/Max (Optimal): Time O(n) — each node visited once. Space O(h) — recursion stack depth equals tree height. O(log n) for balanced, O(n) for skewed (linked-list-like) tree.
Brute Force: Time O(n·h) — each node compares with all its ancestors. Space O(n) — path list grows with depth and is copied at each level.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Tree DFS Pattern

"Each node needs to compare itself with its ancestors. Ancestors are exactly the nodes on the root-to-node path — the call stack of a DFS. This means DFS can naturally carry information down from parent to child as function parameters."

Step 2: Reduce What You Need to Carry

"Do I need the full ancestor list? No. I need to maximize |ancestor − node|, which is maximized when the ancestor is as extreme as possible. Only two values matter: the minimum and maximum seen so far among all ancestors. Everything else can be discarded."

Step 3: Define Base Case and Return Value

"At a null node, the path is complete. cur_max − cur_min is the best |u−v| for any pair on this path. Return it. Each internal node returns max(left_answer, right_answer), propagating the global best upward."

Step 4: Code it Up and State Complexity

"Initialize dfs(root, root.val, root.val). At each call: if null return cur_max−cur_min. Else update cur_min, cur_max, recurse both children, return max. Time O(n), Space O(h). Clean and correct."

🗣️ What to Say to the Interviewer

"Each node needs to compare with all its ancestors on the root-to-node path.
Instead of tracking all ancestors, I'll carry just cur_min and cur_max down via DFS.
At a null child, the path is complete: return cur_max − cur_min.
This captures the maximum |ancestor − node| for this entire path.
Each node returns the max from its two subtrees, bubbling the answer up.
Time O(n) — one visit per node. Space O(h) — recursion stack."
🔮 Likely Follow-up Questions
  • "Can you do it iteratively?" → Yes: use an explicit stack of (node, cur_min, cur_max) tuples
  • "What if we want the minimum difference instead?" → Track a different aggregate; compare node.val to cur_min and cur_max at each node (not at null)
  • "What if the tree can have negative values?" → No change needed; min/max tracking handles negatives naturally
  • "What if it's not a binary tree, but an N-ary tree?" → Same approach: iterate over all children instead of just left/right

7 Summary & Key Takeaways ▾
Pass information down the tree via DFS parameters — the call stack naturally models the ancestor path
Track only cur_min and cur_max: two values replace the need for an explicit ancestor list
Evaluate at null (past leaf), not at the leaf itself — cleaner base case that includes the leaf in the path extremes
Initialize cur_min = cur_max = root.val — the root is the first ancestor for everyone else
Time O(n), Space O(h) — optimal for this problem class; every node visited exactly once

🎯 When to Recognize "Track Min/Max on Path"
  • Problem asks to compare each node with its ancestors (nodes on the root-to-node path)
  • "For each node, find the max/min over all ancestors" → carry extremes as parameters
  • You need to propagate state top-down (root → leaves), not bottom-up
  • The optimal value among ancestors is an extreme (min or max) — not a specific node
