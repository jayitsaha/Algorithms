LeetCode #367

Valid Perfect Square

Given a positive integer num, return true if num is a perfect square, and false otherwise. You must not use any built-in library function such as sqrt.

Easy Binary Search BS on Answer Time: O(log n) Space: O(1)
1 Understanding the Problem ▾

A perfect square is an integer that equals some positive integer multiplied by itself: 1 = 1×1, 4 = 2×2, 9 = 3×3, 16 = 4×4, 25 = 5×5. The question is: does num equal k² for some positive integer k?

💡 Key Insight

The constraint "do not use sqrt()" is a deliberate signal: we must search for the integer square root ourselves. The candidate k lies in [1, num], and since k² is strictly increasing, we can apply binary search — halving the search space at each step rather than checking every value linearly.

✓ Perfect Square: num = 16

k=1 → 1<16, k=2 → 4<16
k=3 → 9<16, k=4 → 16=16 ✓
→ True

✗ Not a Perfect Square: num = 14

k=3 → 9<14
k=4 → 16>14
No integer k satisfies k²=14 → False

Edge cases: num=1 (1 = 1×1 → True), large values near 2³¹−1 (watch for overflow in languages with 32-bit ints when computing mid×mid — use 64-bit), and prime numbers (always False since no integer factor pair gives a square).

🔑 Why Binary Search Applies Here

The function f(k) = k² is strictly monotonically increasing for k ≥ 1. This is the core requirement for binary search. If k² < num, every candidate smaller than k is also too small — we can discard the entire left half. If k² > num, discard the right half. Each comparison halves the remaining candidates, giving O(log n) instead of O(√n) for linear scan.

2 The Algorithm: Binary Search on the Answer ▾

We perform binary search on the candidate value k (not on an array). The search space is [1, num]. At each step we test mid * mid against num and shrink the range accordingly.

1 Initialize bounds: lo = 1, hi = num. The integer square root (if it exists) must live in this range. We'll close in on it.
2 Compute midpoint: mid = (lo + hi) // 2 — our candidate square root this iteration. Then sq = mid * mid.
3 Three-way compare: (a) If sq == num → found it, return True. (b) If sq < num → k is too small, set lo = mid + 1. (c) If sq > num → k is too large, set hi = mid - 1.
4 Exhausted search: If the loop ends (lo > hi) without an exact match, return False — no integer square root exists.
🔑 Bonus: The Odd Number Subtraction Trick

Perfect squares follow a beautiful mathematical pattern: k² = 1 + 3 + 5 + … + (2k−1) (sum of first k odd numbers). So: subtract consecutive odd numbers 1, 3, 5, 7, … from num. If you reach exactly 0, it's a perfect square. This gives O(√n) but is elegant to mention in interviews as a bonus insight.

3 Why It Works: The Invariant ▾
🔐 The Loop Invariant

At every iteration, if a valid integer square root of num exists, it lies within the interval [lo, hi]. We narrow this interval each step without ever excluding the true answer.

Why the invariant is maintained at each step:

  • Initialization: lo=1, hi=num. Any integer square root of num is at most num (since 1²=1 ≤ num), so the invariant holds at start.
  • Case sq < num: mid²<num means mid is not the root. Since f is strictly increasing, any root must be > mid. Setting lo=mid+1 preserves the invariant.
  • Case sq > num: mid²>num means mid is too large. Any root must be < mid. Setting hi=mid−1 preserves the invariant.
  • Case sq == num: mid is exactly the root. Return True immediately.

Termination: Each step either finds the answer or reduces the interval size by at least 1. Eventually lo > hi → loop exits → return False.

⚠️ Integer Overflow in C++ / Java

mid * mid can overflow 32-bit integers when num approaches 2³¹−1 ≈ 2.1×10⁹ (so mid could be ~46,341, and 46341² ≈ 2.1×10⁹ which just barely fits, but higher values overflow). In C++, declare long long sq = (long long)mid * mid;. In Java, use long sq = (long)mid * mid;. In Python there is no overflow — arbitrary precision integers. Always mention this proactively.

Efficiency: Starting with num candidates, binary search takes at most ⌈log₂(num)⌉ iterations. For a 32-bit integer, that's at most 31 steps — we check 31 candidates to answer definitively among 2 billion possibilities.

4 The Code (Annotated) ▾

Solution 1 — Binary Search on Answer (Optimal, Interview Pick)

def isPerfectSquare(num: int) -> bool:
lo, hi = 1, num ← Search space: candidate k in [1, num]
while lo <= hi: ← Inclusive bounds: check single-element ranges too
mid = (lo + hi) // 2 ← Integer midpoint (no overflow risk in Python)
sq = mid * mid ← mid²: the square of our candidate root
if sq == num: ← Exact match: mid is an integer square root
return True
elif sq < num: ← Too small: root must be greater, move lo up
lo = mid + 1
else: ← Too large: root must be smaller, move hi down
hi = mid - 1
return False ← lo > hi: exhausted, no integer root exists
⚠️ Common Mistake: Using while lo < hi

Using strict while lo < hi will miss the final single-element range where lo == hi. That last remaining candidate might be the answer. Always use while lo <= hi for the three-way branch binary search template.

Solution 2 — Odd Number Subtraction (O(√n), elegant math trick)

def isPerfectSquare(num: int) -> bool:
odd = 1 ← Start: subtract 1 (= 1²), then 3 (= 2²-1²), ...
while num > 0: ← Keep subtracting consecutive odd numbers
num -= odd ← Subtract 1, then 3, then 5, then 7, …
odd += 2 ← Move to next odd number
return num == 0 ← If exactly 0: was k² for some k → True

Solution 3 — Newton's Method (O(log n), advanced)

def isPerfectSquare(num: int) -> bool:
x = num ← Initial overestimate of sqrt(num)
while x * x > num: ← Refine until x is floor(sqrt(num))
x = (x + num // x) // 2 ← Newton's update: converges quadratically
return x * x == num ← x is now floor(sqrt(num)); check exact

7 Summary & Key Takeaways ▾
Binary search on the answer — you don't need a sorted array; you need a monotone function. Here, f(k) = k² is monotone, so binary search applies on the value k.
"No built-in sqrt()" = deliberate hint to implement integer square root search yourself via binary search or Newton's method.
The invariant: if an integer square root exists, it always remains in [lo, hi] throughout the search — never mistakenly excluded.
The odd-number math trick (1+3+5+…+(2k−1)=k²) is a beautiful O(√n) alternative that shows mathematical depth in interviews.
Always mention integer overflow for mid*mid when working in C++/Java (use long long / long). Python is overflow-safe.
This is the foundational "binary search on answer" pattern — the same template solves Sqrt(x), Koko Eating Bananas, Capacity to Ship, and many others.
🎯 When to Use Binary Search on the Answer
  • The answer is a number in a known bounded range
  • You can write a check function f(k) that is monotone (all feasible ≤ threshold, all infeasible above it)
  • Problems asking "minimum k such that condition holds" or "is X achievable?"
  • Keywords: "minimize maximum", "maximum minimum", "at least K", "no more than K days"
📐 The Odd Number Math (Bonus Insight)

Sum of first k odd numbers = k². Geometrically: each odd number adds a new L-shaped border to a square. 1-square (1) + L-shape of 3 = 2×2. + L-shape of 5 = 3×3. + L-shape of 7 = 4×4. So: subtracting 1, 3, 5, 7, … from num until 0 (perfect square) or negative (not) is a valid O(√n) algorithm.
