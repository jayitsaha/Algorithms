#409 · Longest Palindrome

Find the length of the longest palindrome buildable from the given characters
Easy String Processing Count Pairs + One Odd Time O(n) Space O(1)
1 Understanding the Problem ▼

We have a string of characters. We can rearrange them in any order. The question: what is the length of the longest palindrome we can form? (We don't need to return the palindrome itself — just the length.)

💡 Key Insight

A palindrome mirrors around its center. Every character on the left side has a twin on the right side. This means characters must come in pairs — one for each side — except at most one character which can sit alone in the center.

What makes a palindrome? The pair structure:

Example: s = "abccccdd" → character counts:

  • 'a' appears 1× (odd) → contributes 0 to pairs; can be center
  • 'b' appears 1× (odd) → contributes 0 to pairs; can also be center
  • 'c' appears 4× (even) → contributes 4 (2 pairs, both sides)
  • 'd' appears 2× (even) → contributes 2 (1 pair, both sides)

Total: 0 + 0 + 4 + 2 = 6, plus +1 for the center (we pick 'a' or 'b') = 7.

2 The Algorithm: Count Pairs + One Odd ▼

The approach is greedy: maximize how many characters we can use, given palindrome constraints.

1 Count frequencies. Build a frequency map of all characters. This takes O(n) time.
2 Harvest even pairs. For each frequency count, add count − (count % 2). This is the largest even number ≤ count. Even count → take all. Odd count → take count−1.
3 Flag odd counts. If any character has an odd count, set has_odd = True. That "leftover" character can go in the center.
4 Return length + (1 if has_odd else 0). The center bonus is at most +1, regardless of how many odd counts exist.
🔑 The Core Formula

For each character frequency count:
Even portion used = count − (count % 2)

This equals count when even, or count − 1 when odd.
Any "wasted" leftover (one at most per odd character) qualifies for the single center slot.

3 Why It Works: The Invariant ▼
🔐 The Invariant

A palindrome requires every non-center character to appear an even number of times across the whole string. Our algorithm greedily takes the maximum even portion of each character's count — count − (count % 2) — which is provably the maximum we can use from that character without violating palindrome structure.

Because at most one center position exists, at most one odd remainder can be promoted to the center. The algorithm flags any odd count but adds only +1 total, not +n.

Proof of Optimality

  • We can't use more than count of any character (it doesn't appear more)
  • We can't use an odd count on both sides — that would make the string asymmetric
  • We can place at most 1 character in the center
  • ∴ Our greedy take is maximal at every step → total is globally maximal ✓

Edge Cases

  • All even counts — has_odd = False, no center; return len(s)
  • Single char — s = "a" → length = 0, has_odd = True → return 1 ✓
  • All same — s = "aaaa" → length = 4, has_odd = False → return 4 ✓
  • All distinct — s = "abc" → length = 0, has_odd = True → return 1 ✓
  • Case sensitive — 'A' and 'a' are treated as different characters
4 The Code (Annotated) ▼

Solution 1 — Frequency Count + Greedy  (Interview Pick)

from collections import Counter def longestPalindrome(s: str) -> int: freq = Counter(s) # O(n) scan; at most 52 unique chars length = 0 # accumulate palindrome length has_odd = False # any character with a leftover? for count in freq.values(): length += count - (count % 2) # take all even pairs (floor to even) if count % 2 == 1: has_odd = True # a leftover char exists return length + (1 if has_odd else 0) # center bonus
⚠️ Common Mistake: Forgetting the center

Students often just sum the even pairs and forget +1 for the center. s = "a" → even sum = 0, but correct answer is 1 (the single 'a' goes in the center). Always add +1 if has_odd is True.

Solution 2 — Set-based Pairing  (Intuitive "Sock Matching")

def longestPalindrome(s: str) -> int: unpaired = set() # chars seen an odd number of times so far length = 0 for c in s: if c in unpaired: # found its pair! length += 2 unpaired.remove(c) # pair consumed else: unpaired.add(c) # still waiting for its pair return length + (1 if unpaired else 0) # any leftover → center
💡 Same Idea, Different View

Solution 2 "matches socks" in real time. Every time you find a pair, count +2. At the end, if any unpaired character remains in the set, one of them becomes the center (+1).

Solution 3 — One-liner Shortcut

from collections import Counter def longestPalindrome(s: str) -> int: odd_count = sum(v % 2 for v in Counter(s).values()) return len(s) - odd_count + (1 if odd_count else 0)

Start with len(s). Each odd-count character wastes 1 (the leftover), so subtract odd_count. But if any waste was present, one reclaimed char goes in the center: +1.

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Recognize the Pattern

Signals: We're building a symmetric structure (palindrome) from a pool of characters. Symmetry → pairing. "How many characters can we use?" with pairing constraints → frequency count + greedy.

Step 2: Explore Palindrome Requirements

Ask: "What rules must a palindrome follow?" Every character except at most one must appear an even number of times across the string. So: maximize even-count usage, then optionally add 1 center.

Step 3: Formalize the Decision

For each character count: even → use all. Odd → use count − 1 (the biggest even ≤ count). Flag that an odd count exists. After processing all chars: if flag set, add 1 for the center.

Step 4: Code and Analyze

Counter(s) in one line. Loop over .values(). Accumulate with count − count%2. Track has_odd. Return length + has_odd. Time O(n), Space O(1).

🗣️ What to Say to the Interviewer

"A palindrome mirrors around its center, so every character must appear in pairs — one for each side — except at most one that can sit in the center. So my approach: count character frequencies, then for each count take the largest even portion (count minus count-mod-2). If any count was odd, I can add +1 for the center. This greedily maximizes the palindrome length. Time is O(n) for counting, space is O(1) since there are only 52 possible characters."
🎯 Potential Follow-ups
  • "Return the actual palindrome string" — Track which char is center; sort remaining pairs; build left half + center + reversed left half
  • "What if we can only use each character at most once?" — Then each char contributes at most 1 to a pair, so collect chars that appear ≥ 2 times
  • "At most k different characters" — Add a filter step before the frequency count
  • "Streaming input" — Maintain a running frequency map and running odd-count tally; answer is n − odd_count + (1 if odd_count else 0)

7 Summary & Key Takeaways ▼
✓ In a palindrome, every character except at most one must appear in pairs
✓ Even-count characters contribute fully; odd-count characters contribute count − 1
✓ At most ONE character can go in the center, providing a +1 bonus
✓ Solution is greedy — the local optimal choice (take max even) is globally optimal
✓ Time O(n), Space O(1) — this is the optimal solution
