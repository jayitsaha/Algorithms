LeetCode #75

Sort Colors

Given an array nums with n objects colored red (0), white (1), or blue (2), sort them in-place so that objects of the same color are adjacent, with colors in order red, white, blue. Do not use the built-in sort function. Solve it in one pass with O(1) extra space.

Medium Sorting Dutch National Flag Time: O(n) Space: O(1)
1 Understanding the Problem ▾

We have an array containing only three values: 0 (red), 1 (white), and 2 (blue). We need to rearrange it in-place so all 0s come first, all 1s in the middle, and all 2s at the end — like sorting a flag into its three coloured bands.

💡 Key Insight

Because there are only three possible values, we don't need a general-purpose sort. We can maintain three regions simultaneously: a "confirmed-0s" band growing from the left, a "confirmed-2s" band growing from the right, and an "unknown" middle region. A single scan pointer moves left to right, immediately placing every element it sees into the correct band. When the scan pointer crosses the right boundary, the unknown region is empty and the array is sorted.

Input — unsorted

[2, 0, 2, 1, 1, 0]
0s scattered at idx 1, 5
1s at idx 3, 4
2s at idx 0, 2

Output — sorted in-place

[0, 0, 1, 1, 2, 2]
0s → left band
1s → middle band
2s → right band

Edge cases to consider: all same value (array already sorted, zero swaps), single element (trivially sorted), [2,1] (one swap), all 1s (no swaps, only mid advances), array of length 1 (lo=mid=hi=0, one iteration).

⚠️ Why Not Just Count and Fill?

A two-pass approach — count each colour, then overwrite — works in O(n)/O(1) and is correct. But the problem explicitly asks for one pass. In an interview, mention count-and-fill first as the brute-force O(n) approach, then offer the Dutch National Flag algorithm as the one-pass optimal solution.

2 The Algorithm: Dutch National Flag (3-Way Partition) ▾

The Dutch National Flag algorithm, invented by Edsger W. Dijkstra (1976), partitions an array into three groups using three pointers in a single pass. Picture the array always divided into four regions:

Confirmed 0s
[0 .. lo)
← lo boundary
Confirmed 1s
[lo .. mid)
mid scans →
Unknown
[mid .. hi]
← shrinks →
Confirmed 2s
(hi .. n)
hi boundary →
1 Initialize: lo = 0, mid = 0, hi = n−1. The entire array is "unknown." The confirmed-0s and confirmed-2s zones are initially empty.
2 Examine nums[mid]. Three cases — if it's 0: swap it to the left (swap mid↔lo, then lo++, mid++). If it's 1: already correct, just mid++. If it's 2: swap it to the right (swap mid↔hi, then hi−−, but do NOT advance mid).
3 Repeat until mid > hi. The unknown region is empty, all elements are correctly placed, and the array is sorted in exactly one pass.

The three cases in detail:

nums[mid] == 0 (Red)
swap(lo, mid)
lo += 1
mid += 1
Red belongs left. The element displaced from lo was a confirmed 1 — it's safe to call it examined. Advance both pointers.
nums[mid] == 1 (White)
mid += 1
White is already in the right zone (between lo and hi). Simply consume it and advance mid. No swap needed.
nums[mid] == 2 (Blue)
swap(mid, hi)
hi -= 1
# do NOT mid += 1
Blue belongs right. The element arriving from hi is unknown — could be 0, 1, or 2. Keep mid fixed to examine it next iteration.
3 Why It Works: The Invariant ▾
🔐 The Invariant (maintained after every iteration)

After each execution of the while-loop body, the following four conditions all hold simultaneously:
• nums[i] == 0 for every i in [0, lo)
• nums[i] == 1 for every i in [lo, mid)
• nums[i] == 2 for every i in (hi, n−1]
• nums[i] is unknown for every i in [mid, hi]

Why each case preserves the invariant:

Case 0 (swap with lo): nums[lo] is guaranteed to be a 1 — it's inside [lo, mid). After the swap, nums[lo] = 0 extends the red zone, nums[mid] = 1 is now examined and known. Both zones grow, both pointers advance, invariant holds.

Case 1 (skip): nums[mid] = 1 expands the white zone [lo, mid) rightward by one. mid advances, shrinking the unknown region. Invariant holds trivially.

Case 2 (swap with hi): nums[hi] extends the blue zone; hi retreats. The element that just landed at mid came from the unexamined region — we don't know its value. If we advanced mid, we'd falsely put an unexamined element in the white zone. So mid stays, preserving the invariant.

Termination: Each iteration either advances mid (cases 0 and 1) or retreats hi (case 2). The unknown region [mid, hi] shrinks by exactly one element per iteration. After at most n iterations, mid > hi and the region is empty. Total operations: O(n).

⚠️ The "Advance Mid on 2-Swap" Trap

The most common mistake: writing hi -= 1; mid += 1 when handling a 2. This is wrong. The element that landed at nums[mid] came from the unexamined zone — if it's a 0, it needs to go left. By advancing mid you'd include an unsorted element in the "confirmed-1s" zone, breaking the invariant and producing an incorrect output. Always leave mid fixed when retreating hi.

💡 Analogy: Sorting a Physical Flag

Imagine holding a physical Dutch flag: red on top, white in middle, blue at bottom. You grab the first stripe from a shuffled flag. If it's red, tuck it at the very top. If it's white, place it at the current middle position. If it's blue, tuck it at the very bottom — but don't immediately look at what just fell to the middle. That's Dutch National Flag in one sentence.

4 The Code (Annotated) ▾

Solution 1 — Dutch National Flag (Optimal, One-Pass O(n)/O(1)) — Interview Pick

def sortColors(nums: list[int]) -> None:
lo, mid, hi = 0, 0, len(nums) - 1← Three pointers: lo=confirmed-0s boundary, mid=scan cursor, hi=confirmed-2s boundary
while mid <= hi:← Unknown region [mid..hi] is non-empty
if nums[mid] == 0:← Red: belongs in left zone
nums[lo], nums[mid] = nums[mid], nums[lo]← Move 0 to red zone; nums[lo] was a confirmed 1
lo += 1← Grow the red zone right by one
mid += 1← The displaced 1 is now examined; safe to advance scan
elif nums[mid] == 1:← White: already in correct middle zone
mid += 1← Consume it and advance scan — no swap needed
else:← nums[mid] == 2: Blue: belongs in right zone
nums[mid], nums[hi] = nums[hi], nums[mid]← Move 2 to blue zone; bring unknown element to mid
hi -= 1← Grow the blue zone left; do NOT advance mid (new element is unexamined)
⚠️ Why advance BOTH lo and mid when swapping a 0?

When nums[mid] == 0, we swap it with nums[lo]. What was at nums[lo]? It was inside [lo, mid), the confirmed-1s zone, so it's guaranteed to be a 1. After the swap: nums[lo] = 0 (confirmed red) and nums[mid] = 1 (confirmed white). We already know the value — no need to re-examine. Both pointers advance.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Read the Constraint Carefully

"Only three distinct values (0, 1, 2), sort in-place, one pass." Three values is the signal — you don't need a general sort. Think partitioning. Two values → two-pointer partition (like Move Zeroes). Three values → three-pointer partition (Dutch National Flag).

Step 2: Propose the Two-Pass Solution First

"The intuitive approach: count each colour in one pass, then overwrite in a second pass — O(n) time, O(1) space. But since the follow-up asks for one pass, I'll think about doing both simultaneously."

Step 3: Derive DNF from First Principles

"I want to place 0s left and 2s right in a single scan. If I use three pointers — lo for the end of confirmed 0s, hi for the start of confirmed 2s, and mid to scan — then every element I see can immediately go to its correct region. That's the Dutch National Flag pattern."

Step 4: Call Out the Subtlety Proactively

"Important edge case: when I swap a 2 to the right, I must NOT advance mid. The element that came from the hi position is from the unexamined region — could be anything. When I swap a 0 to the left, I CAN advance mid, because the element at lo was in the confirmed-1s zone — it's a guaranteed 1."

🗣️ What to Say to the Interviewer

"I'll use Dutch National Flag with three pointers: lo=0, mid=0, hi=n-1.
While mid <= hi, inspect nums[mid]:
- If 0: swap with lo, advance lo and mid (displaced element was a confirmed 1).
- If 1: advance mid, no swap.
- If 2: swap with hi, retreat hi, keep mid (swapped-in element is unexamined).
Single pass, O(n) time, O(1) space."
🔮 Likely Follow-up Questions
  • "What if there are k distinct values?" → k-way partition; general case → counting sort (O(n+k))
  • "Sort 0s and 1s only (two values)?" → Two-pointer partition, like Move Zeroes or Remove Element
  • "Who invented this?" → Edsger W. Dijkstra, 1976; named after the Dutch flag (red/white/blue)
  • "What's the relation to quicksort?" → DNF is the 3-way partition step in Bentley-McIlroy 3-way quicksort, which handles equal elements in O(n) — critical for many-duplicates inputs

Dutch National Flag = 3 pointers, 3 regions, single pass, O(1) space
