LeetCode #686

Repeated String Match

Given two strings a and b, return the minimum number of times a must be repeated so that b is a substring of the result. Return -1 if it is impossible.

Medium String Processing Repeat Until Match Time: O(n·m) Space: O(n·m)
1 Understanding the Problem ▾

We have string a and string b. We can repeat a end-to-end as many times as we like. The question: what is the fewest repetitions needed so that b appears somewhere inside that repeated string?

a ="abcd"
b ="cdabcdab"
a × 2 ="abcdabcd"✗ b not here
a × 3 ="abcdabcdabcd"✓ b found at index 2!
answer =3
💡 Key Insight

You only ever need to check two candidate counts: min_reps = ⌈len(b) / len(a)⌉ and min_reps + 1. The first ensures the repeated string is at least as long as b. The +1 covers the edge case where b starts near a copy boundary. If b isn't found in either, it is mathematically impossible — more copies won't help.

✓ Possible: a="ab", b="ababab"

min_reps = ⌈6/2⌉ = 3
a×3 = "ababab" ← b is there!
Answer = 3

✗ Impossible: a="aa", b="ba"

a×n = "aaaa...aa" (all 'a')
'b' never appears → -1
Chars in b not in a → impossible

Edge cases to consider: b already in a (answer = 1), a has characters not in b (answer = -1), empty b (answer = 0 or 1 by convention), b much longer than a (large min_reps but still just two checks).

2 The Algorithm: Repeat Until Long Enough, Then Check ▾

The approach is clean and relies on a mathematical bound: instead of blindly repeating and checking every step, we compute the minimum possible answer and try just two values.

1 Compute minimum repeats: min_reps = math.ceil(len(b) / len(a)). This is the fewest copies that make the repeated string at least as long as b. Without this many characters, b simply cannot fit.
2 First check — min_reps copies: Build repeated = a * min_reps. Use Python's b in repeated (an O(n·m) optimized substring search). If found, return min_reps.
3 Second check — min_reps + 1 copies: Try b in repeated + a. This handles the boundary case where b starts near the end of one copy and extends into the next. If found, return min_reps + 1.
4 Return -1: If b still isn't found, it is impossible. No additional repetitions can create a substring not implied by the character set and period of a.
🔑 Why Exactly Two Checks Suffice

In any occurrence of b inside a * ∞, b's first character can start at offset 0 to len(a) - 1 within a copy. The worst case is offset len(a) - 1 (last character of a copy), from which b needs len(b) more characters. Total copies required: ⌈((len(a) - 1) + len(b)) / len(a)⌉ = ⌈len(b)/len(a)⌉ + 1. So min_reps + 1 always covers the worst case — we never need min_reps + 2 or more.

3 Why It Works: The Invariant ▾
🔐 The Invariant

If b appears anywhere in a repeated infinitely, it must appear within a * (⌈len(b)/len(a)⌉ + 1). Every occurrence in the infinite string is a shifted copy of an occurrence in this finite window.

Formal bound: Suppose b starts at position p in a * ∞. Define start_offset = p mod len(a) (where b's first char falls within its starting copy). Since start_offset ≤ len(a) - 1, the number of copies of a needed to cover from position p onwards is at most ⌈(start_offset + len(b)) / len(a)⌉ ≤ ⌈((len(a) - 1) + len(b)) / len(a)⌉ = ⌈len(b)/len(a)⌉ + 1. Including the copies before position p does not increase this bound (we start fresh from position p mod len(a)). QED.

Why the +1 is sometimes essential: Consider a="abc", b="ca". min_reps = ⌈2/3⌉ = 1. "abc" does not contain "ca". But "abcabc" (min_reps + 1 = 2 copies) contains "ca" at index 2. Without the +1 check, we'd incorrectly return -1.

⚠️ Common Mistake: Iterative Loop With No Bound

Some beginners write a loop that keeps appending copies of a until b is found or some arbitrary limit is hit. This is equivalent but slower in Python (O(n²) string concatenation). Always prefer a * min_reps over repeated += in a loop — the multiplication builds the string in a single C-level allocation.

Why it's impossible when it is: If b contains a character not in a, no repetition of a will ever produce that character. More subtly, even if all characters match, b might require a character sequence that never cycles into place within a's period — the +1 check catches the algebraic impossibility cleanly.

4 The Code (Annotated) ▾

Solution 1 — Ceil + Two Checks (Interview Pick, O(n·m) time)

import math
def repeatedStringMatch(a: str, b: str) -> int:
min_reps = math.ceil(len(b) / len(a)) ← Minimum copies to be at least as long as b
repeated = a * min_reps ← Single O(1) allocation, faster than += loop
if b in repeated: ← Python's built-in uses optimized C search
return min_reps ← b fits entirely within min_reps copies
if b in repeated + a: ← One more copy handles boundary straddle
return min_reps + 1 ← b straddles the boundary between copies
return -1 ← Impossible: b cannot be formed from repetitions
💡 Why math.ceil(len(b) / len(a)) is the magic formula

a="abc" (len 3), b="abcab" (len 5): ⌈5/3⌉ = 2. a×2 = "abcabc" contains "abcab" at index 0. Answer = 2. a="abc", b="cabcab" (len 6): ⌈6/3⌉ = 2. a×2 = "abcabc", "cabcab" not there. a×3 = "abcabcabc" contains "cabcab" at index 2. Answer = 3 (min_reps + 1).

Solution 2 — Incremental Loop (Easier to Derive, Slightly Slower)

def repeatedStringMatch(a: str, b: str) -> int:
repeated = a ← Start with one copy
count = 1 ← Number of copies built so far
while len(repeated) < len(b): ← Keep building until at least as long as b
repeated += a ← WARNING: O(n²) total due to string copy each time
count += 1 ← Track repetition count
if b in repeated: return count ← Check with minimum copies
if b in repeated + a: return count + 1 ← Try one more for boundary case
return -1 ← Impossible
⚠️ String Concatenation in Loops is O(n²) in Python

Each repeated += a creates a brand-new string of length len(repeated) + len(a), copying all characters. After k iterations, total work is O(k · len(a)) per step → O(k² · len(a)) total. Use a * min_reps (Solution 1) which Python optimizes to a single allocation.

🔑 Advanced: KMP on Cyclic String (O(n+m), no allocation)

For the theoretical optimum, adapt KMP to treat a as a cyclic string: when the index into a reaches len(a), wrap it back to 0 and increment a repetition counter. This avoids allocating the repeated string entirely. Complexity: O(n + m) time, O(m) space for the failure function. Mention this in interviews when asked about optimizations beyond the ceil approach.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Reframe the Problem

Say: "This is equivalent to asking: is b a substring of the infinite string a·a·a·...? I just need to find the minimum prefix of that infinite string that contains b." Signals: "cyclic/repeated string", "minimum repetitions", "b is a substring".

Step 2: Bound the Search Space

Don't try infinite candidates. Key question: "How many copies do I need at minimum?" Answer: ⌈len(b)/len(a)⌉. Then: "Could I ever need one more?" Yes — when b straddles a copy boundary. "Two more?" No — I can prove it's bounded at +1.

Step 3: Code the Two-Check Pattern

Write the ceil, build repeated, check, then check +1, then -1. Mention: "Python's 'in' uses an optimized C-level search — in practice much faster than O(n·m) worst case." Show you know the complexity.

Step 4: Know the Upgrade Path

If asked to optimize: "I'd use KMP with modular index arithmetic — treat position mod len(a) to simulate the cyclic repetition without materializing the string. O(n+m) time, O(m) space."

🗣️ What to Say to the Interviewer

"I see this as: is b a substring of a repeated infinitely?
