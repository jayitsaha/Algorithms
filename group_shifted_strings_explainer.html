LeetCode #249

Group Shifted Strings

Given a list of strings, group all strings that belong to the same shifting sequence together.

Medium Hash Tables Shift Pattern as Key Time: O(n·k) Space: O(n·k)
1 Understanding the Problem ▾

A shifting sequence is formed by repeatedly shifting every character of a string forward by one position in the alphabet — a→b→c→…→z→a (it wraps around). For example, "abc" can shift to "bcd", then "cde", all the way to "xyz", then "yza", "zab", and eventually back to "abc". All such strings belong to the same group.

💡 Key Insight

Two strings belong to the same shifting group if and only if their relative differences between consecutive characters are identical. We can compute a canonical "shift fingerprint" for any string — a tuple of inter-character gaps mod 26 — and use it as a hash map key to bucket all same-pattern strings together.

Think of it this way: "abc" has gaps [1, 1] (b−a=1, c−b=1). "xyz" also has gaps [1, 1]. "zab" has gaps [(a−z+26)%26=1, (b−a)=1] — also [1, 1]. They all belong together. Meanwhile, "ace" has gaps [2, 2] — a completely different group.

2 The Algorithm: Canonical Shift Key via Hash Map ▾

Instead of comparing pairs of strings (expensive and complex), we compute a canonical fingerprint for each string that is invariant to shifting. Then we use a hash map to group strings by fingerprint.

The fingerprint is the tuple of consecutive character gaps mod 26. For string s of length k, the key has k−1 values: ((s[1]-s[0]) % 26, (s[2]-s[1]) % 26, …, (s[k-1]-s[k-2]) % 26). Length-1 strings get the empty tuple ().

1 Create an empty hash map: groups = defaultdict(list). Keys will be shift-pattern tuples; values are lists of strings sharing that pattern.
2 For each string, compute its canonical key: Build a tuple of consecutive character differences mod 26. Use % 26 to handle wraparound correctly — the gap from 'z' to 'a' is 1, not -25.
3 Bucket the string: groups[key].append(s). Multiple strings sharing the same key accumulate in the same list automatically via defaultdict.
4 Return the groups: return list(groups.values()). Each value is a list of all strings in the same shifting sequence.
Brute Force — Wrong Direction

For each pair, try all 26 possible shifts to see if one becomes the other. O(n²·k·26) time — far too slow, and error-prone with wraparound. Avoid this approach entirely.

Canonical Key — Optimal

Normalize each string to its invariant fingerprint in O(k) time. Hash map lookup + insert is O(k) for the key. Total O(n·k). No pairwise comparison needed!

⚠️ The Wraparound Trap — The #1 Bug

The gap from 'z' to 'a' is NOT ord('a') - ord('z') = -25. It must be (ord('a') - ord('z')) % 26 = 1. Without % 26, strings like "az" and "ba" would get keys 25 and -1 respectively — different keys for what should be the same group! Always compute differences as (ord(s[i]) - ord(s[i-1])) % 26.

3 Why It Works: The Invariant ▾
🔐 The Shift-Invariance Theorem

Shifting all characters of a string by the same constant d leaves the pairwise consecutive differences completely unchanged. If gap[i] = (s[i+1] - s[i]) % 26, then after adding d to every character modulo 26:
gap[i]' = ((s[i+1]+d) - (s[i]+d)) % 26 = (s[i+1] - s[i]) % 26 = gap[i].
The gap tuple is therefore invariant to any global shift — it is the perfect canonical key.

Forward direction (same gaps → same group): If two strings s and t have the same gap tuple, shift s by d = (ord(t[0]) - ord(s[0])) % 26. This makes their first characters match. Since all gaps are equal, every subsequent character must also match. Therefore s shifted by d exactly equals t.

Backward direction (different gaps → different group): If any gap position differs, no single global shift d can simultaneously fix the first character discrepancy and maintain the gap discrepancy. The two strings cannot be in the same shifting sequence.

Single-character strings: All length-1 strings get the empty tuple () as their key. This is mathematically correct — there are no consecutive pairs to form gaps, so the gap tuple is empty and identical for all single-char strings. You can always shift one letter to any other.

⚠️ Length Must Match — Automatically Handled

Strings of different lengths trivially produce tuples of different length, which are never equal in Python. Our algorithm handles this without any explicit length check — the canonical keys will simply never collide across different-length strings.

💡 Complexity

Time: O(n·k) — n strings each of max length k. Computing the key is O(k), hashing a length-(k-1) tuple is O(k).
Space: O(n·k) — storing all strings (O(n·k) total chars) plus all keys (O(n·k) total gap values) in the hash map.

🔑 Which Approach to Use in an Interview?

Both are O(n·k). The gap-tuple approach (Solution 1) maps directly to the mathematical insight and has a k−1 element key (slightly smaller). The normalize-to-'a' approach (Solution 2) is visually intuitive — "pretend the string starts with 'a'". Use whichever you can explain clearly; interviewers accept both.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Understand the Grouping Criterion

Confirm: two strings are in the same group if you can shift all characters of one by the same constant to get the other, with wraparound (z → a). Then observe: shifting cannot change string length — so different-length strings are always in different groups. This simplifies the problem.

Step 2: Recognize the Hash Map Grouping Template

Any problem asking to "group items that are equivalent under some transformation" maps to: hash map with canonical key. The hard part is picking the right key. Think: "What property is preserved by the transformation?" Answer: relative gaps between characters. That's your key.

Step 3: Design the Key with Modular Arithmetic

Walk through a wraparound example: "az" vs "ba". Gap for "az": (ord('z')−ord('a')) % 26 = 25. Gap for "ba": (ord('a')−ord('b')) % 26 = (−1+26) % 26 = 25. Same key! Without % 26 you'd get 25 and −1 — that's the bug. State this explicitly; it shows you've thought carefully.

Step 4: Handle Edge Cases Proactively

Single-char strings: key = empty tuple (), all grouped together. Empty input: loop doesn't run, returns []. These are easy once you have the general solution, but mentioning them shows thoroughness.

🗣️ What to Say to the Interviewer

"This is a grouping problem. I'll use a hash map where key = shift-invariant fingerprint of each string.
The fingerprint is the tuple of consecutive character differences mod 26.
Mod 26 is critical — it handles wraparound (z to a = gap 1, not -25).
For each string: compute the gap tuple, append to groups[tuple].
Single-char strings all get the empty tuple and group together.

🔮 Likely Follow-up Questions
  • "What if the alphabet isn't just a-z?" → Use the full Unicode modulus (e.g., % 26 becomes % total_chars_in_alphabet)
  • "Can you sort each group alphabetically?" → Sort inside each value list: groups[key].sort()
  • "How does this differ from Group Anagrams?" → Group Anagrams uses sorted chars as key; here we use char gaps — both are canonical hash map grouping
  • "What's the output format if there are ties?" → Problem says any order is acceptable

The shift-invariant fingerprint is the tuple of consecutive char-gaps mod 26
Always use % 26 — without it, negative differences break wraparound strings like "az" and "ba"
Hash map key = canonical form: the core template for all "group equivalent items" problems
Length-1 strings all get the empty tuple () and correctly group together
Different-length strings always produce different-length keys — automatically separated

🎯 When to Recognize "Shift Pattern as Key"
  • Problem asks to group things that are "equivalent" under a uniform transformation
  • The transformation preserves relative differences between elements
  • You need a canonical representation invariant to the transformation
  • Signal words: "shifting sequence", "cyclic rotation", "same relative pattern"
  • Generalizes to: group by modular arithmetic, group by sorted chars (anagrams), group by char-mapping (isomorphic)
