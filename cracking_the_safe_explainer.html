LeetCode #753

Cracking the Safe

Find the shortest string that contains every possible k-digit password (using digits 0..n-1) as a contiguous substring.

Hard Graph De Bruijn Sequence Eulerian Circuit Time: O(n^k) Space: O(n^k)
1 Understanding the Problem ▾

Imagine a digital safe: you enter a sequence of digits and whenever the last k digits match the password, it unlocks. You don't know the password, but you want to enter the shortest possible string that guarantees you've tried every possible k-digit combination at some point as a sliding window.

💡 Key Insight

There are nk possible k-digit passwords (each of the k positions can be any of n digits). The shortest string that contains all of them as substrings is called a De Bruijn sequence. It has length exactly nk + k − 1 — each new character typed creates exactly one new k-digit window. The key: model this as finding an Eulerian circuit in a directed graph.

Edge Cases: k=1 means single-digit passwords — answer is "012...n-1" (n chars). n=1 means only digit 0 — answer is "000...0" (k chars). The optimal length is always nk + k − 1, which grows exponentially with k.

2 The Algorithm: De Bruijn Graph + Hierholzer's Eulerian Circuit ▾

The breakthrough is mapping the problem to graph theory. Construct a De Bruijn graph: each node is a (k−1)-digit string. A directed edge from node u to node v exists if u[1:] == v[:-1] — meaning: drop the first digit of u, append any digit d, and get v. That edge represents the k-digit combination u + d.

Each edge = one k-digit combination. Every node has out-degree = in-degree = n.

An Eulerian circuit — a cycle that traverses every edge exactly once — covers all nk combinations exactly once. We find it using Hierholzer's algorithm, which runs in O(E) time via DFS with post-order collection.

1 Start from node "0"*(k-1) — the all-zeros node. Maintain a visited set of edges (encoded as node+digit strings, which are exactly the k-digit combinations).
2 DFS: try each digit 0..n-1 from the current node. For each digit d, the edge is node + str(d). If not yet visited, mark it and recurse into the next node node[1:] + str(d).
3 Post-order collection (Hierholzer's trick): After returning from recursion, append the digit d to the result list. This builds the circuit in reverse — the last edge explored gets appended first.
4 Build final string: Reverse the result list (to get forward order), then prepend the starting node "0"*(k-1). Total length = (k-1) + nk = nk + k − 1.
🔑 Why an Eulerian Circuit Exists

In the De Bruijn graph, every node representing a (k-1)-string has exactly n outgoing edges (append digit 0, 1, ..., n-1) and exactly n incoming edges (from each node that matches on k-2 chars). Since in-degree equals out-degree at every node, and the graph is strongly connected, Euler's theorem guarantees an Eulerian circuit exists.

3 Why It Works: The Invariant ▾
🔐 The Invariant

Each directed edge in the De Bruijn graph corresponds to exactly one unique k-digit combination — there is a perfect bijection between edges and combinations. Visiting every edge exactly once (Eulerian circuit) means reading off every k-digit combination exactly once as we traverse the graph. The string produced by the circuit contains each combination as a contiguous window.

Why Hierholzer's post-order trick is correct: When DFS backtracks, it has fully explored all paths forward from the current edge. By appending the digit after returning from recursion, we record edges in the order of last-exit — exactly the reverse of the Eulerian circuit traversal. Reversing the collected list reconstructs the forward Eulerian path. This avoids the classical pitfall of getting "stuck" at a dead end before visiting all edges — the post-order collection always produces a valid complete circuit.

Why the result is shortest possible: The starting (k-1) characters are fixed. Each subsequent character typed creates exactly one new k-digit window. Since there are nk combinations to cover and each new character covers exactly one, we need at least nk additional characters. Total minimum = (k-1) + nk = nk + k − 1. The De Bruijn sequence achieves this bound.

⚠️ Common Trap: Greedy Digit Order vs. Correctness

Trying digits in ascending order (0, 1, 2...) is one valid strategy — any complete ordering works. The post-order Hierholzer approach is what guarantees we don't dead-end early. A naive greedy that stops at any dead-end without backtracking would produce an incorrect incomplete sequence.

Analogy: Think of laying a path through a maze where every corridor (edge) must be walked exactly once. Hierholzer says: walk greedily, and when you hit a dead end, back up and splice your walked path into the circuit. The post-order collection does exactly this splicing automatically via recursion.

4 The Code (Annotated) ▾

Solution 1 — Hierholzer's Eulerian Circuit DFS (Optimal — Interview Pick)

def crackSafe(n: int, k: int) -> str:
visited = set() ← Visited edges, encoded as node+digit = k-char combo string
result = [] ← Post-order digit collection (Hierholzer trick)
start = "0" * (k - 1) ← Starting node is (k-1) zeros; k=1 means start=""
def dfs(node): ← DFS from current (k-1)-digit node in De Bruijn graph
for d in range(n): ← Try each digit in order (ascending gives lex-smallest result)
edge = node + str(d) ← This edge label = the k-digit combination (node prefix + digit)
if edge not in visited: ← Only traverse each k-combination once
visited.add(edge) ← Mark edge used BEFORE recursing (important: pre-mark to avoid revisit)
dfs(node[1:] + str(d)) ← Move to next node: slide window — drop first char, append digit
result.append(str(d)) ← POST-ORDER: append digit AFTER full subtree explored
dfs(start) ← Launch DFS from all-zeros node
return start + "".join(reversed(result)) ← Reverse (post-order→forward) then prepend starting node
⚠️ Why reversed(result)?

Hierholzer's post-order collects digits in the order DFS unwinds — last-in, first-out. The deepest recursive call (the last digit of the circuit) gets appended first to result. Reversing restores the correct forward traversal order. Equivalently: the result is built like a stack that we read in reverse.

Solution 2 — Backtracking (Brute Force, simpler to derive)

def crackSafe_backtrack(n: int, k: int) -> str:
target = n ** k ← Need to cover all n^k combinations
visited = set() ← Track which k-combos have been covered
result = list("0" * k) ← Start with "000...0" (covers first combo)
visited.add("0" * k) ← Mark initial combo as covered
def backtrack() -> bool:
if len(visited) == target: ← All combos covered → success
return True
node = "".join(result[-(k-1):]) ← Current (k-1)-suffix = current De Bruijn graph node
for d in range(n - 1, -1, -1): ← Try digits (descending = heuristic for fewer backtracks)
nxt = node + str(d) ← Candidate next k-combo
if nxt not in visited:
visited.add(nxt); result.append(str(d))
if backtrack(): return True
visited.discard(nxt); result.pop() ← Undo and try next option
return False
backtrack()
return "".join(result)
💡 Algorithm Deep-Dive: Hierholzer's Algorithm

Origin: Carl Hierholzer, 1873. Finds Eulerian circuits in O(E) time and O(E) space.

Core invariant: When DFS exits from an edge, all edges reachable via that edge have already been processed. Post-order collection therefore places each edge exactly where it belongs in the final circuit — in reverse.

When to recognize: "Visit every edge exactly once" + directed graph with equal in/out degrees + strongly connected → Hierholzer's Eulerian circuit. Classic problems: Reconstruct Itinerary, Valid Arrangement of Pairs, this problem.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the overlap structure

Notice that consecutive k-digit passwords share k-1 digits of overlap. "001" and "010" share "01". This screams "sliding window" and "graph where nodes are (k-1)-length strings". State this observation aloud.

Step 2: Build the De Bruijn graph

"Nodes = (k-1)-digit strings. Edges = k-digit combinations. There's an edge from node u to node v when v = u[1:] + some digit. Each edge represents one k-combination. I need to traverse every edge exactly once — this is an Eulerian circuit problem."

Step 3: Prove Eulerian circuit exists

"Every node has n outgoing edges (append any of n digits) and n incoming edges (from n possible predecessors). In-degree equals out-degree everywhere → Eulerian circuit guaranteed by Euler's theorem."

Step 4: Apply Hierholzer's DFS

"Run DFS marking visited edges. Collect digits in post-order and reverse at the end. This is Hierholzer's algorithm — finds Eulerian circuit in O(E) = O(n^k) time and space."

🗣️ What to Say to the Interviewer

"This is equivalent to finding a De Bruijn sequence.
Model as a directed graph: (k-1)-length strings as nodes, k-length combinations as edges.
Every node has equal in/out degree n → Eulerian circuit exists.
Find it with Hierholzer's DFS: mark edges visited, collect digits post-order, then reverse.
Prepend the start node. Result length = n^k + k - 1, which is provably optimal.
Time and space are both O(n^k)."
🔮 Likely Follow-up Questions
  • "How many valid answers exist?" → Exponentially many; any edge-traversal order gives a valid De Bruijn sequence
  • "What's the output length?" → nk + k − 1 (prove: k-1 start chars + nk edges, one char per edge)
  • "Can you do it iteratively?" → Yes, use an explicit stack instead of recursion for Hierholzer's
  • "What if not all digits need to appear?" → Generalize: any k-digit alphabet, same approach
