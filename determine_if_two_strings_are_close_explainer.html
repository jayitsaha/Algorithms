Determine if Two Strings Are Close

LeetCode #1657 · Frequency Sorting Comparison
Medium String Processing Frequency Sort O(n) Time O(1) Space
1 Understanding the Problem ▼

Two strings are "close" if you can transform one into the other using two operations any number of times:

  • Op 1 — Swap any two characters: This means you can rearrange characters in any order. So "abc" and "cab" are always reachable from each other.
  • Op 2 — Swap all occurrences of two characters: e.g., all a's become b's and all b's become a's. This swaps the frequency counts between two characters.

Example: word1 = "aabbcc", word2 = "ccbbaa"

word1: aabbcc

a× 2
b× 2
c× 2

word2: ccbbaa

a× 2
b× 2
c× 2

Same characters, same frequencies → just rearrange with Op 1 → True

💡 Key Insight

Op 1 lets you rearrange freely (any permutation). Op 2 lets you swap frequency labels between characters. So two strings are close iff they have the same character set and the same multiset of frequencies (when sorted).

2 The Algorithm: Three-Check Comparison ▼

We don't need to simulate the operations. Instead, we check three necessary and sufficient conditions:

1️⃣Check 1: Same length?
2️⃣Check 2: Same character set?
3️⃣Check 3: Same sorted frequency list?
✅All pass → return True
❌Any fail → return False
1Length check — If lengths differ, impossible (no operation adds/removes characters)
2Count frequencies — Use Counter for both strings
3Compare character sets — Both must have exactly the same letters (Op 2 can't create new characters)
4Compare sorted frequencies — Sort the frequency lists and check equality (Op 2 can shuffle frequencies between existing characters)
🔑 Why Sorted Frequencies?

Op 2 swaps the counts between two characters. This means we can reassign which character gets which count, but the set of counts itself must remain the same. Sorting both frequency lists normalizes them for comparison.

3 Why It Works: The Invariant ▼
🔐 The Invariant

Neither operation can change: (1) the set of distinct characters, or (2) the multiset of frequency values. Op 1 preserves all counts. Op 2 only swaps counts between two existing characters — same frequencies, different labels.

Op 1 (swap positions): Characters stay the same, just reordered.
→ Set of chars: unchanged. Frequency multiset: unchanged.

Op 2 (swap all a↔b): Count of 'a' and count of 'b' trade places.
→ Set of chars: unchanged (both existed). Frequency multiset: unchanged (same values, swapped labels).

Sufficiency: If char sets match and sorted freqs match, we can:
1. Use Op 2 to assign the right frequency to each character
2. Use Op 1 to arrange characters into the exact target order ✓

Edge Cases

  • Different lengths: Immediately false (no ops add/remove chars)
  • word1 has 'a' but word2 doesn't: False — Op 2 can't create new characters
  • Same chars but different frequency distributions: e.g., [1,1,3] vs [1,2,2] → False
⚠️ Common Mistake

Don't just check if character sets match! "aab" and "abb" have the same char set {a,b} but different sorted frequencies: [1,2] vs [1,2] — wait, those ARE equal. But "aaab" vs "aabb" → {a,b} same, but [1,3] vs [2,2] → NOT close.

4 The Code (Annotated) ▼
# Solution 1 — Counter + Sorted Frequencies (Interview Pick) from collections import Counter def closeStrings(word1: str, word2: str) -> bool: if len(word1) != len(word2): # Check 1: same length return False c1, c2 = Counter(word1), Counter(word2) if c1.keys() != c2.keys(): # Check 2: same char set return False return sorted(c1.values()) == sorted(c2.values()) # Check 3
# Solution 2 — Manual 26-Array (No imports) def closeStrings(word1: str, word2: str) -> bool: if len(word1) != len(word2): return False freq1, freq2 = [0]*26, [0]*26 for c in word1: freq1[ord(c)-97] += 1 for c in word2: freq2[ord(c)-97] += 1 # Same char set: both have non-zero at same positions for i in range(26): if (freq1[i] == 0) != (freq2[i] == 0): return False freq1.sort(); freq2.sort() return freq1 == freq2
⚠️ Interview Tip

Solution 1 is cleaner for interviews. Solution 2 shows you understand the O(1) space argument — only 26 letters, so the frequency arrays are constant size regardless of input.

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Understand What the Operations Preserve

Don't try to simulate! Instead ask: "What properties are invariant under these operations?" Op 1 = any rearrangement → order doesn't matter. Op 2 = swap frequency labels → which char has which count doesn't matter.

Step 2: Identify the Invariants

Two things can NEVER change: (1) the set of characters that appear, and (2) the multiset of frequency values. These are necessary AND sufficient conditions.

Step 3: Translate to Code

Counter for both strings → compare .keys() for char set → compare sorted(.values()) for frequency multiset. Three checks, one-liner each.

🗣️ What to Say to the Interviewer

"The key insight is figuring out what's invariant under these operations. Op 1 lets me rearrange freely, so character positions don't matter — only frequencies. Op 2 lets me swap frequencies between characters, so it doesn't matter which character has which count — only the multiset of counts matters. But Op 2 can't create new characters, so the character set must match. I'll check three things: same length, same character sets, and same sorted frequency lists."
