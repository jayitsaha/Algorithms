LeetCode #445

Add Two Numbers II

Given two non-empty linked lists representing non-negative integers stored most-significant digit first, add the two numbers and return the sum as a linked list in the same format.

Medium Linked List Stack or Reverse Time: O(m+n) Space: O(m+n)
1 Understanding the Problem ▾

You have two linked lists, each representing a non-negative integer with the most significant digit first. For example, the number 7243 is stored as 7 → 2 → 4 → 3. Your goal is to add these two numbers and return the result as a linked list in the same most-significant-first format.

💡 Key Insight

Addition works right-to-left (from the ones place), but the linked list is left-to-right (most significant first). The core challenge is reversing the order of access. The natural tool for "process last-inserted first" is a stack: push all digits, then pop to access them from least-to-most significant.

Input Example

l1: 7 → 2 → 4 → 3 (= 7243)
l2: 5 → 6 → 4 (= 564)
7243 + 564 = 7807

Expected Output

result: 7 → 8 → 0 → 7
(represents 7807)
Same MSD-first format ✓

Why this is harder than LeetCode #2 (Add Two Numbers): In #2, digits are stored least significant first, so you naturally traverse and add in the correct right-to-left order. Here the most significant digit is first — you must reverse the access order without losing the list structure.

Edge cases to consider: Lists of different lengths (the shorter one contributes 0s for higher positions), carry propagating across the entire length (e.g., 999 + 1 = 1000 requires a new leading node), single-element lists, and large carries at the most significant position.

2 The Algorithm: Two Stacks + Prepend ▾

We use two stacks to convert the "most significant first" list traversal into "least significant first" digit access — exactly what grade-school addition requires. We then pop from both stacks simultaneously, add digits plus carry, and prepend each result node to build the output list in the correct order automatically.

1 Push all digits to stacks: Traverse l1, pushing each node.val to s1. Traverse l2, pushing each node.val to s2. After this, the top of each stack is the least significant digit (ones place) — exactly what we need first.
2 Pop and add with carry: While either stack is non-empty OR carry is non-zero, pop the top of each stack (use 0 if empty), compute total = d1 + d2 + carry. The current digit is total % 10. The carry to propagate is total // 10.
3 Prepend to result: Create node = ListNode(total % 10), then set node.next = head and head = node. This inserts the new node at the front. Since we process from least to most significant, each more-significant digit naturally ends up closer to the head.
4 Return head: Once both stacks are empty and carry is 0, head points to the most significant digit of the result — a correctly ordered most-significant-first linked list. No reversal pass needed.
🔑 The Prepend Trick — Why It Works

We process digits from least to most significant (via stack pops). But the output needs most significant first. By always prepending, the digit computed last (the most significant one) ends up at the head — exactly the correct position. No reversal of the result needed.

⚠️ Alternative: Reverse Both Lists In-Place

You could reverse l1 and l2 first (making them LSD-first like problem #2), run the simpler in-order addition, then reverse the result. This uses O(1) extra space but mutates the input lists. Always ask the interviewer: "Am I allowed to modify the input?" If not, the stack approach is the clean answer.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After each loop iteration, head points to a valid linked list where every digit at position i and beyond has been correctly computed, including all carry propagation from positions to the right. We process right-to-left so each new prepended digit is always one position more significant than everything already in the result.

Why stacks give right-to-left access: A stack is Last-In, First-Out (LIFO). If we push 7, 2, 4, 3 (the number 7243 MSD-first), popping yields 3, 4, 2, 7 — the ones, tens, hundreds, and thousands digits in the exact order addition needs them.

Why carry is always 0 or 1: The maximum sum of two decimal digits plus a carry-in is 9 + 9 + 1 = 19. So the carry-out is at most 1 (from 19 // 10 = 1). This means we never have a carry greater than 1, which simplifies reasoning.

Why loop until carry is zero: After both stacks drain, a lingering carry of 1 means the result has one more digit than either input (e.g., 99 + 1 = 100). The condition while s1 or s2 or carry ensures we emit this final leading digit — omitting it is the #1 bug in student implementations.

💡 Why Prepend Gives Correct Order

Prepend reverses insertion order: the last node we insert becomes the head. We insert in order: ones digit (position 0), tens (1), hundreds (2), ... , most significant. So the most significant digit, inserted last, ends up at the head — the required most-significant-first output.

4 The Code (Annotated) ▾

Solution 1 — Two Stacks (Interview Pick: does not modify input)

def addTwoNumbers(l1, l2):
s1, s2 = [], [] ← Two Python lists used as stacks
while l1: ← Push every digit of l1
s1.append(l1.val); l1 = l1.next
while l2: ← Push every digit of l2
s2.append(l2.val); l2 = l2.next
carry, head = 0, None ← carry tracks overflow; head grows via prepend
while s1 or s2 or carry: ← Keep going until no digits AND no carry remain
d1 = s1.pop() if s1 else 0 ← LSD of l1, or 0 if l1 was shorter
d2 = s2.pop() if s2 else 0 ← LSD of l2, or 0 if l2 was shorter
total = d1 + d2 + carry ← Sum of this position's two digits + incoming carry
carry = total // 10 ← At most 1; 1 only when total >= 10
node = ListNode(total % 10) ← Digit for this position (0–9)
node.next = head ← Link new node to current result list
head = node ← New node is the new head (prepend!)
return head ← head = most significant digit of result
⚠️ Common Bug: Missing `or carry` in the Loop Condition

Writing while s1 or s2 without or carry drops the leading carry digit. Example: l1 = [9], l2 = [9]. After one iteration, both stacks are empty but carry = 1. Without the carry check, we return [8] instead of the correct [1,8]. Always check all three conditions.

Solution 2 — Reverse In-Place (O(1) extra space; mutates input)

def reverseList(head):
prev, curr = None, head
while curr:
nxt = curr.next
curr.next = prev
prev, curr = curr, nxt
return prev
def addTwoNumbers(l1, l2):
l1 = reverseList(l1) ← Now l1 is LSD-first (like problem #2)
l2 = reverseList(l2) ← Same for l2
carry, head = 0, None
while l1 or l2 or carry:
d1 = (l1.val if l1 else 0)
d2 = (l2.val if l2 else 0)
if l1: l1 = l1.next
if l2: l2 = l2.next
total = d1 + d2 + carry
carry = total // 10
node = ListNode(total % 10)
node.next = head; head = node ← Still prepend: result comes out MSD-first
return head
💡 Complexity Summary

Stack approach: Time O(m+n), Space O(m+n) — does not modify input
Reverse approach: Time O(m+n), Space O(1) extra — but modifies l1 and l2
Both are optimal in time. The stack solution is preferred when input immutability is required.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Name the Core Tension

Say: "Addition flows right-to-left, but the list is left-to-right. I need to reverse the access order — that's exactly what a stack does." Naming this tension shows clarity of thought and frames everything that follows.

Step 2: Ask the Mutation Question

Ask: "Am I allowed to modify the input lists?" If yes: reverse both, add forward, done. If no (or unspecified): use stacks — O(m+n) space but leaves inputs intact. This clarification often impresses interviewers.

Step 3: Walk Through the Stack Solution

Say: "Push all digits of l1 to s1, all digits of l2 to s2. Pop both simultaneously, add with carry, prepend each result node. Loop until stacks empty AND carry zero." This is one crisp paragraph with no hand-waving.

Step 4: Highlight the Prepend Insight

Explain: "I don't need to reverse the result at the end. Since I process from least to most significant and prepend each node, the most significant digit — computed last — automatically lands at the head."

🗣️ What to Say to the Interviewer

"The key issue: addition is right-to-left but the list is left-to-right. I'll use a stack.
Push all digits of l1 onto s1. Push all digits of l2 onto s2.
Then pop from both simultaneously: d1 = s1.pop(), d2 = s2.pop() (0 if empty).
total = d1 + d2 + carry. carry = total // 10. digit = total % 10.
Prepend a new ListNode(digit) to the result head each time.
Continue while either stack is non-empty OR carry is non-zero.
Return head. Time O(m+n). Space O(m+n)."
🔮 Likely Follow-up Questions
  • "Can you do O(1) extra space?" → Reverse both lists in-place, add (same logic), then the prepend trick already gives the right order.
  • "What if numbers can be negative?" → Track sign separately; add magnitudes; apply sign to result.
  • "Add numbers stored as strings?" → Same idea — push chars to stacks, or just index from the right.
  • "Can you solve with recursion?" → Yes — recursion implicitly uses the call stack, achieving the same reversal effect.

7 Summary & Key Takeaways ▾
When you need right-to-left access on a left-to-right structure — think Stack (LIFO).
Prepending result nodes reverses insertion order: the last node inserted (MSD) becomes the head.
Loop while s1 or s2 or carry — missing the carry check drops the leading digit.
Carry is always 0 or 1 because max(d1+d2+carry) = 9+9+1 = 19 < 20.
Ask about input mutation: stacks preserve inputs; reversing saves O(m+n) space but mutates.

🎯 When to Reach for the Stack-on-Linked-List Pattern
  • Need to process linked list nodes in reverse order without mutating the structure
  • An operation depends on positions from right to left, but traversal is left to right
  • "Do not modify the input" + need reversed access = stack is almost always the answer
  • Need to compare two lists from both ends → one stack per list
