LeetCode #1763

Longest Nice Substring

A string is nice if for every letter that appears in it, both its lowercase and uppercase versions are present. Return the longest nice substring of s. If there are multiple, return the one that occurs earliest. If none exists, return "".

Easy Divide and Conquer Divide on Invalid Chars Time: O(n²) Space: O(n)
1 Understanding the Problem ▾

A string is nice if every character it contains appears in both lowercase and uppercase. For example, "Bb" is nice because both 'B' and 'b' are present. "aA" is nice. But "aAb" is NOT nice — 'b' appears without its uppercase partner 'B'.

💡 Key Insight

If a character is in the string but its case-partner is not, that character is an invalid blocking character. No nice substring can span across an invalid character's position — so it naturally divides the problem. Split there, solve each side recursively, return the longer nice result.

✓ Nice: "YazaAay"

Y → y present ✓
a → A present ✓
z → Z present? ✗ (NOT NICE itself)
A → a present ✓
y → Y present ✓
→ Longest nice piece: "aA" (or "Aa")

✗ Not Nice: "cCccc"

c → C present ✓
C → c present ✓
Wait — "cCccc" IS nice!
But "cccc" is NOT:
c present, C absent → Not Nice

Full walkthrough on "YazaAay": Build the character set: {Y, a, z, A, y}. Scan left to right: Y → y in set? Yes. a → A in set? Yes. z → Z in set? No! So 'z' at index 2 is our split point. Left = "Ya", right = "aAay". Recurse on both and return the longer result.

Edge cases: Single character → always return "" (can't form a pair by itself). Empty string → return "". All chars invalid → every position is a split point, all fragments are length 0 or 1 → return "". Entirely nice string → return the whole string.

2 The Algorithm: Divide on Invalid Characters ▾

This is a textbook Divide and Conquer approach. An invalid character is a guaranteed dividing wall — no nice substring crosses it. We recursively solve both sides and take the winner.

1 Base case: len(s) <= 1 → return "". A single character (or empty string) can never form a case-pair by itself.
2 Build char_set: set(s) collects all characters in the current substring. O(n) time. This enables O(1) membership checks: "is the case-partner present?"
3 Find the first invalid character: Scan left to right. For each character c, check if c.swapcase() is in char_set. The first c for which it is NOT in the set is our split point at index i.
4 Divide: Recurse on s[:i] (everything before the invalid char) and s[i+1:] (everything after, skipping the invalid char itself — it cannot be in any nice substring).
5 Conquer: Compare the two results. Return the longer one. If both are equal length, return the left (which appears earlier in the original string) using >=.
6 Whole string is nice: If the for-loop completes without finding any invalid character, the entire current substring s is nice. Return it directly.

Divide and Conquer Tree for s = "YazaAay":

"YazaAay"
↓ 'z' at idx 2 is invalid (no 'Z' in {"Y","a","z","A","y"}) — split
"Ya" | "aAay"
↓ 'Y' invalid in "Ya"                   ↓ 'y' invalid in "aAay"
"" "Y"→"" | "aA" ← nice! ""
Final: max("","","aA","") = "aA"
3 Why It Works: The Invariant ▾
🔐 The Invariant

If character c appears in substring s but c.swapcase() does not appear in s, then no nice substring of s can include index i (where s[i] == c). The optimal nice substring must lie entirely in s[:i] or in s[i+1:].

Why? Suppose (for contradiction) that some nice substring T spans across index i. Then T contains c = s[i]. Since T is nice, it must also contain c.swapcase(). But T is a substring of s, so its characters are a subset of s's characters. Since c.swapcase() is not in s, it cannot be in T either — contradiction. So no nice substring spans i.

Why splitting at the FIRST invalid character is sufficient: We could split at any invalid character. The first one is chosen for determinism. Every subsequent split happens naturally during recursion — each recursive call re-scans its local substring and finds the first invalid character within that range.

⚠️ Common Mistake: Forgetting the >= in the comparison

When comparing len(left) >= len(right), the >= (not >) ensures that when both halves have the same length, we return left. The left half corresponds to the earlier portion of the string — so equal-length ties go to the earlier-starting result, satisfying the problem's tie-breaking rule.

Complexity analysis:

  • Worst case O(n²): If every character is invalid, we split on every index. The recursion tree becomes a chain of depth n, with O(n) work at each level → O(n²) total.
  • Average/best case O(n log n): If the string has balanced splits (like a BST), each level does O(n) work over log n levels.
  • Space O(n): Recursion depth up to O(n) call frames, each holding an O(n) char_set. But substrings are sliced (Python creates new strings), so total space is O(n) across any single path from root to leaf.
4 The Code (Annotated) ▾

Solution 1 — Divide and Conquer (Interview Pick)

def longestNiceSubstring(s: str) -> str:
if len(s) <= 1: ← base: single char or empty can never be nice
return ""
char_set = set(s) ← O(n): all unique chars in this substring
for i, c in enumerate(s): ← scan left to right for first invalid char
if c.swapcase() not in char_set: ← swapcase: 'a'→'A', 'Z'→'z'. Missing partner = invalid
left = longestNiceSubstring(s[:i]) ← recurse on everything BEFORE the invalid char
right = longestNiceSubstring(s[i+1:]) ← recurse on everything AFTER (skip invalid char)
if len(left) >= len(right): ← >= means: prefer left on ties (earlier in string)
return left
return right
return s ← loop finished without invalid → whole string is nice!
⚠️ Why skip s[i] entirely when dividing?

The character at index i is invalid by definition — its case-partner is absent from the current substring. Including it in either half would only create a non-nice character that would immediately cause further splits (or disqualify that half). Skipping it is correct: we take s[:i] and s[i+1:], completely excluding the invalid char from both recursive calls.

Solution 2 — Brute Force O(n³) (for comparison)

def longestNiceSubstring_brute(s: str) -> str:
best = ""
for i in range(len(s)): ← O(n) starting positions
for j in range(i+2, len(s)+1): ← O(n) ending positions (min length 2)
sub = s[i:j]
chars = set(sub) ← O(n) to build
if all(c.swapcase() in chars for c in chars): ← O(n) niceness check
if len(sub) > len(best):
best = sub ← strictly greater: keeps first occurrence on ties
return best
💡 Complexity Summary

Divide & Conquer: O(n²) worst case (pathological splits), O(n log n) average. Space O(n).
Brute Force: O(n³) time — O(n²) pairs × O(n) per check. Space O(n).

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Clarify what "nice" means

Ask the interviewer: "To confirm — a string is nice if every character that appears has its opposite case also present? So 'Aa' is nice but 'a' alone is not?" This pins down the definition and shows you read carefully.

Step 2: Propose Brute Force

"First approach: try every substring O(n²), check if nice O(n), track longest. O(n³) total. Correct but slow. Can we do better?"

Step 3: The "Invalid Character as Wall" Observation

"If a character's case-partner is absent, no nice substring can contain that character's position. It's a hard boundary — like a wall. This suggests: divide at the first such wall, recurse on both sides."

Step 4: Recognize the D&C Pattern

"This is Divide and Conquer: split on an invalid character, solve each half independently, combine by taking the max. Similar in spirit to QuickSort's partition step."

Step 5: Handle the tie-breaking rule

"When both halves return equal-length results, the left half appears earlier in the string, so use >= in the comparison to prefer the left on ties."

🗣️ What to Say to the Interviewer

"A nice string requires every char to appear in both cases.
Key insight: a char missing its case-partner is an 'invalid' wall — no nice
substring can span it. So I divide at the first invalid character, recurse
on both sides, and return the longer result. If no invalid char exists,
the whole string is nice — return it directly.
Complexity: O(n^2) worst case (highly unbalanced splits), O(n log n) average.
Space: O(n) for the recursion stack."
🔮 Likely Follow-up Questions
  • "Can you do it iteratively?" → Use an explicit stack to simulate the recursion
  • "What's the worst case input?" → Alternating valid/invalid chars: "aAbBcC" — every other char is invalid
  • "Can you solve it in O(n) time?" → Actually O(n log n) with D&C if splits are balanced; O(n²) is the honest worst case for this approach
  • "Find ALL nice substrings?" → Modify to collect results instead of tracking max
