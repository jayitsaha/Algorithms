LeetCode #1547

Minimum Cost to Cut a Stick

Given a wooden stick of length n and cut positions, find the minimum total cost to make all cuts, where cutting a stick of length L costs exactly L.

Hard Dynamic Programming Interval DP Time: O(m³) Space: O(m²)
1 Understanding the Problem ▾

You have a wooden stick of length n units. You receive an array cuts specifying positions where the stick must be cut. The key rule: the cost of any single cut equals the current length of the stick piece you are cutting. Your goal is to order the cuts to minimize the total cost.

💡 Key Insight

The order you make cuts matters enormously. Cutting on a shorter piece is cheaper. So you want to cut larger pieces first only when absolutely necessary, or equivalently — find the optimal split ordering by trying all possible "first cuts" for each sub-interval.

Concrete walkthrough: n=7, cuts=[1,3,4,5]

✗ Naive order: left-to-right

Cut at 1: cost = 7 (full stick)
Cut at 3: cost = 6 (piece [1,7])
Cut at 4: cost = 4 (piece [3,7])
Cut at 5: cost = 3 (piece [4,7])
Total = 20

✓ Optimal: cut at 3 first

Cut at 3: cost = 7 (full stick)
Cut at 5: cost = 4 (piece [3,7])
Cut at 1: cost = 3 (piece [0,3])
Cut at 4: cost = 2 (piece [3,5])
Total = 16

🔑 The Reformulation Trick

Add 0 and n as sentinel values to cuts, then sort. Now define: dp[i][j] = minimum cost to make all cuts strictly between positions cuts[i] and cuts[j]. When you make the first cut at position cuts[k] (where i < k < j), the cost is cuts[j] - cuts[i] (the full length of this sub-stick), and you're left with two independent sub-problems: dp[i][k] and dp[k][j].

Edge cases: Empty cuts array → return 0. One cut → cost equals n. Cuts at extreme positions (near 0 or n).

2 The Algorithm: Interval DP ▾

This is the classic Interval DP pattern — same structure as Matrix Chain Multiplication and Burst Balloons. The key idea: try every possible "first cut" in each interval and recurse on the two resulting sub-intervals.

📐 Recurrence Relation
cuts' = sorted([0] + cuts + [n]) — add sentinels
m = len(cuts')

Base case: dp[i][j] = 0 when j == i+1 (no cuts between adjacent points)
Transition: dp[i][j] = min over k in (i+1 .. j-1) of:
            dp[i][k] + dp[k][j] + (cuts'[j] - cuts'[i])
1 Add sentinels: Insert 0 and n into cuts, then sort. This makes every sub-problem a uniform interval [cuts[i], cuts[j]] without special-casing stick boundaries.
2 Fill by interval length: Iterate length from 2 up to m−1. For length=2: adjacent points, dp=0 (no cuts). For each larger length, all sub-intervals are already solved.
3 Try every first cut: For interval [i,j], try each k as the "first cut made" in this interval. Cost = (length of this piece) + dp[i][k] + dp[k][j].
4 Take minimum: dp[i][j] = minimum over all valid k. Final answer is dp[0][m-1].
⚠️ Why Fill by Interval Length?

When we compute dp[i][j], we need dp[i][k] and dp[k][j] already available. Since both intervals are strictly shorter than [i,j], filling by increasing length guarantees all dependencies are ready when needed.

3 Why It Works: Optimal Substructure ▾
🔐 The Invariant

Once the first cut at position cuts[k] splits an interval [cuts[i], cuts[j]], the two resulting pieces [cuts[i], cuts[k]] and [cuts[k], cuts[j]] are fully independent. No future cut in one piece can ever affect the cost of cutting the other piece.

Optimal Substructure: The min cost for dp[i][j] equals (cost of first cut) + (optimal cost for left sub-problem) + (optimal cost for right sub-problem). The sub-problems don't interact, so we can solve them independently and combine.

Overlapping Subproblems: The interval [2,5] might be needed by both dp[0,5] and dp[1,5]. Without memoization, we'd recompute it many times. The DP table stores each answer once — O(m²) states, each solved in O(m) → O(m³) total.

💡 Analogy: Cutting a Pizza

Imagine cutting a pizza into slices. A cut across the whole pizza is expensive (big diameter), but a cut across a small slice is cheap. DP asks: "what's the cheapest cut to make first?" and then solves each resulting smaller pizza independently. The order of cuts you choose completely determines the total cost.

Why sentinels work: Every sub-problem is now defined by two indices into the extended cuts' array. The cost of cutting in interval [i,j] is always cuts'[j] − cuts'[i] — the exact current piece length. Without sentinels, you'd need to track actual stick endpoints separately, complicating the code significantly.

4 The Code (Annotated) ▾

Solution 1 — Bottom-Up Interval DP (Interview Pick)

def minCost(n: int, cuts: list[int]) -> int:
cuts = sorted([0] + cuts + [n]) ← add sentinels 0 and n, sort; e.g. [0,1,3,4,5,7]
m = len(cuts) ← total boundary points (len(original cuts) + 2)
dp = [[0]*m for _ in range(m)] ← dp[i][j]: min cost for cuts between cuts[i] and cuts[j]
for length in range(2, m): ← length 2 = adjacent (base case, 0); build up to full stick
for i in range(m - length): ← left boundary of current interval
j = i + length ← right boundary
dp[i][j] = float('inf') ← initialize before taking minimum
for k in range(i+1, j): ← try each interior point as the "first cut"
cost = cuts[j]-cuts[i]← cutting this interval costs its full length
dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j] + cost)← left + right + this cut
return dp[0][m-1] ← min cost for the full stick [0, n]
⚠️ Common Mistake: Wrong Iteration Order

Iterating for i in range(m): for j in range(m): without controlling length will attempt to compute dp[i][j] before dp[i][k] and dp[k][j] are ready. Always use the length-first outer loop to ensure all smaller sub-intervals are pre-computed.

Solution 2 — Top-Down Memoization

Time: O(m³) — O(m²) states, O(m) per state. Space: O(m²) for the memo table (or DP array).

5 Interview Strategy: How to Arrive at This ▾
Step 1: Spot the Signal

The cost depends on the current state (stick length), not just fixed positions. The order of operations changes the total cost. Signal: "minimum cost to perform operations on a sequence where each operation's cost depends on the current boundaries." This immediately suggests Interval DP.

Step 2: Identify Subproblems

After any cut, you have two independent sub-sticks. Their costs don't interfere. This is optimal substructure — the hallmark of DP. Define dp[i][j] as the min cost for all cuts in the interval between cuts[i] and cuts[j].

Step 3: Apply the Sentinel Trick

Add 0 and n to the cuts array, sort. Now every sub-problem has the same shape: "make cuts between two known endpoints." The cost formula cuts[j] - cuts[i] works uniformly without special cases.

Step 4: Write the Transition

For each interval [i,j], try every interior index k as the "first cut." The total cost is: dp[i][k] + dp[k][j] + (cuts[j] − cuts[i]). Take the minimum. Use length-first iteration to ensure dependencies.

What to Say to the Interviewer

"The cost of each cut depends on the current piece length, so order matters. This is a signal for Interval DP — I want to find the optimal sequence of operations on a sequence.

My subproblem: dp[i][j] = min cost to cut everything strictly between positions cuts[i] and cuts[j]. When I make cut k as the first cut in this interval, the cost is cuts[j]-cuts[i] (full piece length), and I get two independent sub-problems.

I'll add 0 and n as sentinels so every interval has well-defined boundaries. Then I iterate by interval length to ensure smaller problems are solved first.

Recurrence: dp[i][j] = min over k of (dp[i][k] + dp[k][j] + cuts[j]-cuts[i]).
Time: O(m^3), Space: O(m^2) where m = len(cuts)+2."
💡 Interval DP Recognition Signals
  • "Minimum cost to split/merge a sequence" where cost depends on boundaries
  • "Optimal order of operations" on a contiguous range
  • After any split, the two parts are fully independent
  • Problem can be modeled as: pick the "last merge" or "first split" in each interval
  • Related: Burst Balloons, Matrix Chain Multiplication, Strange Printer

7 Summary & Key Takeaways ▾
Add 0 and n as sentinels and sort — this makes every sub-interval uniform.
dp[i][j] = min cost for all cuts between cuts[i] and cuts[j]; the cost of any cut in this interval is always cuts[j]−cuts[i].
Iterate by interval length (small to large) to guarantee dependencies are ready.
This is Matrix Chain Multiplication with different variable names — recognize the pattern.
Time: O(m³) where m = len(cuts)+2. Space: O(m²).
🎯 When to Use Interval DP
  • Operations on a contiguous range where cost depends on current boundaries
  • After splitting at any point, the two parts are fully independent
  • You need to find the optimal ordering of splits/merges
  • Problem gives you freedom to choose the order of operations
