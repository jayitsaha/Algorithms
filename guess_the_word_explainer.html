843. Guess the Word

Minimax Elimination — Narrow candidates by maximizing information per guess
Hard Minimax / Elimination Interactive O(n²) per guess O(n) space
1 Understanding the Problem ▼

You have a list of 6-letter words. One is the secret. You can call Master.guess(word) which returns how many letters match exactly (same position and same character). You have at most 10 guesses to find the secret.

Reframe: This is an information game. Each guess gives us a number 0–6. We must choose guesses that maximally narrow down which word could be the secret, so we find it within the guess limit.

Example by hand: If we guess "abcdef" and get back 3, then the secret must share exactly 3 characters (in the same positions) with "abcdef". Any word that doesn't share exactly 3 can be eliminated.

💡 Key Insight

After each guess returning match count m, we can eliminate every candidate that doesn't also produce exactly m matches against our guess. The best guess is the one that produces the smallest worst-case group of survivors — this is the minimax strategy.

⚠️ Why Random Guessing Can Fail

With ~100 words and 10 guesses, random guessing risks picking words that have 0 matches with many others. A 0-match result barely narrows the field — most 6-letter words differ in all positions. Minimax avoids this by choosing guesses that spread candidates across multiple match buckets.

2 The Algorithm: Minimax Elimination ▼

We maintain a candidate list — initially all words. Each round, we pick the best word to guess, get the match count, then filter candidates.

1 Initialize: All words are candidates.
2 Minimax selection: For every candidate word W, compute how many candidates would fall into each match bucket (0–6) if W were the guess. The worst case for W is the largest bucket. Pick the W with the smallest worst case.
3 Guess: Call Master.guess(W) → returns match count m.
4 Filter: Keep only candidates whose match count against W equals m. The secret must be in this group.
5 Repeat from step 2 until match count = 6 (found!) or guesses exhausted.

3 Why It Works: The Invariant ▼
🔐 The Invariant

After every guess, the secret word is always in the candidate list. If we guess word G and get back m, then match(G, secret) = m. So we keep only words W where match(G, W) = m. The secret satisfies this, so it survives every filter.

Why minimax guarantees convergence: The minimax strategy picks the guess that minimizes the worst-case number of survivors. In the worst case, the adversary places the secret in the largest bucket. By minimizing this maximum bucket size, we guarantee the candidate list shrinks as fast as possible.

💡 Information Theory View

Each guess returns one of 7 outcomes (0–6 matches). The best guess is one that spreads candidates as evenly as possible across these outcomes, maximizing the information gained. Minimax is the pessimistic version: it optimizes for the worst case rather than the average.

Edge cases:

  • All words identical at some positions: Those positions give no information — algorithm still works because filtering uses all 6 positions.
  • 1 candidate left: Guess it directly — guaranteed match of 6.
  • Many 0-match pairs: Minimax avoids words with large 0-match groups, preferring words that distinguish candidates.
4 The Code (Annotated) ▼
def findSecretWord(self, words, master):
def match(a, b): # Count exact position matches
return sum(x == y for x, y in zip(a, b))
candidates = list(words) # Start with all words as candidates
for _ in range(10): # At most 10 guesses allowed
best, best_max = candidates[0], len(candidates) # Track best guess word
for word in candidates: # Try each candidate as potential guess
groups = [0] * 7 # Buckets for match counts 0-6
for other in candidates: # Count matches with every candidate
groups[match(word, other)] += 1 # Tally into the right bucket
worst = max(groups) # Worst case: largest bucket
if worst < best_max: # Better than current best?
best_max = worst # Update best worst-case
best = word # Update best guess word
result = master.guess(best) # Make the guess
if result == 6: return # Found the secret!
candidates = [w for w in candidates # Keep only words matching
if match(w, best) == result] # same count with our guess
⚠️ Common Mistake: Guessing from all words, not candidates

You must guess from the remaining candidates, not from all original words. Words eliminated in previous rounds cannot be the secret, so guessing them wastes information.

🔑 Why match() is symmetric

match(a, b) == match(b, a) — exact position matches are symmetric. When filtering, we check match(candidate, guess) == m. This is the same as asking "would guessing this candidate against the real guess produce the same count?"

5 🎤 Interview Strategy: How to Arrive at This ▼
Step 1: Recognize the Problem Type

Interactive guessing with feedback → elimination game. Similar to Mastermind / 20-Questions. Key signal: limited guesses + match feedback → need to maximize information per guess.

Step 2: Start Simple — Random + Filter

First propose: pick any word, guess it, filter candidates. This works but might not converge fast enough. Interviewer will ask: "Can you do better?"

Step 3: Optimize Guess Selection

Realize: some guesses split candidates more evenly than others. A word with many 0-match pairs gives little information. The optimal strategy is minimax: pick the word whose worst-case bucket is smallest.

Step 4: Mention Alternative Heuristics

If minimax is too complex to code live, mention: "We could also pick the word with the fewest 0-match pairs — this heuristic works well in practice and is simpler to implement."

🗣️ What to Say to the Interviewer

"This is an interactive elimination problem. After each guess, I get a match count which lets me filter the candidate list. The key question is: which word should I guess to maximize information? I'll use a minimax approach: for each candidate, I compute the distribution of match counts against all other candidates. The 'worst case' is the largest bucket. I pick the word whose worst case is smallest — this guarantees the fastest shrinkage of the candidate set. Each round costs O(n²) to evaluate all pairs, but n shrinks rapidly. Total work is dominated by the first round: O(n²). With 10 guesses and smart selection, this reliably finds the secret."

7 Summary & Key Takeaways ▼
Minimax = minimize the worst case. Pick the guess whose largest match-count bucket is smallest.
Filtering is the core mechanic. After each guess, keep only candidates consistent with the observed match count.
match() is symmetric — match(a,b) == match(b,a), so filtering works regardless of direction.
O(n²) per round converges fast — candidates shrink rapidly, so later rounds are cheap.
Heuristic alternative: Minimizing zero-match pairs is simpler and usually sufficient.
🎯 When to Use This Pattern

Interactive query problems with limited attempts. Anytime you have: (1) a hidden target, (2) a comparison oracle, (3) a guess budget. Examples include Mastermind, 20 Questions, and binary search with errors.
