LeetCode #785

Is Graph Bipartite?

Given an undirected graph, determine if the graph is bipartite — i.e., can its nodes be split into two groups such that every edge connects nodes from different groups?

Medium Graph DFS 2-Coloring Time: O(V + E) Space: O(V)
1 Understanding the Problem ▾

You're given an undirected graph as an adjacency list: graph[i] lists every node directly connected to node i. The graph has n nodes (labeled 0 to n-1). The graph may not be connected (it can have multiple disconnected components).

A graph is bipartite if its nodes can be split into exactly two sets (call them Group A and Group B) such that every single edge connects one node from Group A to one node from Group B. No edge may connect two nodes within the same group.

💡 Key Insight

Bipartiteness is equivalent to 2-colorability: can we paint every node red or blue so no two connected nodes share the same color? This is also equivalent to the graph having no odd-length cycles. A triangle (3 nodes all connected to each other) is the simplest example that kills bipartiteness — you'd need 3 colors to properly color it.

✓ Bipartite: graph = [[1,3],[0,2],[1,3],[0,2]]

Edges: 0-1, 0-3, 1-2, 2-3 (a 4-cycle)
Group A (blue) = {0, 2}
Group B (green) = {1, 3}
Every edge crosses groups → True

✗ Not bipartite: graph = [[1,2,3],[0,2],[0,1,3],[0,2]]

0-1, 0-2, 0-3, 1-2 (includes triangle 0-1-2)
Triangle = odd cycle (length 3)
Can't 2-color without conflict → False

Edge cases: A graph with no edges is always bipartite (each node can be in Group A). Isolated nodes (no edges) don't affect bipartiteness. A disconnected graph is bipartite only if every component is bipartite — so we must check all of them. The problem guarantees no self-loops or parallel edges.

2 The Algorithm: DFS 2-Coloring ▾

We attempt to color the graph with two colors (0 and 1) using DFS. We assign a color to a node, then force all its neighbors to take the opposite color. If a neighbor is already colored with the same color as the current node, we've detected an odd cycle — the graph is not bipartite.

1 Initialize: Create color = [-1] * n where -1 = unvisited. Colors 0 and 1 represent Group A and Group B respectively.
2 Loop over all nodes: For each node not yet visited (color[i] == -1), start a DFS from it with color 0. This handles disconnected components — we never skip an unvisited component.
3 DFS: Color and propagate: Assign current node color c. For each neighbor: if it has the same color c, return False (conflict!). If unvisited, assign color 1-c and recurse.
4 All done: If all components are colored with no conflict, return True.

🔑 BFS Alternative

BFS works identically — process nodes level by level, assigning alternating colors to each level. BFS is often easier to reason about because each BFS "level" naturally corresponds to one color group. Both DFS and BFS run in O(V + E). In interviews, BFS is slightly more common for 2-coloring because the level structure makes the color-flip intuitive.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After DFS colors a connected component: every visited node u has been assigned a color in {0, 1}, and for every edge (u, v) within that component, color[u] ≠ color[v]. If this invariant ever cannot be maintained (a visited neighbor forces a contradiction), the component is not bipartite.

Why does a conflict prove non-bipartiteness? If during DFS we find an edge (u, v) where color[u] == color[v], then following the DFS path from the component's root to u and from the root to v, we have two paths of different parity (one even-length, one odd-length). The edge u-v closes a cycle of odd length. No 2-coloring can resolve an odd cycle — it's mathematically impossible.

Why iterate over all nodes (not just start from node 0)? The graph may be disconnected. A graph like [[],[],[0,3],[2]] has two components: {0} and {1} and {2,3} — node 0 and 1 are isolated, the DFS from them is trivial. But component {2,3} must also be checked. Skipping unvisited nodes means silently missing entire components.

Odd cycle theorem: A graph is 2-colorable ↔ it contains no odd cycle. This is proven by the König-Egerváry theorem (1916) and is a foundational result in graph theory. Our DFS 2-coloring is essentially a constructive proof: it either builds a valid 2-coloring (True) or identifies the contradicting edge in an odd cycle (False).

⚠️ The Classic Mistake: Skipping Unvisited Components

Only starting DFS from node 0 is wrong. A graph like [[1],[0],[3],[2]] has two completely disconnected edges: {0-1} and {2-3}. Both are bipartite individually. But if you only start DFS from node 0, you'd visit {0,1} and call it True — missing {2,3} entirely. Always loop through every unvisited node and start a new DFS from it.

4 The Code (Annotated) ▾

Solution 1 — DFS 2-Coloring (Interview Pick)

def isBipartite(graph): ← graph[i] = neighbors of node i
n = len(graph) ← number of nodes
color = [-1] * n ← -1 = unvisited, 0 = Group A, 1 = Group B
def dfs(node, c): ← Try coloring node with color c
color[node] = c ← Assign this color to the current node
for nei in graph[node]: ← Check every neighbor
if color[nei] == c: ← Same color as us → odd cycle → fail
return False
if color[nei] == -1: ← Unvisited? Assign opposite color and recurse
if not dfs(nei, 1 - c): ← 1-0=1, 1-1=0: flips the color
return False ← Propagate failure up the call stack
return True ← No conflict found in this subtree
for i in range(n): ← Handle ALL components (not just node 0)
if color[i] == -1: ← Only start DFS from unvisited nodes
if not dfs(i, 0): ← Color this component starting with color 0
return False ← Any component fails → whole graph fails
return True ← All components are bipartite
⚠️ Why check color[nei] == c BEFORE color[nei] == -1?

Order matters here. If we check color[nei] == -1 first and recurse, a later check of color[nei] == c would never fire for that neighbor. We want to catch the conflict first: if a neighbor already has our same color, we know immediately there's an odd cycle. Only if the neighbor is unvisited (-1) do we continue the DFS. (If a neighbor has the opposite color already, we can safely skip it — no conflict.)

Solution 2 — BFS 2-Coloring (iterative, avoids recursion depth issues)

from collections import deque
def isBipartite(graph):
n = len(graph)
color = [-1] * n
for start in range(n): ← Handle all components
if color[start] != -1:
continue ← Skip already-visited nodes
color[start] = 0 ← Seed this component with color 0
q = deque([start]) ← BFS queue
while q:
u = q.popleft() ← Process next node
for v in graph[u]: ← Examine neighbors
if color[v] == -1: ← Unvisited: color opposite and enqueue
color[v] = 1 - color[u]
q.append(v)
elif color[v] == color[u]: ← Same color → conflict → not bipartite
return False
return True
💡 Complexity Summary

DFS: Time O(V + E), Space O(V) for the color array + recursion stack (up to O(V) deep).
BFS: Time O(V + E), Space O(V) for the color array + the deque (holds at most O(V) nodes at once).
Both visit every node and every edge at most twice (once in each direction). The color array is the only extra space needed beyond the recursion/queue.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Pattern

Keywords: "can we split nodes into two groups," "no two adjacent nodes same group." This immediately signals 2-coloring / bipartite check. The constraint that the graph is undirected is crucial — directed graphs have a different bipartite concept.

Step 2: Reformulate as Graph Coloring

Restate: "Can I color each node 0 or 1 such that every edge connects nodes with different colors?" This is a constraint propagation problem — once you pick a color for one node, all its neighbors are forced, and their neighbors are forced, and so on.

Step 3: Choose DFS or BFS (either is fine)

Both work. DFS is slightly more compact with recursion. BFS is iterative (avoids stack overflow on large inputs). State explicitly: "I'll use DFS for brevity, but BFS would work identically." This shows you know both.

Step 4: Handle Disconnected Graphs

"The graph may be disconnected. I need to start DFS/BFS from every unvisited node, not just node 0." Mentioning this proactively signals that you understand the edge case — many candidates miss this.

🗣️ What to Say to the Interviewer

"This is a bipartite check — can we 2-color the graph so no two neighbors share a color?
I'll use DFS: start from each unvisited node, assign it color 0,
then DFS: assign neighbors color 1-c. If a neighbor already has the same color, return False.
I need to iterate over all nodes to handle disconnected components.
Time is O(V+E) — we visit each node and edge at most once. Space is O(V) for the color array."
🔮 Likely Follow-up Questions
  • "Can you do it without recursion?" → Yes, use BFS (iterative queue-based approach).
  • "What if the graph is directed?" → Different problem. Bipartite typically applies to undirected graphs. For directed, you'd check SCC or transitive closure properties.
  • "Possible Bipartition (LC 886)?" → Almost identical: build a graph from the "dislike" pairs, then check if it's bipartite.
  • "What's the connection to odd cycles?" → A graph is bipartite iff it contains no odd-length cycle. 2-coloring fails exactly when an odd cycle is encountered.

7 Summary & Key Takeaways ▾
A graph is bipartite iff it can be 2-colored with no adjacent nodes sharing a color
DFS 2-coloring: assign color 0 to start node, force neighbors to color 1, detect same-color conflicts
Must iterate over ALL nodes, not just node 0 — disconnected components must all be checked
Both DFS (recursive) and BFS (iterative) work — O(V + E) time, O(V) space
A conflict (same-color edge) proves an odd-length cycle exists → not bipartite
Trees are always bipartite (no cycles at all). Even-length cycles are fine. Odd-length cycles break bipartiteness.
🎯 When to Use This Pattern
  • Problem asks to divide nodes into two groups based on edge constraints
  • "Can we color this graph with 2 colors?" (general k-coloring for k ≥ 3 is NP-hard, but k=2 is polynomial)
  • Detecting if a relationship graph (dislikes, conflicts) has a consistent 2-partition
  • Checking if a scheduling or assignment problem can be resolved without contradictions
