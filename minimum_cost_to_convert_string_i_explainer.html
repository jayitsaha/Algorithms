LeetCode #2976

Minimum Cost to Convert String I

Given source and target strings and character conversion costs, find the minimum cost to transform source into target one character at a time.

Medium Graph Floyd-Warshall Time: O(26³ + n) Space: O(26²)
1 Understanding the Problem ▾

You have two strings source and target of the same length. You also have arrays original, changed, and cost where each triple (original[i], changed[i], cost[i]) means: converting one character original[i] to changed[i] costs cost[i].

You must transform source into target character by character. For each position i, if source[i] == target[i], no cost is needed. Otherwise, you need the cheapest sequence of single-character conversions to get from source[i] to target[i]. If any position is impossible to convert, return -1.

💡 Key Insight

Each character conversion is an edge in a weighted directed graph on 26 nodes (a–z). The question "what's the cheapest way to convert character X to character Y?" is exactly "what's the shortest path from node X to node Y?" — and Floyd-Warshall computes ALL-PAIRS shortest paths in one shot.

✓ Example: source="abcd", target="acbe"

i=0: a→a, same, cost 0
i=1: b→c, direct edge cost 2
i=2: c→b, direct edge cost 5
i=3: d→e, direct edge cost 3
Total: 0+2+5+3 = 10

Multi-hop path example

a→e: no direct edge
a→b cost 1, b→e cost 3
Cheapest path: a→b→e
Total cost: 1+3 = 4

Edge Cases: position where source[i] != target[i] but no path exists at all (return -1); multiple edges between same pair (keep minimum); cycles are fine (Floyd-Warshall handles them via the update check).

2 The Algorithm: Floyd-Warshall All-Pairs Shortest Path ▾

The character universe is only 26 letters. We can model conversions as a graph and precompute the cheapest cost to convert any character to any other character — then simply sum up costs per position in the strings.

1 Build cost matrix: Initialize a 26×26 matrix dist where dist[i][j] = infinity and dist[i][i] = 0. For each conversion triple, set dist[original][changed] = min(dist[original][changed], cost) — handle duplicate edges by keeping the cheaper one.
2 Floyd-Warshall relaxation: For every intermediate node k (0–25), for every pair (i, j), check if going through k is cheaper: dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]). After all 26 iterations, dist[i][j] holds the cheapest conversion path from character i to character j.
3 Query per position: For each index in the strings, if source[i] == target[i], add 0. Otherwise, look up dist[source[i]][target[i]]. If it's still infinity, that position is unreachable — return -1. Otherwise, add it to the total.
🔑 Why Floyd-Warshall over Dijkstra?

We need shortest paths between all pairs of the 26 characters, not just one source. Running Dijkstra 26 times works too (O(26 × (E + V log V))), but Floyd-Warshall is simpler code: three nested loops over a tiny 26×26 matrix. With only 26 nodes, O(26³) is effectively constant — just 17,576 iterations. Floyd-Warshall wins on elegance here.

3 Why It Works: The Invariant ▾
🔐 The Floyd-Warshall Invariant

After the k-th outer iteration, dist[i][j] holds the minimum cost of any path from node i to node j that uses only nodes 0 through k as intermediate stops. When k reaches 25 (all characters), dist[i][j] is the true global shortest path.

Why this is correct: Consider any optimal path from i to j. The optimal path either passes through node k or it doesn't. If it doesn't, it was already captured in a previous k iteration. If it does, the sub-paths i→k and k→j have also been optimally computed (by induction). So dist[i][k] + dist[k][j] correctly represents the best path through k.

Why each position is independent: Each position in the string is a separate, independent conversion. The conversions at position i do not affect position j. So the total minimum cost is just the sum of per-position costs.

⚠️ Common Trap: Integer Overflow

When computing dist[i][k] + dist[k][j], if either is infinity (represented as a large int), adding them overflows. Use float('inf') in Python (or check for inf before adding) — Python's floats handle inf arithmetic safely: inf + x == inf.

Edge cases: dist[i][i] = 0 for all i (converting a character to itself costs nothing, which is also how we bootstrap the "same character" positions). If the original cost arrays have duplicate (same pair, different cost), we keep the minimum — the matrix initialization does this naturally.

4 The Code (Annotated) ▾

Solution 1 — Floyd-Warshall (Optimal, Interview Pick)

def minimumCost(source, target, original, changed, cost):
INF = float('inf') ← Use float('inf') to avoid overflow in summation
dist = [[INF]*26 for _ in range(26)] ← 26×26 matrix: dist[i][j] = cheapest cost i→j
for i in range(26): dist[i][i] = 0 ← Base case: converting a char to itself costs 0
for o, c, w in zip(original, changed, cost): ← Iterate all given conversion edges
u = ord(o) - ord('a') ← Map character to node index 0–25
v = ord(c) - ord('a') ← Target character to node index 0–25
dist[u][v] = min(dist[u][v], w) ← Keep minimum if duplicate edges exist
for k in range(26): ← Floyd-Warshall: try each node as intermediate
for i in range(26): ← Source of candidate path
for j in range(26): ← Destination of candidate path
if dist[i][k] + dist[k][j] < dist[i][j]: ← Going through k is cheaper?
dist[i][j] = dist[i][k] + dist[k][j] ← Relax: update to cheaper path
total = 0 ← Accumulate total conversion cost
for s, t in zip(source, target): ← Process each position in the strings
if s == t: continue ← Same character: free, skip it
c = dist[ord(s)-ord('a')][ord(t)-ord('a')] ← Look up precomputed shortest path
if c == INF: return -1 ← No path exists → impossible
total += c ← Add this position's cost
return total ← Sum of all per-position minimum costs
⚠️ Why not just use the given cost array directly?

The cost array only has direct, single-hop conversions. Converting 'a'→'c' might not be listed, but if 'a'→'b' (cost 1) and 'b'→'c' (cost 2) are listed, the cheapest path is 3. Floyd-Warshall discovers all such multi-hop paths automatically.

Solution 2 — Dijkstra per source character (also valid)

import heapq
def minimumCost(source, target, original, changed, cost):
graph = [[] for _ in range(26)] ← Adjacency list: graph[u] = [(v, w), ...]
for o, c, w in zip(original, changed, cost):
u, v = ord(o)-ord('a'), ord(c)-ord('a')
graph[u].append((v, w))
def dijkstra(src): ← Standard Dijkstra from a single source
dist = [float('inf')] * 26
dist[src] = 0; heap = [(0, src)]
while heap:
d, u = heapq.heappop(heap)
if d > dist[u]: continue ← Stale entry, skip
for v, w in graph[u]:
if dist[u]+w < dist[v]:
dist[v] = dist[u]+w; heapq.heappush(heap, (dist[v], v))
return dist
needed = set(ord(s)-ord('a') for s,t in zip(source,target) if s!=t)← Only run Dijkstra from needed sources
all_dist = {src: dijkstra(src) for src in needed}
total = 0
for s, t in zip(source, target):
if s == t: continue
c = all_dist[ord(s)-ord('a')][ord(t)-ord('a')]
if c == float('inf'): return -1
total += c
return total
💡 Complexity Summary

Floyd-Warshall: Time O(26³ + n) ≈ O(n), Space O(26²) = O(1) — 26 is constant
Dijkstra × 26: Time O(26 × (E + 26 log 26) + n), Space O(26 + E) — similar in practice
Brute Force: Explore all paths per character pair — exponential, infeasible

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Recognize the Graph Structure

"I see character conversions with costs — this is a weighted directed graph. Each of the 26 letters is a node, and each (original, changed, cost) triple is a directed edge. The key question for each position is: what is the shortest path from source[i] to target[i] in this graph?"

Step 2: Identify All-Pairs Shortest Path Need

"Since different positions may need different source→target pairs, I could run Dijkstra once per unique source character. But since there are at most 26 source characters, I can precompute ALL shortest paths at once. Floyd-Warshall does this in O(V³) — and V = 26 is constant!"

Step 3: Explain the Three-Loop Pattern

"Floyd-Warshall has three nested loops over all nodes. The outer loop picks an intermediate node k. For every pair (i, j), we ask: is going through k cheaper? If dist[i][k] + dist[k][j] < dist[i][j], we update. After all 26 outer iterations, we have the optimal path between every pair."

Step 4: Handling Impossible Cases

"If after Floyd-Warshall, dist[s][t] is still infinity, it means there's no sequence of conversions to get from s to t. Return -1 immediately when we encounter such a position."

🗣️ What to Say to the Interviewer

"Each character conversion is an edge in a 26-node weighted directed graph.
I need the cheapest path from source[i] to target[i] for every position.
Since there are only 26 possible characters, I can precompute ALL-PAIRS shortest paths.
Floyd-Warshall: initialize dist[i][i]=0, direct edges from input, then three nested loops.
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]) for each intermediate k.
After 26 outer iterations, look up each position's cost. Sum them. If any is inf → -1."
🔮 Likely Follow-up Questions
  • "What if strings can have any Unicode characters?" → Use a dict as dist instead of a 26×26 array; run Dijkstra per unique source.
  • "What if conversions have negative costs?" → Floyd-Warshall still works for negative edges (unlike Dijkstra), as long as no negative cycles exist.
  • "Minimum Cost to Convert String II?" → Same idea but with substring replacements — requires Aho-Corasick + DP.
  • "How would you detect an unreachable conversion early?" → Check dist[s][t] before summing; return -1 immediately on first infinity found.

7 Summary & Key Takeaways ▾
Model character conversions as a weighted directed graph — 26 nodes for a–z
Floyd-Warshall precomputes ALL-PAIRS shortest paths in O(26³) ≈ O(1) time
After precomputation, each position in the string is answered in O(1) via table lookup
Duplicate edges (same pair, multiple costs): always keep the minimum
If dist[s][t] == ∞ for any position, the answer is -1 — conversion impossible
Key trick: small fixed alphabet (26) makes a cubic algorithm effectively constant
ApproachTimeSpaceNotes
Brute force (BFS per position) O(n × 26!) O(26) Explore all paths — infeasible
Dijkstra × 26 sources O(26(E + 26 log26) + n) O(26 + E) Valid but more complex to code
Floyd-Warshall ✓ O(26³ + n) ≈ O(n) O(26²) = O(1) Simplest code; fixed alphabet is key
🎯 When to Recognize Floyd-Warshall
  • Need shortest paths between all pairs of a small set of nodes
  • Node count is tiny (≤ a few hundred) — cubic is acceptable
  • "Can I go from X to Y?" for many different X, Y pairs
  • Weighted directed graph with potentially multiple paths between pairs
