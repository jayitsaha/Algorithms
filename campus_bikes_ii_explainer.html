LeetCode #1066

Campus Bikes II

Assign each worker a unique bike to minimize total Manhattan distance. Constraints: W ≤ 4 workers, B ≤ 10 bikes — small enough for bitmask DP.

Medium Dynamic Programming Bitmask DP Time: O(W · 2^B · B) Space: O(2^B)
1 Understanding the Problem ▾

You have a 2D campus with W workers and B bikes (B ≥ W). Assign exactly one unique bike to each worker to minimize the total Manhattan distance summed across all assignments. Manhattan distance between points (x1,y1) and (x2,y2) is |x1−x2| + |y1−y2|.

💡 Key Insight: Use the Small Constraints

W ≤ 4 and B ≤ 10. That means at most 210 = 1024 possible subsets of bikes. We can represent "which bikes are already taken" as a single integer (a bitmask). This unlocks memoization — reuse work when the same bike-subset is reachable via different paths.

✗ Naive: Try All Permutations

P(10,4) = 5040 permutations
Each scanned separately
No shared subproblem reuse
Gets worse exponentially

✓ Bitmask DP: Subset Memoization

Only 2^10 = 1024 mask states
dp[mask] = min dist for that subset
popcount(mask) → which worker next
Total ~40K operations

Critical observation: If a mask has k bits set (k bikes assigned), then workers 0..k−1 have each received one of those bikes in order. Worker k is always assigned next. This means the mask alone encodes both "which bikes are taken" AND "which worker is next" — no extra dimension needed!

Edge Cases: W = B (every bike must be used, no slack), all workers at same position (still need DP — greedy fails), single worker (just find closest bike).

2 The Algorithm: Bottom-Up Bitmask DP ▾
📐 The Recurrence Relation
dp[mask] = min total distance when the bikes in mask
            are assigned to workers 0 .. popcount(mask)−1

worker_k = popcount(mask) ← implicit from the mask

For each bike j NOT in mask:
  new_mask = mask | (1 << j)
  dp[new_mask] = min(dp[new_mask], dp[mask] + dist(workers[k], bikes[j]))

Base: dp[0] = 0  |  Answer: min dp[mask] where popcount(mask) == W
1 Initialize: dp = [∞] × 2B, dp[0] = 0. Zero bikes assigned to zero workers costs nothing.
2 Iterate masks in order 0 to 2B−1: Because new_mask > mask (setting bits only increases value), processing in increasing order guarantees sub-states are ready.
3 Derive worker index k = popcount(mask). If k == W, record ans = min(ans, dp[mask]) and continue. If k > W or dp[mask] == ∞, skip.
4 For each unset bit j in mask (unassigned bike), compute dist(workers[k], bikes[j]) and relax: dp[mask | (1<<j)] = min(dp[mask | (1<<j)], dp[mask] + dist).
5 Return ans. It holds the minimum over all full-assignment masks (those with exactly W bits set).
🔑 Why 1D DP? Why Not dp[worker][mask]?

Since workers are assigned in fixed order (0 then 1 then 2…), the worker index is always exactly popcount(mask). The mask alone determines which worker is next. This eliminates the worker dimension entirely, halving state space from W × 2B to just 2B entries.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing all masks up to m, dp[m] holds the minimum total distance to assign the bikes in m to workers 0..popcount(m)−1, over all possible assignment orderings that result in exactly those bikes being chosen. This is a provably optimal partial assignment.

Optimal Substructure: The best assignment of k workers using some subset of k bikes decomposes as: best assignment of k−1 workers using k−1 of those bikes, plus the marginal cost of the best bike choice for worker k−1. Future assignments cannot change already-incurred costs.

Why increasing mask order is correct: When we update dp[new_mask] from dp[mask], new_mask = mask | (1 << j) > mask (since we only set bits). Processing masks from 0 upward guarantees dp[mask] is finalized before any mask that depends on it is processed — analogous to topological order in DAGs.

Overlapping Subproblems: Many assignment sequences produce the same mask. Example: assigning bike 0 first then bike 2, vs bike 2 first then bike 0, both produce mask 101. The DP collapses all these into one stored value — the minimum cost over all sequences producing that mask.

⚠️ Why Greedy Fails Here

Greedy "assign each worker to the nearest available bike" doesn't minimize the total. Consider workers at [0,0] and [1,0], bikes at [1,0] and [0,1]. Greedy assigns W0 → B0 (dist=1) leaving W1 → B1 (dist=2), total=3. But optimal is W0 → B1 (dist=1) and W1 → B0 (dist=0), total=1. Greedy missed it. DP explores all possibilities.

4 The Code (Annotated) ▾

Solution 1 — Bitmask DP Bottom-Up (Optimal, Interview Pick)

def assignBikes(workers, bikes): ← workers/bikes are [[x,y]] lists
W, B = len(workers), len(bikes) ← W ≤ 4, B ≤ 10 per constraints
INF = float('inf') ← sentinel: "not yet reachable"
dp = [INF] * (1 << B) ← 2^B entries, one per bike subset
dp[0] = 0 ← base: empty assignment = 0 cost
ans = INF ← will hold answer over all full masks
for mask in range(1 << B): ← iterate all 2^B subsets in order
if dp[mask] == INF: continue ← skip unreachable states
k = bin(mask).count('1') ← popcount = next worker index
if k == W: ← all workers assigned!
ans = min(ans, dp[mask]); continue ← record this complete assignment
wx, wy = workers[k] ← coordinates of worker k
for j in range(B): ← try each bike
if mask & (1 << j): continue ← skip: bike j already taken
d = abs(wx-bikes[j][0]) + abs(wy-bikes[j][1])← Manhattan dist
nm = mask | (1 << j) ← new mask: mark bike j as taken
dp[nm] = min(dp[nm], dp[mask] + d) ← relax if cheaper
return ans ← minimum over all complete assignments
⚠️ Common Error: Collecting Answer Wrong

A common mistake is to return min(dp[(1<<B)-1:]) or similar. The correct approach: collect ans at every mask with exactly W bits set — not just masks with B bits. Using all B bits forces every bike to be used, but we only need W of them (W ≤ B).

Solution 2 — Top-Down Memoization (More Intuitive to Derive)

from functools import lru_cache
def assignBikes(workers, bikes):
W, B = len(workers), len(bikes)
@lru_cache(maxsize=None) ← memoize (k, mask) pairs
def dp(k, mask): ← k = current worker, mask = taken bikes
if k == W: return 0 ← base: no more workers to assign
wx, wy = workers[k]
best = float('inf')
for j in range(B):
if mask & (1 << j): continue ← skip taken bikes
d = abs(wx-bikes[j][0]) + abs(wy-bikes[j][1])
best = min(best, d + dp(k+1, mask|(1<<j))) ← try this bike, recurse
return best
return dp(0, 0) ← start: worker 0, no bikes taken
💡 Complexity Summary

States: 2B masks × W workers = W × 2B unique states (but k is implicit in bottom-up)
Transitions per state: O(B) bike choices
Total Time: O(W × 2B × B)
Space: O(2B) for bottom-up, O(W × 2B) for top-down cache
With W=4, B=10: 4 × 1024 × 10 = 40,960 operations — negligible

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Notice the Small Constraints

W ≤ 4, B ≤ 10 — always read the constraints first. 210 = 1024 is tiny. "Small constraint + subset tracking" is the universal bitmask DP signal. Say: "The small value of B suggests we can enumerate bike subsets with a bitmask."

Step 2: Define the State

"What information do I need at each step?" — which bikes are taken. That's a bitmask. Then ask: "Can I derive which worker to assign next from the mask?" Yes — popcount(mask) gives the count of assigned workers, so worker_k = popcount(mask). State = just the mask.

Step 3: Write the Recurrence in Plain English

"To get dp[new_mask], I must have come from dp[mask] where new_mask = mask with one more bit set. The cost delta is the distance for that new assignment." This maps directly to code.

Step 4: Mention Both Approaches

Say: "I can implement top-down with memoization — cleaner to derive. Or bottom-up by iterating masks in order, which is slightly more cache-efficient. Both are O(W × 2B × B)." Implement top-down first if unsure.

🗣️ What to Say to the Interviewer

"I notice W ≤ 4 and B ≤ 10 — small enough to use bitmask DP.
I'll represent which bikes are assigned as an integer mask (bit j = 1 means bike j is taken).
Key insight: popcount(mask) equals the number of workers already assigned,
so the next worker to assign is always worker k = popcount(mask).
dp[mask] = min total distance to assign those bikes to the first k workers.
For each mask, I try assigning each unused bike to worker k and take the minimum.
Iterating masks from 0 to 2^B-1 in order ensures correctness — smaller masks are always ready.
Time: O(W × 2^B × B) ≈ 40K ops. Space: O(2^B) = 1024 entries."
🔮 Likely Follow-up Questions
  • "What if W or B were large (e.g., 1000)?" → Hungarian algorithm O(n³) or min-cost bipartite matching. Bitmask DP is infeasible.
  • "How would you handle maximizing instead of minimizing?" → Initialize dp with −∞, use max() instead of min(). Same structure.
  • "Can you do it with Dijkstra?" → Yes! State = (worker_idx, mask), edge weight = distance. BFS/Dijkstra works but bitmask DP is simpler.
  • "What if a worker can take 0 or multiple bikes?" → State structure changes; the popcount trick breaks.

7 Summary & Key Takeaways ▾
Bitmask DP is triggered by: "small set (≤20), track which items are used, minimize/maximize total"
popcount(mask) implicitly encodes the next worker index — no 2D array needed
Process masks in increasing order — new_mask > mask guarantees sub-states are ready
mask | (1 << j) sets bit j (mark bike j taken); mask & (1 << j) checks bit j
Answer = min dp[mask] over all masks with exactly W bits set — NOT all B bits
ApproachTimeSpaceNotes
Brute Force (all permutations) O(B! / (B−W)!) O(1) No memoization; exponentially worse
Bitmask DP bottom-up ✓ O(W · 2B · B) O(2B) Optimal; ~40K ops for W=4,B=10
Top-Down Memoization O(W · 2B · B) O(W · 2B) Cleaner to derive; higher constant
🎯 When to Recognize Bitmask DP
  • "Assign each A to a unique B" with small A/B counts → bitmask DP
  • "Visit all nodes/elements exactly once" → bitmask DP (TSP family)
  • Constraints say ≤ 15–20 items → subset enumeration is feasible
  • Need to track "which items have been used" across recursive calls
  • Keywords: "minimum total cost", "each item used at most once", "small n"
