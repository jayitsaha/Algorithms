LeetCode #1233

Remove Sub-Folders from the Filesystem

Given a list of folders in a filesystem, remove all sub-folders and return only the top-level folders.

Medium Tries Sort + Skip Time: O(n·L·log n) Space: O(n·L)
1 Understanding the Problem ▾

You receive a list of absolute Unix-style folder paths like ["/a", "/a/b", "/c/d", "/c/d/e", "/c/f"]. Folder B is a sub-folder of A if accessing B requires going through A first — mathematically, if B starts with A + "/". Your job is to remove all such sub-folders and return only the top-level ones.

💡 Key Insight

If we sort folders lexicographically, every sub-folder appears directly after its parent. We only need to remember the last kept folder and check: does the current folder start with last_kept + "/"? If yes — skip. If no — keep and update. One pass, one comparison.

✓ IS a Sub-Folder — REMOVE

A = /a
B = /a/b
B.startswith("/a/") = True
→ Remove /a/b

✗ NOT a Sub-Folder — KEEP

A = /a
B = /ab
B.startswith("/a/") = False
→ Keep /ab

⚠️ The "/" Separator Trap

Simply checking B.startswith(A) is WRONG. /ab starts with /a but is NOT a sub-folder. You MUST check B.startswith(A + "/"). The slash marks a path component boundary — without it, you'll falsely eliminate sibling folders.

Edge cases: single folder (always kept), all folders nested under one root (only root kept), folders with near-identical prefixes like /a and /ab, deeply nested paths like /a/b/c/d/e.

2 The Algorithm: Sort + Single-Pass Skip ▾

The elegant insight: lexicographic sort creates a natural ordering where every sub-folder follows its parent. We exploit this with a single linear scan, comparing each folder against only the most recently kept one.

1 Sort lexicographically. This clusters each parent with all its descendants immediately after. In lex order, "/" (ASCII 47) comes before any letter, so "/a/b" sorts between "/a" and "/ab" — exactly what we need.
2 Keep the first folder unconditionally. The smallest folder in sorted order can never be a sub-folder of anything — nothing precedes it. Initialize result = [folder[0]].
3 Scan remaining folders. For each folder f, let last_kept = result[-1]. Check: does f start with last_kept + "/"? If yes → sub-folder → skip. If no → new top-level → result.append(f).
4 Return result. Because we always compare against the last kept folder (not all previous ones), and because sub-folders sort after their ancestor, a single comparison suffices for any nesting depth.
🔑 Why Only Checking last_kept is Sufficient

After sorting: if B is a sub-folder of A, A appears before B. The last kept folder is the closest ancestor in sorted order. If the current folder is a sub-folder of any previously kept folder, it must start with last_kept + "/" — because last_kept is the most recent non-sub folder in sorted order, which is the closest possible ancestor. Any skipped folders between them are deeper sub-folders and do not change this logic.

💡 Trie Approach (Alternative)

A Trie can also solve this: split each path by "/", insert component by component, mark end-of-path nodes. A folder is a sub-folder if its insertion path passes through a marked node. The Trie runs in O(n·L) time but requires more complex implementation. Sort + Skip is preferred in interviews.

3 Why It Works: The Invariant ▾
🔐 The Invariant

After processing each folder: result contains exactly the top-level (non-sub) folders seen so far in sorted order, and result[-1] is the lexicographically largest non-sub folder seen. Every skipped folder is a proven sub-folder of some folder already in result.

Why lexicographic order is critical: The ASCII value of "/" is 47, which is less than 48 ('0') and any letter. This means in lexicographic comparison, a path component separator "/" sorts before any character that could continue a name. So "/a/b" < "/a0" < "/ab". A parent path A will always sort before any A/child path, but NOT necessarily before all sibling paths sharing the same prefix letters.

Concretely, consider "/a" vs "/ab": In sorted order, "/a" comes first. When we reach "/ab", we check "/ab".startswith("/a/"). This is False — "/ab" does not have a "/" at position 2. So we correctly keep "/ab" as a separate top-level folder. The slash separator acts as a definitive boundary between a folder name and its sub-folders.

Deep nesting works automatically: If "/a", "/a/b", and "/a/b/c" are all in the input, after sorting: we keep "/a" (last_kept = "/a"), skip "/a/b" (starts with "/a/"), skip "/a/b/c" (also starts with "/a/"). We never need to check against "/a/b" because "/a" already eliminates all descendants.

⚠️ The Classic Bug: Checking startswith(A) Instead of startswith(A + "/")

With input ["/a", "/ab"], sorted as ["/a", "/ab"]: "/ab".startswith("/a") is True — this would incorrectly remove "/ab"! But "/ab".startswith("/a/") is False — correctly keeps "/ab". Never forget the slash.

4 The Code (Annotated) ▾

Solution 1 — Sort + Skip (Interview Pick)

def removeSubfolders(folder: List[str]) -> List[str]:
folder.sort() ← Lex sort: parents always precede their sub-folders
result = [folder[0]] ← First folder can never be a sub-folder; always keep it
for f in folder[1:]: ← Scan every remaining folder left to right
last_kept = result[-1] ← Most recently kept top-level folder
if not f.startswith(last_kept + "/"): ← "/" ensures full component match, not just prefix
result.append(f) ← Not a sub-folder → keep it; it becomes the new last_kept
return result ← All non-sub-folder paths
⚠️ Common Interview Mistakes to Avoid
  • Using startswith(last_kept) without the "/" → falsely removes "/ab" when last_kept="/a"
  • Forgetting to sort first → the single-comparison trick only works on sorted input
  • Using a nested loop to check all previous folders → unnecessary; last_kept is enough

Solution 2 — Trie Approach (O(n·L) time)

def removeSubfolders_trie(folder: List[str]) -> List[str]:
trie = {} ← Root of the trie (nested dict)
for path in folder: ← Insert every path into the trie
parts = path.split("/")[1:] ← Split "/a/b" → ["a","b"]; [1:] skips empty string before first "/"
node = trie ← Start at trie root
for p in parts: ← Descend through path components
node = node.setdefault(p, {})← Create child node if missing
node["#"] = True ← Sentinel: this node marks a complete path
result = []
def dfs(node, path): ← DFS through trie to collect top-level folders
if "#" in node: ← Found a complete path → it's a top-level folder
result.append(path); return ← Add to result; DO NOT go deeper (children are sub-folders)
for k, child in node.items(): ← No marker here; keep descending
dfs(child, path + "/" + k)
for k, child in trie.items(): ← Start DFS from each root component
dfs(child, "/" + k)
return result
💡 Complexity Summary

Sort + Skip: Time O(n·L·log n) — sort dominates; each comparison is O(L). Space O(n·L) for output.
Trie: Time O(n·L) build + O(n·L) DFS. Space O(n·L) for the trie nodes.
In practice, Sort + Skip is faster due to cache efficiency and simplicity; Trie is theoretically optimal asymptotically.

5 🎤 Interview Strategy: How to Arrive at This ▾
Step 1: Identify the Core Challenge

"For each folder, I need to check if any other folder is its ancestor. Naively O(n²·L) — check all pairs. I need a smarter structure."

Step 2: Think About What Structure Helps

"A Trie stores paths efficiently and can detect ancestor relationships. But sorting also works — sorted paths group parents with their children immediately after. After sorting, I only need to compare against the last kept folder."

Step 3: Nail the Sub-Folder Check

"f.startswith(last_kept + '/') — I must include the '/' to distinguish '/a/b' (sub-folder of '/a') from '/ab' (not a sub-folder). This is the trickiest part and what interviewers test."

Step 4: Mention Trie as the Alternative

"The Trie solution is O(n·L) — slightly better asymptotically but more code to write under pressure. Sort + Skip is O(n·L·log n) but cleaner. I'll go with Sort + Skip unless you want the Trie implementation."

🗣️ What to Say to the Interviewer

"Let me think about the naive approach first: for each folder, check if any other folder is its ancestor — that's O(n²·L). Can I do better?
If I sort the folders lexicographically, any sub-folder of X will appear directly after X in sorted order. So I can do a single pass: keep track of the last kept folder, and skip any folder that starts with last_kept + '/'. The '/' is critical — without it, '/ab' would be falsely removed when last_kept is '/a'.
Time: O(n·L·log n) for the sort. Space: O(n) for the result. I can also implement this with a Trie in O(n·L) if you prefer."
🔮 Likely Follow-up Questions
  • "Can you implement the Trie solution?" → Split by "/", insert component-by-component, mark endpoints, DFS pruning at marked nodes
  • "What if paths can have trailing slashes?" → Strip them first: f.rstrip("/") in preprocessing
  • "Find all sub-folders of a given folder X?" → Binary search for X + "/" in sorted array, then collect all that start with it
